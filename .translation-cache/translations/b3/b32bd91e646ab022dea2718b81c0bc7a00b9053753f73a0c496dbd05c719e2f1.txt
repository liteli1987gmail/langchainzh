LangSmith SDK profile 允许你将 [API 密钥](/langsmith/create-account-api-key)、端点和工作区 ID 保存在可复用的 JSON 文件中，而无需在每个 shell 会话中重复设置相同的环境变量。

当你频繁切换 [LangSmith Cloud 区域](/langsmith/cloud#regional-storage)、自托管实例或[工作区](/langsmith/administration-overview#workspaces)时，或者希望远程运行时从挂载的文件加载相同的 SDK 配置时，可以使用 profile。

<Warning>
Profile 文件可能包含 API 密钥和 OAuth 刷新令牌。请勿将其提交到源代码管理、嵌入到容器镜像中或在日志中打印。请像对待其他凭证一样谨慎地存储和挂载它们。
</Warning>

## 最低版本

Profile 支持在以下版本中可用：

| 工具或 SDK | 最低版本 |
| --- | --- |
| LangSmith CLI profile 命令 | `v0.2.26` |
| `langsmith auth login` | `v0.2.30` |
| Go SDK | `v0.7.0` |
| Python SDK | `v0.8.1` |
| TypeScript SDK | `v0.6.1` |
| Java SDK | `v0.1.0-beta.3` |

## Profile 文件位置

默认情况下，[SDK](/langsmith/reference) 会在以下位置查找 profile 文件：

```text
~/.langsmith/config.json
```

要使用不同的路径，请设置 `LANGSMITH_CONFIG_FILE`：

```shell
export LANGSMITH_CONFIG_FILE=/path/to/langsmith-config.json
```

TypeScript SDK 仅在 Node.js 类运行时中加载配置文件。浏览器和 Web Worker 运行时无法访问文件系统，因此请在这些环境中显式传递配置。

## 创建配置文件

创建 `~/.langsmith/config.json` 文件，其中包含 `profiles` 对象。每个配置文件可以定义以下字段：

| 字段 | 说明 |
| --- | --- |
| `api_url` | LangSmith API 端点。使用与 `LANGSMITH_ENDPOINT` 相同的值。 |
| `api_key` | LangSmith API 密钥。请参阅[创建账户和 API 密钥](/langsmith/create-account-api-key)。 |
| `workspace_id` | 工作区 ID。当 API 密钥可以访问多个工作区时需要此字段。 |
| `oauth` | LangSmith 工具创建的 OAuth 令牌元数据。 |

```json
{
  "current_profile": "dev",
  "profiles": {
    "dev": {
      "api_url": "https://api.smith.langchain.com",
      "api_key": "<LANGSMITH_API_KEY>",
      "workspace_id": "<WORKSPACE_ID>"
    },
    "eu": {
      "api_url": "https://eu.api.smith.langchain.com",
      "api_key": "<EU_LANGSMITH_API_KEY>",
      "workspace_id": "<EU_WORKSPACE_ID>"
    },
    "apac": {
      "api_url": "https://apac.api.smith.langchain.com",
      "api_key": "<APAC_LANGSMITH_API_KEY>",
      "workspace_id": "<APAC_WORKSPACE_ID>"
    }
  }
}
```

限制文件权限，以便只有您可以读取它：

```shell
chmod 600 ~/.langsmith/config.json
```

## 选择配置文件

SDK 按以下顺序选择配置文件：

1. `LANGSMITH_PROFILE`（如果已设置）。
2. 配置文件中的 `current_profile`（如果已设置）。
3. 名为 `default` 的配置文件（如果存在）。

例如：

```shell
export LANGSMITH_PROFILE=eu
```

此 [LangSmith CLI](/langsmith/langsmith-cli) 还接受一个全局 `--profile` 标志，其优先级高于该命令的 `LANGSMITH_PROFILE`：

```shell
langsmith --profile eu project list
```

## 使用 CLI 管理配置文件

使用 [LangSmith CLI](/langsmith/langsmith-cli) 创建、检查、切换和删除配置文件，无需手动编辑 JSON 文件。

要从现有 API 密钥创建 API 密钥配置文件：

```shell
export LANGSMITH_API_KEY=<LANGSMITH_API_KEY>
langsmith profile create dev \
  --workspace-id <WORKSPACE_ID> \
  --set-current
```

您也可以将密钥和端点作为标志传递。在共享机器上，请优先使用环境变量，因为命令行历史记录可能会记录命令标志。

```shell
langsmith profile create eu \
  --api-key <EU_LANGSMITH_API_KEY> \
  --api-url https://eu.api.smith.langchain.com \
  --workspace-id <EU_WORKSPACE_ID>
```

常用 profile 命令：

| Command | Description |
| --- | --- |
| `langsmith profile list` | 列出已保存的 profile。别名：`langsmith profile ls`。 |
| `langsmith profile show <name>` | 显示已保存的 profile。输出中会对敏感值进行脱敏处理。 |
| `langsmith profile use <name>` | 在 profile 文件中设置 `current_profile`。 |
| `langsmith profile set-workspace <workspace-id>` | 为选定的 profile 设置默认工作空间。 |
| `langsmith profile delete <name>` | 删除已保存的 profile。 |

使用 `--format pretty` 可以输出便于阅读的表格格式：

```shell
langsmith --format pretty profile list
```

## 使用 `langsmith auth login` 进行身份验证

运行 `langsmith auth login` 使用 OAuth 进行身份验证，而无需手动创建 API 密钥配置文件。该命令启动基于浏览器的设备授权流程，将 OAuth 令牌存储在所选配置文件中，并将其设置为当前配置文件。

<Note>
`langsmith auth login` 目前仅支持 LangSmith Cloud（SaaS）。对于自托管或其他非 SaaS LangSmith 端点，请改用 API 密钥配置文件。
</Note>

```shell
langsmith auth login
```

使用 `--profile` 或 `LANGSMITH_PROFILE` 选择配置文件：

```shell
langsmith auth login --profile dev
```

对于无头环境，禁用自动打开浏览器，并传递一个工作区 ID：

```shell
langsmith auth login \
  --profile prod \
  --no-browser \
  --workspace-id <WORKSPACE_ID>
```

`langsmith auth login` 按以下顺序选择配置名称：

1. `--profile`（如果传入）。
2. `LANGSMITH_PROFILE`（如果已设置）。
3. 配置文件中的 `current_profile`（如果已设置）。
4. `default`。

按以下顺序选择 API URL：

1. `--api-url`（如果传入）。
2. `LANGSMITH_ENDPOINT`（如果已设置）。
3. 所选配置的 `api_url`（如果存在）。
4. 默认的 LangSmith Cloud 端点。

登录后，CLI 和 SDK 可以使用保存的配置。CLI 在需要时会刷新 OAuth 令牌，并将刷新的令牌字段写回配置文件。当未设置环境变量或构造函数 API 密钥身份验证时，SDK 也会使用配置中的 OAuth 访问令牌。

## 覆盖配置值

显式的客户端构造函数参数和环境变量优先于配置值。

| 设置 | 优先级 |
| --- | --- |
| 端点 | 构造函数 `api_url` 或 `apiUrl`，然后 `LANGSMITH_ENDPOINT`，然后配置 `api_url`，最后是默认的 LangSmith Cloud 端点。 |
| 身份验证 | 构造函数 API 密钥，然后 `LANGSMITH_API_KEY`，然后配置 OAuth 访问令牌，然后是配置 `api_key`。 |
| 工作区 | 构造函数 `workspace_id` 或 `workspaceId`，然后 `LANGSMITH_WORKSPACE_ID`，然后配置 `workspace_id`。 |

较旧的 `LANGCHAIN_API_KEY`、`LANGCHAIN_ENDPOINT` 和 `LANGCHAIN_WORKSPACE_ID` 别名仍然可用，但新配置建议优先使用 `LANGSMITH_*` 名称。

如果配置同时包含 `oauth.access_token` 和 `api_key`，SDK 会优先使用 OAuth 访问令牌。如果存在 OAuth 刷新令牌且访问令牌已过期或即将过期，SDK 可以刷新令牌并将更新的令牌字段写回配置文件。

<Note>
如果您将配置文件挂载为只读，则 OAuth 令牌刷新无法持久化更新的令牌。只读挂载适用于 API 密钥配置。仅在有意依赖 OAuth 令牌刷新时才使用可写挂载。
</Note>

## 在代码中使用配置

当配置文件存在时，正常创建客户端：

<CodeGroup>

```python Python
from langsmith import Client

client = Client()
```

```typescript TypeScript
import { Client } from "langsmith";

const client = new Client();
```

</CodeGroup>

要在代码中覆盖 profile，请显式传递值：

<CodeGroup>

```python Python
from langsmith import Client

client = Client(api_key="<LANGSMITH_API_KEY>")
```

```typescript TypeScript
import { Client } from "langsmith";

const client = new Client({ apiKey: "<LANGSMITH_API_KEY>" });
```

</CodeGroup>

## 在远程运行时中挂载配置文件

对于远程运行时，请将配置文件作为密钥文件挂载，并设置 `LANGSMITH_CONFIG_FILE` 指向挂载路径。请勿将文件复制到镜像或仓库中。

### Docker

将本地配置文件目录挂载到容器中：

```shell
docker run --rm \
  -e LANGSMITH_CONFIG_FILE=/home/app/.langsmith/config.json \
  -e LANGSMITH_PROFILE=prod \
  -v "$HOME/.langsmith:/home/app/.langsmith:ro" \
  my-image
```

仅在配置文件使用 OAuth 刷新令牌时使用读写挂载：

```shell
docker run --rm \
  -e LANGSMITH_CONFIG_FILE=/home/app/.langsmith/config.json \
  -e LANGSMITH_PROFILE=prod \
  -v "$HOME/.langsmith:/home/app/.langsmith" \
  my-image
```

### Kubernetes

根据配置文件创建 Kubernetes secret：

```shell
kubectl create secret generic langsmith-profile \
  --from-file=config.json="$HOME/.langsmith/config.json"
```

挂载密钥并让 SDK 指向它：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: app
spec:
  template:
    spec:
      containers:
        - name: app
          image: my-image
          env:
            - name: LANGSMITH_CONFIG_FILE
              value: /var/run/langsmith/config.json
            - name: LANGSMITH_PROFILE
              value: prod
          volumeMounts:
            - name: langsmith-profile
              mountPath: /var/run/langsmith
              readOnly: true
      volumes:
        - name: langsmith-profile
          secret:
            secretName: langsmith-profile
```

Kubernetes secret 卷是只读的。对于此模式，请使用 API 密钥配置文件，或者如果您的 OAuth 配置文件必须刷新并持久化令牌，请使用可写的 secret-sync 机制。

### 远程开发和 CI

在远程开发环境或 CI 作业中，将配置文件 JSON 存储在平台的密钥存储中，在运行时将其写入临时文件，并将 `LANGSMITH_CONFIG_FILE` 设置为该文件路径。

```shell
mkdir -p "$RUNNER_TEMP/langsmith"
printf '%s' "$LANGSMITH_PROFILE_JSON" > "$RUNNER_TEMP/langsmith/config.json"
chmod 600 "$RUNNER_TEMP/langsmith/config.json"
export LANGSMITH_CONFIG_FILE="$RUNNER_TEMP/langsmith/config.json"
export LANGSMITH_PROFILE=prod
```

对于托管的 LangSmith Cloud 部署，请将这些值配置为部署环境变量或工作区密钥，除非平台明确支持挂载密钥文件。
