title: 数据集预构建 JSON schema 类型

LangSmith 建议您在数据集 schema 的输入和输出上设置 schema，以确保数据一致性，并确保您的示例格式正确，适合下游处理（如运行评估）。

为了更好地支持 LLM 工作流，LangSmith 支持多种不同的预定义预构建类型。这些 schema 由 LangSmith API 公开托管，您可以使用 [JSON Schema 引用](https://json-schema.org/understanding-json-schema/structuring#dollarref) 在数据集 schema 中定义它们。可用 schema 的表格如下所示：

| 类型 | JSON Schema 引用链接 | 用途 |
| --- | --- | --- |
| Message | [https://api.smith.langchain.com/public/schemas/v1/message.json](https://api.smith.langchain.com/public/schemas/v1/message.json) | 表示发送给聊天模型的消息，采用 OpenAI 标准格式。 |
| Tool | [https://api.smith.langchain.com/public/schemas/v1/tooldef.json](https://api.smith.langchain.com/public/schemas/v1/tooldef.json) | 聊天模型可用于函数调用的工具定义，采用 OpenAI JSON Schema 风格的函数格式定义。 |

LangSmith 允许您定义一系列转换，从您的 traces 中收集上述预构建类型并将其添加到您的数据集中。有关可用转换的更多信息，请参阅我们的[参考文档](/langsmith/dataset-transformations)。
