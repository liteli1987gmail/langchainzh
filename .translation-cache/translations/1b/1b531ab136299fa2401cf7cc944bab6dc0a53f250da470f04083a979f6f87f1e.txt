沙箱是隔离环境，允许代理安全执行潜在危险操作（如运行任意代码或与文件系统交互），而不会影响您的主基础设施。

从 [LangSmith 主页](https://smith.langchain.com) 选择 **沙箱** 来管理您的所有沙箱资源。

![沙箱概览页面](/images/langsmith/sandboxes/sb-overview.png)

## 开始使用

### 1. 安装 SDK

<CodeGroup>


</fragment>

```bash Python
# uv
uv add "langsmith[sandbox]"

# pip
pip install "langsmith[sandbox]"
```

```bash TypeScript
npm install langsmith
```

</CodeGroup>

### 2. 设置您的 API 密钥

```bash
export LANGSMITH_API_KEY="<your-api-key>"
```

### 3. 创建并运行沙箱

<CodeGroup>

</fragment>

```python Python
from langsmith.sandbox import SandboxClient

client = SandboxClient()

with client.sandbox() as sb:
    result = sb.run("python -c 'print(2 + 2)'")
    print(result.stdout)  # "4\n"
```

```ts TypeScript
import { SandboxClient } from "langsmith/sandbox";

const client = new SandboxClient();
const sandbox = await client.createSandbox();
const result = await sandbox.run("node -e 'console.log(2 + 2)'");
console.log(result.stdout); // "4\n"
await sandbox.delete();
```

</CodeGroup>

<Tip>
  喜欢命令行？[Sandbox CLI](/langsmith/sandbox-cli) 可让您创建沙箱、运行命令和打开交互式 Shell，无需编写任何代码。
</Tip>

### 4. 将沙箱与您的代理配合使用

要将沙箱接入代理代码，请参阅开源文档：

- **深度代理**：将 [`LangSmithSandbox`](/oss/integrations/sandboxes/langsmith) 用作后端，涵盖安装、后端创建和清理。
- **沙箱作为代理后端**：将任意沙箱配置为[执行后端](/oss/deepagents/sandboxes)，自动为代理提供 `execute` 和文件系统工具。
- **LangChain / LangGraph 集成**：使用 LangSmith 沙箱作为首选方案，或[连接第三方提供商](/oss/integrations/sandboxes)，如 AgentCore、Daytona、E2B、Modal、Runloop 和 Vercel。
