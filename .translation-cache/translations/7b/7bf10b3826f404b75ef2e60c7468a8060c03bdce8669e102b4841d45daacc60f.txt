LangSmith 提供多种方式来查看您[组织](/langsmith/administration-overview)的使用量，具体取决于您的[计划](/langsmith/pricing-plans)和[部署类型](/langsmith/platform-setup)。本页介绍可用的数据、每个指标的含义，以及[自托管](/langsmith/self-hosted)部署的 limitations。

## 使用量视图

| 视图 | 在哪里找到 | 谁可以查看 | 计划可用性 |
|------|-----------------|----------------|-------------------|
| [使用量图表](#usage-graph) |**企业版**：设置 > 使用量 > 使用量图表<br></br>**自助服务**：设置 > 计费 > 使用量图表 | 所有组织成员 | 所有计划 |
| [详细使用量](#granular-usage) | **企业版**：设置 > 使用量 > 详细使用量<br></br>**自助服务**：设置 > 计费 > 详细使用量 | 所有组织成员 | 所有计划 |
| [合同使用量](#contract-usage) | **企业版**：设置 > 使用量 > 合同使用量<br></br>**自助服务**：设置 > 计费 > 合同使用量 | 仅组织管理员（`organization:manage`） | 仅企业版 |
| [发票](#invoices) | 设置 > 计费 > 发票 | 所有组织成员 | 仅自助云服务 |

## 使用量图表

使用量图表显示您组织的聚合 trace 消耗量，按工作空间细分。它涵盖当前计费周期，不显示支出——要查看支出，请参阅发票。

导航至 **设置** → **计费和使用量** → **使用量图表**。

### 计费指标

| 指标 | 计算内容 |
|--------|----------------|
| **LangSmith Traces（基础费用）** | 计费周期内发送到 LangSmith 的每条 trace，无论数据保留层如何。 |
| **LangSmith Traces（扩展数据保留升级）** | 升级到扩展保留的 trace（默认 400 天，[企业客户可自定义](/langsmith/data-purging-compliance#customize-extended-retention-policy)）。这些在基础费用的基础上额外计费。 |
| **LangSmith Deployment Runs** | 已部署 LangGraph 代理的端到端调用。定价详情请参阅 [LangSmith Deployment 计费](/langsmith/billing#langsmith-deployment-billing)。 |
| **LangSmith Fleet Runs** | [Fleet](/langsmith/fleet) 代理的端到端调用。按云托管和自托管部署分别追踪。 |
| **LangSmith Deployment Nodes Executed** | 已部署代理中的各个 LangGraph 节点执行。已部署代理图中每一步计为一个节点执行。按云托管和自托管部署分别追踪。 |

有关 trace 保留层的更多详情，请参阅[数据保留](/langsmith/usage-and-billing#data-retention)。

<Note>
使用量图表使用 `tenant_id` 作为工作空间 ID 的同义词。
</Note>

## 合同使用量

拥有预付承诺的企业客户可以查看其合同的消耗情况。

导航至 **设置** → **使用量配置** → **合同使用量**。

此视图显示：

- **使用量摘要**：总使用量、总积分和任何超额费用。当使用量超过总预付承诺时会产生超额费用。
- **承诺进度条**：每个承诺的视觉指示器，显示已消耗百分比以及已使用与剩余金额。合同可以有多个适用于不同时期（例如多年合同的第 1 年和第 2 年）或不同产品的承诺。
- **月度使用量图表**：柱状图，显示合同期内每个月的使用量。
- **产品费率**：您有权使用的产品及其定价表格。

<Note>
合同使用量需要 [`organization:manage` 权限](/langsmith/organization-workspace-operations)，仅适用于拥有预付承诺的企业客户。
</Note>

<Note>
如果您的合同涵盖同一计费实体下的多个组织，合同使用量视图显示所有这些组织的**合并**使用量，而不仅仅是您当前查看的组织。[详细使用量](#granular-usage)始终限定在单个组织内，因此要查看特定组织的详细使用量，请切换到该组织并单独查看其详细使用量。
</Note>

## 发票

发票仅在**自助云计划**可用。企业云组织有单独的使用量视图来追踪支出。

导航至 **设置** → **计费和使用量** → **发票**，了解您的使用量如何转化为支出。显示的第一张发票是当月发票的草稿，反映您迄今为止的运行支出。

## 详细使用量

详细使用量按您选择的维度（工作空间、项目、用户或 API 密钥）细分 trace 计数，并按您选择的时间范围显示。这对于内部成本分摊、识别高使用量团队或审计 trace 活动很有用。

导航至 **设置** → **计费和使用量** → **详细使用量**，或使用[详细使用量 API](/langsmith/granular-usage)。

### 这里"traces"的含义

详细使用量视图计算 **traces**：根级别运行及其所有子跨度计为一个单元。这与[计费](/langsmith/billing)使用的单位相同。它不单独计算单独的跨度、token 或模型调用。

### 使用量的记录时间

详细使用量按**插入时间**（trace 被 LangSmith 接收和存储的时间）记录 traces，而不是 trace 在您的应用程序中运行的时间。实际上这种差异通常可以忽略，但有明显延迟发送的 trace（例如缓冲的 SDK 上传）可能会出现在比预期更晚的时间桶中。

有关分组选项、时间桶大小和 API 参考，请参阅[详细计费使用量](/langsmith/granular-usage)。

## 自托管限制

与[云](/langsmith/cloud)相比，[自托管](/langsmith/self-hosted) LangSmith[部署](/langsmith/deployment)可用的使用量视图集不同，这是由于计费基础设施的差异。

| **功能** | **自托管可用性** |
|---------|--------------------------|
| 详细使用量（trace 归因） | 使用功能标志或版本 ≥ 0.13.12 可用 |
| 使用量图表（聚合 traces） | Helm chart 0.9.5 及更高版本可用 |
| 合同使用量 | 启用 Beacon phone-home 时可用 |
| 发票和付款管理 | 不可用（计费在 LangSmith 外部处理） |

### 自托管上的详细使用量

详细使用量在自托管部署上可用，但需要明确选择加入：

- 在 **LangSmith 0.13.12 及更高版本**上，默认启用详细使用量收集。
- 在**早期版本**上，通过设置以下两个环境变量来启用：

    ```env
    DEFAULT_ORG_FEATURE_ENABLE_GRANULAR_USAGE_REPORTING=true
    GRANULAR_USAGE_TABLE_ENABLED=true
    ```

<Warning>
数据收集从功能启用的那一刻开始。启用前不会回填历史使用数据。请在选择启用此功能的时机时相应做好规划。
</Warning>

### 自托管部署的聚合使用量

使用量图表适用于运行 Helm chart 0.9.5 或更高版本的自托管部署。LangSmith 会自动生成并同步组织使用量图表，可通过 **Settings** → **Usage and billing** → **Usage graph** 访问：

- **按工作区统计的使用量**：每个工作区的调用次数（根运行）
- **组织使用量**：整个组织的调用总数

图表每 5 分钟刷新一次以包含新工作区，且不可编辑。

如需以编程方式访问调用次数，请参阅[查看组织的调用次数](/langsmith/self-host-organization-charts)。
