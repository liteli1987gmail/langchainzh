LangSmith 为您提供多种查看您 [组织](/langsmith/administration-overview) 使用量的方式，具体取决于您的 [计划](/langsmith/pricing-plans) 和 [部署类型](/langsmith/platform-setup)。本页说明有哪些可用数据、每个指标的含义，以及哪些限制适用于 [self-hosted](/langsmith/self-hosted) deployments.

## 使用量视图

| 视图 | 在哪里找到 | 谁能查看 | 计划可用性 |
|------|-----------------|----------------|-------------------|
| [使用量图表](#usage-graph) |**企业版**：设置 > 使用量 > 使用量图表<br></br>**Self-serve**：设置 > 计费 > 使用量图表 | 所有组织成员 | 所有计划 |
| [详细使用量](#granular-usage) | **企业版**：设置 > 使用量 > 详细使用量<br></br>**Self-serve**：设置 > 计费 > 详细使用量 | 所有组织成员 | 所有计划 |
| [合同使用量](#contract-usage) | **企业版**：设置 > 使用量 > 合同使用量<br></br>**Self-serve**：设置 > 计费 > 合同使用量 | 仅组织管理员（`organization:manage`）| 仅企业版 |
| [发票](#invoices) | 设置 > 计费 > 发票 | 所有组织成员 | 仅自助云 |

## 使用量图表

使用量图表显示您组织的总体追踪消耗情况，按工作区细分。它涵盖当前计费周期，不显示支出——如需查看支出，请参阅发票。

导航至 **设置** → **计费和用量** → **使用量图表**.

### 计费指标

| 指标 | 计数内容 |
|--------|----------------|
| **LangSmith 追踪（基础费用）** | 计费期间发送至 LangSmith 的每条追踪，无论数据保留层如何。|
| **LangSmith 追踪（扩展数据保留升级）** | 升级至扩展保留的追踪（默认400天， [企业版客户可自定义](/langsmith/data-purging-compliance#customize-extended-retention-policy)）。这些在基础费用之外另行计费。 |
| **LangSmith 部署运行** | 已部署 LangGraph 代理的端到端调用。详见 [LangSmith 部署计费](/langsmith/billing#langsmith-deployment-billing) 了解定价详情。 |
| **LangSmith Fleet 运行** | [Fleet](/langsmith/fleet) 代理的端到端调用。云托管和自托管部署分别追踪。 |
| **LangSmith 部署节点执行数** | 已部署代理中各个 LangGraph 节点的执行次数。已部署代理图中的每一步计为一个节点执行。云托管和自托管部署分别追踪。 |

有关跟踪保留层的更多详细信息，请参阅 [数据保留](/langsmith/usage-and-billing#data-retention).

<Note>
使用量图表使用术语 `tenant_id` 与工作区 ID 互换使用。
</Note>

## 合同使用量

预付费承诺的企业客户可以查看其合同已消耗多少。

导航到 **设置** → **使用量配置** → **合同使用量**.

此视图显示：

- **使用量摘要**：总使用量、总积分和任何超额使用量。当使用量超过您的总预付费承诺时，会产生超额使用量。
- **承诺进度条**：每个承诺的可视化指示器，显示已消耗百分比以及已使用与剩余的美元金额。合同可以有多个承诺，适用于不同时间段（例如，多年合同的第一年和第二年）或不同产品。
- **月度使用量图表**：条形图，显示合同期内每个月的使用量。
- **产品费率**：您有权使用的产品及其定价的表格。

<Note>
合同使用量需要 [`organization:manage` 权限](/langsmith/organization-workspace-operations) ，仅适用于有预付费承诺的企业客户。
</Note>

<Note>
如果您的合同涵盖同一计费实体下的多个组织，合同使用量视图显示所有这些组织的 **合并** 使用量，而非您当前正在查看的组织的使用量。 [精细使用量](#granular-usage) 始终限定在单个组织范围内，因此要查看特定组织的精细使用量，请切换到该组织并单独查看其精细使用量。
</Note>

## 发票

发票仅在 **自助云计划**上可用。企业云组织有单独的使用量视图来跟踪支出。

导航到 **设置** → **账单与使用量** → **发票** 以查看您的使用量如何转化为支出。显示的第一张发票是当月发票的草稿，反映了您迄今为止的运行支出。

## 精细使用量

精细使用量为您提供按您选择的维度（工作区、项目、用户或 API 密钥）细分的跟踪数量，以及您选择的时间范围。这对于内部成本分摊、识别高使用量团队或审计跟踪活动非常有用。

导航到 **设置** → **账单与使用量** → **精细使用量**，或使用 [精细使用量 API](/langsmith/granular-usage).

### 此处 "跟踪" 的含义

精细使用量视图计算 **跟踪**: 根级运行及其所有子跨度被计为单个单位。这也是用于 [计费](/langsmith/billing)的同一单位。它不会单独计算各个跨度、令牌或模型调用。

### 记录使用量时

细粒度使用量按以下方式追踪记录 **插入时间** （追踪被接收并存储在 LangSmith 中的时间），而非在应用程序中运行的时间。在实践中，这种差异通常可以忽略不计，但有显著延迟发送的追踪（例如，缓冲的 SDK 上传）可能会出现在比预期更晚的时间桶中。

有关分组选项、时间桶大小和 API 参考，请参阅 [细粒度计费使用量](/langsmith/granular-usage).

## 自托管限制

[Self-hosted](/langsmith/self-hosted) LangSmith [部署](/langsmith/deployment) 由于计费基础设施的差异，与 [云端](/langsmith/cloud)相比，可用的一组使用量视图有所不同。

| **功能** | **自托管可用性** |
|---------|--------------------------|
| 细粒度使用量（追踪归因）| 通过功能标志或版本 ≥ 0.13.12 可用 |
| 使用量图表（聚合追踪）| Helm chart 0.9.5 及更高版本可用 |
| 合同使用量 | 启用 Beacon phone-home 时可用 |
| 发票和付款管理 | 不可用（计费在 LangSmith 外部处理）|

### 自托管上的细粒度使用量

细粒度使用量在自托管部署上可用，但需要明确选择加入：

- On **LangSmith 0.13.12 及更高版本**，细粒度使用量收集默认启用。
- On **早期版本**，请通过设置以下两个环境变量来启用它：

    ```env
    DEFAULT_ORG_FEATURE_ENABLE_GRANULAR_USAGE_REPORTING=true
    GRANULAR_USAGE_TABLE_ENABLED=true
    ```

<Warning>
数据收集从启用该功能的那一刻开始。启用之前没有历史使用量数据的回填。在选择何时启用此功能时请相应规划。
</Warning>

### 自托管上的聚合使用量

使用量图表在运行 Helm chart 0.9.5 或更高版本的自托管部署上可用。LangSmith 自动生成并同步组织使用量图表，可在 **设置** → **使用量和计费** → **使用量图表**:

- **按工作区划分的使用量**：每个工作区的追踪计数（根运行）
- **组织使用量**：整个组织的总追踪计数

图表每 5 分钟刷新一次以包含新的工作区，且不可编辑。

有关以编程方式访问追踪计数，请参阅 [查看组织中的追踪计数](/langsmith/self-host-organization-charts).
