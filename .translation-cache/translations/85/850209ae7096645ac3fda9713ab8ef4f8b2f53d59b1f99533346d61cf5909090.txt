---
title: MULTIPLE_SUBGRAPHS
---

当您 [在节点内调用子图](/oss/langgraph/use-subgraphs#call-a-subgraph-inside-a-node) 多次，且子图使用 `checkpointer=True` （continuations 模式）编译。

## 故障排除

根据您的需求选择以下选项之一：

:::python
1. **不需要中断？** 使用 `checkpointer=False` 完全禁用检查点：
   ```python
   subgraph = subgraph_builder.compile(checkpointer=False)
   ```

2. **需要中断但不需要跨调用持久化？** 通过省略来使用默认的继承模式 `checkpointer`:
   ```python
   subgraph = subgraph_builder.compile()
   ```
   每次调用都会获得一个唯一的命名空间，因此并行执行可以正常工作。子图每次都会重新启动，但可以使用 `interrupt()`.

3. **需要跨调用持久化？** 使用 `checkpointer=True`。LangGraph 为每次调用分配一个基于位置的命名空间后缀（`calling_node`, `calling_node|1`等）以防止冲突。对于稳定的基于名称的命名空间，请使用唯一的节点名称包装每个子图——请参阅 [并行子图](/oss/langgraph/use-subgraphs#subgraph-persistence).
:::

:::js
1. **不需要中断？** 使用 `checkpointer: false` 完全禁用检查点：
   ```typescript
   const subgraph = subgraphBuilder.compile({ checkpointer: false });
   ```

2. **需要中断但不需要跨调用持久化？** 通过省略来使用默认的继承模式 `checkpointer`:
   ```typescript
   const subgraph = subgraphBuilder.compile();
   ```
   每次调用都会获得一个唯一的命名空间，因此并行执行可以正常工作。子图每次都会重新启动，但可以使用 `interrupt()`.

3. **需要跨调用持久化？** 使用 `checkpointer: true`。LangGraph 为每次调用分配一个基于位置的命名空间后缀（`calling_node`, `calling_node|1`等）以防止冲突。对于稳定的基于名称的命名空间，请使用唯一的节点名称包装每个子图——请参阅 [并行子图](/oss/langgraph/use-subgraphs#subgraph-persistence).
:::
