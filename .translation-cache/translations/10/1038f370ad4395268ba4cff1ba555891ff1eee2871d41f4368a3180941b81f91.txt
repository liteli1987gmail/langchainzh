自托管 LangSmith 部署在您运维的基础设施内运行 [Agent Server](/langsmith/agent-server)、控制平面、数据平面和支持数据库。

LangChain 提供容器镜像、Helm charts 和许可证；您提供 Kubernetes 集群（或 Docker 主机）、PostgreSQL、Redis 以及适合您环境的网络和可观测性工具。当您有数据主权、监管约束、自定义网络或气隙环境需求时，自托管是最佳部署选项。

<Callout icon="clipboard-check" color="#4F46E5" iconType="regular">
自托管部署需要 [Enterprise 计划](https://www.langchain.com/pricing) 和随计划提供的 LangSmith 许可证密钥。有关设置指南，请参阅 [自托管 LangSmith](/langsmith/self-hosted)。
</Callout>

## 拓扑结构

LangSmith 支持三种自托管拓扑结构，在设置复杂性和控制平面功能之间进行权衡。[平台功能](/langsmith/self-hosted-platform-features) 和 [诊断](/langsmith/diagnostics-self-hosted) 的参考页面适用于所有三种拓扑结构。

<CardGroup cols={2}>

<Card title="完整自托管平台" icon="buildings" href="/langsmith/deploy-with-control-plane">
完整的 LangSmith 平台——[控制平面](/langsmith/control-plane) UI 和 API、[数据平面](/langsmith/data-plane) 监听器、可观测性、评估和 Agent 部署管理。最适合需要在自有网络内获得完整 LangSmith 产品体验的团队。
</Card>

<Card title="混合部署" icon="cloud-network" href="/langsmith/hybrid">
LangChain 托管的控制平面，数据平面（Agent Servers 和数据库）在您的基础设施内。最适合需要托管部署工作流但希望 Agent 工作负载和客户数据留在您 VPC 内的场景。
</Card>

<Card title="独立服务器" icon="server" href="/langsmith/deploy-standalone-server">
最轻量的选择——Agent Server 容器（API + 队列 worker）配合您自己的 PostgreSQL 和 Redis。无控制平面，无托管 UI。最适合将运行时嵌入现有基础设施或运行气隙环境。
</Card>

<Card title="平台功能" icon="list-check" href="/langsmith/self-hosted-platform-features">
自托管专用平台行为的参考：自定义 Postgres 和 Redis、监听器和资源定制。
</Card>

<Card title="诊断" icon="stethoscope" href="/langsmith/diagnostics-self-hosted">
收集日志、检查状态并排查自托管安装问题。
</Card>

</CardGroup>

## 谁来管理什么

自托管将基础设施运营的所有权从 LangChain 转移到您的团队，这为您配置和运营该层提供了灵活性：

|                                              | **管理方** | **运行位置**           |
|----------------------------------------------|--------------------|-----------------------------|
| LangSmith 平台（UI、API、数据存储）            | 您                | 您的基础设施         |
| Agent Server 运行时                          | 您                | 您的基础设施         |
| PostgreSQL 和 Redis                           | 您                | 您的基础设施         |
| 您应用的 CI/CD                                | 您                | 您的 CI 环境         |
| 升级、扩缩容和备份                            | 您                | 您的基础设施         |

作为回报，您可以集成自己的 [Postgres](/langsmith/self-hosted-platform-features#custom-postgresql) 和 [Redis](/langsmith/self-hosted-platform-features#custom-redis)，为工作负载调整 [CPU 和内存](/langsmith/self-hosted-platform-features#resource-customization) 大小，并在现有的网络和可观测性堆栈中运行。有关对应的云托管模式，请参阅 [部署到云](/langsmith/deploy-to-cloud-overview)。
