自托管 LangSmith 提供两个发布渠道：稳定渠道（客户在生产环境中运行）和预览渠道（跟踪下一个主要版本）。

## 发布渠道

### 稳定版

当前普遍可用的主要版本。LangSmith 推荐将此渠道用于生产环境。稳定版接收包含关键错误修复和安全补丁的每周补丁版本。主要版本之间，稳定版不会添加新功能、数据迁移或基础设施变更。

在任何给定时间，最新主要版本 (N) 是预览渠道，上一个主要版本 (N-1) 是稳定版。

### 预览版

下一个主要版本的开发构建。预览版包含合并到 LangSmith SaaS 中的新功能和修复，以便您可以在下一个主要版本成为稳定版之前进行评估。预览版构建可能包括数据迁移，但不会添加或移除服务或引入破坏性变更。

预览版适用于测试和预发布环境中的评估。LangSmith 不推荐在生产环境中运行预览版。

## 发布周期

| 渠道 | 周期 |
|---------|---------|
| 预览版 | 与 LangSmith SaaS 发布周期同步发布 |
| 稳定版：新主要版本 (`v0.X.0`) | 大约每 6 周（每季度两次） |
| 稳定版：补丁 (`v0.X.Y`) | 每周（通常在周五），如有变更则跳过。关键客户问题会发布临时发布。 |

## 每个渠道包含的内容

|  | 预览版 | 稳定版补丁 | 下一个主要版本 |
|--|:-:|:-:|:-:|
| 新功能 | 是 | 否 | 是 |
| 错误修复 | 是 | 仅关键修复 | 是 |
| 安全补丁 | 是 | 是 | 是 |
| 数据迁移和回填 | 是 | 否 | 是 |
| 新增或移除服务 | 否 | 否 | 是 |
| 破坏性变更 | 否 | 否 | 是 |

服务新增、服务移除和破坏性变更仅出现在新主要版本中，因此请在计划升级到新主要版本时考虑这一点。

## 版本编号

自托管 LangSmith 使用以下方案：

- `v0.X.0`：主要版本（稳定版正式发布）
- `v0.X.Y`：稳定版补丁发布（仅关键修复）
- `v0.X.0-rcN`：下一个主要版本的预览构建（候选发布版本），其中 `N` 是递增的构建号

## 版本支持

LangSmith 支持当前稳定的主要版本和前两个稳定的主要版本。当 `N` 代表当前稳定的主要版本时：

- `N` 获得主动支持，包括关键错误修复、安全补丁和新补丁发布。
- `N-1` 和 `N-2` 获得关键支持，包括关键错误修复和安全补丁。
- 早于 `N-2` 的版本已终止生命周期，不会收到新补丁发布、错误修复或安全更新。

## 建议

- **在生产环境中运行稳定版本。** 预览版仅供评估使用，可能包含仍在验证中的未发布功能。
- **在测试或预发布环境中使用预览版。** 在非生产环境中运行预览版是尽早发现问题并为下一次主要升级做准备的最佳方式。
- **为重大升级做好规划。** 数据迁移、服务添加或移除以及破坏性变更仅出现在新的主要版本中。请查看 [自托管变更日志](/langsmith/self-hosted-changelog) 在升级之前，并规划所需的任何数据或基础设施变更。
- **保持在支持的版本上。** LangSmith 建议在新主要版本发布后尽快升级，以按照建议的节奏获取架构改进。
