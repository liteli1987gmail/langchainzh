自托管 LangSmith 有两个发布渠道：稳定渠道（供客户在生产环境中运行）和预览渠道（追踪下一个主要版本）。

## 发布渠道

### 稳定版

当前普遍可用的大版本。LangSmith 推荐在生产环境中使用此渠道。稳定版每周发布包含关键错误修复和安全补丁的小版本。除大版本之间外，稳定版不会添加新功能、数据迁移或基础设施变更。

在任何给定时间，最新的大版本（N）是预览渠道，而前一个大版本（N-1）是稳定渠道。

### 预览版

下一个大版本的开发构建。预览版会随着 LangSmith SaaS 合入新功能和修复，让您能够在下一个大版本正式成为稳定版之前对其进行评估。预览版本可能包含数据迁移，但不会添加或移除服务，也不会引入破坏性变更。

预览版适用于测试和预发布环境中的评估。LangSmith 不推荐在生产环境中运行预览版。

## 发布节奏

| 渠道 | 节奏 |
|---------|---------|
| 预览版 | 与 LangSmith SaaS 发布节奏同步发布 |
| 稳定版：新大版本（`v0.X.0`） | 大约每 6 周（每季度两个） |
| 稳定版：小版本（`v0.X.Y`） | 每周（通常在周五），无变更时跳过。关键客户问题会发布临时发布。 |

## 每个渠道发布的内容

|  | 预览版 | 稳定版小版本 | 下一个大版本 |
|--|:-:|:-:|:-:|
| 新功能 | 是 | 否 | 是 |
| 错误修复 | 是 | 仅关键修复 | 是 |
| 安全补丁 | 是 | 是 | 是 |
| 数据迁移和回填 | 是 | 否 | 是 |
| 新增或移除服务 | 否 | 否 | 是 |
| 破坏性变更 | 否 | 否 | 是 |

服务新增、服务移除和破坏性变更仅出现在新的大版本中，因此请在规划大版本升级时注意这一点。

## 版本编号

自托管 LangSmith 使用以下方案：

- `v0.X.0`：大版本（稳定版正式发布）
- `v0.X.Y`：稳定版小版本（仅包含关键修复）
- `v0.X.0-rcN`：下一个大版本的预览构建（候选版本），其中 `N` 是递增的构建号

## 版本支持

LangSmith 支持当前稳定的大版本以及前两个稳定的大版本。当 `N` 代表当前稳定的大版本时：

- `N` 获得主动支持，包括关键错误修复、安全补丁和新的小版本发布。
- `N-1` 和 `N-2` 获得关键支持，包括关键错误修复和安全补丁。
- 早于 `N-2` 的版本已终止生命周期，不再获得新的小版本发布、错误修复或安全更新。

## 建议

- **在生产环境中运行稳定版。** 预览版仅用于评估，可能包含仍在验证中的未发布功能。
- **在测试或预发布环境中使用预览版。** 在非生产环境中运行预览版是尽早发现问题并为下一次大版本升级做准备的最好方式。
- **规划大版本升级。** 数据迁移、服务新增或移除以及破坏性变更仅出现在新的大版本中。在升级前查看[自托管变更日志](/langsmith/self-hosted-changelog)，并规划所需的任何数据或基础设施变更。
- **保持在受支持的版本上。** LangSmith 建议在大版本发布后尽快升级，以获取建议节奏中的架构改进。
