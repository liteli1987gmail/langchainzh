在 AWS 上运行 LangSmith 时 [亚马逊云科技 (AWS)](https://aws.amazon.com/), [self-hosted](/langsmith/self-hosted) 模式会部署一个完整的 LangSmith 平台及其可观测性功能。

本页面提供：

- [初始设置步骤](#initial-setup) 用于部署到 EKS、配置托管服务以及设置身份验证。
- [AWS 特定架构模式](#reference-architecture) 以及参考图表。
- [服务建议](#compute-options) 以及最佳实践。
- [AWS 架构完善的最佳实践](#aws-well-architected-best-practices) 涵盖运维卓越性、安全性和可靠性。

<Note>
LangChain 提供了专门针对 AWS 的 Terraform 模块，以帮助为 LangSmith 配置基础设施。这些模块可以快速设置 EKS 集群、RDS、ElastiCache、S3 和网络资源。

查看 [AWS Terraform 模块](https://github.com/langchain-ai/terraform/tree/main/modules/aws) 获取文档和示例。
</Note>

## 初始设置

<Steps>
  <Step title="Deploy to Kubernetes">
    请遵循 [Kubernetes 安装指南](/langsmith/kubernetes)。LangSmith 已在 Amazon Elastic Kubernetes Service (EKS) 上进行了测试。

    **EKS 特定注意事项：**
    - 确保已安装 EBS CSI Driver 以支持持久化存储
    - 使用 `ebs.csi.aws.com` 存储类 provisioner
  </Step>

  <Step title="Configure external services">
    对于生产部署，请连接到 AWS 托管服务：

    <CardGroup cols={2}>
      <Card title="Amazon S3" icon="database" href="/langsmith/self-host-blob-storage#amazon-s3">
        将追踪数据存储在 S3 中
      </Card>
      <Card title="Amazon RDS" icon="database" href="/langsmith/self-host-external-postgres#amazon-rds">
        PostgreSQL 数据库
      </Card>
      <Card title="Amazon ElastiCache" icon="cpu" href="/langsmith/self-host-external-redis#amazon-elasticache">
        使用 Redis 或 Valkey 进行缓存
      </Card>
      <Card title="ClickHouse Cloud" icon="chart-line" href="/langsmith/self-host-external-clickhouse">
        分析数据库
      </Card>
    </CardGroup>
  </Step>

  <Step title="Set up authentication">
    使用 [IAM Roles for Service Accounts (IRSA)](https://docs.aws.amazon.com/eks/latest/userguide/iam-roles-for-service-accounts.html) 对 LangSmith Pod 进行 AWS 服务身份验证，而无需使用静态凭证。

    **关键页面：**
    - [S3 IRSA 配置](/langsmith/self-host-blob-storage#amazon-s3)
    - [RDS IAM 身份验证](/langsmith/self-host-external-postgres#iam-authentication)
    - [ElastiCache IAM 身份验证](/langsmith/self-host-external-redis#iam-authentication)
  </Step>
</Steps>

完成这些初始设置步骤后，您可以查看下面的完整 AWS 架构和最佳实践。

## 参考架构

我们建议利用 AWS 的托管服务来提供一个可扩展、安全且有弹性的平台。以下架构适用于自托管以及混合部署模式，并符合 [AWS 架构完善框架](https://aws.amazon.com/architecture/well-architected/):

![展示 AWS 与 LangSmith 服务关系的架构图](/langsmith/images/aws-architecture-self-hosted.png)

- <Icon icon="globe" /> **入口与网络**：请求通过 [Amazon Application Load Balancer (ALB)](https://aws.amazon.com/elasticloadbalancing/application-load-balancer/) 进入您的 [VPC](https://aws.amazon.com/vpc/)，并使用 [AWS WAF](https://aws.amazon.com/waf/) 和 [IAM](https://aws.amazon.com/iam/)基于 OIDC 的身份验证进行保护。
- <Icon icon="cube" /> **前端和后端服务：** 容器运行在 [Amazon EKS](https://aws.amazon.com/eks/)，通过 ALB 进行编排。根据需要将请求路由到集群内的其他服务。
- <Icon icon="database" /> **存储和数据库：**
  - [Amazon RDS for PostgreSQL](https://aws.amazon.com/rds/postgresql/) or [Aurora](https://aws.amazon.com/rds/aurora/)：元数据、项目、用户，以及已部署代理的短期和长期记忆。LangSmith 支持 PostgreSQL 14 或更高版本。
  - [Amazon ElastiCache](https://aws.amazon.com/elasticache/) （Redis 或 Valkey）：缓存和作业队列。ElastiCache 可以采用单实例或集群模式。LangSmith 需要 Redis OSS 5.0 或更高版本，或 Valkey 8。
  - ClickHouse + [Amazon EBS](https://aws.amazon.com/ebs/)：分析和追踪存储。
    - 我们建议使用 [外部托管的 ClickHouse 解决方案](/langsmith/self-host-external-clickhouse) 除非出于安全或合规原因
    无法使用外部托管解决方案。
    - 混合部署不需要 ClickHouse。
  - [Amazon S3](https://aws.amazon.com/s3/)：用于追踪工件和遥测数据的对象存储。

- <Icon icon="sparkles" /> **LLM 集成：** 可选代理请求至 [Amazon Bedrock](https://aws.amazon.com/bedrock/) or [Amazon SageMaker](https://aws.amazon.com/sagemaker/) 进行 LLM 推理。
- <Icon icon="chart-line" /> **监控与可观测性：** 与...集成 [Amazon CloudWatch](https://aws.amazon.com/cloudwatch/)


## 计算选项

LangSmith根据您的需求支持多种计算选项：

| 计算选项 | 描述 | 适用于 |
|-----------------|-------------|--------------|
| **弹性 Kubernetes 服务（首选）** | 高级扩展和多租户支持 | 大型企业 |
| **EC2-based** | 完全控制、自建基础设施 | 受监管或气隙环境 |

## AWS 架构完善最佳实践

本参考旨在与 AWS 架构完善框架的六大支柱保持一致：

### 卓越运营

- 使用 IaC ([CloudFormation](https://aws.amazon.com/cloudformation/) / [Terraform](https://www.terraform.io/)).
- 使用 [AWS Systems Manager Parameter Store](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-parameter-store.html) 进行配置。
- 配置您的 LangSmith 实例以 [导出遥测数据](/langsmith/export-backend) 并通过...持续监控 [CloudWatch Logs](https://docs.aws.amazon.com/AmazonCloudWatch/latest/logs/WhatIsCloudWatchLogs.html).
- 管理 [LangSmith 部署](/langsmith/deployment) 的首选方法是创建一个构建 [Agent Server](/langsmith/agent-server) 镜像并将其推送到 [ECR](https://aws.amazon.com/ecr/)在将新版本部署到预发布或生产环境之前，为拉取请求创建测试部署。

### 安全性

- 使用 [IAM](https://aws.amazon.com/iam/) 具有最小权限策略的角色。
- 启用静态加密 ([RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Overview.Encryption.html), [S3](https://docs.aws.amazon.com/AmazonS3/latest/userguide/UsingEncryption.html)，ClickHouse 卷) 和传输中加密 (TLS 1.2+)。
- 与 [AWS Secrets Manager](https://aws.amazon.com/secrets-manager/) 集成以管理凭据。
- 使用 [Amazon Cognito](https://aws.amazon.com/cognito/) 作为身份提供商，结合 LangSmith 内置的身份验证和授权功能来保护对代理及其工具的访问安全。

### 可靠性

- 复制 LangSmith [数据平面](/langsmith/data-plane) 跨区域：在不同区域将相同的数据平面部署到 Kubernetes 集群以实现 LangSmith 部署。部署 [RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Concepts.MultiAZSingleStandby.html) 和 [ECS](https://aws.amazon.com/ecs/) 服务跨 [Multi-AZ](https://aws.amazon.com/about-aws/global-infrastructure/regions_az/).
- 实施 [auto-scaling](https://aws.amazon.com/autoscaling/) 用于后端工作程序。
- 使用 [Amazon Route 53](https://aws.amazon.com/route53/) 运行状况检查和故障转移策略。

### 性能效率

- 利用 [EC2](https://aws.amazon.com/ec2/) 实例以优化计算。
- 使用 [S3 Intelligent-Tiering](https://aws.amazon.com/s3/storage-classes/intelligent-tiering/) 存储不频繁访问的跟踪数据。

### 成本优化

- Right-size [EKS](https://aws.amazon.com/eks/) 集群使用 [Compute Savings Plans](https://aws.amazon.com/savingsplans/compute-pricing/).
- 使用 [AWS Cost Explorer](https://aws.amazon.com/aws-cost-management/aws-cost-explorer/) dashboards.

### 可持续性

- 通过按需计算最大限度地减少闲置工作负载。
- 在低延迟、低成本层级中存储遥测数据。
- 为非生产环境启用自动关闭。

## 安全性和合规性

LangSmith 可配置为：

- [PrivateLink](https://aws.amazon.com/privatelink/)专有访问（无公共互联网暴露，除计费所需的出口外）。
- [KMS](https://aws.amazon.com/kms/)基于加密密钥用于 S3、RDS 和 EBS。
- 审计日志到 [CloudWatch](https://aws.amazon.com/cloudwatch/) 和 [AWS CloudTrail](https://aws.amazon.com/cloudtrail/).

客户可部署在 [GovCloud](https://aws.amazon.com/govcloud-us/)、ISO 或 HIPAA 区域（如需要）。

## 监控和评估

使用 LangSmith：

- 捕获在 [Bedrock](https://aws.amazon.com/bedrock/) or [SageMaker](https://aws.amazon.com/sagemaker/).
- 上运行的 LLM 应用的跟踪。 [LangSmith 数据集](/langsmith/manage-datasets).
- 追踪延迟、令牌使用情况和成功率。

集成方式：

- [AWS CloudWatch](https://aws.amazon.com/cloudwatch/) dashboards.
- [OpenTelemetry](https://opentelemetry.io/) 和 [Prometheus](https://prometheus.io/) exporters.
