<Info>
**适用计划限制**

批量导出仅在 [LangSmith Plus 或 Enterprise 级别](https://www.langchain.com/pricing-langsmith) 可用。
</Info>

LangSmith 可以将追踪数据导出到 Google Cloud Storage (GCS) 存储桶，格式为 Parquet。从此，您可以将数据加载到 BigQuery 作为外部表（从 GCS 就地查询）或作为原生表（复制到 BigQuery 存储）。

本指南涵盖：

- 设置用于 LangSmith 的 GCS 存储桶和 HMAC 凭证。
- 创建批量导出目标位置和导出作业。
- 将导出的数据加载到 BigQuery。

有关批量导出配置选项的完整详情，请参阅[批量导出追踪数据](/langsmith/data-export)和[管理批量导出目标位置](/langsmith/data-export-destinations)。

## 前置条件

- LangSmith [追踪项目](https://smith.langchain.com/projects)中的数据。
- 已安装 [`gcloud` CLI](https://docs.cloud.google.com/sdk/docs/install-sdk)。（您也可以使用 Google Cloud 控制台进行设置。）

## 1. 创建 GCS 存储桶

为 LangSmith 导出创建一个专用的 GCS 存储桶。使用专用存储桶可以更轻松地授予受限权限，而不影响其他数据：

```bash
gcloud storage buckets create gs://YOUR_BUCKET_NAME \
  --location=US \
  --uniform-bucket-level-access
```

选择靠近 BigQuery 数据集的区域，以减少延迟并避免跨区域出口费用。

## 2. 创建服务账号并授予访问权限

创建一个 GCP 服务账号，LangSmith 将使用该账号向 GCS 写入数据：

```bash
gcloud iam service-accounts create langsmith-bulk-export \
  --display-name="LangSmith Bulk Export"
```

授予服务账号对您的存储桶的写访问权限。最低要求的权限是 `storage.objects.create`。授予 `storage.objects.delete` 是可选的，但建议执行。LangSmith 使用它来清理目标验证期间创建的临时测试文件。如果缺少此权限，存储桶中可能会保留一个 `tmp/` 文件夹。

"Storage Object Admin" 预定义角色包含所有必需和推荐的权限：

```bash
gcloud storage buckets add-iam-policy-binding gs://YOUR_BUCKET_NAME \
  --member="serviceAccount:langsmith-bulk-export@YOUR_PROJECT.iam.gserviceaccount.com" \
  --role="roles/storage.objectAdmin"
```

要使用最小自定义角色，请仅授予：

- `storage.objects.create`（必需）
- `storage.objects.delete`（可选，用于测试文件清理）
- `storage.objects.get`（可选但建议，用于文件大小验证）
- `storage.multipartUploads.create`（可选但建议，用于大文件上传）

## 3. 生成 HMAC 密钥

LangSmith 使用 S3 兼容的 XML API 连接到 GCS，这需要 HMAC 密钥，而不是服务账号 JSON 密钥。

为您的服务账号生成 HMAC 密钥：

```bash
gcloud storage hmac create \
  langsmith-bulk-export@YOUR_PROJECT.iam.gserviceaccount.com
```

保存输出中的 `accessId` 和 `secret`。您也可以在 GCP Console 中的 **Cloud Storage → Settings → Interoperability → Create a key for a service account** 下生成 HMAC 密钥。

## 4. 创建批量导出目标

在 LangSmith 中创建一个指向您 GCS 存储桶的目标。将 `endpoint_url` 设置为 `https://storage.googleapis.com` 以使用 GCS S3 兼容 API。

您需要您的 [LangSmith API key](/langsmith/create-account-api-key) 和 [workspace ID](/langsmith/set-up-hierarchy#set-up-a-workspace)。

```bash
curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports/destinations' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "destination_type": "s3",
    "display_name": "GCS for BigQuery",
    "config": {
      "bucket_name": "YOUR_BUCKET_NAME",
      "prefix": "YOUR_PREFIX",
      "endpoint_url": "https://storage.googleapis.com"
    },
    "credentials": {
      "access_key_id": "YOUR_HMAC_ACCESS_ID",
      "secret_access_key": "YOUR_HMAC_SECRET"
    }
  }'
```

`prefix` 是存储桶内的一个路径，LangSmith 将在此路径下写入导出的文件。例如，`langsmith-exports` 或 `data/traces`。选择任何适合您存储桶布局的值。

LangSmith 通过执行测试写入来验证凭证，然后再保存目标。如果请求返回 `400` 错误，请参阅[调试目标错误](/langsmith/data-export-destinations#debug-destination-errors)。

保存响应中的 `id`；您将在下一步需要它。

### 临时验证文件

在创建目标（以及[凭证轮换](#credential-rotation)）期间，LangSmith 会将临时 `.txt` 文件写入 `YOUR_PREFIX/tmp/` 以验证写入权限，然后尝试删除它。删除操作是尽力而为的：如果服务账户缺少 `storage.objects.delete` 权限，文件将不会被删除，`tmp/` 文件夹将保留在您的存储桶中。

`tmp/` 文件夹不会影响导出，但它会被包含在广泛的 GCS URI 通配符中（例如 `gs://YOUR_BUCKET_NAME/YOUR_PREFIX/*`）。

## 5. 创建批量导出作业

创建针对特定项目的导出。使用 `format_version: v2_beta` 以实现 BigQuery 兼容性——它会产生 BigQuery 正确处理的 UTC 时区感知时间戳。

您将需要项目 ID（`session_id`），您可以从[**跟踪项目**列表](https://smith.langchain.com)中的项目视图复制它。

**一次性导出：**

```bash
curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "bulk_export_destination_id": "YOUR_DESTINATION_ID",
    "session_id": "YOUR_PROJECT_ID",
    "start_time": "2024-01-01T00:00:00Z",
    "end_time": "2024-02-01T00:00:00Z",
    "format_version": "v2_beta",
    "compression": "snappy"
  }'
```

**定期（重复）导出：**

```bash
curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "bulk_export_destination_id": "YOUR_DESTINATION_ID",
    "session_id": "YOUR_PROJECT_ID",
    "start_time": "2024-01-01T00:00:00Z",
    "interval_hours": 24,
    "format_version": "v2_beta",
    "compression": "snappy"
  }'
```

Snappy 压缩速度快，且被 BigQuery 广泛支持。所有可用选项，请参阅[批量导出追踪数据](/langsmith/data-export#2-create-an-export-job)，包括字段过滤和过滤表达式。

### 输出文件结构

导出文件将使用 Hive 分区路径结构存入 GCS：

```
gs://YOUR_BUCKET_NAME/YOUR_PREFIX/export_id=<uuid>/tenant_id=<uuid>/session_id=<uuid>/resource=runs/year=<year>/month=<month>/day=<day>/<filename>.parquet
```

路径中的分区列（`export_id`、`tenant_id`、`session_id`、`resource`、`year`、`month`、`day`）在启用 Hive 分区检测后可在 BigQuery 中作为可查询列使用。

## 6. 将数据加载到 BigQuery

BigQuery 提供两种访问导出数据的方式。两者都需要先授予 BigQuery 服务账号对 GCS 存储桶的读取权限。请根据您的需求选择：

- **外部表：** 数据保留在 GCS 中，BigQuery 直接查询原地数据。BigQuery 中不产生存储费用，但查询性能比原生存储慢。请参阅[所需角色](https://docs.cloud.google.com/bigquery/docs/query-cloud-storage-data#required-roles)。
- **原生表：** 数据复制到 BigQuery 存储中。查询更快，支持完整的 BigQuery 功能，但会产生 BigQuery 存储费用。请参阅[所需权限](https://docs.cloud.google.com/bigquery/docs/cloud-storage-transfer#required_permissions)。

### 创建表

<Tabs>
  <Tab title="外部表">
    外部表直接从 GCS 查询数据，无需复制到 BigQuery。

    1. 在 BigQuery 控制台中，在 **Explorer** 窗格中展开您的项目和数据集。
    1. 点击数据集的 **操作** 菜单（三个点）并选择 **创建表**。
    1. 在 **Source** 下：
       - 将 **Create table from** 设置为 **Google Cloud Storage**。
       - 将文件路径设置为 `gs://YOUR_BUCKET_NAME/YOUR_PREFIX/export_id=*`。使用 `export_id=*` 可将 BigQuery 的查询范围限定为 Hive 分区导出目录，并排除 LangSmith 在目标验证期间写入的 `tmp/` 文件夹（请参阅[临时验证文件](#temporary-validation-file)）。
       - 将 **File format** 设置为 **Parquet**。
    1. 勾选 **Source data partitioning**，然后：
       - 将 **Source URI prefix** 设置为 `gs://YOUR_BUCKET_NAME/YOUR_PREFIX`。
       - 将 **Partition inference mode** 设置为 **Automatically infer types**。
    1. 在 **Destination** 下：
       - 选择您的项目和数据集。
       - 输入表名，例如 `langsmith_runs`。
       - 将 **Table type** 设置为 **External table**。
    1. 在 **Schema** 下，启用 **Auto-detect**。
    1. 点击 **Create table**。

    分区路径列（`export_id`、`tenant_id`、`session_id`、`resource`、`year`、`month`、`day`）可作为可查询列使用。在查询中筛选 `year`、`month` 或 `day` 以启用分区裁剪。
  </Tab>
  <Tab title="原生表">
    原生表将 Parquet 数据传输到 BigQuery 存储中，以获得完整的查询性能。

    1. 前往 Google Cloud 控制台中的[数据传输页面](https://console.cloud.google.com/bigquery/transfers)并选择 **+ Create transfer**。
    1. 对于 **Source type**，选择 **Google Cloud Storage**。
    1. 输入 **Transfer name**。您可以在必要时访问并编辑该传输。
    1. 选择 **Schedule option**。如果您不想重复导出，可以选择 **On demand** 并手动触发导出。

    1. 在 BigQuery 控制台中，在 **Explorer** 窗格中展开您的项目和数据集。
    1. 点击数据集的 **操作** 菜单（三个点）并选择 **创建表**。
    1. 在 **Source** 下：
       - 将 **Create table from** 设置为 **Google Cloud Storage**。
       - 将文件路径设置为 `gs://YOUR_BUCKET_NAME/YOUR_PREFIX/export_id=*`。使用 `export_id=*` 可排除 LangSmith 在目标验证期间写入的 `tmp/` 文件夹（请参阅[临时验证文件](#temporary-validation-file)）。
       - 将 **File format** 设置为 **Parquet**。
    1. 勾选 **Source data partitioning**，然后：
       - 将 **Source URI prefix** 设置为 `gs://YOUR_BUCKET_NAME/YOUR_PREFIX`。
       - 将 **Partition inference mode** 设置为 **Automatically infer types**。
    1. 在 **Destination** 下：
       - 选择您的项目和数据集。
       - 输入表名，例如 `langsmith_runs`。
       - 将 **Table type** 设置为 **Native table**。
    1. 在 **Advanced options** 下，将 **Write preference** 设置为 **Write if empty** 用于新表。
    1. 点击 **Create table**。

    BigQuery 运行加载作业来复制数据。Hive 分区列作为常规列出现在表中。可用的数据列完整列表，请参阅[可导出字段](/langsmith/data-export#exportable-fields)。
  </Tab>
</Tabs>

## 凭证轮换

要在不中断活跃导出的情况下轮换您的 HMAC 密钥：

1. **在 GCP 中为同一服务账号生成新的 HMAC 密钥**。
2. **使用新凭据调用 PATCH 端点**：

   ```bash
   curl --request PATCH \
     --url 'https://api.smith.langchain.com/api/v1/bulk-exports/destinations/YOUR_DESTINATION_ID' \
     --header 'Content-Type: application/json' \
     --header 'X-API-Key: YOUR_API_KEY' \
     --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
     --data '{
       "credentials": {
         "access_key_id": "NEW_HMAC_ACCESS_ID",
         "secret_access_key": "NEW_HMAC_SECRET"
       }
     }'
   ```

LangSmith 在保存前会使用测试写入来验证新凭据。在此验证过程中，存储桶中可能会出现新的 `tmp/` 文件（请参阅[临时验证文件](#temporary-validation-file)）。

3. **保持旧的 HMAC 密钥有效**，直到所有进行中的导出运行完成。在过渡期间，两组凭据同时有效。
4. 确认没有进行中的运行使用旧密钥后，**删除 GCP 中的旧 HMAC 密钥**。

如需了解完整详情，请参阅[轮换目标凭据](/langsmith/data-export-destinations#rotate-destination-credentials)。

## 故障排除

| 症状 | 可能原因 | 解决方法 |
|---------|--------------|-----|
| 创建目标时出现 `400 Access denied` | HMAC 凭据缺少写入权限 | 验证服务账号对存储桶具有 `storage.objects.create` 权限 |
| `400 Key ID you provided does not exist` | HMAC 访问 ID 无效 | 在 GCP 中重新生成 HMAC 密钥 |
| `400 Invalid endpoint` | 端点 URL 格式错误 | 必须使用 `https://storage.googleapis.com` |
| BigQuery 表显示无行 | 导出尚未完成 | 使用 `GET /api/v1/bulk-exports/{export_id}` 检查导出状态 |
| BigQuery 分区裁剪不生效 | 源 URI 前缀不正确 | 确保源 URI 前缀在第一个分区键之前结束，例如 `gs://BUCKET/PREFIX` |
| BigQuery 拾取了 `tmp/` 文件 | 文件路径通配符范围过宽 | 在文件路径中使用 `export_id=*` 而不是 `*` |

有关其他错误代码和导出状态详情，请参阅[监控和排查批量导出](/langsmith/data-export-monitor)。
