[Azure OpenAI](https://azure.microsoft.com/products/ai-services/openai-service/) 是一项云服务，可帮助您使用来自 OpenAI、Meta 等多种预构建和精选模型快速开发生成式 AI 体验。

LangChain.js 支持与 [Azure OpenAI](https://azure.microsoft.com/products/ai-services/openai-service/) 集成，使用 OpenAI SDK 中的新 Azure 集成。 [OpenAI SDK](https://github.com/openai/openai-node).

您可以在 [此页面](https://learn.microsoft.com/azure/ai-services/openai/overview)了解有关 Azure OpenAI 及其与 OpenAI API 差异的更多信息。如果您没有 Azure 账户，可以 [创建免费账户](https://azure.microsoft.com/free/) 开始使用。

这将帮助您开始使用 AzureOpenAIEmbeddings [嵌入模型](/oss/integrations/embeddings) 配合 LangChain 使用。有关 `AzureOpenAIEmbeddings` 功能和配置选项的详细文档，请参阅 [API 参考](https://reference.langchain.com/javascript/langchain-openai/AzureOpenAIEmbeddings).


<Info>
**此前，LangChain.js 使用专用的 [Azure OpenAI SDK](https://github.com/Azure/azure-sdk-for-js/tree/main/sdk/openai/openai)支持与 Azure OpenAI 的集成。该 SDK 现已弃用，取而代之的是 OpenAI SDK 中的新 Azure 集成，它允许访问最新的 OpenAI 模型，并在发布当天即可使用，同时支持 OpenAI API 和 Azure OpenAI 之间的无缝切换。**


如果您正在使用已弃用的 SDK 的 Azure OpenAI，请参阅 [迁移指南](#migration-from-azure-openai-sdk) 以更新到新的 API。

</Info>

## 设置

要访问 Azure OpenAI 嵌入模型，您需要创建一个 Azure 账户，获取 API 密钥，并安装 `@langchain/openai` 集成包。

### 凭证

您需要部署一个 Azure OpenAI 实例。您可以按照 [本指南](https://learn.microsoft.com/azure/ai-services/openai/how-to/create-resource?pivots=web-portal).

在 Azure Portal 上部署一个版本。实例运行后，请确保您有实例名称和密钥。您可以在 Azure Portal 中实例的"密钥和终结点"部分找到密钥。

如果您使用的是 Node.js，可以定义以下环境变量来使用该服务：

```bash
AZURE_OPENAI_API_INSTANCE_NAME=<YOUR_INSTANCE_NAME>
AZURE_OPENAI_API_EMBEDDINGS_DEPLOYMENT_NAME=<YOUR_EMBEDDINGS_DEPLOYMENT_NAME>
AZURE_OPENAI_API_KEY=<YOUR_KEY>
AZURE_OPENAI_API_VERSION="2024-02-01"
```

如果您想获取模型调用的自动追踪，可以取消下面注释来设置您的 [LangSmith](/langsmith/observability) API 密钥：

```bash
# export LANGSMITH_TRACING="true"
# export LANGSMITH_API_KEY="your-api-key"
```

### 安装

LangChain AzureOpenAIEmbeddings 集成位于 `@langchain/openai` package:

<CodeGroup>
```bash npm
npm install @langchain/openai @langchain/core
```
```bash yarn
yarn add @langchain/openai @langchain/core
```
```bash pnpm
pnpm add @langchain/openai @langchain/core
```
</CodeGroup>


<Info>
**您可以在 [Azure OpenAI 文档](https://learn.microsoft.com/azure/ai-services/openai/reference).**


</Info>

<Tip>
**If `AZURE_OPENAI_API_EMBEDDINGS_DEPLOYMENT_NAME` 如果未定义，它将回退到以下值： `AZURE_OPENAI_API_DEPLOYMENT_NAME` 用于部署名称。相同的规则适用于 `azureOpenAIApiEmbeddingsDeploymentName` 参数在 `AzureOpenAIEmbeddings` 构造函数中，如果未定义将回退到 `azureOpenAIApiDeploymentName` 的值（如果未定义）。**


</Tip>

## 实例化

现在我们可以实例化模型对象并嵌入文本：

```typescript
import { AzureOpenAIEmbeddings } from "@langchain/openai";

const embeddings = new AzureOpenAIEmbeddings({
  azureOpenAIApiKey: "<your_key>", // In Node.js defaults to process.env.AZURE_OPENAI_API_KEY
  azureOpenAIApiInstanceName: "<your_instance_name>", // In Node.js defaults to process.env.AZURE_OPENAI_API_INSTANCE_NAME
  azureOpenAIApiEmbeddingsDeploymentName: "<your_embeddings_deployment_name>", // In Node.js defaults to process.env.AZURE_OPENAI_API_EMBEDDINGS_DEPLOYMENT_NAME
  azureOpenAIApiVersion: "<api_version>", // In Node.js defaults to process.env.AZURE_OPENAI_API_VERSION
  maxRetries: 1,
});
```

## 索引和检索

嵌入模型常用于检索增强生成（RAG）流程中，既作为索引数据的一部分，也用于后续检索。如需更详细的说明，请参阅 [**学习** 选项卡下的](/oss/learn/).

下面，请查看如何使用 `embeddings` 对象（我们在上面初始化的）来索引和检索数据。在此示例中，我们将使用演示 [`MemoryVectorStore`](/oss/integrations/vectorstores/memory).

```typescript
// Create a vector store with a sample text
import { MemoryVectorStore } from "@langchain/classic/vectorstores/memory";

const text = "LangChain is the framework for building context-aware reasoning applications";

const vectorstore = await MemoryVectorStore.fromDocuments(
  [{ pageContent: text, metadata: {} }],
  embeddings,
);

// Use the vector store as a retriever that returns a single document
const retriever = vectorstore.asRetriever(1);

// Retrieve the most similar text
const retrievedDocuments = await retriever.invoke("What is LangChain?");

retrievedDocuments[0].pageContent;
```

```text
LangChain is the framework for building context-aware reasoning applications
```

## 直接使用

在底层，向量存储和检索器实现调用 `embeddings.embedDocument(...)` 和 `embeddings.embedQuery(...)` 来创建用于 `fromDocuments` 和检索器的 `invoke` 操作中的文本嵌入。

您可以直接调用这些方法来获取嵌入以满足您的使用场景。

const singleVector = await embeddings.embedQuery(text);

## 使用 Azure 托管标识

如果您使用的是 Azure 托管标识，可以按以下方式配置凭据：

```typescript
import {
  DefaultAzureCredential,
  getBearerTokenProvider,
} from "@azure/identity";
import { AzureOpenAIEmbeddings } from "@langchain/openai";

const credentials = new DefaultAzureCredential();
const azureADTokenProvider = getBearerTokenProvider(
  credentials,
  "https://cognitiveservices.azure.com/.default"
);

const modelWithManagedIdentity = new AzureOpenAIEmbeddings({
  azureADTokenProvider,
  azureOpenAIApiInstanceName: "<your_instance_name>",
  azureOpenAIApiEmbeddingsDeploymentName: "<your_embeddings_deployment_name>",
  azureOpenAIApiVersion: "<api_version>",
});

```

## 使用不同的域

如果您的实例托管在默认域以外的域上 `openai.azure.com`，则需要使用备用 `AZURE_OPENAI_BASE_PATH` 环境变量。
例如，以下是连接到该域的方法 `https://westeurope.api.microsoft.com/openai/deployments/{DEPLOYMENT_NAME}`:

```typescript
import { AzureOpenAIEmbeddings } from "@langchain/openai";

const embeddingsDifferentDomain = new AzureOpenAIEmbeddings({
  azureOpenAIApiKey: "<your_key>", // In Node.js defaults to process.env.AZURE_OPENAI_API_KEY
  azureOpenAIApiEmbeddingsDeploymentName: "<your_embedding_deployment_name>", // In Node.js defaults to process.env.AZURE_OPENAI_API_EMBEDDINGS_DEPLOYMENT_NAME
  azureOpenAIApiVersion: "<api_version>", // In Node.js defaults to process.env.AZURE_OPENAI_API_VERSION
  azureOpenAIBasePath:
    "https://westeurope.api.microsoft.com/openai/deployments", // In Node.js defaults to process.env.AZURE_OPENAI_BASE_PATH
});

```

## 自定义请求头

您可以通过传入 `configuration` field:

```typescript
import { AzureOpenAIEmbeddings } from "@langchain/openai";

const embeddingsWithCustomHeaders = new AzureOpenAIEmbeddings({
  azureOpenAIApiKey: "<your_key>",
  azureOpenAIApiInstanceName: "<your_instance_name>",
  azureOpenAIApiEmbeddingsDeploymentName: "<your_embeddings_deployment_name>",
  azureOpenAIApiVersion: "<api_version>",
  configuration: {
    defaultHeaders: {
      "x-custom-header": `SOME_VALUE`,
    },
  },
});
```

该 `configuration` 字段还接受其他 `ClientOptions` 官方 SDK 接受的参数。

**Note:** 特定的请求头 `api-key` 目前无法以这种方式覆盖，将传递来自 `azureOpenAIApiKey`.

## 从 Azure OpenAI SDK 迁移

如果您使用的是已弃用的 Azure OpenAI SDK（使用 `@langchain/azure-openai` 包），可以按照以下步骤更新代码以使用新的 Azure 集成：

1. 安装新的 `@langchain/openai` 包并移除之前的 `@langchain/azure-openai` package:

   ```bash npm
   npm install @langchain/openai
   npm uninstall @langchain/azure-openai
   ```

2. 更新导入语句以使用新的 `AzureOpenAIEmbeddings` 类（来自 `@langchain/openai` package:

   ```typescript
   import { AzureOpenAIEmbeddings } from "@langchain/openai";
   ```

3. 更新代码以使用新的 `AzureOpenAIEmbeddings` 类并传递所需参数：

   ```typescript
   const model = new AzureOpenAIEmbeddings({
     azureOpenAIApiKey: "<your_key>",
     azureOpenAIApiInstanceName: "<your_instance_name>",
     azureOpenAIApiEmbeddingsDeploymentName:
       "<your_embeddings_deployment_name>",
     azureOpenAIApiVersion: "<api_version>",
   });
   ```

   请注意，构造函数现在需要 `azureOpenAIApiInstanceName` 参数，而不是 `azureOpenAIEndpoint` 参数，并且添加了 `azureOpenAIApiVersion` 参数来指定API版本。

   - 如果您使用的是Azure托管标识，现在需要使用 `azureADTokenProvider` 参数到构造函数中，而不是 `credentials`，请参阅 [Azure托管标识](#using-azure-managed-identity) 部分了解更多详情。

   - 如果您使用的是环境变量，现在必须设置 `AZURE_OPENAI_API_INSTANCE_NAME` 环境变量，而不是 `AZURE_OPENAI_API_ENDPOINT`，并添加 `AZURE_OPENAI_API_VERSION` 环境变量来指定API版本。
