本指南将引导您了解如何通过 Isaacus 的 LangChain 集成开始生成法律嵌入。

## 1. 设置您的账户

前往 [Isaacus 平台](https://platform.isaacus.com/accounts/signup/) 创建新账户。

注册后，[添加支付方式](https://platform.isaacus.com/billing/) 以领取您的[免费积分](https://docs.isaacus.com/pricing/credits)。

添加支付方式后，[创建新的 API 密钥](https://platform.isaacus.com/users/api-keys/)。

请确保您的 API 密钥安全。创建后您将无法再次查看它。但别担心，您可以随时生成新的密钥。

## 2. 安装 isaacus API 客户端

现在您已完成账户设置，安装 [Isaacus LangChain](https://pypi.org/project/langchain-isaacus/) 集成包。

<CodeGroup>
```bash pip
pip install langchain-isaacus
```

```bash uv
uv add langchain-isaacus
```
</CodeGroup>

## 3. 嵌入文档

安装好 API 客户端后，让我们开始嵌入第一个法律查询和文档。

首先，你需要**使用 API 密钥初始化客户端**。你可以通过设置 `ISAACUS_API_KEY` 环境变量来做到这一点，也可以直接传递密钥，就像本示例中做的那样。

我们将使用 [Kanon 2 Embedder](https://isaacus.com/blog/introducing-kanon-2-embedder)，这是截至 2025 年 10 月 20 日在 [Massive Legal Embedding Benchmark](https://isaacus.com/blog/introducing-mleb) 上全球最准确的法律嵌入模型。

```python
from langchain_isaacus import IsaacusEmbeddings

# Create an Isaacus API client for Kanon 2 Embedder.
client = IsaacusEmbeddings(
    model="kanon-2-embedder",
    api_key="PASTE_YOUR_API_KEY_HERE",
    # dimensions=1792, # You may optionally wish to specify a lower dimension.
)
```
接下来，让我们获取一个法律文档进行嵌入。在这个示例中，我们将使用 [GitHub 的服务条款](https://github.com/terms)。

```python
import isaacus

tos = isaacus.Isaacus().get(path="https://examples.isaacus.com/github-tos.md", cast_to=str)
```

我们希望根据一个关于 GitHub 服务条款的搜索查询来检索它。

为此，我们将首先使用 API 客户端的 `.embed_documents()` 方法来嵌入文档。使用此方法表示我们正在嵌入文档（而不是搜索查询），这对于确保我们的嵌入针对检索（而不是其他任务（如分类或句子相似性））进行了优化非常重要。

```python
document_embedding = client.embed_documents(texts=[tos])[0]
```

现在，让我们嵌入两个搜索查询，一个与文档明显相关，另一个与文档明显不相关。这次我们将使用 API 客户端的 `.embed_query()` 方法，它表示我们正在嵌入一个搜索查询。

```python
relevant_query_embedding = client.embed_query(text="What are GitHub's billing policies?")
irrelevant_query_embedding = client.embed_query(text="What are Microsoft's billing policies?")
```

为了评估查询与文档的相关性，我们可以计算它们的嵌入与文档嵌入之间的余弦相似度。

余弦相似度衡量两组数字的相似程度（具体来说，是内积空间中两个向量之间夹角的余弦值）。在理论上，它的范围从 $$-1$$ 到 $$1$$，其中 $$1$$ 表示向量完全相同，$$0$$ 表示它们正交（即完全不相似），而 $$-1$$ 表示它们完全相反。然而，在实践中，对于文本嵌入，它通常在 $$0$$ 到 $$1$$ 之间（因为它们通常是非负的）。

Isaacus 的嵌入器已经过优化，使得它们产生的嵌入的余弦相似度大致对应于原始文本在含义上的相似程度。然而，与 Isaacus 的通用分类器不同，Isaacus 嵌入器的分数没有经过校准以被解释为概率，而只是作为相对相似度度量，这使得它们最适合用于排序搜索结果。

为了方便起见，我们的 Python 示例使用 [`numpy`](https://numpy.org/) 的 `dot` 函数来计算嵌入的点积（由于所有嵌入都是 L2 归一化的，这等价于它们的余弦相似度）。如果你愿意，可以使用其他库来计算嵌入的余弦相似度（例如，通过 `torch.nn.functional.cosine_similarity` 使用 [`torch`](https://pytorch.org/)），或者你也可以自己编写实现。

```python
import numpy as np

relevant_similarity = np.dot(relevant_query_embedding, document_embedding)
irrelevant_similarity = np.dot(irrelevant_query_embedding, document_embedding)

print(f"Similarity of relevant query to the document: {relevant_similarity * 100:.2f}")
print(f"Similarity of irrelevant query to the document: {irrelevant_similarity * 100:.2f}")
```

<fragment>

</fragment>
```
Similarity of relevant query to the document: 52.87
Similarity of irrelevant query to the document: 24.86
```

正如您所看到的，相关查询与文档的相似度得分比不相关查询高得多，这表明我们的嵌入器已成功捕获了文本的语义含义。

就是这样！您刚刚成功使用 Isaacus API 和 LangChain 嵌入了一份法律文档和查询。
