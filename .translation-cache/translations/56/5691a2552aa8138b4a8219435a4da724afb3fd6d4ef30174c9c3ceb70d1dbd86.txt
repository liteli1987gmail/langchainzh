本指南将引导你创建第一个具有规划、文件系统和子代理功能的深度智能体。你将构建一个能够进行研究并撰写报告的研究代理。

    **使用 AI 编程助手？**

    - 安装 [LangChain Docs MCP server](/use-these-docs) 为你的代理提供最新的 LangChain 文档和示例访问。
    - 安装 [LangChain Skills](https://github.com/langchain-ai/langchain-skills) 以提升你的代理在 LangChain 生态系统任务上的表现。

## 前提条件

在开始之前，请确保你拥有模型提供商的 API 密钥（例如 Gemini、Anthropic、OpenAI）。

<Note>
    深度智能体需要支持[工具调用](/oss/langchain/models#tool-calling)的模型。有关如何配置模型，请参阅[自定义](/oss/deepagents/customization#model)。
</Note>

## 工作原理

您的 deep agent 会自动执行以下操作：

1. **规划方法** - 使用内置的 [`write_todos`](/oss/deepagents/overview#task-planning) 工具来分解研究任务。
1. **进行研究** - 调用 `internet_search` 工具来收集信息。
1. **管理上下文** - 使用文件系统工具（[`write_file`](/oss/deepagents/overview#virtual-filesystem-access)、[`read_file`](/oss/deepagents/overview#virtual-filesystem-access)）来卸载大型搜索结果。
1. **生成子 agent** - 根据需要生成子 agent，将复杂的子任务委托给专业子 agent。
1. **综合报告** - 将发现内容编译成连贯的响应。

## 流式处理

Deep Agents 内置了 [流式处理](/oss/langchain/event-streaming) 功能，可通过 LangGraph 从 agent 执行中获取实时更新。
这允许您逐步观察输出，并审查和调试 agent 和子 agent 的工作，如工具调用、工具结果和 LLM 响应。

## 下一步

现在您已经构建了您的第一个 deep agent：

- **自定义您的 agent**：了解[自定义选项](/oss/deepagents/customization)，包括自定义系统提示、工具和子 agent。
- **添加长期记忆**：跨对话启用[持久内存](/oss/deepagents/memory)。
- **部署到生产环境**：使用 [托管 Deep Agents](/langsmith/managed-deep-agents-overview) 在 LangSmith 中创建、运行和操作 deep agent。
