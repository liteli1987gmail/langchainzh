本指南将引导您创建您的第一个具备规划、文件系统和子智能体能力的深度智能体。您将构建一个能够进行研究和撰写报告的研究智能体。

<Tip>
    **使用 AI 编码助手？**

    - 安装 [LangChain 文档 MCP 服务器](/use-these-docs) ，让您的智能体能够访问最新的 LangChain 文档和示例。
    - 安装 [LangChain 技能](https://github.com/langchain-ai/langchain-skills) ，以提升您的智能体在 LangChain 生态系统任务中的表现。
</Tip>

## 前提条件

在开始之前，请确保您拥有模型提供商的 API 密钥（例如 Gemini、Anthropic、OpenAI）。

<Note>
    深度智能体需要一个支持 [工具调用](/oss/langchain/models#tool-calling)的模型。请参阅 [自定义](/oss/deepagents/customization#model) ，了解如何配置您的模型。
</Note>

## 步骤 1：安装依赖项

:::python
<CodeGroup>

:::js
<CodeGroup>

    ```bash yarn
    yarn add deepagents langchain @langchain/core @langchain/tavily
    ```

    ```bash pnpm
    pnpm add deepagents langchain @langchain/core @langchain/tavily
    ```

</CodeGroup>
:::

<Note>
    本指南使用 [Tavily](https://tavily.com/) 作为示例搜索提供商，但您可以替换为任何搜索 API（例如 DuckDuckGo、SerpAPI、Brave Search）。
</Note>

## 它是如何工作的？

您的深度智能体自动执行以下操作：

1. **规划其方法** ，使用内置 [`write_todos`](/oss/deepagents/overview#task-planning) 工具来分解研究任务。
1. **进行研究** ，通过调用 `internet_search` 工具来收集信息。
1. **管理上下文** 使用文件系统工具（[`write_file`](/oss/deepagents/overview#virtual-filesystem-access), [`read_file`](/oss/deepagents/overview#virtual-filesystem-access)）来卸载大型搜索结果。
1. **生成子代理** ，以便根据需要将复杂的子任务委托给专业化的子代理。
1. **综合生成报告** ，将调查结果编译成连贯的回复。

要了解您可以使用 Deep Agents 构建的代理、模式和应用程序，请参阅 [示例](https://github.com/langchain-ai/deepagents/tree/main/examples).

## 流式传输

Deep Agents 内置 [流式传输](/oss/langchain/event-streaming) 功能，可通过 LangGraph 获取代理执行的实时更新。
这允许您逐步观察输出，并审查和调试代理和子代理的工作，例如工具调用、工具结果和 LLM 响应。

## 后续步骤

既然您已经构建了您的第一个深度代理：

- **自定义您的代理**：了解 [自定义选项](/oss/deepagents/customization)，包括自定义系统提示、工具和子代理。
- **添加长期记忆**：启用 [持久记忆](/oss/deepagents/memory) 跨对话保留。
- **部署到生产环境**：使用 [托管 Deep Agents](/langsmith/managed-deep-agents-overview) 在 LangSmith 中创建、运行和操作深度代理。
