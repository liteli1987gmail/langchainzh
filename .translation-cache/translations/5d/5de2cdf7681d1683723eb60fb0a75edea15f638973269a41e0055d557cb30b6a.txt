title: INVALID_PROMPT_INPUT
---

{/* TODO: fix link when porting page */}
发生于 [提示词模板](https://github.com/langchain-ai/langchain/blob/v0.3/docs/docs/concepts/prompt_templates.mdx) 接收到缺失或无效的输入变量时。

:::js
一种意想不到的情况是，如果你直接将一个 JSON 对象添加到提示词模板中：

```typescript
import { PromptTemplate } from "@langchain/core/prompts";
import { ChatOpenAI } from "@langchain/openai";

const prompt = PromptTemplate.fromTemplate(`You are a helpful assistant.

Here is an example of how you should respond:

{
  "firstName": "John",
  "lastName": "Doe",
  "age": 21
}

Now, answer the following question:

{question}`);
```

你可能认为上面的提示词模板应该需要一个名为 question 的单一输入键，但 JSON 对象会被解释为一个额外的变量，因为大括号 (`{`) 没有被转义，应该在前面再加一个大括号，像这样：

```typescript
import { PromptTemplate } from "@langchain/core/prompts";
import { ChatOpenAI } from "@langchain/openai";

const prompt = PromptTemplate.fromTemplate(`You are a helpful assistant.

Here is an example of how you should respond:

{{
  "firstName": "John",
  "lastName": "Doe",
  "age": 21
}}

Now, answer the following question:

{question}`);
```
:::


## 故障排除

要解决此错误，你可以：

1. 检查你的提示词模板是否正确。使用 f-string 格式时，确保正确转义大括号：
   * 在 f-string 中使用 `{{` 表示单个大括号
   * 在 f-string 中使用 `{{{{` 表示双重大括号
2. 当使用 `MessagesPlaceholder` 组件时，确认你传递的是消息数组或类似消息的对象。如果使用简写元组，请将变量名用大括号包裹，如 `["placeholder", "{messages}"]`
3. 通过使用 [LangSmith](/langsmith/observability) 或日志记录来检查实际输入到提示词模板的内容，以验证它们是否符合预期
4. 如果从 LangChain [提示词中心](https://smith.langchain.com/prompts)获取提示词，请使用示例输入隔离并测试提示词，以确保其按预期工作
