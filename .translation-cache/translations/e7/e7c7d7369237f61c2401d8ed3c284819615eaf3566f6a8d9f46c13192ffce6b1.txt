Studio 是一个专业的代理 IDE，可实现实现了 Agent Server API 协议的代理系统的可视化、交互和调试。Studio 还集成了 [追踪](/langsmith/observability-concepts), [评估](/langsmith/evaluation)和 [提示词工程](/langsmith/prompt-engineering).

## 功能

Studio 的主要功能：

* 可视化您的图架构
* [运行并与您的代理交互](/langsmith/use-studio#run-application)
* [管理助手](/langsmith/use-studio#manage-assistants)
* [管理线程](/langsmith/use-studio#manage-threads)
* [迭代提示词](/langsmith/observability-studio)
* [在数据集上运行实验](/langsmith/observability-studio#run-experiments-over-a-dataset)
* 管理 [长期记忆](/oss/concepts/memory)
* 通过以下方式调试代理状态 [时间旅行](/oss/langgraph/use-time-travel)
* 一键部署到 LangSmith Cloud。

Studio 适用于部署在 [LangSmith](/langsmith/deployment-quickstart) 上的图，或通过以下方式本地运行的图 [Agent Server](/langsmith/local-dev-testing).

Studio 支持两种模式：

### 图模式

图模式展示了完整的功能集，在您需要获取有关代理执行的尽可能多的详细信息时非常有用，包括遍历的节点、中间状态和 LangSmith 集成（如添加到数据集和 playground）。

### 聊天模式

聊天模式是一个更简单的 UI，用于迭代和测试聊天专用代理。它对业务用户和想要测试整体代理行为的人很有用。聊天模式仅支持状态包含或扩展 [`MessagesState`](/oss/langgraph/use-graph-api#messagesstate).

## 从 Studio 部署

从 [在 Studio 中本地测试图](/langsmith/local-dev-testing) 一键部署到 LangSmith Cloud，直接从 Studio 进行部署。您可以用它来创建全新的部署以进行快速原型设计，或重新部署现有部署。
