LangSmith 应用程序暴露可发送至您所选后端的遥测数据。如果您还没有可观测性堆栈，或希望将 LangSmith 遥测数据与主应用程序分开，可以使用 LangSmith 可观测性 Helm chart 部署一个基本的可观测性堆栈。

# 第 1 节：Prometheus 导出器

如果您只想为自托管部署中的组件部署指标导出器（然后使用遥测工具进行抓取），请使用本节。如果您希望获得完整的可观测性堆栈部署，请前往[端到端部署部分](/langsmith/observability-stack#prerequisites)。

Helm chart 提供了一组 Prometheus 导出器，用于从 [Redis](https://github.com/prometheus-community/helm-charts/tree/main/charts/prometheus-redis-exporter)、[Postgres](https://github.com/prometheus-community/helm-charts/tree/main/charts/prometheus-postgres-exporter)、[Nginx](https://github.com/prometheus-community/helm-charts/tree/main/charts/prometheus-nginx-exporter) 和 [Kube state metrics](https://github.com/prometheus-community/helm-charts/tree/main/charts/kube-state-metrics) 暴露指标。

1. 创建一个名为 `langsmith_obs_config.yaml` 的本地文件
2. 将此[文件](https://github.com/langchain-ai/helm/blob/main/charts/langsmith-observability/examples/metric-exporters-only.yaml)中的值复制到 `langsmith_obs_config.yaml` 中，确保修改值以匹配您的 LangSmith 部署。
3. 通过运行 `helm search repo langchain/langsmith-observability --versions` 查找 chart 的最新版本。
4. 获取最新版本号，然后运行 `helm install langsmith-observability langchain/langsmith-observability --values langsmith_obs_config.yaml --version <version> -n <namespace> --wait --debug`

这将允许您在以下服务端点抓取指标：

* Postgres: `langsmith-observability-postgres-exporter:9187/metrics`
* Redis: `langsmith-observability-redis-exporter:9121/metrics`
* Nginx: `langsmith-observability-nginx-exporter:9113/metrics`
* KubeStateMetrics: `langsmith-observability-kube-state-metrics:8080/metrics`

如果安装成功，您应该会看到以下内容：


# 第 2 部分：完整可观测性堆栈

<Warning>
**这不是生产环境的可观测性堆栈。使用它可以快速了解部署的日志、指标和追踪。此配置仅能处理每天几十 GB 的数据。**
</Warning>

本节将向您展示如何使用 [Helm Chart](https://github.com/langchain-ai/helm/tree/main/charts/langsmith-observability) 部署 LangSmith 的端到端可观测性堆栈。

此 Chart 基于 Grafana 的开源 LGTM 堆栈构建，由以下组件组成：

* [Loki](https://grafana.com/docs/loki/latest/) 用于日志。
* [Mimir](https://grafana.com/docs/mimir/latest/) 用于指标 + 告警。
* [Tempo](https://grafana.com/docs/tempo/latest/) 用于追踪。
* [Grafana](https://grafana.com/docs/grafana/latest/) 用于监控界面。

以及用于收集遥测数据的 [OpenTelemetry Collectors](https://opentelemetry.io/docs/collector/)。

## 前置条件

### 1. 计算资源

堆栈中每个部分的资源请求和限制可以在 Helm Chart 中修改。以下是当前的资源配置（请求/限制）：

* Loki：`2vCPU/3vCPU + 2Gi/4Gi`
* Mimir：`1vCPU/2vCPU + 2Gi/4Gi`
* Tempo：`1vCPU/2vCPU + 4Gi/6Gi`

启动 Helm Chart 前请确保已分配这些资源，或在 Helm 配置文件中修改资源值。

### 2. Cert-Manager

Helm Chart 使用 OpenTelemetry Operator 来配置收集器。Operator 要求您在 Kubernetes 集群中已安装 [cert-manager](https://cert-manager.io/docs/installation/)。

如果尚未安装，可以运行以下命令：

```bash
helm repo add jetstack https://charts.jetstack.iohelm repo updatehelm install cert-manager jetstack/cert-manager -n cert-manager --create-namespace
```

### 3. OpenTelemetry operator

使用以下命令安装 OpenTelemetry Operator：

```bash
helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-chartshelm repo updatehelm install opentelemetry-operator open-telemetry/opentelemetry-operator -n <namespace>
```

## 安装

以下说明将启动 OTel 采集器、LGTM 堆栈、Grafana 和 Prometheus 导出器。

1. 创建名为 `langsmith_obs_config.yaml` 的本地文件
2. 将此[文件](https://github.com/langchain-ai/helm/blob/main/charts/langsmith-observability/examples/e2e-stack.yaml)中的值复制到 `langsmith_obs_config.yaml` 中，并确保修改值以匹配您的 LangSmith 部署。
3. 运行 `helm search repo langchain/langsmith-observability --versions` 查找图表的最新版本。

## 安装后

### 在 LangSmith 中启用日志和追踪

安装可观测性 helm chart 后，您需要在 *LangSmith* helm 配置文件中设置以下值，以启用日志和追踪的收集。

```yaml
commonPodAnnotations:
  # E.g.: "langsmith-observability/langsmith-observability-collector-sidecar"
  sidecar.opentelemetry.io/inject: "${LANGSMITH_OBS_NAMESPACE}/${LANGSMITH_OTEL_CRD_NAME}"
observability:
  tracing:
    enabled: true
    # Replace this with the endpoint of your trace collector.
    # E.g.: "http://langsmith-observability-collector-gateway-collector.langsmith-observability.svc.cluster.local:4318/v1/traces"
    endpoint: "http://${GATEWAY_COLLECTOR_SERVICE_NAME}.${LANGSMITH_OBS_NAMESPACE}.svc.cluster.local:4318/v1/traces"
```

<Info>
1. 要获取 `${LANGSMITH_OTEL_CRD_NAME}`，可以运行 `kubectl get opentelemetrycollectors -n ${LANGSMITH_OBS_NAMESPACE}` 并选择 MODE = `sidecar` 的那个的名称
2. 要获取 `${GATEWAY_COLLECTOR_SERVICE_NAME}` 名称，运行 `kubectl get services -n ${LANGSMITH_OBS_NAMESPACE}` 并选择端口为 4317/4318 且设置了 ClusterIP 的那个。它应该是类似于 `langsmith-observability-collector-gateway-collector` 这样的名称
</Info>

现在运行 `helm upgrade langsmith langchain/langsmith --values langsmith_config.yaml -n <langsmith-namespace> --wait --debug`

升级完成后，如果运行 `kubectl get pods -n <langsmith-namespace>`，你应该看到以下内容（注意 sidecar 收集器的 2/2）：

## Grafana 使用方法

安装完所有组件后，请执行以下操作来获取您的 Grafana 密码：

```bash
echo "Your Grafana password is:" && oc get secret prometheus-grafana -o jsonpath="{.data['admin-password']}" | base64 -D && echo ""
```

```bash
kubectl get secret langsmith-observability-grafana -n <langsmith_observability_namespace> -o jsonpath="{.data.admin-password}" | base64 --decode
```

然后通过端口转发连接到 `langsmith-observability-grafana` 容器，端口为 3000，然后在浏览器中访问 `localhost:3000`。使用用户名 `admin` 和上面 secret 中的密码登录 Grafana。

进入 Grafana 后，您可以使用界面来监控日志、指标和追踪。Grafana 还预置了仪表板集合，用于监控部署的主要组件。
