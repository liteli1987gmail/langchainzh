---
title: 指定自定义运行 ID
description: 如何在 LangSmith 追踪时指定自定义运行 ID。
---

默认情况下，LangSmith 会为每个[运行](/langsmith/observability-concepts#runs)分配一个随机 ID。您可以在需要时使用自定义 ID 覆盖它：

- 提前知道运行 ID（例如，在运行完成后立即附加[反馈](/langsmith/observability-concepts#feedback)）。
- 将 LangSmith 运行与外部系统的 ID 关联。
- 通过重用确定性 ID 使运行具有幂等性。

<Note>
我们建议使用 **UUID v7** 自定义运行 ID。UUIDv7 嵌入时间戳，可保留运行在追踪中的正确时间顺序。当前传递非 UUIDv7 ID 会发出警告，未来版本将强制要求。

[LangSmith SDK](/langsmith/reference) 导出了 `uuid7` 辅助函数（Python v0.4.43+、JS v0.3.80+）：

- **Python**: `from langsmith import uuid7`
- **JS/TS**: `import { uuid7 } from 'langsmith'`
</Note>

接受任何 UUID v7 字符串。您可以传递由 LangSmith SDK 辅助函数生成的 UUID，或者使用您自己的（如果您的系统已使用 UUID v7 标识符）。

<Tabs>
  <Tab title="Python">
    ### 使用 `@traceable`

    调用 @[`@traceable`] 函数时，在 `langsmith_extra` 中传递 `run_id`：

    ```python Python
    from langsmith import traceable, uuid7

    @traceable
    def my_pipeline(question: str) -> str:
        return "answer"

    run_id = uuid7()
    my_pipeline("What is the capital of France?", langsmith_extra={"run_id": run_id})

    # run_id can now be used to attach feedback, query the run, etc.
    ```

### 使用 `trace` 上下文管理器

直接将 `run_id` 传递给 @[trace] 上下文管理器构造函数，为该追踪块设置 ID：

    ```python Python
    from langsmith import trace, uuid7

    run_id = uuid7()

    with trace("my-pipeline", run_id=run_id) as run:
        result = "answer"
        run.end(outputs={"result": result})

    # run_id can now be used to attach feedback, query the run, etc.
    ```
</Tab>
  <Tab title="TypeScript">
    ### 使用 `traceable`

    在传递给 [`traceable`](https://reference.langchain.com/javascript/langsmith/traceable/traceable) 的配置对象中传递 `id`：

</fragment>

    ```typescript TypeScript
    import { traceable } from "langsmith/traceable";
    import { uuid7 } from "langsmith";

    const runId = uuid7();

    const myPipeline = traceable(
      async (question: string) => {
        return "answer";
      },
      { name: "my-pipeline", id: runId }
    );

    await myPipeline("What is the capital of France?");

    // runId can now be used to attach feedback, query the run, etc.
    ```

</Tab>
</Tabs>
