:::python
**Harness 配置文件** 让你打包 Deep Agents 在选择指定提供商或特定模型时应用的配置：系统提示词调整、工具描述覆盖、排除的工具或中间件、额外的中间件，以及通用子代理编辑。这是在不更改调用点的情况下调整 Harness 对特定模型行为的主要方式 `create_deep_agent` 调用点。使用 `HarnessProfile` 在 Python 中构建配置文件；使用 `HarnessProfileConfig` 当 [loading or saving YAML/JSON files](#load-profiles-from-config-files)时。Deep Agents 随附针对 OpenAI 和 Anthropic（Claude）模型的内置 Harness 配置文件。

**Provider 配置文件** 是针对 *model-construction* kwargs 的更窄的配套 API，不会影响 Harness。大多数调用者不需要它们；当你想让 `init_chat_model` 默认值、凭据检查或运行时派生的 kwargs 作为默认值随你的提供商选择一起提供时（例如在打包提供商集成时），可以使用它们。
:::

:::js
**Harness 配置文件** 让你打包 Deep Agents 在选择指定提供商或特定模型时应用的配置：系统提示词调整、工具描述覆盖、排除的工具或中间件、额外的中间件，以及通用子代理编辑。这是在不更改调用点的情况下调整 Harness 对特定模型行为的主要方式 `createDeepAgent` 调用点。使用 `HarnessProfileOptions` 构建配置文件；当 `parseHarnessProfileConfig` 时使用 [loading or saving YAML/JSON files](#load-profiles-from-config-files)。Deep Agents 随附针对 OpenAI 和 Anthropic（Claude）模型的内置 Harness 配置文件。

<Note>
  Provider 配置文件（用于控制模型构造 kwargs）和插件注册系统是仅 Python 专有功能。TypeScript SDK 仅支持 Harness 配置文件。
</Note>
:::

## Harness 配置文件

:::python
A `HarnessProfile` 描述在聊天模型构建后应用的提示词组装、工具可见性、中间件和默认子代理调整： `create_deep_agent` 

```python
from deepagents import (
    GeneralPurposeSubagentProfile,
    HarnessProfile,
    register_harness_profile,
)

register_harness_profile(
    "openai:gpt-5.5",
    HarnessProfile(
        system_prompt_suffix="Respond in under 100 words.",
        excluded_tools={"execute"},
        excluded_middleware={"SummarizationMiddleware"},
        general_purpose_subagent=GeneralPurposeSubagentProfile(enabled=False),
    ),
)
```
:::

:::js
Harness 配置文件描述在聊天模型构建后应用的提示词组装、工具可见性、中间件和默认子代理调整： `createDeepAgent` 

```typescript
import { registerHarnessProfile } from "deepagents";

registerHarnessProfile("openai:gpt-5.5", {
  systemPromptSuffix: "Respond in under 100 words.",
  excludedTools: ["execute"],
  excludedMiddleware: ["SummarizationMiddleware"],
  generalPurposeSubagent: { enabled: false },
});
```
:::

:::python
<ResponseField name="base_system_prompt" type="string">
    替换基础 Deep Agents 系统提示词（`CUSTOM` in [提示词组装](/oss/deepagents/customization#prompt-assembly)).
</ResponseField>

<ResponseField name="system_prompt_suffix" type="string">
    向组装的基础提示词追加文本（`SUFFIX` in [提示词组装](/oss/deepagents/customization#prompt-assembly)）；应用于主代理、声明式子代理和自动添加的通用子代理。
</ResponseField>

<ResponseField name="tool_description_overrides" type="Mapping[str, str]">
    覆盖单个工具描述，按工具名称键控。
</ResponseField>

<ResponseField name="excluded_tools" type="frozenset[str]">
    从工具集中移除特定的 Harness 级别工具。按工具名称（字符串）匹配，作为注入后过滤器应用，因此可以删除用户提供的工具和 Harness 中间件添加的工具。参见 [在不使用默认文件系统工具的情况下运行](/oss/deepagents/harness#virtual-filesystem-access) 以获取完整示例。
</ResponseField>

<ResponseField name="excluded_middleware" type="frozenset[type[AgentMiddleware] | str]">
    从 [默认堆栈](/oss/deepagents/customization#default-stack-main-agent)中剥离特定的中间件类。接受中间件类或字符串名称。
</ResponseField>

<ResponseField name="extra_middleware" type="Sequence[AgentMiddleware] | Callable[[], Sequence[AgentMiddleware]]">
    向此配置文件应用的每个堆栈追加中间件。参见 [默认中间件堆栈](/oss/deepagents/customization#middleware) 了解内置排序。
</ResponseField>

<ResponseField name="general_purpose_subagent" type="GeneralPurposeSubagentProfile">
    禁用、重命名或重新提示通用子代理。当此字段的 `system_prompt` 与...一起设置 `base_system_prompt`，通用子代理提示词优先——见 [通用子代理提示词](/oss/deepagents/customization#general-purpose-subagent-prompt).
</ResponseField>
:::

:::js
<ResponseField name="baseSystemPrompt" type="string">
    替换基础 Deep Agents 系统提示词 (`CUSTOM` in [提示词组装](/oss/deepagents/customization#prompt-assembly)).
</ResponseField>

<ResponseField name="systemPromptSuffix" type="string">
    向已组装的基础提示词追加文本 (`SUFFIX` in [提示词组装](/oss/deepagents/customization#prompt-assembly))；应用于主代理、声明式子代理以及自动添加的通用子代理。
</ResponseField>

<ResponseField name="toolDescriptionOverrides" type="Record<string, string>">
    覆盖单个工具描述，按工具名称作为键。
</ResponseField>

<ResponseField name="excludedTools" type="string[]">
    从工具集中移除特定工具集中的工具。按工具名称匹配，作为注入后过滤器应用，因此可以捕获用户提供的和中间件提供的工具。
</ResponseField>

<ResponseField name="excludedMiddleware" type="string[]">
    从已组装的中间件栈中剥离特定中间件。按每个中间件的 `.name` 属性进行匹配。不可包含必需的脚手架名称 (`FilesystemMiddleware`, `SubAgentMiddleware`).
</ResponseField>

<ResponseField name="extraMiddleware" type="AgentMiddleware[] | (() => AgentMiddleware[])">
    在用户中间件之后追加到栈中的附加中间件。可以是静态数组，也可以是零参数工厂函数，在每次代理构建时返回新实例。
</ResponseField>

<ResponseField name="generalPurposeSubagent" type="GeneralPurposeSubagentConfig">
    禁用、重命名或重新提示通用子代理 (`enabled`, `description`, `systemPrompt`).
</ResponseField>
:::

:::python
<Note>
    Caller-supplied `system_prompt=` 始终位于已组装提示词的前端， `system_prompt_suffix` 始终位于末端——无论选择哪个模型。相同的覆盖规则适用于子代理：每个子代理根据其自己的模型重新运行配置解析。见 [提示词组装](/oss/deepagents/customization#prompt-assembly) 获取完整的逐案细分（主代理、子代理和通用子代理）。
</Note>
:::

:::js
<Note>
    Caller-supplied `systemPrompt` 始终位于已组装提示词的前端， `systemPromptSuffix` 始终位于末端——无论选择哪个模型。相同的覆盖规则适用于子代理：每个子代理根据其自己的模型重新运行配置解析。见 [提示词组装](/oss/deepagents/customization#prompt-assembly) 获取完整的逐案细分（主代理、子代理和通用子代理）。
</Note>
:::

:::python
<Warning>
    要运行一个没有 `task` 工具的代理，请参阅 [无子代理运行](/oss/deepagents/subagents#running-without-subagents) ——设置 `general_purpose_subagent=GeneralPurposeSubagentProfile(enabled=False)` 并且不通过 `subagents=`. `SubAgentMiddleware` （以及 `task` 工具）仅在存在至少一个同步子代理时才附加，因此此配置会干净地将其排除。异步子代理不受影响。

    列表 `FilesystemMiddleware`, `SubAgentMiddleware`，或内部权限中间件在 `excluded_middleware` 引发 `ValueError` ——它们是 [默认中间件栈](/oss/deepagents/customization#default-stack-main-agent)中的必需脚手架。要向模型隐藏它们的工具而不移除中间件，请使用 `excluded_tools` ——请参阅 [运行时不使用默认文件系统工具](/oss/deepagents/harness#virtual-filesystem-access).
</Warning>

中的条目 `excluded_middleware` 接受两种形式：

- 一个中间件 *类* （按精确类型匹配），或一个匹配 `AgentMiddleware.name`的纯字符串。对内置项和公共别名使用纯字符串，例如 `"SummarizationMiddleware"`.
- An `module:Class` import ref（例如， `"my_pkg.middleware:TelemetryMiddleware"`）来从配置文件定位确切的中间件类。导入引用延迟解析，因此仅将其用于可信的本地配置——加载一个会导入 Python 代码。
:::

:::js
<Warning>
    列表 `FilesystemMiddleware` or `SubAgentMiddleware` in `excludedMiddleware` 在构造时抛出——它们是必需的脚手架。要在不移除中间件的情况下隐藏其工具，请使用 `excludedTools` instead.
</Warning>
:::

<Accordion
    title="预配置模型实例的查找顺序"
>
    当你传递一个预配置的聊天模型实例而不是 `provider:model` 字符串时，测试工具会从该实例合成规范 `provider:identifier` 键，并按此顺序查找：

    1. 精确 `provider:identifier` 匹配
    2. 仅限标识符（仅当标识符已包含 `:`)
    3. 仅提供商回退
</Accordion>


## 注册键

两种配置文件类型使用相同的键格式：

- **Provider-level** ——一个裸露的提供商名称，如 `"openai"` 适用于该提供商下的所有模型。
- **Model-level** ——一个完全限定的 `provider:model` 键，如 `"openai:gpt-5.5"` 仅适用于该特定模型。

当提供商级和模型级配置文件同时存在时，它们在解析时合并。未设置的模型级字段继承自提供商级配置文件；显式的模型级值会覆盖它们。

使用现有键重新注册会将新配置文件合并到先前配置文件的顶部——它不会替换它。请参阅 [合并语义](#merge-semantics) 了解每个字段的规则。

:::python
<Note>
    没有通配符键可以匹配每个提供商。要在各处应用相同的覆盖——比如删除 `TodoListMiddleware` 无论选择哪个模型——请在你使用的每个提供商键下注册配置文件。配置文件用于依赖于所选模型的调整。无论模型如何都应应用的全局调整应在 `create_deep_agent` 调用点。
</Note>
:::

:::js
<Note>
    没有通配符键可以匹配每个提供商。要在各处应用相同的覆盖——比如删除 `TodoListMiddleware` 无论选择哪个模型——请在你使用的每个提供商键下注册配置文件。配置文件用于依赖于所选模型的调整。无论模型如何都应应用的全局调整应在 `createDeepAgent` 调用点。
</Note>
:::

## 合并语义

:::python
| 字段 | 合并行为 |
| --- | --- |
| `base_system_prompt`, `system_prompt_suffix` | 设置时新值优先；否则继承 |
| `tool_description_overrides` | 映射按键合并；共享键上新值优先 |
| `excluded_tools`, `excluded_middleware` | 集合并集 |
| `extra_middleware` | 按名称合并：新实例在其位置替换现有实例，新条目追加 |
| `general_purpose_subagent` | 按字段合并（未设置的字段继承）|
:::

:::js
| 字段 | 合并行为 |
| --- | --- |
| `baseSystemPrompt`, `systemPromptSuffix` | 设置时新值优先；否则继承 |
| `toolDescriptionOverrides` | 映射按键合并；共享键上新值优先 |
| `excludedTools`, `excludedMiddleware` | 集合并集 |
| `extraMiddleware` | 按名称合并：新实例在其位置替换现有实例，新条目追加 |
| `generalPurposeSubagent` | 按字段合并（未设置的字段继承）|
:::

:::python
| `init_kwargs` （提供商）| 字典按键合并；共享键上新值优先 |
| `pre_init` (provider) | 可调用链：现有先运行，然后是新调用 |
| `init_kwargs_factory` (provider) | 工厂链及其输出在每次 `resolve_model` 调用时合并 |
:::

## 提供者配置文件

:::python
A `ProviderProfile` 声明 Deep Agents 如何为给定提供者或特定模型规范构建聊天模型。仅当你提供 `provider:model` 字符串创建深度代理时适用，而不是当你使用 @[`init_chat_model`]:

```python
from deepagents import ProviderProfile, register_provider_profile

register_provider_profile(
    "openai",
    ProviderProfile(init_kwargs={"temperature": 0}),
)
```

<ResponseField name="init_kwargs" type="Mapping[str, Any]">
    转发给的静态初始化参数 `init_chat_model`.
</ResponseField>

<ResponseField name="pre_init" type="Callable[[str], None]">
    构建前运行副作用（例如，凭证验证）。
</ResponseField>

<ResponseField name="init_kwargs_factory" type="Callable[[], dict[str, Any]]">
    从运行时状态派生的 kwargs（例如，从环境变量中提取的请求头）。
</ResponseField>
:::

:::js
提供者配置文件（用于控制模型构建 kwargs，例如 `temperature`）是 Python 独有功能，在 TypeScript SDK 中不可用。
:::

## 从配置文件加载配置文件

:::python
For YAML/JSON-backed workflows, use `HarnessProfileConfig`。它镜像 `HarnessProfile` （提示词文本、工具描述覆盖、排除的工具和中间件、通用子代理编辑）的声明式子集，并拥有 `to_dict` / `from_dict`。仅运行时状态 — 中间件实例、工厂和类形式 `excluded_middleware` 条目 — 保留在 `HarnessProfile`.

`register_harness_profile` 接受任一类型，因此基于配置的调用方无需手动转换步骤：
:::

:::js
For YAML/JSON-backed workflows, use `parseHarnessProfileConfig`。它从具有 camelCase 键的普通对象验证和构建 `HarnessProfile` 。仅运行时状态 — `extraMiddleware` instances — cannot be represented in JSON/YAML and must be set programmatically.
:::

:::python
```yaml
# openai.yaml
base_system_prompt: You are helpful.
system_prompt_suffix: Respond briefly.
excluded_tools:
  - execute
  - grep
excluded_middleware:
  - SummarizationMiddleware
  - my_pkg.middleware:TelemetryMiddleware
general_purpose_subagent:
  enabled: false
```
:::

:::js
```yaml
# profile.yaml
baseSystemPrompt: You are helpful.
systemPromptSuffix: Respond briefly.
excludedTools:
  - execute
  - grep
excludedMiddleware:
  - SummarizationMiddleware
generalPurposeSubagent:
  enabled: false
```
:::

:::python
```python
import yaml
from deepagents import HarnessProfileConfig, register_harness_profile

with open("openai.yaml") as f:
    register_harness_profile(
        "openai",
        HarnessProfileConfig.from_dict(yaml.safe_load(f)),
    )
```

要反向操作， `HarnessProfileConfig.from_harness_profile(...)` 仅当使用可序列化功能时，将运行时配置文件导出回声明式形状：

- Class-form `excluded_middleware` 条目序列化为公共别名（当类通过 `serialized_name: ClassVar[str]`) or as a `module:Class` 导入引用公开别名时）
- Non-empty `extra_middleware` 和在 `__main__` 中声明或函数作用域内的中间件和中间件类无法序列化 — 导出抛出 `ValueError`.
:::

:::js
```typescript
import { readFileSync } from "fs";
import YAML from "yaml";
import { parseHarnessProfileConfig, registerHarnessProfile } from "deepagents";

const raw = YAML.parse(readFileSync("profile.yaml", "utf-8"));
registerHarnessProfile("openai", parseHarnessProfileConfig(raw));
```

To serialize a profile back to JSON/YAML, use `serializeProfile`:

```typescript
import { serializeProfile } from "deepagents";

const data = serializeProfile(profile); // JSON-compatible object
```

包含非空 `extraMiddleware` 的配置文件无法序列化 — `serializeProfile` 如果存在中间件实例则抛出异常。
:::

## 将配置文件作为插件发布

:::python
可分发配置文件可以通过 `importlib.metadata` 入口点自行注册，而不是要求调用方手动运行 `register_*_profile` 。加载顺序为 **内置优先，然后是入口点插件，最后是用户代码中的任何直接 `register_*_profile` 调用**；所有三条路径通过相同的累加注册汇聚，因此后续注册在同一键下层叠在早期注册之上。

在发行版的 `pyproject.toml` 下声明适当的组入口点：

```toml
[project.entry-points."deepagents.harness_profiles"]
my_provider = "my_pkg.profiles:register_harness"

[project.entry-points."deepagents.provider_profiles"]
my_provider = "my_pkg.profiles:register_provider"
```

每个目标解析为一个零参数可调用函数，该函数在导入 `deepagents.profiles` 时执行注册：

```python
from deepagents import (
    HarnessProfile,
    ProviderProfile,
    register_harness_profile,
    register_provider_profile,
)


def register_harness() -> None:
    register_harness_profile(
        "my_provider",
        HarnessProfile(system_prompt_suffix="Batch independent tool calls in parallel."),
    )


def register_provider() -> None:
    register_provider_profile(
        "my_provider",
        ProviderProfile(init_kwargs={"temperature": 0}),
    )
```
:::

:::js
插件注册系统（通过包入口点）是 Python 独有功能。在 TypeScript 中，请在应用启动时或包的初始化代码中直接调用 `registerHarnessProfile` 。
:::
