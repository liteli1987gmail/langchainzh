---
title: 计算机使用
description: 通过连接到 Fleet 代理的持久性虚拟计算机运行代码、管理文件和调用已认证的 API。
---

计算机使用功能为您的 Fleet 代理提供对隔离虚拟计算机的访问权限。代理可以编写和执行代码、管理文件、安装软件包，以及调用已认证的外部 API，而不会向语言模型暴露凭证。

<Note>
计算机使用功能适用于 [Plus 和 Enterprise 计划](https://langchain.com/pricing)。
</Note>

## 计算机模式

选择虚拟计算机如何在代理的对话线程之间共享：

| 模式 | 说明 |
|------|-------------|
| **共享计算机** | 所有线程共享一台计算机。文件系统、已安装的软件包和运行中的进程在线程之间保持一致。选择此模式可让文件、依赖项或环境配置在多个对话中累积。共享计算机不会被自动删除。 |
| **每线程一台计算机** | 每个线程获得自己独立的虚拟计算机，启动时保持全新状态，空闲时被归档。选择此模式用于软件工程代理以及其他运行许多并行、写入密集型任务的工作负载，或用于任何线程之间不应看到彼此状态的场景。 |

## 配置计算机使用

<Warning>
计算机模式在创建代理时设置，之后无法更改。若要切换模式，需创建新代理。
</Warning>

<Steps>
  <Step title="Open the Create agent dialog">
    在 [Fleet](https://smith.langchain.com/agents) 左侧导航中的 **我的代理** 下，点击 <Icon icon="plus" /> 并选择 **Create with AI** 或 **Blank agent**。为您的代理输入一个名称。
  </Step>
  <Step title="Enable computer use">
    在 **Should your agent use a computer?** 下，选择 **Yes**，然后选择 **Shared computer** 或 **Computer per thread**。如果选择 **No**（默认），则创建的代理没有计算机访问权限。
  </Step>
  <Step title="Set the base snapshot (optional)">
    展开 **Advanced** 以选择新计算机的 **Snapshot**。
  </Step>
  <Step title="Create the agent">
    点击 **Create Agent**。
  </Step>
</Steps>

## 访问配置文件

使用访问配置文件让您的代理能够调用已认证的外部 API，而无需在提示词中放入凭证或向语言模型暴露这些凭证。发往匹配主机的出站 HTTP 请求通过代理路由，代理在转发前注入配置的请求头。

配置文件包含一个或多个 **自定义规则**。每个规则指定：

- **匹配主机**：规则适用的目标主机名。使用 `*` 作为通配符（例如，`*.example.com` 匹配 `api.example.com`）。
- **源类型和提供者**：凭证来源。选择 **Connection** 用于用户委托的 OAuth，或选择 **Workspace Secret** 用于静态 API 密钥。
- **注入请求头**：代理向匹配请求添加的 HTTP 请求头。使用模板值（如 `{access_token}`）来引用凭证（例如，`Authorization: Bearer {access_token}`）。

配置文件还具有 **网络范围**，用于控制代理计算机的出站流量。默认为 **None (all traffic allowed)**。

### 添加访问配置文件

<Steps>
  <Step title="Create the access profile">
    前往 [Fleet Integrations tab](https://smith.langchain.com/agents/tools) 并导航到 **Computer** 部分。点击 **+ Create profile** 并按照提示配置主机模式和凭证。
  </Step>
  <Step title="Attach the profile to an agent">
    在代理编辑器中，点击 **Computer** 节点。点击 **Access profiles** 旁边的 **+ Add** 并选择您创建的配置文件。
  </Step>
  <Step title="Save changes">
    点击 **Save changes**。
  </Step>
</Steps>

## 计算机生命周期

每个代理有两个生命周期设置，用于控制计算机保持活动状态的时间长度以及停止后保留的时间长度。[在设置弹出窗口中配置这两项](#configure-lifecycle-and-snapshot)。

- **空闲超时**：当计算机未收到任何命令达到此时间长度时，它会暂停并且磁盘被归档。代理稍后可以恢复同一台计算机而不会丢失数据。默认值：**15 分钟**。
- **已停止计算机清理**：当计算机停止运行达到此时间长度后，它会连同所有磁盘数据被永久删除。默认值：**14 天**。

<Note>
**已停止计算机清理** 仅适用于 **每线程一台计算机** 模式。共享计算机不会被自动删除。
</Note>

## 基础快照

快照是用于启动计算机的磁盘映像。默认情况下，所有 Fleet 代理都使用工作区默认快照。若要构建、捕获或配置自定义快照，请参阅 [Sandbox snapshots](/langsmith/sandbox-snapshots)。在设置弹出窗口中[更改代理的快照](#configure-lifecycle-and-snapshot)。

<Note>
快照更改仅对新计算机生效。在 **共享计算机** 模式下，单个共享计算机会保留其原始快照。
</Note>

## 配置生命周期和快照

代理的快照、空闲超时和已停止计算机清理都在设置弹出窗口的 **Computer lifecycle** 部分设置。

<Steps>
  <Step title="Open the settings popover">
    在 [Fleet](https://smith.langchain.com/agents) 中，打开代理并在代理编辑器中点击 <Icon icon="settings" /> 设置图标。
  </Step>
  <Step title="Set the lifecycle fields">
    滚动到 **Computer lifecycle** 部分。设置 **Snapshot**、**Idle timeout**，以及对于 **Computer per thread** 模式，还要设置 **Stopped computer cleanup**。
  </Step>
  <Step title="Save changes">
    点击 **Save changes**。
  </Step>
</Steps>
