---
title: INVALID_TOOL_RESULTS
---

<Note>
    目前仅用于 `langchainjs` (JavaScript/TypeScript).
</Note>

当在工具调用操作期间传递不匹配、不足或过多的 @[`ToolMessage`] 对象给模型时会发生此错误。

此错误源于一个基本要求：包含工具调用的助手消息必须 `tool_calls` 后面必须跟随响应每个工具调用的工具消息 `tool_call_id`.

当模型返回一个 @[`AIMessage`] 且带有工具调用时，您必须为每个工具调用提供恰好一个对应的 @[`ToolMessage`]，并带有匹配的 `tool_call_id` values.

## 常见原因

* **响应不足**：如果模型请求两个工具执行但您只提供一个响应消息，模型会拒绝不完整的消息链
* **重复响应**：为同一工具调用 ID 提供多个 @[`ToolMessage`] 对象会导致拒绝，ID 不匹配也会导致拒绝
* **孤立的工具消息**：发送 @[`ToolMessage`] 而没有前面的包含工具调用的 @[`AIMessage`] 会违反协议要求

:::python
以下是问题模式的示例：

```python
# Model requests two tool calls
response_message.tool_calls  # Returns 2 calls

# But only one ToolMessage provided
chat_history.append(ToolMessage(
    content=str(tool_response),
    tool_call_id=tool_call.get("id")
))

model_with_tools.invoke(chat_history)
```
:::

:::js
以下是问题模式的示例：

```javascript
// Model requests two tool calls
responseMessage.tool_calls // Returns 2 calls

// But only one ToolMessage provided
chatHistory.push({
  role: "tool",
  content: toolResponse,
  tool_call_id: responseMessage.tool_calls[0].id
});

await modelWithTools.invoke(chatHistory); // Fails with INVALID_TOOL_RESULTS
```
:::

## 故障排除

要解决此错误：

* **计算匹配对**：确保前置 @[`ToolMessage`] 中每个工具调用对应一个 @[`AIMessage`]
* **验证 ID**：确认每个 `ToolMessage.tool_call_id` 匹配实际的工具调用标识符
