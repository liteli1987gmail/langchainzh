---
title: 如何添加自定义中间件
sidebarTitle: 添加自定义中间件
---
将智能体部署到 LangSmith 时，您可以向服务器添加自定义中间件来处理日志请求指标、注入或检查请求头以及强制执行安全策略等问题，而无需修改核心服务器逻辑。这与 [添加自定义路由](/langsmith/custom-routes)的方式相同。您只需要提供您自己的 [`Starlette`](https://www.starlette.io/applications/) 应用（包括 [`FastAPI`](https://fastapi.tiangolo.com/), [`FastHTML`](https://fastht.ml/) 和其他兼容的应用）。

添加中间件允许您在整个部署中全局拦截和修改请求和响应，无论它们是访问您的自定义端点还是内置的 LangSmith API。

以下是使用 FastAPI 的示例。

<Note>
仅限 Python
我们目前仅支持在 Python 部署中使用自定义中间件 `langgraph-api>=0.0.26`.
</Note>

## 创建应用

从一个 **现有的** LangSmith 应用开始，将以下中间件代码添加到您的 `webapp.py` 文件中。如果您从头开始，可以使用 CLI 从模板创建新应用。

```bash
langgraph new --template=new-langgraph-project-python my_new_project
```

获得 LangGraph 项目后，添加以下应用代码：

```python {highlight={5}}
# ./src/agent/webapp.py
from fastapi import FastAPI, Request
from starlette.middleware.base import BaseHTTPMiddleware

app = FastAPI()

class CustomHeaderMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        response = await call_next(request)
        response.headers['X-Custom-Header'] = 'Hello from middleware!'
        return response

# Add the middleware to the app
app.add_middleware(CustomHeaderMiddleware)
```

## 配置 `langgraph.json`

将以下内容添加到您的 `langgraph.json` 配置文件中。确保路径指向您上面创建的 `webapp.py` 文件。

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent/graph.py:graph"
  },
  "env": ".env",
  "http": {
    "app": "./src/agent/webapp.py:app"
  }
  // Other configuration options like auth, store, etc.
}
```

### 自定义中间件顺序

默认情况下，自定义中间件在身份验证逻辑之前运行。要在身份验证之后运行自定义中间件，请设置 _之后_ ，在您的 `middleware_order` to `auth_first` 配置中设置 `http` 。（此自定义功能从 API 服务器 v0.4.35 及更高版本开始支持。）

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent/graph.py:graph"
  },
  "env": ".env",
  "http": {
    "app": "./src/agent/webapp.py:app",
    "middleware_order": "auth_first"
  },
  "auth": {
    "path": "./auth.py:my_auth"
  }
}
```

## 启动服务器

在本地测试服务器：

```bash
langgraph dev --no-browser
```

现在任何对服务器的请求都将在响应中包含自定义请求头 `X-Custom-Header` 。

## 部署

您可以按原样将此应用部署到云端或您的自托管平台。

## 后续步骤

现在您已经将自定义中间件添加到部署中，可以使用类似的技术添加 [自定义路由](/langsmith/custom-routes) 或定义 [自定义生命周期事件](/langsmith/custom-lifespan) 来进一步自定义服务器的行为。
