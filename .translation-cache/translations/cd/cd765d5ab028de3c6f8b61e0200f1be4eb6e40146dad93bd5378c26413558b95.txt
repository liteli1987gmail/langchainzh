## 设置

要访问 Google Generative AI 嵌入模型，您需要注册 Google AI 账户、获取 API 密钥，并安装 `@langchain/google-genai` 集成包。

### 凭据

在此处获取 API 密钥：[ai.google.dev/tutorials/setup](https://ai.google.dev/tutorials/setup)。

接下来，将您的密钥设置为名为 `GOOGLE_API_KEY` 的环境变量：

```bash
export GOOGLE_API_KEY="your-api-key"
```

如果您想获取模型调用的自动追踪，您也可以通过取消下面代码的注释来设置您的 [LangSmith](/langsmith/observability) API 密钥：

```bash
# export LANGSMITH_TRACING="true"
# export LANGSMITH_API_KEY="your-api-key"
```

### 安装

LangChain `GoogleGenerativeAIEmbeddings` 集成位于 `@langchain/google-genai` 包中。您可能还需要安装官方 SDK：

<CodeGroup>

</CodeGroup>
```bash npm
npm install @langchain/google-genai @langchain/core @google/generative-ai
```
```bash yarn
yarn add @langchain/google-genai @langchain/core @google/generative-ai
```
```bash pnpm
pnpm add @langchain/google-genai @langchain/core @google/generative-ai
```
</CodeGroup>

## 实例化

现在我们可以实例化模型对象并嵌入文本：

```typescript
import { GoogleGenerativeAIEmbeddings } from "@langchain/google-genai";
import { TaskType } from "@google/generative-ai";

const embeddings = new GoogleGenerativeAIEmbeddings({
  model: "gemini-embedding-001", // 768 dimensions
  taskType: TaskType.RETRIEVAL_DOCUMENT,
  title: "Document title",
});
```

## 索引和检索

嵌入模型常用于检索增强生成（RAG）流程中，既用于索引数据，也用于后续检索数据。如需更详细的说明，请参阅[**学习**标签页](/oss/learn/)下的 RAG 教程。

下方展示了如何使用上面初始化的 `embeddings` 对象来索引和检索数据。在此示例中，我们将使用示例 [`MemoryVectorStore`](/oss/integrations/vectorstores/memory) 来索引和检索示例文档。

```typescript
// Create a vector store with a sample text
import { MemoryVectorStore } from "@langchain/classic/vectorstores/memory";

const text = "LangChain is the framework for building context-aware reasoning applications";

const vectorstore = await MemoryVectorStore.fromDocuments(
  [{ pageContent: text, metadata: {} }],
  embeddings,
);

// Use the vector store as a retriever that returns a single document
const retriever = vectorstore.asRetriever(1);

// Retrieve the most similar text
const retrievedDocuments = await retriever.invoke("What is LangChain?");

retrievedDocuments[0].pageContent;
```

```text
LangChain is the framework for building context-aware reasoning applications
```

## 直接使用

在底层，向量存储和检索器实现调用 `embeddings.embedDocument(...)` 和 `embeddings.embedQuery(...)` 来分别为 `fromDocuments` 中使用的文本和检索器的 `invoke` 操作创建嵌入。

您可以直接调用这些方法来为自己的用例获取嵌入。

### 嵌入单个文本

您可以使用 `embedQuery` 来嵌入搜索查询。这会生成一个针对该查询的向量表示：

```typescript
const singleVector = await embeddings.embedQuery(text);

console.log(singleVector.slice(0, 100));
```

### 嵌入多个文本

你可以使用 `embedDocuments` 嵌入多个文本进行索引。此方法使用的内部实现可能（但不一定）与嵌入查询不同：

```typescript
const text2 = "LangGraph is a library for building stateful, multi-actor applications with LLMs";

const vectors = await embeddings.embedDocuments([text, text2]);

console.log(vectors[0].slice(0, 100));
console.log(vectors[1].slice(0, 100));
```
