<Note>
**私人测试版:** LLM Gateway 处于私人[测试阶段](/langsmith/release-stages)。[报名加入候补名单](https://www.langchain.com/langsmith-llm-gateway-waitlist)以获取访问权限。
</Note>

当 PII 或敏感信息脱敏策略处于活动状态时,网关会在请求到达 LLM 提供商之前扫描出站请求。如果检测到敏感数据,将从请求中脱敏。智能体将继续收到响应。

脱敏内容也会在 LangSmith 跟踪中被脱敏,因此敏感数据也不会保留在您的可观测性数据中。

## PII 检测

网关检测并脱敏以下类别的个人身份信息:

| 类别 | 示例 |
| --- | --- |
| **姓名** | 自然语言中的个人姓名 |
| **国籍、宗教或政治归属** | 国籍 |
| **位置** | 地址、城市、国家 |

检测使用 Presidio 识别命名实体(姓名、位置和 NRP),并使用基于模式的规则检测结构化标识符。

## 敏感信息检测

网关检测并脱敏跨多种提供商和格式的 API 密钥、令牌和凭据:

| 类别 | 检测到的模式 |
| --- | --- |
| **社会安全号码** | 美国 SSN 模式(例如,123-45-6789) |
| **电话号码** | 美国电话号码模式 |
| **LangSmith** | 个人令牌、服务密钥 |
| **AWS** | 访问令牌 |
| **GitHub** | 个人访问令牌、细粒度 PAT、OAuth 令牌、应用令牌 |
| **GitLab** | 个人访问令牌 |
| **AI 提供商** | OpenAI API 密钥、Anthropic API 密钥 |
| **云平台** | GCP API 密钥、Azure AD 客户端密钥 |
| **协作工具** | Slack 机器人/用户/应用令牌、Datadog 访问令牌 |
| **包注册表** | PyPI 上传令牌、npm 访问令牌 |
| **加密** | 私钥 |
| **Stripe** | 访问令牌 |

## 启用脱敏策略

<Warning>
创建和管理策略需要 `organization:manage` 权限。
</Warning>

1. 进入**设置 → 网关 → LLM Gateway**。
1. 点击**创建策略**。
1. 选择**PII 脱敏**或**敏感信息脱敏**作为策略类型。
1. 配置要检测的类别(或启用所有)。
1. 保存。

脱敏策略适用于在其配置范围内通过网关的所有请求。它们会立即生效。

## 脱敏内容的外观

当检测到 PII 或敏感信息时,内容将在发送给提供商的请求和 LangSmith 跟踪中被替换为占位符。例如:

**原始请求:**

```
Please process the refund for John Smith, SSN 123-45-6789.
```

**上游脱敏：**

```
Please process the refund for [SAFE_TO_USE:PERSON_kbqdjxyz], SSN [SAFE_TO_USE:US_SSN_abqxlmwp]
```

占位符遵循以下格式 `[SAFE_TO_USE:<CATEGORY>_<suffix>]`：

- **SAFE_TO_USE:** 固定前缀，标记该值为脱敏占位符。
- **\<CATEGORY\>:** 检测到的类型。例如：`PERSON`、`LOCATION`、`US_SSN`、`US_PHONE_NUMBER`、`OPENAI_API_KEY`、`GITHUB_PAT`、`LANGSMITH_PERSONAL_TOKEN`。
- **\<suffix\>:** 8 字符的随机标签。

LangSmith 中的追踪记录会显示脱敏后的版本，并附带元数据，指示已发生脱敏以及检测到的类别。

**下游去脱敏响应：**

当上游提供商返回响应时，网关将用调用者的原始值替换脱敏占位符。例如，您的智能体可能会看到如下响应：

```
Checking Confirming John Smith's SSN to be 123-45-6789.... Okay! I will process the full refund.
```

## 脱敏覆盖范围

**覆盖范围：**

- 出站请求内容（发送到 LLM 提供商的消息）在离开网关前会被扫描和脱敏。
- 脱敏后的版本会出现在 LangSmith 追踪中。

**不覆盖范围：**

- **LLM 提供商的响应：**如果模型在其响应中生成了敏感数据，该内容不会被脱敏。流式响应脱敏功能正在开发中。
- **已存在于追踪中的数据：**脱敏仅适用于流经网关的请求。直接写入 LangSmith API 的追踪（绕过网关）不会被扫描。
- **平台级数据摄取：**如果您的要求是防止 PII 以任何方式进入 LangSmith（例如数据主权合规），仅靠网关脱敏是不够的。这需要摄取级脱敏，这是一个独立的功能。
- **提示词扫描：**系统提示词、开发者提示词和工具调用参数不会被扫描。
**扫描器故障采用故障关闭策略**：如果 PII 或密钥扫描器无法访问、运行缓慢或出现错误，该阶段会阻止请求继续进行。

此区别至关重要。如果您的安全模型要求敏感数据永远不会到达任何系统（不仅仅是 LLM 提供商），请确保您了解网关覆盖的范围以及哪些范围需要额外的控制。
