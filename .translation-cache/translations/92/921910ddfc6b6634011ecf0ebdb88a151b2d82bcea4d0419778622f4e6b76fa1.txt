<Info>
自托管 LangSmith 是企业计划的附加功能，专为我们最大型、最注重安全性的客户设计。有关更多详细信息，请参阅我们的[定价页面](https://www.langchain.com/pricing)，如果您想在您的环境中试用 LangSmith 获取许可证密钥，请[联系我们的销售团队](https://www.langchain.com/contact-sales)。
</Info>

本页介绍如何在 Kubernetes 集群中设置 **LangSmith**（可观测性、追踪和评估）。您将使用 Helm 来安装 LangSmith 及其依赖项。

完成本页后，您将拥有：

- **LangSmith UI 和 API**：用于[可观测性](/langsmith/observability)、追踪和[评估](/langsmith/evaluation)。
- **后端服务**：（队列、playground、ACE）。
- **数据存储**：（PostgreSQL、Redis、ClickHouse、可选的对象存储）。

关于[代理部署](/langsmith/deployment)：要添加部署能力，请先完成本指南，然后按照[启用 LangSmith 部署、Fleet、Insights 和 Chat](/langsmith/deploy-self-hosted-full-platform)进行操作。

LangChain 已成功在以下 Kubernetes 发行版上测试 LangSmith：

- Google Kubernetes Engine (GKE)
- Amazon Elastic Kubernetes Service (EKS)：有关架构模式和最佳实践，请参阅[在 AWS 上自托管](/langsmith/aws-self-hosted)。
- Azure Kubernetes Service (AKS)：有关架构模式和最佳实践，请参阅[在 AKS 上自托管](/langsmith/azure-self-hosted)。
- OpenShift (4.14+)
- Minikube 和 Kind（用于开发目的）

<Note>
LangChain 提供 Terraform 模块来帮助为 LangSmith 配置基础设施。这些模块可以快速设置 Kubernetes 集群、存储和网络以进行部署。

可用模块：
- [AWS Terraform 模块](https://github.com/langchain-ai/terraform/tree/main/modules/aws)
- [Azure Terraform 模块](https://github.com/langchain-ai/terraform/tree/main/modules/azure)

查看[完整的 Terraform 仓库](https://github.com/langchain-ai/terraform)获取文档和其他资源。
</Note>

## 前置条件

确保您已准备好以下工具/项目。有些项目标记为可选：

1. LangSmith 许可证密钥

   1. 您可以从 LangChain 代表处获取此密钥。如需更多信息，请[联系我们的销售团队](https://www.langchain.com/contact-sales)。

2. Api Key Salt

   1. 这是一个您可以生成的密钥。它应该是一个随机字符串。
   2. 您可以使用以下命令生成它：

   ```bash
   openssl rand -base64 32
   ```

3. JWT 密钥（可选，但用于基本身份验证）

   1. 这是一个可以生成的密钥。它应该是一个随机的字符串。
   2. 您可以使用以下命令生成此密钥：

   ```bash
   openssl rand -base64 32
   ```

### 数据库

LangSmith 使用 PostgreSQL 数据库、Redis 缓存和 ClickHouse 数据库来存储追踪记录。默认情况下，这些服务会安装在你的 Kubernetes 集群内部。但是，我们强烈建议使用外部数据库。对于 PostgreSQL 和 Redis，最佳选择是你云提供商的托管服务。

有关更多信息，请参阅以下外部服务设置指南：

- [PostgreSQL](/langsmith/self-host-external-postgres)
- [Redis](/langsmith/self-host-external-redis)
- [ClickHouse](/langsmith/self-host-external-clickhouse)

有关每个数据存储的最低支持版本，请参阅[自托管依赖项最低版本](/langsmith/self-host-dependency-versions)。

### Kubernetes 集群要求

1. 你需要一个可以通 `kubectl` 访问的工作 Kubernetes 集群。你的集群应满足以下最低要求：

   1. 推荐：至少 16 vCPU、64GB 内存可用

      * 你可能需要根据组织规模/使用情况调整所有不同服务的资源请求/限制。你可以在[自托管扩展指南](/langsmith/self-host-scale)中找到我们的建议。
      * 我们建议使用集群自动扩缩容器来处理基于资源使用情况的节点扩缩容。
      * 我们建议设置指标服务器，以便可以开启自动扩缩容。
      * 如果你在集群内运行 ClickHouse，你必须有一个至少 4 vCPU 和 16GB 内存**可分配**的节点，因为 ClickHouse 默认会请求此数量的资源。

   2. 有效的动态 PV 配置器或集群上有可用的 PV（仅在集群内运行数据库时才需要）

      * 为了启用持久化，我们将尝试为任何在集群内运行的数据库配置卷。
      * 如果在集群中使用 PV，我们强烈建议在生产环境中设置备份。
      * **我们强烈建议使用由 SSD 支持的存储类以获得更好的性能。我们建议 7000 IOPS 和 1000 MiB/s 吞吐量。**
      * 在 EKS 上，你可能需要确保已安装并配置了 `ebs-csi-driver` 以进行动态配置。有关更多信息，请参阅 [EBS CSI Driver 文档](https://docs.aws.amazon.com/eks/latest/userguide/ebs-csi.html)。

      你可以通过运行以下命令来验证这一点：

      ```bash
      kubectl get storageclass
      ```

输出应至少显示一个存储类，其制备器支持动态制备。例如：

      ```bash
      NAME            PROVISIONER                 RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE
      gp2 (default)   ebs.csi.eks.amazonaws.com   Delete          WaitForFirstConsumer   true                   161d
      ```

<Note>
我们强烈建议使用支持卷扩展的存储类。这是因为跟踪数据可能需要大量磁盘空间，您的卷可能需要随时间调整大小。
</Note>

有关存储类的更多信息，请参阅 [Kubernetes 文档](https://kubernetes.io/do/langsmith/observability-concepts/storage/storage-classes/)。

2. Helm

   1. 要安装 `helm`，请参阅 [Helm 文档](https://helm.sh/docs/intro/install/)

3. 出口流量到 `https://beacon.langchain.com`（如果不处于离线模式）

   1. LangSmith 需要出口流量到 `https://beacon.langchain.com` 以进行许可证验证和使用情况报告。这是 LangSmith 正常运行的必要条件。您可以在[出口流量](/langsmith/self-host-egress)部分找到有关出口要求的更多信息。

## 配置您的 Helm charts：

1. 使用上一步中的配置选项创建一个名为 `langsmith_config.yaml` 的新文件。
   1. 您可以在 `langsmith_config.yaml` 文件中设置多个配置选项。您可以在[配置](/langsmith/self-hosted)部分找到有关特定配置选项的更多信息。
   2. 如果您是 Kubernetes 或 Helm 的新手，我们建议从 Helm Chart 仓库的 examples 目录中的一个示例配置开始：[LangSmith helm chart examples](https://github.com/langchain-ai/helm/tree/main/charts/langsmith/examples)。
   3. 您可以在 Helm Chart 仓库的 `values.yaml` 文件中查看完整的配置选项列表：[LangSmith Helm Chart](https://github.com/langchain-ai/helm/tree/main/charts/langsmith/values.yaml)

<Warning>
只覆盖 `langsmith_config.yaml` 中您需要设置的配置项；不要复制整个 `values.yaml`。
保持配置最小化可确保您继续从 Helm chart 中继承新的默认值和升级。
</Warning>

<Tip>
如果您的集群强制执行非 root 或只读容器策略，请从[只读 Helm 配置示例](https://github.com/langchain-ai/helm/blob/main/charts/langsmith/examples/read_only_config.yaml)开始。LangSmith 容器不需要 root 权限。该示例展示了如何设置 `runAsNonRoot`、服务 UID 和 GID、`fsGroup`、`RuntimeDefault` seccomp 配置、删除的 capabilities、禁用的 privilege escalation，以及需要临时存储的服务可写的 `emptyDir` 挂载。
</Tip>

2. 至少需要设置以下配置选项（使用基本认证）：

<Warning>
仅设置一次 `apiKeySalt`，之后不要更改。此值用于对所有 API 密钥进行静态哈希。更改它将永久失效您组织中的所有现有 API 密钥，要求所有用户重新生成其密钥。
</Warning>

   ```yaml
   config:
     langsmithLicenseKey: "<your license key>"
     apiKeySalt: "<your api key salt>"
     authType: mixed
     initialOrgAdminEmail: "admin@langchain.dev" # Change this to your admin email address
     basicAuth:
       enabled: true
       initialOrgAdminPassword: "secure-password" # Must be at least 12 characters long and have at least one lowercase, uppercase, and symbol
       jwtSecret: <your jwt salt> # A random string of characters used to sign JWT tokens for basic auth.
   ```
您还需要为所使用的任何外部数据库指定连接详情。

## 部署到 Kubernetes：

1. 验证您可以连接到 Kubernetes 集群（注：我们强烈建议安装到空的命名空间中）

   1. 运行 `kubectl get pods`

      输出应类似于：

   ```

      ```bash
        langsmith-eks-2vauP7wf 21:07:46 No resources found in default namespace.
      ```

<Note>
如果您使用的不是默认命名空间，则需要使用 `-n <namespace>` 标志在 `helm` 和 `kubectl` 命令中指定命名空间。
</Note>

2. 确保您已添加 LangChain Helm 仓库。（如果您使用的是本地 charts，请跳过此步骤）

   ```bash
   helm repo add langchain https://langchain-ai.github.io/helm
   ```

3. 找到 chart 的最新版本。您可以在 [Helm Chart 仓库](https://github.com/langchain-ai/helm/releases) 中找到可用的版本。

   * 我们通常建议使用最新版本。
   * 您也可以运行 `helm search repo langchain/langsmith --versions` 来查看可用的版本。输出将类似于：

   ```
    langchain/langsmith              	0.13.0      	0.13.1    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.34      	0.12.73    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.33      	0.12.72    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.32      	0.12.70    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.31      	0.12.69    	Helm chart to deploy the langsmith application ...
   ```

4. 运行 `helm upgrade -i langsmith langchain/langsmith --values langsmith_config.yaml --version <version> -n <namespace> --wait --debug`

   * 将 `<namespace>` 替换为您想要部署 LangSmith 的命名空间。
   * 将 `<version>` 替换为您想要安装的 LangSmith 版本，可参考上一步骤获取可用版本。大多数用户应安装最新可用版本。

   `helm install` 命令成功运行并完成后，您应该会看到类似的输出：

   ```
   NAME: langsmith
   LAST DEPLOYED: Fri Sep 17 21:08:47 2021
   NAMESPACE: langsmith
   STATUS: deployed
   REVISION: 1
   TEST SUITE: None
   ```

这可能需要几分钟才能完成，因为它将创建多个 Kubernetes 资源并运行多个作业来初始化数据库和其他服务。

5. 运行 `kubectl get pods`，输出现在应该类似于以下内容（请注意，确切的 Pod 名称可能因您使用的版本和配置而异）：

   ```
    langsmith-ace-backend-98fbd468c-x9gjl         1/1     Running   0
    langsmith-backend-84999bbcb7-dfhml            1/1     Running   0
    langsmith-clickhouse-0                        1/1     Running   0
    langsmith-frontend-79bdcbccc6-r7pt7           1/1     Running   0
    langsmith-ingest-queue-cbb67748-8rl8x         1/1     Running   0
    langsmith-platform-backend-586bd9d97c-2g5mv   1/1     Running   0
    langsmith-playground-859d44b46c-fjqjh         1/1     Running   0
    langsmith-postgres-0                          1/1     Running   0
    langsmith-queue-7bd6cb8b9b-bmvxm              1/1     Running   0
    langsmith-redis-0                             1/1     Running   0
   ```

## 使用 LangSmith

现在 LangSmith 已经运行，您可以开始使用它来追踪您的代码。您可以在[自托管使用指南](/langsmith/self-hosted)中找到更多关于如何使用自托管 LangSmith 的信息。

您的 LangSmith 实例现在正在运行，但可能尚未完全设置。

如果您使用了基本配置之一，将为您创建一个默认的管理员用户账户。您可以使用在 `langsmith_config.yaml` 文件中指定的电子邮件地址和密码登录。

作为下一步，强烈建议您与基础设施管理员合作：

* 为您的 LangSmith 实例设置 DNS，以便于访问
* 配置 SSL 以确保提交到 LangSmith 的追踪数据的传输加密
* 使用[单点登录](/langsmith/self-host-sso)配置 LangSmith，以保护您的 LangSmith 实例
* 将 LangSmith 连接到外部 Postgres 和 Redis 实例
* 设置 [Blob Storage](/langsmith/self-host-blob-storage) 用于存储大文件

查看我们的[配置部分](/langsmith/self-hosted)，了解有关如何配置这些选项的更多信息。

## 启用 LangSmith Deployment、Fleet、Insights 和 Chat

要超越可观测性、追踪和评估，您可以在自托管实例上启用以下功能：

- **[LangSmith Deployment](/langsmith/deployment)**：通过 LangSmith UI 部署、扩展和管理代理。
- **[Fleet](/langsmith/fleet/index)**：无需编写代码即可创建和管理 AI 代理。
- **[Insights](/langsmith/insights)**：获取由 AI 驱动的追踪数据和应用程序数据分析。
- **[Chat](/langsmith/chat)**：跨 LangSmith 的工作区内聊天体验，帮助您分析追踪、线程、提示和实验结果。

按照[启用 LangSmith Deployment、Fleet、Insights 和 Chat](/langsmith/deploy-self-hosted-full-platform)指南设置这些组件。
