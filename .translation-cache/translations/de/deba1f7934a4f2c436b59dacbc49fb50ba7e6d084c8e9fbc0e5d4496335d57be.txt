## 设置与安装

要使用此功能，请安装 `@sap/hana-langchain` 包及其对等依赖项：

```bash
npm install @sap/hana-langchain @langchain/core@latest @langchain/classic@latest langchain@latest
```

创建与您的 SAP HANA Cloud 实例的连接

```typescript
import * as dotenv from 'dotenv';
// Load environment variables if needed
dotenv.config();

import hanaClient from "@sap/hana-client";

const connectionParams = {
  host: process.env.HANA_DB_ADDRESS,
  port: process.env.HANA_DB_PORT,
  user: process.env.HANA_DB_USER,
  password: process.env.HANA_DB_PASSWORD,
};
const client = hanaClient.createConnection(connectionParams);

// connect to hanaDB
await new Promise<void>((resolve, reject) => {
  client.connect((err: Error) => {
    // Use arrow function here
    if (err) {
      reject(err);
    } else {
      console.log("Connected to SAP HANA successfully.");
      resolve();
    }
  });
});
```

`HanaSparqlQAChain` 将以下内容整合在一起：

1. **支持模式的 SPARQL 生成**
2. **针对 SAP HANA 的查询执行**
3. **自然语言答案格式化**

## 初始化

* 一个 **LLM** 用于生成和解释查询
* 一个 **`HanaRdfGraph`**（包含连接、`graph_uri` 和本体）

请按照此处 [HanaRdfGraph](/oss/integrations/graphs/sap_hana_rdf_graph) 的步骤操作，详细了解如何创建 `HanaRdfGraph` 实例。

导入 `HanaSparqlQAChain`

```typescript
import { HanaSparqlQAChain } from "@sap/hana-langchain";
const qaChain = HanaSparqlQAChain.fromLLM({ llm, graph, allowDangerousRequests: true, verbose: true });
```

## 流水线概述

1. SPARQL 生成
   - 使用 SPARQL 生成提示词
   - 输入：`schema`（来自 `graph.getSchema()` 的 Turtle 格式）、`prompt`（用户问题）
2. 查询后处理
   - 从模型输出中提取 SPARQL
   - 如果缺失则注入 `FROM <graph_uri>`
   - 确保常用前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）
3. 执行
   - `graph.query(generatedSparql)`
4. 答案生成
   - 使用问答提示词，输入：`context`（结果）、`prompt`（原始问题）

## 提示词模板

### "SPARQL 生成"提示词

`sparqlGenerationPrompt` 用于指导 LLM 根据用户问题和提供的架构生成 SPARQL 查询。

### 问答提示词

`qaPrompt` 指示 LLM 仅根据数据库结果生成自然语言答案。
默认提示词可在此处找到：[`prompts.ts`](https://github.com/SAP/langchainjs-integration-for-sap-hana-cloud/blob/main/src/chains/graph_qa/prompts.ts)

## 自定义提示词

您可以在初始化时覆盖默认值：

```typescript
const qaChain = HanaSparqlQAChain.fromLLM({
  llm,
  graph,
  allowDangerousRequests: true,
  verbose: true,
  sparqlGenerationPrompt: YOUR_SPARQL_PROMPT,
  qaPrompt: YOUR_QA_PROMPT,
});
```

> - `sparql_generation_prompt` 必须有输入变量：`["schema", "prompt"]`
> - `qa_prompt` 必须有输入变量：`["context", "prompt"]`

## 示例：针对"电影"知识图谱的问答

**前提条件**：
您必须拥有启用了 **triple store** 功能的 SAP HANA Cloud 实例。
有关详细说明，请参阅：[启用 Triple Store](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/enable-triple-store/)<br />
加载 `kgdocu_movies` 示例数据。请参阅[知识图谱示例](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/knowledge-graph-example)。

下面我们将：

1. 实例化指向"movies"数据图的 `HanaRdfGraph`
2. 使用 LLM 驱动的 `HanaSparqlQAChain` 包装它
3. 提出自然语言问题并打印出链的响应

这展示了 LLM 如何在后台生成 SPARQL，针对 SAP HANA 执行，并返回人类可读的答案。

首先，创建到 SAP HANA Cloud 实例的连接。

```typescript

// connect to hanaDB

  client.connect((err: Error) => {

      reject(err);
    } else {
```

然后，设置知识图谱实例

```typescript
import { HanaRdfGraph, HanaRdfGraphOptions } from "@sap/hana-langchain";

const graphOptions: HanaRdfGraphOptions = {
  connection: client,
  graphUri: "kgdocu_movies",
  autoExtractOntology: true,
};

// create a Graph instance from a source URI
const graph = new HanaRdfGraph(graphOptions);

// need to initialize once an instance is created.
await graph.initialize(graphOptions);
```

```typescript
// Serialise the graph schema (optional)
```

```typescript
  debug: true,

// const query = "Which movies are in the data?"

const response = await chain.invoke({ query });
```

## 底层发生了什么？

1. **SPARQL 生成**
   该链使用您的 Turtle 格式本体（`graph.getSchema()`）和用户的问题，通过 `SPARQL_GENERATION_SELECT_PROMPT` 调用 LLM。然后 LLM 会根据您的架构发出有效的 `SELECT` 查询。
2. **预处理与执行**
   * **提取与清洗**：从 LLM 的响应中提取原始 SPARQL 文本。
   * **注入图上下文**：如果缺少则添加 `FROM <graph_uri>`，并确保声明常用前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）。
   * **在 HANA 上运行**：通过 `HanaRdfGraph.query()` 在您的命名图上执行最终确定的查询。
3. **答案组织**
   返回的 CSV（或 Turtle）结果再次输入到 LLM——这次使用 `SPARQL_QA_PROMPT`。LLM 会严格根据检索到的数据生成简洁、易读的答案，不会产生幻觉。
