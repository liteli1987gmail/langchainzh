为使用 `createAgent`创建的代理构建丰富的交互式前端。这些
模式涵盖了从基本消息渲染到高级工作流的所有内容
如人在回路审批、排队提交、持久流重新连接和
时间旅行调试。

LangChain 前端 SDK 专为 **代理应用**而构建，不仅用于
令牌流聊天机器人。相同的钩子不仅渲染消息，还暴露了
代理的持久线程状态、工具调用生命周期、中断、检查点
历史记录和自定义状态值，使您的 UI 可以成为
长期运行的代理工作的控制面板。

<Note>
这些模式使用 v1 前端 SDK 包。如果您使用的是早期版本，请参阅 [React](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-react/docs/v1-migration.md), [Vue](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-vue/docs/v1-migration.md), [Svelte](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-svelte/docs/v1-migration.md)、 [Angular](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-angular/docs/v1-migration.md).
</Note>

每个模式遵循相同的架构： `createAgent` 后端通过 SDK 流 API 将状态流式传输到前端。

在后端， `createAgent` 生成一个编译后的 LangGraph 图，暴露流式 API。在前端，流处理器连接到该 API 并提供响应式状态——消息、工具调用、中断、值和线程元数据——您可以使用任何框架进行渲染。

## 为什么使用 LangChain 前端 SDK？

大多数 AI UI 库帮助您将流式文本附加到聊天记录中。
LangChain 的 SDK 暴露了生产代理更丰富的运行时语义
need:

| 功能 | 它在您的 UI 中实现的功能 |
| --- | --- |
| **持久线程** | 重新加载页面、切换设备或重新加入运行而不会丢失对话状态。 |
| **类型化代理状态** | 渲染任意状态键，而不仅仅是消息：待办事项、管道输出、引用、沙箱文件、指标或自定义业务对象。 |
| **工具调用生命周期** | 将待处理、已完成和失败的工具调用显示为专用 UI 卡片，而不是原始 JSON。 |
| **中断** | 暂停执行以进行人工审批、编辑或补充缺失信息，然后从代理停止的确切位置恢复。 |
| **检查点** | 从持久化状态快照构建编辑、重试、分支、审计和时间旅行流程。 |
| **嵌套执行** | 可视化深度代理、子代理和图节点，而不会将所有内容扁平化为一条难以阅读的流。 |
| **框架原生响应式** | 在 React、Vue、Svelte 或 Angular 中使用相同协议，同时保持惯用的钩子、组合式、存储或信号。 |

这些原语让您能够设计用户可以检查、引导、暂停、
恢复和分叉正在运行的代理工作的 UI。

React、Vue 和 Svelte 使用 `useStream`。Angular 使用 `injectStream`:

```ts
import { useStream } from "@langchain/react";      // React
import { useStream } from "@langchain/vue";        // Vue
import { useStream } from "@langchain/svelte";     // Svelte
import { injectStream } from "@langchain/angular"; // Angular
```

## 类型推断

向 @[`useStream`] (or @[`injectStream`]（Angular 中）传递类型参数，以实现对 `stream.messages`, `stream.toolCalls`, `stream.interrupt`, `stream.values` 及其他响应式状态的安全类型访问。

:::python

定义一个 TypeScript 接口，使其与你的 agent 状态模式匹配，并将其作为类型参数传递：

```ts
import type { BaseMessage } from "langchain";

interface AgentState {
  messages: BaseMessage[];
}

const stream = useStream<AgentState>({
  apiUrl: "http://localhost:2024",
  assistantId: "agent",
});
```

使用 `langgraph.json` as `assistantId`中的图名称。在本指南的模式示例中，将 `typeof myAgent` 替换为你的接口名称（例如， `AgentState`).

如果你的 agent 暴露了自定义状态键，请扩展该接口：

```ts
import type { BaseMessage, Todo } from "langchain";

interface AgentState {
  messages: BaseMessage[];
  todos: Todo[];
}
```

:::

:::js

导入你的 agent 并将 `typeof myAgent` 作为类型参数传递。TypeScript 会从编译后的图中推断出状态模式：

```ts
import type { myAgent } from "./agent";

const stream = useStream<typeof myAgent>({
  apiUrl: "http://localhost:2024",
  assistantId: "agent",
});
```

自定义状态键会被自动推断，无需手动定义接口。

:::

## 集成

流式 API 与 UI 无关。可与任何组件库或生成式 UI 配合使用
框架。组件库可以拥有表示层，而 LangChain 的
SDK 拥有代理运行时状态、可恢复性、中断和检查点
底层语义。
