为使用 `createAgent` 创建的代理构建丰富的交互式前端。这些模式涵盖从基本消息渲染到高级工作流（如人机交互审批、队列提交、持久流重连和时间旅行调试）的一切。

LangChain 前端 SDK 专为**代理应用**而构建，而非仅用于 token 流式聊天机器人。同一渲染消息的 hook 也会暴露代理的持久线程状态、工具调用生命周期、中断、检查点历史记录和自定义状态值，使您的 UI 能够成为长时间运行的代理工作的控制平面。

<Note>
这些模式使用 v1 前端 SDK 包。如果您使用的是早期版本，请参阅 [React](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-react/docs/v1-migration.md)、[Vue](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-vue/docs/v1-migration.md)、[Svelte](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-svelte/docs/v1-migration.md) 和 [Angular](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-angular/docs/v1-migration.md) 的迁移指南。
</Note>

## 架构

每种模式都遵循相同的架构：`createAgent` 后端通过 SDK 流式 API 向前端流式传输状态。

在后端，`createAgent` 会生成一个编译后的 LangGraph 图，该图暴露了一个流式 API。在前端，流处理器连接到该 API 并提供响应式状态——消息、工具调用、中断、值和线程元数据——你可以使用任何框架来渲染这些内容。

## 为什么要使用 LangChain 前端 SDK？

大多数 AI UI 库帮助你将流式文本追加到聊天记录中。
LangChain 的 SDK 暴露了生产级代理所需的高级运行时语义：

| 功能 | 在 UI 中的作用 |
| --- | --- |
| **持久化线程** | 重新加载页面、切换设备或重新加入运行，而不会丢失对话状态。 |
| **类型化代理状态** | 渲染任何状态键，而不仅仅是消息：待办事项、管道输出、引用、沙盒文件、指标或自定义业务对象。 |
| **工具调用生命周期** | 将待处理、已完成和失败的工具调用显示为专用 UI 卡片，而不是原始 JSON。 |
| **中断** | 暂停执行以等待人工批准、编辑或补充缺失信息，然后从代理停止的精确位置恢复。 |
| **检查点** | 从持久化的状态快照构建编辑、重试、分支、审计和时间回溯流程。 |
| **嵌套执行** | 可视化深度代理、子代理和图节点，而不会将所有内容扁平化为一条难以阅读的流。 |
| **框架原生响应式** | 在 React、Vue、Svelte 或 Angular 中使用相同的协议，同时保持惯用的 hooks、composables、stores 或 signals。 |

这些原语让你能够设计用户界面，在代理工作进行时检查、引导、暂停、恢复和分叉。

React、Vue 和 Svelte 使用 `useStream`。Angular 使用 `injectStream`：

```ts
import { useStream } from "@langchain/react";      // React
import { useStream } from "@langchain/vue";        // Vue
import { useStream } from "@langchain/svelte";     // Svelte
import { injectStream } from "@langchain/angular"; // Angular
```

## 类型推断

向 @[`useStream`]（或在 Angular 中向 @[`injectStream`]）传递类型参数，以获得对 `stream.messages`、`stream.toolCalls`、`stream.interrupt`、`stream.values` 等响应式状态进行类型安全访问的能力。

:::python

定义一个 TypeScript 接口来匹配您的 agent 状态 schema，并将其作为类型参数传递：

```ts
import type { BaseMessage } from "langchain";

interface AgentState {
  messages: BaseMessage[];
}

const stream = useStream<AgentState>({
  apiUrl: "http://localhost:2024",
  assistantId: "agent",
});
```

使用 `langgraph.json` 中的图形名称作为 `assistantId`。在本指南的模式示例中，将 `typeof myAgent` 替换为您的接口名称（例如 `AgentState`）。

如果您的代理暴露了自定义状态键，请扩展该接口：

```ts
import type { BaseMessage, Todo } from "langchain";

interface AgentState {
  messages: BaseMessage[];
  todos: Todo[];
}
```

:::

:::js

导入你的 agent 并将 `typeof myAgent` 作为类型参数传递。TypeScript 会从编译后的图中推断状态模式：

:::

```ts
import type { myAgent } from "./agent";

const stream = useStream<typeof myAgent>({
  apiUrl: "http://localhost:2024",
  assistantId: "agent",
});
```

自定义状态键会自动推断，无需手动编写接口。

:::

## 选择前端模式

从应用程序需要解决的体验问题出发：

| 如果用户需要... | 请使用 |
| --- | --- |
| 了解智能体正在做什么 | [工具调用](/oss/langchain/frontend/tool-calling) 和 [推理令牌](/oss/langchain/frontend/reasoning-tokens) |
| 安全地审批敏感操作 | [人机交互](/oss/langchain/frontend/human-in-the-loop) |

| 将状态呈现为应用程序，而非聊天 | [结构化输出](/oss/langchain/frontend/structured-output)、[生成式 UI](/oss/langchain/frontend/generative-ui) 和 [Deep Agents 前端模式](/oss/deepagents/frontend/overview) |

## 集成

LangChain 的流 API 与 UI 无关。可与任何组件库或生成式 UI 框架配合使用。组件库可以负责表示层，而 LangChain 的 SDK 在底层管理智能体的运行时状态、可恢复性、中断和检查点语义。
