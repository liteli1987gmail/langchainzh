_运行_是对[助手](/langsmith/assistants)的调用。执行运行时，你需要指定使用哪个助手——通过 graph ID 指定默认助手，或通过 assistant ID 指定特定配置。

```mermaid actions={false}
flowchart TB
    subgraph deploy[Deployment]
        G[Graph Code<br/>━━━━━━━━━<br/>Deployed logic]
    end

    subgraph config[Configuration]
        A1[Assistant 1<br/>GPT-4, Formal]
        A2[Assistant 2<br/>Claude, Casual]
    end

    subgraph state[State]
        T1[Thread 1<br/>User A]
        T2[Thread 2<br/>User B]
    end

    subgraph runs[Runs]
        A1T1["Run: A1 + T1"]
        A1T2["Run: A1 + T2"]
        A2T1["Run: A2 + T1"]
    end

    A1 -.-> T1
    A1 -.-> T2
    A2 -.-> T1

    A1T1 --> G
    A1T2 --> G
    A2T1 --> G

    style G fill:#E5F4FF,stroke:#006DDD,stroke-width:2px,color:#030710
    style A1 fill:#B3E0F2,stroke:#4A90E2,stroke-width:2px,color:#1E3A5F
    style A2 fill:#B3E0F2,stroke:#4A90E2,stroke-width:2px,color:#1E3A5F
    style T1 fill:#FFE0B3,stroke:#7E65AE,stroke-width:2px,color:#504B5F
    style T2 fill:#FFE0B3,stroke:#7E65AE,stroke-width:2px,color:#504B5F

    style A1T2 fill:#B3F2C9,stroke:#10B981,stroke-width:2px,color:#2E3900
    style A2T1 fill:#B3F2C9,stroke:#10B981,stroke-width:2px,color:#2E3900
```

此图展示了 **run** 如何将助手与线程结合来执行图：

- **Graph** (浅蓝色)：包含代理逻辑的已部署代码
- **Assistants** (蓝色)：配置选项（模型、提示词、工具）
- **Threads** (橙色)：用于存储对话历史的状态容器
- **Runs** (绿色)：将助手与线程配对执行的单元

**组合示例：**
- **Run: A1 + T1**：将助手 1 配置应用于用户 A 的对话
- **Run: A1 + T2**：同一助手服务于用户 B（不同对话）
- **Run: A2 + T1**：将不同助手应用于用户 A 的对话（配置切换）

执行 run 时：

- 每个 run 可以有自己的输入、配置覆盖项和元数据。
- Runs 可以是无状态的（无线程），也可以是有状态的（在 [thread](/langsmith/use-threads) 上执行以保持对话持久性）。
- 多个 run 可以使用相同的助手配置。
- 助手的配置会影响底层图的执行方式。

Agent Server API 提供了多个端点用于创建和管理 runs。如需了解更多详情，请参阅 [API 参考文档](/langsmith/server-api-ref)。
