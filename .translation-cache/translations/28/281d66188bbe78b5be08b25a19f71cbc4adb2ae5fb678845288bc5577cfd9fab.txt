LangGraph 是一个低级编排框架和运行时，用于构建、管理和部署长时间运行的有状态代理，受到塑造代理未来的公司（包括 Klarna、Uber、J.P. Morgan 等）的信赖。

LangGraph 是非常低级的，完全专注于代理 **编排**。在使用 LangGraph 之前，我们建议您熟悉一些用于构建代理的组件，从 [模型](/oss/langchain/models) 和 [工具](/oss/langchain/tools).

我们通常会使用 [LangChain](/oss/langchain/overview) 文档中的组件来集成模型和工具，但您不需要使用 LangChain 来使用 LangGraph。如果您是代理的新手或想要更高级的抽象，我们建议您使用 LangChain 的 [代理](/oss/langchain/agents) 它为常见的 LLM 和工具调用循环提供预构建架构。

LangGraph 专注于对代理编排重要的底层能力：持久执行、流式处理、人工介入等。

<Expandable title="how LangChain products fit together" defaultOpen={false}>

- [深度代理](/oss/deepagents/overview) is an [代理测试框架](/oss/concepts/products#agent-harnesses-like-the-deep-agents-sdk)：在 LangGraph 之上的规划、子代理、文件系统工具和上下文管理。
- [LangChain](/oss/langchain/overview) 是代理框架：用于模型、工具和代理循环的抽象和集成。
- [LangGraph](/oss/langgraph/overview) 是编排运行时：持久执行、流式处理、人工介入和持久化。
- [LangSmith](/langsmith/observability) 是跨框架的追踪、评估、提示和部署平台。
- [LangSmith Engine](/langsmith/engine) 检测 LangGraph 代理追踪中的问题并提出修复建议。您可以直接从 Engine 选项卡打开包含建议修复的拉取请求。
- [LangSmith Fleet](/langsmith/fleet/index) 是用于模板、集成和流程自动化的无代码代理构建器。

阅读 [框架、运行时和测试框架](/oss/concepts/products) 以了解开源技术栈的比较。
</Expandable>

## <Icon icon="download" size={20} /> 安装

:::python

```bash pip

```

```bash uv
uv add langgraph
```
</CodeGroup>
:::

:::js
<CodeGroup>
```bash npm
npm install @langchain/langgraph @langchain/core
```

```bash pnpm
pnpm add @langchain/langgraph @langchain/core
```

```bash yarn
yarn add @langchain/langgraph @langchain/core
```

```bash bun
bun add @langchain/langgraph @langchain/core
```
</CodeGroup>
:::

然后，创建一个简单的 hello world 示例：

:::python
```python
from langgraph.graph import StateGraph, MessagesState, START, END

def mock_llm(state: MessagesState):
    return {"messages": [{"role": "ai", "content": "hello world"}]}

graph = StateGraph(MessagesState)
graph.add_node(mock_llm)
graph.add_edge(START, "mock_llm")
graph.add_edge("mock_llm", END)
graph = graph.compile()

graph.invoke({"messages": [{"role": "user", "content": "hi!"}]})
```
:::

:::js
```typescript
import { StateSchema, MessagesValue, type GraphNode, StateGraph, START, END } from "@langchain/langgraph";

const State = new StateSchema({
  messages: MessagesValue,
});

const mockLlm: GraphNode<typeof State> = (state) => {
  return { messages: [{ role: "ai", content: "hello world" }] };
};

const graph = new StateGraph(State)
  .addNode("mock_llm", mockLlm)
  .addEdge(START, "mock_llm")
  .addEdge("mock_llm", END)
  .compile();

await graph.invoke({ messages: [{ role: "user", content: "hi!" }] });
```
:::

<Tip>
使用 [LangSmith](/langsmith/observability) 来追踪请求、调试代理行为和评估输出。设置 `LANGSMITH_TRACING=true` 和您的 API 密钥以开始使用。按照 [追踪快速入门](/langsmith/trace-with-langchain) 进行设置。我们还建议您设置 [LangSmith Engine](/langsmith/engine) 它可以监控您的追踪、检测问题并提出修复建议。
</Tip>

## 核心优势

LangGraph 为 *任何* 长时间运行的有状态工作流或代理提供低级支持基础设施。LangGraph 不抽象提示或架构，并提供以下核心优势：

* [持久性](/oss/langgraph/persistence)：构建能够在失败中持续运行并长时间执行的智能体，从中断处恢复。
* [Human-in-the-loop](/oss/langgraph/interrupts)：通过在任何时点检查和修改智能体状态来引入人工监督。
* [全面记忆](/oss/concepts/memory)：创建有状态的智能体，同时具备用于持续推理的短期工作记忆和跨会话的长期记忆。
* [使用 LangSmith 进行调试](/langsmith/observability)：通过可视化工具深入了解复杂的智能体行为，这些工具可追踪执行路径、捕获状态转换并提供详细的运行时指标。
* [生产级部署](/langsmith/deployment)：使用可扩展的基础设施自信地部署复杂的智能体系统，该基础设施专为处理有状态的长时间运行工作流的独特挑战而设计。
