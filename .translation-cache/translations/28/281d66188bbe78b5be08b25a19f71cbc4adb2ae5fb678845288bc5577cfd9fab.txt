LangGraph 是一个低级编排框架和运行时，用于构建、管理和部署长时间运行的、有状态的智能体。Klarna、Uber、J.P. Morgan 等塑造智能体未来的公司都在使用它。

LangGraph 非常底层，完全专注于智能体**编排**。在使用 LangGraph 之前，我们建议您熟悉构建智能体所用的一些组件，从[模型](/oss/langchain/models)和[工具](/oss/langchain/tools)开始。

我们将在文档中经常使用 [LangChain](/oss/langchain/overview) 组件来集成模型和工具，但使用 LangGraph 不一定需要使用 LangChain。如果您是刚接触智能体或想要更高级的抽象，我们建议您使用 LangChain 的[智能体](/oss/langchain/agents)，它们为常见的 LLM 和工具调用循环提供预构建架构。

LangGraph 专注于智能体编排的重要底层能力：持久执行、流式处理、人机交互等。

<Expandable title="LangChain 产品如何协同工作" defaultOpen={false}>

- [Deep Agents](/oss/deepagents/overview) 是一个[智能体工具包](/oss/concepts/products#agent-harnesses-like-the-deep-agents-sdk)：在 LangGraph 之上提供规划、子智能体、文件系统工具和上下文管理。
- [LangChain](/oss/langchain/overview) 是智能体框架：为模型、工具和智能体循环提供抽象和集成。
- [LangGraph](/oss/langgraph/overview) 是编排运行时：持久执行、流式处理、人机交互和持久化。
- [LangSmith](/langsmith/observability) 是跨框架的追踪、评估、提示词和部署平台。
- [LangSmith Engine](/langsmith/engine) 可以检测 LangGraph 智能体追踪中的问题并提出修复建议。您可以直接从 Engine 选项卡打开包含建议修复的拉取请求。
- [LangSmith Fleet](/langsmith/fleet/index) 是用于模板、集成和日常自动化的无代码智能体构建器。

请阅读[框架、运行时和工具包](/oss/concepts/products)以了解开源堆栈的比较。
</Expandable>

## <Icon icon="download" size={20} /> 安装

:::python
<CodeGroup>


```bash uv
uv add langgraph
```
</CodeGroup>
:::

:::js
<CodeGroup>
```bash npm
npm install @langchain/langgraph @langchain/core
```

```bash pnpm
pnpm add @langchain/langgraph @langchain/core
```

```bash yarn
yarn add @langchain/langgraph @langchain/core
```

```bash bun
bun add @langchain/langgraph @langchain/core
```
</CodeGroup>
:::

然后，创建一个简单的 hello world 示例：

:::python
```python
from langgraph.graph import StateGraph, MessagesState, START, END

def mock_llm(state: MessagesState):
    return {"messages": [{"role": "ai", "content": "hello world"}]}

graph = StateGraph(MessagesState)
graph.add_node(mock_llm)
graph.add_edge(START, "mock_llm")
graph.add_edge("mock_llm", END)
graph = graph.compile()

graph.invoke({"messages": [{"role": "user", "content": "hi!"}]})
```
:::

:::js
```typescript
import { StateSchema, MessagesValue, type GraphNode, StateGraph, START, END } from "@langchain/langgraph";

const State = new StateSchema({
  messages: MessagesValue,
});

const mockLlm: GraphNode<typeof State> = (state) => {
  return { messages: [{ role: "ai", content: "hello world" }] };
};

const graph = new StateGraph(State)
  .addNode("mock_llm", mockLlm)
  .addEdge(START, "mock_llm")
  .addEdge("mock_llm", END)
  .compile();

await graph.invoke({ messages: [{ role: "user", content: "hi!" }] });
```
:::

<Tip>
使用 [LangSmith](/langsmith/observability) 追踪请求、调试 Agent 行为并评估输出。设置 `LANGSMITH_TRACING=true` 并配置您的 API 密钥即可开始使用。按照[追踪快速入门](/langsmith/trace-with-langchain)完成设置。我们还建议您设置 [LangSmith Engine](/langsmith/engine)，它可以监控您的追踪数据、检测问题并提出修复建议。
</Tip>

## 核心优势

LangGraph 为*任何*长期运行、有状态的工作流或 Agent 提供底层支持基础设施。LangGraph 不抽象化提示词或架构，提供以下核心优势：

* [持久化](/oss/langgraph/persistence)：构建可在故障时持久运行并能长时间执行、从中断处恢复的 Agent。
* [人工介入](/oss/langgraph/interrupts)：通过在任意时刻检查和修改 Agent 状态来引入人工监督。
* [全面的记忆](/oss/concepts/memory)：创建具有短期工作记忆（用于持续推理）和长期记忆（跨会话）的有状态 Agent。
* [使用 LangSmith 调试](/langsmith/observability)：通过可视化工具深入了解复杂的 Agent 行为，这些工具可以追踪执行路径、捕获状态转换并提供详细的运行时指标。
* [生产级部署](/langsmith/deployment)：使用专为处理有状态长期运行工作流的独特挑战而设计的高扩展性基础设施，自信地部署复杂的 Agent 系统。
