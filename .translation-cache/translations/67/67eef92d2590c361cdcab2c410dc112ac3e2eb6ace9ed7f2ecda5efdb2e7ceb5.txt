LangSmith 可以使用 OpenTelemetry 检测来捕获 [LiveKit Agents](https://docs.livekit.io/agents/) 生成的追踪。本指南展示了如何自动捕获 LiveKit 语音 AI 代理的追踪，并将其发送到 LangSmith 进行监控和分析。

有关追踪语音代理的高级指导原则，请参阅[语音追踪基础](/langsmith/trace-voice-fundamentals)。

如需完整的实现，请参阅[语音演示仓库](https://github.com/langchain-ai/voice-demo)。

## 安装

安装所需的包：

<CodeGroup>

```bash pip
pip install langsmith livekit livekit-agents livekit-plugins-openai livekit-plugins-silero livekit-plugins-turn-detector opentelemetry-exporter-otlp python-dotenv
```

```bash uv

```

</CodeGroup>

## 快速入门教程

按照此分步教程，使用 LiveKit 和 LangSmith 追踪功能创建一个语音 AI 代理。您将通过复制和粘贴代码片段来构建一个完整的可工作示例。

### 步骤 1：设置您的环境

在您的项目目录中创建一个 `.env` 文件：

```bash .env
OTEL_EXPORTER_OTLP_ENDPOINT=https://api.smith.langchain.com/otel
OTEL_EXPORTER_OTLP_HEADERS=x-api-key=<your-langsmith-api-key>, Langsmith-Project=livekit-voice
LIVEKIT_URL=<your-livekit-url>
LIVEKIT_API_KEY=<your-livekit-api-key>
LIVEKIT_API_SECRET=<your-livekit-api-secret>
OPENAI_API_KEY=<your-openai-api-key>
```

### Step 2: 下载 span processor

LiveKit 会发送 OpenTelemetry span，但大部分有用数据都附加在 LiveKit 特定属性中，LangSmith 默认无法识别这些属性。自定义 span processor 会转换这些属性，使追踪在 LangSmith 中正确渲染。

添加[自定义 span processor 文件](https://github.com/langchain-ai/voice-demo/blob/main/src/voice_demo/livekit/processor.py)，并将其保存为项目目录中的 `langsmith_processor.py`。

<Accordion title="这个 span processor 有什么作用？">

span processor 会使用 LangSmith 兼容的属性丰富 LiveKit Agents 的 OpenTelemetry span，使追踪在 LangSmith 中正确显示。

**核心功能：**

- 将 LiveKit span 类型（stt、llm、tts、agent、session、job）转换为 LangSmith 格式。
- 添加 `gen_ai.prompt.*` 和 `gen_ai.completion.*` 属性以实现消息可视化。
- 将 LiveKit 的指标（首 token 时间、首字节时间、端到端延迟以及其他 `lk.*` 分析数据）作为运行元数据公开。
- 在整个对话中跟踪和聚合消息。
- 渲染完整对话记录并将通话录音附加到根运行。

它只会重塑它能识别为 LiveKit 的 span；任何其他 span（例如，嵌套的 LangChain 或 LangGraph 运行）都会原样通过。当你在代码中导入它时，processor 就会激活。

</Accordion>

### Step 3: 创建语音 agent 文件

创建一个名为 `agent.py` 的新文件，并添加以下代码。我们将逐步构建它，以便你可以复制粘贴每个部分。

#### 第一部分：导入依赖项并设置追踪

```python
import sys
import os
from pathlib import Path
from dotenv import load_dotenv

# Load environment variables
load_dotenv()

# Import LiveKit components
from livekit import agents
from livekit.agents import AgentServer, AgentSession, Agent
from livekit.agents.telemetry import set_tracer_provider
from livekit.plugins import openai, silero
from livekit.plugins.turn_detector.multilingual import MultilingualModel
from opentelemetry.sdk.trace import TracerProvider

# Import span processor to enable LangSmith tracing
from langsmith_processor import LangSmithSpanProcessor

# Set up LangSmith tracing
def setup_langsmith():
    """Setup OpenTelemetry tracing to export spans to LangSmith."""
    endpoint = os.getenv("OTEL_EXPORTER_OTLP_ENDPOINT")
    headers = os.getenv("OTEL_EXPORTER_OTLP_HEADERS")

    if not endpoint or not headers:
        print("⚠️  Warning: OTEL environment variables not set. Tracing disabled.")
        return

    # Create tracer provider with custom span processor
    trace_provider = TracerProvider()
    trace_provider.add_span_processor(LangSmithSpanProcessor())

    # Set as LiveKit's tracer provider
    set_tracer_provider(trace_provider)
    print("✅ LangSmith tracing enabled")

# Enable tracing before creating agents
setup_langsmith()
```

#### 第二部分：定义你的代理

```python
class Assistant(Agent):
    def __init__(self) -> None:
        super().__init__(
            instructions="""You are a helpful voice AI assistant.
            You eagerly assist users with their questions.
            Keep responses concise and conversational.""",
        )
```

#### 第 3 部分：设置代理服务器

```python
server = AgentServer()

@server.rtc_session()
async def my_agent(ctx: agents.JobContext):
    # Create agent session with STT, LLM, TTS, and VAD
    session = AgentSession(
        stt="deepgram/nova-2:en",
        llm="openai/gpt-5.4-mini",
        tts=openai.TTS(model="tts-1", voice="alloy"),
        vad=silero.VAD.load(),
        turn_detection=MultilingualModel(),
    )

    # Start the session
    await session.start(
        room=ctx.room,
        agent=Assistant(),
    )

if __name__ == "__main__":
    # Run in console mode for local testing
    sys.argv = [sys.argv[0], "console"]
    agents.cli.run_app(server)
```

### 第 4 步：运行你的代理

在控制台模式下运行你的语音代理进行本地测试：

```bash
python agent.py console
```

您的 agent 将启动并连接到 LiveKit。通过麦克风说话，所有 trace 将自动出现在 LangSmith 中。

## 高级用法

### 追踪语音到语音模型

前面的示例使用了级联管道（分离的 STT、LLM 和 TTS 服务）。LiveKit 还支持语音到语音模型，其中单个实时模型处理输入和输出音频。要追踪此类模型，请使用实时模型构建会话，而不是使用 STT/LLM/TTS 堆栈。追踪设置是相同的，span processor 保持不变：

</fragment>

```python
from livekit.plugins import openai as lk_openai

session = AgentSession(llm=lk_openai.realtime.RealtimeModel(voice="marin"))
```

### 附加对话音频

要同时收听对话及其转录内容，请录制通话并将音频文件附加到根运行。录制播放给客户的内容，使录音反映实际听到的内容，包括因中断而被截断的话语。有关底层附件 API，请参阅[使用跟踪上传文件](/langsmith/upload-files-with-traces)。

### 自定义元数据和标签

您可以使用 span 属性向跟踪添加自定义元数据：

```python
from opentelemetry import trace

class Assistant(Agent):
    def __init__(self) -> None:
        super().__init__(
            instructions="You are a helpful assistant.",
        )

        # Get current span and add custom attributes
        tracer = trace.get_tracer(__name__)
        span = trace.get_current_span()
        if span:
            span.set_attribute("langsmith.metadata.agent_type", "voice_assistant")
            span.set_attribute("langsmith.metadata.version", "1.0")
            span.set_attribute("langsmith.span.tags", "livekit,voice-ai,production")
```

## 故障排除

### LangSmith 中未显示 Span

如果追踪未在 LangSmith 中显示：

1. **验证环境变量**：确保 `.env` 文件中 `OTEL_EXPORTER_OTLP_ENDPOINT` 和 `OTEL_EXPORTER_OTLP_HEADERS` 设置正确。
2. **检查设置顺序**：确保 `setup_langsmith()` 在创建 `AgentServer` **之前**被调用。
3. **检查 API 密钥**：确认您的 LangSmith API 密钥具有写入权限。
4. **查看确认信息**：启动时应看到控制台输出"✅ LangSmith tracing enabled"。

### 消息未正确显示

如果对话消息未正确显示：

1. **检查 span processor**：验证 `langsmith_processor.py` 在您的项目目录中且已正确导入。
2. **验证导入**：确保 `LangSmithSpanProcessor` 在您的 agent.py 中已导入。
3. **启用调试日志**：在您的环境中设置 `LANGSMITH_PROCESSOR_DEBUG=true` 以查看详细日志。

### 连接问题

如果您的代理无法连接到 LiveKit：

1. **验证 LiveKit URL**：检查 `.env` 文件中的 `LIVEKIT_URL` 设置是否正确。
2. **检查凭证**：确保 `LIVEKIT_API_KEY` 和 `LIVEKIT_API_SECRET` 正确。
3. **测试连接**：首先使用 LiveKit CLI 尝试连接到您的 LiveKit 服务器。
4. **控制台模式**：对于本地测试，始终使用：`python agent.py console`。

### 导入错误

如果您遇到导入错误：

1. **安装依赖**：运行第 1 步中的完整 pip install 命令。
2. **检查 Python 版本**：确保您使用的是 Python 3.9 或更高版本。
3. **验证 langsmith_processor**：确保 `langsmith_processor.py` 已下载并位于 `agent.py` 的同一目录中。
4. **检查 LiveKit 插件**：确保您为 STT/LLM/TTS 提供商安装了正确的 LiveKit 插件。

### 代理无响应

如果您的代理已连接但无响应：

1. **检查 API 密钥**：验证您的 OpenAI API 密钥（或其他提供商密钥）正确。
2. **测试服务**：确保您的 STT、LLM 和 TTS 服务可访问。
3. **检查指令**：确保您的代理有正确的指令。
4. **查看日志**：在控制台输出中查找错误。
