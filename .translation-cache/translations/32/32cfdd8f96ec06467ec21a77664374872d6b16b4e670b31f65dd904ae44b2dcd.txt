配置工作区密钥并管理 Fleet 代理和用户的消费限额。

## 工作区密钥

Fleet 使用 [工作区密钥](/langsmith/set-up-hierarchy#configure-workspace-settings) 来存储模型和工具的 API 密钥。以下是可用的密钥类型：

- **必需的模型密钥**：需要 OpenAI 或 Anthropic API 密钥才能让 Fleet 进行 LLM API 调用。代理图从工作区密钥加载此密钥用于推理。
- **Fleet 专用密钥**：以 `FLEET_` 为前缀的密钥在 Fleet 内部优先于工作区密钥。这样，您可以更好地追踪 Fleet 与使用相同密钥的 LangSmith 其他部分的用量。如果您同时拥有 `OPENAI_API_KEY` 和 `FLEET_OPENAI_API_KEY`，则 `FLEET_OPENAI_API_KEY` 密钥将被使用。
- **可选工具密钥**：为您启用的任何工具添加密钥。这些在运行时从工作区密钥读取。
  - `EXA_API_KEY`：Exa 搜索工具（通用网络和 LinkedIn 个人资料搜索）必需。
  - `TAVILY_API_KEY`：Tavily 网络搜索必需。
  - `TWITTER_API_KEY` 和 `TWITTER_API_KEY_SECRET`: Required for Twitter/X read operations (app-only bearer). Posting/media upload is not enabled.
- **MCP 服务器配置**：Fleet 可以从一个或多个远程 [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) 服务器拉取工具。在您的 [工作区](/langsmith/administration-overview#workspaces) 设置中配置 MCP 服务器和请求头。Fleet 自动发现工具并在调用时应用配置的请求头。更多信息，请参阅 [远程 MCP 服务器](/langsmith/fleet/remote-mcp-servers) page.

<Note icon="wand" iconType="regular">
  Fleet 支持每个代理使用自定义模型。参见 [自定义模型](/langsmith/fleet/essentials#custom-models) 了解更多信息。
</Note>

### 添加密钥

添加密钥：

1. 在 [LangSmith UI](https://smith.langchain.com)中，导航至 <Icon icon="settings" /> **设置** ，然后进入 **密钥** tab.
1. 选择 **添加密钥** 并输入密钥 **名称** （例如， `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`）并将您的密钥作为 **值**.

    <Note>
    确保密钥名称与模型提供商预期的环境变量名称相匹配。
    </Note>

1. 选择 **保存密钥**.

## 用量和消费限额

 **用量** 页面让工作区管理员可以查看 Fleet 消费情况，并能够为代理和用户设置消费限额。此页面仅对具有以下权限的用户可见 `fleet:read-admin-config` permission.

### 查看当前消费

该 **使用量** 页面显示您工作区在所选时间段内的总消费（**最近 7 天** or **最近 14 天**），以及总线程数和总运行次数。

每日消费图表提供了所选时间段内成本的直观明细。 **明细** 部分让您可以通过两种方式查看消费明细：

- **按代理**：查看每个代理的总成本、运行次数、首次和最后使用日期、所有者以及每周限额。
- **按用户**：查看每个用户的消费和活动情况。

### 设置消费限额

消费限额让您可以控制代理和用户的消费金额。管理消费限额需要 `fleet:write-admin-config` permission.

#### 默认每周消费限额

在 **默认每周消费限额** 部分中，您可以配置：

- **每个代理的默认限额（美元）**：设置适用于工作区中所有代理的默认每周消费限额。
- **每个用户的默认限额（美元）**：设置适用于工作区中所有用户的默认每周消费限额。

限额为本周至今的累计值，会在周一重置。

#### 为单个代理和用户覆盖限额

您可以为单个代理或用户覆盖默认消费限额，设置自定义的每周限额。

#### 消费限额行为

- 消费限额的更改可能需要几分钟才能传播到所有正在运行的代理。
- 消费限额在每次运行开始时检查。如果运行开始时用量低于限额，即使最终成本超过限额，运行也将被允许完成。
- 消费计算基于追踪记录。删除追踪记录将影响报告的用量和消费执行。
