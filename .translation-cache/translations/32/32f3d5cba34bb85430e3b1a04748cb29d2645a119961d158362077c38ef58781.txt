## 部署选项

选择托管 LangSmith 的方式，以满足您的基础设施和数据驻留要求。

<CardGroup cols={3}>
  <Card title="云端" icon="cloud" href="/langsmith/cloud">
    在 LangSmith 托管的云端中托管 LangSmith，支持美国或欧盟数据驻留。
  </Card>
  <Card title="混合部署" icon="topology-complex" href="/langsmith/hybrid">
    在 LangSmith 的云端运行控制平面，在您自己的 VPC 中运行数据平面，以实现完全的数据隔离。
  </Card>
  <Card title="自托管" icon="server-2" href="/langsmith/self-hosted">
    使用 Docker Compose 或 Kubernetes 在您自己的基础设施内完全部署 LangSmith。
  </Card>
</CardGroup>

## 访问控制

控制组织内谁可以访问什么。

<CardGroup cols={2}>
  <Card title="基于角色的访问控制 (RBAC)" icon="shield-lock" href="/langsmith/rbac">
    使用内置或自定义角色定义每个工作区的权限。仅在企业计划中可用。
  </Card>
  <Card title="基于属性的访问控制 (ABAC)" icon="tag" href="/langsmith/abac">
    应用细粒度的基于标签的访问策略来限制资源访问——包括阻止特定用户访问 PII 数据。
  </Card>
  <Card title="工作负载隔离" icon="layout-sidebar" href="/langsmith/workload-isolation">
    使用多工作区模型隔离团队、建立信任边界并分离环境。
  </Card>
  <Card title="资源标签" icon="tag" href="/langsmith/set-up-resource-tags">
    标记资源以用于 ABAC 策略，并组织 dev、staging 和 prod 等环境。
  </Card>
</CardGroup>
