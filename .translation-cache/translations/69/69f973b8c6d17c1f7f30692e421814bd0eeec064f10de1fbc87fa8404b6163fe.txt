# LangGraph

LangGraph 是一个低级编排框架和运行时，用于构建、管理和部署长时间运行的有状态智能体。它提供持久执行、流式输出、人工介入交互和时间回溯调试功能。

## 使用场景

在以下场景中使用 LangGraph：
- **设计自定义智能体工作流** 具有显式的基于图的控制流
- **添加持久执行能力** 使智能体能够在故障和重启后继续运行
- **实现人工介入** 包含中断和审批步骤
- **构建多智能体系统** 在多个智能体之间共享状态
- **流式输出中间结果** 来自长时间运行的智能体任务
- **时间回溯调试** 从任意检查点重放智能体执行过程

## 不适用场景

- 对于简单的工具调用智能体，请使用 [LangChain](https://docs.langchain.com/oss/langchain/overview) 智能体——常见模式更少样板代码
- 对于包含规划和子智能体的完整智能体，请使用 [Deep Agents](https://docs.langchain.com/oss/deepagents/overview) 代替
- LangGraph 是 **编排层**——当您需要对智能体行为进行细粒度控制时使用

## 快速参考

### 图 API（推荐用于大多数用例）

```python
from langgraph.graph import StateGraph, MessagesState, START, END

def my_node(state: MessagesState):
    return {"messages": [{"role": "ai", "content": "hello world"}]}

graph = StateGraph(MessagesState)
graph.add_node(my_node)
graph.add_edge(START, "my_node")
graph.add_edge("my_node", END)
graph = graph.compile()

result = graph.invoke(
    {"messages": [{"role": "user", "content": "Hello!"}]}
)
```

### 函数式 API（用于简单管道）

```python
from langgraph.func import entrypoint, task

@task
def step_one(input: str) -> str:
    return f"processed: {input}"

@entrypoint()
def pipeline(input: str) -> str:
    return step_one(input).result()
```

### 添加人工介入

```python
from langgraph.types import interrupt

def human_approval(state: MessagesState):
    answer = interrupt({"question": "Approve this action?"})
    return {"messages": [{"role": "user", "content": answer}]}
```

## 核心概念

| 概念 | 描述 |
|---------|-------------|
| `StateGraph` | 定义构成智能体控制流的节点和边 |
| `MessagesState` | 内置聊天智能体状态架构 |
| `compile()` | 将图构建器编译为可执行图 |
| `interrupt()` | 暂停执行并等待人工输入 |
| 检查点器 | 持久化状态以实现持久执行和时间回溯 |
| 图 API 与函数式 API | 图 API 适用于复杂工作流；函数式 API 适用于线性管道 |
