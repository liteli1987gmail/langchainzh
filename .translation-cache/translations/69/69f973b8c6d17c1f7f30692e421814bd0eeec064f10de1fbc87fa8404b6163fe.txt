# LangGraph

LangGraph 是一个低层次的编排框架和运行时，用于构建、管理和部署长时间运行的有状态智能体。它提供持久化执行、流式输出、人机交互和时间回溯调试功能。

## 适用场景

在以下情况下使用 LangGraph：
- **设计自定义智能体工作流**，使用显式的基于图的控制流
- **添加持久化执行**，使智能体能够在故障和重启后继续运行
- **实现人机交互**，使用中断和审批步骤
- **构建多智能体系统**，在智能体之间共享状态
- **流式输出中间结果**，处理长时间运行的智能体任务
- **时间回溯调试**，从任何检查点回放智能体执行

## 不适用场景

- 对于简单的工具调用智能体，请改用 [LangChain](https://docs.langchain.com/oss/langchain/overview) 智能体——常见模式更少样板代码
- 对于包含规划和子智能体的全能智能体，请改用 [Deep Agents](https://docs.langchain.com/oss/deepagents/overview)
- LangGraph 是**编排层**——在你需要对智能体行为进行细粒度控制时使用

## 快速参考

### Graph API（推荐用于大多数用例）

```python
from langgraph.graph import StateGraph, MessagesState, START, END

def my_node(state: MessagesState):
    return {"messages": [{"role": "ai", "content": "hello world"}]}

graph = StateGraph(MessagesState)
graph.add_node(my_node)
graph.add_edge(START, "my_node")
graph.add_edge("my_node", END)
graph = graph.compile()

result = graph.invoke(
    {"messages": [{"role": "user", "content": "Hello!"}]}
)
```

### 函数式 API（用于简单管道）

```python
from langgraph.func import entrypoint, task

@task
def step_one(input: str) -> str:
    return f"processed: {input}"

@entrypoint()
def pipeline(input: str) -> str:
    return step_one(input).result()
```

### 添加人工干预

```python
from langgraph.types import interrupt

def human_approval(state: MessagesState):
    answer = interrupt({"question": "Approve this action?"})
    return {"messages": [{"role": "user", "content": answer}]}
```

## 核心概念

| 概念 | 描述 |
|---------|-------------|
| `StateGraph` | 定义构成代理控制流的节点和边 |
| `MessagesState` | 用于聊天型代理的内置状态模式 |
| `compile()` | 将图构建器编译为可执行图 |
| `interrupt()` | 暂停执行并等待人工输入 |
| Checkpointer | 持久化状态以支持持久执行和时间旅行 |
| Graph API vs Functional API | Graph API 用于复杂工作流；Functional API 用于线性管道 |
