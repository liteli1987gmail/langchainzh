> 来自 [Unstructured.IO](https://www.unstructured.io/) 的 `unstructured` 包可以从 PDF 和 Word 文档等原始源文档中提取干净的文本。
本页面介绍如何在 LangChain 中使用 [`unstructured`](https://github.com/Unstructured-IO/unstructured) 生态系统。

## 安装和设置

如果您使用的是本地运行的加载器，请按照以下步骤来运行 `unstructured` 及其依赖项。

- 如需最小的安装占用空间并利用开源 `unstructured` 包中不可用的功能，请使用 `pip install unstructured-client` 安装 Python SDK，同时配合 `pip install langchain-unstructured` 来使用 `UnstructuredLoader` 并通过 Unstructured API 进行远程分区。此加载器位于 LangChain 合作伙伴仓库中，而非 `langchain-community` 仓库中，您需要一个 `api_key`。您可以在 [Unstructured API 密钥页面生成免费密钥](https://unstructured.io/api-key/)。
    - Unstructured 的 SDK 文档可在此处找到：
      https://docs.unstructured.io/api-reference/api-services/sdk

- 如需在本地运行所有内容，请使用 `pip install unstructured` 安装开源 Python 包，同时配合 `pip install langchain-community` 并使用上述相同的 `UnstructuredLoader`。
    - 您可以使用 extras 安装特定文档的依赖项，例如 `pip install "unstructured[docx]"`。在 [完整安装文档](https://docs.unstructured.io/open-source/installation/full-installation) 中了解更多关于 extras 的信息。
    - 如需安装所有文档类型的依赖项，请使用 `pip install "unstructured[all-docs]"`。
- 如果系统上尚未安装，请使用例如 Mac 上的 `brew install` 来安装以下系统依赖项。根据您正在解析的文档类型，可能不需要所有这些依赖项。
    - `libmagic-dev`（文件类型检测）
    - `poppler-utils`（图像和 PDF）
    - `tesseract-ocr`（图像和 PDF）
    - `qpdf`（PDF）
    - `libreoffice`（MS Office 文档）
    - `pandoc`（EPUB）
- 在本地运行时，Unstructured 还建议按照 [此指南](https://docs.unstructured.io/open-source/installation/docker-installation) 使用 Docker，以确保所有系统依赖项正确安装。

Unstructured API 需要 API 密钥来发出请求。
您可以 [申请 API 密钥](https://unstructured.io/api-key-hosted) 并立即开始使用！
查看 [Unstructured API README](https://github.com/Unstructured-IO/unstructured-api) 开始发出 API 调用。
我们非常乐意听取您的反馈，请在我们的 [社区 Slack](https://join.slack.com/t/unstructuredw-kbe4326/shared_invite/zt-1x7cgo0pg-PTptXWylzPQF9xZolzCnwQ) 中告诉我们使用情况。
请持续关注质量和性能方面的改进！
