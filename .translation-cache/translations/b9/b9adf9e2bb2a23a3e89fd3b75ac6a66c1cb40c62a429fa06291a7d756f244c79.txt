在本地使用 LangChain 构建智能体时，对其内部发生的情况进行可视化、实时交互以及在问题发生时进行调试会非常有帮助。 **LangSmith Studio** 是一个免费的图形界面，用于从本地机器开发和测试您的 LangChain 智能体。

Studio 连接到本地运行的智能体，向您展示智能体执行的每个步骤：发送给模型的提示词、工具调用及其结果，以及最终输出。您可以测试不同的输入、检查中间状态，并在不编写额外代码或部署的情况下迭代优化智能体的行为。

本页介绍如何将 Studio 与您的本地 LangChain 智能体进行设置。

## 前提条件

在开始之前，请确保您满足以下要求：

- **一个 LangSmith 账户**：在 [smith.langchain.com](https://smith.langchain.com).
- **注册（免费）或登录**一个 LangSmith API 密钥 [：按照](/langsmith/create-account-api-key) guide.
- 创建 API 密钥 [的说明进行操作。如果您不想将数据](/langsmith/observability-concepts#traces) 追踪 `LANGSMITH_TRACING=false` 到 LangSmith，请在应用程序的 `.env` 文件中设置

## 。禁用追踪后，数据不会离开您的本地服务器。

### 设置本地智能体服务器

1. 安装 LangGraph CLI [LangGraph CLI](/langsmith/cli) 提供了一个本地开发服务器（也称为 [智能体服务器](/langsmith/agent-server)），用于将您的智能体连接到 Studio。

:::python
```shell
# Python >= 3.11 is required.
pip install --upgrade "langgraph-cli[inmem]"
```
:::

:::js
```shell
npx @langchain/langgraph-cli
```
:::

### 2. 准备您的智能体

如果您已有 LangChain 智能体，可以直接使用。本示例使用一个简单的电子邮件智能体：

:::python
```python title="agent.py"
from langchain.agents import create_agent

def send_email(to: str, subject: str, body: str):
    """Send an email"""
    email = {
        "to": to,
        "subject": subject,
        "body": body
    }
    # ... email sending logic

    return f"Email sent to {to}"

agent = create_agent(
    "gpt-5.5",
    tools=[send_email],
    system_prompt="You are an email assistant. Always use the send_email tool.",
)
```
:::

:::js
```typescript title="agent.ts"
import { createAgent } from "@langchain/langgraph";

function sendEmail(to: string, subject: string, body: string): string {
    // Send an email
    const email = {
        to: to,
        subject: subject,
        body: body
    };
    // ... email sending logic

    return `Email sent to ${to}`;
}

const agent = createAgent({
    model: "gpt-5.5",
    tools: [sendEmail],
    systemPrompt: "You are an email assistant. Always use the sendEmail tool.",
});

export { agent };
```
:::

### 3. 环境变量

Studio 需要一个 LangSmith API 密钥来连接您的本地智能体。在项目根目录创建一个 `.env` 文件，并添加您从 [LangSmith](https://smith.langchain.com/settings).

<Warning>
    获取的 API 密钥。确保您的 `.env` 文件不会被提交到版本控制系统（如 Git）中。
</Warning>

```bash .env
LANGSMITH_API_KEY=lsv2...
```

### 4. 创建 LangGraph 配置文件

LangGraph CLI 使用配置文件来定位您的智能体并管理依赖项。在您的应用目录中创建一个 `langgraph.json` 文件：

:::python
```json title="langgraph.json"
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent.py:agent"
  },
  "env": ".env"
}
```
:::

:::js
```json title="langgraph.json"
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent.ts:agent"
  },
  "env": ".env"
}
```
:::

:::python
@[`create_agent`] 函数会自动返回一个已编译的 LangGraph 图，这正是配置文件中 `graphs` 键所期望的格式。
:::

:::js
@[`createAgent`] 函数会自动返回一个已编译的 LangGraph 图，这正是配置文件中 `graphs` 键所期望的格式。
:::

<Info>
有关配置文件 JSON 对象中每个键的详细说明，请参阅 [LangGraph 配置文件参考](/langsmith/cli#configuration-file).
</Info>

此时，项目结构应如下所示：

:::python
```bash
my-app/
├── src
│   └── agent.py
├── .env
└── langgraph.json
```
:::

:::js
```bash
my-app/
├── src
│   └── agent.ts
├── .env
├── package.json
└── langgraph.json
```
:::

### 5. 安装依赖项

:::python
从根目录安装项目依赖项：

<CodeGroup>
```shell pip
pip install langchain langchain-openai
```
```shell uv
uv add langchain langchain-openai
```
</CodeGroup>
:::

:::js
```shell
yarn install
```
:::

### 6. 在 Studio 中查看您的代理

启动开发服务器以将您的代理连接到 Studio：

:::python
```shell
langgraph dev
```
:::

:::js
```shell
npx @langchain/langgraph-cli dev
```
:::

<Warning>
Safari 浏览器会阻止 `localhost` 连接到 Studio。为了解决这个问题，请使用以下命令运行 `--tunnel` 通过安全隧道访问 Studio。您需要通过点击 **连接到本地服务器** 在 Studio UI 中手动将隧道 URL 添加到允许的来源。请参阅 [故障排除指南](/langsmith/troubleshooting-studio#safari-connection-issues) 了解步骤。
</Warning>

服务器运行后，您的代理可通过 API 访问，地址为 `http://127.0.0.1:2024` 以及通过 Studio UI 访问，地址为 `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`:

![Studio UI 中的代理视图](/oss/images/studio_create-agent.png)

With Studio connected to your local agent, you can iterate quickly on your agent's behavior. Run a test input, inspect the full execution trace including prompts, tool arguments, return values, and token/latency metrics in [LangSmith](/langsmith/observability-studio)。当出现问题时，Studio 会捕获异常及其周围状态，帮助您了解发生了什么。

开发服务器支持热重载——修改代码中的提示或工具签名后，Studio 会立即反映这些更改。您可以从任何步骤重新运行对话线程来测试更改，无需从头开始。这种工作流程适用于简单的单工具代理和复杂的多节点图。
