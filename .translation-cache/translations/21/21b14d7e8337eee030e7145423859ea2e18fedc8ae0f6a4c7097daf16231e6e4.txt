Playground 允许您使用任何符合 OpenAI API 的模型。您可以通过在 Playground 中设置代理提供商来使用您的模型。

## 部署 OpenAI 兼容模型

许多提供商提供 OpenAI 兼容模型或代理服务。这方面的例子包括：

* [LiteLLM Proxy](https://github.com/BerriAI/litellm?tab=readme-ov-file#quick-start-proxy---cli)
* [Ollama](https://ollama.com/)

您可以使用这些提供商来部署您的模型并获取符合 OpenAI API 的 API 端点。

查看完整的规范了解更多信息。

## 在 Playground 中使用模型

部署模型服务器后，您可以在 Playground 中使用它。

访问 **Prompt Settings** 菜单：

1. 在 **Prompts** 标题下，选择模型名称旁边的齿轮 <Icon icon="settings" iconType="solid" /> 图标。
1. 在 **Model Configuration** 选项卡中，在下拉菜单中选择要编辑的模型。
1. 对于 **Provider** 下拉菜单，选择 **OpenAI Compatible Endpoint**。
1. 将您的 OpenAI 兼容端点添加到 **Base URL** 输入框中。参见 Base URL 格式了解示例。

如果设置正确，您应该会在 Playground 中看到模型的响应。您还可以使用此功能调用下游管道。

有关如何存储模型配置的信息，请参阅配置提示设置。

如果您的 OpenAI 兼容端点位于 OAuth2 网关后面，请将 OAuth `client_credentials` 存储在模型配置中，而不是将静态 API 密钥作为工作区密钥分发。参见 OAuth 客户端凭据。

## Base URL 格式

**Base URL** 应指向 OpenAI 兼容 API 服务器的根目录。
LangSmith 会自动附加 `/chat/completions`——请勿在 Base URL 中包含它。

### 示例 Base URL

| 提供商 | 示例 Base URL |
|----------|-----------------|
| [Ollama](https://ollama.com/)（本地） | `http://localhost:11434/v1` |
| [LiteLLM Proxy](https://github.com/BerriAI/litellm)（本地） | `http://localhost:4000` |
| [vLLM](https://docs.vllm.ai/)（本地） | `http://localhost:8000/v1` |
| 自托管（远程） | `https://my-model-server.example.com/v1` |

支持自定义路径前缀。如果您的服务器在 `/api/v2/chat/completions` 提供补全功能，
请将 Base URL 设置为 `https://my-server.example.com/api/v2`。
