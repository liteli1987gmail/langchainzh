## Pull Request 要求

<Warning>
    **所有 Pull Request 必须链接到一个由维护者批准解决方案的 Issue 或讨论。** 在维护者批准您的方案并将为关联的 Issue 分配给您之前，请不要提交 Pull Request。提前提交的 PR 可能会被自动关闭，在分配之前不会进行审查。
</Warning>

所有 Pull Request 都应展示有意义的 effort 和上下文理解。**如果创建 Pull Request 所需的 effort 小于维护者审查所需的 effort，则不应提交此类贡献。** 低 effort 的一次性贡献——无论其产生方式如何——往往在上下文相关性、准确性和质量方面偏离目标。大量自动化贡献对我们的 effort 来说是一种拒绝服务攻击。

所有外部 Pull Request 必须满足以下要求：

* Pull Request 必须链接到一个由维护者批准解决方案的 Issue 或讨论，且贡献者在打开 PR 之前必须被分配到该 Issue。
* Pull Request 必须填写仓库的 Pull Request 模板。

如果未满足这些要求，维护者保留不附带说明直接关闭 PR 的权利。**我们将关闭看起来是低 effort 垃圾内容的 Pull Request 和 Issue。**

## 语言政策

所有贡献——Issue、Pull Request、代码审查和讨论——必须使用英语。这使我们的全球贡献者社区能够访问和搜索通信内容。

如果英语不是您的第一语言，请不要担心。我们重视清晰的沟通胜过完美的语法，欢迎使用翻译工具。

## LLM 的可接受使用方式

当您**验证每个更改**时，可以使用 AI 助手帮助起草或修改贡献：运行和测试代码，根据代码库和官方提供商文档检查事实，并确保结果符合仓库风格。请勿提交大量未经审查的生成内容。无论内容是如何产生的，我们会关闭看起来是低 effort 或垃圾内容的 Pull Request。
