title: 管理模型配置
description: 管理模型配置并控制其在 LangSmith 功能中的可用性。
---

模型配置定义了 LangSmith 功能在调用 AI 提供商时使用的模型和参数。单个共享配置库覆盖您的整个 [工作区](/langsmith/administration-overview#workspaces)，因此您创建的任何配置都可以在以下功能中使用，而无需重复：

- [**Playground**](/langsmith/prompt-engineering-concepts)
- [**Evaluators**](/langsmith/evaluation)
- [**Fleet**](/langsmith/fleet/index)
- [**Chat**](/langsmith/chat)
- [**Insights**](/langsmith/insights)

[工作区管理员](/langsmith/rbac#workspace-admin) 可以创建、编辑和删除配置，并控制每个功能可用的提供商和模型。非管理员成员可以查看配置，但无法修改。

配置还可以包含 [OAuth 客户端凭据](#oauth-client-credentials)，因此 LangSmith 在请求时向您的 IdP 颁发短期持有者令牌，而不是使用静态 API 密钥。

## 功能访问权限

 **功能访问权限** 表独立控制每个 LangSmith 功能中提供商和模型的可用性。

| **功能** | **模型选择体验** |
|---------|---------------------------|
| Playground | 完整模型控制—查看和调整所有参数。无内置模型；依赖工作区配置。 |
| Evaluators | 完整模型控制—查看和调整所有参数。无内置模型；依赖工作区配置。 |
| Fleet | 默认从精选列表中选择。您也可以添加自定义工作区配置。 |
| Chat | 默认从精选列表中选择。您也可以添加自定义工作区配置。 |
| Insights (Thinking) | 用于深度分析的模型。默认从带提供商推荐的精选列表中选择。您也可以添加自定义工作区配置。 |
| Insights (Summarization) | 用于轻量级摘要的模型。默认从带提供商推荐的精选列表中选择。您也可以添加自定义工作区配置。 |

所有功能都支持自定义工作区配置，因此您可以使用任何提供商或模型——即使对于默认显示精选列表的功能也是如此。

<Note>
**Insights** 使用两个独立的行——一个用于分析，一个用于摘要。如果为任一行选择不兼容的提供商或非推荐模型，界面会显示警告。
</Note>

### 配置功能访问权限

在 [UI](https://smith.langchain.com):

1. 导航到 **设置** > **模型配置**.
1. 在 **功能访问权限** 表中，找到要配置的功能。
1. 点击 **已启用的提供商** 并为该功能切换提供商的启用或禁用状态。
1. 点击 **可用模型** 并选择用户可以选择的模型。
1. 使用 **默认模型** 下拉菜单，用于设置用户打开功能时预选的模型。

## 配置

该 **配置** 表是工作区的共享模型配置命名库。您在 LangSmith 中创建的配置（包括在 [Playground](/langsmith/managing-model-configurations)）中创建的配置）会显示在此处，您可以在所有功能中重复使用它们。

### 创建配置

1. 导航到 **设置** > **模型配置**.
1. 在 **配置**下，点击 **+ 创建**.
1. 选择一个 **提供商** 和 **模型**.
1. 输入 **API 密钥名称**—工作区中存储提供商 API 密钥的密钥名称。
1. 根据需要调整参数。参数分为以下几组：
    - **标准采样设置**：温度、top P、top K、存在惩罚、频率惩罚、最大输出 tokens
    - **推理**：推理工作强度、服务层级
    - **提供商配置**：提供商 API、基础 URL
    - **选项**：停止序列、种子、JSON 模式、额外请求头、每秒请求数、额外参数

    可用参数因提供商而异——请参阅提供商文档了解详情。
1. 点击 **保存**.

### 编辑配置

1. 在 **配置** 表中，点击该配置旁边的溢出菜单 <Icon icon="dots-vertical" iconType="regular" /> 。
1. 选择 **编辑**.
1. 更新配置并点击 **保存**.

### 删除配置

1. 在 **配置** 表中，点击该配置旁边的溢出菜单 <Icon icon="dots-vertical" iconType="regular" /> 。
1. 选择 <Icon icon="trash" iconType="regular" /> **删除** 并确认。

## OAuth 客户端凭证

<Note>
OAuth 客户端凭证在 LangSmith 上可用 [云端](/langsmith/cloud). [Self-hosted](/langsmith/self-hosted) 支持计划在即将发布的版本中推出。
</Note>

当模型配置位于 OAuth2 网关后面时，您可以直接在配置中存储 OAuth `client_credentials` 凭据，而不是分发静态 API 密钥。LangSmith 在请求时将这些凭据交换为短期 bearer token，将其附加到 `Authorization: Bearer <token>` 出站 LLM 调用上，并在其过期前刷新令牌。这是一种每个配置的自助服务方案，替代通过 [LLM 身份验证代理](/langsmith/llm-auth-proxy-self-hosted)路由工作区；两者在每个配置上是互斥的。

OAuth 客户端凭据在每个支持自定义模型配置的 [计划](/langsmith/pricing-plans) 上都可用。 **使用自定义 OAuth** 开关适用于 bearer token 提供商（OpenAI、Anthropic、OpenAI 兼容端点等），不支持 Bedrock、Google Vertex AI 或 Google GenAI，后者使用原生云身份进行身份验证。对于 **LangServe（已弃用）** preset.

### 也隐藏了该开关。在模型配置上配置 OAuth

配置 OAuth 需要 [工作区管理员](/langsmith/rbac#workspace-admin) 角色，或具有 [自定义角色](/langsmith/rbac#custom-roles) 的 `workspaces:manage-model-configs` 权限的成员。没有此权限的成员会看到 OAuth 字段被禁用，并带有掩码密钥提示。在 [LangSmith UI](https://smith.langchain.com):

1. 导航到 **设置** > **模型配置** ，然后点击 **+ 创建** 或通过溢出菜单打开现有行 <Icon icon="dots-vertical" iconType="regular" /> > **编辑**.
1. 选择兼容的提供商并照常配置模型参数。
1. 切换 **使用自定义 OAuth** on.
1. 填写 OAuth 字段：
    - **令牌 URL**：IdP 令牌端点，例如 `https://login.example.com/oauth/token`.
    - **客户端 ID**：OAuth 客户端标识符。
    - **客户端密钥**：OAuth 客户端密钥。静态加密。
    - **令牌端点身份验证方法**: `client_secret_basic` or `client_secret_post`.
    - **额外参数**: key/value rows sent in the token request body. Use these rows for `scope`, `audience`, `resource`，或 IdP 期望的任何其他参数。发送多个作用域时，每行添加一个值；重复的密钥作为多值对发送。
    - **额外请求头**：与令牌请求一起发送的附加请求头。保留请求头如 `Authorization` 在保存时被拒绝。
1. 点击 **保存**.

### 编辑语义

OAuth 字段遵循保护存储密钥的编辑行为：

- **密钥往返**：服务器将密钥返回为 `********`。输入框渲染为空，并带有一个 *“密钥已设置。输入以替换。”* 提示。不重新输入直接提交会保持存储的密钥不变。
- **关闭开关保留凭据**：切换 **使用自定义 OAuth** 关闭会停用 OAuth 流程但保留存储的字段。重新开启后会继续使用相同的凭据。
- **清除字段**：编辑配置并将该字段留空以明确清除。
- **通过另存为预设来克隆**：将一次性配置保存为新预设时，非密钥 OAuth 字段会复制到新行中。密钥无法转移，因为它从不暴露供读取，因此克隆上的 OAuth 会被强制禁用，直到您重新输入密钥。

### 请求如何流动

当请求针对启用 OAuth 的配置运行时，LangSmith 会通过内部代理生成 bearer 令牌，缓存结果，并在缓存的令牌过期之前在每个出站 LLM 调用上附加该 bearer。

```mermaid actions={false}
sequenceDiagram
    autonumber
    participant Caller as LangSmith service<br/>(Playground, Evals, Insights, Chat, Fleet)
    participant Broker as Token broker
    participant Cache as Redis cache
    participant IdP as Customer IdP
    participant LLM as Upstream LLM provider

    Caller->>Broker: Mint token for configuration X
    Broker->>Cache: Lookup cached token for X
    alt Cache hit
        Cache-->>Broker: Cached token
    else Cache miss or expired
        Broker->>IdP: client_credentials grant
        IdP-->>Broker: access_token + expires_in
        Broker->>Cache: Store until expiry
    end
    Broker-->>Caller: access_token
    Caller->>LLM: Request with Authorization: Bearer <token>
    LLM-->>Caller: Response
```

在 OAuth 和 [LLM 身份验证代理](/langsmith/llm-auth-proxy-self-hosted) 是按配置而非按组织。每个请求根据配置的 OAuth 状态解析到 OAuth 或 LLM 认证代理。单个多模型作业（例如， [洞察](/langsmith/insights) 使用独立的 Thinking 和 Summarization 模型）可以混合两种流程，因为每个模型是独立解析的。

### 回退行为

如果代理无法生成令牌（IdP 不可达、凭证无效、请求准备和执行之间配置被删除），则请求回退到提供商的静态工作区 API 密钥。如果没有设置工作区密钥，则预期在出站调用时收到提供商 401。

令牌轮换仅在缓存的 bearer 过期后才传播。请围绕 IdP 上配置的访问令牌 TTL 来计划轮换。

### 覆盖范围

OAuth 启用的配置在模型配置被使用的任何地方都会被遵循：

- [**Playground**](/langsmith/prompt-engineering-concepts)：聊天运行和实验运行。
- [**Evaluators**](/langsmith/evaluation)：LLM-as-judge 配置、Reuse、Preview Test 和 Evaluator Details Test 在每个提示都解析到 OAuth 启用的配置时，都会跳过工作区密钥提示。
- [**洞察**](/langsmith/insights)：Thinking 和 Summarization 配置是独立解析的。
- [**聊天**](/langsmith/chat)
- [**Fleet**](/langsmith/fleet/index)

当在配置上启用 OAuth 时，LangSmith 不会提示该配置的工作区密钥，因为代理在请求时提供凭证。

### 安全与审计

- **静态加密**：客户端密钥使用与 [工作区密钥](/langsmith/administration-overview#workspaces).
- **Bearer 缓存**：访问令牌会被缓存至过期，且绝不会写入日志。

### FAQ

<Accordion title="Can a single set of OAuth credentials be shared across workspaces?">
不能。OAuth 凭证存储在模型配置中，该配置按工作区作用域划分。每个工作区输入各自的凭证，即使凭证指向同一个 IdP 客户端。
</Accordion>

<Accordion title="Why is my OAuth-enabled configuration suddenly using a static workspace key?">
如果代理无法生成 bearer（IdP 不可达、凭证无效、或请求准备与执行之间配置被删除），则请求回退到提供商的静态工作区 API 密钥。请重新打开模型配置，验证 Token URL 可达、Client ID 和密钥是最新的，以及 Token Endpoint Auth Method 与 IdP 期望的方式匹配。
</Accordion>

<Accordion title="How do I rotate the client secret?">
编辑模型配置，在 **Client Secret（客户端密钥）** 字段中重新输入密钥。先前的密钥在保存时被覆盖。Redis 缓存的 bearer 将继续有效，直至其 TTL 过期，之后代理将使用轮换后的密钥生成新的 bearer。
</Accordion>

<Accordion title="Can OAuth and the LLM auth proxy be used together?">
是的。路由按配置进行。启用 OAuth 的配置使用 OAuth；未启用的配置在组织级别启用代理时会回退到 LLM 认证代理。单个多模型作业可以混合使用这两种流程。
</Accordion>
