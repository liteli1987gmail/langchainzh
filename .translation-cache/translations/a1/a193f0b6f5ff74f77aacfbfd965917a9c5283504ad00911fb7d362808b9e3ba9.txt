构建可实时可视化深度代理工作流的界面。这些模式展示了如何渲染子代理进度、任务规划、流式内容以及由 `createDeepAgent` 创建的代理提供的类 IDE 沙箱体验。

深度代理最为有用之处在于，当界面使委托过程可见时。与其显示一个单一的不透明助手气泡，LangChain SDK 公开了协调器、子代理发现、自定义状态和沙箱支持的产物，让用户可以检查长时间运行的任务是如何被分解和完成的。

<Note>
这些模式使用 v1 前端 SDK 包。如果您使用的是早期版本，请参阅 [React](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-react/docs/v1-migration.md)、[Vue](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-vue/docs/v1-migration.md)、[Svelte](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-svelte/docs/v1-migration.md) 和 [Angular](https://github.com/langchain-ai/langgraphjs/blob/main/libs/sdk-angular/docs/v1-migration.md) 的迁移指南。
</Note>

## 架构

Deep Agents 采用协调器-工作者架构。主代理规划任务并委托给专门的子代理，每个子代理在隔离环境中运行。在前端，v1 流处理程序在根流上公开协调器消息，并为作用域子代理视图提供子代理发现快照。

在前端，使用 @[`useStream`] 连接，方式与 `createAgent` 相同。传入一个[类型参数](/oss/langchain/frontend/overview)以获得类型安全的流状态。深度 Agent 模式使用 `stream.subagents`、选择器辅助函数（如 `useMessages(stream, subagent)`）以及自定义状态值（如 `stream.values.todos`）来渲染子 Agent 特定的 UI。

```ts
import { useStream } from "@langchain/react";

function App() {
  const stream = useStream<typeof agent>({
    apiUrl: "http://localhost:2024",
    assistantId: "agent",
  });

  // Deep agent state beyond messages
  const todos = stream.values?.todos;
  const subagents = [...stream.subagents.values()];
}
```

## SDK 暴露的内容

深度智能体 UI 通常需要的不只是最终答案。前端 SDK 为您提供用户关心的运行环节的结构化投射：

| 投射 | 用于 |
| --- | --- |
| `stream.messages` | 协调器对话和最终综合。 |
| `stream.subagents` | 实时发现专家工作线程，包括状态和任务元数据。 |
| `stream.values` | 共享状态，如待办事项、计划、报告章节、沙盒元数据，或您的智能体写入的任何自定义键。 |
| 工具调用状态 | 将文件系统、搜索、浏览器或域工具呈现为带有进度和结果的卡片。 |
| 中断 | 暂停委托的工作以等待用户批准或缺失输入，同时不丢失运行状态。 |

这让您能够构建更像 IDE、任务看板或工作流监控器的界面，而不是简单的聊天记录。

## 相关模式

[LangChain 前端模式](/oss/langchain/frontend/overview)，包括 markdown 消息、工具调用和人机交互，都同样适用于深度智能体。深度智能体构建在同一个 LangGraph 运行时之上，因此 `useStream` 提供相同的核心 API。

如需更低级别的图形可视化，请参阅 [LangGraph 前端模式](/oss/langgraph/frontend/overview)。它们展示了如何将图形节点和状态键直接映射到 UI 组件。
