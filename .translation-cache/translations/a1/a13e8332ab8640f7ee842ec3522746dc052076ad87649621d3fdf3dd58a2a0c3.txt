本页面涵盖 LangSmith 中评估的两个方面：

1. **[评估类型](#offline-evaluation-types)**：_何时_以及_为何_进行评估。离线评估类型（基准测试、单元测试、回归测试）用于部署前测试，在线评估类型（监控、异常检测）用于生产环境。
2. **[评估器实现](#implement-evaluators)**：_如何_进行评估。可用的评估器方法（LLM-as-judge、代码、复合、汇总、配对）以及配置位置（UI 或 SDK、离线或在线）。

理解这两个方面有助于您构建全面的评估策略，在部署前验证功能并在生产环境中监控质量。

## 离线评估类型

离线评估在部署前使用精选数据集测试应用程序。通过在有参考输出的示例上运行评估，团队可以比较版本、验证功能，并在将更改暴露给用户之前建立信心。

使用 LangSmith SDK（@[Python][langsmith-python] 或 @[TypeScript][langsmith-js]）在客户端运行离线评估，或通过 [Playground](/langsmith/prompt-engineering-concepts#playground) 或[将评估器绑定到数据集](/langsmith/bind-evaluator-to-dataset)在服务端运行。

![离线](/langsmith/images/offline.png)

### 基准测试

_基准测试_在精选数据集上比较多个应用程序版本，以识别最佳表现者。此过程包括创建代表性输入数据集、定义性能指标以及测试每个版本。

基准测试需要使用黄金标准参考输出进行数据集策划，并设计良好的比较指标。例如：
- **RAG 问答机器人**：问题和参考答案的数据集，使用 LLM-as-judge 评估器检查实际答案和参考答案之间的语义等价性。
- **ReAct 代理**：用户请求和参考工具调用的数据集，使用启发式评估器验证是否进行了所有预期的工具调用。

### 单元测试

_单元测试_验证单个系统组件的正确性。在 LLM 上下文中，[单元测试通常是针对输入或输出的基于规则的断言](https://hamel.dev/blog/posts/evals/#level-1-unit-tests)（例如，验证 LLM 生成的代码能编译、JSON 成功加载），用于验证基本功能。

单元测试通常期望一致通过的结果，因此适用于 CI 管道。在 CI 中运行时，配置缓存以减少 LLM API 调用和相关成本。

更多详情请参阅 [Pytest](/langsmith/pytest) 和 [Vitest/Jest](/langsmith/vitest-jest) 页面。

### 回归测试

_回归测试_测量应用程序版本随时间的性能一致性。它们确保新版本不会在当前版本正确处理的案例上降低性能，并理想情况下展示相对于基线的改进。这些测试通常在进行预期会影响用户体验的更新时运行（例如，模型或架构变更）。

LangSmith 的比较视图突出显示相对于基线的回归（红色）和改进（绿色），实现变更的快速识别。

![比较视图](/langsmith/images/comparison-diff-view-light.png)

### 回测

_回测_使用历史生产数据评估新应用程序版本。生产日志被转换为数据集，然后较新版本处理这些示例以评估在过去的、真实的用户输入上的性能。

此方法常用于评估新模型发布。例如，当新模型可用时，在最近的生产运行上测试它，并将结果与实际生产结果进行比较。

### 配对评估

_配对评估_通过确定相对质量来比较两个版本的输出，而不是分配绝对分数。对于某些任务，[确定"版本 A 比 B 好"](https://www.oreilly.com/radar/what-we-learned-from-a-year-of-building-with-llms-part-i/) 比独立评分每个版本更容易。

这种方法在主观任务的 LLM-as-judge 评估中特别有用。例如，在摘要中，确定"哪个摘要更清晰、更简洁？"通常比分配数字清晰度分数更简单。

了解[如何运行配对评估](/langsmith/evaluate-pairwise)。

## 在线评估类型

在线评估近乎实时地评估生产应用程序输出。没有参考输出，这些评估专注于检测问题、监控质量趋势以及识别边缘情况，为未来的离线测试提供信息。

在线评估器通常在服务端运行。LangSmith 提供内置的 [LLM-as-judge 评估器](/langsmith/llm-as-judge)用于配置，并支持在 LangSmith 内运行的自定义代码评估器。

![在线](/langsmith/images/online.png)

### 实时监控

在用户与系统交互时持续监控应用程序质量。在线评估自动在生产流量上运行，为每个交互提供即时反馈。这使得能够在质量下降、异常模式或意外行为影响大量用户之前检测到它们。

### 异常检测

识别偏离预期模式的异常值和边缘情况。在线评估器可以标记具有异常特征的运行——异常长或短的响应、意外的错误率，或未能通过安全检查的输出——以供人工审查并可能添加到离线数据集中。

### 生产反馈循环

利用生产洞察改进离线评估。在线评估揭示了可能在精选数据集中不会出现现实世界问题和使用模式。失败的生产运行成为数据集示例的候选者，创建了一个迭代循环，生产经验不断改进测试覆盖率。

## 实现评估器

上述评估类型描述了_何时_进行评估。LangSmith 提供了多种方法来实现跨这些评估类型工作的评估器。

### LLM-as-a-judge

使用 LLM 根据提示中定义的标准对输出进行评分。这种方法非常适用于语气、清晰度或语义正确性等难以用确定性规则捕获的主观质量。

常见用例包括对照参考输出评估事实准确性（离线）或检查生产响应中的毒性（在线）。例如，对 RAG 系统进行基准测试可能使用 LLM-as-judge 评估器来检查生成答案和参考答案之间的语义等价性。

为 LLM-as-judge 评估器配置：
- 程序化离线评估：[使用 SDK](/langsmith/llm-as-judge-sdk)
- 数据集上的离线评估：[在 UI 中](/langsmith/llm-as-judge)
- 生产追踪上的在线评估：[在 UI 中](/langsmith/online-evaluations-llm-as-judge)

### 代码评估器

编写确定性的、基于规则的函数来检查特定条件。这些评估器执行自定义逻辑来验证结构、检查模式或应用业务规则。

代码评估器对单元测试特别有用——验证生成的代码编译、JSON 正确解析或存在必需字段。在回归测试中，它们可以跟踪结构化输出的一致性。对于在线监控，它们实时捕获格式违规。

为代码评估器定义：
- 数据集上的离线评估：[在 UI 中](/langsmith/code-evaluator-ui)
- 程序化离线评估：[使用 SDK](/langsmith/code-evaluator-sdk)
- 生产追踪上的在线评估：[在 UI 中](/langsmith/online-evaluations-code)

### 复合评估器

使用加权平均值或求和将多个评估器分数组合成单个指标。这创建了反映多个评估标准同时的聚合质量分数。

对于基准测试，复合分数有助于在多个维度上比较版本（例如，70% 准确率 + 20% 清晰度 + 10% 简洁性）。在在线监控中，它们为仪表板和警报提供单一指标。例如，将整体聊天机器人质量作为有用性、正确性和语气分数的加权组合进行跟踪。

设置复合评估器：
- 预定义聚合的离线评估：[在 UI 中](/langsmith/composite-evaluators-ui)
- 自定义聚合逻辑的离线评估：[使用 SDK](/langsmith/composite-evaluators-sdk)
- 生产追踪上的在线评估：[在 UI 中](/langsmith/online-evaluations-composite)

### 汇总评估器

在整个实验而非单个示例上计算指标。这些评估器接收数据集的所有输出并计算聚合统计数据，如精确率、召回率、F1 分数或分布分析。

汇总评估器对于需要数据集级指标的基准测试至关重要——跨版本比较整体性能，而不是逐个示例评分。它们仅与离线评估配合使用，因为需要处理完整数据集。

实现汇总评估器：
- 离线评估的自定义聚合函数：[使用 SDK](/langsmith/summary)

### 配对评估器

比较两个版本的输出以确定相对质量。这种方法在前面的[配对评估](#pairwise-evaluation)中介绍过，当绝对评分困难但确定"哪个更好"很直接时很有帮助。

运行配对评估：
- 比较现有实验：[使用 SDK](/langsmith/evaluate-pairwise)
