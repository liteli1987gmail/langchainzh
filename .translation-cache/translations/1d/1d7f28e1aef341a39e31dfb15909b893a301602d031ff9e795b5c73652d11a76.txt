LangSmith 可观测性让您可以记录、检查和分析 LLM 应用程序执行的每个步骤。本页面介绍 LangSmith 中的数据结构以及如何发送追踪。

## LangSmith 如何组织数据

LangSmith 将多个 [_追踪_](#traces) 归入一个 [_项目_](#projects)。每个追踪记录应用程序针对单个操作执行的步骤序列，这些步骤由独立的 [_运行_](#runs)组成。您可以将多轮对话的追踪链接为一个 [_线程_](#threads).

```mermaid actions={false}
graph TB
    Project -->|contains| Trace1[Trace]
    Project -->|contains| Trace2[Trace]
    Trace1 -->|contains| Run1[Run]
    Trace1 -->|contains| Run2[Run]
    Trace1 -->|contains| Run3[Run]
    Trace1 -->|part of| Thread
    Trace2 -->|part of| Thread
```

### 项目

A _项目_ 是包含单个应用程序或服务所有相关追踪的容器。

[将追踪记录到项目](/langsmith/log-traces-to-project).

### 追踪

A _追踪_ 是单个操作的运行集合。例如，如果用户请求触发了一个链，该链调用了 LLM 然后调用了输出解析器，所有这些运行都属于同一个追踪。运行通过唯一的追踪 ID 绑定到追踪。如果您熟悉 [OpenTelemetry](https://opentelemetry.io/)，可以将 LangSmith 追踪视为 spans 的集合。

<Note><MaxRunsPerTrace /></Note>

### 运行

A _运行_ 是 LLM 应用程序中单个工作单元的 span：调用 LLM、格式化提示词、检索调用或任何其他离散操作。如果您熟悉 [OpenTelemetry](https://opentelemetry.io/)，可以将运行视为一个 span。

### 线程

A _线程_ 是表示单个对话的追踪序列。多轮对话中的每一轮都是自己的追踪，但追踪通过共享标识符链接。要将追踪分组到线程中，请传递带有唯一值的特殊元数据键 (`session_id`, `thread_id`, or `conversation_id`)。

[了解如何配置线程](/langsmith/threads).

<Callout type="info" icon="feather">
使用 **[Chat](/langsmith/chat)** 分析追踪、运行和线程。Chat 帮助您了解代理性能、调试问题并从对话线程中获取洞察，而无需手动深入挖掘数据。
</Callout>

## 追踪增强

### 反馈

_反馈_ 允许您根据某些标准对单个运行进行评分。每个反馈条目由标签和分数组成，并通过唯一的运行 ID 绑定到运行。反馈可以是连续的或离散的（分类的），标签可以在组织内的运行中重复使用。

有关反馈存储方式的更多信息，请参阅 [反馈数据格式指南](/langsmith/feedback-data-format).

### 标签

_标签_ 是您可以附加到运行上的字符串，用于在 LangSmith UI 中对运行进行分类、筛选和分组。

[了解如何将标签附加到追踪](/langsmith/add-metadata-tags).

### 元数据

_元数据_ 是一个键值对集合，可附加到运行。例如，应用程序版本、环境或任何其他上下文信息。与标签类似，您可以使用元数据来过滤和分组运行。

[了解如何向您的轨迹添加元数据](/langsmith/add-metadata-tags).

## 发送轨迹

向 LangSmith 发送轨迹数据有两种方式。

### 集成

LangSmith _集成_ 为流行的 LLM 提供商和代理框架提供自动追踪（相当于通用可观测性中的自动插桩）。当您使用支持的框架（如 LangChain、LangGraph、OpenAI、Anthropic 或 CrewAI）时，集成会捕获输入、输出和元数据，无需手动更改代码。

[浏览所有集成](/langsmith/integrations).

### 手动插桩

_手动插桩_ 允许您向任何代码添加追踪，不受框架限制。当您没有使用支持的集成或需要精细控制追踪内容时使用。LangSmith 提供三种机制：

- `@traceable` / `traceable`：一个用于追踪任何函数的装饰器
- `trace` 上下文管理器（Python）：包装特定代码块
- `RunTree` API：低级、显式追踪构造

[了解如何添加手动插桩](/langsmith/annotate-code).

## 数据保留

LangSmith (SaaS) 从数据获取之日起保留轨迹数据 400 天。超过此期限后，轨迹将被永久删除，仅保留有限的元数据用于使用统计信息。 [使用和计费：数据保留](/langsmith/usage-and-billing#data-retention).

<Note>
要将数据保留超过保留期限，请将其添加到 [数据集](/langsmith/manage-datasets)。数据集会无限期保留，即使源轨迹被删除也不会受影响。
</Note>

如需在到期日前删除轨迹，请参阅 [管理轨迹](/langsmith/manage-trace#delete-a-trace).
