该列表包含 LangSmith 中的 API 操作以及：

- 可执行每个操作的系统角色。
- 所需的具体权限字符串。
- 关于部分访问或特殊情况的说明。

<Info>
有关 LangSmith RBAC 系统、角色定义和权限概念的概述，请参阅[基于角色的访问控制](/langsmith/rbac)。
</Info>

## 图例

- ✓ **允许**：具有此角色的用户可以执行此操作
- ✗ **不允许**：具有此角色的用户不能执行此操作
- ⚠ **部分**：用户访问受限（见说明）

## 权限继承

### 组织到工作区

- [组织管理员](/langsmith/rbac#organization-admin) 在所有工作区自动拥有完全权限。
- [组织操作员](/langsmith/rbac#organization-operator) 仅在明确添加到具有工作区级角色的工作区时获得工作区访问权限（或他们创建的工作区）。
- [组织用户](/langsmith/rbac#organization-user) 和 [组织查看者](/langsmith/rbac#organization-viewer) 仅在明确添加到具有工作区级角色的工作区时获得工作区访问权限。

有关详细角色定义，请参阅[组织角色](/langsmith/rbac#organization-roles)和[工作区角色](/langsmith/rbac#workspace-roles)。

### 工作区角色独立性

- 用户可以在不同工作区拥有不同的工作区角色。
- 用户可能在一个工作区是[工作区管理员](/langsmith/rbac#workspace-admin)，而在另一个工作区是[工作区查看者](/langsmith/rbac#workspace-viewer)。
