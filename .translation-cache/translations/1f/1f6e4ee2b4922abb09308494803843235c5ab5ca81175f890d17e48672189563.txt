本指南从头开始构建数据分析代理，使用 `create_agent` 和 Deep Agents 中间件。

`create_agent` 和 `create_deep_agent` 都为你提供了对工具、内存等的细粒度控制。
两者的主要区别在于 Deep Agents 已经内置了一系列常用功能，如规划、文件系统工具和子代理。
如果 Deep Agents 的默认框架不符合你的需求，本指南将展示如何从 `create_agent` 开始，逐步组装框架，这样你可以清楚地看到每个组件的作用，并且只添加你需要的部分。

按照本指南构建一个代理，实现以下功能：

1. 接收 CSV 文件进行分析
2. 在隔离的沙箱中编写和执行 Python 代码
3. 将可视化工作委托给专门的子代理
4. 从技能文件中加载数据分析模式

最终堆栈与 `create_deep_agent` 默认组装的内容相同。

## 你将学到什么

每个步骤为同一个数据分析代理添加一项功能：

| 步骤 | 没有它的问题 | 添加的内容 |
|------|-------------------|--------------|
| 最小代理 | — | 基线循环：模型 + 工具，无框架 |
| 沙箱 + 文件系统 | 代理无法读取 CSV 或运行 Python | 隔离的[后端](/oss/deepagents/backends) + 文件和执行工具 |
| 摘要 | 长会话达到上下文限制 | 自动历史压缩 |
| 技能 | 领域规则使系统提示膨胀 | 通过[渐进式披露](/oss/langchain/multi-agent/skills-sql-assistant)按需获取专业知识 |
| 子代理 | 图表迭代挤占主线程 | 隔离的工作线程 + 并行委托 |

## 添加沙箱后端

为了高效分析数据，Agent 需要在文件上运行代码。这需要两件事：

- 一个隔离的 [sandbox](/oss/deepagents/sandboxes)，Agent 可以在其中放置文件并对文件运行代码，而无需访问你的主机。
- 一个 [backend](/oss/deepagents/backends)，它提供文件系统工具来使用 @[`FilesystemMiddleware`] 处理沙箱中的文件（`read_file`、`write_file`、`edit_file`、`glob` 和 `grep`）：**。由于 `LangSmithSandbox` 后端实现了沙箱协议，@[`FilesystemMiddleware`] 也会添加 `execute` 工具，允许 Agent 运行 shell 命令。

@[`LangSmithSandbox`] 是文件存放和命令运行的地方。@[`FilesystemMiddleware`] 是将该环境作为工具暴露给模型的内容。如果你稍后交换后端，相同的中间件也适用于其他后端。

## 添加上下文管理

在步骤 2 之后，每个工具结果都保留在消息历史中。实际的分析会话（多个图表、失败的脚本、大量 `read_file` 输出）会迅速填满上下文窗口。

@[`SummarizationMiddleware`] 会在历史记录增长过大时压缩较早的对话轮次，这样代理可以持续工作而无需你手动修剪消息。这对于第一个 `sales.csv` 问题影响不大，但对于后续问题（例如"现在按产品细分并绘制每月趋势"）更为重要。

## 添加技能

[技能](/oss/langchain/multi-agent/skills-sql-assistant)提供了一种按需向代理赋予领域知识的方式，采用渐进式披露。技能可以包括多步骤工作流、规则和约定。将此信息放在技能中，默认情况下不会添加到系统提示中，这样确保只有在任务需要技能中的信息时才使用相关令牌。

当代理启动时，它只看到每个技能的轻量级元数据。当任务需要某个技能时，代理会按需加载完整的技能文件。

在 skills 目录中创建一个技能文件：

```
skills/
  pandas-patterns/
    SKILL.md
```

```markdown
---
name: pandas-patterns
description: Common pandas and matplotlib patterns for data analysis and visualization
---

## Data loading
Use `pd.read_csv()` for CSV files. Always check `df.info()` and `df.describe()` first.

## Visualization
Use `matplotlib` for bar charts, `seaborn` for statistical plots.
Save figures with `plt.savefig("output.png", dpi=150, bbox_inches="tight")`.

## Reporting
Write a markdown summary to `report.md` alongside any generated charts.
```

此技能包含有关如何进行可视化操作的信息。

## 添加可视化子智能体

某些任务会产生大量中间输出（脚本草稿、失败的运行、文件读取），如果保留在一个线程中，会挤占主智能体的上下文。[subagent](/oss/deepagents/subagents) 在其自己的上下文窗口中运行，因此监督者只看到最终结果，而非沿途的每个工具调用。这使得主分析保持专注，并为后续问题留出空间。

使用子智能体有意义的一个例子是图表生成。绘图通常意味着迭代 Python 脚本、安装包，以及在图表准备好之前读取错误输出。下面的 `visualizer` 子智能体可以隔离处理这些工作，而主智能体继续进行规划和分析。通过 @[`TodoListMiddleware`]，主智能体还可以并行委派图表工作，而不是在每个图表上阻塞。
