本指南从第一性原理出发，使用 `create_agent` 和 Deep Agents 中间件构建数据分析智能体。

两者都 `create_agent` 和 `create_deep_agent` 为您提供了对工具、内存等的细粒度控制。
两者之间的主要区别在于，Deep Agents 已经内置了一系列常用功能，如规划、文件系统工具和子智能体。
如果 Deep Agents 默认的 harness 不符合您的需求，本指南将向您展示如何从 `create_agent` 开始，逐步组装 harness，以便您可以清楚地看到每个组件的作用，并仅替换您用例所需的部分。

按照本指南构建一个能够执行以下操作的智能体：

1. 接收 CSV 文件进行分析
2. 在隔离的沙箱中编写和执行 Python 代码
3. 将可视化工作委托给专门的子智能体
4. 从技能文件中加载数据分析模式

最终的堆栈与 `create_deep_agent` 默认组装的结构相同。

## 您将学到的内容

每个步骤为同一个数据分析智能体添加一项功能：

| 步骤 | 没有它的困扰 | 添加的内容 |
|------|-------------------|--------------|
| 最小智能体 | — | 基线循环：模型 + 工具，无测试框架 |
| 沙箱 + 文件系统 | 智能体无法读取 CSV 或运行 Python | 隔离环境 | [后端](/oss/deepagents/backends) + 文件和执行工具 |
| 摘要总结 | 长会话会触及上下文限制 | 自动历史压缩 |
| 技能 | 领域规则使系统提示膨胀 | 按需专业知识通过 [渐进式披露](/oss/langchain/multi-agent/skills-sql-assistant) |
| 子智能体 | 图表迭代挤占主线程 | 隔离工作线程 + 并行委托 |

## 添加沙盒后端

为了高效分析数据，智能体需要在文件上运行代码。这需要两样东西：

- 一个隔离的 [沙盒](/oss/deepagents/sandboxes) 智能体可以在其中放置文件并在文件上运行代码的环境，而不会获得对您主机的访问权限。
- A [后端](/oss/deepagents/backends) 它提供文件系统工具来操作沙盒（`read_file`, `write_file`, `edit_file`, `glob`，以及 `grep`）使用 @[`FilesystemMiddleware`]:**。因为 `LangSmithSandbox` 后端实现了沙盒协议，@[`FilesystemMiddleware`] 还添加了 `execute` 工具，该工具允许智能体运行 shell 命令。

@[`LangSmithSandbox`] 是文件存放和命令运行的位置。@[`FilesystemMiddleware`] 用于将环境作为工具暴露给模型。如果稍后更换后端，相同的中间件可与其他后端配合工作。

## 添加上下文管理

在步骤2之后，每个工具结果都保留在消息历史中。一个真实的分析会话（多个图表、失败的脚本、大的 `read_file` 输出）会迅速填满上下文窗口。

@[`SummarizationMiddleware`] 在历史记录增长过大时压缩较早的轮次，这样代理就可以继续工作，而无需你手动修剪消息。这在第一个 `sales.csv` 问题上影响较小，但在后续问题（如“现在按产品细分并绘制每月趋势”）上影响更大。

## 添加技能

[技能](/oss/langchain/multi-agent/skills-sql-assistant) 提供了一种使用渐进式披露的方式，在需要时向代理提供按需的领域知识。技能可以包括多步骤工作流程、规则和约定。通过将此信息放入技能中，它默认不会被添加到系统提示中，这确保了令牌仅在任务需要技能中的信息时才被使用。

当代理启动时，它只看到每个技能的轻量级元数据。当任务需要某个技能时，代理会按需加载完整的技能文件。

在技能目录中创建一个技能文件：

```
skills/
  pandas-patterns/
    SKILL.md
```

```markdown
---
name: pandas-patterns
description: Common pandas and matplotlib patterns for data analysis and visualization
---

## Data loading
Use `pd.read_csv()` for CSV files. Always check `df.info()` and `df.describe()` first.

## Visualization
Use `matplotlib` for bar charts, `seaborn` for statistical plots.
Save figures with `plt.savefig("output.png", dpi=150, bbox_inches="tight")`.

## Reporting
Write a markdown summary to `report.md` alongside any generated charts.
```

此技能包含有关如何进行可视化的信息。

## 添加可视化子代理

某些任务会产生大量中间输出（脚本草稿、失败的运行、文件读取），如果保留在一个线程中，会挤占主代理的上下文。 [子代理](/oss/deepagents/subagents) 在自己的上下文窗口中运行，因此监督者只看到最终结果，而不是过程中的每个工具调用。这样可以保持主分析的专注性，并为后续问题留出空间。

使用子代理的一个典型场景是图表生成。绘图通常需要反复修改 Python 脚本、安装依赖包、读取错误输出，直到图表最终生成。 `visualizer` 子代理可以在隔离环境中处理这些工作，而主代理继续进行规划和分析。通过 @[`TodoListMiddleware`]，主代理也可以并行委托这些图表工作，而不是阻塞在每个图表上。
