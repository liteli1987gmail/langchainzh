## 设置与安装

要使用此功能，请安装 `langchain-hana` package:

```python
pip install langchain_hana
```

然后，创建到 SAP HANA Cloud 实例的连接。

```python
import os

from dotenv import load_dotenv
from hdbcli import dbapi

# Load environment variables if needed
load_dotenv()

# Establish connection to SAP HANA Cloud
connection = dbapi.connect(
    address=os.environ.get("HANA_DB_ADDRESS"),
    port=os.environ.get("HANA_DB_PORT"),
    user=os.environ.get("HANA_DB_USER"),
    password=os.environ.get("HANA_DB_PASSWORD")
)
```

`HanaSparqlQAChain` 整合了：

1. **支持架构的 SPARQL 生成**
2. **查询执行** 针对 SAP HANA
3. **自然语言答案格式化**

## 初始化

您需要：

* An **LLM** 用于生成和解释查询
* A **`HanaRdfGraph`** （使用连接， `graph_uri`和本体）

按照此处的步骤 [HanaRdfGraph](/oss/integrations/graphs/sap_hana_rdf_graph) 了解更多关于创建 `HanaRdfGraph` instance.

导入 `HanaSparqlQAChain`

```python
from langchain_hana import HanaSparqlQAChain
```

```python
qa_chain = HanaSparqlQAChain.from_llm(
    llm=llm, graph=graph, allow_dangerous_requests=True, verbose=True
)
```

## 管道概览

1. **SPARQL 生成**
   * 使用 `SPARQL_GENERATION_SELECT_PROMPT`
   * Inputs:
     * `schema` （来自 Turtle `graph.get_schema`)
     * `prompt` （用户问题）
2. **查询后处理**
   * 从 LLM 输出中提取 SPARQL 代码。
   * 注入 `FROM <graph_uri>` 如果缺失
   * 确保声明所需通用前缀（`rdf:`, `rdfs:`, `owl:`, `xsd:`)
3. **执行**
   * 调用 `graph.query(generated_sparql)`
4. **答案生成**
   * 使用 `SPARQL_QA_PROMPT`
   * Inputs:
     * `context` （原始查询结果）
     * `prompt` （原始问题）

## 提示模板

### “SPARQL 生成”提示

 `sparql_generation_prompt` 用于根据用户问题和提供的架构引导 LLM 生成 SPARQL 查询。

### 问答提示

 `qa_prompt` 指示 LLM 仅根据数据库结果生成自然语言答案。

默认提示可在此处找到： [`prompts.py`](https://github.com/SAP/langchain-integration-for-sap-hana-cloud/blob/main/langchain_hana/chains/graph_qa/prompts.py)

## 自定义提示

您可以在初始化时覆盖默认值：

```python
qa_chain = HanaSparqlQAChain.from_llm(
    llm=llm,
    graph=graph,
    allow_dangerous_requests=True,
    verbose=True,
    sparql_generation_prompt=YOUR_SPARQL_PROMPT,
    qa_prompt=YOUR_QA_PROMPT
)
```

> - `sparql_generation_prompt` 必须包含输入变量： `["schema", "prompt"]`
> - `qa_prompt` 必须包含输入变量： `["context", "prompt"]`

## 示例：针对“电影”知识图的问答

**前置条件**:
您必须拥有启用 **三元组存储** 功能的 SAP HANA Cloud 实例。
有关详细说明，请参阅： [启用三元组存储](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/enable-triple-store/)<br />
加载 `kgdocu_movies` 示例数据。请参阅 [知识图谱示例](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/knowledge-graph-example).

下面我们将：

1. 实例化 `HanaRdfGraph` 指向我们的“movies”数据图
2. 将其包装在 `HanaSparqlQAChain` 由 LLM 驱动
3. 提出自然语言问题并打印链路的响应

这演示了 LLM 如何在后台生成 SPARQL，对 SAP HANA 执行查询，并返回人类可读的答案。

## 幕后发生了什么？

1. **SPARQL 生成**
   该链使用 Turtle 格式的本体论（`graph.get_schema`）和用户的问题，通过 `SPARQL_GENERATION_SELECT_PROMPT`. 然后 LLM 会生成有效的 `SELECT` 查询语句，该查询会根据您的架构进行定制。
2. **预处理与执行**
   * **提取与清洗**：从 LLM 的响应中提取原始 SPARQL 文本。
   * **注入图上下文**：添加 `FROM <graph_uri>` （如果缺失）并确保声明了常用前缀（`rdf:`, `rdfs:`, `owl:`, `xsd:`）。
   * **在 HANA 上运行**：通过 `HanaRdfGraph.query()` 在您的命名图上执行最终确定的查询。
3. **答案组织**
   返回的 CSV（或 Turtle）结果会再次输入到 LLM——这次带有 `SPARQL_QA_PROMPT`。LLM 会根据检索到的数据生成简洁、可读的答案，不会产生幻觉内容。
