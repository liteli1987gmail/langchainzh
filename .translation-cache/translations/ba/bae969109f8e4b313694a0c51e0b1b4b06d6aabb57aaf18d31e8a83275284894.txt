此页面列出了自托管 LangSmith 所使用的数据库、工具和基础设施的最低支持版本。在部署或升级之前，请安装满足或超过最低要求的版本。使用低于最低要求的版本不受支持，可能会导致安装或运行时失败。

<Tip>
如果您使用的是托管服务（如 Amazon RDS、Google Cloud SQL 或 Azure Database），请选择等于或高于此处列出的最低版本。请参阅链接的“外部服务”页面以获取连接和身份验证详情。
</Tip>

## 数据存储

| 依赖项 | 最低版本 | 说明 |
|------------|------------------|-------|
| [PostgreSQL](/langsmith/self-host-external-postgres) | 14 | 操作数据的主要关系存储。LangSmith 和独立 Agent Server 部署都需要此组件。用于安装 `btree_gin`、`btree_gist`、`pgcrypto`、`citext`、`ltree` 和 `pg_trgm` 扩展。 |
| [Redis](/langsmith/self-host-external-redis) | 5 | 用于队列和缓存。独立模式和 Redis Cluster 模式均支持。 |
| [Valkey](/langsmith/self-host-external-redis) | 8 | 作为 Redis 的直接替代品正式支持。独立模式和 Cluster 模式均支持。 |
| [ClickHouse](/langsmith/self-host-external-clickhouse) | [LangSmith Helm chart](https://github.com/langchain-ai/helm/releases) 中指定的版本或更高版本 | 存储跟踪和反馈。ClickHouse 版本 >= 24.2 需要 LangSmith v0.6 或更高版本。不支持降级。 |

<Warning>
**不支持 Redis < 5 和 PostgreSQL < 14。** 指向旧版 Redis 或 PostgreSQL 实例的 LangSmith 安装可能无法启动或行为异常。在安装或升级 LangSmith 之前，请先升级您的数据存储。
</Warning>

## 计算和编排

| 依赖项 | 最低版本 | 说明 |
|------------|------------------|-------|
| [Kubernetes](/langsmith/kubernetes) | [上游 Kubernetes 发布周期](https://kubernetes.io/releases/) 支持的任何版本 | LangSmith 定期在 GKE、EKS、AKS、Minikube 和 Kind 上进行测试。 |
| [OpenShift](/langsmith/kubernetes) | 4.14 | 支持作为 LangSmith 的 Kubernetes 发行版。 |
| [Helm](https://helm.sh/docs/intro/install/) | 3 | 用于安装和升级 LangSmith Helm chart。 |
| Docker | 与 [Docker Compose v2](https://docs.docker.com/compose/) 兼容的版本 | Docker 独立部署 Agent Server 所需。 |

## LangSmith chart 和平台

| 依赖项 | 最低版本 | 说明 |
|------------|------------------|-------|
| [LangSmith Helm chart](https://github.com/langchain-ai/helm/releases) | 最新稳定版 | 我们建议绑定到最新的稳定 chart 版本。请参阅[自托管更新日志](/langsmith/self-hosted-changelog)以获取每个版本的升级说明。 |
| 出站流量到 `https://beacon.langchain.com` | LangSmith 0.9.0 或更高版本 | 除非在[离线模式](/langsmith/self-host-egress)下运行，否则许可证验证和使用报告需要此功能。 |

## 这些版本的执行位置

- PostgreSQL `>= 14`：请参阅[连接到外部 PostgreSQL 数据库](/langsmith/self-host-external-postgres#requirements)。
- Redis `>= 5` 和 Valkey `8`：请参阅[连接到外部 Redis 或 Valkey 数据库](/langsmith/self-host-external-redis#requirements)。
- ClickHouse：使用 [LangSmith Helm chart](https://github.com/langchain-ai/helm/releases) 中指定的版本或更高版本：请参阅[连接到外部 ClickHouse 数据库](/langsmith/self-host-external-clickhouse#requirements)。
- Kubernetes 集群前提条件：请参阅[在 Kubernetes 上自托管 LangSmith](/langsmith/kubernetes#prerequisites)。

如果您不确定当前运行的版本，请联系您的数据库管理员或参阅云提供商的控制台。如果您需要在升级 LangSmith 之前升级依赖项，请参阅供应商的文档。
