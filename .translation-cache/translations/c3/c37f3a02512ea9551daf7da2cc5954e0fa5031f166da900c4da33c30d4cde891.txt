<Note>
**私人测试版：** LLM Gateway 目前处于私人[测试版](/langsmith/release-stages)。[注册候补名单](https://www.langchain.com/langsmith-llm-gateway-waitlist)以获取访问权限。
</Note>

为您的 LangSmith [组织](/langsmith/administration-overview#organizations) 启用 LLM Gateway 的一次性设置。[组织管理员](/langsmith/rbac#organization-admin) 应在个人用户通过网关路由调用之前完成此操作。

## 前提条件

您需要在 LangSmith 中拥有 [`organization:manage` 权限](/langsmith/organization-workspace-operations)。[第 3 步选项 A](/langsmith/llm-gateway-admin-setup#option-a-create-a-custom-workspace-role-recommended) 还需要包含 [RBAC](/langsmith/rbac)（自定义角色）的计划。

## 1. 启用 LLM Gateway

在私人测试期间，网关必须由 LangChain 为您的组织启用。一旦您被接受进入测试版，您的组织将开启 `llm_gateway_enabled` 功能标志。

在正式发布时，这将成为自助服务设置。

## 2. 添加提供商密钥

网关从您工作区的提供商密钥中解析提供商 API 密钥——这是它向上游提供商代理调用的方式，无需个人用户在本地保存提供商密钥副本。

转到 **Settings → Integrations → Provider Secrets** 并添加您想要通过网关代理的提供商的密钥：

| 密钥名称 | 提供商 |
| --- | --- |
| `ANTHROPIC_API_KEY` | Anthropic |
| `AWS_BEARER_TOKEN_BEDROCK` | AWS Bedrock |
| `BASETEN_API_KEY` | Baseten |
| `FIREWORKS_API_KEY` | Fireworks |
| `GOOGLE_API_KEY` | Google Gemini |
| `OPENAI_API_KEY` | OpenAI |
| `VERTEX_SERVICE_ACCOUNT_JSON` | Google Vertex AI |

仅添加您组织使用的提供商。如果用户尝试调用其密钥尚未添加的提供商，网关将返回错误。

## 3. 为用户配置网关访问权限

内置角色 `WORKSPACE_USER` 和 `WORKSPACE_VIEWER` 不包含 `gateway:invoke` 权限，且无法编辑。您有两个选项来授予网关访问权限：

### 选项 A：创建自定义工作区角色（推荐）

需要启用 RBAC 的计划。

1. 转到 **Settings → Members/Roles**。
2. 创建新的工作区角色。
3. 至少授予其 `gateway:invoke` 和 `workspaces:read` 权限。
4. 将需要网关访问权限的用户分配到此角色。

当您想要授予特定用户网关访问权限但又不给他们完整的工作区管理员权限时使用此选项。这使您能够最大程度地控制谁可以使用网关。

### 选项 B：使用工作区管理员角色

无计划要求。

`WORKSPACE_ADMIN` 角色默认已包含 `gateway:invoke` 和 `workspaces:read`。将需要网关访问权限的用户分配到此角色。

如果您不需要细粒度访问控制，或者没有启用 RBAC，请使用此选项。

## 4. 配置策略（可选）

网关策略管理需要 `organization:manage` 权限。

转到 **Settings → Gateway → LLM Gateway** 创建治理策略。您可以配置：

- **消费限额：** 在组织、工作区、API 密钥或用户级别设置硬性上限。请参阅[消费策略](/langsmith/llm-gateway-spend-policies)。
- **PII 和密钥重识别：** 在敏感数据到达模型之前检测并重识别。请参阅 [PII 和密钥重识别](/langsmith/llm-gateway-redaction)。

初始设置期间策略是可选的。在您配置策略之前，网关将自由允许调用。

## 5. 向用户分发 API 密钥

为需要网关访问权限的用户创建工作区范围的[服务密钥](/langsmith/administration-overview#service-keys)。每个密钥应附加到包含 `gateway:invoke` 和 `workspaces:read` 的角色。

使用工作区范围的密钥，而非组织范围的密钥。详细信息请参阅 [API 密钥范围](/langsmith/llm-gateway-access#api-key-scoping)。

与每个用户共享密钥和网关端点，或通过 MDM（移动设备管理）进行分发以实现全公司范围的编码代理部署。逐个代理的配置说明，请参阅[设置编码代理](/langsmith/llm-gateway-coding-agents)。

## 验证

让用户运行[快速入门中的验证 curl](/langsmith/llm-gateway-quickstart#2-make-a-call)。`200` 响应确认网关、API 密钥、提供商密钥和角色权限都已正确配置。调用将显示为工作区中 **gateway** 追踪项目中的追踪记录。
