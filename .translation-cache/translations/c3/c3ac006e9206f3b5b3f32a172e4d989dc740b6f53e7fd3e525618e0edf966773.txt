LangSmith Deployment 可运行任何框架。对于不是基于 Deep Agents、LangChain 或 LangGraph 构建的代理，可以使用 [`deployments-wrap-sdk`](https://pypi.org/project/deployments-wrap-sdk/) 包（Google ADK）或 [LangGraph Functional API](/oss/langgraph/functional-api)（Claude Agent SDK、Strands、CrewAI、AutoGen 和其他库）进行部署。

<Tip>
对于新项目，请考虑 [Deep Agents](/oss/deepagents/overview)，这是一个用于规划、使用工具、委派给子代理以及长时间工作的开源框架。Deep Agents 可直接部署到 LangSmith Deployment，并提供 [托管 Deep Agents](/langsmith/deploy-managed-deep-agent) 作为完全托管的运行时。
</Tip>

## Functional API 的工作原理

当 Agent Server 收到针对 Functional API 包装代理的运行时：

1. 平台使用运行输入和同一线程上先前轮次的任何保存状态（作为 `previous` 参数传递）调用您的 `@entrypoint` 装饰的 `agent` 函数。
2. 入口点调用您的 `@task` 装饰的函数，该函数委托给框架代理（Claude Agent SDK、Strands、CrewAI、AutoGen 或其他库）。
3. 入口点返回 `entrypoint.final(value=..., save=...)`。`value` 是本轮的响应；`save` 是用作下一轮 `previous` 的检查点状态。
4. Agent Server 持久化检查点、在支持时流式传输部分输出，并在配置了跟踪时记录跟踪。

此模式保留了您的框架的执行语义，同时为您提供标准 Agent Server 功能：持久运行、多线程持久化、流式传输端点和 LangSmith 可观测性。

## 先决条件

无论使用何种框架，您需要：

* Python 3.10+（适用于 Functional API 框架）（Strands Agents 支持 Python 3.9+）
* 一个 [LangSmith API 密钥](/langsmith/create-account-api-key)

## 通用部署模式

<Step title="使用 Functional API 包装">

通过使用 `@entrypoint` 装饰的名为 `agent` 的函数来暴露你的代理。在内部，使用 `@task` 作为调用框架的工作单元。使用 `entrypoint.final()` 返回响应，并在同一线程上的多次交互中保持对话历史。

<Tabs>
<Tab title="Claude Agent SDK">

</fragment>

```python
import operator

from claude_agent_sdk import ClaudeSDKClient
from langgraph.func import entrypoint, task

@task
async def invoke_claude(prompt: str) -> str:
    async with ClaudeSDKClient(options=options) as client:
        await client.query(prompt)
        chunks: list[str] = []
        async for message in client.receive_response():
            chunks.append(str(message))
        return "\n".join(chunks)

@entrypoint()
async def agent(messages: list[dict], previous: list[dict] | None = None):
    history = operator.add(previous or [], messages)
    prompt = history[-1]["content"]
    response = await invoke_claude(prompt)
    new_message = {"role": "assistant", "content": response}
    return entrypoint.final(
        value=[new_message],
        save=operator.add(history, [new_message]),
    )
```

</Tab>
<Tab title="Strands Agents">

以下示例将智能体定义、函数式 API 包装、追踪设置以及 `agent` 符号的导出组合在单个 `agent.py` 文件中。选择与您的框架对应的标签页。

每个示例都有两个要点：

1. 在模块作用域中，将带有 `@entrypoint` 装饰器的函数导出为 `agent`。Agent Server 在服务图时导入此符号。
2. 返回 `entrypoint.final()` 并使用 `save` 参数，以便对话状态在同一线程的多次交互中保持一致。

## 启用追踪

使用 [项目布局](#project-layout) 中特定于框架的 `.env` 模板。当 `langgraph.json` 中设置 `"env": ".env"` 时，Agent Server 会加载此文件。

在该文件中设置 `LANGSMITH_PROJECT` 和您的框架提供商凭据。对于 Claude Agent SDK 和 Strands Agents，还要设置 `LANGSMITH_TRACING=true`。对于 CrewAI 和 AutoGen，追踪通过 `agent.py` 中的 `OtelSpanProcessor()` 和框架检测工具启用，因此只需设置 `LANGSMITH_API_KEY` 和 `LANGSMITH_PROJECT`。

[追踪](/langsmith/observability) 会在 [LangSmith UI](https://smith.langchain.com) 中显示 agent 调用、工具调用和 LLM 交互。了解框架特定的追踪选项，请参阅 [配置追踪](#configure-tracing) 中的链接。

## 本地运行

使用 [LangGraph CLI](/langsmith/cli) 启动本地 Agent Server：

```bash
langgraph dev
```

<Note>
如果 `langgraph dev` 报告 `langgraph-api` 缺失，请在相同环境中安装 `langgraph-cli[inmem]`。
</Note>

此命令在 `http://127.0.0.1:2024` 启动代理服务并打开 [LangSmith Studio](/langsmith/studio)。使用 `curl` 发送请求：

<Note>
`langgraph dev` 可能会在不同的端口上运行。请检查终端输出中的 URL，并根据需要更新下面的 `curl` 命令。
</Note>

<Tabs>
<Tab title="Claude Agent SDK, CrewAI, AutoGen">


</fragment>

```bash
# Create a thread
THREAD=$(curl -s -X POST http://127.0.0.1:2024/threads \
  -H "Content-Type: application/json" -d '{}' | python -c "import sys, json; print(json.load(sys.stdin)['thread_id'])")

# Run the agent and wait for the final response
curl -s -X POST "http://127.0.0.1:2024/threads/$THREAD/runs/wait" \
  -H "Content-Type: application/json" \
  -d '{
    "assistant_id": "ASSISTANT_ID",
    "input": [{"role": "user", "content": "Hello"}]
  }'
```

</Tab>

```bash
# Create a thread
THREAD=$(curl -s -X POST http://127.0.0.1:2024/threads \
  -H "Content-Type: application/json" -d '{}' | python -c "import sys, json; print(json.load(sys.stdin)['thread_id'])")

# Run the agent and wait for the final response
curl -s -X POST "http://127.0.0.1:2024/threads/$THREAD/runs/wait" \
  -H "Content-Type: application/json" \
  -d '{
    "assistant_id": "ASSISTANT_ID",
    "input": [
      {
        "role": "user",
        "content": [
          {"type": "text", "text": "Hello"}
        ]
      }
    ]
  }'
```

<Note>
如果此请求失败并出现 `NoCredentialsError`，请为您的模型提供商配置 AWS 凭证（例如 `AWS_PROFILE` 或 `AWS_ACCESS_KEY_ID` 和 `AWS_SECRET_ACCESS_KEY`），然后重启 `langgraph dev`。
</Note>

</Tab>
</Tabs>

将 `ASSISTANT_ID` 替换为 `langgraph.json` 中 `graphs` 对象内的图键。例如，如果您的配置是 `"graphs": {"claude_agent": "./agent.py:agent"}`，则使用 `claude_agent`；如果您的配置是 `"graphs": {"strands_agent": "./agent.py:agent"}`，则使用 `strands_agent`。

<Note>
在部署之前，[验证 LangGraph API 是否在本地运行](/langsmith/local-dev-testing)。如果 `langgraph dev` 失败，则部署到 LangSmith 也会失败。
</Note>

## 部署到 LangSmith

在代理本地运行后，使用 `langgraph deploy` 将其部署：

```bash
langgraph deploy --name my-agent
```

有关环境配置、部署类型和修订管理，请参阅[部署到云端](/langsmith/deploy-to-cloud)。有关自托管设置，请参阅[自托管部署](/langsmith/self-hosted)。有关不使用控制平面的纯 Docker 托管，请参阅[部署独立服务器](/langsmith/deploy-standalone-server)。
