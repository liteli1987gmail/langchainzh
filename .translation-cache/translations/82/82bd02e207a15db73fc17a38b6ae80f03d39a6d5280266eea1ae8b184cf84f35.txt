## LLM

### Azure OpenAI

> [Microsoft Azure](https://en.wikipedia.org/wiki/Microsoft_Azure)，通常称为 `Azure` 是由运营的云计算平台 `Microsoft`，提供全球数据中心访问、管理和开发应用程序及服务。它提供一系列功能，包括软件即服务（SaaS）、平台即服务（PaaS）和基础设施即服务（IaaS）。 `Microsoft Azure` 支持多种编程语言、工具和框架，包括 Microsoft 特定以及第三方软件和系统。

> [Azure OpenAI](https://azure.microsoft.com/products/ai-services/openai-service/) 是一项云服务，帮助您使用来自 OpenAI、Meta 等公司的多样化预构建和精选模型快速开发生成式 AI 体验。

LangChain.js 支持与 [Azure OpenAI](https://azure.microsoft.com/products/ai-services/openai-service/) 集成，使用 OpenAI SDK 中的新 Azure 集成功能 [OpenAI SDK](https://github.com/openai/openai-node).

您可以在 [此页面](https://learn.microsoft.com/azure/ai-services/openai/overview)上了解更多关于 Azure OpenAI 及其与 OpenAI API 的区别。如果您没有 Azure 账户，可以 [创建一个免费账户](https://azure.microsoft.com/free/) 开始使用。

您需要部署一个 Azure OpenAI 实例。您可以按照 [本指南](https://learn.microsoft.com/azure/ai-services/openai/how-to/create-resource?pivots=web-portal).

在 Azure 门户上部署一个版本。一旦您的实例运行，请确保您拥有实例名称和密钥。您可以在 Azure 门户中实例的“密钥和终结点”部分找到密钥。

如果您使用的是 Node.js，可以定义以下环境变量来使用该服务：

```bash
AZURE_OPENAI_API_INSTANCE_NAME=<YOUR_INSTANCE_NAME>
AZURE_OPENAI_API_DEPLOYMENT_NAME=<YOUR_DEPLOYMENT_NAME>
AZURE_OPENAI_API_EMBEDDINGS_DEPLOYMENT_NAME=<YOUR_EMBEDDINGS_DEPLOYMENT_NAME>
AZURE_OPENAI_API_KEY=<YOUR_KEY>
AZURE_OPENAI_API_VERSION="2024-02-01"
```

<Info>
**您可以在 [Azure OpenAI 文档](https://learn.microsoft.com/azure/ai-services/openai/reference).**


</Info>

<Tip>
中找到支持的 API 版本列表。另请参阅 [部分了解安装 LangChain 包的一般说明](/oss/langchain/install).
</Tip>


```typescript
  azureOpenAIApiDeploymentName: "<your_deployment_name>", // In Node.js defaults to process.env.AZURE_OPENAI_API_DEPLOYMENT_NAME
  azureOpenAIApiVersion: "<api_version>", // In Node.js defaults to process.env.AZURE_OPENAI_API_VERSION
```

## 向量存储

### Azure Cosmos DB for NoSQL

> [Azure Cosmos DB for NoSQL](https://learn.microsoft.com/azure/cosmos-db/nosql/) 提供对具有灵活架构的项目的查询支持以及对 JSON 的原生支持。它现在提供向量索引和搜索功能。此功能旨在处理高维向量，在任何规模下实现高效准确的向量搜索。您现在可以直接在文档中随数据一起存储向量。数据库中的每个文档不仅可包含传统的无架构数据，还可包含作为文档其他属性的高维向量。

```bash npm
npm install @langchain/azure-cosmosdb @langchain/core
```
查看 [使用示例](/oss/integrations/vectorstores/azure_cosmosdb_nosql).

```typescript
import { AzureCosmosDBNoSQLVectorStore } from "@langchain/azure-cosmosdb";
```
### Azure DocumentDB

> [Azure DocumentDB](https://learn.microsoft.com/azure/documentdb/) （原 Azure Cosmos DB for MongoDB vCore）可轻松创建具有完整原生 MongoDB 支持的数据库。您可以运用您的 MongoDB 经验，通过将应用程序指向连接字符串来继续使用您喜爱的 MongoDB 驱动程序、SDK 和工具。使用 Azure DocumentDB 中的向量搜索可将基于 AI 的应用程序与存储在 Azure DocumentDB 中的数据无缝集成。

```bash npm
npm install @langchain/azure-cosmosdb @langchain/core
```
查看 [使用示例](/oss/integrations/vectorstores/azure_documentdb).

```typescript
import { AzureDocumentDBVectorStore } from "@langchain/azure-cosmosdb";
```
## 语义缓存

### Azure Cosmos DB NoSQL 语义缓存

> 语义缓存功能支持 Azure Cosmos DB for NoSQL 集成，使用户能够根据用户输入与之前缓存结果之间的语义相似性检索缓存响应。它利用 [AzureCosmosDBNoSQLVectorStore](/oss/integrations/vectorstores/azure_cosmosdb_nosql)来存储缓存提示的向量嵌入。这些嵌入支持基于相似性的搜索，允许系统检索相关的缓存结果。

```bash npm
npm install @langchain/azure-cosmosdb @langchain/core
```
查看 [使用示例](/oss/integrations/llm_caching/azure_cosmosdb_nosql).

```typescript
import { AzureCosmosDBNoSQLSemanticCache } from "@langchain/azure-cosmosdb";
```
