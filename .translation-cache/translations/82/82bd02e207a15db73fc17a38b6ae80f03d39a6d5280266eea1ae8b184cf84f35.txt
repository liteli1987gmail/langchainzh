
## 大型语言模型

### Azure OpenAI

> [Microsoft Azure](https://en.wikipedia.org/wiki/Microsoft_Azure)，通常简称为 `Azure`，是由 `Microsoft` 运营的云计算平台，通过全球数据中心提供访问、管理和开发应用程序及服务。它提供一系列功能，包括软件即服务（SaaS）、平台即服务（PaaS）和基础设施即服务（IaaS）。`Microsoft Azure` 支持多种编程语言、工具和框架，包括 Microsoft 特定的以及第三方软件和系统。

> [Azure OpenAI](https://azure.microsoft.com/products/ai-services/openai-service/) 是一项云服务，帮助您通过 OpenAI、Meta 等多种预构建和精选模型快速开发生成式 AI 应用。

LangChain.js 支持使用 [OpenAI SDK](https://github.com/openai/openai-node) 中的新 Azure 集成与 [Azure OpenAI](https://azure.microsoft.com/products/ai-services/openai-service/) 进行集成。

您可以在[此页面](https://learn.microsoft.com/azure/ai-services/openai/overview)了解更多关于 Azure OpenAI 及其与 OpenAI API 的区别。如果您没有 Azure 账户，可以[创建免费账户](https://azure.microsoft.com/free/)开始使用。

您需要部署一个 Azure OpenAI 实例。按照[本指南](https://learn.microsoft.com/azure/ai-services/openai/how-to/create-resource?pivots=web-portal)可在 Azure Portal 上部署一个版本。

实例运行后，请确保您拥有实例名称和密钥。您可以在 Azure Portal 中实例的"密钥和终结点"部分找到密钥。

如果您使用的是 Node.js，可以定义以下环境变量来使用该服务：

```bash
AZURE_OPENAI_API_INSTANCE_NAME=<YOUR_INSTANCE_NAME>
AZURE_OPENAI_API_DEPLOYMENT_NAME=<YOUR_DEPLOYMENT_NAME>
AZURE_OPENAI_API_EMBEDDINGS_DEPLOYMENT_NAME=<YOUR_EMBEDDINGS_DEPLOYMENT_NAME>
AZURE_OPENAI_API_KEY=<YOUR_KEY>
AZURE_OPENAI_API_VERSION="2024-02-01"
```

<Info>
**您可以在 [Azure OpenAI 文档](https://learn.microsoft.com/azure/ai-services/openai/reference) 中找到支持的 API 版本列表。**


</Info>

<Tip>
请参阅[本节了解安装 LangChain 包的一般说明](/oss/langchain/install)。
</Tip>

```bash npm
npm install @langchain/openai @langchain/core
```
查看[使用示例](/oss/integrations/llms/azure)。

```typescript
import { AzureOpenAI } from "@langchain/openai";

const model = new AzureOpenAI({
  azureOpenAIApiKey: "<your_key>", // In Node.js defaults to process.env.AZURE_OPENAI_API_KEY
  azureOpenAIApiInstanceName: "<your_instance_name>", // In Node.js defaults to process.env.AZURE_OPENAI_API_INSTANCE_NAME
  azureOpenAIApiDeploymentName: "<your_deployment_name>", // In Node.js defaults to process.env.AZURE_OPENAI_API_DEPLOYMENT_NAME
  azureOpenAIApiVersion: "<api_version>", // In Node.js defaults to process.env.AZURE_OPENAI_API_VERSION
});
```

## 向量存储

### Azure Cosmos DB for NoSQL

> [Azure Cosmos DB for NoSQL](https://learn.microsoft.com/azure/cosmos-db/nosql/) 支持使用灵活架构查询数据，并原生支持 JSON。它现在提供向量索引和搜索功能。此功能专为处理高维向量而设计，能够在任何规模下实现高效、准确的向量搜索。您现在可以将向量直接存储在文档中，与您的数据并存。数据库中的每个文档不仅包含传统的无架构数据，还可以将高维向量作为文档的其他属性。

```bash npm
npm install @langchain/azure-cosmosdb @langchain/core
```
查看[使用示例](/oss/integrations/vectorstores/azure_cosmosdb_nosql)。

```typescript
import { AzureCosmosDBNoSQLVectorStore } from "@langchain/azure-cosmosdb";
```
### Azure DocumentDB

> [Azure DocumentDB](https://learn.microsoft.com/azure/documentdb/)（原 Azure Cosmos DB for MongoDB vCore）让您可以轻松创建具有完整原生 MongoDB 支持的数据库。您可以利用自己的 MongoDB 经验，通过将应用程序指向连接字符串来继续使用您喜欢的 MongoDB 驱动程序、SDK 和工具。使用 Azure DocumentDB 中的矢量搜索，将基于 AI 的应用程序与存储在 Azure DocumentDB 中的数据无缝集成。

```bash npm
npm install @langchain/azure-cosmosdb @langchain/core
```
请参阅[用法示例](/oss/integrations/vectorstores/azure_documentdb)。

```typescript
import { AzureDocumentDBVectorStore } from "@langchain/azure-cosmosdb";
```
## 语义缓存

### Azure Cosmos DB NoSQL 语义缓存

> 语义缓存功能支持 Azure Cosmos DB for NoSQL 集成，使用户能够根据用户输入与之前缓存的结果之间的语义相似性来检索缓存响应。它利用 [AzureCosmosDBNoSQLVectorStore](/oss/integrations/vectorstores/azure_cosmosdb_nosql)，该功能存储缓存提示词的向量嵌入。这些嵌入支持基于相似性的搜索，使系统能够检索相关的缓存结果。

```bash npm
npm install @langchain/azure-cosmosdb @langchain/core
```
查看[使用示例](/oss/integrations/llm_caching/azure_cosmosdb_nosql)。

```typescript
import { AzureCosmosDBNoSQLSemanticCache } from "@langchain/azure-cosmosdb";
```
