## `LANGSMITH_API_KEY`

要将追踪发送至自托管的 LangSmith 实例，请设置 `LANGSMITH_API_KEY` 为在自托管实例中创建的 API 密钥。

## `LANGSMITH_ENDPOINT`

要将追踪发送至自托管的 LangSmith 实例，请设置 `LANGSMITH_ENDPOINT` 为自托管实例的主机名。

## `MOUNT_PREFIX`

设置 `MOUNT_PREFIX` 以在特定路径前缀下提供 Agent Server。当服务器位于需要特定路径前缀的反向代理或负载均衡器后面时，这很有用。

例如，如果服务器要在 `https://example.com/langgraph`下提供，请设置 `MOUNT_PREFIX` to `/langgraph`.

## `POSTGRES_URI_CUSTOM`

指定 `POSTGRES_URI_CUSTOM` 以使用自定义 Postgres 实例。该值 `POSTGRES_URI_CUSTOM` 必须是有效的 [Postgres 连接 URI](https://www.postgresql.org/docs/current/libpq-connect.html#LIBPQ-CONNSTRING-URIS).

Postgres:

* 版本 15.8 或更高。
* 必须存在初始数据库，且连接 URI 必须引用该数据库。

控制平面功能：

* If `POSTGRES_URI_CUSTOM` 时，控制平面将不会为服务器配置数据库。
* If `POSTGRES_URI_CUSTOM` 时，控制平面将不会为服务器配置数据库，也不会删除外部管理的 Postgres 实例。
* If `POSTGRES_URI_CUSTOM` 时，修订版本的部署将不会成功。一旦 `POSTGRES_URI_CUSTOM` ，则必须在部署的整个生命周期内始终设置该值。
* 如果部署被删除，控制平面不会删除外部管理的 Postgres 实例。
* 的值 `POSTGRES_URI_CUSTOM` 可以更新。例如，URI 中的密码可以更新。

数据库连接：

* 自定义 Postgres 实例必须可由 Agent Server 访问。用户负责确保连接性。

## `REDIS_CLUSTER`

<Warning>
此功能处于 Alpha 阶段。
</Warning>

设置 `REDIS_CLUSTER` to `True` 以启用 Redis 集群模式。启用后，系统将使用集群模式连接到 Redis。连接到 Redis 集群部署时这很有用。

默认为 `False`.

## `REDIS_URI_CUSTOM`

指定 `REDIS_URI_CUSTOM` 以使用自定义 Redis 实例。该值 `REDIS_URI_CUSTOM` 必须是有效的 [Redis 连接 URI](https://redis-py.readthedocs.io/en/stable/connections.html#redis.Redis.from_url).
