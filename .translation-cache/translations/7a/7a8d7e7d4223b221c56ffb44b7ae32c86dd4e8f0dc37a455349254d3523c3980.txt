本页面说明 [LangChain Deep Agents](/oss/deepagents/overview) 如何与 [Claude Agent SDK](https://platform.anthropic.com/docs/en/agent-sdk/overview)进行对比。两者都是用于构建自定义代理的框架，但在执行环境、部署和供应商耦合方面做出了不同的权衡。

<Info>
    Deep Agents 在生产环境中被 [OpenSWE](https://github.com/langchain-ai/open-swe) 和 [LangSmith Fleet](/langsmith/fleet/index).
</Info>

## 概览

| | **Deep Agents** | **Claude Agent SDK** |
| --- | --- | --- |
| **代理运行位置** | 在沙箱内，或在沙箱外远程执行命令 | 在沙箱内 |
| **执行后端** | 可插拔： [本地、虚拟文件系统、远程沙箱或自定义](/oss/deepagents/backends) | 运行沙箱的本地文件系统 |
| **模型提供商** | 任意（Anthropic、OpenAI、Google、100+ 其他）| Claude（Anthropic、Bedrock、Vertex、Azure）|
| **Per-provider/model tuning** | [框架配置](/oss/deepagents/profiles) （测试版）：系统提示、工具、中间件和子代理调整的声明性捆绑包，按提供商或特定模型注册 | 在每个模型调用站点通过代码配置 |
| **部署** | [托管 Deep Agents](/langsmith/managed-deep-agents-overview) 在 LangSmith 中，或自行托管 [独立镜像](/langsmith/deploy-standalone-server) 通过 [`langgraph build`](/langsmith/cli#build) | [Self-host](https://code.claude.com/docs/en/agent-sdk/hosting)。您需要构建服务器、身份验证和流式传输层。 [Claude 托管代理](https://platform.claude.com/docs/en/managed-agents/overview) 是独立产品 |
| **Multi-tenancy** | [Built-in](/oss/deepagents/going-to-production#multi-tenancy)：作用域线程、每用户沙箱、RBAC | 自行构建 |
| **许可证** | MIT | MIT（Claude Code 本身是专有的）|

## 主要差异

### 代理和执行环境

有两种 [将代理连接到沙箱的模式](https://www.langchain.com/blog/the-two-patterns-by-which-agents-connect-sandboxes)：在 *内部* 沙箱内运行代理，或在沙箱外运行代理并 **将沙箱作为工具使用**.

Claude Agent SDK 仅支持第一种模式。您的代理在沙箱内运行，并针对沙箱的本地文件系统执行工具。Anthropic 的托管模型 [Claude 托管代理](https://platform.claude.com/docs/en/managed-agents/overview) 使用解耦模型，这反映了生产代理架构的发展方向。

Deep Agents 同时支持两种模式，并允许您选择 [后端](/oss/deepagents/backends#quickstart) 来将它们连接在一起。在实践中，这意味着您可以：

- 在沙箱内运行代理（与 Claude Agent SDK 相同的模型）。
- 在长期运行的容器中运行代理，并 [使用远程沙箱作为工具](https://www.langchain.com/blog/the-two-patterns-by-which-agents-connect-sandboxes)，通过网络安全执行命令。
- 为测试换入虚拟文件系统，或为您的自有基础设施换入自定义后端。

### Multi-tenancy

当您将应用程序投入生产时，您通常会将其暴露给众多最终用户，并且必须为每个用户隔离环境。

在 Claude Agent SDK 中，SDK 将代理绑定到其沙箱。要为每个用户提供独立的执行环境，您必须构建一个 API 包装器，为每个用户启动一个沙箱，跟踪哪个沙箱属于谁，然后在之后将其拆除。

Deep Agents 直接处理这个问题：在工具中配置沙箱 [每个用户或每个助手](/oss/deepagents/going-to-production#lifecycle) ，通过作用域线程、运行历史记录和 [RBAC](/oss/deepagents/going-to-production#team-access-control-rbac) 包括在内。如果您使用 [LangSmith Sandbox](/langsmith/sandbox-auth-proxy)，您还可以开箱即用地获得身份验证代理，以便最终用户可以从沙箱调用第三方 API，而无需您为每个用户配置凭据。

### 生产级代理服务器

要公开一个 [自托管 Claude Agent SDK](https://code.claude.com/docs/en/agent-sdk/hosting) app to end users, you write your own HTTP/WebSocket or SSE server that invokes the agent, streams tokens back, and manages conversation threads. That server is yours to build, operate, and secure.

Deep Agents 部署包括一个 [开箱即用：流式端点、线程管理、运行历史记录、Webhook 和](/langsmith/agent-server) 身份验证 [托管云或自托管](/langsmith/auth).

### Claude Agent SDK 部署是

managed cloud or self-hosted [self-hosted](https://code.claude.com/docs/en/agent-sdk/hosting)。SDK 和 [Claude 托管代理](https://platform.claude.com/docs/en/managed-agents/overview) 是独立的产品。使用 SDK 编写的代码不能直接部署到托管服务。

Deep 代理以两种模式运行，无需修改代码：

- **Managed:** 通过 [LangSmith 托管 Deep 代理](/langsmith/managed-deep-agents-overview) 创建、运行和操作 deep 代理。
- **Self-hosted:** 运行 [`langgraph build`](/langsmith/cli#build) 可生成 [独立 Docker 镜像](/langsmith/deploy-standalone-server) ，可部署到任何环境。

<Tip>
    如需跨任何模型提供商的托管代理平台，请使用 [LangSmith Fleet](/langsmith/fleet/index). [Claude 托管代理](https://platform.claude.com/docs/en/managed-agents/overview) 仅限 Anthropic 生态系统。
</Tip>

### LLM

Claude Agent SDK 执行包将模型、后端和部署打包在一起，并优化三者之间的支持。

使用 Deep 代理，您可以独立选择模型提供商、执行后端和部署目标。通过选择这种架构，您可以在模型和基础设施方面保持最大的灵活性。

### 生态系统

Claude Agent SDK 专为 Claude 和 Anthropic 产品生态设计。Deep 代理与更广泛的 LangChain 生态系统集成，包括用于可观测性、评估和部署的 LangSmith，并可跨任何模型提供商运行。

## 总结

- **选择 Deep 代理** 如果您需要模型和基础设施的灵活性、内置多租户部署，以及无需代码更改即可运行托管或自托管的选项。
- **选择 Claude Agent SDK** 如果您已经在 Anthropic 生态系统中投入了大量资源，并希望自行托管以及构建 API、身份验证和多租户层。
