在本地使用 LangChain 构建代理时,可视化地查看代理内部发生的情况、实时与其交互以及调试出现的问题是很有帮助的。 **LangSmith Studio** 是一个免费的视觉界面,用于从本地机器开发和测试您的 LangChain 代理。

Studio 连接到您在本地运行的代理,向您展示代理执行的每个步骤:发送给模型的提示词、工具调用及其结果,以及最终输出。您可以测试不同的输入、检查中间状态,并在无需额外代码或部署的情况下迭代改进代理的行为。

本页介绍如何设置 Studio 与您的本地 LangChain 代理。

## 前提条件

在开始之前,请确保您具备以下条件:

- **LangSmith 账户**:(免费)注册或登录于 [smith.langchain.com](https://smith.langchain.com).
- **LangSmith API 密钥**:请按照 [创建 API 密钥](/langsmith/create-account-api-key) guide.
- 如果您不想将数据 [追踪](/langsmith/observability-concepts#traces) 到 LangSmith,请在应用程序的 `LANGSMITH_TRACING=false` 文件中设置 `.env` 。禁用追踪后,不会有数据离开您的本地服务器。

## 设置本地代理服务器

### 1. 安装 LangGraph CLI

该 [LangGraph CLI](/langsmith/cli) 提供本地开发服务器(也称为 [代理服务器](/langsmith/agent-server)将您的代理连接到 Studio) 

```shell
# Python >= 3.11 is required.
pip install --upgrade "langgraph-cli[inmem]"
```

### 2. 准备您的代理

如果您已有 LangChain 代理，可以直接使用。此示例使用一个简单的邮件代理：

```python title="agent.py"
from langchain.agents import create_agent

def send_email(to: str, subject: str, body: str):
    """Send an email"""
    email = {
        "to": to,
        "subject": subject,
        "body": body
    }
    # ... email sending logic

    return f"Email sent to {to}"

agent = create_agent(
    "gpt-5.5",
    tools=[send_email],
    system_prompt="You are an email assistant. Always use the send_email tool.",
)
```

### 3. 环境变量

Studio 需要 LangSmith API 密钥来连接您的本地代理。创建一个 `.env` 文件在项目根目录，并从以下位置添加您的 API 密钥 [LangSmith](https://smith.langchain.com/settings).

<Warning>
    确保您的 `.env` 文件不会被提交到版本控制系统（如 Git）中。
</Warning>

```bash .env
LANGSMITH_API_KEY=lsv2...
```

### 4. 创建 LangGraph 配置文件

LangGraph CLI 使用配置文件来定位您的代理并管理依赖项。创建一个 `langgraph.json` 文件在您的应用程序目录中：

```json title="langgraph.json"
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent.py:agent"
  },
  "env": ".env"
}
```

@[`create_agent`] 函数自动返回编译后的 LangGraph 图，这正是 `graphs` 键在配置文件中预期的值。

<Info>
有关配置文件 JSON 对象中每个键的详细说明，请参阅 [LangGraph 配置文件参考](/langsmith/cli#configuration-file).
</Info>

此时，项目结构将如下所示：

```bash
my-app/
├── src
│   └── agent.py
├── .env
└── langgraph.json
```

### 5. 安装依赖项

从根目录安装项目依赖项：

<CodeGroup>
```shell pip
pip install langchain langchain-openai
```
```shell uv
uv add langchain langchain-openai
```
</CodeGroup>

### 6. 在 Studio 中查看您的代理

启动开发服务器以将您的代理连接到 Studio：

```shell
langgraph dev
```

<Warning>
Safari 阻止 `localhost` 到 Studio 的连接。作为变通方法，请使用 `--tunnel` 通过安全隧道访问 Studio。
</Warning>

服务器运行后，您的代理既可以通过 API 访问 `http://127.0.0.1:2024` 也可以通过 Studio UI 访问，访问地址为 `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`:

<Frame>
![Studio UI 中的代理视图](/oss/images/studio_create-agent.png)
</Frame>

With Studio connected to your local agent, you can iterate quickly on your agent's behavior. Run a test input, inspect the full execution trace including prompts, tool arguments, return values, and token/latency metrics. When something goes wrong, Studio captures exceptions with the surrounding state to help you understand what happened.

开发服务器支持热重载——对代码中的提示词或工具签名进行更改后，Studio 会立即反映这些变化。从任何步骤重新运行对话线程来测试您的更改，无需从头开始。此工作流程可扩展至从简单的单工具代理到复杂的多节点图。
