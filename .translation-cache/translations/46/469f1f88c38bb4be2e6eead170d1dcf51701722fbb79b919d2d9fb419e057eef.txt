当您准备好将 LangChain 代理部署到生产环境时，请选择适合您技术栈的托管模式。 **[LangSmith Cloud](/langsmith/deploy-to-cloud)** 为有状态、长时运行的代理提供完全托管的基础设施，支持持久状态和后台执行。

:::js
您还可以部署在 **JavaScript 框架和平台** 上，例如 Next.js、SvelteKit、Nuxt、Cloudflare Workers 和 Deno Deploy，使用相同的 [代理流式传输协议](https://github.com/langchain-ai/agent-protocol/tree/main/streaming).

:::

<Tip>
LangSmith 提供多种除 Cloud 之外的部署选项，包括 [混合](/langsmith/hybrid), [独立服务器](/langsmith/deploy-standalone-server)和 [带控制平面的自托管](/langsmith/deploy-with-control-plane)。如需更多信息，请参阅 [LangSmith 部署概述](/langsmith/deployment).
</Tip>

## LangSmith Cloud

本节将引导您完成从 GitHub 仓库将代理部署到 LangSmith Cloud 的过程。LangSmith 处理基础设施、扩展和运维相关的事务。

### 前提条件

在开始之前，请确保您具备以下条件：

- A [GitHub 账户](https://github.com/)
- A [LangSmith 账户](https://smith.langchain.com) （免费注册）

### 部署您的代理

#### 1. 在 GitHub 上创建仓库

您的应用程序代码必须位于 GitHub 仓库中才能在 LangSmith 上部署。公共仓库和私有仓库均支持。对于此快速入门，请先按照 [本地服务器设置指南](/oss/langchain/studio)确保您的应用与 LangGraph 兼容。然后，将您的代码推送到仓库。
