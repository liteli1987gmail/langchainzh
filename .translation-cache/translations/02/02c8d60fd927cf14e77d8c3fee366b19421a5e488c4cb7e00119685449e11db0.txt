**集成是 LangChain 的核心组件。**

LangChain 为几个不同的组件（语言模型、向量存储等）提供标准接口，这些组件在构建 LLM 应用程序时至关重要。实现新的集成有助于扩展 LangChain 的生态系统，并使您的服务能够被数百万开发者发现。

<Warning>
    新的集成**不会作为 PR 接受**到任何 `langchain-ai` 仓库。所有新集成必须作为独立包发布到 PyPI（例如 `langchain-yourprovider`）。您应该向 `langchain-ai` 仓库提交的 PR 是为已发布的包添加文档。
</Warning>

## 为什么要实现 LangChain 集成？

<Card title="可发现性" icon="search">
    LangChain 是构建 LLM 应用程序使用最广泛的框架，每月下载量超过 2 亿次。
</Card>

<Card title="互操作性" icon="refresh">
    LangChain 组件公开标准接口，允许开发者轻松地在它们之间切换。如果您实现了 LangChain 集成，任何使用不同组件的开发者都能轻松地将其替换为您的。
</Card>

<Card title="最佳实践" icon="star">
    通过标准接口，LangChain 组件鼓励并促进最佳实践（流式传输、异步等），这些实践改善了开发者体验和应用程序性能。
</Card>

## 待集成的组件

虽然任何组件都可以集成到 LangChain，但我们特别鼓励以下特定类型的集成：

**集成这些 ✅**：

- [**聊天模型**](/oss/integrations/chat)：使用最活跃的组件类型
- [**工具/工具包**](/oss/integrations/tools)：启用代理能力
- [**检索器**](/oss/integrations/retrievers)：RAG 应用程序的核心
- [**嵌入模型**](/oss/integrations/embeddings)：向量操作的基础
- [**向量存储**](/oss/integrations/vectorstores)：语义搜索的必备组件
- [**中间件**](/oss/integrations/middleware)：使用钩子扩展代理行为
- [**沙箱**](/oss/deepagents/sandboxes)：使用深度代理安全运行代码

<Accordion title="额外的第三方沙箱集成标准">
请注意，我们仅在以下情况下推荐第三方沙箱集成：

- 该集成由提供沙箱的公司编写和维护。
- **或者**该集成被广泛使用，即该集成在 PyPI 或 npm 上的每日下载量必须至少达到 10,000 次才能被考虑推荐。
</Accordion>

**不要集成这些 ❌**：

- **LLM（文本补全模型）**：已被弃用，改用[聊天模型](/oss/integrations/chat)
- [**文档加载器**](/oss/integrations/document_loaders)：维护负担高
- [**键值存储**](/oss/integrations/stores)：使用量有限
- **文档转换器**：小众用例
- **模型缓存**：基础设施相关
- **图**：复杂的抽象
- **消息历史**：存储抽象
- **回调**：系统级组件
- **聊天加载器**：需求有限
- **适配器**：边缘用例工具

## 如何贡献一个集成

<Steps>
    <Step title="实现您的包">
        <Card title="如何实现 LangChain 集成" icon="link" href="/oss/contributing/implement-langchain" arrow />
    </Step>

    <Step title="通过标准测试">
        如果适用，为您的集成实现对 LangChain [标准测试](/oss/contributing/standard-tests-langchain) 套件的支持，并成功运行它们。
    </Step>

    <Step title="发布集成">
        <Card title="如何发布集成" icon="upload" href="/oss/contributing/publish-langchain" arrow />
    </Step>

    <Step title="添加文档">
        提交一个 PR，为官方 LangChain 文档添加您的集成文档。

        <Accordion title="集成文档指南" icon="book">
            一个集成的价值取决于其文档的质量。为确保用户体验的一致性，所有新集成都需要文档。我们为每种类型的集成提供了标准的起点模板，供您复制和修改。

            在 LangChain [docs 仓库](https://github.com/langchain-ai/docs)的新 PR 中，使用适当的模板文件在相关目录下创建新文件：`src/oss/python/integrations/<component_type>/integration_name.mdx`：

            - [聊天模型](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx)
            - [工具和工具包](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx)
            - [中间件](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/middleware/TEMPLATE.mdx)
            - 检索器 - 即将推出
            - 文本分割器 - 即将推出
            - 嵌入模型 - 即将推出
            - [向量存储](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx)
            - 文档加载器 - 即将推出
            - 键值存储 - 即将推出
        </Accordion>
    </Step>

    <Step title="联合营销" icon="speakerphone">
        （可选）与 LangChain 团队合作进行联合[联合营销](/oss/contributing/comarketing)。
    </Step>
</Steps>
