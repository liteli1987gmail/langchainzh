<Update label="June 15-19, 2026" rss={{ title: "2026-06-15 - Cloud update" }}>

## 可观测性与评估

### 自动化

- [自动化](/langsmith/rules) 现在允许您按操作控制跟踪保留，以便匹配的跟踪可以保持基础保留级别，而不会被升级。

### 引擎

- 该 [引擎](/langsmith/engine) 引擎问题看板现已在未连接GitHub时显示连接GitHub操作，以便您可以在不离开看板的情况下设置拉取请求创建。
- [引擎](/langsmith/engine) 现已提供统一的启用界面和访问请求功能，组织设置在一个位置整合了引擎使用情况和限制。
- 组织管理员现在会收到 [引擎](/langsmith/engine) 在消费超过每个配置的阈值时发送消费邮件，暂停或禁用引擎现需要确认。

### 数据集与实验

- [实验](/langsmith/analyze-an-experiment) 现已在标题栏和进度列中显示实时加载进度，以便您可以实时跟踪已完成和已评估的运行。
- [评估器](/langsmith/evaluators) 现已在高级选项中包含跟踪保留切换，以便评分跟踪可以在适合您工作流程时保持基础保留。
- [评估器](/langsmith/evaluators) 提示编辑现提供高级模式，可直接编辑分离变量映射的Mustache模板。
- 现在创建 [数据集](/langsmith/manage-datasets)时可以应用资源标签，包括从头创建、文件上传或克隆。
- 自动附加断言 [评估器](/langsmith/evaluators) 现从参考输出读取断言，以便实验分数反映实际通过和失败结果。

### 提示与实验场

- [OAuth客户端凭证](/langsmith/model-configurations#oauth-client-credentials) 现支持模型配置中每个工作区的设置，以便工作区管理员可以自助使用保存的提示和模型上的OAuth。
- 该 [Playground](/langsmith/playground-model-providers) 现在在 Responses API 上为 OpenAI 推理模型公开了推理摘要选项。
- 模型下拉菜单不再为 OpenAI 兼容端点推荐 OpenAI 模型，因此您可以输入自己的 [自定义模型名称](/langsmith/model-configurations).

### 追踪

- [追踪查询语法](/langsmith/trace-query-syntax) 现在包含完整的操作符参考、字段表和快速示例，使 API 过滤更易于发现。
- 该 [OpenTelemetry 指南](/langsmith/trace-with-opentelemetry) 现在说明了如何将跨度链接到现有 LangSmith SDK 追踪，以及当父跨度从未到达时会发生什么，从而使跨进程追踪更易于调试。

### 监控和告警

- [仪表板](/langsmith/dashboards) 现在包含图表生成器（带图表模板）、创建和编辑窗格，以及时间序列图表上的刷选和系列控制功能。
- 您现在可以发送 [告警](/langsmith/alerts) 到 Slack 作为原生通知目标，并从界面中连接或断开 Slack 应用。

## 部署

- 预览 [预览部署](/langsmith/deployment) 现在为预览提交构建镜像，而不是重复使用父部署的镜像。

## 沙盒

- [沙盒认证代理](/langsmith/sandbox-auth-proxy) 现在记录了 GCP 规则和服务账户处理，使通过代理访问 Google API 更加清晰。
- [沙盒](/langsmith/sandboxes) 现在将 AWS US SaaS 可用性标记为正式发布，使区域表反映了当前的推广进度。
- [沙盒](/langsmith/sandboxes) 现在支持 Git 挂载和 Google Cloud Storage 存储桶挂载。

## 管理及计费

### 管理

- [组织设置](/langsmith/administration-overview) now clarify that SSO/SCIM group names can omit spaces, so enterprise IdPs that disallow spaces still work cleanly.
- Vanta MCP 集成现已向所有工作区正式推出。
- 创建数据集、提示和项目时应用标签现在由专门的 [创建时标签权限](/langsmith/administration-overview).

### LLM 网关

- 该 [LLM 网关](/langsmith/llm-gateway) 现在支持 Vertex AI 的原生 Gemini 路由和 OpenAI 嵌入端点。
- [网关守护](/langsmith/llm-gateway) 策略现在接受细粒度的 PII 配置和可自定义的超时操作。

### 使用及计费

- [细粒度可计费使用](/langsmith/granular-usage) 现在阐明了组织范围划分，使您可以更准确地解读使用总量。

</Update>

## 可观测性和评估

### 引擎

- [引擎](/langsmith/engine) 现在仅在项目视图中显示项目级支出，因此组织级支出保留在组织设置界面中。
- [引擎](/langsmith/engine) 现在将 Slack 问题告警卡固定在滚动问题列表上方，因此在浏览时提醒始终可见。

实验表现在显示加载进度条，展示已完成和已评估的运行次数，早于此功能的实验显示占位符进度条。

- Categorical feedback now shows derived percentages in experiment tables, so pass/fail metrics are easier to scan.

### 提示词和 Playground

- [Playground](/langsmith/playground-model-providers) 现在从头到尾为启用 OAuth 的预设生成 OAuth 承载者令牌，因此长时间运行的批处理和流可以继续工作。

## 沙箱

- [沙箱认证代理](/langsmith/sandbox-auth-proxy) 现在支持 GCP 认证流程，因此沙箱工作负载可以通过代理访问 Google API。

## 修复

- Engine 试用模态框不再显示粗略计算的 LCU 标注，因此定价文案不再那么误导人。

</Update>

## 可观测性和评估

### 自动化

- [运行规则](/langsmith/rules) webhook 有效载荷现在为每次运行包含跟踪深度链接，因此下游系统可以直接跳回跟踪。

### Engine

- Per-workspace [Engine](/langsmith/engine) spend 功能现已普遍可用：您可以直接在 Engine 设置页面查看 LCU 和 USD 支出，包括会话级别的支出。
- Engine 设置页面现在在一处显示额外的 Engine 详细信息。
- 您可以轮换 [Engine 问题板 webhook](/langsmith/engine-webhooks) API 和 webhook 设置 UI 的签名密钥。
- 引擎问题列表添加了按追踪数量排序的选项。

### 数据集和实验

- 一个新的开箱即用的 [断言评估器](/langsmith/assertions) 根据参考输出中指定的明确标准列表对输出进行评分，当您向数据集添加断言式示例时，会自动附加断言规则。
- 实验详情中的评估器指标已改进， [对比](/langsmith/compare-experiment-results)，以及全局实验表中。

### 提示词和 playground

-  [Playground](/langsmith/playground-model-providers) 支持 Amazon Bedrock API 密钥身份验证，允许您使用 bearer token 而不是 AWS 凭证进行身份验证。

### 追踪

-  [追踪视图](/langsmith/view-traces) 现在在运行的操作菜单上显示未读指示符，当该运行有待查看的审查者备注时。
- 瀑布视图现在是全高且标题固定，这样您在滚动长追踪时能保持当前位置。
- 全局搜索现在包含上下文和沙箱

## 部署

- 您现在可以从 [Studio](/langsmith/studio) page.
- 触发 LangSmith 部署 [部署 Google Agent Development Kit (ADK) 代理](/langsmith/deploy-google-adk).

## 沙箱

- [沙盒代理规则](/langsmith/sandbox-auth-proxy) 现在支持配置 AWS 身份验证，因此沙盒可以通过带签名请求的代理访问 AWS 服务。
- 沙盒可以创建 [快照](/langsmith/sandbox-snapshots) 从 Dockerfile 构建源。

## 管理后台和计费

### 管理后台

- 组织管理员现在可以从 [组织设置](/langsmith/administration-overview) page.

### 使用量和计费

- [精细化计费使用量](/langsmith/granular-usage) 现在支持按保留层级进行过滤和分组，将长生命周期和短生命周期的追踪分开。
- 精细化计费使用量页面现在显示 LangSmith 部署使用量，包括执行的节点、代理运行次数和代理运行时间，以及追踪使用量。

## 修复

- 大型追踪加载性能改进。
- 现在重新打开筛选下拉菜单进行编辑时，元数据的筛选值会被保留。
- 数据集创建现在使用多选下拉菜单来选择 CSV 字段。

</Update>

## 可观测性和评估

### 洞察

-  [洞察代理](/langsmith/insights) 现在支持按每日、每周或自定义 cron 间隔生成计划报告，因此报告生成无需手动触发。时间范围动态计算，因此"最近 24 小时"报告在运行时总是反映最近的时间窗口，而不是配置时的时间窗口。

### 数据集和实验

- 您现在可以将任何实验固定为基准实验。固定的实验会保持在 [实验](/langsmith/compare-experiment-results) 视图的顶部，并作为后续运行的自动对比点，显示每个列的性能差异，使改进和回归立即清晰可见。

</Update>

<Update label="February 2-6, 2026" rss={{ title: "2026-02-02 - Cloud update" }}>

## 可观测性和评估

### 成本追踪

- [成本追踪](/langsmith/cost-tracking) 现在超越了 LLM 调用。为任何运行提交自定义成本元数据，例如昂贵的工具调用、第三方 API 或检索步骤，以便从单一仪表板监控、调试和优化整个代理堆栈的支出。

### 追踪

- 您现在可以 [配置追踪表中的哪些部分](/langsmith/configure-input-output-preview) 显示追踪的输入和输出，因此使用自定义追踪格式的团队可以显示最相关的字段，减少杂乱，并更快地识别需要仔细查看的追踪。

</Update>

## 可观测性和评估

### 标注和人工反馈

- 新的成对 [标注队列](/langsmith/annotation-queues) let reviewers compare two runs side by side and choose whether option A is better, option B is better, or the two are equal across rubric items. LangSmith automatically pairs runs between two experiments and manages queues, reviewer assignments, and trace access, so you can run A/B evaluations across agents, prompts, and models, including for subjective dimensions like tone, correctness, usefulness, or style.

</Update>

## 可观测性和评估

### 追踪

- LangSmith Fetch 是一个新的命令行工具，可以将 LangSmith 追踪直接带到您的终端、编码环境或 IDE。使用 `pip install langsmith-fetch`安装，然后使用 `--limit`, `--after`等过滤器检索追踪，以及 `--last-n-minutes`将追踪和线程批量导出到文件以进行分析、脚本编写或数据集创建。
