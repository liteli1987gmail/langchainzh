## 可观测性与评估

### 自动化

- [自动化规则](/langsmith/rules) 现在允许您按操作控制跟踪保留，因此被规则匹配的跟踪可以保持基础保留级别，而不会被升级。

### Engine

- 当 GitHub 未连接时，[Engine](/langsmith/engine) 问题板现在会显示"连接 GitHub"操作，这样您就可以在不离开问题板的情况下设置拉取请求创建。
- [Engine](/langsmith/engine) 现在有一个统一的启用屏幕，包含访问请求，且组织设置在一个位置整合了 Engine 使用情况和限制。
- 组织管理员现在会在支出超过每个配置的阈值时收到 [Engine](/langsmith/engine) 支出邮件，且暂停或禁用 Engine 现在需要确认。

### 数据集与实验

- [实验](/langsmith/analyze-an-experiment) 现在在标题和进度列中显示实时加载进度，因此您可以实时跟踪已完成的运行和已评估的运行。
- [评估器](/langsmith/evaluators) 现在在高级选项中包含跟踪保留切换，因此评分后的跟踪在适合您的工作流程时可以保持基础保留级别。
- [评估器](/langsmith/evaluators) 提示编辑现在提供高级模式，可直接编辑 Mustache 模板并使用独立的变量映射。
- 您现在可以在创建[数据集](/langsmith/manage-datasets)时应用资源标签，包括从头创建、通过文件上传或克隆。
- 自动附加的断言[评估器](/langsmith/evaluators) 现在从参考输出中读取断言，因此实验分数反映实际的通过和失败结果。

### 提示词与 Playground

- [OAuth 客户端凭证](/langsmith/model-configurations#oauth-client-credentials) 现在支持在模型配置上进行每个工作区独立设置，因此工作区管理员可以自助管理保存的提示词和模型的 OAuth。
- [Playground](/langsmith/playground-model-providers) 现在为 OpenAI 推理模型在 Responses API 上公开推理摘要选项。
- 模型下拉菜单不再为 OpenAI 兼容端点建议 OpenAI 模型，因此您可以输入自己的[自定义模型名称](/langsmith/model-configurations)。

### 跟踪

- [跟踪查询语法](/langsmith/trace-query-syntax) 现在有完整的操作符参考、字段表和快速示例，因此 API 过滤更容易发现。
- [OpenTelemetry 指南](/langsmith/trace-with-opentelemetry) 现在说明了如何将跨度链接到现有 LangSmith SDK 跟踪，以及当父跨度永远不到达时会发生什么，因此跨进程跟踪更容易调试。

### 监控与告警

- [仪表板](/langsmith/dashboards) 现在包含图表构建器，具有图表模板、创建和编辑窗格，以及时间序列图表上的刷选和系列控制。
- 您现在可以将[告警](/langsmith/alerts)作为原生通知目标发送到 Slack，并可从 UI 连接或断开 Slack 应用。

## 部署

- 预览[部署](/langsmith/deployment) 现在为预览提交构建镜像，而不是重用父部署的镜像。

## 沙箱

- [沙箱认证代理](/langsmith/sandbox-auth-proxy) 现在记录了 GCP 规则和服务账户处理，因此通过代理访问 Google API 更加清晰。
- [沙箱](/langsmith/sandboxes) 现在将 AWS US SaaS 可用性标记为正式发布，因此区域表反映了当前的推广进度。
- [沙箱](/langsmith/sandboxes) 现在支持 Git 挂载和 Google Cloud Storage 存储桶挂载。

## 管理员与计费

### 管理

- [组织设置](/langsmith/administration-overview) 现在明确了 SSO/SCIM 组名可以省略空格，因此不允许空格的企业 IdP 仍能正常工作。
- Vanta MCP 集成现在对所有工作区正式发布。
- 在创建数据集、提示词和项目时应用标签现在由专门的[创建时标签权限](/langsmith/administration-overview)控制。

### LLM Gateway

- [LLM gateway](/langsmith/llm-gateway) 现在支持 Vertex AI 的原生 Gemini 路由和 OpenAI embeddings 端点。
- [Gateway guard](/langsmith/llm-gateway) 策略现在接受粒度 PII 配置和可配置的的超时操作。

### 使用与计费

- [粒度计费使用量](/langsmith/granular-usage) 现在明确了组织范围，因此您可以更准确地解读使用量总计。

## 可观测性与评估

### Engine

- [Engine](/langsmith/engine) 现在只在项目视图中显示项目级支出，因此组织范围的支出保持在组织设置界面中。
- [Engine](/langsmith/engine) 现在将 Slack 问题告警卡片固定在滚动问题列表上方，因此当您浏览时提示始终可见。

### 数据集与实验

实验表现在显示加载进度条，显示已完成的运行和已评估的运行数量，在此功能之前的实验显示占位进度条。
- [仪表板](/langsmith/dashboards) 现在支持由 v2 图表 API 支持的时间序列柱状图和折线图，因此监控指标可以使用较新的图表类型。
- 分类反馈现在在实验表中显示派生的百分比，因此通过/失败指标更容易扫描。

### 提示词与 Playground

- [Playground](/langsmith/playground-model-providers) 现在为启用 OAuth 的预设端到端生成 OAuth bearer，因此长时间运行的批处理和流可以继续工作。

## 沙箱

- [沙箱认证代理](/langsmith/sandbox-auth-proxy) 现在支持 GCP 认证流程，因此沙箱工作负载可以通过代理访问 Google API。

## 修复

- Engine 试用模态框不再显示粗略计算 LCU 子弹项目，因此定价文案误导性更低。

</Update>

## 可观测性与评估

### 自动化

- [运行规则](/langsmith/rules) webhook 有效载荷现在为每个运行包含一个跟踪深度链接，因此下游系统可以直接跳转回跟踪。

### Engine

- 每个工作区 [Engine](/langsmith/engine) 支出现在正式发布：您可以直接在 Engine 设置页面查看 LCU 和 USD 支出，包括会话级支出。
- Engine 设置页面现在在一个位置显示额外的 Engine 详情。
- 您可以从 API 和 webhook 设置 UI 轮换 [Engine 问题板 webhook](/langsmith/engine-webhooks) 签名密钥。
- Engine 问题列表添加了按跟踪数量排序的选项。

### 数据集与实验

- 新的开箱即用[断言评估器](/langsmith/assertions) 根据参考输出中指定的标准列表对输出进行评分，当您向数据集添加断言式示例时会自动附加断言规则。
- 实验详情、[比较](/langsmith/compare-experiment-results)和全局实验表中的评估器指标已改进。

### 提示词与 Playground

- [Playground](/langsmith/playground-model-providers) 支持 Amazon Bedrock API 密钥认证，让您使用 bearer token 而不是 AWS 凭证进行认证。

### 跟踪

- [跟踪视图](/langsmith/view-traces) 现在在运行的菜单上显示未读指示器，当运行有您尚未看到的审核者备注时。
- 瀑布视图现在为全高度并带有粘性轮次标题，因此在滚动浏览长跟踪时您可以保持您的位置。
- 全局搜索现在包含上下文和沙箱

## 部署

- 您现在可以从 [Studio](/langsmith/studio) 页面触发 LangSmith 部署。
- LangSmith 部署现在支持[部署 Google Agent Development Kit (ADK) 代理](/langsmith/deploy-google-adk)。

## 沙箱

- [沙箱代理规则](/langsmith/sandbox-auth-proxy) 现在支持配置 AWS 认证，因此沙箱可以通过带签名请求的代理访问 AWS 服务。
- 沙箱可以从 Dockerfile 构建源创建[快照](/langsmith/sandbox-snapshots)。

## 管理员与计费

### 管理

- 组织管理员现在可以从[组织设置](/langsmith/administration-overview)页面禁用个人访问令牌创建。

### 使用与计费

- [粒度计费使用量](/langsmith/granular-usage) 现在支持按保留层级过滤和分组，区分长期和短期跟踪。
- 粒度计费使用量页面现在显示 LangSmith 部署使用量，包括执行的节点、代理运行和代理运行时间，以及跟踪使用量。

## 修复

- 大跟踪加载的性能改进。
- 重新打开过滤器下拉菜单进行编辑时，元数据的过滤器值现在会被保留。
- 数据集创建现在使用多选下拉菜单来选择 CSV 字段。

</Update>

## 可观测性与评估

### Insights

- [Insights Agent](/langsmith/insights) 现在支持按每日、每周或自定义 cron 间隔的计划报告，因此报告生成无需手动触发。时间范围动态计算，因此"最近 24 小时"报告在其运行时始终反映最新的时间窗口，而不是您配置它的时间。

### 数据集与实验

- 您现在可以将任何实验固定为基准。固定的实验保持在[实验](/langsmith/compare-experiment-results)视图顶部，并作为后续运行的自动比较点，实时显示每列的性能差异，使改进和回归一目了然。

</Update>

## 可观测性与评估

### 成本跟踪

- [成本跟踪](/langsmith/cost-tracking) 现在不仅限于 LLM 调用。为任何运行提交自定义成本元数据，例如昂贵的工具调用、第三方 API 或检索步骤，可从单个仪表板监控、调试和优化整个代理堆栈的支出。

### 跟踪

- 您现在可以[配置跟踪的输入和输出](/langsmith/configure-input-output-preview)的哪些部分显示在跟踪表中，因此使用自定义跟踪格式的团队可以显示最相关的字段，减少混乱，并更快地识别需要仔细查看的跟踪。

</Update>
