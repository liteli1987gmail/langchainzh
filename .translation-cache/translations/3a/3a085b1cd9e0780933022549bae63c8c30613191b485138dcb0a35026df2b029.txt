[OpenGradient](https://www.opengradient.ai/) 是一个去中心化 AI 计算网络，实现全球可访问、无需许可且可验证的 ML 模型推理。

OpenGradient langchain 包目前提供了一套工具包，允许开发者为其在 OpenGradient 网络上的模型构建自定义 ML 推理工具。这在以前是一个难题，因为大型模型参数会污染上下文窗口——想象一下不得不给您的代理一个 200x200 的浮点数据数组！

该工具包通过将所有数据处理逻辑封装在工具定义本身中来解决这个问题。这种方法使代理的上下文窗口保持干净，同时让开发者可以完全灵活地为他们的 ML 模型实现自定义数据处理和实时数据检索。

## 安装和设置

确保您拥有 OpenGradient API 密钥以访问 OpenGradient 网络。如果您已有 API 密钥，只需设置环境变量：

```python
!export OPENGRADIENT_PRIVATE_KEY="your-api-key"
```

如果您需要设置新的 API 密钥，请下载 opengradient SDK 并按照说明初始化新配置。

```python
!pip install opengradient
!opengradient config init
```

设置好 API 密钥后，安装 langchain-opengradient 包。

```python
pip install -U langchain-opengradient
```

## OpenGradient 工具包

OpenGradientToolkit 使开发者能够基于 [ML 模型](https://hub.opengradient.ai/models) 和 [工作流](https://docs.opengradient.ai/developers/sdk/ml_workflows.html) 部署在 OpenGradient 去中心化网络上创建专业工具。此集成使 LangChain 代理能够访问强大的 ML 功能，同时保持高效的上下文使用。

### 主要优势

* 🔄 实时数据集成 - 在工具内处理实时数据源

* 🎯 动态处理 - 可适应特定代理输入的自定义数据管道

* 🧠 上下文效率 - 处理复杂的 ML 操作而不淹没您的上下文窗口

* 🔌 无缝部署 - 轻松集成 OpenGradient 网络上已有的模型

* 🔧 完全可定制 - 通过 [OpenGradient SDK](https://docs.opengradient.ai/developers/sdk/model_management.html)创建并部署您自己的特定模型，然后从中构建自定义工具

* 🔐 可验证推理 - 所有推理都在去中心化 OpenGradient 网络上运行，允许用户选择各种安全方案，如 ZKML 和 TEE，实现无需信任、可验证的模型执行

有关详细示例和实现指南，请查看我们的 [综合教程](/oss/integrations/tools/opengradient_toolkit.ipynb).
