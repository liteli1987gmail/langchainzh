>[Parallel](https://platform.parallel.ai/) 是一个专为 LLM 和 AI 应用构建的实时网络搜索和内容提取平台。

`ChatParallel` 是 Parallel 模型的 OpenAI 兼容聊天接口。`speed` 模型是低延迟的对话模型，无引用；研究模型（`lite`、`base`、`core`）会浏览网页，并通过 JSON schema 返回每个字段的引用和结构化输出。

<Note>
    `ChatParallel` 是规范类名。早期的 `ChatParallelWeb` 继续作为同一类的别名使用。
</Note>

### 选择模型

| 模型 | 延迟 | 网页浏览 | 引用 | 结构化输出 | 使用场景 |
| :--- | :--- | :---: | :---: | :---: | :--- |
| `speed` | 低 | ❌ | ❌ | ❌ | 来自模型参数知识的对话回答。 |
| `lite` | 中等 | ✅ | ✅ | ✅ | 带引用的查事实。 |
| `base` | 中高 | ✅ | ✅ | ✅ | 带引用的中深度研究。 |
| `core` | 较高 | ✅ | ✅ | ✅ | 带引用的多源研究。 |

`speed` 不支持 `response_format`，因此在该模型上调用 `with_structured_output()` 会抛出明确的错误。当需要解析的 pydantic 输出或每个字段的引用时，请使用研究模型。

## 设置

要访问 Parallel 模型，请安装 `langchain-parallel` 集成包并获取 [Parallel](https://platform.parallel.ai) API 密钥。

### 安装

<CodeGroup>
    ```bash pip
    pip install -U langchain-parallel
    ```
    ```bash uv
    uv add langchain-parallel
    ```
</CodeGroup>

### 凭证

访问 [Parallel](https://platform.parallel.ai) 注册并生成 API 密钥。在您的环境中设置 `PARALLEL_API_KEY`：

```python
import getpass
import os

if not os.environ.get("PARALLEL_API_KEY"):
    os.environ["PARALLEL_API_KEY"] = getpass.getpass("Parallel API key:\n")
```

## 实例化

```python
from langchain_parallel import ChatParallel

llm = ChatParallel(
    model="speed",
    # timeout=None,
    # max_retries=2,
    # api_key="...",  # optional if PARALLEL_API_KEY is set
    # base_url="https://api.parallel.ai",  # default
)
```

请参阅 @[`ChatParallel`] API 参考文档以获取完整的可用参数列表。

## 调用

```python
messages = [
    ("system", "You are a helpful assistant with access to real-time web information."),
    ("human", "What are the latest developments in AI?"),
]
ai_msg = llm.invoke(messages)
print(ai_msg.content)
```

## 链式调用

使用提示词模板链接模型：

```python
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate(
    [
        (
            "system",
            "You are a research assistant with access to real-time web information. "
            "Answer questions about {topic} using current sources.",
        ),
        ("human", "{question}"),
    ]
)

chain = prompt | llm
chain.invoke(
    {
        "topic": "artificial intelligence",
        "question": "What are the most significant AI breakthroughs in 2026?",
    }
)
```

## 结构化输出

在研究模型（`lite`、`base`、`core`）上，`ChatParallel.with_structured_output(...)` 绑定一个 JSON-schema `response_format` 并返回一个解析后的 pydantic 对象（或字典）。在 `speed` 上调用会引发 `ValueError`，因为 `speed` 会静默忽略 `response_format`。

```python
from pydantic import BaseModel, Field

class Founder(BaseModel):
    name: str = Field(description="Full name of the founder")
    company: str = Field(description="Company they founded")

structured = ChatParallel(model="lite").with_structured_output(Founder)
parsed = structured.invoke([("human", "Who founded SpaceX?")])
print(parsed)
```

```text
name='Elon Musk' company='SpaceX'
```

`method="json_schema"`（默认）、`method="json_mode"` 和 `method="function_calling"` 均受支持。传入 `include_raw=True` 以接收完整的 `{"raw", "parsed", "parsing_error"}` 包装，并捕获解析器错误：

```python
structured = ChatParallel(model="lite").with_structured_output(Founder, include_raw=True)
res = structured.invoke([("human", "Who founded SpaceX?")])
res["parsed"]          # Founder(...) or None
res["parsing_error"]   # Exception or None
res["raw"]             # original AIMessage
```

## 引用

研究模型会在 `AIMessage.response_metadata["basis"]` 中填充每个字段的引用、模型推理过程和置信度标签。`response_metadata["interaction_id"]` 会暴露出来以支持多轮对话的上下文链接；当存在时，`system_fingerprint` 也会被传递。

```python
cited = ChatParallel(model="lite").invoke([
    ("human", "Who is the current CEO of OpenAI? One sentence."),
])
print(cited.content)
print("\nbasis:", cited.response_metadata.get("basis"))
print("interaction_id:", cited.response_metadata.get("interaction_id"))
```

## 流式传输

`ChatParallel` 支持按令牌流式传输：

```python
stream = llm.stream_events(messages, version="v3")
for token in stream.text:
    print(token, end="", flush=True)
```

## 异步

```python
ai_msg = await llm.ainvoke(messages)

stream = await llm.astream_events(messages, version="v3")
async for token in stream.text:
    print(token, end="", flush=True)
```

## Token 使用量

Parallel 目前不提供 token 使用量元数据。`usage_metadata` 为 `None`。

```python
ai_msg = llm.invoke(messages)
print(ai_msg.usage_metadata)
# None
```

## 响应元数据

```python
ai_msg = llm.invoke(messages)
print(ai_msg.response_metadata)
# {'model_name': 'speed', 'finish_reason': 'stop', 'created': 1764043410}
```

对于研究模型，`response_metadata` 额外包含 `basis`（按字段引用）、`interaction_id`（用于多轮链式调用），以及在可用时的 `system_fingerprint`。

## 错误处理

集成会在常见失败情况下抛出带有描述性消息的 `ValueError`：

```python
from langchain_parallel import ChatParallel

try:
    llm = ChatParallel(api_key="invalid-key")
    response = llm.invoke([("human", "Hello")])
except ValueError as e:
    if "Authentication failed" in str(e):
        print("Invalid API key provided")
    elif "Rate limit exceeded" in str(e):
        print("API rate limit exceeded, please try again later")
```

## OpenAI 兼容性

`ChatParallel` 接受许多 [OpenAI Chat Completions API](https://platform.openai.com/docs/api-reference/chat) 参数，用于直接替换 OpenAI 客户端迁移。`tools`、`tool_choice`、`top_p` 和 `frequency_penalty` 等高级参数会被接受，但会被 Parallel API 忽略。

```python
llm = ChatParallel(
    model="speed",
    # accepted but ignored by Parallel:
    tools=[{"type": "function", "function": {"name": "example"}}],
    tool_choice="auto",
    top_p=1.0,
    frequency_penalty=0.0,
    presence_penalty=0.0,
    logit_bias={},
    seed=42,
    user="user-123",
)
```

对于结构化输出，推荐使用 `ChatParallel.with_structured_output(...)`（参见[结构化输出](#structured-output)），而不是直接传递 `response_format`。它适用于研究模型，并返回一个已解析的对象。

## 消息处理

该集成会合并连续的类型相同的消息，以满足 API 要求：

```python
from langchain.messages import HumanMessage, SystemMessage

# Consecutive system messages are automatically merged before the API call.
messages = [
    SystemMessage("You are a helpful assistant."),
    SystemMessage("Always be polite and concise."),
    HumanMessage("What is the weather like today?"),
]

response = llm.invoke(messages)
```
