[Harbor](https://harborframework.com/docs) 是一个用于在沙箱环境中评估和优化智能体及语言模型的框架，来自 [Terminal-Bench](https://www.tbench.ai)。Harbor 在隔离容器中运行每个试验，因此您可以同时并行化多个环境中的评估和部署。

该 `langsmith` Harbor 环境在 LangSmith 沙箱上运行这些试验。使用 `-e langsmith` 在 LangSmith 基础设施上执行 Harbor 作业，与 AgentCore、Daytona、E2B 和 Modal 等提供商并列。

## 前提条件

- A [LangSmith 账户](https://smith.langchain.com) 和 API 密钥。
- Python 及 `pip`.

## 安装

使用 `langsmith` extra:

```bash
pip install "harbor[langsmith]"
```

## 身份验证

Harbor 使用您的 LangSmith 凭据进行身份验证。设置 API 密钥：

```bash
export LANGSMITH_API_KEY="<LANGSMITH_API_KEY>"
```

`LANGCHAIN_API_KEY` 也可以使用。或者，选择 [LangSmith SDK 配置文件](/langsmith/profile-configuration) 而不是导出密钥：

```bash
export LANGSMITH_PROFILE=prod
```

## 运行评估

运行 Harbor 作业并使用 `-e langsmith`:

```bash
harbor run -d "<org/name>" \
  -m "<model>" \
  -a "<agent>" \
  -e langsmith \
  -n "<n-parallel-trials>"
```

Harbor 为每个试验创建一个 LangSmith 沙箱，在其中运行智能体和验证器，试验完成后销毁沙箱。

## 配置沙箱环境

LangSmith 环境从文件系统快照启动每个沙箱。在 Harbor 任务中提供以下选项之一：

- **预构建镜像**：设置 `[environment].docker_image` in `task.toml`。Harbor 会重用或从该镜像创建快照。
- **现有快照**：传递 `environment.kwargs.snapshot_name` 以从 [快照](/langsmith/sandbox-snapshots) 启动。
- **Dockerfile**：包含 `environment/Dockerfile`。Harbor 使用 [从 Dockerfile 构建流程](/langsmith/sandbox-snapshots#build-a-snapshot-from-a-dockerfile)从中构建快照，使用任务 `environment/` 目录作为构建上下文。

使用环境参数调整沙箱生命周期，通过命令行传递 `--ek`:

```bash
harbor run -d "<org/name>" \
  -m "<model>" \
  -a "<agent>" \
  -e langsmith \
  -n "<n-parallel-trials>" \
  --ek idle_ttl_seconds=0 \
  --ek delete_after_stop_seconds=7200
```

- `idle_ttl_seconds`：在此秒数后停止空闲沙箱。设置 `0` 以禁用空闲超时。
- `delete_after_stop_seconds`：在此秒数后删除已停止的沙箱。

## 在 LangSmith 上运行 Deep Agents

Deep Agents 作为自定义 Harbor 智能体针对 LangSmith 环境运行。要构建和运行 Deep Agent，请参阅 [Deep Agents 文档](/oss/deepagents/overview)。Harbor 包装器位于 [`deepagents-evals` 包中，它提供](https://github.com/langchain-ai/deepagents/tree/main/libs/evals)并包含现成的 `deepagents_harbor:DeepAgentsWrapper` 示例。 `make run-terminal-bench-*` targets. 在与 Harbor 相同的环境中安装它：

```bash
pip install "harbor[langsmith]"

# From a checkout of langchain-ai/deepagents:
pip install -e libs/evals
```

设置您的 LangSmith 和模型凭证，然后使用包装器运行 Harbor：

```bash
export LANGSMITH_PROFILE=prod
export LANGSMITH_TRACING=true
export LANGSMITH_PROJECT=harbor-deepagents
export ANTHROPIC_API_KEY="<ANTHROPIC_API_KEY>"

harbor run -d "terminal-bench@2.0" \
  --agent-import-path deepagents_harbor:DeepAgentsWrapper \
  --model anthropic:claude-opus-4-8 \
  -e langsmith \
  -n 10 \
  -l 10 \
  --yes \
  --ek idle_ttl_seconds=0 \
  --ek delete_after_stop_seconds=7200
```

将 API 密钥保留在您的 shell 环境中，而不是在作业配置文件中。

## 使用配置文件

在 Harbor 作业配置中捕获相同的运行：

```yaml
jobs_dir: jobs/deepagents-langsmith
n_attempts: 1
n_concurrent_trials: 10
environment:
  type: langsmith
  delete: true
  kwargs:
    idle_ttl_seconds: 0
    delete_after_stop_seconds: 7200
agents:
  - import_path: deepagents_harbor:DeepAgentsWrapper
    model_name: anthropic:claude-opus-4-8
datasets:
  - name: terminal-bench
    version: "2.0"
    n_tasks: 10
```

## 多容器任务

LangSmith 环境支持多容器任务。在任务定义中包含 `environment/docker-compose.yaml` 文件，以便在每次试验中运行多个容器。请参阅 [Harbor 沙箱文档](https://harborframework.com/docs/run-jobs/cloud-sandboxes) 了解更多详情。
