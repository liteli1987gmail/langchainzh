title: 删除组织
sidebarTitle: 删除组织
---

LangSmith UI 目前不支持从 LangSmith 自托管实例中删除单个组织。但是，这可以通过直接从 ClickHouse 中的所有物化视图（runs_history 视图除外）以及 runs 和 feedbacks 表中移除所有痕迹，然后从 Postgres tenants 表中移除该组织来实现。

此命令使用组织 ID 作为参数。

### 前置条件

确保您已准备好以下工具/项目。

1. kubectl

   * [https://kubernetes.io/docs/tasks/tools/](https://kubernetes.io/docs/tasks/tools/)

2. PostgreSQL 客户端

   * [https://www.postgresql.org/download/](https://www.postgresql.org/download/)

3. PostgreSQL 数据库连接：

   * Host（主机）
   * Port（端口）
   * Username（用户名）
     * 如果使用捆绑版本，则为 `postgres`
   * Password（密码）
     * 如果使用捆绑版本，则为 `postgres`
   * Database name（数据库名称）
     * 如果使用捆绑版本，则为 `postgres`

4. Clickhouse 数据库凭据

   * Host（主机）
   * Port（端口）
   * Username（用户名）
     * 如果使用捆绑版本，则为 `default`
   * Password（密码）
     * 如果使用捆绑版本，则为 `password`
   * Database name（数据库名称）
     * 如果使用捆绑版本，则为 `default`

5. 从运行迁移脚本的机器到 PostgreSQL 数据库的连接。

   * 如果您使用的是捆绑版本，可能需要将 postgresql 服务端口转发到本地机器。
   * 运行 `kubectl port-forward svc/langsmith-postgres 5432:5432` 将 postgresql 服务端口转发到本地机器。

6. 从运行迁移脚本的机器到 Clickhouse 数据库的连接。

   * 如果您使用的是捆绑版本，可能需要将 clickhouse 服务端口转发到本地机器。
     * 运行 `kubectl port-forward svc/langsmith-clickhouse 8123:8123` 将 clickhouse 服务端口转发到本地机器。
   * 如果您使用的是 Clickhouse Cloud，则需要指定 --ssl 标志并使用端口 `8443`

7. 用于删除组织的脚本

   * 下载[删除组织脚本](https://github.com/langchain-ai/helm/blob/main/charts/langsmith/scripts/delete_organization_sh)

### 运行单个组织的删除脚本

运行以下命令以执行组织移除脚本：

```bash
sh delete_organization.sh <postgres_url> <clickhouse_url> --organization_id <organization_id>
```

例如，如果您正在使用捆绑版本并使用端口转发，则命令应如下所示：

```bash
sh delete_organization.sh "postgres://postgres:postgres@localhost:5432/postgres" "clickhouse://default:password@localhost:8123/default" --organization_id 4ec70ec7-0808-416a-b836-7100aeec934b
```

如果您访问 LangSmith UI，您现在应该会看到组织已不再存在。
