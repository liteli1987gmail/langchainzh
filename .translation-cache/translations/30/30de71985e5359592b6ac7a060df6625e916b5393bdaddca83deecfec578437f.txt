本指南将引导您完成使用 LangSmith 自托管实例的过程。

<Info>
本指南假设您已经部署了 LangSmith 自托管实例。如果没有，请参阅 [Kubernetes 部署指南](/langsmith/kubernetes)。
</Info>

### 配置要与 LangSmith 配合使用的应用程序

LangSmith 有一个统一的 API，可用于与 hub 和 LangSmith 后端进行交互。

1. 部署实例后，您可以在 `http(s)://<host>` 访问 LangSmith UI。
2. LangSmith API 将在 `http(s)://<host>/api/v1` 提供
3. LangSmith 控制平面将在 `http(s)://<host>/api-host` 提供

要使用实例的 API，您需要在应用程序中设置以下环境变量：

```bash
LANGSMITH_ENDPOINT=http://<host>/api/v1
LANGSMITH_API_KEY=foo # Set to a legitimate API key if using OAuth
```

您也可以直接在 LangSmith SDK 客户端中配置这些变量：

```python
import langsmith
langsmith_client = langsmith.Client(
    api_key='<api_key>',
    api_url='http(s)://<host>/api/v1',
)
```

完成上述设置后，您应该能够运行代码，并在您的自托管实例中看到结果。我们建议您通读[*快速入门指南*](https://docs.smith.langchain.com/#quick-start)来了解如何使用 LangSmith。

### 自签名证书

如果您为自托管的 LangSmith 实例使用自签名证书，可能会出现问题，因为 Python 自带一组受信任的证书，其中可能不包含您的自签名证书。要解决此问题，您可能需要使用 `truststore` 之类的工具将系统证书加载到 Python 环境中。

您可以按如下步骤操作：

1. pip install truststore（或根据您使用的包管理器使用类似的命令）

然后使用以下代码加载系统证书：

```python
import truststore
truststore.inject_into_ssl()
# The rest of your code
import langsmith
langsmith_client = langsmith.Client(
    api_key='<api_key>',
    api_url='http(s)://<host>/api/v1',
)
```
