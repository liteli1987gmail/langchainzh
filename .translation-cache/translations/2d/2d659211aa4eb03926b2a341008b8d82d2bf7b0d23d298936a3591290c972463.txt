    ## 发布节奏

    :::python
    随着 LangChain 1.0 的发布， **次要** 版本（例如，从 `1.0.x` to `1.1.0`) of `langchain` 到 `langchain-core` ）遵循语义化版本控制，可能会频繁发布。次要版本包含新功能和改进，但不包含破坏性更改。

    随着 LangChain 1.0 的发布， **次要** 版本（例如，从 `1.0.x` to `1.1.0`) of `langchain` 到 `@langchain/core` ）遵循语义化版本控制，可能会频繁发布。次要版本包含新功能和改进，但不包含破坏性更改。

    补丁版本会频繁发布，每周可达数次，因为它们包含错误修复和小幅改进。

    ## API 稳定性

    :::python
    LLM 应用的开发是一个快速发展的领域，我们不断从用户和社区中学习。因此，我们预计 `langchain` 和 `langchain-core` 中的 API 将继续发展，以更好地满足用户需求。
    :::
    :::js
    LLM 应用的开发是一个快速发展的领域，我们不断从用户和社区中学习。因此，我们预计 `langchain` 和 `@langchain/core` 中的 API 将继续发展，以更好地满足用户需求。
    :::

    随着 LangChain 1.0 采用语义化版本控制：

    - 对公共 API 的破坏性更改仅会在主要版本发布时发生（例如， `2.0.0`)
    - 次要版本升级（例如， `1.0.0` to `1.1.0`）添加新功能，不会破坏现有功能
    - 补丁版本升级（例如， `1.0.0` to `1.0.1`）包含错误修复和小幅改进

    我们通常会尽量避免不必要的更改，并为即将移除的功能提供弃用政策。

    ### 其他包的稳定性

    LangChain 生态系统中其他包的稳定性可能会有所不同：

    - **LangChain 维护的合作伙伴包** （例如 `langchain-openai` 和 `langchain-anthropic`）遵循语义化版本控制，预计在 1.0 版本后保持稳定。其他合作伙伴包可能遵循不同的稳定性和版本控制政策，用户应参考这些包的文档以获取更多信息。

    ## 弃用政策

    我们通常会避免弃用功能，直到有更好的替代方案可用。

    :::python
    通过 LangChain 1.0 的语义化版本控制方法，弃用的功能将在整个 1.x 版本系列中继续工作。破坏性更改（包括移除弃用的功能）仅会在主要版本发布时发生（例如 2.0）。

    当某个功能在 `langchain` or `langchain-core`中被弃用时，我们将：
    - 在代码和文档中明确标记为已弃用
    - 提供迁移指南，指向推荐替代方案
    - 在整个 1.x 次要版本中为已弃用功能提供安全更新
    :::

    在某些情况下，如果已弃用功能不会造成维护问题，我们可能会允许其在代码库中保留更长时间，以进一步减轻用户的负担。

    ## 长期支持 (LTS)

    LangChain 遵循长期支持 (LTS) 策略，为生产应用程序提供稳定性：

    ### 发布状态定义

    包会标记为以下状态之一：
    - **ACTIVE**：当前活跃开发，包括错误修复、安全补丁和新功能
    - **MAINTENANCE**：接收所有安全补丁和关键错误修复，但不包含新功能

    ### 当前 LTS 版本

    **LangChain 1.0** 被指定为 LTS 版本：
    - **状态**：ACTIVE 状态，直至 2.0 发布
    - **支持周期**：2.0 发布后，1.0 将进入 MAINTENANCE 模式，至少持续 1 年
    - **语义化版本合规性**：用户可以在次要版本之间升级（例如从 1.0 到 1.1）而不会产生破坏性变更

    ### 旧版本支持

    **LangChain 0.3**:

    - **状态**：MAINTENANCE 模式
    - **支持周期**：直至 2026 年 12 月
    - **支持包括**：安全补丁和关键错误修复

    <Tab title="LangGraph">
    LangGraph 遵循结构化的发布策略，以确保为构建生产应用程序的用户提供稳定性和可预测性。

    ## 发布节奏

    我们预计将 **主** 版本发布间隔至少 6-12 个月，以期为生产应用程序提供稳定性。

    **次要** 版本通常每隔 1-2 个月发布一次，包含新功能和改进。

    **补丁** 版本按需发布，通常每周一次，以解决错误和安全问题。

    ## API 稳定性

    ### 稳定 API

    所有没有特殊前缀的 API 都被视为稳定的，可用于生产环境。我们在一个主版本内保持稳定功能的向后兼容性。

    ### Beta 功能

    标记为 `beta` 的功能在文档中标注如下：

    - 功能完整且已测试
    - 可安全用于生产环境，但需了解它们可能会发生变化
    - 可能会根据用户反馈进行小幅 API 调整

    ### 实验性功能

    标记为 `experimental` or `alpha`:

    - 正在积极开发中
    - 可能会发生重大变化或被移除
    - 在生产环境中使用时应谨慎

    ### 内部 API

    以下划线（_）为前缀的 API（`_`）或明确标记为内部的：

    - 不属于公共 API 的一部分
    - 可能会在不通知的情况下发生变化
    - 不应直接使用

    ## 弃用策略

    弃用功能时：

    1. **弃用通知**：功能被标记为弃用，并提供清晰的迁移指导
    2. **宽限期**：弃用的功能在至少一个次要版本中保持可用
    3. **移除**：功能仅在主要版本发布时移除
    4. **迁移支持**：我们提供迁移指南，并在可能的情况下提供自动化工具

    ## 平台兼容性

    :::python
    ### Python 支持

    - 我们支持由 Python Software Foundation 积极维护的 Python 版本
    - Python 版本要求仅在主要版本中可能发生变化
    - 当前需要 Python 3.10 或更高版本
    :::

    :::js
    ### JavaScript/TypeScript support

    - 我们支持 Node.js LTS 版本
    - 为所有公共 API 提供 TypeScript 定义
    - 客户端组件的浏览器兼容性已有文档记录
    :::

    ## 破坏性变更

    破坏性变更仅在主要版本中引入，包括：

    - 移除已弃用的 API
    - 必需参数的变更
    - 影响现有应用程序的默认行为变更
    - Minimum Python/Node.js version updates

    ## 迁移支持

    对于主要版本升级，我们提供：

    - 全面的迁移指南
    - 在可行的情况下提供自动迁移脚本
    - 为前一个主要版本提供扩展支持期
    - 明确记录所有破坏性变更

    ## 长期支持 (LTS)

    LangGraph 采用长期支持 (LTS) 策略，为生产应用程序提供稳定性：

    ### 发布状态定义

    包被标记为以下状态之一：

    - **ACTIVE**：当前正在积极开发，包括错误修复、安全补丁和新功能
    - **MAINTENANCE**接收所有安全补丁和关键错误修复，但不包含新功能

    ### 当前的 LTS 版本

    **LangGraph 1.0** 被指定为 LTS 版本：

    - **状态**：在 2.0 发布前为 ACTIVE 状态
    - **支持周期**：在 2.0 发布后，1.0 将进入 MAINTENANCE 模式，至少持续 1 年
    - **语义化版本兼容**：用户可以在次版本之间升级（如 1.0 到 1.1）而不会产生破坏性变更

    ### 旧版本支持

    **LangGraph 0.4**:

    - **状态**：MAINTENANCE 模式
    - **支持周期**：截至 2026 年 12 月
    - **支持内容包含**：所有安全补丁和关键错误修复

    <Tab title="Deep Agents">
    :::python
    Deep Agents (`deepagents`）是一个快速发展的包，用于构建高级智能体架构。

    ## 1.0 前状态

    Deep Agents 目前处于 1.0 前开发阶段。虽然我们尽可能减少破坏性变更，但随着我们吸收社区反馈并适应快速变化的智能体应用领域，API 可能会继续演进。

    ## 发布节奏

    **次版本** 发布（如 `0.1.0` to `0.2.0`）可能包含新功能和潜在的破坏性变更，因为该包正在逐步成熟。

    **补丁** 发布（如 `0.1.0` to `0.1.1`）包含错误修复和小幅改进，不会有破坏性变更。

    ## API 稳定性

    作为一个 1.0 前版本：

    - API 在次版本之间可能会发生变化
    - 我们致力于通过在变更发生时提供清晰的迁移指南来最大程度地减少干扰
    - 标记为 `experimental` or `alpha` 的功能可能会发生更重大的变更

    ## 弃用政策

    在弃用 Deep Agents 中的功能时：

    - 我们尽可能提前提供弃用警告
    - 迁移指南包含在发布说明中
    - 弃用的功能会在后续次版本中移除

    ## 通往 1.0 的路径

    Deep Agents 将在以下情况下达到 1.0：

    - 核心 API 已基于社区反馈稳定下来
    - 该包已在生产环境中经过实战检验
    - 核心功能不再预期会有破坏性变更

    1.0 之后，Deep Agents 将遵循与 LangChain 和 LangGraph 相同的语义版本控制和 LTS 策略。
