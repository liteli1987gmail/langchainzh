    ## 发布周期

    :::python
    随着 LangChain 1.0 的发布，`langchain` 和 `langchain-core` 的**次要**版本（例如从 `1.0.x` 到 `1.1.0`）遵循语义化版本控制，可能频繁发布。次要版本包含新功能和改进，但不包含破坏性更改。

    补丁版本会频繁发布，每周最多几次，因为它们包含错误修复和微小改进。

    ## API 稳定性

    :::python
    LLM 应用的开发是一个快速发展的领域，我们不断从用户和社区中学习。因此，我们预计 `langchain` 和 `langchain-core` 中的 API 将继续发展，以更好地满足用户需求。
    :::
    :::js
    LLM 应用的开发是一个快速发展的领域，我们不断从用户和社区中学习。因此，我们预计 `langchain` 和 `@langchain/core` 中的 API 将继续发展，以更好地满足用户需求。
    :::

    随着 LangChain 1.0 采用语义化版本控制：

    - 对公共 API 的破坏性更改只会在主要版本发布时发生（例如 `2.0.0`）
    - 次要版本升级（例如 `1.0.0` 到 `1.1.0`）添加新功能而不破坏现有功能
    - 补丁版本升级（例如 `1.0.0` 到 `1.0.1`）包含错误修复和微小改进

    我们通常会尽量避免不必要的更改，并为即将移除的功能提供弃用策略。

    ### 其他包的稳定性

    LangChain 生态系统中其他包的稳定性可能有所不同：

    - **由 LangChain 维护的合作伙伴包**（如 `langchain-openai` 和 `langchain-anthropic`）遵循语义化版本控制，预计在 1.0 版本后稳定。其他合作伙伴包可能遵循不同的稳定性和版本控制策略，用户应参阅这些包的文档获取更多信息。

    ## 弃用策略

    在有更好的替代方案之前，我们通常会避免弃用功能。

    :::python
    凭借 LangChain 1.0 的语义化版本控制方法，弃用的功能将在整个 1.x 版本系列中继续工作。破坏性更改（包括移除弃用功能）只会在主要版本发布时发生（例如 2.0）。

    当某个功能在 `langchain` 或 `langchain-core` 中被弃用时，我们将：
    - 在代码和文档中清楚地标记为已弃用
    - 提供迁移指导以推荐替代方案
    - 在整个 1.x 次要版本中为弃用功能提供安全更新
    :::

    ## 长期支持 (LTS)

    LangChain 遵循长期支持 (LTS) 策略，为生产应用提供稳定性：

    ### 发布状态定义

    包会标记为以下状态之一：
    - **ACTIVE**：当前活跃开发，包括错误修复、安全补丁和新功能
    - **MAINTENANCE**：接收所有安全补丁和关键错误修复，但不包含新功能

    ### 当前的 LTS 版本

    **LangChain 1.0** 被指定为 LTS 版本：
    - **状态**：在 2.0 发布之前为 ACTIVE
    - **支持期限**：2.0 发布后，1.0 将进入 MAINTENANCE 模式至少 1 年
    - **语义版本合规性**：用户可以在次要版本之间升级（例如 1.0 到 1.1）而不会破坏现有功能

    ### 旧版本支持

    **LangChain 0.3**：

    - **状态**：MAINTENANCE 模式
    - **支持期限**：直到 2026 年 12 月
    - **支持包含**：安全补丁和关键错误修复

    <Tab title="LangGraph">
    LangGraph 遵循结构化的发布策略，以确保为构建生产应用的用户提供稳定性和可预测性。

    ## 发布周期

    我们预计主要版本发布间隔至少为 6-12 个月，以，为生产应用提供稳定性。

    **次要**版本通常每 1-2 个月发布一次，包含新功能和改进。

    **补丁**版本按需发布，通常每周一次，以解决错误和安全问题。

    ## API 稳定性

    ### 稳定 API

    所有没有特殊前缀的 API 都被视为稳定，可用于生产环境。我们维护主要版本内稳定功能的后向兼容性。

    ### Beta 功能

    在文档中标记为 `beta` 的功能：

    - 功能完整且经过测试
    - 可安全用于生产环境，但需了解它们可能会更改
    - 根据用户反馈可能会进行次要 API 调整

    ### 实验性功能

    标记为 `experimental` 或 `alpha` 的功能：

    - 正在积极开发中
    - 可能会发生重大变化或被移除
    - 在生产环境中使用时应谨慎

    ### 内部 API

    带下划线 (`_`) 前缀或明确标记为内部使用的 API：

    - 不属于公共 API 的一部分
    - 可能会在不通知的情况下更改
    - 不应直接使用

    ## 弃用策略

    弃用功能时：

    1. **弃用通知**：功能标记为已弃用，并提供清晰的迁移指导
    2. **宽限期**：弃用的功能至少在下一个次要版本中保持可用
    3. **移除**：功能只在主要版本发布时移除
    4. **迁移支持**：我们提供迁移指南，在可行的情况下提供自动化工具

    ## 平台兼容性

    :::python
    ### Python 支持

    - 我们支持由 Python Software Foundation 积极维护的 Python 版本
    - Python 版本要求可能只在主要版本中更改
    - 当前需要 Python 3.10 或更高版本
    :::

    :::js
    ### JavaScript/TypeScript 支持

    - 我们支持 Node.js LTS 版本
    - 为所有公共 API 提供 TypeScript 类型定义
    - 客户端组件的浏览器兼容性有文档记录
    :::

    ## 破坏性更改

    破坏性更改只在主要版本中引入，包括：

    - 移除已弃用的 API
    - 更改必需参数
    - 更改影响现有应用的默认行为
    - 最低 Python/Node.js 版本更新

    ## 迁移支持

    对于主要版本升级，我们提供：

    - 全面的迁移指南
    - 在可行的情况下提供自动化迁移脚本
    - 上一个主要版本的延长支持期
    - 所有破坏性更改的清晰文档

    ## 长期支持 (LTS)

    LangGraph 遵循长期支持 (LTS) 策略，为生产应用提供稳定性：

    ### 发布状态定义

    包会标记为以下状态之一：

    - **ACTIVE**：当前活跃开发，包括错误修复、安全补丁和新功能
    - **MAINTENANCE**：接收所有安全补丁和关键错误修复，但不包含新功能

    ### 当前的 LTS 版本

    **LangGraph 1.0** 被指定为 LTS 版本：

    - **状态**：在 2.0 发布之前为 ACTIVE
    - **支持期限**：2.0 发布后，1.0 将进入 MAINTENANCE 模式至少 1 年
    - **语义版本合规性**：用户可以在次要版本之间升级（例如 1.0 到 1.1）而不会破坏现有功能

    ### 旧版本支持

    **LangGraph 0.4**：

    - **状态**：MAINTENANCE 模式
    - **支持期限**：直到 2026 年 12 月
    - **支持包含**：所有安全补丁和关键错误修复

    <Tab title="Deep Agents">
    :::python
    Deep Agents（`deepagents`）是一个快速发展的包，用于构建高级代理架构。

    ## 1.0 之前的状态

    Deep Agents 目前处于 1.0 之前的开发阶段。虽然我们尽可能减少破坏性更改，但随着我们整合社区的学习成果并适应快速变化的代理应用领域，API 可能会发展。

    ## 发布周期

    **次要**版本（例如 `0.1.0` 到 `0.2.0`）可能包含新功能，并随着包的成熟可能包含破坏性更改。

    **补丁**版本（例如 `0.1.0` 到 `0.1.1`）包含错误修复和微小改进，不会破坏现有功能。

    ## API 稳定性

    作为 1.0 之前的包：

    - API 可能在次要版本之间发生变化
    - 我们旨在通过在发生更改时提供清晰的迁移指导来最大程度减少干扰
    - 标记为 `experimental` 或 `alpha` 的功能可能会发生更重大的更改

    ## 弃用策略

    在 Deep Agents 中弃用功能时：

    - 我们尽可能提前提供弃用警告
    - 迁移指导包含在发布说明中
    - 弃用的功能在后续次要版本中移除

    ## 通往 1.0 的道路

    Deep Agents 将在以下情况下达到 1.0：

    - 核心 API 基于社区反馈已稳定
    - 该包已在生产环境中经过实战测试
    - 核心功能不再期望有破坏性更改

    在 1.0 之后，Deep Agents 将遵循与 LangChain 和 LangGraph 相同的语义化版本控制和 LTS 策略。
