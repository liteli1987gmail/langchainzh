Deep Agents Code（`dcode`）是一个基于 [Deep Agents SDK](/oss/deepagents/quickstart) 构建的开源编程智能体。
它可与任何大型语言模型配合使用，并支持在会话中途切换提供商或模型。
持久化记忆可在对话之间保持上下文，可定制的技能塑造其行为，审批控制可限制代码执行。

## 快速开始

<Steps>

    <Step title="安装并启动" icon="terminal">
        ```bash
        curl -LsSf https://langch.in/dcode | bash
        ```
{/* ![Deep Agents Code](/oss/images/deepagents/deepagents-cli.png) */}
        {/* TODO: maybe add back when updating to current */}

    </Step>

    <Step title="Add provider credentials" icon="key">
        Deep Agents Code 可与任何工具调用型 LLM 配合使用。OpenAI、Anthropic 和 Google 开箱即用。

        使用 `/auth` 命令连接提供商。参见 [Providers](/oss/deepagents/code/providers) 获取完整列表和凭证详情。

        <Note>
            Web 搜索使用 [Tavily](https://tavily.com)。从 `/auth` 添加密钥或设置 `TAVILY_API_KEY`。参见[启用 Web 搜索](/oss/deepagents/code/configuration#enable-web-search-with-tavily)。
        </Note>
    </Step>

    <Step title="Choose a model (optional)" icon="cpu">
        在会话中运行 `/model` 打开交互式切换器，或使用 `--model` 启动：


</fragment>

        ```bash
        dcode --model anthropic:claude-opus-4-8
        dcode --model openai:gpt-5.5
        dcode --model fireworks:accounts/fireworks/models/deepseek-v4-pro
        dcode --model baseten:moonshotai/Kimi-K2.7-Code
        ```

请参阅[模型提供商](/oss/deepagents/code/providers)以获取完整提供商列表、开源权重选项和凭证详情。
</Step>

<Step title="给代理分配任务" icon="message">
        ```txt
        Create a Python script that prints "Hello, World!"
        ```

代理会解读您的查询，并在修改文件之前通过差异对比（diffs）提出更改建议以供您审批。如有需要，它还可以运行 shell 命令来测试代码、查看文档或搜索网络以获取最新信息。
    </Step>

    <Step title="启用追踪（可选）" icon="chart-dots">
        要在 LangSmith 中记录代理操作、工具调用和决策，请将以下内容添加到 `~/.deepagents/.env` 或在您的 shell 中导出这些变量：

        ```bash title="~/.deepagents/.env"
        LANGSMITH_TRACING=true
        LANGSMITH_API_KEY=lsv2_...
        LANGSMITH_PROJECT=optional-project-name  # Specify a project name or default to "deepagents-code"
        ```

有关更多详情和使用方法，请参阅[使用 LangSmith 进行追踪](#trace-with-langsmith)。
    </Step>
</Steps>

<Note>
    Deep Agents Code 在 Windows 上不受官方支持。Windows 用户可以尝试在 [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) 下运行。
</Note>

## 功能

Deep Agents Code 具有以下内置功能：

* <Icon icon="file" size={16} /> **文件操作** - 读取、写入和编辑磁盘上的文件。
* <Icon icon="terminal" size={16} /> **Shell 执行** - 执行命令以运行测试、构建项目、管理依赖项以及与版本控制系统交互。
* <Icon icon="cloud" size={16} /> **[远程沙盒](/oss/deepagents/code/remote-sandboxes)** - 在远程运行代理工具，而非本地机器。
* <Icon icon="search" size={16} /> **网页搜索** - 搜索网络以获取最新信息和文档。需要 [Tavily API 密钥](/oss/deepagents/code/configuration#enable-web-search-with-tavily)。
* <Icon icon="list-check" size={16} /> **任务规划与追踪** - 将复杂任务分解为离散的步骤并追踪进度。
* <Icon icon="users" size={16} /> **[子代理](/oss/deepagents/code/subagents)** - 将工作委托给任务特定的子代理。
* <Icon icon="brain" size={16} /> **[记忆存储与检索](/oss/deepagents/code/memory-and-skills#memory)** - 跨会话存储和检索信息，使代理能够记住项目约定和已学习的模式。
* <Icon icon="arrows-minimize" size={16} /> **上下文压缩与卸载** - 总结较旧的对话消息并将原始消息卸载到存储中。
* <Icon icon="user" size={16} /> **人工介入** - 对敏感的工具操作需要人工批准。
* <Icon icon="puzzle" size={16} /> **[技能](/oss/deepagents/code/memory-and-skills#skills)** - 使用自定义专业知识和指令扩展代理能力。
* <Icon icon="plug" size={16} /> **[MCP 工具](/oss/deepagents/code/mcp-tools)** - 从 [Model Context Protocol](https://modelcontextprotocol.io/) 服务器加载外部工具。
* <Icon icon="chart-dots" size={16} /> **[追踪](/oss/deepagents/code/overview#trace-with-langsmith)** - 在 LangSmith 中追踪代理操作，以实现可观测性和调试。

<Accordion title="内置工具完整列表">
    ## 内置工具

    代理附带以下内置工具，无需配置即可使用：

    | 工具 | 描述 | 人工介入 |
    |------|-------------|-------------------|
    | `ls` | 列出文件和目录 | - |
    | `read_file` | 读取文件内容；为图像、音频、视频和 PDF 返回多模态块 | - |
    | `write_file` | 创建或覆盖文件 | 需要<sup>1</sup> |
    | `edit_file` | 对现有文件进行针对性编辑 | 需要<sup>1</sup> |
    | `glob` | 查找匹配模式的文件 | - |
    | `grep` | 在文件中搜索文本模式 | - |
    | `execute` | 在本地或[远程沙盒](/oss/deepagents/code/remote-sandboxes)中执行 shell 命令 | 需要<sup>1</sup> |
    | `web_search` | 使用 Tavily 进行网页搜索（请参阅[启用网页搜索](/oss/deepagents/code/configuration#enable-web-search-with-tavily)） | 需要<sup>1</sup> |
    | `fetch_url` | 获取并转换网页为 markdown | 需要<sup>1</sup> |
    | `task` | 将工作委托给[子代理](/oss/deepagents/code/subagents)进行并行执行<sup>3</sup> | 需要<sup>1</sup> |
    | `ask_user` | 向用户提出自由形式或多项选择问题 | - |
    | `compact_conversation` | 总结较旧的消息，将原始消息卸载到后端存储，并用摘要替换上下文中的原始消息 | 混合<sup>2</sup> |
    | `write_todos` | 创建和管理复杂工作的任务列表 | - |
    | `get_current_thread_id` | 返回当前线程 ID，用于 LangSmith 或 MCP 工具 | - |

    <sup>1</sup>：潜在破坏性操作需要用户在执行前批准。要绕过人工批准，您可以切换自动批准（shift+tab）或使用以下选项启动：

    ```bash
    dcode --auto-approve
    # shorter alias:
    dcode -y
    ```

<Note>
        非交互模式默认禁用 shell。使用 `-S`/`--shell-allow-list`（或 `DEEPAGENTS_CODE_SHELL_ALLOW_LIST`）将命令加入白名单。使用 `recommended` 可获得只读安全默认值，或使用 `all` 允许任何操作。请参阅[非交互模式和管道传输](#non-interactive-mode-and-piping)。
    </Note>

    <sup>2</sup>：Deep Agents Code 会在 token 使用量超过模型感知阈值时自动在后台卸载对话。卸载时通过 LLM 对较旧的消息进行摘要，并将原始内容弹出到存储（`/conversation_history/{thread_id}.md`），同时用摘要替换上下文中的原始内容。必要时，代理仍可从已卸载的文件中检索完整历史记录。`compact_conversation` 工具允许代理（或您）按需触发卸载。作为工具调用时，默认需要用户批准。

    <sup>3</sup>：当通过 `config.toml` 中的 `[async_subagents]` 部分配置异步子代理时（请参阅[异步子代理](/oss/deepagents/async-subagents)），可使用额外的工具：`start_async_task`、`update_async_task` 和 `cancel_async_task`（均需批准），以及 `check_async_task` 和 `list_async_tasks`。
</Accordion>

## 命令参考

```bash
# Use a specific agent configuration
dcode --agent mybot

# Use a specific model (provider:model format or auto-detect)
dcode --model anthropic:claude-opus-4-8
dcode --model gpt-5.5

# Auto-approve tool usage (skip human-in-the-loop prompts)
dcode -y

# list directory contents, then summarize directory as first prompt—the command runs first, then the prompt is submitted
# the prompt does NOT have access to the command output
dcode --startup-cmd "ls -la" -m "Summarize what's in this directory"

# Non-interactive with startup command: show git status before the task runs
# the task does NOT have access to the command output
dcode --startup-cmd "git diff --stat" -n "Review these changes"
```

        | `--no-stream`          | 缓冲完整响应并一次性写入 stdout，而不是流式输出。需要 `-n` 或管道 stdin |
        | `--stdin`              | 显式从 stdin 读取输入而不是自动检测。当 stdin 不可用或为 TTY 时会明确报错 |
        | `-y`, `--auto-approve` | 自动批准所有工具调用而不提示（禁用人在回路）。在交互式会话期间可通过 `Shift+Tab` 切换 |

## 配置

完整参考（包括 `config.toml` 模式、提供商参数、profile overrides 和 hook 配置）请参见[配置](/oss/deepagents/code/configuration)。

Deep Agents Code 在 `~/.deepagents/` 下存储所有配置。在该目录中，每个代理都有自己的子目录（默认：`agent`）：

| 路径 | 用途 |
|------|------|
| `~/.deepagents/config.toml` | 模型和代理默认值、提供商设置、构造函数参数、profile overrides、主题、更新设置 |
| `~/.deepagents/.env` | 全局 API 密钥和密钥。请参见[配置](/oss/deepagents/code/configuration#environment-variables) |
| `~/.deepagents/hooks.json` | [生命周期事件 hooks](/oss/deepagents/code/configuration#hooks)（会话开始/结束、任务完成等） |
| `~/.deepagents/<agent_name>/` | 按代理的记忆、技能和对话线程 |
| `.deepagents/`（项目根目录） | 项目特定的记忆和技能，在 git 仓库内运行时加载 |

## 交互模式

像在聊天界面中一样自然地输入。代理使用其内置工具、技能和记忆来帮助您完成任务。

<AccordionGroup>
    <Accordion title="斜杠命令" icon="slash">
        在 Deep Agents Code 会话中使用这些命令：

        - `/model` - 切换模型或打开交互式模型选择器。
        - `/agents` - 热切换预配置的代理而无需重新启动。详细信息请参见[命令参考](/oss/deepagents/code/overview#command-reference)
        - `/auth` - 管理模型提供商和服务的已存储 API 密钥（如 Tavily 网络搜索）。详细信息请参见[提供商凭据](/oss/deepagents/code/configuration#provider-credentials)
        - `/remember [context]` - 回顾对话并更新记忆和技能。可选传递额外上下文
        - `/skill:<name> [args]` - 直接按名称调用技能。技能的 `SKILL.md` 说明会连同您提供的任何参数一起注入到提示中
        - `/skill-creator [task]` - 创建有效代理技能的指南
        - `/offload`（别名 `/compact`）- 通过将消息卸载到存储并用摘要占位符替换来释放上下文窗口空间。代理可以根据需要从卸载的文件中检索完整历史记录
        - `/tokens` - 显示当前上下文窗口令牌使用情况细分
        - `/clear` - 清除对话历史并开始新线程
        - `/copy` - 将最新的助手消息复制到剪贴板
        - `/threads` - 浏览并恢复之前的对话线程
        - `/mcp [login <server> | reconnect]` - 显示活动的 MCP 服务器和工具。`login <server>` 为服务器运行 OAuth 流程；`reconnect` 加载延迟的登录
        - `/notifications` - 配置启动警告偏好
        - `/reload` - 重新读取 `.env` 文件、刷新配置并重新发现技能而无需重启。会话状态会保留。参见[`DEEPAGENTS_CODE_` 前缀](/oss/deepagents/code/configuration#deepagents_code_-prefix)以了解覆盖行为
        - `/theme` - 打开交互式主题选择器以切换颜色主题。可用内置主题以及任何[用户定义的主题](/oss/deepagents/code/configuration#themes)
        - `/update` - 检查并安装 Deep Agents Code 更新。检测您的安装方式（uv、Homebrew、pip）并运行适当的升级命令
        - `/auto-update` - 切换自动更新开关
        - `/install` - 安装可选额外依赖（例如 `quickjs`、`daytona`、`fireworks`）
        - `/trace` - 在 LangSmith 中打开当前线程（需要 `LANGSMITH_API_KEY`）
        - `/editor` - 在外部编辑器（`$VISUAL` / `$EDITOR`）中打开当前提示。参见[外部编辑器](/oss/deepagents/code/configuration#external-editor)
        - `/timestamps` - 切换消息时间戳页脚
        - `/changelog` - 在浏览器中打开 Deep Agents Code 更新日志
        - `/docs` - 在浏览器中打开文档
        - `/feedback` - 打开 GitHub issues 页面提交错误报告或功能请求
        - `/version` - 显示已安装的 `deepagents-code` 和 SDK 版本
        - `/help` - 显示帮助和可用命令
        - `/quit` - 退出应用程序
    </Accordion>

    <Accordion title="Shell 命令" icon="prompt">
        输入 `!` 进入 shell 模式，然后输入您的命令。

</AccordionGroup>

        ```bash
        git status
        npm test
        ls -la
        ```
</Accordion>

    <Accordion title="键盘快捷键" icon="keyboard">
        **通用**

        | 快捷键 | 操作 |
        |-|-|
        | `Enter` | 提交提示 |
        | `Shift+Enter`、`Ctrl+J`、`Alt+Enter` 或 `Ctrl+Enter` | 插入换行 |
        | `@filename` | 自动补全文件并注入内容 |
        | `Shift+Tab` 或 `Ctrl+T` | 切换自动批准 |
        | `Ctrl+X` | 在外部编辑器中打开提示 |
        | `Ctrl+N` | 查看待处理通知 |
        | `Ctrl+O` | 展开/折叠最近的工具输出 |
        | `Escape` | 中断当前操作 |
        | `Ctrl+C` | 中断或退出 |
        | `Ctrl+D` | 退出 |

        **提示框文本编辑**

        聊天输入框使用标准的 readline 风格绑定：

        | 快捷键 | 操作 |
        |-|-|
        | `Ctrl+A` 或 `Home` | 将光标移动到行首 |
        | `Ctrl+E` 或 `End` | 将光标移动到行尾 |
        | `Ctrl+U` | 删除从光标到行首的内容 |
        | `Ctrl+K` | 删除从光标到行尾的内容 |
        | `Ctrl+W` 或 `Ctrl+Backspace` | 删除左侧单词 |
        | `Ctrl+Left` / `Ctrl+Right` | 光标向左/右移动一个单词 |

        <Note>
            **macOS `Cmd+Left` / `Cmd+Right` / `Cmd+Delete`**

            终端模拟器会在按键到达运行中的应用程序之前拦截 `Cmd` 修改键，因此 Deep Agents Code 不会直接接收它们。相反，终端会将它们转换为上述的 readline 快捷键。

            - **Ghostty：** 开箱即用。`Cmd+Left`、`Cmd+Right` 和 `Cmd+Delete` 默认会被转换为 `Ctrl+A`、`Ctrl+E` 和 `Ctrl+U`。
            - **iTerm2：** 默认未绑定。在 **设置 → 配置 → 键 → 键映射** 下添加以下内容，选择 **发送带有 vim 特殊字符的文本**：
                - `Cmd+Left` → `\x01` (Ctrl+A)
                - `Cmd+Right` → `\x05` (Ctrl+E)
                - `Cmd+Delete` → `\x15` (Ctrl+U)
            - **Terminal.app：** 没有原生 UI 进行此重映射。请直接使用基于 `Ctrl` 的快捷键。

            单词级移动（`Option+Left` / `Option+Right`）的处理方式相同：终端发送 `Esc+b` / `Esc+f`，Deep Agents Code 会将其解释为向左/右移动一个单词。
        </Note>
    </Accordion>
</AccordionGroup>

## 非交互模式和管道传输

使用 `-n` 运行单个任务而不启动交互式界面：

```bash
dcode -n "Write a Python script that prints hello world"
```

每次非交互式运行都会启动一个新线程——对话历史不会在调用之间保留。文件状态（记忆、技能、配置）会持续保存。

您也可以通过 stdin 管道输入。当输入通过管道传输时，Deep Agents Code 会自动以非交互模式运行：

```bash
echo "Explain this code" | dcode
cat error.log | dcode -n "What's causing this error?"
git diff | dcode -n "Review these changes"
git diff | dcode --skill code-review -n 'summarize changes'
```

当您将管道输入与 `-n` 或 `-m` 结合使用时，管道内容会首先显示，然后是您传递给该标志的文本。

<Note>
    管道输入的最大大小为 10 MiB。
</Note>

在非交互模式下，Shell 执行默认是禁用的。使用 `-S`/`--shell-allow-list` 可以启用特定命令（例如 `-S "pytest,git,make"`）、`recommended` 以获得安全默认值，或 `all` 以允许任何命令。

<AccordionGroup>
    <Accordion title="限制轮次计数" icon="gauge">
        在 CI/CD 流水线中，长时间运行或行为异常的代理可能会无限循环。`--max-turns N` 为操作员提供硬性上限，无需触及 SDK 内部：


</AccordionGroup>

        ```bash
        dcode -n "fix the failing tests" --max-turns 10
        ```

`N` 必须是正整数，用于覆盖内部安全默认值（该默认值用于限制失控循环）。当超出预算时，退出码为 124（与 GNU `timeout` 一致），以便 CI 区分预算超限与常规故障。需要 `-n` 或管道输入 stdin；否则退出码为 2。

如需基于时间的限制（替代或补充基于轮次数的限制），请参阅[使用 `--timeout` 限制墙钟时间](#non-interactive-mode-and-piping)。
</Accordion>

<Accordion title="限制墙钟时间" icon="clock">
`--timeout SECONDS` 对非交互式运行强制执行硬性墙钟时间限制。它与 `--max-turns`（轮次计数）互补，提供基于时间的预算——先达到任一限制都会取消代理。

        ```bash
        # Fail fast in CI if the task takes more than 2 minutes
        dcode -n "run the test suite and summarise failures" --timeout 120

        # Combine with --max-turns—whichever limit is hit first stops the agent
        dcode -n "refactor auth module" --timeout 300 --max-turns 20
        ```

过期时，代理被取消，进程以退出码 124 退出，与 `--max-turns` 使用的退出码相同，因此 CI 可以统一处理这两种预算超限情况。需要 `-n` 或管道输入 stdin；否则以退出码 2 退出。
    </Accordion>

    <Accordion title="清洁输出和缓冲" icon="buffer">
        使用 `-q` 可以获得适合管道传输到其他命令的清洁输出，使用 `--no-stream` 可以缓冲完整响应后再写入 stdout（而非流式传输）：


</fragment>

        ```bash
        dcode -n "Generate a .gitignore for Python" -q > .gitignore
        dcode -n "List dependencies" -q --no-stream | sort
        ```

<fragment>

        在非交互模式下，代理会被指示做出合理的假设并自主继续执行，而不是提出澄清问题。它还倾向于使用非交互式命令变体（例如 `npm init -y`、`apt-get install -y`）。
    </Accordion>

    <Accordion title="Shell 执行示例" icon="shield-check">

</fragment>
        ```bash
        # Allow specific commands (validated against the list)
        dcode -n "Run the tests and fix failures" -S "pytest,git,make"

        # Use the curated safe-command list
        dcode -n "Build the project" -S recommended

        # Allow any shell command
        dcode -n "Fix the build" -S all
        ```
</Accordion>
</AccordionGroup>

<Warning>
    **请谨慎使用。**

    `-S all`（或 `--shell-allow-list all`）允许代理执行任意 shell 命令，无需人工确认。
</Warning>

## 使用 LangSmith 追踪

启用 [LangSmith](https://smith.langchain.com) 追踪功能，在 LangSmith 项目中查看代理操作、工具调用和决策过程。

将追踪密钥添加到 `~/.deepagents/.env`，这样每次会话都会启用追踪，无需每次手动导出：

```bash title="~/.deepagents/.env"
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=lsv2_...
LANGSMITH_PROJECT=optional-project-name  # Specify a project name or default to "deepagents-code"
```

若要为特定项目进行覆盖，请在项目目录中的 `.env` 文件中添加相同的键。请参阅[环境变量](/oss/deepagents/code/configuration#environment-variables)了解完整的加载顺序。

如果你愿意，也可以将这些设置为 shell 环境变量。Shell 导出始终优先于 `.env` 值，因此，这对于临时覆盖或测试来说是一个很好的选择：

```bash
export LANGSMITH_TRACING=false
```

<Accordion title="分离代理追踪与应用追踪">
    Deep Agents Code 可以生成两种 LangSmith 追踪：

    - `Agent traces` 是 Deep Agents Code 自身的模型调用、工具调用、编排和中间件。
    - `Shell-command traces` 是 Deep Agents Code 在 shell 中为您运行的代码发出的追踪，例如测试、脚本或本地 LangGraph 应用。

    要将 Deep Agents Code 自身的追踪发送到专用项目，请设置 `DEEPAGENTS_CODE_LANGSMITH_PROJECT`：

</Accordion>

    ```bash title="~/.deepagents/.env"
    # Example value; use any LangSmith project name you want.
    DEEPAGENTS_CODE_LANGSMITH_PROJECT=deepagents-code
    ```

然后为您的应用追踪配置 `LANGSMITH_PROJECT`：

    ```bash title=".env"
    LANGSMITH_PROJECT=customer-support-agent
    ```

例如，假设您要求 Deep Agents Code 调试一个失败的 LangGraph 测试：

    ```bash
    uv run pytest tests/test_escalation_flow.py
    ```

如果该测试使用 LangSmith 跟踪运行您的应用，则这些应用跟踪由 shell 进程创建并发送到 `customer-support-agent`。Deep Agents Code 自己的推理和工具使用跟踪发送到 `deepagents-code`。

您还可以使用 [`DEEPAGENTS_CODE_` 前缀](/oss/deepagents/code/configuration#deepagents_code_-prefix) 将 LangSmith 凭据限定为 Deep Agents Code（例如 `DEEPAGENTS_CODE_LANGSMITH_API_KEY`）。
</Accordion>

<Accordion title="双写跟踪到第二个项目">
要代理跟踪到第二个 LangSmith 项目，请设置 `DEEPAGENTS_CODE_LANGSMITH_REPLICA_PROJECTS`。这对于将相同的跟踪同时发送到个人项目和共享团队项目很有用。

</fragment>

    ```bash title="~/.deepagents/.env"
    DEEPAGENTS_CODE_LANGSMITH_REPLICA_PROJECTS=team-shared
    ```

设置后且追踪处于活动状态时，每次智能体运行都会被记录到主项目（`DEEPAGENTS_CODE_LANGSMITH_PROJECT`，默认情况下为 `deepagents-code`）以及您在此命名的项目。如果不想写入多个项目，请勿设置该变量，按往常一样写入单个项目。
</Accordion>

配置完成后，Deep Agents Code 会显示一条状态栏，其中包含指向 LangSmith 项目的链接。在支持的终端中，点击链接即可直接打开。您也可以使用 `/trace` 打印 URL 并在浏览器中打开。

```sh
✓ LangSmith tracing: 'my-project'
```

<Tip>
我们建议您同时设置 [LangSmith Engine](/langsmith/engine)，它可以监控您的跟踪、检测问题并提出修复建议。
</Tip>
