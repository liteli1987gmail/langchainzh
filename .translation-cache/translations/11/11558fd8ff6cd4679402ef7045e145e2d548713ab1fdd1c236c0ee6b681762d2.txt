>[Parallel](https://platform.parallel.ai/) 是一个专为 LLM 和 AI 应用构建的实时网络搜索和内容提取平台。

`ParallelFindAllTool` 调用 Parallel 的 [FindAll API](https://docs.parallel.ai/findall-api/findall-quickstart) 进行实体发现。给定一个自然语言目标和布尔匹配条件列表，它返回满足所有条件的排名候选结果。

### 凭证

前往 [Parallel](https://platform.parallel.ai) 注册并生成 API 密钥。在您的环境中设置 `PARALLEL_API_KEY`：

```python
import getpass
import os

if not os.environ.get("PARALLEL_API_KEY"):
    os.environ["PARALLEL_API_KEY"] = getpass.getpass("Parallel API key:\n")
```

## 实例化

`generator` 是一个工具级设置。使用 `"preview"`（免费，上限为 10 个候选项）用于快速迭代；切换到 `"base"`（默认值）、`"core"` 或 `"pro"` 以获得更高质量的运行。

```python
from langchain_parallel import ParallelFindAllTool

tool = ParallelFindAllTool(generator="preview")
```

## 调用

### 快速发现 (preview 生成器)

`preview` 生成器在数秒内返回，最多 10 个候选项。`match_limit` 是必需的；对于 `preview`，它必须在 `[5, 10]` 范围内。

```python
from langchain_parallel import FindAllMatchCondition

result = await tool.ainvoke({
    "objective": "Pure-play public LLM API providers",
    "entity_type": "company",
    "match_conditions": [
        FindAllMatchCondition(
            name="public_us",
            description="Company is publicly traded on a US exchange",
        ),
        FindAllMatchCondition(
            name="llm_api_revenue",
            description="Primary revenue is selling LLM inference via API",
        ),
    ],
    "match_limit": 5,
})

for c in result["candidates"]:
    print(c["name"], "—", c["url"])
```

```text
Anthropic — https://www.anthropic.com
OpenAI — https://www.openai.com
...
```

### 更高质量的运行

切换到 `"base"`、`"core"` 或 `"pro"` 以将 `match_limit` 上限提升至 1000。这些模式需要几分钟；工具会持续轮询直到运行达到终止状态（`"completed"`、`"cancelled"` 或 `"failed"`）。

```python
deep = ParallelFindAllTool(generator="core")

result = await deep.ainvoke({
    "objective": "Independent solar-installer companies based in the EU",
    "entity_type": "company",
    "match_conditions": [
        FindAllMatchCondition(
            name="eu_hq",
            description="Headquartered in an EU country",
        ),
        FindAllMatchCondition(
            name="residential_solar_pv",
            description="Primarily installs residential solar PV",
        ),
    ],
    "match_limit": 50,
})
```

### 排除已查看的候选人

传入 `exclude_list=[FindAllExcludeEntry(name=..., url=...)]` 以排除已处理的候选人。`name` 和 `url` 都是必需的。

```python
from langchain_parallel import FindAllExcludeEntry

result = await tool.ainvoke({
    "objective": "Pure-play public LLM API providers",
    "entity_type": "company",
    "match_conditions": [
        FindAllMatchCondition(
            name="llm_api_revenue",
            description="Primary revenue is selling LLM inference via API",
        ),
    ],
    "match_limit": 5,
    "exclude_list": [
        FindAllExcludeEntry(name="OpenAI", url="https://www.openai.com"),
        FindAllExcludeEntry(name="Anthropic", url="https://www.anthropic.com"),
    ],
})
```

### 取消

`cancel()` 通过 id 中止正在运行的任务。id 会返回给启动该任务的调用方；如果你在长时间运行的任务中使用 `tool.ainvoke(...)` 启动了它，请在 await 完成之前从任务中捕获 `findall_id`。

```python
# from another task / handler:
tool.cancel(findall_id)        # sync
await tool.acancel(findall_id) # async
```

### 参数

#### 必填参数

- `objective`：用自然语言描述要查找的内容。
- `entity_type`：描述候选类别的简短名词（如 `"company"`、`"researcher"`、`"product"` 等）。
- `match_conditions`：`FindAllMatchCondition(name=..., description=...)` 列表。每个条件对象的两个字段均为必填。
- `match_limit`：整数，范围 `[5, 1000]`。`preview` 生成器进一步限制为最多 10 个。

#### 可选参数

- `exclude_list`：`FindAllExcludeEntry(name=..., url=...)` 列表，用于排除的候选项。
- `webhook`：`FindAllWebhook(url=..., event_types=[...])`，用于接收运行/候选事件。
- `metadata`：自由格式的元数据，会在运行中持久化保存。
- `timeout`：轮询超时时间（秒），默认 600。

## 链接

将该工具绑定到任何支持工具调用的聊天模型，并使用 [`create_agent`](/oss/langchain/agents) 驱动智能体：

```python
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model

llm = init_chat_model(model="claude-haiku-4-5", model_provider="anthropic")
agent = create_agent(model=llm, tools=[tool])

agent.invoke({"messages": [("human", "Find me a few independent EU solar installers.")]})
```

```python
            "match_status": "matched",  # or "generated" / "unmatched"
            "output": {
                "<condition_name>": {
                    "type": "match_condition",
                    "value": True,
                    "is_matched": True,
                },
            },
            "basis": [...],  # citations + reasoning per output field
        },
    ],
    "run": {...},          # status info
    "last_event_id": "...",
}
```
