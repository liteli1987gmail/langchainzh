<Info>
**本指南适用于自托管 LangSmith 客户** 已 [启用 LangSmith 部署](/langsmith/deploy-self-hosted-full-platform#enable-langsmith-deployment) 功能。对于云端客户，请参阅 [在云端部署](/langsmith/deploy-to-cloud)。对于没有控制平面的独立 Agent 服务器，请参阅 [自托管独立服务器](/langsmith/deploy-standalone-server).
</Info>

本指南介绍如何将您的应用程序部署到 [self-hosted](/langsmith/self-hosted) LangSmith 实例，使用 [控制平面](/langsmith/control-plane)。使用控制平面，您可以在本地构建 Docker 镜像，将其推送到 Kubernetes 集群可访问的注册表，然后使用 [LangSmith UI](https://smith.langchain.com).

## 拓扑

在现有的自托管 LangSmith 实例上启用 LangSmith 部署会添加一个控制平面、一个数据平面监听器和一个在集群中配置 Agent Server 的运算符。基础 LangSmith 平台继续处理可观测性、评估和提示；已部署的 Agent Server 会将跟踪数据发送回平台。

有关启用 LangSmith 部署添加的组件的详细信息，请参阅 [启用 LangSmith 部署](/langsmith/deploy-self-hosted-full-platform#enable-langsmith-deployment).

## 概览

部署到带有控制平面的自托管 LangSmith 实例的应用程序使用 Docker 镜像。在本指南中，应用程序部署工作流程如下：

1. 使用 `langgraph dev` or [工作室](/langsmith/studio).
1. 使用以下工具构建 Docker 镜像 `langgraph build` command.
1. 将镜像推送到您基础设施可访问的容器镜像仓库。
1. 从以下位置部署 [控制平面 UI](/langsmith/control-plane#control-plane-ui) 通过指定镜像 URL。

## 前提条件

在完成本指南之前，您需要满足以下条件：

- [已启用 LangSmith Deployment](/langsmith/deploy-self-hosted-full-platform#enable-langsmith-deployment) 在您的自托管 LangSmith 实例上。
- 访问 [LangSmith UI](https://smith.langchain.com) 并已启用 LangSmith Deployment。
- 一个您 Kubernetes 集群可访问的容器镜像仓库。如果使用需要身份验证的私有镜像仓库，您必须将镜像拉取密钥配置为基础设施设置的一部分。请参阅 [私有镜像仓库身份验证](#private-registry-authentication).

## 步骤 1. 本地测试

在部署之前，在本地测试您的应用程序。您可以使用 [LangGraph CLI](/langsmith/cli#dev) 以开发模式运行 Agent 服务器：

```bash
langgraph dev
```

有关本地测试的完整指南，请参阅 [本地服务器快速入门](/langsmith/local-dev-testing).

## 步骤 2. 构建 Docker 镜像

使用以下工具构建应用程序的 Docker 镜像 [`langgraph build`](/langsmith/cli#build) command:

```bash
langgraph build -t my-image
```

构建命令选项包括：

| 选项 | 默认值 | 描述 |
|--------|---------|-------------|
| `-t, --tag TEXT` | 必填 | Docker 镜像的标签 |
| `--platform TEXT` | | 目标构建平台（例如， `linux/amd64,linux/arm64`) |
| `--pull / --no-pull` | `--pull` | 使用最新的远程 Docker 镜像构建 |
| `-c, --config FILE` | `langgraph.json` | 配置文件路径 |

指定平台的示例：

```bash
langgraph build --platform linux/amd64 -t my-image:v1.0.0
```

详细信息，请参阅 [CLI 参考文档](/langsmith/cli#build).

## 步骤 3. 推送到容器注册表

将镜像推送到您的 Kubernetes 集群可访问的容器注册表。具体命令取决于您的注册表提供商。

<Tip>
使用版本信息标记您的镜像（例如， `my-registry.com/my-app:v1.0.0`）以便于回滚。
</Tip>

## 步骤 4. 使用控制平面 UI 部署

 [控制平面 UI](/langsmith/control-plane#control-plane-ui) 允许您创建和管理部署、查看日志和指标以及更新配置。要在 [LangSmith UI](https://smith.langchain.com):

1. 中创建新部署，请在左侧导航面板中选择 **部署**.
1. 在右上角，选择 **+ 新建部署**.
1. 在部署配置面板中，提供：
   - **镜像 URL**：您在 [步骤 3](#step-3-push-to-container-registry).
   - **Listener/Compute ID**中推送的完整镜像 URL：选择为您的基础设施配置的监听器。
   - **命名空间**：要部署到的 Kubernetes 命名空间。
   - **环境变量**：任何必需的配置（API 密钥等）。
   - 根据需要设置其他部署选项。
1. 选择 **提交**.

控制平面将协调您的 [数据平面](/langsmith/data-plane) 监听器来部署您的应用程序。

创建部署后，基础设施将 [异步配置](/langsmith/control-plane#asynchronous-deployment)。部署可能需要长达几分钟时间，初始部署由于需要创建数据库会花费更长时间。

From the control plane UI, you can view build logs, server logs, and deployment metrics including CPU/memory usage, replicas, and API performance. For more details, refer to the [控制平面监控文档](/langsmith/control-plane#monitoring).

<Note>
A [LangSmith 可观测性追踪项目](/langsmith/observability) 会自动为每个部署创建，其名称与部署名称相同。追踪环境变量由控制平面自动设置。
</Note>

## 更新部署

要部署新版本的应用程序，请创建 [新修订版本](/langsmith/control-plane#revisions):

从 LangSmith UI 开始：

1. 在左侧导航面板中，选择 **部署**.
1. 选择一个现有部署。
1. 在部署视图中，选择 **+ 新建修订版本** 位于右上角。
1. 更新配置：
   - 将 **镜像 URL** 更新为新镜像版本。
   - 如需要，请更新环境变量。
   - 根据需要调整其他设置。
1. 选择 **提交**.

## 私有镜像仓库认证

如果您的容器镜像仓库需要认证（例如 AWS ECR、Azure ACR、GCP Artifact Registry、私有 Docker 镜像仓库），则必须在部署应用程序之前配置 Kubernetes 镜像拉取密钥。这是一次性的基础设施配置。

<Note>
**此配置在基础设施层面完成，而非按部署进行。** 配置完成后，所有部署将自动继承镜像仓库凭证。
</Note>

在您的 LangSmith Helm chart 的 `imagePullSecrets` 中配置 `values.yaml` 文件中进行配置。详见 [《启用 LangSmith 部署指南》](/langsmith/deploy-self-hosted-full-platform#enable-langsmith-deployment).

如需获取创建镜像拉取密钥的详细步骤，请参阅 [Kubernetes 从私有镜像仓库拉取镜像的相关文档](https://kubernetes.io/docs/tasks/configure-pod-container/pull-image-private-registry/).
