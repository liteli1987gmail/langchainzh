---
title: 基础身份验证（邮箱和密码）
sidebarTitle: 设置基础身份验证
---

LangSmith 支持通过用户名/密码登录，但有以下限制：

* 您不能将现有安装从基础身份验证模式更改为 OAuth 与 PKCE（已弃用）或反之亦然 - 安装必须是其中一种。**基础身份验证安装需要完全全新的安装，包括单独的 PostgreSQL 数据库/架构，除非从现有的 `None` 类型安装迁移（见下文）。**
* 用户在被邀请时必须获得其初始自动生成的密码。此密码可由任何组织管理员稍后更改。
* 您不能同时使用基础身份验证和带客户端密钥的 OAuth。

## 要求和功能

* 初始安装期间会配置一个单一的 `Default` 组织，不支持创建其他组织
* 您设置的初始密码（见下方配置）必须至少 12 个字符，且必须包含至少一个小写字母、一个大写字母和一个符号
* JWT 签名密钥没有严格要求，但我们建议安全生成至少 32 个字符的字符串。例如：`openssl rand -base64 32`

### 从无身份验证迁移

**仅在 0.7 及以上版本中支持。**

从 [None](/langsmith/authentication-methods#none) 身份验证模式迁移安装，会将单一的"default"用户替换为具有配置凭证的用户，并保留所有现有资源。迁移后单一的预先存在的 workspace ID 保持为 `00000000-0000-0000-0000-000000000000`，但迁移后安装的其他方面均为标准基础身份验证安装。

要迁移，只需按下方所示更新配置，然后照常运行 `helm upgrade`（或 `docker-compose up`）。

### 配置

<Note>
更改 JWT 密钥将使用户登出
</Note>

<CodeGroup>

</CodeGroup>

```yaml Helm
config:
  authType: mixed
  basicAuth:
    enabled: true
    initialOrgAdminEmail: <YOUR EMAIL ADDRESS>
    initialOrgAdminPassword: <PASSWORD> # Must be at least 12 characters long and have at least one lowercase, uppercase, and symbol
    jwtSecret: <SECRET>
```

```bash Docker
# In your .env file
AUTH_TYPE=mixed
BASIC_AUTH_ENABLED=true
INITIAL_ORG_ADMIN_EMAIL=<YOUR EMAIL ADDRESS>
INITIAL_ORG_ADMIN_PASSWORD=<PASSWORD> # Must be at least 12 characters long and have at least one lowercase, uppercase, and symbol
BASIC_AUTH_JWT_SECRET=<SECRET>
```

</CodeGroup>

此外，在 docker-compose 中，你需要运行 bootstrap 命令来创建初始组织和用户：

```bash
docker-compose exec langchain-backend python hooks/auth_bootstrap.pyc
```

配置完成后，您将看到如下登录界面。您应该能够使用 `initialOrgAdminEmail` 和 `initialOrgAdminPassword` 的值进行登录，您的用户将被自动配置为 `Organization Admin` 角色。有关组织角色的更多详细信息，请参阅[管理员指南](/langsmith/administration-overview#organization-roles)。
