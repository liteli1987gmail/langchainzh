<Note>
支持删除工作区 **LangSmith 自托管 v0.10 原生支持**。查看 [删除工作区的说明](/langsmith/set-up-hierarchy#delete-a-workspace).

请按照以下指南操作 v0.10 之前的自托管版本。
</Note>

The LangSmith UI does not currently support the deletion of an individual workspace from an organization. This, however, can be accomplished by directly removing all traces from all materialized views in ClickHouse (except the runs\_历史视图）以及运行和反馈表，然后从 Postgres 租户表中删除工作区。

此命令使用工作区 ID 作为参数。

### 前提条件

Ensure you have the following tools/items ready.

1. kubectl

   * [https://kubernetes.io/docs/tasks/tools/](https://kubernetes.io/docs/tasks/tools/)

2. PostgreSQL 客户端

   * [https://www.postgresql.org/download/](https://www.postgresql.org/download/)

3. PostgreSQL 数据库连接：

   * 主机
   * 端口
   * 用户名
     * 如果使用捆绑版本，则为 `postgres`
   * 密码
     * 如果使用捆绑版本，则为 `postgres`
   * 数据库名称
     * 如果使用捆绑版本，则为 `postgres`

4. Clickhouse 数据库凭据

   * 主机
   * 端口
   * 用户名
     * 如果使用捆绑版本，则为 `default`
   * 密码
     * 如果使用捆绑版本，则为 `password`
   * 数据库名称
     * 如果使用捆绑版本，则为 `default`

5. 确保从运行迁移脚本的计算机能够连接到 PostgreSQL 数据库。

   * 如果使用捆绑版本，可能需要将 postgresql 服务端口转发到本地计算机。
   * 运行 `kubectl port-forward svc/langsmith-postgres 5432:5432` 将 postgresql 服务端口转发到本地计算机。

6. 确保从运行迁移脚本的计算机能够连接到 Clickhouse 数据库。

   * 如果使用捆绑版本，可能需要将 clickhouse 服务端口转发到本地计算机。
     * 运行 `kubectl port-forward svc/langsmith-clickhouse 8123:8123` 将 clickhouse 服务端口转发到本地计算机。
   * 如果使用 Clickhouse Cloud，需要指定 --ssl 标志并使用端口 `8443`

7. 删除工作区的脚本

   * 下载 [工作区脚本](https://github.com/langchain-ai/helm/blob/main/charts/langsmith/scripts/delete_workspace.sh)

### 运行删除单个工作区的脚本

运行以下命令执行工作区删除脚本：

```bash
sh delete_workspace.sh <postgres_url> <clickhouse_url> --workspace_id <workspace_id>
```

例如，如果使用带端口转发的捆绑版本，命令如下：

```bash
sh delete_workspace.sh "postgres://postgres:postgres@localhost:5432/postgres" "clickhouse://default:password@localhost:8123/default" --workspace_id 4ec70ec7-0808-416a-b836-7100aeec934b
```

如果访问 LangSmith UI，现在应该看到工作区已被删除。
