在排查 LangSmith 自托管实例的问题时，LangChain 团队可能会要求您生成 LangSmith 查询统计数据，帮助我们了解驱动 LangSmith 产品体验的各种查询的性能。

此命令将生成一个可以与 LangChain 团队共享的 CSV 文件。

### 前置要求

确保您已准备好以下工具/项目。

1. kubectl

   * [https://kubernetes.io/docs/tasks/tools/](https://kubernetes.io/docs/tasks/tools/)

2. Clickhouse 数据库凭证

   * 主机
   * 端口
   * 用户名
     * 如果使用捆绑版本，则为 `default`
   * 密码
     * 如果使用捆绑版本，则为 `password`
   * 数据库名称
     * 如果使用捆绑版本，则为 `default`

3. 确保能够从运行 `get_query_stats` 脚本的机器连接到 Clickhouse 数据库。

   * 如果使用捆绑版本，可能需要将 Clickhouse 服务端口转发到本地机器。
   * 运行 `kubectl port-forward svc/langsmith-clickhouse 8123:8123` 将 Clickhouse 服务端口转发到本地机器。

4. 用于生成查询统计的脚本

   * 下载[查询统计脚本](https://github.com/langchain-ai/helm/blob/main/charts/langsmith/scripts/get_query_stats.sh)

### 运行查询统计生成脚本

运行以下命令以运行统计生成脚本：

```bash
sh get_query_stats.sh <clickhouse_url> --output path/to/file.csv
```

例如，如果您正在使用带有端口转发的捆绑版本，命令将如下所示：

```bash
sh get_query_stats.sh "clickhouse://default:password@localhost:8123/default" --output query_stats.csv
```

然后运行此命令后，你应该会看到文件 query_stats.csv 已被创建，其中包含 LangSmith 查询统计信息。
