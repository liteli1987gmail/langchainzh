LangSmith Deployment 是一个专为代理工作负载构建的工作流编排运行时。它提供代理在生产环境中可靠运行所需的管理基础设施，支持从本地开发到部署的完整生命周期。

## 可部署的产品

LangSmith Deployment 与框架无关，这意味着您可以部署使用以下框架构建的代理：

## 部署环境

您可以在多种托管模式中运行相同的 [Agent Server](/langsmith/agent-server) 运行时。**独立服务器**是最轻量的选项：您自行运行容器，无需 LangSmith [控制平面](/langsmith/control-plane)。对于通过 UI 和 API 进行托管部署，请使用 **Cloud** 或 **Self-hosted**（在您的基础设施中运行完整平台）。

<CardGroup cols={2}>

<Card
  title="Cloud"
  cta="查看指南"
  href="/langsmith/deploy-to-cloud"
  icon="cloud"
>
  由 LangChain 完全托管，运行在 AWS 和 GCP 上。在 LangSmith UI 中或使用 [`langgraph deploy`](/langsmith/cli#deploy) 从 GitHub 创建部署。需要 [Plus 计划或更高版本](https://www.langchain.com/pricing)。
</Card>

<Card
  title="Standalone server"
  cta="查看指南"
  href="/langsmith/deploy-standalone-server"
  icon="server"
>
  使用 Docker、Compose 或 Kubernetes 部署 Agent Server。自带 PostgreSQL、Redis 和 LangSmith 许可证；无需控制平面。可选 [LangSmith 追踪](/langsmith/observability) 到 Cloud 或自托管实例。
</Card>

<Card
  title="Self-hosted"
  cta="查看指南"
  href="/langsmith/self-hosted"
  icon="buildings"
>
  在您的云中运行完整的 LangSmith 平台，包括控制平面和数据平面（例如在 Kubernetes 上）。需要 [Enterprise 计划](https://www.langchain.com/pricing)。将可观测性、评估和代理部署集成在一个私有堆栈中。
</Card>

</CardGroup>

相同的运行时，相同的 API。变化的是谁管理基础设施。
有关功能级别的比较和基础设施设置，请参阅 [平台设置](/langsmith/platform-setup)。

## 基于 Agent Server 运行时构建

部署后，代理与 [Agent Server](/langsmith/assistants) 的执行模型配合使用：**assistants** 用于配置，**threads** 用于状态，**runs** 用于工作负载。有关功能特性、教程、服务器自定义和操作，请参阅 [开发代理](/langsmith/develop-agents-overview)。
