LangSmith 部署是一个专为智能体工作负载构建的工作流编排运行时。它提供智能体在生产环境中大规模可靠运行所需的托管基础设施，支持从本地开发到部署的完整生命周期。

## 可部署产品

LangSmith 部署与框架无关，这意味着您可以部署使用以下框架构建的智能体：

## 部署环境

您可以在多种托管模式中运行相同的 [智能体服务器](/langsmith/agent-server) 运行时。 **独立服务器** 是最轻量的选项：您自己运行容器，无需 LangSmith [控制平面](/langsmith/control-plane)。如需通过 UI 和 API 进行托管部署，请使用 **云** or **Self-hosted** （在您的基础设施中提供完整平台）。

<CardGroup cols={2}>

<Card
  title="云"
  cta="查看指南"
  href="/langsmith/deploy-to-cloud"
  icon="cloud"
>
  由 LangChain 完全管理，运行在 AWS 和 GCP 上。在 LangSmith UI 中或使用 [`langgraph deploy`](/langsmith/cli#deploy)从 GitHub 创建部署。需要 [Plus 计划或更高级别](https://www.langchain.com/pricing).
</Card>

<Card
  title="独立服务器"
  cta="查看指南"
  href="/langsmith/deploy-standalone-server"
  icon="server"
>
  使用 Docker、Compose 或 Kubernetes 部署智能体服务器。使用您自己的 PostgreSQL、Redis 和 LangSmith 许可证；无控制平面。可选 [LangSmith 追踪](/langsmith/observability) 到云端或自托管实例。
</Card>

<Card
  title="自托管"
  cta="查看指南"
  href="/langsmith/self-hosted"
  icon="buildings"
>
  在您的云中运行完整的 LangSmith 平台，包括控制平面和数据平面（例如在 Kubernetes 上）。需要 [企业版计划](https://www.langchain.com/pricing)。在一个私有堆栈中集成可观测性、评估和代理部署。
</Card>

## 基于 Agent Server 运行时构建

部署后，代理使用 [Agent Server](/langsmith/assistants)的执行模型： **助手** 用于配置， **线程** 用于状态，以及 **运行** 用于工作负载。关于功能、教程、服务器定制和运维，请参阅 [开发代理](/langsmith/develop-agents-overview).
