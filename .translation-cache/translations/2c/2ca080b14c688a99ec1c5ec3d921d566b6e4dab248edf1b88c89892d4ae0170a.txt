LangChain由几个核心信念驱动：

- 大语言模型（LLMs）是一项伟大而强大的新技术。
- 当你将大语言模型与外部数据源结合使用时，效果会更好。
- 大语言模型将改变未来应用的面貌。具体来说，未来的应用将变得越来越具有代理性。
- 这场变革仍处于非常早期的阶段。
- 虽然构建这些代理应用的原型很容易，但构建足够可靠以投入生产环境的代理仍然非常困难。

今天，开发者可以选择如何构建代理：使用 [LangChain](/oss/langchain/overview) 以获得最大的灵活性和控制力，或者 [Deep Agents](/oss/langchain/overview) 它提供类似的灵活性和控制力，但内置了固执己见的规划、文件系统工具、子代理和上下文管理功能。两者都建立在 [LangGraph](/oss/langgraph/overview).

使用LangChain，我们有两个核心关注点：

<Steps>
    <Step title="We want to enable developers to build with the best models.">
        不同的提供商公开不同的API，具有不同的模型参数和不同的消息格式。
        标准化这些模型输入和输出是一个核心关注点，使开发者能够轻松切换到最新的最先进的模型，避免被锁定。
    </Step>
    <Step title="We want to make it easy to use models to orchestrate more complex flows that interact with other data and computation.">
        模型应该被用于 *文本生成* 之外更多的用途——它们还应该用于编排与外部数据交互的更复杂流程。LangChain使定义 [工具](/oss/langchain/tools) 变得容易，大语言模型可以动态使用这些工具，同时帮助解析和访问非结构化数据。
    </Step>
</Steps>

## 历史

鉴于该领域不断变化的步伐，LangChain也在不断演进。以下是LangChain多年来的变化时间线，与使用大语言模型构建的含义同步发展：

<Update label="2022-10-24" description="v0.0.1">
    在ChatGPT发布前一个月， **LangChain作为Python包发布**。它由两个主要组件组成：

    - LLM抽象层
    - “链”，或为常见用例运行的可预测计算步骤。例如 - RAG：运行检索步骤，然后运行生成步骤。

    LangChain这个名字来自"Language"（如语言模型）和"Chains"（链）。
</Update>

<Update label="2022-12">
    第一个通用代理被添加到LangChain中。

    这些通用代理基于 [ReAct论文](https://arxiv.org/abs/2210.03629) （ReAct代表推理和行动）。它们使用大语言模型生成代表工具调用的JSON，然后解析该JSON以确定要调用哪些工具。
</Update>

<Update label="2023-01">
    OpenAI发布了“聊天补全”API。

    在此之前，模型接受字符串输入并返回字符串。在ChatCompletions API中，它们演进为接受消息列表并返回消息。其他模型提供商也纷纷效仿，LangChain更新为处理消息列表。
</Update>

<Update label="2023-01">
    LangChain发布了JavaScript版本。

    大语言模型和代理将改变应用的构建方式，而JavaScript是应用开发者的语言。
</Update>

<Update label="2023-02">
    **LangChain Inc. 作为一家公司成立** 围绕开源 LangChain 项目。

    主要目标是"让智能代理无处不在"。团队认识到，虽然 LangChain 是一个关键部分（LangChain 使开始使用 LLM 变得简单），但也需要其他组件。
</Update>

<Update label="2023-03">
    OpenAI 在其 API 中发布了"函数调用"功能。

    这允许 API 明确生成代表工具调用的 payload。其他模型提供商也纷纷效仿，LangChain 被更新为使用此方法作为工具调用的首选方式（而不是解析 JSON）。
</Update>

<Update label="2023-06">
    **LangSmith 发布** 作为 LangChain Inc. 的闭源平台，提供可观察性和评估。

    构建代理的主要问题是让它们变得可靠，而 LangSmith 提供了可观察性和评估，正是为了解决这一需求而构建的。LangChain 被更新为与 LangSmith 无缝集成。
</Update>

<Update label="2024-01" description="v0.1.0">
    **LangChain 发布 0.1.0 版本**，即首个非 0.0.x 版本。

    整个行业从原型开发走向生产环境，因此 LangChain 加强了对稳定性的重视。
</Update>

<Update label="2024-02">
    **LangGraph 发布** 作为开源库。

    原始 LangChain 有两个重点：LLM 抽象，以及用于快速上手常见应用的高级接口；然而，它缺少一个允许开发者控制代理精确流程的低级编排层。这就是 LangGraph 的由来。

    在构建 LangGraph 时，我们从构建 LangChain 的经验中吸取教训，并添加了我们发现需要的功能：流式输出、持久化执行、短期记忆、人机交互等。
</Update>

    **LangChain 拥有超过 700 个集成。**

<Update label="2024-10">
    LangGraph 成为构建任何超出单个 LLM 调用范围的 AI 应用的首选方式。

    当开发者尝试提高应用的可靠性时，他们需要比高级接口提供的更多的控制。LangGraph 提供了这种低级灵活性。LangChain 中的大多数链和代理都被标记为已弃用，并提供了迁移到 LangGraph 的指南。LangGraph 中仍然有一个高级抽象：代理抽象。它构建在低级 LangGraph 之上，具有与 LangChain 中 ReAct 代理相同的接口。
</Update>

<Update label="2025-10-20" description="v1.0.0">
    **LangChain 发布 1.0 版本** 包含两个重大变化：

    1. 彻底重构了所有链和代理 `langchain`。所有链和代理现在都被一个高级抽象取代：构建在 LangGraph 之上的代理抽象。这是一个最初在 LangGraph 中创建的高级抽象，只是移到了 LangChain 中。

        :::python

    2. 标准消息内容格式：模型 API 从返回包含简单内容字符串的消息演变为更复杂的输出类型——推理块、引用、服务端工具调用等。LangChain 演化其消息格式以跨提供商标准化这些内容。
</Update>

<Update label="2026-03-15" description="v0.5.3">
    **Deep Agents 发布** 作为一个基于 LangGraph 构建的开源智能体工具集。

    虽然 LangChain 为自定义智能体架构提供了灵活的构建模块， [Deep Agents](/oss/langchain/overview) 为研究、编码等复杂的长时间运行任务提供了包含完整功能的解决方案。它添加了内置的规划工具、具有可插拔后端（内存、磁盘、LangGraph 存储、沙箱）的虚拟文件系统，以及用于上下文隔离的子智能体生成功能。使用 Deep Agents 可以获得具有预定义工具的更自主的智能体；使用 LangChain 可以完全控制您的智能体架构。
</Update>
