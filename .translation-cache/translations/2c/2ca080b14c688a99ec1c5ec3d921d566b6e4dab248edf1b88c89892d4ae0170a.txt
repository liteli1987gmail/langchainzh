LangChain 由几个核心信念驱动：

- 大语言模型（LLM）是一项伟大而强大的新技术。
- 当 LLM 与外部数据源结合时，它们会变得更加强大。
- LLM 将改变未来应用的面貌。具体来说，未来的应用将越来越具备自主代理特性。
- 在这场变革中，我们仍然处于非常早期的阶段。
- 虽然为这些代理应用构建原型很容易，但要构建足够可靠以投入生产的代理仍然非常困难。

如今，开发者可以选择构建代理的方式：使用 [LangChain](/oss/langchain/overview) 以获得最大的灵活性和控制权，或使用 [Deep Agents](/oss/langchain/overview)，它提供类似的灵活性和控制权，但内置了固执己见的规划、文件系统工具、子代理和上下文管理。两者都建立在 [LangGraph](/oss/langgraph/overview) 之上。

使用 LangChain，我们有两个核心重点：

<Steps>
    <Step title="我们希望让开发者能够使用最佳模型进行构建。">
        不同的提供商公开不同的 API，具有不同的模型参数和不同的消息格式。
        标准化这些模型输入和输出是一个核心重点，使开发者能够轻松切换到最新的前沿模型，避免被锁定。
    </Step>
    <Step title="我们希望让使用模型编排与数据和计算交互的更复杂流程变得容易。">
        模型不仅应用于*文本生成*，还应用于编排与数据交互的更复杂流程。LangChain 使定义 [工具](/oss/langchain/tools) 变得容易，LLM 可以动态使用这些工具，同时帮助解析和访问非结构化数据。
    </Step>
</Steps>

## 历史

鉴于该领域不断变化的步伐，LangChain 也随着时间推移而发展。以下是 LangChain 多年来如何变化的简要时间线，与基于 LLM 构建的含义同步演进：

<Update label="2022-10-24" description="v0.0.1">
    在 ChatGPT 发布前一个月，**LangChain 作为 Python 包发布**。它由两个主要组件组成：

    - LLM 抽象层
    - "链"，或用于常见用例的预定义计算步骤。例如 - RAG：运行检索步骤，然后运行生成步骤。

    LangChain 这个名字来自"语言"（如语言模型）和"链"。
</Update>

<Update label="2022-12">
    第一个通用代理被添加到 LangChain。

    这些通用代理基于 [ReAct 论文](https://arxiv.org/abs/2210.03629)（ReAct 代表推理和行动）。它们使用 LLM 生成表示工具调用的 JSON，然后解析该 JSON 以确定要调用哪些工具。
</Update>

<Update label="2023-01">
    OpenAI 发布了"聊天补全"API。

    之前，模型接收字符串并返回字符串。在 ChatCompletions API 中，它们演变为接收消息列表并返回消息。其他模型提供商纷纷效仿，LangChain 也更新为使用消息列表。
</Update>

<Update label="2023-01">
    LangChain 发布 JavaScript 版本。

    LLM 和代理将改变应用程序的构建方式，而 JavaScript 是应用程序开发者的语言。
</Update>

<Update label="2023-02">
    **LangChain Inc. 作为公司成立**，围绕开源 LangChain 项目。

    主要目标是"让智能代理无处不在"。团队认识到，虽然 LangChain 是一个关键部分（LangChain 使开始使用 LLM 变得简单），但也需要其他组件。
</Update>

<Update label="2023-03">
    OpenAI 在其 API 中发布了"函数调用"。

    这允许 API 显式生成表示工具调用的有效载荷。其他模型提供商纷纷效仿，LangChain 被更新为使用函数调用作为工具调用的首选方法（而不是解析 JSON）。
</Update>

<Update label="2023-06">
    **LangSmith 由 LangChain Inc. 发布为闭源平台**，提供可观测性和评估。

    构建代理的主要问题是让它们变得可靠，而 LangSmith 提供可观测性和评估，正是为了解决这一需求。LangChain 被更新为与 LangSmith 无缝集成。
</Update>

<Update label="2024-01" description="v0.1.0">
    **LangChain 发布 0.1.0**，这是其首个非 0.0.x 版本。

    行业从原型成熟到生产，因此 LangChain 加强了对稳定性的关注。
</Update>

<Update label="2024-02">
    **LangGraph 作为开源库发布**。

    最初的 LangChain 有两个重点：LLM 抽象层，以及用于快速入门常见应用程序的高级接口；但是，它缺少一个允许开发者控制代理精确流程的低级编排层。Enter: LangGraph。

    在构建 LangGraph 时，我们从构建 LangChain 的经验中吸取教训，并添加了我们发现需要的功能：流式输出、持久执行、短期记忆、人机交互等。
</Update>

    :::js

<Update label="2024-10">
    LangGraph 成为构建任何超过单次 LLM 调用的 AI 应用的首选方式。

    随着开发者尝试提高应用程序的可靠性，他们需要比高级接口提供的更多的控制。LangGraph 提供了这种低级灵活性。LangChain 中的大多数链和代理被标记为弃用，并提供了迁移到 LangGraph 的指南。LangGraph 中仍然创建了一个高级抽象：代理抽象。它构建在低级 LangGraph 之上，具有与 LangChain 中的 ReAct 代理相同的接口。
</Update>

<Update label="2025-10-20" description="v1.0.0">
    **LangChain 发布 1.0**，包含两个重大变化：

    1. 全面改造 `langchain` 中的所有链和代理。所有链和代理现在都被替换为一个高级抽象：构建在 LangGraph 之上的代理抽象。这是最初在 LangGraph 中创建的高级抽象，只是移到了 LangChain。

        :::js
        对于仍在使用旧 LangChain 链/代理且不想升级的用户（注意：我们建议您升级），您可以通过安装 `@langchain/classic` 包继续使用旧 LangChain。

    2. 标准消息内容格式：模型 API 从返回带有简单内容字符串的消息演变为更复杂的输出类型——推理块、引用、服务端工具调用等。LangChain 演进其消息格式以跨提供商标准化这些内容。
</Update>

<Update label="2026-03-15" description="v0.5.3">
    **Deep Agents 作为构建在 LangGraph 之上的开源代理工具包发布**。

    虽然 LangChain 为自定义代理架构提供了灵活的构建块，[Deep Agents](/oss/langchain/overview) 为复杂、长运行任务（如研究和编码）提供了开箱即用的选项。它添加了内置规划工具、具有可插拔后端（内存、磁盘、LangGraph 存储、沙箱）的虚拟文件系统，以及用于上下文隔离的子代理生成。使用 Deep Agents 获得具有预定义工具的更自主代理；使用 LangChain 完全控制您的代理架构。
</Update>
