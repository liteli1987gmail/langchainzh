---
title: 导入导出的数据
description: 将 LangSmith 批量导出的 Parquet 数据导入到 BigQuery、Snowflake、Redshift、Clickhouse 或 DuckDB。
---

从 S3 和 Parquet 格式导入数据被大多数分析系统广泛支持。请参阅以下文档链接：

## BigQuery

要将数据导入 BigQuery，请参阅[从 Parquet 加载数据](https://cloud.google.com/bigquery/docs/loading-data-cloud-storage-parquet)以及
[Hive 分区加载](https://cloud.google.com/bigquery/docs/hive-partitioned-loads-gcs)。

## Snowflake

您可以按照[从云文档加载](https://docs.snowflake.com/en/user-guide/tutorials/load-from-cloud-tutorial)中的说明将数据从 S3 加载到 Snowflake。

## Redshift

您可以按照 [AWS COPY 命令文档](https://docs.aws.amazon.com/redshift/latest/dg/r_COPY.html)中的说明，将数据从 S3 或 Parquet COPY 到 Amazon Redshift。

## Clickhouse

您可以直接在 Clickhouse 中查询 S3 / Parquet 格式的数据。例如，如果使用 GCS，可以按如下方式查询数据：

```sql
SELECT count(distinct id) FROM s3('https://storage.googleapis.com/<bucket>/<prefix>/export_id=<export_id>/**',
 'access_key_id', 'access_secret', 'Parquet')
```

有关更多信息，请参阅 [Clickhouse S3 集成文档](https://clickhouse.com/docs/en/engines/table-engines/integrations/s3)。

## DuckDB

您可以使用 DuckDB 通过 SQL 在内存中查询 S3 中的数据。请参阅 [S3 导入文档](https://duckdb.org/docs/guides/network_cloud_storage/s3_import.html)。
