---
title: 导入导出的数据
description: 将 LangSmith 批量导出的 Parquet 数据导入到 BigQuery、Snowflake、Redshift、Clickhouse 或 DuckDB。
---

大多数分析系统通常支持从 S3 和 Parquet 格式导入数据。参见以下文档链接：

## BigQuery

要将数据导入 BigQuery，请参见 [从 Parquet 加载数据](https://cloud.google.com/bigquery/docs/loading-data-cloud-storage-parquet) 以及
[Hive 分区加载](https://cloud.google.com/bigquery/docs/hive-partitioned-loads-gcs).

## Snowflake

您可以按照以下方式从 S3 将数据加载到 Snowflake [从云文档加载](https://docs.snowflake.com/en/user-guide/tutorials/load-from-cloud-tutorial).

## RedShift

您可以按照以下方式将数据从 S3 或 Parquet COPY 到 Amazon Redshift [AWS COPY 命令文档](https://docs.aws.amazon.com/redshift/latest/dg/r_COPY.html).

## Clickhouse

You can directly query data in S3 / Parquet format in Clickhouse. As an example, if using GCS, you can query the data as follows:

```sql
SELECT count(distinct id) FROM s3('https://storage.googleapis.com/<bucket>/<prefix>/export_id=<export_id>/**',
 'access_key_id', 'access_secret', 'Parquet')
```

请参见 [Clickhouse S3 集成文档](https://clickhouse.com/docs/en/engines/table-engines/integrations/s3) 了解更多。

## DuckDB

您可以使用 DuckDB 通过 SQL 从 S3 查询内存中的数据。参见 [S3 导入文档](https://duckdb.org/docs/guides/network_cloud_storage/s3_import.html).
