[Studio](/langsmith/studio) 在 [LangSmith 部署界面](https://smith.langchain.com) 支持连接两种类型的图：

- 部署在 [云端或自托管](#deployed-graphs).
- 使用 [Agent Server](#local-development-server).

## 已部署的图

Studio 可在 [LangSmith 界面](https://smith.langchain.com) 的 **部署** navigation.

对于 [已部署](/langsmith/deployment-quickstart)的应用程序，您可以作为该部署的一部分访问 Studio。为此，请在界面中导航到该部署并选择 **Studio**.

这将加载已连接到您实时部署的 Studio，允许您创建、读取和更新该部署中的 [线程](/oss/langgraph/checkpointers#threads), [助手](/langsmith/assistants)和 [内存](/oss/concepts/memory) 。

## 本地开发服务器

### 前提条件

使用 Studio 在本地测试您的应用程序：

- 请按照 [本地应用程序快速入门](/langsmith/local-dev-testing) first.
- 如果您不想让数据 [被追踪](/langsmith/observability-concepts#traces) 到 LangSmith，请在 `LANGSMITH_TRACING=false` 中设置 `.env` 。禁用追踪后，数据不会离开您的本地服务器。

### 设置

1. 安装 [LangGraph CLI](/langsmith/cli):

    <CodeGroup>
    ```bash pip
    pip install -U "langgraph-cli[inmem]"
    langgraph dev
    ```

    ```bash uv
    uv add "langgraph-cli[inmem]"
    langgraph dev
    ```

    ```bash npm
    npx @langchain/langgraph-cli dev
    ```
    </CodeGroup>

    <Warning>
    **浏览器兼容性**
    Safari 浏览器阻止 `localhost` 到 Studio 的连接。作为替代方案，请使用 `--tunnel` 通过安全隧道访问 Studio。您需要通过点击在 Studio 界面中手动将隧道 URL 添加到允许的来源 **连接到本地服务器** 。请参阅 [故障排除指南](/langsmith/troubleshooting-studio#safari-connection-issues) 了解具体步骤。
    </Warning>

    这将本地启动 Agent Server，在内存中运行。服务器将以监视模式运行，监听并自动重新启动代码变更。阅读本 [参考资料](/langsmith/cli#dev) 了解启动 API 服务器的所有选项。

    您将看到以下日志：

    ```
    > Ready!
    >
    > - API: [http://localhost:2024](http://localhost:2024/)
    >
    > - Docs: http://localhost:2024/docs
    >
    > - LangSmith Studio Web UI: https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
    ```

    运行后，您将自动被定向到 Studio。

1. 对于正在运行的服务器，请使用以下方式之一访问调试器：
    1. 直接导航到以下 URL： `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`.
    1. 导航到 **部署** 在 UI 中，点击 **Studio** 部署上的按钮，输入 `http://127.0.0.1:2024` 然后点击 **Connect**.

    如果您的服务器在不同的主机或端口上运行，请更新 `baseUrl` 以匹配。

### （可选）附加调试器

要使用断点和变量检查进行逐步调试，请运行以下命令：

    <CodeGroup>
    ```bash pip
    # Install debugpy package
    pip install debugpy
    # Start server with debugging enabled
    langgraph dev --debug-port 5678
    ```

    ```bash uv
    # Install debugpy package
    uv add debugpy
    # Start server with debugging enabled
    langgraph dev --debug-port 5678
    ```
    </CodeGroup>

然后附加您首选的调试器：

<Tabs>
    <Tab title="VS Code">
    将此配置添加到 `launch.json`:

    ```json
    {
        "name": "Attach to LangGraph",
        "type": "debugpy",
        "request": "attach",
        "connect": {
          "host": "0.0.0.0",
          "port": 5678
        }
    }
    ```
    </Tab>
    <Tab title="PyCharm">
    1. 前往运行 → 编辑配置
    2. 点击 + 并选择"Python Debug Server"
    3. 设置 IDE 主机名： `localhost`
    4. 设置端口： `5678` （或您在上一步中选择的端口号）
    5. 点击"确定"并开始调试
    </Tab>
</Tabs>

<Tip>
如需获取帮助，请参阅 [故障排除指南](/langsmith/troubleshooting-studio).
</Tip>
