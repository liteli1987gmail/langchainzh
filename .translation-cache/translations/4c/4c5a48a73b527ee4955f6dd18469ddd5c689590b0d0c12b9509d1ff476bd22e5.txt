[Studio](/langsmith/studio) 在 [LangSmith 部署 UI](https://smith.langchain.com) 中支持连接两种类型的图：

- 部署在[云端或自托管](#deployed-graphs)的图。
- 使用 [Agent Server](#local-development-server) 在本地运行的图。

## 部署的图

在 LangSmith UI 中，通过 **Deployments** 导航访问 Studio。

对于已[部署](/langsmith/deployment-quickstart)的应用程序，您可以在该部署中访问 Studio。具体操作是：在 UI 中导航到该部署，然后选择 **Studio**。

这将加载与您的实时部署连接的 Studio，允许您在该部署中创建、读取和更新[线程](/oss/langgraph/checkpointers#threads)、[助手](/langsmith/assistants)和[内存](/oss/concepts/memory)。

## 本地开发服务器

### 前置条件

要使用 Studio 在本地测试您的应用程序：

- 首先按照[本地应用程序快速开始](/langsmith/local-dev-testing)进行操作。
- 如果您不希望数据被[追踪](/langsmith/observability-concepts#traces)到 LangSmith，请在应用程序的 `.env` 文件中设置 `LANGSMITH_TRACING=false`。禁用追踪后，数据不会离开您的本地服务器。

### 设置

1. 安装 [LangGraph CLI](/langsmith/cli)：

    <CodeGroup>

</fragment>
    ```bash pip
    pip install -U "langgraph-cli[inmem]"
    langgraph dev
    ```

    ```bash uv
    uv add "langgraph-cli[inmem]"
    langgraph dev
    ```

    ```bash npm
    npx @langchain/langgraph-cli dev
    ```
</CodeGroup>

<Warning>
**浏览器兼容性**
Safari 会阻止与 Studio 的 `localhost` 连接。要解决此问题，请使用 `--tunnel` 运行命令，通过安全隧道访问 Studio。您需要通过点击 Studio UI 中的 **连接本地服务器** 手动将隧道 URL 添加到允许的来源。请参阅[故障排除指南](/langsmith/troubleshooting-studio#safari-connection-issues)了解步骤。
</Warning>

这将在本地启动 Agent Server，并在内存中运行。服务器将以监视模式运行，监听代码更改并自动重启。请阅读此[参考文档](/langsmith/cli#dev)了解启动 API 服务器的所有选项。

您将看到以下日志：

    ```
    > Ready!
    >
    > - API: [http://localhost:2024](http://localhost:2024/)
    >
    > - Docs: http://localhost:2024/docs
    >
    > - LangSmith Studio Web UI: https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
    ```

<fragment>

    Once running, you will automatically be directed to Studio.

1. For a running server, access the Dbugger with one of the following:
    1. Directly navigate to the following URL: `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`.
    1. Navigate to **Deployments** in the UI, click the **Studio** button on a deployment, enter `http://127.0.0.1:2024` and click **Connect**.

    If running your server at a different host or port, update the `baseUrl` to match.

### （可选）附加调试器

For step-by-step debugging with breakpoints and variable inspection, run the following:

    <CodeGroup>

</fragment>
    ```bash pip
    # Install debugpy package
    pip install debugpy
    # Start server with debugging enabled
    langgraph dev --debug-port 5678
    ```

    ```bash uv
    # Install debugpy package
    uv add debugpy
    # Start server with debugging enabled
    langgraph dev --debug-port 5678
    ```
</CodeGroup>

然后连接你喜欢的调试器：

<Tabs>
    <Tab title="VS Code">
    将此配置添加到 `launch.json` 中：


</fragment>

    ```json
    {
        "name": "Attach to LangGraph",
        "type": "debugpy",
        "request": "attach",
        "connect": {
          "host": "0.0.0.0",
          "port": 5678
        }
    }
    ```
</Tab>
    <Tab title="PyCharm">
    1. 前往 Run → Edit Configurations
    2. 点击 + 并选择"Python Debug Server"
    3. 设置 IDE 主机名：`localhost`
    4. 设置端口：`5678`（或上一步中选择的端口号）
    5. 点击"确定"并开始调试
    </Tab>
</Tabs>

<Tip>
如需获取帮助，请参阅[故障排除指南](/langsmith/troubleshooting-studio)。
</Tip>
