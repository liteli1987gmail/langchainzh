[**LangSmith Fleet**](/langsmith/fleet/index) 是一个企业级代理平台，用于在整个组织内构建、共享和管理代理。本页将其与类似平台进行比较，帮助您为团队选择合适的平台。

<div className="compact-first-col">

| **平台** | **选择条件...** |
| ------------ | ---------------- |
| [LangSmith Fleet](/langsmith/fleet/index) | 您希望在组织内构建和共享专用代理，保持模型无关性，并通过 LangSmith 实现完整的可观测性。**Fleet** 是唯一提供自托管部署路径和通过 [Deep Agents](/oss/deepagents/overview) 将代理导出为代码的选项。 |
| Claude Cowork | 您希望将开放性任务委托给桌面上的 Claude 进行个人知识工作，且设备上的数据存储满足您的隐私要求。 |
| Amazon Quick | 您已使用 AWS，希望获得可直接访问 AWS 数据源和企业集成的 AI 助手。 |
| Google Workspace Studio | 您的组织使用 Google Workspace，希望构建无需离开 Google 生态系统的原生无代码代理，可直接在 Gmail、Drive 和 Sheets 中使用。 |
| Microsoft Copilot | 您的组织使用 Microsoft 365，希望通过 Copilot Studio 构建低代码代理（可原生发布到 Teams 和 Microsoft 365 Copilot），并通过 Power Platform 管理中心进行治理。 |

</div>

### 目标用户

**Fleet** 兼顾组织级和个人用例。团队可以构建可在整个组织内共享的专用代理（例如，服务于整个运营团队的人员入职代理，或为每个客户经理每周一节省三十分钟的周报代理），任何用户都可以通过 Fleet 的通用默认聊天获取任何任务的帮助。其他平台专注于个人生产力、生态系统特定自动化或两者结合，但没有平台将无代码代理构建与组织内共享和代码导出相结合。

**Fleet** 还允许您设置工具级审批要求，使代理在执行敏感步骤前与您确认，并通过[集中收件箱](https://smith.langchain.com/agents/inbox)进行审核、编辑和批准操作。本比较中没有其他平台提供跨所有代理的单一集中审批收件箱。

### 企业控制与访问

**Fleet** 提供基于角色的访问控制（RBAC）、基于属性的访问控制以及每个代理的共享权限（克隆、运行和编辑）。在比较的平台中，只有 Fleet 记录了每个 MCP 服务器的基于属性的访问控制。所有平台都提供某种形式的 RBAC，但粒度有所不同。

**Fleet** 在工作区级别管理支出。关于企业计费选项，[联系销售](https://www.langchain.com/contact-sales)。

### 模型灵活性

**Fleet** 通过 OpenAI 或 Anthropic 聊天规范支持任何 LLM，包括自托管提供商，无生态系统依赖。Microsoft Copilot 提供精选的多供应商模型和通过 Azure AI Foundry 的自有路径，但完全灵活性需要 Azure 基础设施。Google Workspace Studio 和 Amazon Quick 更受限于各自的供应商生态系统。

在比较的平台中，只有 Fleet 可与任何 OpenAI 或 Anthropic 兼容的 API 端点配合使用，不受云提供商限制。

### 记忆、自我更新与学习

### 可观测性与治理

**Fleet** 最显著的优势是与 LangSmith 的原生连接。每个代理运行都会在 LangSmith 中进行追踪，便于调试性能并进行大规模评估。其他平台提供基本的日志和审计跟踪，但没有平台能通过专用可观测性平台匹配 Fleet 的深度 LLM 感知追踪、评估和调试能力。

### 代码导出与托管

**Fleet** 允许您通过 [Deep Agents](/oss/deepagents/overview)（Fleet 运行的开源代理运行时）将构建的任何代理导出为代码。导出的代理采用 MIT 许可，可以独立于 Fleet 部署、在代码中修改，或通过 [API](/langsmith/fleet/code) 直接集成到您自己的应用程序中。本比较中的其他平台都不提供代码导出路径。

**Fleet** 是本比较中唯一提供自托管部署选项的平台。对于有合规要求的团队，自托管和 BYOC（携带您自己的云）配置让您可以在自己的基础设施内完全运行 Fleet。所有其他平台仅提供纯云托管服务。
