[**LangSmith Fleet**](/langsmith/fleet/index) 是一个企业级智能体平台，用于在整个组织内构建、共享和治理智能体。本页面将其与类似平台进行对比，帮助您为团队选择合适的平台。

<div className="compact-first-col">

| **平台** | **选择条件...** |
| ------------ | ---------------- |
| [LangSmith Fleet](/langsmith/fleet/index) | 您希望在整个组织内构建和共享专用智能体，保持模型无关性，并通过 LangSmith 实现完整的可观测性。 **Fleet** 是唯一提供自托管部署路径和通过 [Deep Agents](/oss/deepagents/overview). |
| Claude Cowork | 您希望将开放式任务委托给桌面上的 Claude 用于个人知识工作，且设备上的数据存储满足您的隐私要求。 |
| Amazon Quick | 您已在使用 AWS，希望获得能直接访问 AWS 数据源和企业集成的 AI 助手。 |
| Google Workspace Studio | 您的组织使用 Google Workspace，您希望获得无需离开 Google 生态系统的无代码智能体，原生集成于 Gmail、Drive 和 Sheets。 |
| Microsoft Copilot | 您的组织使用 Microsoft 365，您希望获得低代码智能体（通过 Copilot Studio），原生发布到 Teams 和 Microsoft 365 Copilot，通过 Power Platform 管理中心进行治理。 |

</div>

### 目标用户

**Fleet** 涵盖组织级和个人使用场景。团队可以构建专用代理在整个组织内共享（例如，一个服务于整个运营团队的供应商入驻代理，或者一个每周报告代理，每周为每位客户经理节省半小时的工作时间），任何用户都可以通过 Fleet 的通用默认聊天功能使用任何工具完成任何任务。其他平台专注于个人生产力、生态系统特定的自动化或两者兼有，但没有哪个平台能将以无代码方式构建代理与组织级共享和代码导出相结合。

**Fleet** 还允许您设置工具级别的审批要求，让代理在执行敏感步骤前征得您的同意，并通过 [集中审批收件箱](https://smith.langchain.com/agents/inbox) 来审查、编辑和批准操作。此对比中的其他平台均不提供跨所有代理的单一集中审批收件箱。

### 企业控制与访问

**Fleet** 提供基于角色的访问控制（RBAC）、基于属性的访问控制和每个代理的共享权限（克隆、运行和编辑）。在本次对比的平台中，只有 Fleet 文档化了每个 MCP 服务器的基于属性的访问控制。所有平台都提供某种形式的 RBAC，但粒度有所不同。

**Fleet** 在工作区级别管理支出。如需企业级计费选项， [请联系销售](https://www.langchain.com/contact-sales).

### 模型灵活性

**Fleet** 支持通过 OpenAI 或 Anthropic 聊天规范使用任何 LLM，包括自托管提供商，无需任何生态系统依赖。Microsoft Copilot 提供精选的多供应商模型和通过 Azure AI Foundry 的自带路径，但完全灵活性需要 Azure 基础设施。Google Workspace Studio 和 Amazon Quick 更局限于各自的供应商生态系统。

在比较的平台中，只有 Fleet 可以与任何 OpenAI 或 Anthropic 兼容的 API 端点配合使用，无论云提供商如何。

### 记忆、自我更新和学习

### 可观测性和治理

**Fleet 的** 最明显优势是与 LangSmith 的原生连接。每个代理运行都在 LangSmith 中进行追踪，使调试性能和规模化运行评估变得容易。其他平台提供基本的日志记录和审计跟踪，但没有平台能匹配 Fleet 通过专用可观测性平台提供的 LLM 感知追踪、评估和调试深度。

### 代码导出和托管

**Fleet** 让您可以将构建的任何代理导出为代码，通过 [Deep Agents](/oss/deepagents/overview)，这是 Fleet 运行的开源代理运行时。导出的代理采用 MIT 许可，可以独立于 Fleet 部署、在代码中修改，或通过以下方式直接集成到您自己的应用中 [API](/langsmith/fleet/code)。此对比中的其他平台均不提供代码导出路径。

**Fleet** 是此对比中唯一提供自托管部署选项的平台。对于有合规要求的团队，自托管和 BYOC（使用您自己的云）配置让您可以在自己的基础设施中完全运行 Fleet。所有其他平台都是仅云托管的托管服务。
