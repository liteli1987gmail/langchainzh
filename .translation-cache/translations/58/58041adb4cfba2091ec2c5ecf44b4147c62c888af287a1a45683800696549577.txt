title: 技能
---

在 **技能** 架构中，专业能力被封装为可调用的"技能"，用于增强 [智能体的](/oss/langchain/agents) 行为。技能主要是由提示词驱动的专业化能力，智能体可以按需调用。
有关内置技能支持，请参阅 [Deep Agents](/oss/deepagents/skills).

<Tip>
此模式在概念上与 [Agent Skills](https://agentskills.io/) 和 [llms.txt](https://llmstxt.org/) （由 Jeremy Howard 引入），它使用工具调用来实现文档的渐进式披露。技能模式将渐进式披露应用于专业提示词和领域知识，而不仅仅是文档页面。

有关可立即使用的技能（可提升智能体在 LangChain 生态系统任务中的性能），请参阅 [LangChain Skills](https://github.com/langchain-ai/langchain-skills) repository.
</Tip>

## 关键特征

* 提示词驱动的专业化：技能主要由专业提示词定义
* 渐进式披露：技能根据上下文或用户需求变得可用
* 团队分布：不同团队可以独立开发和维护技能
* 轻量级组合：技能比完整的子智能体更简单
* 引用感知：技能可以引用脚本、模板和其他资源

## 使用场景

在以下情况下使用技能模式：当您想要一个 [智能体](/oss/langchain/agents) 具有多种可能的专长、不需要在技能之间强制执行特定约束，或者不同团队需要独立开发能力时。常见示例包括编码助手（不同语言或任务的技能）、知识库（不同领域的技能）和创意助手（不同格式的技能）。

## 基本实现

:::python
```python
from langchain.tools import tool
from langchain.agents import create_agent

@tool
def load_skill(skill_name: str) -> str:
    """Load a specialized skill prompt.

    Available skills:
    - write_sql: SQL query writing expert
    - review_legal_doc: Legal document reviewer

    Returns the skill's prompt and context.
    """
    # Load skill content from file/database
    ...

agent = create_agent(
    model="gpt-5.5",
    tools=[load_skill],
    system_prompt=(
        "You are a helpful assistant. "
        "You have access to two skills: "
        "write_sql and review_legal_doc. "
        "Use load_skill to access them."
    ),
)
```
:::
:::js
```typescript
import { tool, createAgent } from "langchain";
import * as z from "zod";

const loadSkill = tool(
  async ({ skillName }) => {
    // Load skill content from file/database
    return "";
  },
  {
    name: "load_skill",
    description: `Load a specialized skill.

Available skills:
- write_sql: SQL query writing expert
- review_legal_doc: Legal document reviewer

Returns the skill's prompt and context.`,
    schema: z.object({
      skillName: z
        .string()
        .describe("Name of skill to load")
    })
  }
);

const agent = createAgent({
  model: "gpt-5.5",
  tools: [loadSkill],
  systemPrompt: (
    "You are a helpful assistant. " +
    "You have access to two skills: " +
    "write_sql and review_legal_doc. " +
    "Use load_skill to access them."
  ),
});
```

## 扩展模式

编写自定义实现时，可以通过多种方式扩展基本技能模式：

- **动态工具注册**：将渐进式披露与状态管理相结合，以注册新 [工具](/oss/langchain/tools) 当技能加载时。例如，加载"数据库_管理员"技能可以同时添加专业上下文并注册数据库特定工具（备份、恢复、迁移）。这使用了跨多智能体模式相同的工具和状态机制——工具更新状态以动态改变智能体能力。

- **层级技能**：技能可以在树结构中定义其他技能，创建嵌套的专业化。例如，加载"数据_科学"技能可能会使子技能可用，如"pandas_专家"、"可视化"和"统计_analysis"。每个子技能都可以根据需要独立加载，从而实现领域知识的细粒度渐进式展示。这种分层方法通过将能力组织成逻辑分组来帮助管理大型知识库，这些分组可以按需发现和加载。

- **引用感知**：虽然每个技能只有一个提示词，但该提示词可以引用其他资源的位置，并提供关于智能体何时应使用这些资源的信息。
当这些资源变得相关时，智能体会知道这些文件的存在，并在需要时将其读取到内存中以完成任务。
这也遵循渐进式披露模式，并限制了上下文窗口中的信息量。
