---
title: 概述
description: 将 useStream 连接到任何 React UI 组件库或生成式 UI 框架
---

@[`useStream`] 是 UI 无关的。它返回包含消息、工具调用、加载状态、值和线程元数据的纯响应式状态，你可以将其连接到任何你选择的视觉层。这些页面展示了不同库如何与 LangChain 前端集成，每个库都有不同的 AI 聊天和生成式 UI 构建理念。

## 集成

<CardGroup cols={2}>
  <Card title="CopilotKit" icon="package" href="/oss/langchain/frontend/integrations/copilotkit">
    完整的 AI 聊天运行时，支持结构化生成式 UI。向 LangGraph 部署添加自定义 CopilotKit 端点，然后在 React 中渲染动态组件树。
  </Card>
  <Card title="AI Elements" icon="package" href="/oss/langchain/frontend/integrations/ai-elements">
    可组合的基于 shadcn/ui 的 AI 聊天组件。直接使用 `Conversation`、`Message`、`Tool` 和 `Reasoning` 组件，并将它们直接连接到 `stream.messages`。
  </Card>
  <Card title="assistant-ui" icon="package" href="/oss/langchain/frontend/integrations/assistant-ui">
    带完整运行时层的无头 React 框架。通过 `useExternalStoreRuntime` 适配器将 `useStream` 桥接到 `AssistantRuntimeProvider`。
  </Card>
  <Card title="OpenUI" icon="package" href="/oss/langchain/frontend/integrations/openui">
    生成式 UI 库，让代理能够用声明式组件 DSL 生成完整的交互式仪表板。专为数据丰富、报告风格的 UI 设计。
  </Card>
</CardGroup>

## 选择库

每个库都适合略微不同的集成模式。选择取决于你要构建什么样的 UI：

| | CopilotKit | AI Elements | assistant-ui | OpenUI |
| --- | --- | --- | --- | --- |
| **最佳场景** | 完整聊天运行时加上结构化生成式 UI | 带丰富消息类型的聊天 | 功能齐全且设置简单的聊天 | 生成的仪表板和报告 |
| **UI 风格** | CopilotKit 聊天外壳 + 自定义消息渲染器 | 可组合的 shadcn/ui 组件 | 无头插槽 + 默认主题 | 带声明式 DSL 的预构建组件库 |
| **自定义** | 自定义后端端点、代理上下文和渲染器 | 直接编辑源文件 | 覆盖组件插槽 | 通过 CSS 自定义属性设置主题 |
| **流式体验** | 运行时管理的聊天流，包含结构化助手负载 | 组件级渐进式渲染 | 内置线程管理 | 提升——外壳立即显示，数据随后填充 |
| **工具调用** | 通过 CopilotKit 运行时和自定义渲染器 | `Tool` / `ToolHeader` / `ToolOutput` | 通过消息插槽自定义 | 内联在生成的 UI 中 |
| **代理格式** | 结构化助手响应加上可选的 Markdown | 任意 `stream.messages` | 任意 `stream.messages` | 代理输出 openui-lang 文本 |

所有四个库都与 LangChain 代理配合良好，后三个还可以直接连接到 @[`useStream`]。当你想要更丰富的运行时层和专用端点来配合 [LangGraph](/oss/langgraph/overview) 部署时，CopilotKit 特别有用。
