---
title: 概览
description: 将 useStream 连接到任何 React UI 组件库或生成式 UI 框架
---

@[`useStream`是 UI 无关的。它返回带有消息、工具调用、加载标志、值和线程元数据的纯响应式状态，您可以将其连接到任何选择的视觉层。这些页面展示了不同库如何与 LangChain 前端集成，每个库都有不同的 AI 聊天和生成式 UI 构建理念。

## 集成

<CardGroup cols={2}>
  <Card title="CopilotKit" icon="package" href="/oss/langchain/frontend/integrations/copilotkit">
    支持结构化生成式 UI 的完整 AI 聊天运行时。将自定义 CopilotKit 端点添加到您的 LangGraph 部署中，然后在 React 中渲染动态组件树。
  </Card>
  <Card title="AI Elements" icon="package" href="/oss/langchain/frontend/integrations/ai-elements">
    Composable shadcn/ui-based components for AI chat. Drop in `Conversation`, `Message`, `Tool`和 `Reasoning` 并将它们直接连接到 `stream.messages`.
  </Card>
  <Card title="assistant-ui" icon="package" href="/oss/langchain/frontend/integrations/assistant-ui">
    具有完整运行时层的无头 React 框架。桥接 `useStream` to `AssistantRuntimeProvider` 通过 `useExternalStoreRuntime` adapter.
  </Card>
  <Card title="OpenUI" icon="package" href="/oss/langchain/frontend/integrations/openui">
    一个生成式 UI 库，让代理能够使用声明式组件 DSL 生成完整的交互式仪表板。专为数据丰富的报告风格 UI 构建。
  </Card>
</CardGroup>

## 选择库

每个库都有略微不同的集成模式。选择取决于您构建的 UI 类型：

| | CopilotKit | AI Elements | assistant-ui | OpenUI |
| --- | --- | --- | --- | --- |
| **最佳适用场景** | 完整聊天运行时加结构化生成式 UI | 带丰富消息类型的聊天 | 最小设置的全功能聊天 | 生成的仪表板和报告 |
| **UI 风格** | CopilotKit chat shell + custom message renderers | Composable shadcn/ui components | Headless slots + default theme | Prebuilt component library with declarative DSL |
| **自定义** | 自定义后端端点、代理上下文和渲染器 | 直接编辑源文件 | 覆盖组件插槽 | 通过 CSS 自定义属性主题 |
| **流式 UX** | 运行时管理的聊天流，带结构化助手负载 | 组件级渐进式渲染 | 内置线程管理 | 提升——外壳立即出现，数据逐步填充 |
| **工具调用** | 通过 CopilotKit 运行时和自定义渲染器 | `Tool` / `ToolHeader` / `ToolOutput` | 通过消息插槽自定义 | 内联在生成的 UI 中 |
| **代理格式** | 结构化助手响应加可选 Markdown | 任意 `stream.messages` | 任意 `stream.messages` | 代理输出 openui-lang 文本 |

这四个库都能很好地与 LangChain 代理配合使用，后三个库还可以直接连接到 @[`useStream`]。当您想要更丰富的运行时层和可以与 [LangGraph](/oss/langgraph/overview) deployment.
