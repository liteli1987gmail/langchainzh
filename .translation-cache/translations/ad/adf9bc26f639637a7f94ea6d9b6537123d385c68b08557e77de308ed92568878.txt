**让您的集成对社区可用。**

:::python
<Warning>
**请勿向 LangChain 或 Deep Agents 代码仓库提交集成的 PR。**

新的集成应作为 **独立的 PyPI 包** 在您自己的 GitHub 组织或账户下发布（例如， `langchain-yourservice`），而不是作为 PR 提交到 [`langchain-ai/langchain`](https://github.com/langchain-ai/langchain) repository.

主代码仓库仅包含一小部分由 LangChain 团队维护的第一方集成（如 OpenAI、Anthropic 和 Ollama）。
</Warning>

现在您的包已实现并通过测试，您可以发布它并添加文档以使其能被社区发现。

## 发布您的包

<Info>
    本指南假设您已经实现了您的包并为其编写了测试。如果您还没有，请参阅 [实现指南](/oss/contributing/implement-langchain) 和 [测试指南](/oss/contributing/standard-tests-langchain).
</Info>

出于本指南的目的，我们将使用 PyPI 作为包注册表。如果您选择发布到其他注册表，说明可能会有所不同。

### 设置凭据

首先，确保您拥有 PyPI 账户：

<AccordionGroup>
    <Accordion title="How to create a PyPI Token" icon="key">
        <Steps>
            <Step title="Create account">
                前往 [PyPI 网站](https://pypi.org/) 并创建一个账户
            </Step>

            <Step title="Verify email">
                通过点击 PyPI 发送给您邮件中的链接来验证您的电子邮件地址
            </Step>

            <Step title="Enable 2FA">
                前往您的账户设置，点击"生成恢复代码"以启用双因素身份验证 (2FA)。要生成 API 令牌，您 **必须** 启用双因素认证 (2FA)
            </Step>

            <Step title="Generate token">
                前往您的账户设置并 [生成一个新的 API 令牌](https://pypi.org/manage/account/token/)
            </Step>
        </Steps>
    </Accordion>
</AccordionGroup>

## 添加文档

要在此站点的 [集成标签页](/oss/integrations/providers/overview)下为您的包添加文档，您需要创建相关的文档页面，并在 [LangChain 文档仓库](https://github.com/langchain-ai/docs).

### 编写文档

根据您构建的集成类型，您需要创建不同类型的文档页面。LangChain 为不同类型的集成提供了模板，帮助您快速上手。

<CardGroup>
    <Card title="Chat models" icon="message" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx" arrow/>
    <Card title="Tools/toolkits" icon="tool" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx" arrow/>
    <Card title="Middleware" icon="plug" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/middleware/TEMPLATE.mdx" arrow/>
    <Card title="Vector stores" icon="database" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx" arrow/>
</CardGroup>

<Tip>
    要参考现有文档，您可以查看 [集成列表](/oss/integrations/providers/overview) 并找到与您的类似的集成。

    要查看原始 Markdown 格式的文档页面，请使用页面右上角"复制页面"按钮旁边的下拉菜单，选择"查看为 Markdown"。
</Tip>

### 向文档仓库提交 PR

创建 [LangChain 文档仓库](https://github.com/langchain-ai/docs) （不是主 `langchain` 仓库），在个人 GitHub 账户下创建，并克隆到本地。为您的集成创建一个新分支。复制模板并使用您喜欢的 Markdown 文本编辑器进行修改。务必参考并遵循 [文档指南](/oss/contributing/documentation) 编写文档。

<Info>
    此 PR 仅用于 **文档编写**。您的集成包本身应位于您 GitHub 组织或账户下的独立仓库中，作为独立包发布到 PyPI。
</Info>

<Warning>
    如果出现以下情况，我们可能会拒绝 PR 或要求修改：
    - CI 检查失败
    - 存在严重的语法错误或拼写错误
    - [Mintlify 组件](/oss/contributing/documentation#mintlify-components) 使用不当
    - 页面缺少 [前置元数据](/oss/contributing/documentation#page-structure)
    - [本地化](/oss/contributing/documentation#localization) 缺失（如果适用）
    - [代码示例](/oss/contributing/documentation#in-code-documentation) 无法运行或存在错误
    - [质量标准](/oss/contributing/documentation#quality-standards) 未达标
</Warning>

请耐心等待，我们处理大量 PR 会需要一些时间。我们会尽快审核您的 PR 并提供反馈或进行合并。 **请勿反复提及维护者关于您的 PR。**

<Note>
    如果您的 PR 包含 AI 生成的内容，您必须遵守我们的 [LLM 可接受使用规范](/oss/contributing/overview#acceptable-uses-of-llms) policy.
</Note>
