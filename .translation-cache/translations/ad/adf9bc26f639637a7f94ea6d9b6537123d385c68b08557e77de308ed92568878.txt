<Warning>
**请勿向 LangChain 或 Deep Agents 代码仓库提交集成 PR。**

新集成应作为**独立 PyPI 包**发布在您自己的 GitHub 组织或账户下（例如 `langchain-yourservice`），而不是作为 PR 提交到 [`langchain-ai/langchain`](https://github.com/langchain-ai/langchain) 代码仓库。

## 发布您的包

<Info>
    本指南假设您已经实现了您的包并为其编写了测试。如果您还没有这样做，请参阅[实现指南](/oss/contributing/implement-langchain)和[测试指南](/oss/contributing/standard-tests-langchain)。
</Info>

为便于说明，本指南将使用 PyPI 作为包注册中心。如果您愿意，也可以发布到其他注册中心；具体说明会有所不同。

### 设置凭证

首先，确保您有一个 PyPI 账户：

<AccordionGroup>
    <Accordion title="如何创建 PyPI Token" icon="key">
        <Steps>
            <Step title="创建账户">
                访问 [PyPI 网站](https://pypi.org/) 并创建一个账户
            </Step>

            <Step title="验证邮箱">
                点击 PyPI 发送给您的邮件中的链接来验证您的邮箱地址
            </Step>

            <Step title="启用双因素认证">
                进入您的账户设置，点击"生成恢复码"以启用双因素认证。要生成 API token，您**必须**启用双因素认证
            </Step>

            <Step title="生成 token">
                进入您的账户设置，[生成一个新的 API token](https://pypi.org/manage/account/token/)
            </Step>
        </Steps>
    </Accordion>
</AccordionGroup>

## 添加文档

要为您的包在此站点的[集成选项卡](/oss/integrations/providers/overview)下添加文档，您需要创建相关文档页面，并在 [LangChain 文档仓库](https://github.com/langchain-ai/docs)中提交 PR。

### 编写文档

根据您构建的集成类型，您需要创建不同类型的文档页面。LangChain 为不同类型的集成提供了模板来帮助您入门。

<CardGroup>
    <Card title="聊天模型" icon="message" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx" arrow/>
    <Card title="工具/工具包" icon="tool" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx" arrow/>
    <Card title="中间件" icon="plug" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/middleware/TEMPLATE.mdx" arrow/>
    <Card title="向量存储" icon="database" href="https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx" arrow/>
</CardGroup>

<Tip>
    要参考现有文档，您可以查看[集成列表](/oss/integrations/providers/overview)并找到与您类似的集成。

    要查看给定文档页面的原始 markdown，请使用页面右上角"复制页面"旁边的下拉按钮，选择"查看为 Markdown"。
</Tip>

### 提交 PR 到文档仓库

在您个人的 GitHub 账户下创建 [LangChain 文档仓库](https://github.com/langchain-ai/docs)（不是主 `langchain` 仓库）的分支，并在本地克隆它。为您的集成创建一个新分支。复制模板并使用您喜欢的 markdown 文本编辑器进行修改。编写文档时请务必参考并遵循[文档指南](/oss/contributing/documentation)。

<Info>
    此 PR 仅用于**文档**。您的集成包本身应存在于您 GitHub 组织或账户下的独立仓库中，作为独立包发布到 PyPI。
</Info>

<Warning>
    在以下情况下，我们可能会拒绝 PR 或要求修改：
    - CI 检查失败
    - 存在严重的语法错误或拼写错误
    - [Mintlify 组件](/oss/contributing/documentation#mintlify-components) 使用不当
    - 页面缺少 [frontmatter](/oss/contributing/documentation#page-structure)
    - 缺少[本地化](/oss/contributing/documentation#localization)（如适用）
    - [代码示例](/oss/contributing/documentation#in-code-documentation) 无法运行或存在错误
    - 未达到[质量标准](/oss/contributing/documentation#quality-standards)
</Warning>

请耐心等待，我们处理的 PR 数量很大。我们会尽快审查您的 PR 并提供反馈或合并。**请不要反复提及维护者询问您的 PR 状态。**

<Note>
    如果您的 PR 包含 AI 生成的内容，您必须遵守我们的 [LLM 合理使用政策](/oss/contributing/overview#acceptable-uses-of-llms)。
</Note>
