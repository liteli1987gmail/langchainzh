要开始使用 LangSmith，您需要创建一个账户。您可以在以下位置注册免费账户 [LangSmith UI](https://smith.langchain.com)。LangSmith 支持使用 Google、GitHub 和电子邮件登录。

## API 密钥

LangSmith 支持两种类型的 API 密钥。您可以使用这两种令牌来验证对 LangSmith API 的请求，但它们有不同的用例：

- [**个人访问令牌 (PAT)**](/langsmith/administration-overview#personal-access-tokens-pats) 继承创建它们的用户的权限。将 PAT 用于个人脚本或工具。
- [**服务密钥**](/langsmith/administration-overview#service-keys) 可以限定在特定的 [工作区](/langsmith/administration-overview#workspaces) 或整个 [组织](/langsmith/administration-overview#organizations)。将服务密钥用于应用程序和生产服务。

要使用 LangSmith 记录追踪并运行评估，请创建 API 密钥来验证您的请求。

<Steps>
  <Step title="Open API Keys settings" icon="settings">
    导航至 [设置页面](https://smith.langchain.com/settings) 并选择 **API 密钥** section.
  </Step>
  <Step title="Configure the key type" icon="key">
    对于服务密钥，请在组织限定密钥和工作区限定密钥之间进行选择。如果密钥是工作区限定的，您必须指定工作区。

    [企业版](/langsmith/pricing-plans) 用户还可以 [为服务密钥分配特定的工作区角色](/langsmith/administration-overview#workspace-roles-rbac) ，这会独立于任何用户调整其权限。
  </Step>
  <Step title="Set expiration" icon="calendar">
    设置密钥的过期时间。密钥在选定的天数后失效，如果选择了相应选项，则永不过期。
  </Step>
  <Step title="Create the key" icon="circle-check">
    点击 **创建 API 密钥。** LangSmith 将仅显示一次 API 密钥，因此请务必复制并将其存储在安全的地方。
  </Step>
</Steps>

<Tip>
  要删除 API 密钥，请导航至 [设置页面](https://smith.langchain.com/settings)，在 **API 密钥** 部分找到该密钥，然后选择垃圾桶图标 <Icon icon="trash" iconType="solid"/> 在 **操作** column.
</Tip>

<Tip>
  [企业版](/langsmith/pricing-plans) 组织管理员可以编辑现有服务密钥的 [角色](/langsmith/administration-overview#workspace-roles-rbac) 而无需轮换密钥。在 [设置页面](https://smith.langchain.com/settings) **API 密钥** 部分，切换到 **服务** 标签页，然后点击任意服务密钥行以打开编辑对话框。更新工作区角色（对于组织限定密钥，还包括组织角色），然后点击 **保存**—密钥字符串本身保持不变。
</Tip>

## 配置 SDK

安装适用于您语言的 SDK：

<Tabs>
  <Tab title="Python">
    <CodeGroup>
      ```bash pip
      pip install langsmith
      ```
      ```bash uv
      uv add langsmith
      ```
    </CodeGroup>
  </Tab>
  <Tab title="TypeScript">
    ```bash
    npm install langsmith
    ```
  </Tab>
</Tabs>

有关完整详情，请参阅 [Python SDK](/langsmith/smith-python-sdk) or [JS/TS SDK](/langsmith/smith-js-ts-sdk) reference.

然后，设置您的 API 密钥并启用追踪：

```bash
export LANGSMITH_API_KEY=<your-api-key>
export LANGSMITH_TRACING=true
```

您可能还需要以下额外环境变量。

`LANGSMITH_ENDPOINT` 控制 SDK 向哪个 LangSmith 服务器发送数据。默认为 `https://api.smith.langchain.com` （GCP 美国）。仅在您使用不同部署时才设置此项。对于区域性 SaaS，请将其设置为您所在区域的 API URL：

<SaasRegionUrls prefix="api.smith" />

`LANGSMITH_WORKSPACE_ID` 仅在您的 API 密钥作用域超过一个时才需要 [工作区](/langsmith/administration-overview#workspaces)。在以下位置找到您的工作区 ID [设置页面](https://smith.langchain.com/settings) 位于 **常规**:

`LANGSMITH_WORKSPACE_ID=<Workspace ID>`

要跨本地 shell 或远程运行时重复使用端点、API 密钥和工作区设置，请参阅 [配置文件](/langsmith/profile-configuration).
