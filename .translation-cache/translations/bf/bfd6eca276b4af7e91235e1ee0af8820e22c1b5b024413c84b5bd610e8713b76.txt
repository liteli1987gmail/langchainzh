要开始使用 LangSmith，您需要创建一个账户。您可以在 [LangSmith UI](https://smith.langchain.com) 注册一个免费账户。LangSmith 支持使用 Google、GitHub 和电子邮件登录。

## API 密钥

LangSmith 支持两种类型的 API 密钥。您可以使用这两种类型的令牌来验证对 LangSmith API 的请求，但它们的用例不同：

- [**个人访问令牌 (PATs)**](/langsmith/administration-overview#personal-access-tokens-pats) 继承创建它们的用户的权限。请将 PAT 用于个人脚本或工具。
- [**服务密钥**](/langsmith/administration-overview#service-keys) 可以被限定在特定的[工作空间](/langsmith/administration-overview#workspaces)或整个[组织](/langsmith/administration-overview#organizations)。请将服务密钥用于应用程序和生产服务。

要使用 LangSmith 记录跟踪和运行评估，请创建一个 API 密钥来验证您的请求。

<Steps>
  <Step title="打开 API 密钥设置" icon="settings">
    导航到[设置页面](https://smith.langchain.com/settings)并选择 **API 密钥**部分。
  </Step>
  <Step title="配置密钥类型" icon="key">
    对于服务密钥，请在组织范围和工作空间范围的密钥之间进行选择。如果密钥是工作空间范围的，则必须指定工作空间。

    [企业版](/langsmith/pricing-plans)用户还可以为服务密钥[分配特定的工作空间角色](/langsmith/administration-overview#workspace-roles-rbac)，这会独立于任何用户调整其权限。
  </Step>
  <Step title="设置过期时间" icon="calendar">
    设置密钥的过期时间。密钥在所选天数后变得不可使用，或者如果选择了该项，则永不过期。
  </Step>
  <Step title="创建密钥" icon="circle-check">
    点击 **创建 API 密钥**。LangSmith 将仅显示一次 API 密钥，因此请确保复制并将其存储在安全的地方。
  </Step>
</Steps>

<Tip>
  要删除 API 密钥，请导航到[设置页面](https://smith.langchain.com/settings)，在 **API 密钥**部分找到该密钥，然后在 **操作**列中选择垃圾桶图标 <Icon icon="trash" iconType="solid"/>。
</Tip>

<Tip>
  [企业版](/langsmith/pricing-plans)组织管理员可以在不轮换密钥的情况下编辑现有服务密钥的[角色](/langsmith/administration-overview#workspace-roles-rbac)。在[设置页面](https://smith.langchain.com/settings) **API 密钥**部分，切换到 **服务**选项卡，然后点击任意服务密钥行以打开编辑对话框。更新工作空间角色（对于组织范围的密钥，还要更新组织角色），然后点击 **保存**——密钥字符串本身保持不变。
</Tip>

## 配置 SDK

为您的语言安装 SDK：

<Tabs>
  <Tab title="Python">
    <CodeGroup>

</fragment>
      ```bash pip
      pip install langsmith
      ```
      ```bash uv
      uv add langsmith
      ```
</CodeGroup>
  </Tab>
  <Tab title="TypeScript">

</fragment>
    ```bash
    npm install langsmith
    ```
</Tab>
</Tabs>

有关完整详细信息，请参阅 [Python SDK](/langsmith/smith-python-sdk) 或 [JS/TS SDK](/langsmith/smith-js-ts-sdk) 参考文档。

然后，设置您的 API 密钥并启用跟踪：

```bash
export LANGSMITH_API_KEY=<your-api-key>
export LANGSMITH_TRACING=true
```

您可能还需要以下附加环境变量。

`LANGSMITH_ENDPOINT` 控制 SDK 向哪个 LangSmith 服务器发送数据。默认为 `https://api.smith.langchain.com`（GCP US）。仅当您使用不同部署时设置此项。对于区域 SaaS，请将其设置为您所在区域的 API URL：

<SaasRegionUrls prefix="api.smith" />

`LANGSMITH_WORKSPACE_ID` 仅当您的 API 密钥作用域超过一个 [workspace](/langsmith/administration-overview#workspaces) 时才需要。在[设置页面](https://smith.langchain.com/settings)下的 **常规** 中找到您的工作区 ID：

`LANGSMITH_WORKSPACE_ID=<Workspace ID>`

若要在本地 shell 或远程运行时中复用端点、API 密钥和工作区设置，请参阅[配置文件](/langsmith/profile-configuration)。
