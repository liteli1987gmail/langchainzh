    ## `deepagents` v0.6.0

    - **[`CodeInterpreterMiddleware`](/oss/deepagents/interpreters)**：（实验性）`deepagents` 现在支持通过作用域 QuickJS 运行时进行代码执行和编程式工具调用。
    - 在 `stream_events` / `astream_events` 中支持 `version="v3"`。详见[事件流](/oss/deepagents/event-streaming)指南。
    - **[`DeltaChannel`](/oss/langgraph/pregel#deltachannel)（测试版）**（[博客](https://www.langchain.com/blog/delta-channels-evolving-agent-runtime)）：Deep Agents 现在使用 `DeltaChannel` 处理消息历史和代理文件。不再在每次检查点重新序列化完整的累积值，而是只存储每步写入的增量——随着线程增长，保持检查点大小较小。
    - **[Harness profiles](/oss/deepagents/profiles)**：注册按提供商或按模型的配置包（`HarnessProfile`），`create_deep_agent` 在选择模型时自动应用——系统提示调整、工具覆盖、中间件更改和子代理默认值——无需修改调用方。
    - **[`ContextHubBackend`](/oss/deepagents/backends#contexthubbackend)**（[博客](https://www.langchain.com/blog/introducing-context-hub)）：由 LangSmith Hub 支持的新的文件系统后端。代理文件——技能、记忆和其他持久化上下文——存储为 Hub 提交，每次写入都提供版本历史，且无需配置单独的 LangGraph 存储即可获得 LangSmith 原生的持久性。
</Update>

    ## `langgraph` v1.2.0

    此版本添加了对节点执行更细粒度的控制（超时、错误恢复和优雅关闭），一种新的通道类型以减少长时间运行线程的检查点开销，以及一个新的以内容块为中心的流 API（v3），具有类型化的逐通道投影。

    - **[`DeltaChannel`](/oss/langgraph/pregel#deltachannel)（测试版）**：一种新的通道类型，每步只存储增量增量，而不是重新序列化完整累积值。最适用于随时间增长的大型通道，例如长时间运行线程中的消息列表。使用 `snapshot_frequency=K` 每 K 步写入一次完整快照并限制读取延迟。

    - **[逐节点超时](/oss/langgraph/fault-tolerance#timeouts)**：向 @[`add_node`] 传递 `timeout=` 以限制单次尝试可运行的时间。设置硬性挂钟限制（`run_timeout`）、在有进展时重置的空闲限制（`idle_timeout`），或通过 @[`TimeoutPolicy`] 两者结合使用。当限制触发时，LangGraph 抛出 @[`NodeTimeoutError`]，清除该尝试的写入，并将控制权交给重试策略。仅限异步节点。

    - **[节点级错误处理器](/oss/langgraph/fault-tolerance#error-handling)**：向 @[`add_node`] 传递 `error_handler=` 以在所有重试耗尽后运行恢复函数。处理器接收类型化 @[`NodeError`] 并可返回 @[`Command`] 来更新状态并路由到不同节点，适用于 Saga/补偿模式。

    - **[优雅关闭](/oss/langgraph/fault-tolerance#graceful-shutdown)**：在当前超级步骤完成后协作停止运行中的任务，并保存可恢复的检查点。创建 @[`RunControl`] 并从任何线程调用 `request_drain()`；运行将抛出 `GraphDrained`，以后可使用相同配置恢复。

    - **新的事件流 API（测试版）**：向 `stream_events()` / `astream_events()` 传递 `version="v3"` 以获取以内容块为中心的协议，具有类型化的逐通道投影（`run.values`、`run.messages`、`run.lifecycle`、`run.subgraphs`）以及可选的更新、自定义事件、检查点、任务和调试转换器。`run.messages` 为每次 LLM 调用生成一个 `ChatModelStream`，具有文本、推理、工具调用和用量的类型化子投影。`version="v1"` 和 `version="v2"` 保持不变。

    超时和错误处理器仅适用于 Python；重试策略继续在 Python 和 TypeScript 中工作。
</Update>
<Update label="2026年4月7日" tags={["deepagents"]} rss={{ title: "2026年4月7日 - deepagents" }}>
    ## `deepagents` v0.5.0

    - **[异步子代理](/oss/deepagents/async-subagents)**：Deep Agents 可以启动非阻塞后台任务，因此用户可以在子代理并发工作时继续与代理交互。需要 [LangSmith Deployment](/langsmith/deployment) 用于子代理。

    - **多模态支持**：`read_file` 工具现在除了图像外还支持 PDF、音频和视频文件。

    - **后端更改**：我们对 Deep Agents [后端协议](https://github.com/langchain-ai/deepagents/blob/main/libs/deepagents/deepagents/backends/protocol.py)进行了向后兼容的更改：
      - 更新了 [State 和 Store 后端](/oss/deepagents/backends) 中存储的文件格式以支持二进制文件。
      - 改进了从后端到工具的错误传播。
      - 现在可以直接实例化 `StateBackend()` 和 `StoreBackend()`。使用工厂指定（例如 `backend=(lambda rt: StateBackend(rt))`）已弃用。

    - **Anthropic 提示缓存改进**：我们对 Anthropic 模型的提示缓存性能进行了一些改进。
</Update>
<Update label="2026年3月10日" tags={["langgraph"]} rss={{ title: "2026年3月10日 - langgraph" }}>
    ## `langgraph` v1.1.0

    - **类型安全流（`version="v2"`）**：向 `stream()` / `astream()` 传递 `version="v2"` 以获取统一的 `StreamPart` 输出，每个块都有 `type`、`ns` 和 `data` 键。每种模式都有自己的 `TypedDict`，可从 `langgraph.types` 导入。请参阅[流文档](/oss/langgraph/streaming#stream-output-format-v2)。

    - **类型安全调用（`version="v2"`）**：向 `invoke()` / `ainvoke()` 传递 `version="v2"` 以获取具有 `.value` 和 `.interrupts` 属性的 `GraphOutput` 对象。请参阅[调用文档](/oss/langgraph/streaming#v2-invoke-format)。

    - **Pydantic 和 dataclass 强制转换**：使用 `version="v2"` 时，`invoke()` 和 `values` 模式流输出会自动强制转换为您声明的 Pydantic 模型或 dataclass 类型。

    - **修复了带中断和子图的时间旅行**：重放不再重用过期的 `RESUME` 值，子图正确恢复父级历史状态的检查点。

    - **完全向后兼容**：`version="v2"` 是可选的。`GraphOutput` 支持弃用的字典式访问以实现渐进式迁移。
</Update>
<Update label="2026年2月10日" tags={["deepagents"]} rss={{ title: "2026年2月10日 - deepagents" }}>
    ## `deepagents` v0.4.0

    * 可插拔沙箱的新集成包：[`langchain-modal`](https://pypi.org/project/langchain-modal/)、[`langchain-daytona`](https://pypi.org/project/langchain-daytona/) 和 [`langchain-runloop`](https://pypi.org/project/langchain-runloop/)。请参阅[沙箱指南](/oss/deepagents/sandboxes)和示例[数据分析教程](/oss/deepagents/data-analysis)。
    * [对话历史摘要](/oss/deepagents/context-engineering#summarization)的更改：
      * 摘要现在通过 `wrap_model_call` 事件在模型节点中进行。因此，我们在图状态中保留了完整的消息历史。
      * 更准确的令牌计数。
      * 摘要现在将在聊天模型抛出 @[`ContextOverflowError`]（在 `langchain-core` 中定义）时自动触发。目前 `langchain-anthropic` 和 `langchain-openai` 支持此功能。
    * 我们现在默认为以 `"openai:"` 为前缀的模型字符串使用 Responses API。
        <Accordion
            title="使用 Responses API 禁用数据保留"
        >
            ```python
            from langchain.chat_models import init_chat_model

            agent = create_deep_agent(
                model=init_chat_model(
                    "openai:...",
                    use_responses_api=True,
                    store=False,
                    include=["reasoning.encrypted_content"],
                )
            )
            ```

<Update label="2025年11月25日" tags={["langchain"]} rss={{ title: "2025年11月25日 - langchain" }}>
    ## `langchain` v1.1.0

    * [模型配置](/oss/langchain/models#model-profiles)：聊天模型现在通过 `.profile` 属性公开支持的功能和特性。这些数据来源于 [models.dev](https://models.dev)，一个提供模型能力数据的开源项目。
    * [摘要中间件](/oss/langchain/middleware/built-in#summarization)：更新为使用模型配置支持灵活的触发点，实现上下文感知的摘要。
    * [结构化输出](/oss/langchain/structured-output)：`ProviderStrategy` 支持（原生结构化输出）现在可从模型配置中推断。
    * [`create_agent` 的 `SystemMessage`](/oss/langchain/middleware/custom#dynamic-prompt)：支持直接将 `SystemMessage` 实例传递给 `create_agent` 的 `system_prompt` 参数，启用缓存控制和结构化内容块等高级功能。
    * [模型重试中间件](/oss/langchain/middleware/built-in#model-retry)：用于自动重试失败模型调用的新中间件，支持可配置的指数退避。
    * [内容审核中间件](/oss/integrations/middleware/openai#content-moderation)：用于检测和处理 agent 交互中不安全内容的 OpenAI 内容审核中间件。支持检查用户输入、模型输出和工具结果。
</Update>
