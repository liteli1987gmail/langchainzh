## 什么是分支对话？

分支对话将对话视为带检查点的时间线，而不是
扁平列表。每条消息都有元数据，指向该消息创建之前的
检查点。编辑消息或重新生成回复会从该检查点提交一个新的
运行。

主要功能：

- **编辑任意用户消息：** 重写之前的提示词并从该点重新运行智能体
- **重新生成任意 AI 回复：** 让智能体针对相同的输入产生不同的答案
- **查看历史记录：** 需要分支时间线时，使用 LangGraph 客户端加载检查点

## 理解消息元数据

该 `useMessageMetadata(stream, messageId)` 辅助函数返回一条消息的 @[MessageMetadata]
在渲染每条消息的组件中使用它，这样
元数据就被限制在该消息 ID 的范围内：

```tsx
import type { BaseMessage } from "langchain";
import { useState } from "react";
import { useMessageMetadata, useStream } from "@langchain/react";

function Chat() {
  const stream = useStream<typeof myAgent>({
    apiUrl: AGENT_URL,
    assistantId: "simple_agent",
  });

  return stream.messages.map((message) => (
    <MessageWithForkControls
      key={message.id}
      stream={stream}
      message={message}
    />
  ));
}

function MessageWithForkControls({
  stream,
  message,
}: {
  stream: ReturnType<typeof useStream>;
  message: BaseMessage;
}) {
  const metadata = useMessageMetadata(stream, message.id);
  const checkpointId = metadata?.parentCheckpointId;
  const [editedText, setEditedText] = useState(message.text);

  return (
    <form
      onSubmit={(event) => {
        event.preventDefault();
        if (!checkpointId) return;

        stream.submit(
          { messages: [{ type: "human", content: editedText }] },
          { forkFrom: { checkpointId } }
        );
      }}
    >
      <textarea
        value={editedText}
        onChange={(event) => setEditedText(event.target.value)}
      />
      <button disabled={!checkpointId || editedText === message.text}>
        Submit edited branch
      </button>
    </form>
  );
}
```

`parentCheckpointId` 是消息之前的检查点。将其用作
编辑和重新生成的分叉点。

## 编辑消息

要编辑用户消息并分支对话：

1. 从消息的元数据中获取 `parentCheckpointId` 提交带有
2. 的已编辑消息 `forkFrom: { checkpointId }`
3. 智能体从该点重新运行

```ts
function handleEdit(
  stream: ReturnType<typeof useStream>,
  originalMsg: HumanMessage,
  metadata: MessageMetadata | undefined,
  newText: string
) {
  if (!metadata?.parentCheckpointId) return;

  stream.submit(
    {
      messages: [{ type: "human", content: newText }],
    },
    { forkFrom: { checkpointId: metadata.parentCheckpointId } }
  );
}
```

编辑后：

- 智能体从分叉点使用更新的消息重新运行
- 原始路径在线程历史中仍然可用

## 重新生成回复

要在不更改输入的情况下重新生成 AI 回复：

1. 获取 `parent_checkpoint` 从 AI 消息的元数据中
2. 使用空输入和 `forkFrom: { checkpointId }`
3. 提交

```ts
function handleRegenerate(
  stream: ReturnType<typeof useStream>,
  metadata: MessageMetadata | undefined
) {
  if (!metadata?.parentCheckpointId) return;

  stream.submit(undefined, {
    forkFrom: { checkpointId: metadata.parentCheckpointId },
  });
}
```

每次重新生成都会为该位置的 AI 消息创建新路径。

<Tip>
重新生成对于非确定性智能体很有用。由于 LLM 输出会有所不同
使用 temperature，重新生成相同的提示词通常会产生有意义的
不同响应。
</Tip>

## 分支在底层是如何工作的

LangGraph 将每个状态转换持久化为 **检查点**。当您提交
配合 `forkFrom`，后端会从该点开始一条新的执行路径，而不是
追加到当前对话。结果是一个树状结构：

```
User: "What is React?"
  └─ AI: "React is a JavaScript library..." (branch A)
  └─ AI: "React is a UI framework..." (branch B, regenerated)

User: "Tell me about hooks" (branch A)
  └─ AI: "Hooks are functions..."

User: "Tell me about JSX" (edited from branch A)
  └─ AI: "JSX is a syntax extension..."
```

每条路径都保存在检查点存储中。使用
`stream.client.threads.getHistory(threadId)` 当您想构建一个独立的
跨检查点的时间线视图。

## 最佳实践

- **在消息附近读取元数据**：在组件中调用 `useMessageMetadata` 中调用
  渲染消息控件的组件中调用。
- **悬停时显示分支控件**：编辑和重新生成按钮应该
  悬停时出现，以保持 UI 整洁。
- **按需刷新历史记录**：仅在 `client.threads.getHistory()` 仅在
  渲染时间线或分支合并后调用。
- **流式传输时禁用控件**：不要允许编辑或重新生成
  当智能体正在流式传输响应时。检查 `stream.isLoading`
  后再启用这些操作。
- **取消时保留编辑文本**：如果用户开始编辑，然后取消，
  将文本框重置为原始消息内容。
- **使用深层检查点树进行测试**：频繁编辑和重新生成的用户
  可以创建许多路径。确保时间线渲染保持高性能。
