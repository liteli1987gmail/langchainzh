## 设置

要访问 Perplexity 模型，你需要创建一个 Perplexity 账户、获取 API 密钥，并安装 `@langchain/perplexity` 集成包。

### 凭证

前往 [Perplexity API 密钥控制台](https://www.perplexity.ai/account/api/keys) 注册并生成 API 密钥。完成此操作后，设置 `PERPLEXITY_API_KEY` 环境变量：

```bash
export PERPLEXITY_API_KEY="your-api-key"
```

<fragment>

如果您想获取模型调用的自动化追踪，您还可以通过取消下方注释来设置您的 [LangSmith](/langsmith/observability) API 密钥：

</fragment>

```bash
# export LANGSMITH_TRACING="true"
# export LANGSMITH_API_KEY="your-api-key"
```

### 安装

LangChain Perplexity 集成位于 `@langchain/perplexity` 包中：

<CodeGroup>

</CodeGroup>
```bash npm
npm install @langchain/perplexity @langchain/core
```
```bash yarn
yarn add @langchain/perplexity @langchain/core
```
```bash pnpm
pnpm add @langchain/perplexity @langchain/core
```
</CodeGroup>

## 实例化

现在您可以实例化模型：

</fragment>

```typescript
import { ChatPerplexity } from "@langchain/perplexity";

const llm = new ChatPerplexity({
  model: "openai/gpt-5.5",
  temperature: 0,
  maxTokens: undefined,
  timeout: undefined,
  maxRetries: 2,
  // other params...
});
```

## Agent API 支持（`useResponsesApi`）

`ChatPerplexity` 还可以通过设置 `useResponsesApi` 将请求路由到 Perplexity 的 [Agent API](https://docs.perplexity.ai/docs/agent-api/quickstart)（Perplexity 风格的 Responses API）。这与 [`ChatOpenAI` 的 Responses 模式](https://reference.langchain.com/javascript/langchain-openai/) 一致：一套类，两个端点，由单一选项控制。

| 值 | 端点 | 说明 |
| :--- | :--- | :--- |
| `undefined`（默认）| 自动检测 | 当请求使用 Perplexity 内置工具（`web_search`、`fetch_url`、`finance_search`、`people_search`）或包含 Responses 专用字段（`previousResponseId`、`instructions`、`input`、`include`）时路由到 Agent API。否则路由到 Chat Completions。 |
| `true` | Agent API | 始终使用 `client.responses.create()`。 |
| `false` | Chat Completions | 始终使用 `client.chat.completions.create()`。 |

Agent API 为 `ChatPerplexity` 提供了 Perplexity 的内置工具（实时网络搜索、URL 获取、金融和人物搜索）以及有状态的代理字段（`previousResponseId`、`instructions`、`include`），这些在 Chat Completions 上不可用。现有的 `new ChatPerplexity({ model: "sonar" })` 调用不会看到行为变化——Chat Completions 路径仍然是纯文本请求的默认选项。

```typescript
import { ChatPerplexity } from "@langchain/perplexity";

const chat = new ChatPerplexity({
  model: "openai/gpt-5.5",
  useResponsesApi: true,
});

const response = await chat.invoke("What did Apple announce at WWDC this week?");
console.log(response.content);
```

你也可以绑定一个内置工具，并让自动检测路由请求 — 无需设置任何选项：

```typescript
import { ChatPerplexity } from "@langchain/perplexity";

const chat = new ChatPerplexity({ model: "openai/gpt-5.5" });

const response = await chat.invoke(
  "Summarize the latest LangChain release notes.",
  { tools: [{ type: "web_search" }] },
);
console.log(response.content);
```

当通过 Agent API 路由时，响应对象包含更丰富的元数据：

- `usage_metadata` 从 Responses 格式的 `usage` 负载中填充（`input_tokens`、`output_tokens`、`total_tokens`）。
- `response_metadata` 携带传输级字段（`id`、`model`、`status`、`object`），以及存在时来自 Perplexity 的搜索输出：`citations`、`images`、`related_questions` 和 `search_results`。
- `additional_kwargs.responses_output` 保存原始 Agent API 输出项。
- 模型返回的工具调用会在 `response.tool_calls` 上呈现，与 `ChatOpenAI` 的表现完全一致。

<Note>
    `_toResponsesPayload` 转换直接将 `temperature`、`topP` 和 `toolChoice` 传递到 Agent API。Chat-Completions 特有的旋钮（非原生 Responses 字段，例如 `topK`、`stop`、`metadata`）通过 `extra_body` 转发。
</Note>

请参阅 [Perplexity Agent API 模型列表](https://docs.perplexity.ai/docs/agent-api/models) 获取通过此端点可用的完整模型集合（例如 `openai/gpt-5.5`、`anthropic/claude-sonnet-4-6`、`google/gemini-3-1-pro`）。

## 相关集成

`@langchain/perplexity` 包还包含不使用聊天 API 的搜索组件：

- [`PerplexitySearchRetriever`](/oss/integrations/retrievers/perplexity_search)：从 Perplexity Search API 返回 `Document` 对象，用于 RAG 管道

请参阅 [Perplexity 提供商概述](/oss/integrations/providers/perplexity) 了解这三个组件的设置。
