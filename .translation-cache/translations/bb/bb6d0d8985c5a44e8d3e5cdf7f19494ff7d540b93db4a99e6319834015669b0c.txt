以下页面详细介绍了在 [Nuxt 4](https://nuxt.com) 项目中部署 LangChain **深度代理**的示例应用：流式聊天界面、子代理详情视图、线程历史记录和推理令牌流，所有功能都基于实现为 Nitro 路由处理器（HTTP + SSE）的[代理流式协议](https://github.com/langchain-ai/agent-protocol/tree/main/streaming)。无需单独的后端进程。

源代码：[部署烹饪书](https://github.com/langchain-ai/deployment-cookbook/tree/main/js-nuxt) 中的 [`js-nuxt`](https://github.com/langchain-ai/deployment-cookbook/tree/main/js-nuxt)。

## 必需的 API 端点

应用通过 `/api/threads/...` 公开代理流式协议。Nitro 路由处理器位于 `server/api/threads/`。

### 最小配置（流式聊天）

这三个端点足以使用 `@langchain/vue` 的 `HttpAgentServerAdapter` 运行单线程流式聊天：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `POST` | `/api/threads/:threadId/commands` | 接受协议命令（`run.start`，…）并启动代理运行 |
| `POST` | `/api/threads/:threadId/stream` | 运行的协议事件 SSE 流 |
| `GET` / `POST` | `/api/threads/:threadId/state` | 读取和引导检查点线程状态 |

客户端通过 `GET /state` 引导线程（404 时使用 `POST /state`），以便在发送第一条消息之前水合不会 404。

### 可选配置（线程侧边栏）

此示例还实现了线程历史侧边栏的端点。如果你的 UI 不需要多线程管理，可以省略它们：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/api/threads` | 列出检查点已知的所有线程 |
| `DELETE` | `/api/threads/:threadId` | 删除线程的会话和检查点 |
| `POST` | `/api/threads/:threadId/history` | 分页检查点历史（代理协议） |

### 请求流程

1. 引导线程状态（`GET`/`POST /state`）。
2. 提交时，SDK 向 `/commands` 发送 `run.start` 并接收 `run_id`。
3. SDK 订阅 `/stream`（SSE）以获取回放和实时协议事件。
4. 子代理（`task`）运行发出的命名空间事件作为 `stream.subagents` 暴露。

## Nitro 后端设计

| 问题 | 实现 |
| --- | --- |
| 前端 | `app/` 中的 Vue 组件（为 SSE 包装在 `<ClientOnly>` 中） |
| API 层 | `server/api/threads/` 中的 Nitro 路由处理器 |
| 运行时 | Node.js（Nitro 预设取决于部署目标） |
| SSE 回放 | 进程本地的 `LocalThreadSession`（`server/utils/session.ts`） |
| 代理运行 | 同一 Nitro 进程；事件缓存在 LangGraph `StreamChannel` 中 |
| 线程存储 | 内存中的 `MemorySaver` 检查点器（`server/agent/index.ts`） |
| 密钥 | 本地为 `.env`；生产环境为主机环境变量 |

代理的检查点器是线程的唯一真实来源。没有客户端缓存：侧边栏始终从服务器获取，重新启动服务器会清除所有线程。

## 生产环境持久化

开箱即用，代理使用内存中的 `MemorySaver` 检查点器（`server/agent/index.ts`）和进程本地的会话映射（`server/utils/runtime.ts`）。这适用于本地开发和单实例服务器，但在无服务器或多实例主机上，对话状态在冷启动或副本之间**不是持久化的**。

对于生产环境，换用[持久化检查点器](/oss/langgraph/checkpointers#checkpointer-libraries)：

| 包 | 后端 |
| --- | --- |
| [`@langchain/langgraph-checkpoint-redis`](https://www.npmjs.com/package/@langchain/langgraph-checkpoint-redis) | Redis（`RedisSaver`） |
| [`@langchain/langgraph-checkpoint-postgres`](https://www.npmjs.com/package/@langchain/langgraph-checkpoint-postgres) | Postgres（`PostgresSaver`） |
| [`@langchain/langgraph-checkpoint-sqlite`](https://www.npmjs.com/package/@langchain/langgraph-checkpoint-sqlite) | SQLite（`SqliteSaver`） |

替换 `server/agent/index.ts` 中的 `MemorySaver`，并将新的检查点器传递给 `createDeepAgent`。Nitro 路由处理器和 `server/utils/threads.ts` 辅助函数保持不变。

还需要在 `server/utils/runtime.ts` 中设置一个共享的会话/回放存储，以便 SSE 重连在无服务器调用之间正常工作。

更多信息请参阅[检查点器库](/oss/langgraph/checkpointers#checkpointer-libraries)和[添加内存/持久化](/oss/langgraph/add-memory)。

## 本地开发

```bash
cp .env.example .env   # set OPENAI_API_KEY
pnpm install
pnpm dev
```

打开 [http://localhost:3000](http://localhost:3000)。发送一个将任务委托给子代理的提示词，观察他们的工作流式显示在专用卡片中。

```bash
pnpm build      # production build
pnpm preview    # preview the production build
pnpm typecheck  # vue-tsc over the project
```

## 项目布局

<AccordionGroup>

<Accordion title="项目结构">

- `server/agent/` — 深度代理 (`createDeepAgent`)，包含 `researcher` 和 `math-whiz` 子代理、模拟工具和 `stripReasoningReplay` 中间件。
- `server/utils/` — 协议服务器逻辑：`session.ts`（SSE 运行）、`threads.ts`（检查点支持的状态）、`serialize.ts`、`runtime.ts`。
- `server/api/threads/` — 上述协议端点的 Nitro 路由处理程序。
- `app/components/` — 使用 `@langchain/vue` 的 Vue 聊天 UI（`ChatApp`、`Chat`、`ThreadHistory`、`SubagentList`、`MessageReasoning` …）。
- `app/utils/threads.ts` — 服务器驱动的线程辅助函数和 LangGraph SDK 引导。

</Accordion>

<Accordion title="后端详情">

- `server/agent/index.ts` — 协调器通过 Responses API 使用推理模型；使用工具的子代理使用聊天补全（以避免通过检查点重放推理项）。
- `server/agent/middleware.ts` — 从 `content` + `tool_calls` 重建之前的助手消息，确保过时的推理 ID 永远不会被重放到 Responses API。
- `server/utils/session.ts` — `LocalThreadSession` 缓冲协议事件，并通过 `matchesSubscription` 将匹配的帧分发给 SSE。
- `server/api/threads/index.get.ts` — `GET /api/threads`，检查点支持的线程列表。
- `server/api/threads/[threadId]/…` — `commands`、`stream`、`state`（GET/POST）、`history` 和 `DELETE` 的处理程序。

</Accordion>

<Accordion title="前端详情">

- `app/components/ChatThread.vue` — 构建 `HttpAgentServerAdapter` 并调用 `provideStream({ transport, threadId })`。
- `app/components/Chat.vue` — 带有输入框和每个子代理详情视图的消息视图（带面包屑导航）。
- `app/components/SubagentList.vue` / `SubagentDetail.vue` — 内联子代理卡片和作用域子代理聊天（`useMessages` 绑定到命名空间）。
- `app/components/MessageReasoning.vue` — 可折叠的推理摘要"思考"块。

</Accordion>

</AccordionGroup>
