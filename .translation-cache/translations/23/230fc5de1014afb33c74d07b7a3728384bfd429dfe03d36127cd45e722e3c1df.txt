您可以[公开分享跟踪](#share-a-trace)，并[查看](#view-server-logs)与跟踪执行关联的服务器日志。

## 分享跟踪

<Warning>
**公开分享跟踪会让任何拥有链接的人都能访问它。请确保您没有分享敏感信息。**

如果您的 [self-hosted](/langsmith/self-hosted) LangSmith 部署在 VPC 内，则该公开链接仅供通过 VPC 身份验证的成员访问。为增强安全性，建议将实例配置为仅供有权访问您网络的用户访问的私有 URL。
</Warning>

公开分享跟踪：

1. 在 [UI](https://smith.langchain.com) 中打开任何跟踪。
2. 点击 Details 视图顶部的更多 <Icon icon="dots-vertical" /> 菜单中的 **Share** 按钮。
3. 在出现的对话框中，复制公开链接。

拥有链接的任何人都可以访问分享的跟踪，无需 LangSmith 账户。他们可以查看跟踪但无法编辑。

取消分享跟踪，请使用以下任一方法：

1. 打开已分享的跟踪，点击 Details 视图顶部工具栏中的 **Public**，然后在对话框中点击 **Unshare**。
2. 进入 **Settings** → **Shared URLs** 查看所选工作区中所有公开分享的跟踪。点击要取消分享的跟踪旁边的 **Unshare**。

## 查看服务器日志

<Note>
查看跟踪的服务器日志仅适用于 [Cloud SaaS](/langsmith/cloud) 和 [完全自托管](/langsmith/self-hosted) 部署选项。
</Note>

当查看由 LangSmith 中运行生成的跟踪时，您可以直接从 Details 视图访问关联的服务器日志。

在 Details 视图中，使用右上角的 **See Logs** 按钮，该按钮位于 **Run in Studio** 按钮旁边。

点击此按钮将跳转到 LangSmith 中关联部署的服务器日志视图。

服务器日志视图显示以下两部分的日志：

- **Agent Server 自身的操作日志**：内部服务器操作、API 调用和系统事件
- **用户应用程序日志**：在您的图中编写的日志：
  - Python：使用 `logging` 或 `structlog` 库。
  - JavaScript：使用从 `@langchain/langgraph-sdk/logging` 重新导出的 Winston 日志记录器：

    ```javascript
    import { getLogger } from "@langchain/langgraph-sdk/logging";

    const logger = getLogger();
    logger.info("Your log message");
    ```

当从详情视图导航时，**筛选器**框会自动预填您刚刚查看的跟踪的 Trace ID，这样您可以快速筛选日志，仅查看与该特定跟踪执行相关的日志。

## 删除跟踪

如果您需要在 LangSmith 中的跟踪过期之前将其移除，可以删除整个项目或删除特定的跟踪。

### 删除整个项目

- 在 [LangSmith UI](https://smith.langchain.com) 中，选择项目溢出菜单上的**删除**选项。
- 使用 [`delete_tracer_sessions`](/langsmith/smith-api/tracer-sessions/delete-tracer-session) API 端点。
- 使用 LangSmith SDK 中的 `delete_project()` ([Python](https://reference.langchain.com/python/langsmith/observability/sdk/)) 或 `deleteProject()` ([JS/TS](https://reference.langchain.com/javascript/modules/langsmith.html))。

### 删除特定的跟踪

使用 [`delete_runs`](/langsmith/smith-api/run/delete-runs) API 端点通过跟踪 ID 或元数据键值对删除运行。请求体接受：

- `session_id`：将删除范围限定在特定项目。
- `trace_ids`：要删除的跟踪 ID 列表。
- `metadata`：删除与给定元数据键值对匹配的所有运行。

有关完整的 API 使用，包括代码示例、每请求 1000 条跟踪的限制、删除时间线以及元数据匹配行为，请参阅[合规性数据清除](/langsmith/data-purging-compliance#trace-deletes)。
