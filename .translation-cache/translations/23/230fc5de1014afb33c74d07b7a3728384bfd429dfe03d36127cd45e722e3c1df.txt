## 分享跟踪

<Warning>
**公开分享跟踪将使任何拥有链接的人都可以访问它。请确保您没有分享敏感信息。**

如果您的 [self-hosted](/langsmith/self-hosted) LangSmith 部署在 VPC 内，则公开链接仅对在 VPC 内通过身份验证的成员可访问。为增强安全性，我们建议将您的实例配置为使用私有 URL，仅供可访问您网络的用户使用。
</Warning>

公开分享跟踪的方法：

1. 在 [UI](https://smith.langchain.com).
1. 中打开任意跟踪，点击 **分享** 按钮在详情视图顶部的 <Icon icon="dots-vertical" /> 更多菜单中。
1. 在出现的对话框中，复制公开链接。

拥有链接的任何人都可以查看分享的跟踪，即使没有 LangSmith 账户。他们可以查看跟踪但不能编辑。

取消分享跟踪的方法：

1. 打开分享的跟踪，点击 **公开** 在详情视图顶部的工具栏中，然后点击 **取消分享** 在对话框中。
1. Go to **设置** → **分享的链接** 查看所选工作区中所有公开分享的跟踪。点击 **取消分享** 在要取消分享的跟踪旁边。

## 查看服务器日志

<Note>
查看跟踪的服务器日志仅适用于 [云 SaaS](/langsmith/cloud) 和 [完全自托管](/langsmith/self-hosted) 部署选项。
</Note>

当查看由 LangSmith 中运行生成的跟踪时，您可以直接从详情视图访问相关的服务器日志。

在详情视图中，使用 **查看日志** 右上角的按钮，紧邻 **在 Studio 中运行** button.

点击此按钮将带您进入 LangSmith 中关联部署的服务器日志视图。

服务器日志视图显示以下两个来源的日志：

- **Agent Server 自身的操作日志**：内部服务器操作、API 调用和系统事件
- **用户应用日志**：在您的图中编写的日志，使用：
  - Python：使用 `logging` or `structlog` libraries.
  - JavaScript：使用从重新导出的 Winston logger `@langchain/langgraph-sdk/logging`:

    ```javascript
    import { getLogger } from "@langchain/langgraph-sdk/logging";

    const logger = getLogger();
    logger.info("Your log message");
    ```

当您从详情视图导航时， **筛选器** 框会自动预先填充您刚才查看的追踪的追踪 ID，因此您可以快速筛选日志，只查看与特定追踪执行相关的日志。

![部署服务器日志筛选器](/langsmith/images/lgp-server-logs-filters.png)

## 删除追踪

如果您需要在追踪到期日之前从 LangSmith 中移除追踪，可以删除整个项目或删除特定追踪。

### 删除整个项目

- 在 [LangSmith UI](https://smith.langchain.com)中，选择 **删除** 项目溢出菜单上的选项。
- 使用 [`delete_tracer_sessions`](/langsmith/smith-api/tracer-sessions/delete-tracer-session) API 端点。
- 使用 `delete_project()` ([Python](https://reference.langchain.com/python/langsmith/observability/sdk/)) or `deleteProject()` ([JS/TS](https://reference.langchain.com/javascript/modules/langsmith.html)) 在 LangSmith SDK 中。

### 删除特定追踪：

使用 [`delete_runs`](/langsmith/smith-api/run/delete-runs) API 端点按追踪 ID 或元数据键值对删除运行。请求正文接受：

- `session_id`：将删除范围限定到特定项目。
- `trace_ids`：要删除的追踪 ID 列表。
- `metadata`：删除与给定元数据键值对匹配的所有运行。

有关完整的 API 使用方法，包括代码示例、每请求 1000 条追踪的限制、删除时间表和元数据匹配行为，请参阅 [合规性数据清理](/langsmith/data-purging-compliance#trace-deletes).
