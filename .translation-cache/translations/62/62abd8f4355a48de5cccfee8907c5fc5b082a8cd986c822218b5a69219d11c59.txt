[聊天模型](/oss/langchain/models) 是使用一系列 [消息](/oss/langchain/messages) 作为输入并返回消息作为输出 <Tooltip tip="Older models that do not follow the chat model interface and instead use an interface that takes a string as input and returns a string as output. These models typically do not include the prefix 'Chat' in their name or include 'LLM' as a suffix.">（与纯文本相反）</Tooltip>.

## 安装和使用

<Tip>
    请参阅 [本节了解安装 LangChain 包的一般说明](/oss/langchain/install).
</Tip>

<AccordionGroup>
    <Accordion title="OpenAI">
        Install:

        <CodeGroup>
            ```bash npm
            npm install @langchain/openai @langchain/core
            ```
            ```bash yarn
            yarn add @langchain/openai @langchain/core
            ```
            ```bash pnpm
            pnpm add @langchain/openai @langchain/core
            ```
        </CodeGroup>

        添加环境变量：

        ```bash
        OPENAI_API_KEY=your-api-key
        ```

        实例化模型：

        ```typescript
        import { ChatOpenAI } from "@langchain/openai";

        const model = new ChatOpenAI({ model: "gpt-5.4-mini" });
        ```
        ```javascript
        await model.invoke("Hello, world!")
        ```
    </Accordion>

## 路由器和代理

路由器和代理让您通过单一 API 和凭证访问来自多个提供商的模型。它们可以简化计费，让您在不更改集成的情况下切换模型，并提供自动故障转移等功能。

| 提供商 | 集成 | 描述 |
|-|-|-|
| [OpenRouter](https://openrouter.ai/) | [`ChatOpenRouter`](/oss/integrations/chat/openrouter) | 通过统一接口访问 OpenAI、Anthropic、Google、Meta 等模型 |

## 聊天补全 API

某些模型提供商提供与 OpenAI（遗留版本）的 [聊天补全 API](https://platform.openai.com/docs/guides/completions)兼容的端点。在这种情况下，您可以使用 [`ChatOpenAI`](/oss/integrations/chat/openai) 和自定义 `base_url` 连接到这些端点。请注意，基于聊天补全 API 构建的功能可能无法被 `ChatOpenAI`完全支持；在这种情况下，请考虑使用提供商特定的类（如果有）。
