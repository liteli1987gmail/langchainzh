文档加载器提供了一个 **标准接口** 用于将来自不同来源（如 Slack、Notion 或 Google Drive）的数据读取到 LangChain 的 @[Document] 格式中。
这确保了数据可以始终如一地处理，无论其来源如何。

所有文档加载器都实现了 @[BaseLoader] 接口。

<Warning>
社区文档加载器由用户贡献且未经验证。LangChain 不会审查或认可这些集成；使用风险自负。
</Warning>

## 接口

每个文档加载器可以定义自己的参数，但它们共享一个通用 API：

- `load()`：一次性加载所有文档。
- `loadAndSplit()`：一次性加载所有文档并将其分割成更小的文档。

```typescript
import { OracleDocLoader } from "@oracle/langchain-oracledb";

const loader = new OracleDocLoader(,
  ...  // <-- Integration specific parameters here
);
const data = await loader.load();
```

## 按类别

LangChain.js 以两种不同方式对文档加载器进行分类：

- [文件加载器](/oss/integrations/document_loaders/file_loaders/)，用于从本地文件系统将数据加载到 LangChain 格式中。
- [网络加载器](/oss/integrations/document_loaders/web_loaders/)，用于从远程来源加载数据。

### 文件加载器

    如果您想贡献集成，请参阅 [贡献集成](/oss/contributing#add-a-new-integration).
</Info>

#### 常见文件类型

| Document Loader | Description | Package/API |
|----------------|-------------|-------------|
| [`DirectoryLoader`](/oss/integrations/document_loaders/file_loaders/directory) | 使用自定义加载器映射从目录加载所有文件 | 包 |
| [JSON](/oss/integrations/document_loaders/file_loaders/json) | 使用 JSON 指针加载 JSON 文件以定位特定键 | 包 |
| [`JSONLines`](/oss/integrations/document_loaders/file_loaders/jsonlines) | Load data from JSONLines/JSONL files | Package |
| [`Text`](/oss/integrations/document_loaders/file_loaders/text) | 加载纯文本文件 | 包 |

#### 专用文件加载器

| Document Loader | Description | Package/API |
|----------------|-------------|-------------|
| [`MultiFileLoader`](/oss/integrations/document_loaders/file_loaders/multi_file) | 从多个单独的文件路径加载数据 | 包 |
| [`OracleDocLoader`](/oss/integrations/document_loaders/file_loaders/oracleai) | 从 Oracle AI Vector Search 表或 Oracle Text 支持的文件中提取数据 | 包 |
