文档加载器提供了一个**标准接口**，用于从不同的数据源（如 Slack、Notion 或 Google Drive）读取数据并转换为 LangChain 的 @[Document] 格式。
这确保了无论数据来源如何，都能以一致的方式处理数据。

所有文档加载器都实现了 @[BaseLoader] 接口。

<Warning>
社区文档加载器由用户贡献且未经核实。LangChain 不会审查或认可这些集成；使用风险自负。
</Warning>

## 接口

每个文档加载器可以定义自己的参数，但它们共享一个通用 API：

- `load()`：一次性加载所有文档。
- `loadAndSplit()`：一次性加载所有文档，并将其分割成更小的文档。

```typescript
import { OracleDocLoader } from "@oracle/langchain-oracledb";

const loader = new OracleDocLoader(,
  ...  // <-- Integration specific parameters here
);
const data = await loader.load();
```

## 按类别

LangChain.js 以两种不同的方式对文档加载器进行分类：

- [文件加载器](/oss/integrations/document_loaders/file_loaders/)，从本地文件系统将数据加载到 LangChain 格式。
- [Web 加载器](/oss/integrations/document_loaders/web_loaders/)，从远程源加载数据。

### 文件加载器

<Info>
    如果您想贡献一个集成，请参阅[贡献集成](/oss/contributing#add-a-new-integration)。
</Info>

#### 常见文件类型

| 文档加载器 | 描述 | 包/API |
|----------------|-------------|-------------|
| [`DirectoryLoader`](/oss/integrations/document_loaders/file_loaders/directory) | 使用自定义加载器映射从目录加载所有文件 | Package |
| [JSON](/oss/integrations/document_loaders/file_loaders/json) | 使用 JSON pointer 定位特定键来加载 JSON 文件 | Package |
| [`JSONLines`](/oss/integrations/document_loaders/file_loaders/jsonlines) | 从 JSONLines/JSONL 文件加载数据 | Package |
| [`Text`](/oss/integrations/document_loaders/file_loaders/text) | 加载纯文本文件 | Package |

#### 专用文件加载器

| 文档加载器 | 描述 | 包/API |
|----------------|-------------|-------------|
| [`MultiFileLoader`](/oss/integrations/document_loaders/file_loaders/multi_file) | 从多个独立的文件路径加载数据 | Package |
| [`OracleDocLoader`](/oss/integrations/document_loaders/file_loaders/oracleai) | 引入 Oracle AI Vector Search 表或 Oracle Text 支持的文件 | Package |
