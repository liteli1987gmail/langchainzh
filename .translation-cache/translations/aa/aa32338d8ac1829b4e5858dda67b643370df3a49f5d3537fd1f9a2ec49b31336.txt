Deep Agents Code 支持任意[与 LangChain 兼容的聊天模型提供商](/oss/integrations/chat)，解锁了对几乎所有支持工具调用的 LLM 的使用。任何公开 OpenAI 兼容或 Anthropic 兼容 API 的服务也可以开箱即用——请参阅[兼容的 API](/oss/deepagents/code/configuration#compatible-apis)。

## 快速开始

Deep Agents Code 会自动集成[以下模型提供商](#provider-reference)：除了安装相关提供商包外，无需额外配置。

1. **安装提供商包**

    每个模型提供商都需要其对应的 LangChain 集成包。这些包作为可选依赖项发布，以保持应用程序轻量化。OpenAI、Anthropic 和 Gemini 默认包含。使用 `/install` 在会话中安装任何其他依赖项，或使用 `dcode --install` 在 shell 中安装：

    <CodeGroup>
        ```txt In session
        /install groq
        ```

        ```bash Shell
        dcode --install groq
        ```
</CodeGroup>

运行 `/install` 不带参数运行以列出有效的额外组件。若要在初始 CLI 安装期间预装额外组件，请设置 `DEEPAGENTS_CODE_EXTRAS`：

    ```bash
    DEEPAGENTS_CODE_EXTRAS="baseten,groq" curl -LsSf https://langch.in/dcode | bash
    ```

2. **设置凭据**

    使用 [`/auth`](/oss/deepagents/code/configuration#use-%2Fauth-recommended) 凭据管理器为您的提供商添加 API 密钥：

    ```txt
    /auth
    ```

对于非交互式运行、CI/CD 或没有 TUI 的环境，请使用 [`dcode auth set`](/oss/deepagents/code/configuration#manage-credentials-from-the-shell-dcode-auth) 从 shell 中存储相同的密钥，或改用提供商的 environment variable 变量。详见 [Provider credentials](/oss/deepagents/code/configuration#provider-credentials) 获取完整的密钥解析顺序、[`DEEPAGENTS_CODE_` 前缀](/oss/deepagents/code/configuration#deepagents_code_-prefix) 用于将密钥限定在 Deep Agents Code 范围内，以及 [Provider reference](#provider-reference) 中各提供商的 environment variable 变量。

要配置模型参数，请参阅 [Model parameters](#model-parameters)。

### Sign in with ChatGPT

`openai_codex` 提供商允许您使用付费 **ChatGPT** 订阅而非 `OPENAI_API_KEY` 来使用 OpenAI 的 Codex 模型。您使用 ChatGPT 账户登录后，它会在 `/auth` 和 `/model` 切换器中显示为独立的提供商，与基于 API 密钥的 `openai` 提供商分开。

<Steps>
    <Step title="Start the sign-in">
        在任何会话中运行 `/auth` 并选择 **`openai_codex`**。由于 ChatGPT 通过浏览器让您登录，这会启动浏览器登录流程，而非要求输入 API 密钥。
    </Step>

    <Step title="Authorize in your browser">
        Deep Agents Code 会打开您的浏览器进入 ChatGPT 登录页面。如果无法打开浏览器（例如通过 SSH），它也会在屏幕上显示登录 URL，以便您可以复制到另一台设备的浏览器中。
    </Step>

    <Step title="Select a Codex model">
        登录后，Codex 模型会出现在 `/model` 切换器的 `openai_codex` 提供商下。您可以通过其 spec 直接切换到其中一个：

</Steps>

        ```txt
        /model openai_codex:gpt-5.5
        ```
</Step>
</Steps>

您的登录状态会在会话之间保持。要查看状态或退出登录，请运行 `/auth`，选择 `openai_codex`，然后选择重新认证或退出登录。

<Note>
    `openai_codex` 与 `openai` 是独立的。如果要使用标准 API 密钥调用 OpenAI 模型，请使用常规的 `openai` 提供商（例如 `/model openai:gpt-5.5`）。
</Note>

:::

## 切换模型

要在 Deep Agents Code 中切换模型，请执行以下操作之一：

1. **使用交互式模型切换器** - 使用 `/model` 命令。

    <Note>
        并非所有模型都会在此处显示。如果您的模型不在列表中，请直接传递模型名称（例如 `/model gpt-5.5`），或将其添加到 `config.toml` 中。
    </Note>
2. **直接指定模型名称** 作为参数，例如 `/model gpt-5.5`。您可以使用所选提供商支持的任何模型，无论其是否出现在选项 1 的列表中。模型名称将传递给 API 请求。
3. **在启动时指定模型** 通过 `--model`，例如

    ```txt
    dcode --model openai:gpt-5.5
    ```

<Accordion title="模型解析顺序" icon="list-numbers">
    Deep Agents Code 启动时，会按以下顺序确定要使用的模型：

    1. **`--model` 标志**——如果提供，始终优先。
    2. **`[models].default`** 在 `~/.deepagents/config.toml` 中——用户的长期偏好。
    3. **`[models].recent`** 在 `~/.deepagents/config.toml` 中——最近通过 `/model` 切换的模型。自动写入；不会覆盖 `[models].default`。
    4. **环境自动检测**：回退到第一个可用的启动凭证，按顺序检查：`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GOOGLE_API_KEY`、`GOOGLE_CLOUD_PROJECT`（Vertex AI）。

    此启动回退仅检查这四个凭证。其他支持的提供商（例如 Groq）仍可通过 `--model`、`/model` 和保存的默认值（`[models].default` / `[models].recent`）使用。
</Accordion>

### 切换器中显示哪些模型

`/model` 选择器根据已安装的提供商包动态构建其列表。展开下方查看完整标准和故障排除方法。

<Accordion title="切换器如何构建其模型列表" icon="list-search">
    交互式 `/model` 选择器根据已安装的提供商包和 `config.toml` 中配置的模型构建其列表。

    模型在以下情况下会显示：

    1. 提供商包已安装。
    2. 模型可从提供商包、本地提供商或您的 `config.toml` 获取。
    3. 模型配置未将文本输入或输出标记为不支持。

    如果某个模型缺失，可直接使用 `/model <provider>:<model>` 或将其添加到 [`[models.providers.<name>].models`](/oss/deepagents/code/configuration#adding-models-to-the-interactive-switcher)。

    <Tip>
        凭证状态**不会**影响模型是否被列出。您仍然可以选择凭证缺失的模型。提供商会在请求时报告身份验证错误。
    </Tip>
</Accordion>

### 开源模型

如果您想使用开源模型，根据您偏好本地推理还是云端托管推理，有两条常用路径。

**使用 Ollama 进行本地推理**是最简单的免费入门方式，无需 API 密钥：

1. [安装 Ollama](https://ollama.com/) 并拉取一个模型，例如：

    ```bash
    ollama pull qwen3:4b
    ```

2. 安装 Ollama 扩展：

<CodeGroup>
        ```txt In session
        /install ollama
        ```

        ```bash Shell
        dcode --install ollama
        ```
</CodeGroup>

3. 选择模型：

<CodeGroup>
        ```txt In session
        /model
        ```

        ```bash Shell
        dcode --model ollama:qwen3:4b
        ```
</CodeGroup>

    使用交互式切换器，或直接通过 `/model ollama:qwen3:4b` 传递模型。

**通过 Groq 云托管的开源模型**让您无需本地运行任何内容即可获得快速推理：

1. 在以下网址获取免费 API 密钥：[console.groq.com](https://console.groq.com/)。

2. 安装 Groq 扩展包：

    <CodeGroup>

</fragment>
        ```txt In session
        /install groq
        ```

        ```bash Shell
        dcode --install groq
        ```
</CodeGroup>

3. 选择模型：

    <CodeGroup>
        ```txt In session
        /model
        ```

        ```bash Shell
        GROQ_API_KEY="your-api-key" dcode --model groq:openai/gpt-oss-120b
        ```
</CodeGroup>

使用交互式切换器，或直接使用 `/model groq:openai/gpt-oss-120b` 传递模型。

**Fireworks** 是另一个支持开源模型的热门云服务提供商：

<CodeGroup>
    ```txt In session
    /install fireworks
    /model
    ```

    ```bash Shell
    dcode --install fireworks
    FIREWORKS_API_KEY="your-api-key" dcode --model fireworks:accounts/fireworks/models/deepseek-v4-pro
    ```
</CodeGroup>

使用交互式切换器，或通过 `/model fireworks:accounts/fireworks/models/deepseek-v4-pro` 直接传递模型。

**Baseten** 是另一个为开源权重模型提供支持的云服务提供商：

<CodeGroup>
    ```txt In session
    /install baseten
    /model
    ```

    ```bash Shell
    dcode --install baseten
    BASETEN_API_KEY="your-api-key" dcode --model baseten:moonshotai/Kimi-K2.7-Code
    ```
</CodeGroup>

使用交互式切换器，或直接通过 `/model baseten:moonshotai/Kimi-K2.7-Code` 传递模型。

<Tip>
    如果你希望在安装 CLI 的同时预装提供商，请在初始安装期间使用 `DEEPAGENTS_CODE_EXTRAS`：

</fragment>

    ```bash
    DEEPAGENTS_CODE_EXTRAS="fireworks" curl -LsSf https://langch.in/dcode | bash
    ```

你可以组合多个提供商：`DEEPAGENTS_CODE_EXTRAS="groq,fireworks,ollama"`。如果已安装 Deep Agents Code，请在会话中使用 `/install <extra>`，或从 shell 中使用 `dcode --install <extra>`。

**Together**、**OpenRouter** 和 **Hugging Face**（`langchain-huggingface`）是云托管开源权重的其他选项。请参阅[提供商参考](#provider-reference)获取凭据和包名。

### 设置默认模型

你可以设置一个持久化的默认模型，它将应用于所有未来的 CLI 启动：

* **通过模型选择器：** 打开 `/model`，导航到所需模型，然后按 `Ctrl+S` 将其固定为默认。再次在当前默认模型上按 `Ctrl+S` 可清除它。
* **通过命令：** `/model --default provider:model`（例如 `/model --default anthropic:claude-opus-4-8`）
* **通过配置文件：** 在 `~/.deepagents/config.toml` 中设置 `[models].default`（参见[配置](/oss/deepagents/code/configuration)）。
* **从 shell：**

    ```bash
    dcode --default-model anthropic:claude-opus-4-8
    ```

查看当前默认值：

```bash
dcode --default-model
```

要清除默认值：

* **从 Shell：**

    ```bash
    dcode --clear-default-model
    ```

* **通过命令：** `/model --default --clear`
* **通过模型选择器：** 在当前固定的默认模型上按 `Ctrl+S`。

如果没有设置默认模型，Deep Agents Code 会使用最近使用的模型。

### 模型参数

向模型传递额外的构造函数 kwargs——采样控制、推理/思考预算、上下文窗口大小、请求超时，以及底层聊天模型类接受的任何其他参数。设置它们的三个位置，按优先级排序（最高优先）：

1. **启动时一次性设置 `--model-params`。** JSON 字符串，仅限当前会话：

    ```bash
    # OpenAI reasoning effort
    dcode --model openai:gpt-5.5 --model-params '{"reasoning": {"effort": "high"}}'

    # Anthropic extended thinking
    dcode --model anthropic:claude-opus-4-8 --model-params '{"thinking": {"type": "enabled", "budget_tokens": 10000}, "max_tokens": 16000}'
    ```

2. **通过 `/model --model-params` 进行会话中切换。** 相同的 JSON 语法——在不需要重启的情况下切换参数（以及可选的模型）：

    ```txt
    /model --model-params '{"temperature": 0.7}' anthropic:claude-opus-4-8
    /model --model-params '{"num_ctx": 16384}'           # opens selector, applies params to choice
    ```

3. **持久化在 `config.toml` 中。** 在每次启动时应用的 Provider 级别默认值（包含可选的 per-model 子表）：

    ```toml
    [models.providers.anthropic.params]
    thinking = { type = "enabled", budget_tokens = 10000 }
    max_tokens = 16000

    [models.providers.openai.params]
    reasoning = { effort = "high", summary = "auto" }
    output_version = "responses/v1"

    [models.providers.ollama.params]
    num_ctx = 16384
    temperature = 0

    # Per-model override—wins over provider-level keys
    [models.providers.ollama.params."qwen3:4b"]
    temperature = 0.5
    ```

CLI 标志会覆盖配置文件中的 `params`，且仅在当前会话有效（会话中途的更改不会持久化）。`config.toml` 中每个模型对应的子表会覆盖 provider 级别的键（浅层合并——详见[模型构造函数参数](/oss/deepagents/code/configuration#model-constructor-params)了解完整语义）。`--model-params` 不能与 `--default` 组合使用。

对于重试次数，请优先使用 `--max-retries` 或顶层 `[retries]` 配置。

<Tip>
    底层聊天模型构造函数接受的任何 kwarg 都有效。具体支持的参数请参考各 provider 的参考文档——例如 [`ChatAnthropic`](https://reference.langchain.com/python/langchain-anthropic/langchain_anthropic/chat_models/ChatAnthropic)、[`ChatOpenAI`](https://reference.langchain.com/python/langchain-openai/langchain_openai/chat_models/base/ChatOpenAI)、[`ChatOllama`](https://reference.langchain.com/python/langchain-ollama/langchain_ollama/chat_models/ChatOllama)。未识别的 kwargs 会被直接转发到上游 API 请求，因此新发布的参数无需更新 CLI 即可使用。
</Tip>

<Note>
    请勿将凭证（`api_key`）放在 `params` 中——请使用 [`api_key_env`](/oss/deepagents/code/configuration#provider-configuration) 指向环境变量。
</Note>

要在模型的运行时 *profile*（`max_input_tokens`、`tool_calling`、功能标志）上覆盖字段（这与构造函数参数不同），请参阅[Profile 覆盖](/oss/deepagents/code/configuration#profile-overrides-advanced)。
