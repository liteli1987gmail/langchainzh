LangSmith 提供了多种工具来帮助您有效地管理您的[_提示词_](/langsmith/prompt-engineering-concepts)。本页面介绍以下功能：

- [环境](#environments)用于通过 **Staging** 和 **Production** 提升提交。
- [提交标签](#commit-tags)用于版本控制和环境管理。
- [提示词所有者](#prompt-owners)用于控制谁可以提升提交和删除提示词。
- [Webhook 触发器](#trigger-a-webhook-on-prompt-commit)用于在提示词更新时自动化工作流程。
- [公共提示词中心](#public-prompt-hub)用于发现和使用社区创建的提示词。

## 提示词详情页

从[**提示词**表](/langsmith/create-a-prompt#view-your-prompts)中选择一个提示词以打开其详情页，该页面采用双栏布局：左侧显示提交历史和环境，右侧显示提交详情。

您可以通过切换右上角的 **Diff** 来将提交与其之前的版本进行比较。

## 环境

环境代表命名的部署目标，**Staging** 和 **Production**，您可以将特定的提交分配给这些目标。它们允许您跟踪每个环境中哪个版本的提示词处于活跃状态，并在不同环境之间提升提交。

环境由保留的[提交标签](#commit-tags)（`staging` 和 `production`）定义，这些标签通过提升 UI 管理，而不是通过自由格式的标签选择器管理。

### 提升提交

提升提交将其分配给某个环境。您可以将任何提交提升到 Staging 或 Production。

要提升提交：

1. 将鼠标悬停在左侧窗格中的提交上以显示 **Promote**，或点击页面右上角的 **Promote**。从下拉列表中选择 **Staging** 或 **Production**。
2. 打开一个部署模态框，显示当前分配给该环境的提交以及将被替换的提交。
3. 确认提升。环境指针会立即更新。

<Note>
将提交提升到 Production 不会将其从 Staging 中移除。如果一个提交已在 Staging 中且您将其提升到 Production，它仍然会保留在 Staging 中。
</Note>

### 回滚环境

每个环境都维护一个有序的历史记录，记录哪些提交被分配到该环境以及何时分配。要回滚到之前的提交：

1. 在左侧窗格中，找到您想要回滚的环境。
2. 点击该环境的回滚图标。
3. 从显示的 **回滚历史记录** 中，选择您想要回滚到的提交。环境指针将更新到该提交。

## 提交标签

[_提交标签_](/langsmith/prompt-engineering-concepts#tags) 是引用提示词版本历史中特定[_提交_](/langsmith/prompt-engineering-concepts#commits)的标签。它们帮助您标记重要版本，并控制哪些版本在不同的环境中运行。通过在代码中引用标签而不是提交 ID，您可以更新正在使用的版本而无需修改代码本身。

每个标签恰好引用一个提交，但您可以将标签重新分配给不同的提交。

<Note>
**保留标签：** `staging` 和 `production` 标签保留用于环境管理，在自由格式标签选择器中不可用。使用[提升流程](#promote-a-commit)将提交分配给这些环境。
</Note>

<Note>
**不要与资源标签混淆**：提交标签特定于提示词版本控制，引用提示词历史中的单个提交。[资源标签](/langsmith/set-up-resource-tags)是用于组织工作区资源（如项目、数据集和提示词）的键值对。虽然两者可以使用相似的命名约定（如 `prod` 或 `staging`），但提交标签控制提示词的**哪个版本**运行，而资源标签帮助您**组织和筛选**整个工作区中的资源。
</Note>

### 创建标签

要创建标签，请在提示词详情页的左侧窗格中选择您想要打标签的提交。点击右侧窗格右上角的 **Tag**。在下拉列表中，点击 **Commit Tag** 并输入名称。

### 移动标签

要将标签指向不同的提交，请在提示词详情页的左侧窗格中选择目标提交。点击右侧窗格右上角的 **Tag**。在下拉列表中，选择您想要移动的标签。这会自动更新标签以指向新的提交。

### 删除标签

要删除标签，请点击右侧窗格右上角的 **Tag**。（选择哪个提交无关紧要）。在下拉列表中，点击要删除的标签旁边的删除图标。这将完全移除该标签，它将不再与任何提交关联。

### 在代码中使用标签

标签提供了一种稳定的方式来引用代码中提示词的特定版本。与其直接使用提交哈希，不如引用可以更新而无需更改代码的标签。

以下是在 Python 中按标签拉取提示词的示例：

```python
prompt = client.pull_prompt("joke-generator:production")
# If production tag points to commit a1b2c3d4, this is equivalent to:
prompt = client.pull_prompt("joke-generator:a1b2c3d4")
```

有关如何在代码中使用提示的更多信息，请参阅[以编程方式管理提示](/langsmith/manage-prompts-programmatically)。

## 提示词所有者

提示词所有者功能让您可以精细控制谁可以为提交添加标签和删除特定提示词。这在生产推广流程中非常有用，您希望限制哪些团队成员可以通过分配或移动标签来将提交推广到某个环境。

### 访问模式

每个提示词有两个访问模式，在 [UI](https://smith.langchain.com) 的 **访问和权限** 下配置：

- **工作区授权用户**（默认）：任何具有 [`prompts:tag`](/langsmith/organization-workspace-operations#prompts) 权限的工作区用户都可以创建、更新和删除标签，以及删除提示词。
- **仅所有者**：只有被添加为提示词所有者的用户才能创建或更新提交标签、将提交推广到环境，以及删除提示词。

LangSmith 会自动将提示词创建者添加为所有者。

### 配置访问和权限

1. 在 LangSmith UI 中打开提示词。
2. 点击右上角的 <Icon icon="dots-vertical"/> **更多** 图标，选择 **访问和权限**。
3. 选择 **仅所有者** 模式。
4. 添加或从所有者组中移除用户。

<Warning>
如果您保存了移除自己作为所有者的更改，您将失去管理所有者或将提示词切换回工作区授权用户模式的能力。只有另一个所有者可以恢复您的访问权限。
</Warning>

当 **仅所有者** 模式处于激活状态时，只有所有者可以禁用它或添加或移除其他所有者。

## 在提示词提交时触发 Webhook

您可以配置 webhook，以便在向提示词提交代码时触发。

常见的用例包括：

* 在提示词更新时触发 CI/CD 管道。
* 将提示词与 GitHub 仓库同步。
* 通知团队成员有关提示词的修改。

### 配置 Webhook

在左侧边栏或应用程序主页中导航到 **提示词** 部分。点击右上角的 `+ Webhook` 按钮。

添加 Webhook URL 和任何必需的请求头。

<Note>
每个工作区只能配置一个 webhook。如果您想为每个工作区配置多个 webhook 或为每个提示词设置不同的 webhook，请在 [LangChain 论坛](https://forum.langchain.com/) 告诉我们。
</Note>

要测试 webhook，请点击 **发送测试通知** 按钮。这将向您提供的 webhook URL 发送一个带有示例负载的测试通知。

示例负载是一个包含以下字段的 JSON 对象：

- `prompt_id`：被提交的提示词的 ID。
- `prompt_name`：被提交的提示词的名称。
- `commit_hash`：提示词的提交哈希。
- `created_at`：提交的日期。
- `created_by`：提交的作者。
- `manifest`：提示词的清单。

### 触发 Webhook

向提示词提交代码以触发您配置的 webhook。

#### 使用 Playground

如果您在 Playground 中进行操作，系统会提示您取消选择您想避免触发的 webhook。

![Commit Playground](/langsmith/images/commit-playground.png)

#### 使用 API

如果您通过 API 提交，可以通过将 `skip_webhooks` 参数设置为 `true` 或要忽略的 webhook ID 数组来指定跳过触发 webhook。更多信息请参阅 [API 文档](/langsmith/smith-api/commits/create-a-commit)。

## 公共提示词中心

LangSmith 的公共提示词中心是 LangChain 社区创建的提示词集合，您可以将这些提示词用作参考。

<Note>
请注意，提示词由用户生成且未经审核。LangChain 不审查或认可公共提示词，使用风险自负。提示词中心的使用受我们的[服务条款](https://www.langchain.com/terms-of-service)约束。
</Note>

导航到左侧边栏的 **提示词** 部分，点击 **浏览 LangChain 中心的所有公共提示词**。

在这里您可以找到 LangChain 中心中所有公开列出的提示词。您可以按名称、句柄、用例、描述或模型搜索提示词。您可以将提示词分叉到您的个人组织、查看提示词的详细信息，以及在 Playground 中运行提示词。您可以使用 SDK[将任何公共提示词拉取到您的代码中](/langsmith/manage-prompts-programmatically)。

要查看与您工作区关联的提示词，请导航到侧边栏中的 **提示词**。
