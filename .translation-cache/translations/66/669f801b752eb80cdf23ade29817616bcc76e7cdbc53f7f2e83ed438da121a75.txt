本指南将引导您创建第一个具有规划、文件系统工具和子代理功能的深度代理。您将构建一个可以进行研究并撰写报告的研究代理。

<Tip>
    **使用 AI 编码助手？**

    - 安装 [LangChain 文档 MCP 服务器](/use-these-docs) 为您的代理提供对最新 LangChain 文档和示例的访问权限。
    - 安装 [LangChain 技能](https://github.com/langchain-ai/langchain-skills) 以提高您的代理在 LangChain 生态系统任务中的性能。
</Tip>

## 前提条件

在开始之前，请确保您有模型提供商（例如 Gemini、Anthropic、OpenAI）的 API 密钥。

<Note>
    深度代理需要一个支持 [工具调用](/oss/langchain/models#tool-calling)的模型。请参阅 [自定义](/oss/deepagents/customization#model) 了解如何配置您的模型。
</Note>

## 步骤 1：安装依赖项

:::python
<CodeGroup>

</CodeGroup>

:::js

    ```bash yarn
    yarn add deepagents langchain @langchain/core @langchain/tavily
    ```

    ```bash pnpm

    ```
</CodeGroup>
:::

<Note>
    本指南使用 [Tavily](https://tavily.com/) 作为示例搜索提供商，但您可以替换任何搜索 API（例如 DuckDuckGo、SerpAPI、Brave Search）。

## 它是如何工作的？

您的深度代理自动：

1. **规划其方法** 使用内置的 [`write_todos`](/oss/deepagents/harness#task-planning) 工具来分解研究任务。
1. **进行研究** 通过调用 `internet_search` 工具来收集信息。
1. **管理上下文** 通过使用文件系统工具 ([`write_file`](/oss/deepagents/harness#virtual-filesystem-access), [`read_file`](/oss/deepagents/harness#virtual-filesystem-access)) 来卸载大型搜索结果。
1. **生成子代理** 根据需要将复杂子任务委托给专业子代理。
1. **综合生成报告** 将调查结果编译成连贯的响应。

## 流式传输

Deep Agents 内置 [流式传输](/oss/langchain/event-streaming) 功能，可通过 LangGraph 获取代理执行的实时更新。
这允许您逐步观察输出，并审查和调试代理和子代理的工作，例如工具调用、工具结果和 LLM 响应。

## 后续步骤

既然您已经构建了您的第一个深度代理：

- **自定义您的代理**：了解 [自定义选项](/oss/deepagents/customization)，包括自定义系统提示、工具和子代理。
- **添加长期记忆**：启用 [持久记忆](/oss/deepagents/memory) 跨对话功能。
- **部署到生产环境**：使用 [托管式 Deep Agents](/langsmith/managed-deep-agents-overview) 在 LangSmith 中创建、运行和操作深度代理。
