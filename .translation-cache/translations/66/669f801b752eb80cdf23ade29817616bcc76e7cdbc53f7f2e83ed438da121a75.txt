本指南将引导您创建您的第一个深度智能体，具备规划、文件系统和子智能体功能。您将构建一个能够进行研究并撰写报告的研究智能体。

<Tip>
    **使用 AI 编程助手？**

    - 安装 [LangChain 文档 MCP 服务器](/use-these-docs)，为您的智能体提供最新的 LangChain 文档和示例。
    - 安装 [LangChain Skills](https://github.com/langchain-ai/langchain-skills)，提升您的智能体在 LangChain 生态系统任务中的表现。
</Tip>

## 前置条件

在开始之前，请确保您拥有模型提供商的 API 密钥（例如 Gemini、Anthropic、OpenAI）。

<Note>
    Deep Agents 需要支持[工具调用](/oss/langchain/models#tool-calling)的模型。请参阅[自定义](/oss/deepagents/customization#model)了解如何配置您的模型。
</Note>

## 工作原理

您的深度代理会自动：

1. **规划其方法** - 使用内置的 [`write_todos`](/oss/deepagents/harness#task-planning) 工具分解研究任务。
2. **开展研究** - 调用 `internet_search` 工具收集信息。
3. **管理上下文** - 使用文件系统工具（[`write_file`](/oss/deepagents/harness#virtual-filesystem-access)、[`read_file`](/oss/deepagents/harness#virtual-filesystem-access)）卸载大型搜索结果。
4. **生成子代理** - 根据需要生成子代理，将复杂子任务委托给专门的子代理。
5. **综合报告** - 将发现编译成连贯的响应。

## 流式传输

Deep Agents 内置了 [流式传输](/oss/langchain/event-streaming) 功能，可通过 LangGraph 实时更新代理执行情况。
这使您能够逐步观察输出，并审查和调试代理和子代理的工作，例如工具调用、工具结果和 LLM 响应。

## 下一步

现在您已经构建了第一个深度代理：

- **自定义您的代理**：了解 [自定义选项](/oss/deepagents/customization)，包括自定义系统提示、工具和子代理。
- **添加长期记忆**：跨对话启用 [持久记忆](/oss/deepagents/memory)。
- **部署到生产环境**：使用 [托管深度代理](/langsmith/managed-deep-agents-overview) 在 LangSmith 中创建、运行和操作深度代理。
