## langsmith-0.16.0-rc.4

- 评估器分离确认对话框显示"分离"按钮而不是"删除"。
- "包含扩展统计"功能现已对所有组织开放代码评估器使用。
- 新增两个专用权限 `bulk-exports:read` 和 `bulk-exports:manage`，用于获取和创建/更新批量导出。
- 修复了 Engine"连接 GitHub"流程中 GitHub App 已通过同一组织中的其他工作区安装的情况。
- 在 `smith-frontend` 中将 `@langchain/langgraph-sdk` 升级至 1.9.4。
- 在 `SMITH_ACE_SANDBOX_IMPLEMENTATION=v2` 后面添加了可选的 Smith-ACE v2 沙盒实现。
- 线程表现在在 *Last Output* 列中显示实际最后输出，并在新的 *Last Error* 列中显示线程级错误。
- 在追踪树视图中隐藏了工具调用的 $0.00 成本徽章。
- 用户现在可以在同一个代理上使用相同的表达式创建多个 cron 计划。
- Agent Builder 的"查看代理追踪"和"查看追踪"链接现在始终在 fleet 追踪项目中打开。
- 为 `gemini-3.1-flash-lite` 添加了 LangSmith 模型定价条目。
- LLM-as-judge 评估器现在可以选择"包含扩展统计"并从 `run.*` 字段映射提示变量。
- 默认沙盒 rootfs 镜像现在包含 Docker Compose 并自动启动 Docker 守护进程。
- 为 `gemini-3.5-flash` 添加了成本追踪。
- Gateway 支出上限策略现在可以配置为每周周期。
- 添加了 Centralize 作为 MCP 市场集成。
- 启用了沙盒的代理现在在其系统提示中看到配置的代理配置文件（主机、注入的头部密钥、网络规则、OAuth 提供商），取代了旧的主机专用 auth-proxy 部分。
- 在 `SANDBOX_FEATURE_ENABLED` 关闭的区域隐藏了沙盒导航项和 `/sandboxes` 页面。
- 自托管 DockerHub 镜像现在包含 Cosign 签名和签名的 SPDX SBOM 证明。
- 修复了一个 bug，其中线程 ID 中的特殊字符未编码，导致 UI 无法查询这些线程。
- 修复了打开大型追踪时出现的"查询超时超出"错误。
- 自托管 OIDC：修复了 SSO 群组同步在登录期间静默无操作的问题。
- Managed Deep Agents 私有预览现在支持使用基于头部认证的 MCP 服务器注册。
- 从 `queue` workers 发出 Prometheus 指标。
- 澄清了应用文本过滤器时统计数据不可用的消息。
- 上下文仓库现在支持从 Hub 溢出菜单更新和删除元数据。
- 为 Claude Opus 4.8 添加了令牌定价支持。
- Agent Builder 现在将 Claude Opus 4.8 作为内置 Anthropic 模型提供。
- 组织管理员现在可以通过 service-keys API 更新现有 API 密钥的角色，而无需轮换密钥。
- Managed Deep Agents MCP 服务器设置现在在 `/v1/deepagents` API 命名空间下支持 OAuth。
- 为 Bedrock Nova 2（以及任何其他需要 camelCase API 字段的提供商）输入的额外参数现在在 Playground 中保存和重新加载模型配置时保留其原始键大小写。
- 自托管 OIDC 用户现在从 `name` / `given_name`+`family_name` id_token 声明中解析显示名称。
- 修复了 `playground` 服务的 SSRF 策略，使其遵守 `SSRF_ALLOW_K8S_INTERNAL`。
- 修复了一个 LLM gateway 数据保护 bug，该 bug 可能在启用 PII 脱敏时损坏 Anthropic 图像或文档。
- 隐藏沙盒文件资源管理器控件，同时允许显式沙盒摘要下载。
- Engine 现在支持可选的每月 LCU 支出限制（由财务、计划或组织管理员设置），一旦达到该限制将暂停新的 Engine 运行。
- agent builder/fleet 中的聊天输入文件上传在启用沙盒时现在到达沙盒文件系统中的 `/tmp/uploads/`。
- Fleet Default 现在在符合条件的计划的模型选择器中排在首位。
- 修复了项目统计侧边栏追踪计数标签和标题布局。
- 数据平面无访问权限屏幕上的工作区切换器仅列出当前组织的工作区。
- 恢复了自 2026 年 3 月初以来一直静默失败的 enterprise Fleet 代理的 cron 执行。
- Run rules webhook 有效载荷现在为每个运行包含一个 `trace_url` 深度链接。
- 实验加载进度条现在在实验表中显示已完成和评估的运行数量。
- 沙盒现在允许非 root 用户使用基于密码的 SSH，同时保持 root SSH 登录仅限密钥。

- 修复了安全漏洞。请参阅 CVE-2026-45736、CVE-2026-44664、CVE-2025-71176 了解详情。

**下载 Helm chart：** [`langsmith-0.16.0-rc.4.tgz`](https://github.com/langchain-ai/helm/releases/download/langsmith-0.16.0-rc.4/langsmith-0.16.0-rc.4.tgz)
{/* langsmith-release-image: 0.16.0-rc.4 0.16.5-51eaa96f56bc7d7a37eef03b4b8bb601359e4653 */}
</Update>

## langsmith-0.15.11

- 修复了删除活动代理后的 Fleet 重定向。
- 从 LangSmith 数据集表中删除了 Type 列。

- Fleet 代理 API 现在要求沙盒支持代理使用 `thread_scoped_sandbox` 或 `agent_scoped_sandbox`。

## langsmith-0.15.0

LangSmith 自托管 v0.15 引入了**可复用评估器和包含 30+ 评估器模板的库**，集中管理整个工作区的评估；推出了**每个示例的断言**功能，可与标注队列中的参考输出一起使用；支持下载 **Insights 报告**为 PDF 用于离线分析；并推出 **Context Hub**，用于对 agent 指令和工具进行版本控制和环境感知管理。在升级前有几个破坏性变更值得注意：`agent-bootstrap` 脚本已弃用，Agent Builder 更名为 [Fleet](/langsmith/fleet) 可能需要更新 workload-identity 服务账号，`projects:update-retention` 权限拆分为 `projects:increase-trace-tier` 和 `projects:decrease-trace-tier`。

请按照[升级说明](/langsmith/self-host-upgrades)获取所有功能。如需预约 LangChain 支持团队协助升级，请通过[支持门户](https://support.langchain.com)联系团队。

### 破坏性变更

- 弃用了 `agent-bootstrap` 脚本。LangSmith agents 现在是独立服务，通过 Helm chart 部署，而非通过 LangSmith Deployment 控制平面。如果您之前通过此脚本使用 [Fleet](/langsmith/fleet)，可能需要进行迁移。请联系支持团队以获取迁移指导。
- 将 Agent Builder 重命名为 [Fleet](/langsmith/fleet)。如果您使用 workload identity，可能需要更新任何服务账号。
- `POST /workspaces/current/members` 现在要求[启用 RBAC](/langsmith/rbac)的组织提供 `role_id`。不带此参数的请求将返回 `400`，而非默认为 `WORKSPACE_ADMIN`。
- 弃用了 `USAGE_EXPORT_ADMIN_EMAILS` 环境变量。请改用 `INSTANCE_ADMIN_EMAILS`。
- 将 `projects:update-retention` 权限替换为 `projects:increase-trace-tier` 和 `projects:decrease-trace-tier`，以分别控制提高和降低 trace 保留期限。权限已回填到现有角色，因此现有角色无需更改。新角色应使用新权限。请参阅 [RBAC 权限](/langsmith/rbac)。
- 新增了 `fleet-admin:read` 权限，用于访问新的 Fleet Admin 部分。现有租户的管理员需要授予此权限。权限已回填到现有角色，因此现有角色无需更改。新角色应使用新权限。请参阅 [RBAC 权限](/langsmith/rbac)。

### 基础设施变更

- **由于 Fleet 重命名而导致的板块重命名** — 作为 Agent Builder 更名为 [Fleet](/langsmith/fleet) 的一部分（请参阅[破坏性变更](#breaking-changes)），多个板块已重命名。您可能需要更新服务账号或调整配置中的值。

### 新功能

- **Context Hub** — 对 agent 指令和工具进行版本控制和环境感知管理。创建和管理版本化的 [skill 和 agent 仓库](/langsmith/context-engineering-concepts)，将 commit 提升到 `staging` 或 `production` 环境，并在运行时按环境标签解析上下文。请参阅[使用 Context Hub](/langsmith/use-the-context-hub)和[使用 SDK 管理 contexts](/langsmith/manage-contexts-sdk) 开始使用。
- **可复用评估器和评估器模板** — 全新的[评估器](/langsmith/evaluators)标签页集中管理您工作区中的所有评估器，提供 30+ 模板覆盖安全性、响应质量、轨迹、用户行为和多模态评估。只需几秒即可将现有评估器附加到新的 tracing 项目中，无需维护重复副本。
- **每个示例的断言** — 在编辑[标注队列](/langsmith/annotation-queues)中的示例时，编写[断言](/langsmith/assertions)来替代或配合参考输出使用。
- **可下载的 Insights 报告** — 从报告详情页面将 [Insights](/langsmith/insights) 报告下载为 PDF 以进行离线分析。

### 管理变更

- **扩展的 ABAC 覆盖范围** — [ABAC](/langsmith/abac) 现在适用于 `POST /runs` 和 `POST /runs/batch` 上的 `runs:create`，以及剩余的 `/sessions/{session_id}/` 端点。
- **修复 SCIM 邮箱大小写不匹配问题** — 发送不同邮箱大小写的身份提供商不再被拒绝为邮箱更改尝试。

## langsmith-0.14.0

LangSmith Self-Hosted v0.14 为自托管带来了**聊天功能**（我们的产品内跟踪和运行聊天）、将 **ABAC 和审计日志** 正式发布（默认启用），并默认启用 **LLM Auth Proxy**，支持 URL 白名单和更丰富的 JWT 声明。管理员可获得跨代理构建器、聊天、洞察、操场和评估器的**统一模型配置**，以及细粒度的**提示所有者**权限控制，用于锁定谁能提升或删除单个提示。评估器获得**多模态支持**，工作区现在可以在跟踪项目上设置**成本警报**。操场模型支持扩展（通过 Vertex AI 的 Anthropic、自定义 Azure 模型、Bedrock 推理配置文件、Gemini 3.1 Pro、GPT-5.3/5.4、Baseten + GLM-5），新的代理工具和触发器支持 Google Sheets & Docs、Outlook、Teams 和 Salesforce SOQL。在基础设施方面，v0.14 添加了用于 blob 存储的 **GCS 工作负载身份**支持、**Valkey** 作为 Redis 的直接替代品，以及用于更安全部署的预升级迁移钩子。

请按照[升级说明](/langsmith/self-host-upgrades)获取所有功能。要预约 LangChain 支持团队进行升级，请通过[支持门户](https://support.langchain.com)联系团队。

### 重大变更

- 修复了 `host-backend` 未正确获取 `commonEnv` 的问题。这可能导致重复的环境变量，需要手动移除。

### 基础设施变更

- 迁移现在作为 `Pre-upgrade` 钩子在镜像版本推出之前运行。这将防止迁移失败时出现问题。
- **GCS 工作负载身份支持** — 使用云原生工作负载身份而非长期凭证向 GCS blob 存储进行身份验证。
- **Valkey 支持** — Valkey 现在可用作 Redis 的直接替代品。

### 新功能

- **自托管聊天** — 用于理解跟踪、运行和评估器反馈的产品内聊天功能现已可在自托管中使用。
- **ABAC 和审计日志正式发布** — 属性级访问控制和审计日志在自托管部署中默认启用。
- **LLM Auth Proxy 默认启用** — URL 白名单可防止凭证被转发到非预期主机，JWT 现在包含 `organization_name` 和 `workspace_name` 声明。
- **统一模型配置** — 代理构建器、聊天、洞察、操场和评估器现在共享一组模型配置，工作区管理员可控制跨所有 AI 功能的模型访问权限。
- **提示所有者** — 指定特定用户组，授予细粒度权限以提升或删除单个提示，而无需授予更广泛的组织访问权限。
- **多模态评估器** — 直接将附件和 base64 内容（图像、音频、PDF）传递到评估器。
- **跟踪项目成本警报** — 可在跟踪项目级别设置成本警报，与现有 LangSmith 警报配合使用。
- **扩展操场模型支持** — 通过 Vertex AI 的 Anthropic、自定义 Azure 模型、Bedrock 推理配置文件和可配置的基础 URL、Gemini 3.1 Pro、GPT-5.3/5.4（现为默认）、以及 Baseten + GLM-5。
- **新的代理工具和触发器** — Google Sheets 和 Docs、Outlook 邮件和日历、Microsoft Teams、Salesforce SOQL、具有刷新令牌的 Gmail OAuth v2，以及 Outlook 触发器。
- **洞察增强** — 计划的洞察报告、按时间趋势的类别、分析中的完整反馈评论，以及更低的最小作业间隔（6小时 → 1小时）。
- **标注和审核升级** — 每个队列必需的审核者、尊重 `reviewer_access_mode` 的成对队列、"分配给我"筛选器、每个标注者的 CSV 导出以及批量表格操作。
- **提示中心和工具注册表** — 提交标签搜索、模板创建中的模型选择、作用域工具注册表 API 和私有注册表 UI。
- **评估器工作流改进** — 预构建的 LLM 评估器默认使用严格结构化输出，评估器支持标签和复用、重试不再丢失分数，以及新的 API 可编程运行操场实验。
- **自定义 iframe 输出渲染器** — 在实验和跟踪视图中可拖动调整大小的 HTML 图表输出。
- **线程和收件箱 UX** — 自动生成的线程标题、线程中重新设计的运行详情、会话级反馈统计、键盘快捷键，以及过滤内部辅助线程。
