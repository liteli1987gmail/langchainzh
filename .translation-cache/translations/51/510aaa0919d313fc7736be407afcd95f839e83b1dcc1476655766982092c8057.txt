## langsmith-0.16.0-rc.4

- 评估器分离确认对话框显示的是「分离」按钮而不是「删除」按钮。
- 「包含扩展统计」功能现已对所有组织开放代码评估器使用。
- 新增两个专用权限 `bulk-exports:read` 和 `bulk-exports:manage` for fetching and creating/updating bulk exports.
- 修复了当 GitHub App 已通过同一组织中的其他工作区安装时，Engine「连接 GitHub」流程的问题。
- 已升级 `@langchain/langgraph-sdk` to 1.9.4 in `smith-frontend`.
- 在 `SMITH_ACE_SANDBOX_IMPLEMENTATION=v2`.
- 线程表现在在 *最后输出* 列中显示实际的最后输出，并在新的 *最后错误* column.
- 中显示线程级错误。在追踪树视图中隐藏了工具调用的 $0.00 成本标签。
- 用户现在可以在同一个智能体上使用相同表达式创建多个 cron 计划。
- Agent Builder 的「查看智能体追踪」和「查看追踪」链接始终在 fleet 追踪项目中打开。
- 为 `gemini-3.1-flash-lite`.
- 添加了 LangSmith 模型定价条目。LLM-as-judge 评估器现在可以选择「包含扩展统计」并从 `run.*` fields.
- 默认沙箱 rootfs 镜像现在包含 Docker Compose 并自动启动 Docker 守护进程。
- 为 `gemini-3.5-flash`.
- 添加了成本追踪。Gateway 支出上限策略现在可以配置为每周周期。
- 将 Centralize 添加为 MCP 市场集成。
- 启用沙箱的智能体现在可以在系统提示中看到配置的代理配置文件（主机、注入的请求头密钥、网络规则、OAuth 提供商），取代了旧的仅主机 auth-proxy 部分。
- 在 `/sandboxes` 关闭的地区隐藏了沙箱导航项和 `SANDBOX_FEATURE_ENABLED` 页面。
- 自托管 DockerHub 镜像现在包含 Cosign 签名和签名的 SPDX SBOM 证明。
- 修复了线程 ID 中的特殊字符未被编码导致 UI 无法查询这些线程的 bug。
- 修复了打开大型跟踪时出现的“查询超时”错误。
- 自托管 OIDC：修复了 SSO 组同步在登录期间静默无效的问题。
- 托管 Deep Agents 私有预览版现在支持使用基于 Header 认证的 MCP 服务器注册。
- 从以下来源发出 Prometheus 指标 `queue` workers.
- 明确了应用文本过滤器时统计信息不可用的消息。
- 上下文仓库现在支持从 Hub 溢出菜单更新元数据和删除。
- 添加了对 Claude Opus 4.8 的令牌定价支持。
- Agent Builder 现在提供 Claude Opus 4.8 作为内置 Anthropic 模型。
- 组织管理员现在可以通过 service-keys API 更新现有 API 密钥的角色，而无需轮换密钥。
- 托管 Deep Agents MCP 服务器设置现在支持 `/v1/deepagents` API 命名空间下的 OAuth。
- 为 Bedrock Nova 2（以及任何其他需要 camelCase API 字段的提供商）输入的额外参数现在在 Playground 中保存和重新加载模型配置时保持其原始键名大小写。
- 自托管 OIDC 用户现在可以从 `name` / `given_name`+`family_name` id_token 声明中解析显示名称。
- 修复了 `playground` 服务的 SSRF 策略，使其遵守 `SSRF_ALLOW_K8S_INTERNAL`.
- 修复了一个可能导致在启用 PII 脱敏时损坏 Anthropic 图像或文档的 LLM 网关数据保护 bug。
- 隐藏沙箱文件浏览器控件，同时允许显式沙箱摘要下载。
- 引擎现在支持可选的月度 LCU 消费限额（由财务、计划或组织管理员设置），一旦达到该限额，新引擎运行将暂停。
- Chat-input file uploads in agent builder/fleet now reached the sandbox filesystem at `/tmp/uploads/` 当沙箱被启用时。
- “Fleet Default”现在在符合条件的计划的模型选择器中排在首位。
- 修复了项目统计侧边栏跟踪计数标签和标题布局。
- 数据平面无访问权限屏幕上的工作区切换器仅列出当前组织的工作区。
- 恢复了自 2026 年 3 月初以来静默失败的企业 Fleet 代理的 cron 执行。
- 运行规则 webhook 载荷现在包含 `trace_url` 每个运行的深度链接。
- 实验加载进度条在实验表中显示已完成和已评估的运行数量。
- 沙箱现在允许非 root 用户使用基于密码的 SSH，同时保持 root SSH 登录仅限密钥认证。

- 修复了安全漏洞。详见 CVE-2026-45736、CVE-2026-44664、CVE-2025-71176。

**下载 Helm chart：** [`langsmith-0.16.0-rc.4.tgz`](https://github.com/langchain-ai/helm/releases/download/langsmith-0.16.0-rc.4/langsmith-0.16.0-rc.4.tgz)

- 评估器分离确认对话框显示的是"分离"按钮而非"删除"。
- 扩展统计现在已对所有组织的代码评估器可用。
- 新增了两个专用权限 `bulk-exports:read` 和 `bulk-exports:manage` for fetching and creating/updating bulk exports.
- 修复了当 GitHub App 已通过同一组织中的另一个工作区安装时 Engine"连接 GitHub"流程的问题。
- 升级了 `@langchain/langgraph-sdk` to 1.9.4 in `smith-frontend`.
- 在...后添加了可选的 Smith-ACE v2 沙箱实现 `SMITH_ACE_SANDBOX_IMPLEMENTATION=v2`.
- 线程表现在显示实际的最后输出在 *最后输出* 列中，并在新的 *最后错误* column.
- 在追踪树视图中隐藏了工具调用的$0.00成本徽章。
- 用户现在可以在同一代理上使用相同的表达式创建多个 cron 计划。
- Agent Builder 的"查看代理追踪"和"查看追踪"链接现在始终在 fleet tracing 项目中打开。
- 为...添加了 LangSmith 模型定价条目 `gemini-3.1-flash-lite`.
- LLM-as-judge 评估器现在可以选择包含扩展统计并从...映射提示变量 `run.*` fields.
- 默认沙箱 rootfs 镜像现在包含 Docker Compose 并自动启动 Docker 守护进程。
- 添加了...的成本追踪 `gemini-3.5-flash`.
- Gateway 消费上限策略现在可以配置为每周周期。
- 将 Centralize 添加为 MCP marketplace 集成。
- 启用沙箱的代理现在在其系统提示中看到配置的代理配置文件（主机、注入的头部密钥、网络规则、OAuth 提供商），替换了旧的仅主机 auth-proxy 部分。
- 在...的区域隐藏了沙箱导航入口和 `/sandboxes` 页面 `SANDBOX_FEATURE_ENABLED` 已关闭。
- 自托管 DockerHub 镜像现在包含 Cosign 签名和签名的 SPDX SBOM 证明。
- 修复了线程 ID 中的特殊字符未被编码的 bug，导致 UI 无法查询这些线程。
- 修复了打开大型追踪时"查询超时"的错误。
- 自托管 OIDC：修复了登录期间 SSO Groups Sync 静默无效的问题。
- 托管 Deep Agents 私有预览现在支持使用基于头部认证的 MCP 服务器注册。
- 从...发出 Prometheus 指标 `queue` workers.
- 澄清了应用文本过滤器时统计不可用的消息。
- 上下文仓库现在支持从 Hub 溢出菜单进行元数据更新和删除。
- Fleet 的类型化响应和标准错误信封 `/v1/fleet/agents/{agent_id}/connections` (List / Create / Delete) endpoints.
- 沙箱快照现在可以导出在沙箱内构建的 Docker 镜像。
- 修复了 ACE 子进程处理，以便早期子进程退出返回请求失败而不是导致服务崩溃。
- 修复了本机运行摄取负载中的大整数保留问题。
- 瀑布视图现在如果可用则占用完整高度。
- 组织管理员现在可以在计划自动启用 Engine 时禁用它；他们的明确选择会在 UI 和后端网关中持续生效。
- 工作区管理员可以在评估器侧边栏中按每条评估器规则覆盖工作区默认的每周支出上限，而非管理员则只能将解析后的上限作为只读文本查看。
- 修复了元数据分面建议不正确的问题，并改善了具有丰富运行元数据的项目的分组统计延迟。
- 现已支持运行计数、错误、延迟和成本的告警规则 `<`, `<=`, `>`，以及 `>=` 比较运算符（此前界面仅允许 `>=`).
- Fleet `/v1/fleet/auth-agents/{agent_id}/connections` 端点已迁移至 `/v1/fleet/agents/{agent_id}/connections` ，现包含类型化响应、请求验证和标准 Fleet 错误格式；旧 URL 返回 404。
- 修复了删除活跃代理后 Fleet 重定向的问题。
- 已从 LangSmith 数据集表中移除类型列。
- Encrypted/redacted "reasoning" content blocks no longer appeared as empty or garbled cards in the trace messages view; meaningful extended-thinking content continued to render normally.
- Fleet 代理 API 现要求 `thread_scoped_sandbox` or `agent_scoped_sandbox` 用于沙箱支持的代理。
- 允许通过新的 `all_experiments` 参数导出工作区中的所有实验，每次导出限制为 250 个实验，可根据请求增加。
- Fleet 使用 langchain-fireworks 1.4.2 进行 Fireworks 模型调用。
- 这使得运行详情面板能够重新设计，提高了可读性和消息解析的稳健性。
- Fleet/Agent Builder now included Gemini 3.5 Flash as a selectable built-in model.
- 计算机使用现已为符合条件的通用聊天用户提供应用内调用提示。
- 修复了 blob 存储横幅在页面加载时错误闪烁的问题。
- 这使得用户能够在运行详情面板中直接留下运行反馈。
- 已添加对 Claude Opus 4.8 的代币定价支持。
- Agent Builder 现已将 Claude Opus 4.8 作为内置 Anthropic 模型提供。
- 组织管理员现在可以通过 service-keys API 更新现有 API 密钥的角色，而无需轮换密钥。
- 托管 Deep Agents MCP 服务器设置现支持 `/v1/deepagents` API 命名空间下的 OAuth。
- 为 Bedrock Nova 2（及其他需要 camelCase API 字段的提供商）输入的额外参数现在在 Playground 中保存和重新加载模型配置时保留了其原始键名大小写。
- 自托管 OIDC 用户现在可以从 `name` / `given_name`+`family_name` id_令牌声明中获取显示名称。
- 修复了 `playground` 服务的 SSRF 策略，使其遵守 `SSRF_ALLOW_K8S_INTERNAL`.
- 修复了一个 LLM 网关数据保护错误，该错误可能在启用 PII 编辑时损坏 Anthropic 图像或文档。
- 隐藏了沙箱文件浏览器控件，同时允许显式下载沙箱摘要。
- Engine 现支持可选的每月 LCU 支出上限（由财务、计划或组织管理员设置），一旦达到该上限将暂停新的 Engine 运行。

## langsmith-0.15.0

LangSmith 自托管 v0.15 带来了 **可重用评估器和包含 30+ 评估器模板的库** 集中整个工作区的评估的发布 **每示例断言** 在标注队列中与参考输出一起显示，让您可以下载 **Insights 报告** 为 PDF 格式用于离线分析，并引入了 **Context Hub** 用于对代理指令和工具进行版本控制、环境感知的管理。在升级之前有几个重大变更值得查看： `agent-bootstrap` 脚本已弃用，Agent Builder 重命名为 [Fleet](/langsmith/fleet) 可能需要更新工作负载身份服务账号，以及 `projects:update-retention` 权限拆分为 `projects:increase-trace-tier` 和 `projects:decrease-trace-tier`.

请参阅 [升级说明](/langsmith/self-host-upgrades) 以获取所有功能。如需预约 LangChain 支持团队协助您的升级，请通过 [支持门户](https://support.langchain.com).

### 重大变更

- 已弃用 `agent-bootstrap` 脚本。LangSmith 代理现在是独立服务，通过 Helm chart 部署，而不是通过 LangSmith Deployment 控制平面。如果您之前在使用 [Fleet](/langsmith/fleet) 通过此脚本进行部署，可能需要迁移。请联系支持团队以协助完成迁移流程。
- 将 Agent Builder 重命名为 [Fleet](/langsmith/fleet)。如果您使用工作负载身份，可能需要更新服务账号。
- `POST /workspaces/current/members` 现在需要 `role_id` 用于 [RBAC](/langsmith/rbac)启用的组织。如果没有该设置，请求将返回 `400` 而不是默认返回 `WORKSPACE_ADMIN`.
- 已弃用 `USAGE_EXPORT_ADMIN_EMAILS` 环境变量。请使用 `INSTANCE_ADMIN_EMAILS` instead.
- 已替换 `projects:update-retention` 权限替换为 `projects:increase-trace-tier` 和 `projects:decrease-trace-tier` 用于分别控制提高和降低追踪保留期。权限已回填到现有角色，因此现有角色无需更改。新角色应使用新权限。请参阅 [RBAC 权限](/langsmith/rbac).
- 添加了 `fleet-admin:read` 控制新 Fleet Admin 部分的权限。现有租户的管理员需要授予此权限。权限已回填到现有角色，因此现有角色无需更改。新角色应使用新权限。请参阅 [RBAC 权限](/langsmith/rbac).

### 基础设施变更

- **因 Fleet 重命名导致的章节重命名** —— 作为 Agent Builder 到 [Fleet](/langsmith/fleet) 重命名的一部分（请参阅 [重大变更](#breaking-changes)）。您可能需要更新服务账号或调整配置中的值。

### 新功能

- **Context Hub** —— 代理指令和工具的版本控制、环境感知管理。创建和管理版本化的 [技能和代理仓库](/langsmith/context-engineering-concepts)，将提交晋升到 `staging` or `production` 环境，并在运行时通过环境标签解析上下文。参见 [使用上下文中心](/langsmith/use-the-context-hub) 和 [使用 SDK 管理上下文](/langsmith/manage-contexts-sdk) 以开始使用。
- **可复用评估器和评估器模板** — 一个新的 [评估器](/langsmith/evaluators) 选项卡集中管理您工作区中的所有评估器，提供 30+ 模板，覆盖安全性、响应质量、轨迹、用户行为和多模态评估。在几秒钟内将现有评估器附加到新的追踪项目中，无需维护重复副本。
- **按示例断言** — 编写 [断言](/langsmith/assertions) 而不是或与参考输出一起，在编辑中的示例时 [标注队列](/langsmith/annotation-queues).
- **可下载的洞察报告** — 下载 [洞察](/langsmith/insights) 报告为 PDF，用于报告详情页的离线分析。

### 管理员变更

- **扩展的 ABAC 覆盖范围** — [ABAC](/langsmith/abac) 现在适用于 `runs:create` on `POST /runs` 和 `POST /runs/batch`，加上其余的 `/sessions/{session_id}/` endpoints.
- **SCIM 邮箱大小写不匹配修复** — 发送不同邮箱大小写的身份提供者不再被视为邮箱更改尝试而被拒绝。

## langsmith-0.14.0

LangSmith 自托管 v0.14 版本带来 **聊天** （我们产品内用于跟踪和运行的聊天功能）到自托管环境，并带来 **ABAC 和审计日志** 正式发布（默认开启），并启用 **LLM 身份验证代理** 默认启用 URL 白名单和更丰富的 JWT 声明。管理员可以获得 **统一的模型配置** 跨 Agent Builder、聊天、洞察、 Playground 和评估器共享，以及细粒度的 **提示词所有者** 用于控制谁可以升级或删除单个提示词。评估器获得 **多模态支持** 工作区现在可以设置 **成本警报** on tracing projects. Playground model support expands (Anthropic via Vertex AI, custom Azure models, Bedrock inference profiles, Gemini 3.1 Pro, GPT-5.3 / 5.4, Baseten + GLM-5), and new agent tools and triggers land for Google Sheets & Docs, Outlook, Teams, and Salesforce SOQL. On the infrastructure side, v0.14 adds **GCS 工作负载身份** 支持 blob 存储， **Valkey** 作为 Redis 的即插即用替代品，以及升级前的迁移钩子以实现更安全的部署。

按照 [升级说明](/langsmith/self-host-upgrades) 获取所有功能。要预约 LangChain 支持团队协助您的升级，请通过 [支持门户](https://support.langchain.com).

### 重大变更

- 修复了一个问题，其中 `host-backend` 未检测到 `commonEnv`。这可能导致重复的环境变量需要被移除。

### 基础设施变更

- 迁移现在作为 `Pre-upgrade` 钩子运行，在镜像版本推出之前执行。这将防止迁移失败时出现问题。
- **GCS 工作负载身份支持** — 使用云原生工作负载身份验证 GCS Blob 存储，而不是使用长期凭证。
- **Valkey 支持** — Valkey 现在可用作 Redis 的直接替代品。

### 新功能

- **自托管版聊天功能** — 产品内聊天功能现已可用于自托管版，帮助理解追踪、运行和评估器反馈。
- **ABAC 和审计日志正式版** — 基于属性的访问控制和审计日志现已默认启用，适用于自托管部署。
- **LLM 身份验证代理默认开启** — URL 白名单可防止凭证被转发到非预期主机，JWT 现在携带 `organization_name` 和 `workspace_name` claims.
- **统一模型配置** — Agent Builder、聊天、洞察、Playground 和评估器现在共享同一套模型配置，工作区管理员可控制所有 AI 功能的模型访问权限。
- **提示词所有者** — 指定特定用户组具有精细权限，可提升或删除单个提示词，无需授予更广泛的组织访问权限。
- **多模态评估器** — 可将附件和 base64 内容（图像、音频、PDF）直接传入评估器。
- **追踪项目的成本告警** — 在现有 LangSmith 警报之外，设置追踪项目级成本的警报。
- **扩展的 Playground 模型支持** — Anthropic via Vertex AI, custom Azure models, Bedrock inference profiles and configurable base URLs, Gemini 3.1 Pro, GPT-5.3 / 5.4 (now default), and Baseten + GLM-5.
- **新的 Agent 工具和触发器** — Google Sheets 和 Docs、Outlook 邮件和日历、Microsoft Teams、Salesforce SOQL、带有刷新令牌的 Gmail OAuth v2，以及 Outlook 触发器。
- **Insights 增强功能** — 计划的 Insights 报告、随时间变化的类别趋势、分析中的完整反馈评论，以及更低的最小作业间隔（6小时 → 1小时）。
- **标注和审核升级** — 每个队列所需审核者、遵循的成对队列 `reviewer_access_mode`、"Assigned to me"筛选器、每位标注者的 CSV 导出和批量表格操作。
- **Prompt Hub 和工具注册表** — 提交标签搜索、模板创建中的模型选择、工作空间级别的工具注册 API 以及私有注册表 UI。
- **评估器工作流改进** — 预置 LLM 评估器默认使用严格结构化输出，评估器支持标签化和复用，重试不再丢失评分，以及新的 API 支持以编程方式运行实验场实验。
- **自定义 iframe 输出渲染器** — 在实验和追踪视图中可拖动调整大小的 HTML 图表输出。
- **线程和收件箱 UX** — 自动生成的线程标题、重新设计的线程运行详情、会话级反馈统计、键盘快捷键，以及过滤内部辅助线程。
