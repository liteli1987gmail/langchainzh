## 包含内容

自托管 LangSmith 实例包含：

**服务：**
- LangSmith 前端 UI
- LangSmith 后端 API
- LangSmith Platform 后端
- LangSmith Playground
- LangSmith 队列
- LangSmith ACE（任意代码执行）后端

**存储服务：**
- ClickHouse（追踪和反馈数据）
- PostgreSQL（操作数据）
- Redis（队列和缓存）
- Blob 存储（可选，但建议用于生产环境）

要访问 LangSmith UI 并发送 API 请求，您需要公开 [LangSmith 前端](#services) 服务。根据您的安装方式，这可以是负载均衡器或主机上公开的端口。

### 服务

| 服务 | 描述 |
|---------|-------------|
| <a id="langsmith-frontend"></a> **LangSmith 前端** | 前端使用 Nginx 来提供 LangSmith UI 并将 API 请求路由到其他服务器。这是应用程序的入口点，是唯一必须向用户公开的组件。 |
| <a id="langsmith-backend"></a> **LangSmith 后端** | 后端是 CRUD API 请求的主要入口点，处理应用程序的大部分业务逻辑。这包括处理来自前端和 SDK 的请求、准备用于摄取的追踪，以及支持 Hub API。 |
| <a id="langsmith-queue"></a> **LangSmith 队列** | 队列处理传入的追踪和反馈，以确保它们被异步摄取并持久化到追踪和反馈数据存储中，处理数据完整性检查并确保成功插入数据存储，在数据库错误或暂时无法连接到数据库等情况下处理重试。 |
| <a id="langsmith-platform-backend"></a> **LangSmith Platform 后端** | Platform 后端是另一个关键服务，主要处理身份验证、运行摄取和其他高容量任务。 |
| <a id="langsmith-playground"></a> **LangSmith Playground** | Playground 是一项将请求转发到各种 LLM API 以支持 Playground 功能的服务。这也可以用于连接您自己的自定义模型服务器。 |
| <a id="langsmith-ace-arbitrary-code-execution-backend"></a> **LangSmith ACE（任意代码执行）后端** | ACE 后端是一项在安全环境中执行任意代码的服务。这用于支持在 LangSmith 中运行自定义代码。 |

### 存储服务

<Note>
LangSmith 默认会打包所有存储服务。您可以将其配置为使用所有存储服务的外部版本。在生产环境中，我们**强烈建议使用外部存储服务**。
</Note>

| 服务 | 描述 |
|---------|-------------|
| <a id="clickhouse"></a> **ClickHouse** | [ClickHouse](https://clickhouse.com/docs/en/intro) 是一个高性能的列式 SQL 数据库管理系统（DBMS），用于在线分析处理（OLAP）。<br/><br/>LangSmith 使用 ClickHouse 作为追踪和反馈（高容量数据）的主要数据存储。<br/><br/>💡 [连接外部 ClickHouse](/langsmith/self-host-external-clickhouse) |
| <a id="postgresql"></a> **PostgreSQL** | [PostgreSQL](https://www.postgresql.org/about/) 是一个强大的开源对象关系数据库系统，它使用并扩展了结合了许多功能的 SQL 语言，可以安全地存储和扩展最复杂的数据工作负载。<br/><br/>LangSmith 使用 PostgreSQL 作为事务工作负载和操作数据的主要数据存储（除追踪和反馈之外的几乎所有内容）。<br/><br/>💡 [连接外部 PostgreSQL](/langsmith/self-host-external-postgres) - AWS RDS、GCP Cloud SQL、Azure Database |
| <a id="redis"></a> **Redis / Valkey** | [Redis](https://github.com/redis/redis) 是一个强大的内存键值数据库，会持久化到磁盘。由于将数据保存在内存中，Redis 为缓存等操作提供了高性能。<br/><br/>LangSmith 使用 Redis 来支持队列和缓存操作。[Valkey](https://valkey.io/) 也被官方支持作为 Redis 的直接替代品。<br/><br/>💡 [连接外部 Redis 或 Valkey](/langsmith/self-host-external-redis) - AWS ElastiCache、GCP Memorystore、Azure Cache |
| <a id="blob-storage"></a> **Blob 存储** | LangSmith 支持多种 blob 存储提供商，包括 [AWS S3](https://aws.amazon.com/s3/)、[Azure Blob Storage](https://azure.microsoft.com/en-us/services/storage/blobs/) 和 [Google Cloud Storage](https://cloud.google.com/storage)。<br/><br/>LangSmith 使用 blob 存储来存储大文件，如追踪工件、反馈附件和其他大型数据对象。Blob 存储是可选的，但强烈建议用于生产部署。<br/><br/>💡 [启用 blob 存储](/langsmith/self-host-blob-storage) - AWS S3、GCP GCS、Azure Blob |

要安装，请按照 [Kubernetes 设置指南](/langsmith/kubernetes)操作。
