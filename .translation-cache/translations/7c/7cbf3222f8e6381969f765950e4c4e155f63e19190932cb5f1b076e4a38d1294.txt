本页面介绍仅适用于以下内容的平台功能 [self-hosted](/langsmith/self-hosted) deployments.

## 自定义 PostgreSQL

可以使用自定义 PostgreSQL 实例来替代 [控制平面自动创建的那个](/langsmith/cloud-platform-features#database-provisioning)。请指定 [`POSTGRES_URI_CUSTOM`](/langsmith/env-var-self-hosted) 环境变量以使用自定义 PostgreSQL 实例。

多个部署可以共享同一个 PostgreSQL 实例。例如，对于 `Deployment A`, `POSTGRES_URI_CUSTOM` 可以设置为 `postgres://<user>:<password>@/<database_name_1>?host=<hostname_1>` 对于 `Deployment B`, `POSTGRES_URI_CUSTOM` 可以设置为 `postgres://<user>:<password>@/<database_name_2>?host=<hostname_1>`. `<database_name_1>` 和 `<database_name_2>` 是同一实例中的不同数据库，但 `<hostname_1>` 是共享的。 **同一数据库不能用于独立部署**.

## 自定义 Redis

可以使用自定义 Redis 实例来替代控制平面自动创建的那个。请指定 [`REDIS_URI_CUSTOM`](/langsmith/env-var-self-hosted) 环境变量以使用自定义 Redis 实例。

多个部署可以共享同一个 Redis 实例。例如，对于 `Deployment A`, `REDIS_URI_CUSTOM` 可以设置为 `redis://<hostname_1>:<port>/1` 对于 `Deployment B`, `REDIS_URI_CUSTOM` 可以设置为 `redis://<hostname_1>:<port>/2`. `1` 和 `2` 是同一实例中的不同数据库编号，但 `<hostname_1>` 是共享的。 **同一数据库编号不能用于独立部署**.

## 监听器

监听器是 [监听器应用](/langsmith/data-plane#listener-application)的实例。监听器包含关于应用程序的元数据（如版本）和关于可以部署到的计算基础设施的元数据（如 Kubernetes 命名空间）。

监听器数据模型仅适用于自托管部署。控制平面和监听器协调配置和调和运行在您自己的基础设施上的部署。

## 资源定制

自托管部署的资源可以完全定制。与公开固定 `Development` 和 `Production` [部署类型](/langsmith/cloud-platform-features#deployment-types)的云不同，自托管部署根据您的基础设施配置来调整 CPU、内存、副本和存储大小。请参阅 [配置 Agent Server 以实现扩展](/langsmith/agent-server-scale) 以获取调优指南。
