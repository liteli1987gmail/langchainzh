本页介绍仅适用于[自托管](/langsmith/self-hosted)部署的平台功能。

## 自定义 PostgreSQL

可以使用自定义 PostgreSQL 实例来替代[控制平面自动创建的](/langsmith/cloud-platform-features#database-provisioning)实例。请指定 [`POSTGRES_URI_CUSTOM`](/langsmith/env-var-self-hosted) 环境变量以使用自定义 PostgreSQL 实例。

多个部署可以共享同一个 PostgreSQL 实例。例如，对于 `Deployment A`，可以将 `POSTGRES_URI_CUSTOM` 设置为 `postgres://<user>:<password>@/<database_name_1>?host=<hostname_1>`，对于 `Deployment B`，可以将 `POSTGRES_URI_CUSTOM` 设置为 `postgres://<user>:<password>@/<database_name_2>?host=<hostname_1>`。`<database_name_1>` 和 `<database_name_2>` 是同一实例中的不同数据库，但 `<hostname_1>` 是共享的。**同一个数据库不能用于不同的部署**。

## 自定义 Redis

可以使用自定义 Redis 实例来替代控制平面自动创建的实例。请指定 [`REDIS_URI_CUSTOM`](/langsmith/env-var-self-hosted) 环境变量以使用自定义 Redis 实例。

多个部署可以共享同一个 Redis 实例。例如，对于 `Deployment A`，可以将 `REDIS_URI_CUSTOM` 设置为 `redis://<hostname_1>:<port>/1`，对于 `Deployment B`，可以将 `REDIS_URI_CUSTOM` 设置为 `redis://<hostname_1>:<port>/2`。`1` 和 `2` 是同一实例中的不同数据库编号，但 `<hostname_1>` 是共享的。**同一个数据库编号不能用于不同的部署**。

## 监听器

监听器是[监听器应用程序](/langsmith/data-plane#listener-application)的实例。监听器包含有关应用程序的元数据（如版本）以及有关可部署计算基础设施的元数据（如 Kubernetes 命名空间）。

监听器数据模型仅适用于自托管部署。控制平面和监听器协调配置并在您自己的基础设施上协调部署。

## 资源定制

自托管部署的资源可以完全定制。与云端提供固定的 `Development` 和 `Production` [部署类型](/langsmith/cloud-platform-features#deployment-types)不同，自托管部署根据您的基础设施配置调整 CPU、内存、副本和存储。请参阅[为扩展配置 Agent Server](/langsmith/agent-server-scale) 以获取调优指导。
