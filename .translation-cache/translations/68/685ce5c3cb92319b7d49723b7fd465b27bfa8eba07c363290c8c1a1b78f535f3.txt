LangSmith 中的 [Evaluators（评估器）](/langsmith/evaluation-concepts#evaluators) 是[工作区级别](/langsmith/administration-overview#workspaces)的资源。您可以将单个评估器附加到多个[跟踪项目](/langsmith/observability-concepts#projects)和[数据集](/langsmith/evaluation-concepts#datasets)，这样就可以在工作中应用一致的评估逻辑，而无需每次都重新创建。

<Tip>
[LangSmith Engine](/langsmith/engine) 会为检测到的问题推荐自定义评估器，并支持一键部署。
</Tip>

## 查看评估器

在 [LangSmith UI](https://smith.langchain.com) 中，选择左侧边栏中的 **Evaluators（评估器）** 来查看工作区中的所有评估器。

评估器表格显示以下列：

| 列 | 描述 |
|--------|-------------|
| Name（名称） | 评估器名称 |
| Type（类型） | **LLM 作为评判者** 或 **代码**。复合评分评估器仅限于单个跟踪项目和数据集，不会显示在此处。 |
| Feedback Key（反馈键） | 评估器生成的反馈键 |
| Projects & Datasets（项目和数据集） | 此评估器附加到的跟踪项目和数据集 |
| Evaluator Trace Count (this week)（评估器跟踪计数（本周）） | 该评估器在过去一周内运行的跟踪数量。仅在启用支出跟踪时显示；**–** 表示代码评估器或没有附加规则的评估器。 |
| Spend (this week)（支出（本周）） | 该评估器在过去一周的预计 USD 支出。仅在启用支出跟踪时显示；**–** 表示代码评估器或没有附加规则的评估器。 |
| Spend Status（支出状态） | 评估器是 **在限额内**、**无限额** 还是达到一个或多个已配置的支出限额。仅在启用支出跟踪时显示；**–** 表示代码评估器。 |
| Created By（创建者） | 创建该评估器的工作区成员 |
| Updated At（更新时间） | 评估器上次修改的时间 |
| Created At（创建时间） | 评估器的创建时间 |

## 创建评估器

1. 在 [LangSmith UI](https://smith.langchain.com) 中，选择左侧边栏中的 **Evaluators（评估器）**。
1. 点击 **+ Evaluator（评估器）** 打开新建评估器面板。
1. 该面板允许您：
   - **从零创建**：构建一个新的 [LLM-as-a-Judge（LLM 作为评判者）](/langsmith/llm-as-judge) 或 [Code（代码）](/langsmith/online-evaluations-code) 评估器。
   - **从模板创建**：从一个现成的评估器（也称为预构建评估器）开始，用于常见的评估模式。**Recommended（推荐）** 部分首先展示热门模板，然后按以下类别组织模板：

     | 类别 | 描述 |
     |----------|-------------|
     | Security（安全） | 检测泄露、注入和对抗性输入。 |
     | Safety（安全） | 评估内容安全性和审核。 |
     | Quality（质量） | 衡量输出质量和准确性。 |
     | Conversation（对话） | 评估对话质量和用户体验。 |
     | Trajectory（轨迹） | 评估智能体工具使用和决策路径。 |
     | Image Evaluations（图像评估） | 评估图像内容质量和安全性。 |
     | Voice Evaluation（语音评估） | 评估语音和音频交互质量。 |

您也可以直接从[跟踪项目](/langsmith/observability-concepts#projects)或[数据集](/langsmith/evaluation-concepts#datasets)添加评估器。在此流程中，您还可以**附加工作区中现有的评估器**，或创建[Composite（复合）](/langsmith/composite-evaluators-ui)评估器。请参阅[设置 LLM-as-a-judge 在线评估器](/langsmith/online-evaluations-llm-as-judge)和[在实验上自动运行评估器](/langsmith/bind-evaluator-to-dataset)。

## 查看评估器详情

点击表格中的任意评估器打开其详情视图。详情视图有四个标签页：

- **Overview（概览）**：评估器的反馈配置和提示词或代码定义。
- **Traces（跟踪）**：该评估器在所有附加资源上处理的跟踪。
- **Logs（日志）**：该评估器在所有附加资源上的执行日志。
- **Resources（资源）**：该评估器附加到的跟踪项目和数据集。

## 编辑评估器

打开评估器。在 **Overview（概览）** 标签页中，点击 **Edit evaluator（编辑评估器）** <Icon icon="pencil"/> 图标打开 **Configure Evaluator（配置评估器）** 面板。更新评估器的配置。点击 **Save（保存）**。

由于评估器是共享的，更改将应用于其附加的所有跟踪项目和数据集。

## 管理评估器跟踪保留

当在线评估器对跟踪进行评分时，它会将反馈附加到该跟踪上，这会[自动将跟踪升级到扩展保留期](/langsmith/usage-and-billing#data-retention-auto-upgrades)。扩展保留期会延长跟踪的保留时间，但成本更高。当您在[跟踪项目](/langsmith/observability-concepts#projects)上设置在线评估器时，可以选择退出此升级，使评分的跟踪保持在项目的基线保留期。

此控制仅在项目的[默认保留期](/langsmith/billing#change-project-level-default-retention)为[基础级别](/langsmith/usage-and-billing#how-it-works)时可用。如果项目默认使用扩展保留期（[在项目或工作区级别设置](/langsmith/data-purging-compliance#data-retention)），则评估器评分的跟踪将遵循该默认设置，且此选项被锁定。

要退出已评分跟踪的扩展保留：

1. 当您[创建](#create-an-evaluator)或[编辑](#edit-an-evaluator)在线评估器时，将来源设置为[跟踪项目](/langsmith/observability-concepts#projects)而非[数据集](/langsmith/evaluation-concepts#datasets)。
1. 在评估器配置面板中展开 **Advanced（高级）** 部分。
1. 取消选中 **Extend trace retention（扩展跟踪保留）**。

此更改适用于保存评估器后评分的跟踪。已评分的现有跟踪保持其当前的保留级别。

## 删除评估器

评估器在附加到跟踪项目或数据集时无法删除。要删除评估器，请先通过 **Resources（资源）** 标签页将其从所有资源中移除，然后再删除。
