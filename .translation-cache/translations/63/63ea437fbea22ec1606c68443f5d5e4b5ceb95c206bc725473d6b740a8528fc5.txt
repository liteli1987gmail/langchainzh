---
title: 沙箱访问权限
sidebarTitle: 权限
description: 控制工作区中谁可以在沙箱创建后与其交互。
---

每个沙箱都有一个记录的 **创建者**，即使用其 API 密钥或会话创建沙箱的工作区成员。默认情况下，只有创建者可以在该沙箱上运行命令、读取或写入文件、打开隧道或访问服务 URL。其他工作区成员需要 `sandboxes:exec` [权限](/langsmith/rbac) 才能与其未创建的沙箱进行交互。沙箱永远无法从创建它们的工作区以外的其他工作区访问。

## 谁可以做什么

| 调用者 | 默认 | 使用 `sandboxes:exec` |
| --- | --- | --- |
| 沙箱创建者 | ✅ 所有运行时操作 | ✅ 所有运行时操作 |
| 其他工作区成员 | ❌ 拒绝 | ✅ 所有运行时操作 |
| 不同工作区 | ❌ 隐藏（视为未找到） | ❌ 隐藏（视为未找到） |

"运行时操作"涵盖了创建后在运行中的沙箱上进行交互的四种方式：

- **执行** 命令（`langsmith sandbox exec`、`SandboxClient.exec`）
- **文件** 操作（读取、写入、列出沙箱内的路径）
- **隧道** 将 TCP 端口回传至您的机器（`langsmith sandbox tunnel`）
- **代理** 通过[服务 URL](/langsmith/sandbox-service-urls) 转发请求

生命周期操作——创建、列出、更新、删除沙箱——继续使用现有的 `sandboxes:create` / `sandboxes:read` / `sandboxes:update` / `sandboxes:delete` 权限。这些权限保持不变。

## 拒绝的请求

当请求被拒绝时，沙箱返回 `HTTP 403`，响应体会指明触发的规则：

```json
{
  "detail": {
    "error": "Forbidden",
    "message": "sandbox access denied: not the creator and missing sandboxes:exec"
  }
}
```

对存在于其他工作区中的沙盒的请求会返回 `404 Not Found` 而不是 `403`，因此响应不会泄露沙盒是否存在于其他地方。

## 共享沙盒

您有两种方式让团队成员使用您拥有的沙盒：

1. 为自定义角色授予 `sandboxes:exec` 权限，并在工作区中分配该角色。拥有该角色的任何人都可以与工作区中的每个沙盒进行交互。
2. 为沙盒内运行的 HTTP 服务使用[服务 URL](/langsmith/sandbox-service-urls)。服务 URL 使用自己的访问令牌，无需接收者成为工作区成员。

对于临时协作，服务 URL 方法通常更简单；当团队成员需要广泛的访问权限来操作非自己创建的沙盒时，可以使用 `sandboxes:exec`。
