虽然传统软件开发通过编写代码来实现，但 AI 应用通常从提示词中获取其逻辑。

本指南将介绍 LangSmith 中提示词工程的关键概念。

## 为什么进行提示词工程？

提示词在不改变模型底层能力的情况下引导模型的行为。通过提供指令、示例和上下文，提示词塑造了模型如何响应输入。

提示词工程之所以重要，是因为它允许你修改模型行为。虽然存在其他方法（如微调），但提示词工程通常具有最低的入门门槛，并且往往能带来最高的投资回报。

提示词工程通常是一项多学科协作工作。最有效的提示词工程师可能是产品经理、领域专家或其他非技术团队成员，而不是构建应用的软件工程师。适当的工具和基础设施对于支持这种跨职能协作至关重要。

## 提示词类型

提示词有两种不同的格式类型：`chat` 风格提示词和 `completion` 风格提示词。

**聊天提示词**是由多条消息组成的列表，每条消息都有一个角色（如 `system`、`user` 或 `assistant`）。这是大多数当前模型 API 支持的提示风格，也是推荐的格式。

**补全提示词**是单个字符串。这是一种较旧的提示风格，主要为向后兼容而保留。

<Note>
除非你有特定原因需要使用补全提示词，否则请在新项目中使用聊天提示词。聊天提示词为多轮对话提供了更好的结构，并且得到现代 LLM 的更好支持。
</Note>

## 提示词与提示词模板

虽然"提示词"和"提示词模板"经常被交替使用，但理解它们之间的区别有助于阐明 LangSmith 如何管理和评估你的 AI 应用。

- **提示词**指的是传递给语言模型的消息。
- **提示词模板**让你能够创建包含动态占位符的可重用提示词，这些占位符在运行时被填充。你可以定义变量而不是硬编码值，LangSmith 会在每次运行提示词时用不同的输入替换这些变量。这使得提示词更加灵活、可测试，并且更容易迭代。

以下是模板在实践中的工作方式：

1. **定义模板**：创建一个包含变量（用大括号标记）的提示词，这些变量将在运行时被替换：

    ```
    You are a customer support agent. This is the refund policy:

    {refund_policy}

    Please respond to the user's question:

    {question}
    ```

1. **提供输入值**：为每个变量提供实际值：

    ```json
    {
    "refund_policy": "no refunds under any circumstances",
    "question": "can I get a refund for this hat?"
    }
    ```

1. **获取最终提示词**: LangSmith 会用您的输入替换变量，生成发送给模型的提示词：

    ```
    You are a customer support agent. This is the refund policy:

    no refunds under any circumstances

    Please respond to the user's question:

    Can I get a refund for this hat?
    ```

<Tip>
在[提示词模板格式](/langsmith/create-a-prompt#template-format)指南中了解更多关于模板变量语法和格式选项的信息。
</Tip>

## LangSmith 中的提示词

您可以在 LangSmith 中存储和版本化管理提示词模板。这些模板可以在 Playground 中进行测试，使用提交和标签进行版本控制，并可拉到您的应用程序代码中使用。

<Callout type="info" color="#4F46E5" icon="player-play" iconType="regular">
打开 [Playground](https://smith.langchain.com/playground) 来创建和测试您的第一个提示词模板。如需分步指导，请参阅[创建提示词](/langsmith/create-a-prompt)。
</Callout>

以下部分描述了提示词模板的关键方面。

### F-string 与 Mustache

您可以使用 [f-string](https://realpython.com/python-f-strings/) 或 [mustache](https://mustache.github.io/mustache.5.html) 格式来格式化您的提示词模板中的输入变量。

有关如何在 Playground 中使用这些格式的详细信息，请参阅[模板格式](/langsmith/create-a-prompt#template-format)。

<Check>
[Playground](https://smith.langchain.com/playground) 默认使用 `f-string` 作为模板格式，但您可以在提示词设置/模板格式部分切换到 `mustache` 格式。`mustache` 在条件变量、循环和嵌套键方面为您提供更大的灵活性。对于条件变量，您需要手动在"输入"部分添加 JSON 变量。请阅读[文档](https://mustache.github.io/mustache.5.html)
</Check>

### 工具

[工具](/langsmith/use-tools)是 LLM 可以用来与外部世界交互的接口。工具由名称、描述和用于调用工具的 JSON schema 组成。

### 结构化输出

结构化输出是大多数先进 LLM 的功能，它不是生成原始文本作为输出，而是遵守指定的 schema。这可能使用也可能不使用[工具](#tools)。

<Check>
结构化输出与工具类似，但在几个关键方面有所不同。对于工具，LLM 选择调用哪个工具（或者可能选择不调用任何工具）；对于结构化输出，LLM **始终**以此格式响应。对于工具，LLM 可以选择**多个**工具；对于结构化输出，只会生成一个响应。
</Check>

### 模型

您可以选择将模型配置与提示词模板一起存储。这包括模型名称和任何其他参数（temperature 等）。

## 提示词版本控制

版本控制是与提示词协作迭代的关键组件。

### 提交

每次保存对提示词的更新都会创建一个带有唯一提交哈希的新提交。这允许您：

- 查看提示词更改的完整历史记录。
- 审查早期版本。
- 在需要时回滚到之前的状态。
- 使用提交哈希在代码中引用特定版本（例如，`client.pull_prompt("prompt_name:commit_hash")`）。

在 UI 中，您可以通过在[提示词详情页](/langsmith/manage-prompts#prompt-detail-page)右上角切换 **Diff** 来将提交与其先前版本进行比较。

### 标签

提交标签是指向提示词历史记录中特定提交的人类可读标签。与提交哈希不同，标签可以移动以指向不同的提交，允许您更新代码引用的版本而无需更改代码本身。

提交标签的用例包括：

- **环境**：`staging` 和 `production` 标签保留用于[环境](/langsmith/manage-prompts#environments)功能，该功能允许您在命名部署目标之间提升提交，并在不更改代码的情况下切换版本。
- **版本控制**：标记提示词的稳定版本，例如 `v1`、`v2`，让您可以在代码中引用特定版本并随时间跟踪更改。
- **协作**：标记供审查的版本，使您能够与协作者分享特定版本并获取反馈。

<Note>
**不要与资源标签混淆**：提交标签引用特定的提示词版本。[资源标签](/langsmith/set-up-resource-tags)是用于组织工作区资源的关键值对。
</Note>

有关创建和管理提交标签的详细信息，请参阅[管理提示词](/langsmith/manage-prompts#commit-tags)。

## Playground

Playground 提供了一个用于迭代和测试提示词的界面。您可以从侧边栏或直接从已保存的提示词访问 Playground。

在 Playground 中，您可以：

* 更改使用的模型
* 更改使用的提示词模板
* 更改输出 schema
* 更改可用的工具
* 输入要通过提示词模板运行的输入变量
* 通过模型运行提示词
* 观察输出

<Callout type="info" icon="feather">
使用 Playground 中的**[聊天](/langsmith/chat)**来获得 AI 辅助优化提示词、生成工具和创建输出 schema。
</Callout>
