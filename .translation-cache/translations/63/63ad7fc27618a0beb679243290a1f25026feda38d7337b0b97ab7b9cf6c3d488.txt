[LangSmith SDK](https://reference.langchain.com/python/langsmith/) 和 [REST API](/langsmith/smith-api-ref) 允许使用一组过滤参数和结构化过滤查询语言，以编程方式过滤、查询和导出[运行记录](/langsmith/observability-concepts#runs)。本文档记录了过滤参数和查询语言，并提供了常见查询的示例。

有关将这些过滤器与 SDK 结合使用的可运行端到端示例，请参阅[使用 SDK 查询轨迹](/langsmith/export-traces)。

## 过滤参数

| 键 | 描述 |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `project_id` / `project_name` | 要获取运行记录的项目，可以是单个项目或项目列表。 |
| `trace_id` | 获取属于特定轨迹的运行记录。 |
| `run_type` | 要获取的[运行类型](/langsmith/run-data-format#run-types)，例如 `llm`、`chain`、`tool`、`retriever`。 |
| `dataset_name` / `dataset_id` | 获取与指定数据集中某一行示例关联的运行记录。这对于比较给定数据集上的提示词或模型很有用。 |
| `reference_example_id` | 获取与特定示例行关联的运行记录。这对于比较给定输入上的提示词或模型很有用。 |
| `parent_run_id` | 获取作为给定运行子级的运行记录。这对于获取使用上下文管理器分组的运行记录或获取智能体轨迹很有用。 |
| `error` | 获取出错或未出错的运行记录。 |
| `run_ids` | 使用给定的运行 ID 列表获取运行记录。注意：**这将忽略所有其他过滤参数。** |
| `filter` | 获取匹配给定结构化过滤语句的运行记录。有关更多详细信息，请参阅[过滤查询语言](#filter-query-language)部分。 |
| `trace_filter` | 应用于轨迹根运行的过滤条件。与 `filter` 结合使用可根据根运行的属性进行筛选。 |
| `tree_filter` | 应用于轨迹树中任何运行（根、同级或子级）的过滤条件。与 `filter` 结合使用可根据轨迹内任何运行的属性进行筛选。 |
| `is_root` | 仅返回根运行。 |
| `select` | 选择要在响应中返回的字段。默认返回所有字段。可用字段请参阅[运行数据格式](/langsmith/run-data-format)。 |
| `query`（_实验性_） | 自然语言查询，将您的查询转换为过滤语句。 |

<Note>
**性能提示**：传递 `select` 参数并从列表中排除 `inputs` 和 `outputs` 可以显著提高查询性能并减小响应大小，特别是对于大型运行记录。
</Note>

## 过滤查询语言

LangSmith 支持使用过滤查询语言进行过滤功能，以在获取运行记录时执行复杂的过滤操作。这在使用 SDK 或 API 以编程方式查询轨迹时特别有用。例如，在[评估](/langsmith/evaluation)流水线、监控脚本或检查先前[轨迹](/langsmith/observability-concepts#traces)的智能体工作流中。

### 比较器

过滤语法基于应用于运行对象字段的比较器函数：

| 比较器 | 描述 | 示例 |
| ---------- | -------------------------------------------------------- | ------- |
| `eq` | 等于 | `eq(run_type, "llm")` |
| `neq` | 不等于 | `neq(status, "error")` |
| `gt` | 大于 | `gt(latency, "5s")` |
| `gte` | 大于或等于 | `gte(latency, 1.5)` |
| `lt` | 小于 | `lt(start_time, "2024-01-01T00:00:00Z")` |
| `lte` | 小于或等于 | `lte(feedback_score, 0.5)` |
| `has` | 检查运行是否包含标签或元数据键值对 | `has(tags, "production")` |
| `search` | 在所有字符串字段中搜索子字符串 | `search("invoice")` |
| `in` | 检查字段值是否在列表中 | `in(metadata_key, ["session_id", "thread_id"])` |

### 逻辑运算符

使用 `and` 和 `or` 组合多个比较器：

```text
and(eq(run_type, "llm"), gt(latency, "2s"))
or(eq(status, "error"), and(eq(feedback_key, "score"), lt(feedback_score, 0.5)))
```

### 可筛选字段

| 字段 | 类型 | 说明 |
| ----------------- | --------------- | ----- |
| `id` | string (UUID) | 运行 ID |
| `name` | string | 运行名称 |
| `run_type` | string | 值为 `llm`、`chain`、`tool`、`retriever`、`embedding`、`prompt`、`parser` 之一 |
| `status` | string | `"success"`、`"error"` 或 `"pending"`。用于筛选出错的运行与成功的运行。 |
| `start_time` | ISO 8601 字符串 | 例如 `"2024-01-15T00:00:00Z"` |
| `end_time` | ISO 8601 字符串 | |
| `latency` | 持续时间字符串或数字 | 秒数，例如 `"5s"`、`"1.5s"` 或 `1.5`。仅支持 `s` 后缀。 |
| `tags` | 字符串列表 | 使用 `has(tags, "value")` |
| `metadata_key` | string | 运行元数据字典中的键 |
| `metadata_value` | string | 运行元数据字典中的值 |
| `feedback_key` | string | 反馈分数的名称 |
| `feedback_score` | number | 反馈分数的数值 |

### 值格式化

- **字符串**：使用双引号或单引号包裹，如 `eq(name, "MyChain")` 或 `eq(name, 'MyChain')`。
- **时间戳**：ISO 8601 格式，如 `"2024-06-01T00:00:00Z"`。
- **持续时间**：秒为单位，可以是数字或带 `s` 后缀的字符串，如 `"5s"`、`"1.5s"`、`"90s"` 或 `1.5`。不支持其他单位后缀（`m`、`h`）。
- **列表**：JSON 数组语法，如 `["session_id", "conversation_id"]`。

## 快速参考示例

以下示例仅展示筛选字符串。请将该字符串作为 `filter`、`trace_filter` 或 `tree_filter` 参数传递给 `client.list_runs()` 或 `/runs/query` API 端点。

<Tip>
有关 Python、TypeScript 和 Java 代码的完整 SDK 示例，请参阅[使用 SDK 查询跟踪](/langsmith/export-traces#use-filter-query-language)。
</Tip>

### 按运行名称筛选

```text
eq(name, "my_chain")
```

### 按错误状态筛选

```text
# Runs that errored
eq(status, "error")

# Runs that did not error
eq(status, "success")
```

### 按延迟筛选

```text
# Runs slower than 5 seconds
gt(latency, "5s")

# Runs faster than 1 second
lt(latency, "1s")
```

### 按时间范围筛选

```text
and(gt(start_time, "2024-01-01T00:00:00Z"), lt(start_time, "2024-02-01T00:00:00Z"))
```

### 按标签筛选

```text
has(tags, "production")

# Multiple tags (any match)
or(has(tags, "production"), has(tags, "staging"))
```

### 按元数据键或值筛选

```text
# Runs with a "user_id" metadata key
eq(metadata_key, "user_id")

# Runs with a specific user ID value
and(eq(metadata_key, "user_id"), eq(metadata_value, "usr_abc123"))

# Runs from the production environment
and(eq(metadata_key, "environment"), eq(metadata_value, "production"))
```

### 按对话或主题 ID 筛选

```text
and(in(metadata_key, ["session_id", "conversation_id", "thread_id"]), eq(metadata_value, "<your_thread_id>"))
```

### 按反馈分数筛选

```text
# Runs with a "thumbs_up" score of 1
and(eq(feedback_key, "thumbs_up"), eq(feedback_score, 1))

# Runs with a "correctness" score below 0.5
and(eq(feedback_key, "correctness"), lt(feedback_score, 0.5))
```

### 对所有字符串字段进行全文搜索

```text
search("my search term")
```

### 组合条件

```text
# Errors that started after a specific time
and(gt(start_time, "2024-06-01T00:00:00Z"), eq(status, "error"))

# Slow LLM runs with low correctness feedback
and(gt(latency, "10s"), eq(feedback_key, "correctness"), lt(feedback_score, 0.5))

# Complex: errors OR low score, both starting after a timestamp
and(gt(start_time, "2023-07-15T12:34:56Z"), or(eq(status, "error"), and(eq(feedback_key, "Correctness"), eq(feedback_score, 0.0))))
```

### 使用 trace_filter 和 tree_filter

`filter` 适用于返回的运行。`trace_filter` 适用于跟踪的根运行。`tree_filter` 适用于跟踪树中任意位置的运行。

```text
# filter: the run you want
eq(name, "RetrieveDocs")

# trace_filter: condition on the root run (e.g. human feedback on the overall trace)
and(eq(feedback_key, "user_score"), eq(feedback_score, 1))

# tree_filter: condition on any run in the trace tree
eq(name, "ExpandQuery")
```

<Note>
`tree_filter` 将相同的查询语法应用于追踪树中任意位置的运行。对于任意嵌套的子运行字段的谓词（如返回的 `inputs`、`outputs` 或 `extra` 有效载荷路径），首先使用服务端过滤器缩小候选范围，然后用子运行填充根追踪，并在本地遍历它们。请参阅[使用子运行谓词查询追踪树](/langsmith/export-traces#query-trace-trees-with-child-run-predicates)。
</Note>
