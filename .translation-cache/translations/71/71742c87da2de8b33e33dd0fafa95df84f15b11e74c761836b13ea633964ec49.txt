## 监控导出状态

要监控导出任务的状态，请使用以下 cURL 命令：

```bash
curl --request GET \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports/{export_id}' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID'
```

将 `{export_id}` 替换为您要监控的导出任务的 ID。此命令用于获取指定导出任务的当前状态。

## 列出导出任务的所有运行

导出任务通常会被拆分为多个运行，每个运行对应一个特定的日期分区进行导出。要列出与特定导出任务关联的所有运行，请使用以下 cURL 命令：

```bash
curl --request GET \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports/{export_id}/runs' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID'
```

此命令获取与指定导出相关的所有运行信息，提供运行 ID、状态、创建时间、已导出行数等详细信息。

## 列出所有导出

要获取所有导出作业的列表，请使用以下 cURL 命令：

```bash
curl --request GET \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID'
```

此命令返回所有导出作业的列表，以及它们当前的状态和创建时间戳。

## 停止导出

要停止现有导出，请使用以下 cURL 命令：

```bash
curl --request PATCH \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports/{export_id}' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "status": "Cancelled"
}'
```

将 `{export_id}` 替换为您要取消的导出 ID。请注意，一旦作业被取消，将无法重新启动，您需要创建一个新的导出作业。

## 失败模式与重试策略

LangSmith 批量导出会自动处理临时性故障和基础设施问题，以确保弹性。

每个批量导出被划分为多个 _runs_，每个 run 处理特定日期分区的数据（通常按天组织）。Run 相互独立处理，这使得：

- 不同时间段可以并行处理。
- 每个 run 有独立的重试逻辑。
- 如果中断，可以从特定检查点恢复。

导出中的每个 run（日期范围）都有各自的[故障处理](#failure-scenarios)和[重试预算](#automatic-retry-behavior)。如果某个 run 在重试次数用尽后失败，则整个导出将被标记为 `FAILED`。

### 自动重试行为

导出作业会自动重试临时性故障，具体行为如下：

- **最大重试次数**：每个 run 20 次重试（可能更改）。
- **重试延迟**：每次尝试间隔 30 秒（固定，无指数退避）。
- **Run 超时**：每个 run 最长 4 小时。
- **整体工作流超时**：整个导出 72 小时。

### 故障场景

| 故障类型 | 原因 | 自动重试？ | 需要采取的操作 |
|----------|------|------------|----------------|
| **基础设施中断** | [部署](/langsmith/deployment)、服务器重启、工作进程崩溃 | 是，自动重新排队并保留剩余重试次数。 | 无，作业会自动恢复。 |
| **Run 超时** | 单个 run 超过 4 小时限制 | 是，最多重试 20 次（可能更改）。 | 如持续存在，请缩小日期范围、添加筛选条件或[限制导出的字段](/langsmith/data-export#limit-exported-fields)。 |
| **工作流超时** | 整个导出超过 72 小时 | 否 | 减少导出范围（日期范围、筛选条件）或拆分为更小的导出。 |
| **存储/目标错误** | [无效凭证](/langsmith/data-export-destinations#credentials-configuration)、[存储桶缺失](/langsmith/data-export-destinations#configuration-fields)、[权限问题](/langsmith/data-export-destinations#permissions-required) | 否 | 修复目标配置并创建新导出。 |
| **目标已删除** | 导出期间存储桶被移除 | 否 | 重新创建目标并重新启动导出。 |
| **最终处理错误** | 数据序列化问题、资源耗尽 | 是，最多重试 20 次（可能更改）。 | 检查 run 错误详情；可能需要进一步调查。 |

<Note>
任何单个 run 失败（重试次数用尽后）都会导致整个导出失败。
</Note>

### 导出状态生命周期

导出可以具有以下状态：

| 状态 | 描述 |
|------|------|
| `CREATED` | 导出已创建但尚未开始处理。 |
| `RUNNING` | 导出正在积极处理 runs。 |
| `COMPLETED` | 所有 runs 均已成功导出。 |
| `FAILED` | 一个或多个 runs 在重试次数用尽后失败。 |
| `CANCELLED` | 导出已被用户手动取消。 |
| `TIMEDOUT` | 导出超出了 48 小时的工作流超时限制。 |

单个 runs 可以具有相同的状态：`CREATED`、`RUNNING`、`COMPLETED`、`FAILED`、`CANCELLED` 或 `TIMEDOUT`。

### 并发与速率限制

为确保系统稳定性，导出受以下限制约束：

- **每个导出的最大并发 runs 数**：45
- **每个工作区的最大并发导出数**：15

如果您有多个导出正在运行，新的 run 作业将排队等待，直到有可用容量。

### 进度跟踪与可恢复性

导出系统为每个 run 维护详细的进度元数据：
- 数据流中的最新游标位置。
- 已导出的行数。
- 已写入的 Parquet 文件列表。

此进度跟踪支持：
- **优雅恢复**：如果某个 run 被中断（例如通过部署），它将从上一个检查点恢复，而不是重新开始。
- **进度监控**：通过 API 跟踪已导出多少数据。
- **高效重试**：失败的 runs 不会重新导出已成功写入的数据。

### 排查导出失败问题

如果您的导出失败，请按以下步骤操作：

1. **检查导出状态**：使用 [`GET /api/v1/bulk-exports/{export_id}` 端点](/langsmith/smith-api/bulk-exports/get-bulk-export) 检索导出详情和状态。
2. **查看 run 错误**：您可以使用 [List Runs API](#list-runs-for-an-export) 监控您的 runs。每个 run 都包含一个 `errors` 字段，其中包含按重试尝试（如 `retry_0`、`retry_1`）键控的详细错误消息。
3. **验证目标访问**：确保您的[目标存储桶](/langsmith/data-export-destinations#configuration-fields)仍然存在且[凭证](/langsmith/data-export-destinations#credentials-configuration)有效。
4. **检查 run 大小**：如果您看到超时错误，您的日期分区可能包含过多数据。[限制导出的字段](/langsmith/data-export#limit-exported-fields)可能会有所帮助。
5. **检查系统限制**：确保您没有达到[并发限制](#concurrency-and-rate-limits)（每个导出 5 个 runs，每个工作区 3 个导出）。

对于存储相关错误，您可以在重试导出之前使用 AWS CLI 或 gsutil 测试您的目标配置。
