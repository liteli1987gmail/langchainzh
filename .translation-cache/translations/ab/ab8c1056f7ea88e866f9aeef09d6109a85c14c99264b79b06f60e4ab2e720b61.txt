## 前置要求

在开始之前，请确认以下内容：

- 您的[组织管理员](/langsmith/rbac#organization-admin)已启用 LLM Gateway 并将提供商 API 密钥添加到工作区密钥中。要设置此项，请参阅[管理员设置](/langsmith/llm-gateway-admin-setup)。
- 您有一个附加了具有 `gateway:invoke` 和 `workspaces:read` [权限](/langsmith/organization-workspace-operations)角色的工作区范围的 [LangSmith API 密钥](/langsmith/create-account-api-key)。如不确定，请咨询您的组织管理员。

## 1. 设置环境变量

在终端中设置以下内容（或将其添加到您的 `~/.zshrc` 中以跨会话保持）：

```bash
export BASE_URL="https://gateway.smith.langchain.com"

export ANTHROPIC_BASE_URL="$BASE_URL/anthropic"
export OPENAI_BASE_URL="$BASE_URL/openai/v1"
export GOOGLE_GEMINI_BASE_URL="$BASE_URL/gemini"

export LANGSMITH_API_KEY="lsv2_..._....cbed3e"

export ANTHROPIC_API_KEY="$LANGSMITH_API_KEY"
export OPENAI_API_KEY="$LANGSMITH_API_KEY"
export GEMINI_API_KEY="$LANGSMITH_API_KEY"
export GOOGLE_API_KEY="$LANGSMITH_API_KEY"
```

这会将所有提供商 SDK 指向 LangSmith Gateway，并使用您的 LangSmith API 密钥进行身份验证。Gateway 会从您工作区中的提供商密钥（Provider Secrets）解析实际的提供商密钥——您永远不需要在本地存储提供商 API 密钥的副本。

## 2. 发起调用

<CodeGroup>

</CodeGroup>

```bash curl
curl https://gateway.smith.langchain.com/openai/v1/chat/completions \
    -H "Authorization: Bearer $LANGSMITH_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"ping"}]}'
```

```python OpenAI SDK
import os

from openai import OpenAI

client = OpenAI(
    base_url=os.environ["OPENAI_BASE_URL"],
    api_key=os.environ["LANGSMITH_API_KEY"],
)
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "ping"}],
)
print(response.choices[0].message.content)
```

```python Anthropic SDK
import os

import anthropic

client = anthropic.Anthropic(
    base_url=os.environ["ANTHROPIC_BASE_URL"],
    api_key=os.environ["LANGSMITH_API_KEY"],
)
message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "ping"}],
)
print(message.content[0].text)
```

```python LangChain
import os

from langchain.agents import create_agent
from langchain.chat_models import init_chat_model

# Anthropic shown here; OpenAI and Gemini are also supported by swapping base_url and model_provider.
model = init_chat_model(
    model="claude-sonnet-4-6",
    model_provider="anthropic",
    base_url=os.environ["ANTHROPIC_BASE_URL"],
    api_key=os.environ["LANGSMITH_API_KEY"],
)
agent = create_agent(model=model, system_prompt="You are a helpful assistant.")
result = agent.invoke({"messages": [{"role": "user", "content": "ping"}]})
print(result["messages"][-1].content)
```

</CodeGroup>

返回 `200` 响应并带有聊天补全，表示网关、您的 API 密钥和角色权限都正常工作。

## 3. 查看您的追踪记录

打开 [LangSmith UI](https://smith.langchain.com)，导航到与您的 API 密钥关联的工作区中名为 `gateway` 或 `gateway-<short_api_key>-<api_key_id>` 的追踪项目。您应该会看到刚刚调用的新追踪记录。

<Note>
如果您的应用程序也发送自己的 LangSmith 追踪（例如通过 [LangChain 或 LangGraph 追踪](/langsmith/observability)），网关追踪和您的应用程序追踪会显示为独立的运行。尚不支持将网关追踪链接到父应用程序运行。
</Note>

## 4. 设置消费策略（可选）

在 LangSmith 中进入 **设置 → 网关 → LLM 网关** 创建消费策略。例如，您可以为 API 密钥设置每日 $10 的上限。达到上限时，网关返回 `402` 响应，消息为：`"Request blocked by gateway policies: R&D Spend Cap"`。

有关策略维度、时间窗口和冲突解决的完整指南，请参阅[消费策略](/langsmith/llm-gateway-spend-policies)。

## 网关如何处理请求

以下是您发出调用时网关执行的操作：

1. **认证** 您的请求（使用 LangSmith API 密钥）。
1. **解析** 工作区提供商密钥中的实际提供商 API 密钥（例如 `OPENAI_API_KEY`）。
1. **评估** 任何活跃策略（消费限额、PII 删除、密钥删除）。
1. **代理** 请求到上游提供商。
1. **追踪** 调用到 LangSmith，包括令牌数量、成本和任何策略事件。

通过网关路由无需更改应用程序代码。
