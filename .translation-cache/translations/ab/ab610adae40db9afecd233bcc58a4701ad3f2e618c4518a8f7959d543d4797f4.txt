**Agent = Model + Harness。** LangChain 提供 `create_agent`：一个轻量级、高度可配置的框架。框架包含模型循环周围的所有内容：提示词、工具以及任何塑造行为的中间件。从基础组件开始，精确组合出你用例所需的内容。支持 [OpenAI、Anthropic、Google 等](/oss/integrations/providers/overview).

<Tip>
**LangChain vs. LangGraph vs. Deep Agents**

从 [Deep Agents](/oss/deepagents/overview/) 开始，这是一个"功能齐全"的 agent，具有自动上下文压缩、虚拟文件系统和子 agent 生成等功能。Deep Agents 基于 LangChain [agents](/oss/langchain/agents/) 构建，你也可以直接使用这些 agents。

使用 [LangChain](/oss/langchain/agents) (`create_agent`获取高度可定制的框架，可轻松适应你的用例和数据。

使用 [LangGraph](/oss/langgraph/overview)，我们的底层编排框架，满足结合确定性和 agent 工作流的高级需求。

使用 [LangSmith](/langsmith/observability) 用于追踪、调试和评估使用这些框架构建的智能体。请按照 [追踪快速入门](/langsmith/trace-with-langchain) 进行设置。我们还建议您设置 [LangSmith Engine](/langsmith/engine) 它可以监控您的追踪、检测问题并提出修复建议。
</Tip>

## <Icon icon="wand" /> 创建智能体

此示例演示了如何创建一个带有自定义工具的简单 LangChain 智能体：

:::python
<CodeGroup>
    ```python OpenAI
    # pip install -qU langchain "langchain[openai]"
    from langchain.agents import create_agent

    def get_weather(city: str) -> str:
        """Get weather for a given city."""
        return f"It's always sunny in {city}!"

    agent = create_agent(
        model="openai:gpt-5.5",
        tools=[get_weather],
        system_prompt="You are a helpful assistant",
    )

    result = agent.invoke(
        {"messages": [{"role": "user", "content": "What's the weather in San Francisco?"}]}
    )
    print(result["messages"][-1].content_blocks)
    ```

    ```ts OpenAI
    // First install: npm install langchain zod @langchain/openai
    import { createAgent, tool } from "langchain";
    import * as z from "zod";

    const getWeather = tool(
      (input) => `It's always sunny in ${input.city}!`,
      {
        name: "get_weather",
        description: "Get the weather for a given city",
        schema: z.object({
          city: z.string().describe("The city to get the weather for"),
        }),
      }
    );

    const agent = createAgent({
      model: "gpt-5.5",
      tools: [getWeather],
    });

    console.log(
      await agent.invoke({
        messages: [{ role: "user", content: "What's the weather in San Francisco?" }],
      })
    );
    ```

## <Icon icon="star" size={20} /> 核心优势

<Columns cols={2}>
    <Card title="Standard model interface" icon="refresh" href="/oss/langchain/models" arrow cta="Learn more">
        使用一个接口处理聊天模型、嵌入等跨提供商的功能。通过最少的代码更改切换模型，并随着需求的演变保持应用程序的可移植性。
    </Card>

    <Card title="Highly configurable harness" icon="wand" href="/oss/langchain/agents" arrow cta="Learn more">
        从 `create_agent` 开始作为最小化框架，并通过中间件逐步添加功能。只组合您的用例所需的内容，从防护栏和重试到路由和自定义工具策略。
    </Card>

    <Card title="Built on top of LangGraph" icon="/images/brand/langgraph-icon.png" href="/oss/langgraph/overview" arrow cta="Learn more">
        LangChain 的智能体建立在 LangGraph 之上。这使我们能够利用 LangGraph 的持久执行、人工干预支持、持久化等功能。
    </Card>

    <Card title="Debug with LangSmith" icon="/images/brand/observability-icon-dark.png" href="/langsmith/observability" arrow cta="Learn more">
        在一个地方检查追踪、工具调用、状态转换和延迟。通过执行数据发现故障模式、评估质量并改进智能体行为。
    </Card>
</Columns>
