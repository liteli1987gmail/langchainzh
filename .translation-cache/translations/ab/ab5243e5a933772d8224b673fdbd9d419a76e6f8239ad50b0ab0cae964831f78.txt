<Tip>
**前提条件**
本指南假定您熟悉 [LangSmith](/langsmith/observability), [持久化](/oss/langgraph/persistence)，以及 [跨线程持久化](/oss/langgraph/stores) concepts.
</Tip>

LangSmith 会持久化 [检查点](/oss/langgraph/checkpointers#checkpoints) （线程状态）和 [跨线程记忆](/oss/langgraph/stores) （存储项）。您可以在 [`langgraph.json`](/langsmith/application-structure#configuration-file) 中配置生存时间 (TTL) 策略，以自动管理此数据的生命周期，防止无限积累。

## 配置线程和检查点 TTL

检查点捕获对话线程的状态。设置 TTL 可确保旧的检查点和线程元数据被自动删除。

在您的 `checkpointer.ttl` 中添加 `langgraph.json` file:

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./agent.py:graph"
  },
  "checkpointer": {
    "ttl": {
      "strategy": "delete",
      "sweep_interval_minutes": 60,
      "default_ttl": 43200
    }
  }
}
```

- `strategy`：指定到期时采取的操作。
    - `"delete"`：TTL 到期时删除整个线程，包括所有相关的运行和检查点数据。
    - `"keep_latest"`：保留线程和最新检查点，但删除后续运行不需要的旧检查点数据。
- `sweep_interval_minutes`：定义系统检查过期检查点的频率（以分钟为单位）。
- `default_ttl`: Sets the default lifespan of threads (and corresponding checkpoints) in minutes (e.g., 43200 minutes = 30 days). Applies only to checkpoints created after this configuration is deployed; existing checkpoints/threads are not changed. To clear older data, delete it explicitly.

## 配置存储项 TTL

存储项允许跨线程数据持久化。为存储项配置 TTL 有助于通过删除过时数据来管理内存。

在您的 `store.ttl` 中添加 `langgraph.json` file:

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./agent.py:graph"
  },
  "store": {
    "ttl": {
      "refresh_on_read": true,
      "sweep_interval_minutes": 120,
      "default_ttl": 10080
    }
  }
}
```

- `refresh_on_read`：(可选，默认值 `true`) If `true`，通过 `get` or `search` 访问项会重置其过期计时器。如果 `false`，则 TTL 仅在 `put`.
- `sweep_interval_minutes`：(可选) 定义系统检查过期项的频率（以分钟为单位）。如果省略，则不进行清理。
- `default_ttl`：(可选) 设置存储项的默认生存时间（以分钟为单位，例如 10080 分钟 = 7 天）。仅适用于此配置部署后创建的项目；现有项目不会更改。如果需要清除旧项目，请手动删除。如果省略，项目默认不过期。

## 组合 TTL 配置

您可以在同一个 `langgraph.json` 文件中为检查点和存储项配置 TTL，为每种数据类型设置不同的策略。示例如下：

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./agent.py:graph"
  },
  "checkpointer": {
    "ttl": {
      "strategy": "delete",
      "sweep_interval_minutes": 60,
      "default_ttl": 43200
    }
  },
  "store": {
    "ttl": {
      "refresh_on_read": true,
      "sweep_interval_minutes": 120,
      "default_ttl": 10080
    }
  }
}
```

## 配置按线程 TTL

您可以为每个线程应用 [TTL 配置](https://reference.langchain.com/python/langsmith/deployment/sdk/#langgraph_sdk.client.ThreadsClient.create).

```python
thread = await client.threads.create(
    ttl={
        "strategy": "delete",
        "ttl": 43200  # 30 days in minutes
    }
)
```

<Note>
线程级 TTL 也会删除所有关联的检查点。因此，您可以设置线程级 TTL 而无需单独为检查点设置 TTL。
</Note>

## 运行时覆盖

来自 `store.ttl` 的默认 `langgraph.json` 设置可以在运行时通过在 SDK 方法调用中提供特定的 TTL 值来覆盖，如 `get`, `put`和 `search`.

## 部署流程

在 `langgraph.json`中配置 TTL 后，部署或重启您的 LangGraph 应用以使更改生效。使用 [`langgraph dev`](/langsmith/local-dev-testing#langgraph-dev) 用于本地开发或 [`langgraph up`](/langsmith/local-dev-testing#langgraph-up) 用于 Docker 部署。

有关其他可配置选项的详细信息，请参阅 [LangGraph CLI 参考页面](/langsmith/cli#configuration-file).
