_助手_ 是一个 [Agent Server](/langsmith/agent-server) 概念，允许您将配置（例如提示词、LLM选择、工具）与图形核心逻辑分开管理。这使您能够创建同一图形架构的多个专门版本，在运行时具有不同的行为。通过配置变体（而非结构性的图形更改），每个助手都针对不同的 [用例进行优化](#use-cases).

例如，设想一个基于通用图形架构构建的通用写作代理。虽然结构保持不变，但不同的写作风格（如博客文章和推文）需要定制的配置来优化性能。为了支持这些变体，您可以创建多个助手（例如，一个用于博客，另一个用于推文），它们共享底层图形，但在模型选择和系统提示词方面有所不同。

![助手版本](/langsmith/images/assistants.png)

Agent Server API 提供了多个端点来创建和管理助手及其版本。请参阅 [API 参考](/langsmith/server-api-ref) 了解更多详情。

<Info>
助手是一种 [LangSmith 部署](/langsmith/deployment) 概念。它们在开源 LangGraph 库中不可用。
</Info>

## 默认助手

当您使用 LangSmith 部署部署图形时， [Agent Server](/langsmith/agent-server) 会自动创建一个 **默认助手** 绑定到该图形的默认配置。然后，您可以为同一图形创建其他助手，每个助手都有自己的配置。

如果您的部署在 [`langgraph.json`](/langsmith/application-structure#configuration-file)中定义了多个图形，每个图形都会获得自己的默认助手：

```json
{
    "graphs": {
        "graph_id_1": "path_to_graph_id_1",  // default assistant created for graph_id_1
        "graph_id_2": "path_to_graph_id_2"   // default assistant created for graph_id_2
    }
}
```

助手具有以下几个关键特性：

- **[通过 API 和 UI 管理](/langsmith/configuration-cloud)**: Create, list, update, version, and get assistants using the Agent Server/LangGraph SDKs or the [LangSmith UI](https://smith.langchain.com).
- **一个图形，多个助手**：单个已部署的图形可以支持多个助手，每个助手具有不同的配置（例如提示词、模型、工具）。
- **[版本化](#versioning) 配置**：每个助手通过版本控制维护自己的配置历史记录。编辑助手会创建新版本，您可以升级或回滚到任何版本。
- **[配置](#configuration) 更新，无需更改图形**：通过助手配置更新提示词、模型选择和其他设置，从而无需修改或重新部署图形代码即可实现快速迭代。

<Note>
调用助手时，您可以指定 [`langgraph.json`](/langsmith/application-structure#configuration-file):

- A **图形 ID** （中的键 `langgraph.json`, e.g., `"agent"`）：使用该图形的默认助手。
- An **助手 ID** （UUID）：使用特定的助手配置。

这种灵活性使您能够使用默认设置快速测试，或精确控制使用的配置。
</Note>

## 配置

助手基于 LangGraph 开源概念的 [配置](/oss/langgraph/graph-api#runtime-context).

虽然配置在开源 LangGraph 库中可用，但助手仅存在于 [LangSmith 部署](/langsmith/deployment) 中，因为它们与您部署的图紧密耦合。部署后， [Agent Server](/langsmith/agent-server) 将自动使用图的默认配置设置为每个图创建一个默认助手。

实际上，助手只是一个具有特定配置的图的 _实例_ 。因此，多个助手可以引用同一个图，但可以包含不同的配置（例如提示词、模型、工具）。LangSmith 部署 API 提供了多个端点用于创建和管理助手。请参阅 [API 参考](/langsmith/server-api-ref) 和 [本操作指南](/langsmith/configuration-cloud) 以获取有关如何创建助手的更多详细信息。

### 使用场景

当您需要使用不同配置部署相同的图架构时，助手是理想的选择。常见的使用场景包括：

- **用户级个性化**
  - 根据每个用户自定义模型选择、系统提示词或工具可用性。
  - 存储用户偏好并在每次交互时自动应用。
  - 让用户能够在不同的 AI 个性或专业水平之间进行选择。

- **客户或组织特定配置**
  - 为不同客户或组织维护单独的配置文件。
  - 为每个客户端自定义行为，而无需部署独立的基础设施。
  - 将配置变更隔离到特定客户。

- **环境特定配置**
  - 为开发、预发布和生产环境使用不同的模型或设置。
  - 在预发布环境中测试配置变更后再推广到生产环境。
  - 在非生产环境中使用更小的模型来降低成本。

- **A/B testing and experimentation**
  - 比较不同的提示词、模型或参数设置。
  - 逐步向部分用户推出配置变更。
  - 测量不同配置变体之间的性能差异。

- **专门任务变体**
  - 创建通用代理的领域特定版本。
  - 针对不同语言、地区或行业优化配置。
  - 在保持一致的图逻辑的同时改变执行细节。

## 助手如何与部署配合工作

当您使用 LangSmith 部署部署图时， [Agent Server](/langsmith/agent-server) 自动创建一个 **默认助手** ，绑定到该图的默认配置。您随后可以为同一图创建额外的助手，每个助手都有各自的配置。

如果您的部署在 [`langgraph.json`](/langsmith/application-structure#configuration-file)中定义了多个图，每个图都会有自己的默认助手：

```json
{
    "graphs": {
        "graph_id_1": "path_to_graph_id_1",  // default assistant created for graph_id_1
        "graph_id_2": "path_to_graph_id_2"   // default assistant created for graph_id_2
    }
}
```

也就是说，可以有多个默认助手——每个在部署中定义的图都有一个。

助手具有以下几个关键特性：

- **[通过 API 和 UI 管理](/langsmith/configuration-cloud)**: Create, list, update, version, and get assistants using the Agent Server/LangGraph SDKs or the [LangSmith UI](https://smith.langchain.com).
- **一个图，多个助手**：单个已部署的图可以支持多个助手，每个助手具有不同的配置（例如提示词、模型、工具）。
- **[版本控制](#versioning) 配置**：每个助手通过版本控制维护自己的配置历史。编辑助手会创建新版本，您可以升级或回滚到任何版本。
- **[配置](#configuration) 更新无需更改图**：通过助手配置更新提示词、模型选择和其他设置，实现快速迭代，无需修改或重新部署图代码。

<Note>
调用助手时，您可以指定 [`langgraph.json`](/langsmith/application-structure#configuration-file):
- A **图 ID** (e.g., `"agent"`）：使用该图的默认助手
- An **助手 ID** （UUID）：使用特定的助手配置

这种灵活性使您能够快速测试默认设置或精确控制使用的配置。
</Note>

### 配置

助手基于 LangGraph 开源概念中的 [配置](/oss/langgraph/graph-api#runtime-context).

虽然配置在开源 LangGraph 库中可用，但助手仅在 [LangSmith 部署](/langsmith/deployment) 中可用，因为它们与您部署的图紧密耦合。部署时， [Agent Server](/langsmith/agent-server) 将使用图的默认配置设置自动为每个图创建一个默认助手。

实际上，助手只是具有特定配置的图的 _实例_ 。因此，多个助手可以引用同一个图，但可以包含不同的配置（例如提示词、模型、工具）。LangSmith 部署 API 提供了多个端点来创建和管理助手。请参阅 [API 参考](/langsmith/server-api-ref) 和 [本操作指南](/langsmith/configuration-cloud) 了解更多关于如何创建助手的信息。

### 版本控制

助手支持版本控制以跟踪随时间的变化。创建助手后，后续编辑将自动创建新版本。

- 每次更新都会创建助手的新版本。
- 您可以将任何版本升级为活动版本。
- 回滚到之前的版本就像设置其为活动版本一样简单。
- 所有版本都保持可用，以便参考和回滚。

<Warning>
更新助手时，您必须提供完整的配置负载。更新端点从头创建新版本，不会与之前的版本合并。请确保包含您要保留的所有配置字段。
</Warning>

有关如何管理助手版本的更多详细信息，请参阅 [管理助手指南](/langsmith/configuration-cloud#create-a-new-version-for-your-assistant).
