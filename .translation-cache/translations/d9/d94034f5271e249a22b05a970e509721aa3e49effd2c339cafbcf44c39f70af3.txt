**助手 (Assistants)** 是一种 [Agent Server](/langsmith/agent-server) 概念，它允许您将配置（例如提示词、LLM 选择、工具）与图的核心逻辑分离管理。这使您能够创建同一图架构的多个专用版本，并在运行时具有不同的行为。通过配置变体（而非结构性的图变更），每个助手都可以针对不同的[用例](#use-cases)进行优化。

例如，假设您构建了一个基于通用图架构的写作代理。虽然结构保持不变，但不同的写作风格（如博客文章和推文）需要定制化的配置以优化性能。为了支持这些变体，您可以创建多个助手（例如，一个用于博客，另一个用于推文），它们共享底层图，但在模型选择和系统提示词上有所不同。

![assistant versions](/langsmith/images/assistants.png)

Agent Server API 提供了多个端点用于创建和管理助手及其版本。有关更多详情，请参阅 [API 参考文档](/langsmith/server-api-ref)。

<Info>
助手是 [LangSmith 部署](/langsmith/deployment) 的概念。它们在开源 LangGraph 库中不可用。
</Info>

## 默认助手

当您使用 LangSmith 部署部署一个图时，[Agent Server](/langsmith/agent-server) 会自动创建一个与该图的默认配置绑定的**默认助手**。然后，您可以为同一个图创建额外的助手，每个助手都有自己的配置。

如果您的部署在 [`langgraph.json`](/langsmith/application-structure#configuration-file) 中定义了多个图，则每个图都会有自己的默认助手：

```json
{
    "graphs": {
        "graph_id_1": "path_to_graph_id_1",  // default assistant created for graph_id_1
        "graph_id_2": "path_to_graph_id_2"   // default assistant created for graph_id_2
    }
}
```

助手具有以下几个关键特性：

- **[通过 API 和 UI 管理](/langsmith/configuration-cloud)**：使用 Agent Server/LangGraph SDK 或 [LangSmith UI](https://smith.langchain.com) 创建、列出、更新、版本控制和获取助手。
- **一个图，多个助手**：单个已部署的图可以支持多个助手，每个助手具有不同的配置（例如，提示词、模型、工具）。
- **[版本化](#versioning) 配置**：每个助手通过版本控制维护自己的配置历史记录。编辑助手会创建新版本，你可以提升或回滚到任何版本。
- **[配置](#configuration) 更新无需修改图**：通过助手配置更新提示词、模型选择和其他设置，无需修改或重新部署图代码。

<Note>
调用助手时，你可以在 [`langgraph.json`](/langsmith/application-structure#configuration-file) 中指定以下任一内容：

- **图 ID**（`langgraph.json` 中的键，例如 `"agent"`）：使用该图的默认助手。
- **助手 ID**（UUID）：使用特定助手配置。

这种灵活性使你能够快速使用默认设置进行测试，或精确控制所使用的配置。
</Note>

## 配置

助手建立在 LangGraph 开源概念的 [配置](/oss/langgraph/graph-api#runtime-context) 基础之上。

虽然开源 LangGraph 库中提供了配置功能，但助手仅存在于 [LangSmith 部署](/langsmith/deployment) 中，因为它们与已部署的图紧密耦合。部署时，[Agent Server](/langsmith/agent-server) 会使用图的默认配置设置自动为每个图创建一个默认助手。

在实践中，助手只是具有特定配置的图的 _实例_。因此，多个助手可以引用同一个图，但可以包含不同的配置（例如，提示词、模型、工具）。LangSmith 部署 API 提供了多个用于创建和管理助手的端点。有关创建助手的更多详细信息，请参阅 [API 参考](/langsmith/server-api-ref) 和[此操作指南](/langsmith/configuration-cloud)。

### 使用场景

当你需要使用不同配置部署相同的图架构时，助手是理想的选择。常见使用场景包括：

- **用户级个性化**
  - 根据用户自定义模型选择、系统提示词或工具可用性。
  - 存储用户偏好并在每次交互中自动应用。
  - 让用户在不同 AI 个性或专业水平之间进行选择。

- **客户或组织特定配置**
  - 为不同客户或组织维护单独的配置。
  - 无需部署单独的基础架构即可为每个客户端定制行为。
  - 将配置变更隔离到特定客户。

- **Environment-specific configurations**
  - Use different models or settings for development, staging, and production.
  - Test configuration changes in staging before promoting to production.
  - Reduce costs in non-production environments with smaller models.

- **A/B testing and experimentation**
  - Compare different prompts, models, or parameter settings.
  - Roll out configuration changes gradually to a subset of users.
  - Measure performance differences between configuration variants.

- **Specialized task variants**
  - Create domain-specific versions of a general-purpose agent.
  - Optimize configurations for different languages, regions, or industries.
  - Maintain consistent graph logic while varying the execution details.

## 助手如何与部署配合工作

使用 LangSmith Deployment 部署图时，[Agent Server](/langsmith/agent-server) 会自动创建一个绑定到该图默认配置的**默认助手**。然后，您可以为同一个图创建额外的助手，每个助手都有自己的配置。

如果您的部署在 [`langgraph.json`](/langsmith/application-structure#configuration-file) 中定义了多个图，每个图都会有自己的默认助手：

```json
{
    "graphs": {
        "graph_id_1": "path_to_graph_id_1",  // default assistant created for graph_id_1
        "graph_id_2": "path_to_graph_id_2"   // default assistant created for graph_id_2
    }
}
```

也就是说，对于部署中定义的每个图，都可以有多个默认助手。

助手具有以下几个关键特性：

- **[通过 API 和 UI 进行管理](/langsmith/configuration-cloud)**：使用 Agent Server/LangGraph SDK 或 [LangSmith UI](https://smith.langchain.com) 创建、列出、更新、版本控制和获取助手。
- **一个图，多个助手**：单个已部署的图可以支持多个助手，每个助手具有不同的配置（例如提示词、模型、工具）。
- **[版本化](#versioning) 配置**：每个助手都通过版本控制维护自己的配置历史记录。编辑助手会创建新版本，您可以提升或回滚到任何版本。
- **[无需更改图即可更新配置](#configuration)**：通过助手配置更新提示词、模型选择和其他设置，实现快速迭代，而无需修改或重新部署图代码。

<Note>
调用助手时，可以在 [`langgraph.json`](/langsmith/application-structure#configuration-file) 中指定：
- **图 ID**（例如 `"agent"`）：使用该图的默认助手
- **助手 ID**（UUID）：使用特定助手配置

这种灵活性允许您使用默认设置快速测试，或精确控制所使用的配置。
</Note>

### 配置

助手基于 LangGraph 开源概念中的[配置](/oss/langgraph/graph-api#runtime-context)构建。

虽然配置在开源 LangGraph 库中可用，但助手仅存在于 [LangSmith Deployment](/langsmith/deployment) 中，因为它们与已部署的图紧密耦合。部署后，[Agent Server](/langsmith/agent-server) 将使用图的默认配置设置自动为每个图创建一个默认助手。

在实践中，助手只是具有特定配置的图的*实例*。因此，多个助手可以引用同一个图，但可以包含不同的配置（例如提示词、模型、工具）。LangSmith Deployment API 提供了多个端点用于创建和管理助手。请参阅 [API 参考](/langsmith/server-api-ref) 和[此操作指南](/langsmith/configuration-cloud)了解如何创建助手的更多详细信息。

### 版本控制

助手支持版本控制以跟踪随时间的变更。创建助手后，后续编辑将自动创建新版本。

- 每次更新都会创建新版本的助手。
- 您可以将任何版本提升为活动版本。
- 回滚到之前的版本就像将其设置为主动版本一样简单。
- 所有版本都保持可用以供参考和回滚。

<Warning>
更新助手时，必须提供完整的配置负载。更新端点会从头创建新版本，不会与之前的版本合并。请确保包含您希望保留的所有配置字段。
</Warning>

有关如何管理助手版本的更多详细信息，请参阅[管理助手指南](/langsmith/configuration-cloud#create-a-new-version-for-your-assistant)。
