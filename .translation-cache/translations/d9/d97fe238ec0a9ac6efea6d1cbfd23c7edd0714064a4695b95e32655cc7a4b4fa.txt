>[Redis](https://redis.io/docs/latest/) 是一个内存数据平台，具有向量搜索、全文搜索和语义缓存功能，专为实时 AI 应用而设计。

本笔记本展示了如何使用与 `RedisVectorStore`.

## 设置

要使用 `RedisVectorStore` ，您首先需要安装合作伙伴包，以及本笔记本中使用的其他包。

```bash
pip install -qU langchain-redis langchain-openai
```

您需要一个运行中的 Redis 实例，并启用 Redis Search 功能。我们推荐 Redis 8.4+。您可以使用 Docker 在本地启动一个：

```bash
docker run -d --name redis -p 6379:6379 redis:8.4
```

或使用 [Redis Cloud](https://redis.io/cloud/) 进行托管部署。

（可选）下载 [RedisInsight](https://redis.io/insight/) — 一个用于检查数据和索引的浏览器图形界面。

```bash
docker run -d --name redisinsight -p 5540:5540 redis/redisinsight:latest
```

### 凭证

通过设置以下环境变量来配置 Redis 连接：

```bash
export REDIS_URL="redis://localhost:6379"
```

对于需要认证的 Redis：

```bash
export REDIS_URL="redis://username:password@localhost:6379"
```

对于 Redis Cloud，请使用数据库仪表板中的连接字符串：

```bash
export REDIS_URL="redis://default:<password>@<host>:<port>"
```

This package also supports SSL/TLS (`rediss://`）和高可用性 Redis Sentinel 部署（`redis+sentinel://`）。请参阅 [langchain-redis README](https://github.com/langchain-ai/langchain-redis/blob/main/libs/redis/README.md#redis-connection-options) 获取完整的连接选项。

设置您的 OpenAI API 密钥：

```bash
export OPENAI_API_KEY="your-api-key-here"
```

<Tip>
该 [`langchain-redis`](https://github.com/langchain-ai/langchain-redis/blob/main/libs/redis/README.md) 包还提供 `RedisChatMessageHistory` 用于对话内存，以及 `RedisSemanticCache` 用于 LLM 响应缓存 — 所有功能都由与您的向量存储相同的 Redis 实例支持。
</Tip>

## 初始化

```python
import os
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
```

```python
from langchain_redis import RedisConfig, RedisVectorStore

config = RedisConfig(
    index_name="my_vectors",
    redis_url=os.getenv("REDIS_URL", "redis://localhost:6379"),
    distance_metric="COSINE",  # Options: COSINE, L2, IP
    metadata_schema=[
        {"name": "baz", "type": "tag"},
        {"name": "foo", "type": "tag"},
    ]
)

vector_store = RedisVectorStore(embeddings=embeddings, config=config)
```

<Note>
该 `metadata_schema` 参数告诉 Redis 要索引哪些元数据字段。未在此列出的字段不能用于过滤器。对于分类字符串值使用 `"tag"` ，对于数字使用 `"numeric"` 。
</Note>

## 管理向量存储

### 向向量存储添加项目

```python
texts = ["foo", "bar"]
metadatas = [{"baz": "bar"}, {"foo": "baz"}]
vector_store.add_texts(texts, metadatas=metadatas)
```

您也可以使用自定义键添加文档：

```python
custom_keys = ["doc1", "doc2"]
vector_store.add_texts(texts, metadatas=metadatas, keys=custom_keys)
```

或与 LangChain 一起使用 `add_documents`  `Document` objects:

```python
from langchain_core.documents import Document

document_1 = Document(page_content="foo", metadata={"baz": "bar"})
document_2 = Document(page_content="bar", metadata={"foo": "baz"})
document_3 = Document(page_content="to be deleted")

documents = [document_1, document_2, document_3]
ids = ["1", "2", "3"]
vector_store.add_documents(documents=documents, ids=ids)
```

### 从向量存储删除项目

```python
vector_store.delete(ids=["3"])
```

## 查询向量存储

一旦您的向量存储创建完成并添加了相关文档，您很可能希望在链或代理运行期间对其进行查询。

### 直接查询

执行简单的相似性搜索可以如下进行：

```python
query = "foo"
docs = vector_store.similarity_search(query, k=2)
for doc in docs:
    print(f"* {doc.page_content} [{doc.metadata}]")
```

#### 带评分的相似性搜索

```python
docs_and_scores = vector_store.similarity_search_with_score(query, k=2)
for doc, score in docs_and_scores:
    print(f"* [SIM={score:.3f}] {doc.page_content} [{doc.metadata}]")
```

#### 带过滤的相似性搜索

```python
from redisvl.query.filter import Tag

results = vector_store.similarity_search(
    query="foo",
    k=1,
    filter=Tag("baz") == "bar"
)
for doc in results:
    print(f"* {doc.page_content} [{doc.metadata}]")
```

#### 最大边际相关性搜索

```python
docs = vector_store.max_marginal_relevance_search(query, k=2, fetch_k=10)
for doc in docs:
    print(f"* {doc.page_content} [{doc.metadata}]")
```

#### 其他搜索方法

本笔记本未列出更多搜索方法，要查找所有方法，请务必阅读 @[`API reference`][RedisVectorStore]。

<Tip>
Redis 支持将向量相似性与全文搜索相结合的混合搜索。参见 @[`API reference`][RedisVectorStore] 获取 `similarity_search` 参数包括 `return_all` 和 `distance_threshold`.
</Tip>

### 通过转换为检索器进行查询

您还可以将向量存储转换为检索器，以便在链中更轻松地使用。

```python
retriever = vector_store.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 1, "fetch_k": 2, "lambda_mult": 0.5},
)
retriever.invoke("your query here")
```
