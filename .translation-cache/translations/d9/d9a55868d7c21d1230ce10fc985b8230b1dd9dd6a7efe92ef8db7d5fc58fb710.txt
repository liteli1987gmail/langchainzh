<Note>
本指南假设您已阅读[管理员指南](/langsmith/administration-overview)和[组织设置指南](/langsmith/set-up-hierarchy#set-up-an-organization)。
</Note>

LangSmith 提供额外的用户管理自定义功能，使用功能开关。

## 功能

### 工作区级邀请加入组织

LangSmith 中的默认行为要求用户必须是组织管理员才能邀请新用户加入组织。对于希望将此责任委托给工作区管理员的自托管客户，可以设置功能开关，使工作区管理员能够在**工作区级别**邀请新用户加入组织及其特定工作区。

启用此功能后，工作区管理员可以在`设置` > `工作区`下的`工作区成员`标签页中添加新用户。在工作区级别邀请时支持以下两种情况，而组织级别的邀请功能与之前相同。

1. 邀请尚未在组织中激活的用户：这将把用户作为待处理状态添加到组织及特定工作区
2. 邀请已在组织中激活的用户：直接将用户作为活跃成员添加到工作区（无待处理状态）。

管理员可以同时为这两种情况邀请用户。

#### 配置

<CodeGroup>

```yaml Helm
config:
  workspaceScopeOrgInvitesEnabled: true
```

```bash Docker
# In your .env file
WORKSPACE_SCOPE_ORG_INVITES_ENABLED="true"
```

### SSO 新成员登录流程

从 helm **v0.11.10** 开始，使用 OAuth SSO 的自托管部署将不再需要手动在 LangSmith 设置中添加成员以使其加入。部署将有一个 <b>默认</b> 组织，新用户在其首次登录 LangSmith 时将自动被添加到该组织中。

对于您的 **默认** 组织，您可以设置分配给新成员的 workspace(s) 和 workspace 角色。对于 **非默认** 组织，邀请流程保持不变。
用户加入组织后，对其 workspace 或角色的任何更改（超出默认组织设置的）都必须通过 LangSmith 设置（如之前所述）或通过 SCIM 进行管理。
<Note>
默认情况下，所有新用户都会被添加到组织最初配置的 workspace（默认为 **Workspace 1**），并被赋予 **Workspace 编辑者** 角色。
</Note>
![更新 SSO 成员设置](/langsmith/images/sso-member-settings-update.png)

<Note>
要更改默认组织，请在组织选择器下拉菜单中使用 **设为默认组织**。（源组织和目标组织都需要组织管理员权限。）
</Note>

### SSO 群组同步

<Note>
自托管上的 SSO 群组同步需要 LangSmith chart 版本 **0.15.0-rc.3**（应用版本 **0.15.2rc1**）或更高版本。
</Note>

[SSO 群组同步](/langsmith/user-management#sso-groups-sync-alternative) 从 OIDC ID 令牌中读取群组成员资格，并使用 [SCIM 命名约定](/langsmith/user-management#group-naming-convention) 分配组织和 workspace 角色。对于自托管组织而言，如果其 IdP 可以在 OIDC 令牌中包含群组但无法轻松运行 SCIM 配置，这是一个比 [SCIM](/langsmith/user-management#set-up-scim-for-your-organization) 更简单的替代方案。

有关 IdP 端配置（声明、范围），请参阅 [OIDC SSO 设置指南中的 SSO 群组同步部分](/langsmith/self-host-sso#sso-groups-sync)。有关设置参考和行为，请参阅 [SSO 群组同步主要文档](/langsmith/user-management#sso-groups-sync-alternative)。

### 禁用组织创建

默认情况下，任何用户都可以在 LangSmith 中创建组织。对于自托管客户，管理员可能希望在设置初始组织后限制此能力。此功能标志允许管理员禁用用户创建新组织的能力。

#### 配置

<Note>
对于使用 [基本认证](/langsmith/self-host-basic-auth) 或 [SSO](/langsmith/self-host-sso) 的组织，`userOrgCreationDisabled` 功能标志默认设置为 `true`。
</Note>
<CodeGroup>

```yaml Helm
config:
  userOrgCreationDisabled: true
```

```bash Docker
# In your .env file
FF_ORG_CREATION_DISABLED="true"
```

### 禁用个人组织

默认情况下，任何登录 LangSmith 的用户都会自动创建一个个人组织。对于自托管客户，管理员可能希望限制此功能。此功能标志允许管理员禁用用户创建个人组织的能力。

#### 配置

<Note>
对于使用[基本认证](/langsmith/self-host-basic-auth)或 [SSO](/langsmith/self-host-sso) 的组织，`personalOrgsDisabled` 功能标志默认设置为 `true`。
</Note>

<CodeGroup>

```yaml Helm
config:
  personalOrgsDisabled: true
```

```bash Docker
# In your .env file
FF_PERSONAL_ORGS_DISABLED="true"
```

</CodeGroup>

### 禁用个人访问令牌创建

<Note>
此功能需要 Helm chart 版本 0.13.12（应用版本 0.13.12）或更高版本。
</Note>

默认情况下，用户可以在任何组织中创建个人访问令牌 (PAT)。对于自托管客户，管理员可能希望在整个实例范围内全局禁用 PAT 创建。此环境变量允许管理员阻止用户在实例上的任何组织中创建新的 PAT。

如需仅为单个组织禁用 PAT 创建，请参阅[按组织 API 选项](/langsmith/manage-organization-by-api#security-settings)。

#### 配置

<CodeGroup>

</CodeGroup>

```yaml Helm
commonEnv:
  - name: PAT_CREATION_DISABLED
    value: "true"
```

```bash Docker
# In your .env file
PAT_CREATION_DISABLED="true"
```

</CodeGroup>
