Deep Agents 是开始构建由 LLM 驱动的代理和应用程序的最简单方式——内置任务规划、用于上下文管理的文件系统、子代理生成和长期记忆功能。
您可以将深度代理用于任何任务，包括复杂的多步骤任务。

Deep Agents 具有以下内置功能：

- **在环境中执行操作**：通过工具执行操作、读写文件、执行代码
- **连接您的数据**：在合适的时机加载记忆、技能和领域知识
- **管理不断增长的上下文**：在长时间运行中总结历史并卸载大型结果
- **并行化任务**：委托给在隔离上下文窗口中运行的一般或专业子代理
- **保持信息同步**：在关键决策点暂停以等待人工批准
- **随时间改进**：根据实际使用情况更新记忆、技能和提示

请参阅[核心能力](#core-capabilities)了解每个组件的完整分解。

## 执行环境

执行环境是代理运行的地方。它有四个层级：

- **[工具](#tools-and-mcp)**：代理可以调用的自定义函数、API 和数据库
- **[虚拟文件系统](#virtual-filesystem-access)**：由可插拔后端支持的文件工具
- **[文件系统权限](#filesystem-permissions)**：对代理可以读取或写入的路径的声明式访问控制
- **[代码执行](#code-execution)**：沙盒化 shell 执行和进程内 JavaScript 解释器

**[流式传输](#streaming)** 允许您使用消息、工具、值和委托任务的类型化事件流来跟踪所有发生的事情。

### 工具和 MCP

通过 `tools=` 参数传递自定义函数、LangChain 工具或来自任何 [MCP 服务器](/oss/deepagents/tools#mcp-tools) 的工具。Deep Agents 完全支持 [Model Context Protocol (MCP)](/oss/langchain/mcp)，让您可以通过标准接口连接数据库、API、文件系统等。

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[search, fetch_page, run_query],
)
```

有关定义自定义工具、使用 MCP 服务器以及内置 harness 工具完整列表的更多信息，请参阅[工具](/oss/deepagents/tools)。

### 虚拟文件系统访问

Harness 提供了一个可配置的虚拟文件系统，支持多种[可插拔后端](/oss/deepagents/backends)：内存状态、本地磁盘、LangGraph store、复合路由，或带有[权限规则](/oss/deepagents/permissions)的自定义后端，用于控制读写访问。

后端支持以下文件系统操作：

| 工具 | 描述 |
| ---- | ----------- |
| `ls` | 列出目录中的文件及其元数据（大小、修改时间） |
| `read_file` | 读取文件内容并显示行号，支持大文件的偏移量/限制读取。对于非文本文件（图片、视频、音频和文档），还支持返回多模态内容块。详见下方支持的文件扩展名。 |
| `write_file` | 创建新文件 |
| `edit_file` | 对文件执行精确字符串替换（支持全局替换模式） |
| `glob` | 查找匹配模式的文件（例如 `**/*.py`） |
| `grep` | 搜索文件内容，支持多种输出模式（仅文件、带上下文的文件名和内容，或计数） |
| `execute` | 在环境中运行 shell 命令（仅在[沙箱后端](/oss/deepagents/sandboxes)下可用） |

<Accordion title="支持的多模态文件扩展名">

| 类型 | 扩展名 |
| ---- | ---------- |
| [图片](/oss/langchain/messages#multimodal) | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.heic`, `.heif` |
| [视频](/oss/langchain/messages#multimodal) | `.mp4`, `.mpeg`, `.mov`, `.avi`, `.flv`, `.mpg`, `.webm`, `.wmv`, `.3gpp` |
| [音频](/oss/langchain/messages#multimodal) | `.wav`, `.mp3`, `.aiff`, `.aac`, `.ogg`, `.flac` |
| [文件](/oss/langchain/messages#multimodal) | `.pdf`, `.ppt`, `.pptx` |

</Accordion>

<Accordion title="运行不带默认文件系统工具" icon="ban">
    要向模型隐藏上述文件系统工具，请使用 `excluded_tools` 注册一个 [harness profile](/oss/deepagents/profiles#harness-profiles)：

    ```python
    from deepagents import HarnessProfile, register_harness_profile

    register_harness_profile(
        "anthropic:claude-sonnet-4-6",
        HarnessProfile(
            excluded_tools=frozenset(
                {"ls", "read_file", "write_file", "edit_file", "glob", "grep"}
            ),
        ),
    )
    ```

通过 `excluded_middleware` 移除 @[`FilesystemMiddleware`] 本身会被故意拒绝——它是[默认中间件堆栈](/oss/deepagents/customization#default-stack-main-agent)中的必需支架。使用 `excluded_tools` 仅隐藏模型可见的工具表面，同时保留中间件。要移除 `task` 工具，请参阅[无子代理运行](/oss/deepagents/subagents#running-without-subagents)。
</Accordion>

虚拟文件系统被多项其他工具功能使用，例如技能、记忆、代码执行和上下文管理。
在为 Deep Agents 构建自定义工具和中间件时，也可以使用文件系统。

更多信息，请参阅[后端](/oss/deepagents/backends)。

### 文件系统权限

工具支持声明式权限规则，用于控制代理可以读取或写入的文件和目录。权限适用于上述内置文件系统工具，并按声明顺序评估，采用首次匹配优先语义。

在创建代理时通过 `permissions=` 传递规则列表来定义权限。每条规则包括：
- `operations`：`"read"` 和/或 `"write"`
- `paths`：文件和目录的 Glob 模式
- `mode`：`"allow"` 或 `"deny"`

规则从上到下评估，首次匹配的规则生效。若无规则匹配，则允许操作。

此模型允许将代理限制在特定目录（例如 `/workspace/`），保护敏感文件如 `.env` 或凭证，并赋予子代理比父代理更窄的访问权限。

权限不适用于[沙箱后端](/oss/deepagents/sandboxes)，后者通过 `execute` 工具支持任意命令执行。若需自定义验证逻辑，请使用[后端策略钩子](/oss/deepagents/backends#add-policy-hooks)。

完整的规则结构、示例和子代理继承信息，请参阅[权限](/oss/deepagents/permissions)。

### 代码执行

Deep Agents 支持两种代码执行方式：

- [沙箱后端](/oss/deepagents/sandboxes)提供 `execute` 工具，用于在隔离环境中执行 shell 命令。
- [解释器](/oss/deepagents/interpreters)添加 `eval` 工具，在作用域化的 QuickJS 运行时中运行 JavaScript。

当代理需要安装依赖、运行测试、调用 CLI 或与操作系统文件系统交互时，请使用沙箱后端。沙箱后端实现 `SandboxBackendProtocolV2`；检测到后，工具会将 `execute` 工具添加到代理的可用工具中。

当代理需要轻量级的可编程层来处理循环、批处理、确定性数据转换或编程式工具调用时，请使用解释器。解释器不提供 shell 访问、包安装或文件系统和网络访问。

沙箱设置、提供商和文件传输 API 的信息，请参阅[沙箱](/oss/deepagents/sandboxes)。关于 QuickJS 运行时和编程式工具调用，请参阅[解释器](/oss/deepagents/interpreters)。

### 流式传输

[事件流式传输](/oss/deepagents/event-streaming)将代理运行公开为消息、工具调用、值和输出的类型化投影。Deep Agents 添加了 `stream.subagents`，使每个委托任务获得独立的消息、工具调用和嵌套子代理流。

## 上下文管理

上下文管理组件控制代理了解什么、在令牌限制内能运行多长时间，以及跨会话保留什么内容。它有四层：

- **[技能](#skills)**——从技能文件按需渐进加载的领域知识
- **[记忆](#memory)**——启动时从 `AGENTS.md` 文件加载的持久化指令和偏好设置
- **[摘要和上下文卸载](#summarization-and-context-offloading)**——自动压缩对话历史和大型工具结果
- **[提示缓存](#prompt-caching)**——静态提示部分可缓存，以加速推理并在支持的模型上降低成本

### 技能

技能为您的深度代理打包了专业工作流程、领域知识和自定义指令。

每个技能都遵循 [Agent Skills 标准](https://agentskills.io/)，存在于包含 `SKILL.md` 文件的目录中。技能还可以包括脚本、模板、参考文档和其他支持资源。

Deep Agents 采用渐进式披露方式加载技能：代理在启动时读取 `SKILL.md` frontmatter，然后在任务需要时才读取完整的技能内容。这保持了启动上下文的紧凑性，同时仍能按需提供丰富的功能。

更多信息，请参阅[技能](/oss/deepagents/skills)。

### 记忆

记忆为您的深度代理提供跨对话的持久上下文，例如编码风格、偏好设置、约定和项目指南。

记忆使用 [`AGENTS.md` 文件](https://agents.md/)，您可以在创建代理时通过 `memory` 参数传入。与技能不同，记忆文件始终加载，内容存储在配置的 backend（`StateBackend`、`StoreBackend` 或 `FilesystemBackend`）中。

代理还可以根据交互和反馈更新记忆，因此偏好设置和模式可以延续，无需在每个线程中重复说明。

配置详情和示例，请参阅[记忆](/oss/deepagents/customization#memory)。

### 摘要和上下文卸载

工具管理上下文，使深度代理能够在令牌限制内处理长时间运行的工作，同时保持最相关的信息在范围内。

此上下文流有四个部分：
- **输入上下文**：系统提示、记忆、技能和工具提示定义了代理的起始状态。
- **压缩**：内置卸载和摘要压缩对话历史和大型中间结果。
- **隔离**：子代理隔离重型子任务，只返回最终结果（参见[委托](#delegation)）。
- **长期记忆**：虚拟文件系统中的持久存储跨线程携带信息。

这些机制共同支持超出单个上下文窗口的多步骤任务，同时减少手动上下文修剪和令牌使用。

配置详情，请参阅[上下文工程](/oss/deepagents/context-engineering)。

### 提示缓存

对于 Anthropic 模型，`create_deep_agent` 自动将提示缓存应用到系统提示的静态部分——每次对话都重复的基础代理指令、记忆和技能内容。这避免了跨调用重复处理相同的令牌，减少了长时间运行代理的延迟和成本。

使用 Anthropic 模型时，提示缓存默认启用。无需配置。

对于其他提供商，请参阅[中间件集成](/oss/integrations/middleware#official-integrations)以获取可用的提供商特定缓存中间件。

## 委托

委托组件使代理能够将大问题分解为更小的、可并行的工作单元。它有两层：

- **[任务规划](#task-planning)**：内置的 `write_todos` 工具，用于结构化任务跟踪
- **[子代理](#subagents)**：处理隔离子任务的临时子代理

### 任务规划

工具提供 `write_todos` 工具，允许代理在执行过程中维护结构化任务列表。

任务支持状态跟踪（`'pending'`、`'in_progress'`、`'completed'`）并在代理状态中持久化。这为代理提供了轻量级的规划层，用于组织长时间运行和多步骤的工作。

### 子代理

工具包含内置的 `task` 工具，允许主代理创建用于隔离、长时间运行、多步骤或并行任务的临时子代理。

子代理执行提供：
- **全新上下文**：每次调用创建一个具有自己上下文的新代理实例。
- **自主执行**：子代理独立运行直到完成。
- **单一交接**：它向主代理返回一份最终报告。
- **可配置策略**：使用[默认的 `general-purpose` 子代理](/oss/deepagents/subagents#default-subagent)（默认启用）或定义[自定义子代理](/oss/deepagents/subagents#custom-subagents)。
- **无状态消息传递**：子代理是无状态的，无法发送多条消息回复。
- **上下文和令牌效率**：重型子任务工作保持隔离，并被压缩成紧凑的结果。

<Accordion title="无子代理运行（无 `task` 工具）" icon="ban">
    要运行不带 `task` 工具的代理，请参阅[无子代理运行](/oss/deepagents/subagents#running-without-subagents)。不要尝试通过 `excluded_middleware` 移除 @[`SubAgentMiddleware`]——这是故意被拒绝的。请改为通过[工具配置文件](/oss/deepagents/profiles#harness-profiles)禁用自动添加的子代理，并通过 `subagents=` 不传入同步子代理。异步子代理不受影响。完整的排序请参阅[默认中间件堆栈](/oss/deepagents/customization#default-stack-main-agent)。
</Accordion>

更多信息，请参阅[子代理](/oss/deepagents/subagents)。

## 引导

引导组件使人类能够在运行时控制代理行为，并为代理工作设置文件系统权限。

### 人在回路

Deep Agents 与 LangGraph 中断集成，使您可以在敏感工具调用时暂停以等待批准。在 `create_deep_agent` 中通过 `interrupt_on` 参数启用此行为。

`interrupt_on` 接受工具名称到中断配置的映射。例如，`interrupt_on={"edit_file": True}` 在每次编辑前暂停，让您可以批准调用、添加指导或在执行前修改工具输入。

这为您提供了运行时安全和控制层，用于破坏性操作、昂贵的 API 调用和交互式调试。

更多信息，请参阅[人在回路](/oss/deepagents/human-in-the-loop)。
