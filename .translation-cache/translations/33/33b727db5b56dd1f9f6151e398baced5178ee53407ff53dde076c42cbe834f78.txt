Deep Agents 是开始构建由 LLM 驱动的代理和应用程序的最简单方式——内置任务规划、上下文管理文件系统、子代理生成和长期记忆等功能。
您可以将 deep agents 用于任何任务，包括复杂的多步骤任务。

Deep Agents 具有以下内置功能：

- **在环境中执行操作**：通过工具执行操作、读写文件、执行代码
- **连接您的数据**：在合适的时机加载记忆、技能和领域知识
- **管理不断增长的上下文**：总结历史并在长时间运行中卸载大型结果
- **并行化任务**：委托给在隔离上下文窗口中运行的一般或专业子代理
- **保持参与**：在关键决策点暂停以等待人工批准
- **随时间改进**：根据实际使用情况更新记忆、技能和提示

请参阅 [核心功能](#core-capabilities) 以获取每个组件的完整分解。

## 执行环境

执行环境是智能体运行的地方。它有四个层次：

- **[工具](#tools-and-mcp)**：智能体可以调用的自定义函数、API 和数据库
- **[虚拟文件系统](#virtual-filesystem-access)**：由可插拔后端支持的文件工具
- **[文件系统权限](#filesystem-permissions)**：声明式访问控制，规定智能体可以读取或写入哪些路径
- **[代码执行](#code-execution)**：沙箱化 shell 执行和进程内 JavaScript 解释器

**[流式处理](#streaming)** 允许您使用类型化事件流来跟踪消息、工具、值和委托任务的所有活动。

### 工具和 MCP

通过 [MCP 服务器](/oss/deepagents/tools#mcp-tools) 参数传递自定义函数、LangChain 工具或任何 `tools=` 的工具。Deep Agents 完全支持 [Model Context Protocol (MCP)](/oss/langchain/mcp)，让您可以通过标准接口连接数据库、API、文件系统等。

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[search, fetch_page, run_query],
)
```

有关定义自定义工具、使用 MCP 服务器和内置工具完整列表的更多信息，请参阅 [工具](/oss/deepagents/tools).

### 虚拟文件系统访问

该工具提供了一个可配置的虚拟文件系统，可由不同的 [可插拔后端](/oss/deepagents/backends)支持：内存状态、本地磁盘、LangGraph 存储、复合路由或带有 [权限规则](/oss/deepagents/permissions) 的自定义后端。

这些后端支持以下文件系统操作：

| 工具 | 描述 |
| ---- | ----------- |
| `ls` | 列出目录中的文件及元数据（大小、修改时间） |
| `read_file` | Read file contents with line numbers, supports offset/limit for large files. Also supports returning multimodal content blocks for non-text files (images, video, audio, and documents). See supported extensions below. |
| `write_file` | 创建新文件 |
| `edit_file` | 在文件中执行精确字符串替换（支持全局替换模式） |
| `glob` | 查找匹配模式的文件（例如， `**/*.py`) |
| `grep` | 搜索文件内容，支持多种输出模式（仅文件、带上下文的文件内容或计数） |
| `execute` | 在环境中运行 shell 命令（适用于 [沙箱后端](/oss/deepagents/sandboxes) ） |

<Accordion title="Supported multimodal file extensions">

| 类型 | 扩展名 |
| ---- | ---------- |
| [图片](/oss/langchain/messages#multimodal) | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.heic`, `.heif` |
| [视频](/oss/langchain/messages#multimodal) | `.mp4`, `.mpeg`, `.mov`, `.avi`, `.flv`, `.mpg`, `.webm`, `.wmv`, `.3gpp` |
| [音频](/oss/langchain/messages#multimodal) | `.wav`, `.mp3`, `.aiff`, `.aac`, `.ogg`, `.flac` |
| [文件](/oss/langchain/messages#multimodal) | `.pdf`, `.ppt`, `.pptx` |

</Accordion>

<Accordion title="Running without the default filesystem tools" icon="ban">
    要向模型隐藏上述文件系统工具，请注册一个 [harness 配置](/oss/deepagents/profiles#harness-profiles) 并设置 `excluded_tools`:

    ```python
    from deepagents import HarnessProfile, register_harness_profile

    register_harness_profile(
        "anthropic:claude-sonnet-4-6",
        HarnessProfile(
            excluded_tools=frozenset(
                {"ls", "read_file", "write_file", "edit_file", "glob", "grep"}
            ),
        ),
    )
    ```

    要移除`FilesystemMiddleware`] 本身请通过 `excluded_middleware` 操作——它是 [默认中间件堆栈中](/oss/deepagents/customization#default-stack-main-agent)必需的框架结构。使用 `excluded_tools` 可以仅隐藏模型可见的工具界面而保留中间件。要移除 `task` 工具，请参见 [无子代理运行](/oss/deepagents/subagents#running-without-subagents).
</Accordion>

虚拟文件系统被多项其他 harness 功能使用，如技能、记忆、代码执行和上下文管理。
在构建自定义工具和 Deep Agents 中间件时，也可以使用文件系统。

更多信息请参见 [后端](/oss/deepagents/backends).

### 文件系统权限

harness 支持声明式权限规则，用于控制代理可以读取或写入的文件和目录。权限适用于上述内置文件系统工具，并按声明顺序进行评估，匹配即生效。

通过在创建代理时传递规则列表来定义权限 `permissions=` 。每条规则包括：
- `operations`: `"read"` and/or `"write"`
- `paths`：文件或目录的 Glob 模式
- `mode`: `"allow"` or `"deny"`

规则从上到下评估，首条匹配规则生效。若无规则匹配，则允许操作。

此模型允许将代理限制在特定目录（例如 `/workspace/`），保护敏感文件（如 `.env` 或凭据），并为子代理提供比父代理更窄的访问权限。

权限不适用于 [沙箱后端](/oss/deepagents/sandboxes)，后者支持通过 `execute` 工具执行任意命令。如需自定义验证逻辑，请使用 [后端策略钩子](/oss/deepagents/backends#add-policy-hooks).

完整的规则结构、示例和子代理继承机制请参见 [权限](/oss/deepagents/permissions).

### 代码执行

Deep Agents 支持两种代码执行方式：

- [沙箱后端](/oss/deepagents/sandboxes) 提供 `execute` 工具，用于在隔离环境中执行 shell 命令。
- [解释器](/oss/deepagents/interpreters) 添加一个 `eval` 在作用域受限的 QuickJS 运行时中运行 JavaScript 的工具。

当智能体需要安装依赖项、运行测试、调用 CLI 或使用操作系统文件系统时，请使用沙箱后端。沙箱后端实现 `SandboxBackendProtocolV2`；检测到时，测试工具会添加 `execute` 工具到智能体的可用工具列表。

当智能体需要轻量级可编程层来处理循环、批处理、确定性数据转换或编程式工具调用时，请使用解释器。解释器不提供 shell 访问、程序包安装或文件系统和网络访问。

有关沙箱设置、提供商和文件传输 API，请参阅 [沙箱](/oss/deepagents/sandboxes)。有关 QuickJS 运行时和编程式工具调用，请参阅 [解释器](/oss/deepagents/interpreters).

### 流式传输

[事件流](/oss/deepagents/event-streaming) 将智能体运行作为消息、工具调用、值和输出的类型化投影公开。深度智能体添加 `stream.subagents` 使每个委托的任务获得自己的句柄，具有独立的消息、工具调用和嵌套子智能体流。

## 上下文管理

上下文管理组件控制智能体知道什么、它能在 token 限制内操作多长时间，以及它在会话之间保留什么。它有四层：

- **[技能](#skills)**—从技能文件按需加载的领域知识
- **[记忆](#memory)**—在启动时从 `AGENTS.md` 文件加载的持久性指令和偏好设置
- **[摘要和上下文卸载](#summarization-and-context-offloading)**—对话历史记录和大型工具结果的自动压缩
- **[提示缓存](#prompt-caching)**—静态提示部分可缓存，以加快推理速度并降低支持模型的成本

### 技能

技能为您的深度智能体打包专业工作流程、领域知识和自定义指令。

每个技能遵循 [智能体技能标准](https://agentskills.io/) 并存在于带有 `SKILL.md` 文件的目录中。技能还可以包括脚本、模板、参考文档和其他支持资源。

深度智能体使用渐进式披露加载技能：智能体在启动时读取 `SKILL.md` frontmatter，然后在任务需要时读取完整技能内容。这使启动上下文保持紧凑，同时仍按需提供丰富的能力。

有关更多信息，请参阅 [技能](/oss/deepagents/skills).

### 记忆

记忆为您的深度智能体提供跨对话的持久上下文，例如编码风格、偏好设置、约定和项目指南。

记忆使用 [`AGENTS.md` 文件](https://agents.md/) 您在创建智能体时通过 `memory` 参数传递这些文件。与技能不同，记忆文件始终加载，内容存储在配置的后端中（`StateBackend`, `StoreBackend`, or `FilesystemBackend`).

智能体还可以根据交互和反馈更新记忆，因此偏好设置和模式可以在不需要在每个线程中重新陈述的情况下延续。

有关配置详情和示例，请参阅 [内存](/oss/deepagents/customization#memory).

### 摘要和上下文卸载

测试框架管理上下文，使深度代理能够在令牌限制内处理长时间运行的工作，同时保持最相关的信息在范围内。

此上下文流程包含四个部分：
- **输入上下文**：系统提示词、内存、技能和工具提示词定义了代理的起始状态。
- **压缩**：内置卸载和摘要功能会压缩对话历史和大型中间结果。
- **隔离**：子代理隔离重型子任务，只返回最终结果（见 [委托](#delegation)).
- **长期记忆**：虚拟文件系统中的持久化存储跨线程传递信息。

这些机制共同支持超出单个上下文窗口的多步骤任务，同时减少手动上下文修剪和令牌使用量。

有关配置详情，请参阅 [上下文工程](/oss/deepagents/context-engineering).

### 提示词缓存

对于 Anthropic 模型， `create_deep_agent` 会自动对系统提示词的静态部分应用提示词缓存——即每次交互都会重复的基础代理指令、内存和技能内容。这避免了跨调用重复处理相同的令牌，减少了长时间运行代理的延迟和成本。

使用 Anthropic 模型时，提示词缓存默认启用。无需任何配置。

对于其他提供商，请参阅 [中间件集成](/oss/integrations/middleware#official-integrations) 了解可用的提供商特定缓存中间件。

## 委托

委托组件使代理能够将大型问题分解为更小的、可并行处理的工作单元。它包含两个层次：

- **[任务规划](#task-planning)**：内置 `write_todos` 用于结构化任务跟踪的工具
- **[子代理](#subagents)**：处理隔离子任务的临时子代理

### 任务规划

测试框架提供了一个 `write_todos` 工具，允许代理在执行期间维护结构化任务列表。

任务支持状态跟踪（`'pending'`, `'in_progress'`, `'completed'`），并持久化在代理状态中。这为代理提供了一个轻量级的规划层，用于组织长时间运行和多步骤的工作。

### 子代理

测试框架包含一个内置 `task` 工具，允许主代理创建临时子代理来处理隔离的、长时间运行的、多步骤或并行任务。

子代理执行提供：
- **全新上下文**：每次调用都会创建一个具有自己上下文的新代理实例。
- **自主执行**：子代理独立运行直到完成。
- **单一交接**：它向主代理返回一个最终报告。
- **可配置策略**：使用 [默认 `general-purpose` 子代理](/oss/deepagents/subagents#default-subagent) （默认启用）或定义 [自定义子代理](/oss/deepagents/subagents#custom-subagents).
- **无状态消息传递**：子代理是无状态的，不能发送多条消息回来。
- **上下文和令牌效率**：繁重的子任务工作保持隔离，并被压缩成紧凑的结果。

<Accordion title="Running without subagents (no `task` tool)" icon="ban">
    要在没有 `task` 工具的情况下运行代理，请参阅 [无子代理运行](/oss/deepagents/subagents#running-without-subagents)。不要尝试通过`SubAgentMiddleware`删除 @[ `excluded_middleware`，这会被故意拒绝。请通过 [禁用自动添加的子代理，并通过](/oss/deepagents/profiles#harness-profiles) 不传递同步子代理。异步子代理不受影响。请参阅 `subagents=`工具配置文件 [获取完整排序。](/oss/deepagents/customization#default-stack-main-agent) 默认中间件堆栈
</Accordion>

有关更多信息，请参阅 [子代理](/oss/deepagents/subagents).

## 操控

操控组件让人类能够在运行时控制代理行为，并设置代理工作的文件系统权限。

### Human-in-the-loop

Deep Agents 与 LangGraph 中断集成，以便您可以在敏感工具调用时暂停以获得批准。通过 `interrupt_on` 参数启用此行为 `create_deep_agent`.

`interrupt_on` 接受工具名称到中断配置的对象映射。例如， `interrupt_on={"edit_file": True}` 在每次编辑前暂停，让您批准调用、添加指导或在执行前修改工具输入。

这为您提供了破坏性操作、昂贵 API 调用和交互式调试的运行时安全和控制层。

有关更多信息，请参阅 [Human-in-the-loop](/oss/deepagents/human-in-the-loop).
