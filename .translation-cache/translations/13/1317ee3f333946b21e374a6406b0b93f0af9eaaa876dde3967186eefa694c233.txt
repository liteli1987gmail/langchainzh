## 概览

AgentSystems Notary 为 AI 智能体交互创建防篡改的审计追踪。

## 为什么

当客户、审计人员、监管机构、保险公司等对 AI 行为提出质疑时，您需要证明实际发生了什么。传统日志无法满足这一需求：由于您控制着日志，第三方必须信任您没有篡改它们。

防篡改日志消除了这种信任需求。

## 工作原理

原始 LLM 交互保留在您的存储中。正常运行期间，第三方无法查看。但每次交互的密码学哈希会同时写入独立的防篡改存储（Arweave 或 AgentSystems 托管服务）中。

如果发生审计或争议，您可以提供原始日志。审计人员会重新计算哈希值并与存储的哈希值进行比对。一致表示日志未被篡改，不一致则表明存在篡改或损坏。

您控制数据，但无法在不被发现的情况下修改它。

**记录内容：**
- 存储在您的存储中：完整的原始 LLM 负载（提示、响应、元数据、时间戳）
- 存储在哈希存储中：SHA-256 哈希 + 元数据（例如命名空间、会话 ID、时间戳）

## 哈希存储选项

哈希（而非原始数据）可以写入以下任一存储选项：

| 存储 | 适用场景 | 功能 |
|---------|----------|----------|
| [去中心化 (Arweave)](https://docs.agentsystems.ai/notary/configuration/hash-storage?utm_source=langchain-docs&utm_medium=docs) | 无供应商锁定 | 公共只追加账本、开源验证、无需账户 |
| [`托管`](https://agentsystems.ai/notary?utm_source=langchain-docs&utm_medium=docs) | 托管合规性 | 一次写入存储、验证界面、用于审计的签名证明 |

<Info>
[托管方案](https://agentsystems.ai/notary?utm_source=langchain-docs&utm_medium=docs) 提供 WORM 合规哈希存储、托管签名和签名证明。
</Info>

## 先决条件

```shell
pip install agentsystems-notary langchain langchain-anthropic python-dotenv
```

## 示例（去中心化）

<Steps>
  <Step title="生成签名密钥">
    ```shell
    openssl genrsa -out arweave-key.pem 4096
    ```
<Warning>
    保留此密钥。它是验证时证明链上哈希所有权所必需的。
    </Warning>

    对于生产环境，请使用[云密钥管理服务](https://docs.agentsystems.ai/notary/configuration/signing?utm_source=langchain-docs&utm_medium=docs)。
  </Step>
  <Step title="创建 .env 文件">
    在项目根目录中创建 `.env` 文件：

</fragment>
    ```
    # AWS S3 for raw payload storage
    ORG_AWS_S3_BUCKET_NAME=your-bucket
    ORG_AWS_S3_ACCESS_KEY_ID=AKIA...
    ORG_AWS_S3_SECRET_ACCESS_KEY=...
    ORG_AWS_S3_REGION=us-east-1

    # Path to signing key
    ARWEAVE_PRIVATE_KEY_PATH=./arweave-key.pem

    # Anthropic
    ANTHROPIC_API_KEY=sk-ant-...
    ```
</Step>
  <Step title="运行示例">

</fragment>
    ```python
    import os

    from agentsystems_notary import (
        LangChainNotary,
        ArweaveHashStorage,
        AwsS3StorageConfig,
        LocalKeySignerConfig,
        RawPayloadStorage,
    )
    from langchain_anthropic import ChatAnthropic
    from dotenv import load_dotenv

    load_dotenv()

    # Your S3 bucket for raw LLM payloads
    s3_config = AwsS3StorageConfig(
        bucket_name=os.environ["ORG_AWS_S3_BUCKET_NAME"],
        aws_access_key_id=os.environ["ORG_AWS_S3_ACCESS_KEY_ID"],
        aws_secret_access_key=os.environ["ORG_AWS_S3_SECRET_ACCESS_KEY"],
        aws_region=os.environ["ORG_AWS_S3_REGION"],
    )
    raw_payload_storage = RawPayloadStorage(storage=s3_config)

    # Local RSA key for signing
    signer = LocalKeySignerConfig(
        private_key_path=os.environ["ARWEAVE_PRIVATE_KEY_PATH"],
    )

    # Arweave for decentralized hash storage
    # Namespace is public — written to the ledger and used to segment stored data
    # Namespace should be one anonymous ID per customer, agent, or environment
    # Retain a record of your namespace mappings
    arweave_storage = ArweaveHashStorage(
        namespace="tenant_a1b2c3d4",  # See namespace comments above
        signer=signer,
    )

    # Create notary callback
    notary = LangChainNotary(
        raw_payload_storage=raw_payload_storage,
        hash_storage=[arweave_storage],
        debug=True,
    )

    # Attach to model
    model = ChatAnthropic(
        model="claude-sonnet-4-6",
        api_key=os.environ["ANTHROPIC_API_KEY"],
        callbacks=[notary],
    )

    response = model.invoke("What is the capital of France?")
    print(response.content)
    ```
</Step>
</Steps>

## 验证

**去中心化（Arweave）**：从存储桶下载原始数据，压缩后使用开源 CLI 进行验证：

```shell
aws s3 sync s3://your-bucket/arweave/tenant_a1b2c3d4/ ./logs
zip -r logs.zip logs
npm install -g agentsystems-verify
agentsystems-verify --logs logs.zip
```

CLI 会对每个负载进行重新哈希，并与存储在 Arweave 上的哈希值进行比较。详见[完整验证指南](https://docs.agentsystems.ai/notary/verification/arweave?utm_source=langchain-docs&utm_medium=docs)。

或者，[验证界面](https://verify.agentsystems.ai?utm_source=langchain-docs&utm_medium=docs)同时支持去中心化和托管验证。
