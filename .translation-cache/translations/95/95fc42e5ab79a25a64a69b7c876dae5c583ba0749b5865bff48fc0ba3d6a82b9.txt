---
title: 沙盒 CLI
description: 从命令行创建、检查、连接和隧道访问 LangSmith 沙盒。
---

[LangSmith CLI](/langsmith/langsmith-cli) 包含用于创建快照、启动沙盒、运行命令、打开交互式 shell 以及将 TCP 连接隧道传输到沙盒的沙盒命令。

沙盒 CLI 命令需要 LangSmith CLI `v0.2.26` 或更高版本。

## 安装和身份验证

安装或升级 LangSmith CLI：

```bash
curl -fsSL https://cli.langsmith.com/install.sh | sh
langsmith self-update
```

使用您的 LangSmith API 密钥验证 CLI 身份：

```bash
export LANGSMITH_API_KEY="<LANGSMITH_API_KEY>"
```

CLI 输出默认是 JSON。添加 `--format pretty` 以便于阅读的表格形式列出命令：

```bash
langsmith --format pretty sandbox list
```

## 端到端工作流

创建一个沙箱，然后在其中运行命令：

```bash
langsmith sandbox create my-vm \
  --vcpus 2 \
  --memory 1gb \
  --wait

langsmith sandbox exec my-vm -- python --version
```

当您完成沙箱的使用后，请将其删除：

```bash
langsmith sandbox delete my-vm
```

## 管理快照

从 Docker 镜像构建快照：

```bash
langsmith sandbox snapshot build my-snapshot \
  --docker-image ubuntu:24.04 \
  --capacity 8gb \
  --wait
```

对于私有仓库，从环境变量传递仓库凭证：

```bash
langsmith sandbox snapshot build internal-python \
  --docker-image registry.example.com/internal/python:3.12 \
  --registry-url https://registry.example.com \
  --registry-username "$REGISTRY_USERNAME" \
  --registry-password "$REGISTRY_PASSWORD" \
  --wait
```

从运行中的沙箱捕获文件系统：

```bash
langsmith sandbox snapshot capture ml-ready \
  --box my-vm \
  --wait
```

列出、检查、等待和删除快照：

```bash
langsmith sandbox snapshot list
langsmith sandbox snapshot get <SNAPSHOT_ID>
langsmith sandbox snapshot wait <SNAPSHOT_ID>
langsmith sandbox snapshot delete <SNAPSHOT_ID>
```

## 管理沙箱

使用默认运行时创建沙箱。仅在需要从可重用的自定义快照启动时添加 `--snapshot-id`：

```bash
langsmith sandbox create my-vm \
  --vcpus 4 \
  --memory 1gb \
  --rootfs-capacity 8gb \
  --wait
```

列出并检查沙箱：

```bash
langsmith sandbox list
langsmith sandbox get my-vm
langsmith sandbox wait my-vm
```

停止并启动沙箱，同时保留其文件系统：

```bash
langsmith sandbox stop my-vm
langsmith sandbox start my-vm --wait
```

更新资源或代理配置：

```bash
langsmith sandbox update my-vm --vcpus 8 --memory 2gb
langsmith sandbox update my-vm --proxy-config @proxy.json
```

资源更改将在沙箱下次启动时生效。代理配置更改会立即生效。

### 代理配置

使用 `create` 或 `update` 命令上的 `--proxy-config @proxy.json` 来配置沙箱身份验证代理。优先使用工作区密钥进行凭证注入，而不是将原始密钥放在本地文件中。

```json
{
  "rules": [
    {
      "name": "openai",
      "match_hosts": ["api.openai.com"],
      "match_paths": [],
      "headers": [
        {
          "name": "Authorization",
          "type": "workspace_secret",
          "value": "Bearer {OPENAI_API_KEY}"
        }
      ],
      "enabled": true
    }
  ],
  "access_control": {
    "allow_list": ["api.openai.com"],
    "deny_list": []
  }
}
```

有关代理规则的更多信息，请参阅 [Sandbox auth proxy](/langsmith/sandbox-auth-proxy)。

## 运行命令

使用 `sandbox exec` 运行一次性命令：

```bash
langsmith sandbox exec my-vm -- uname -a
langsmith sandbox exec my-vm -- ls -la /
langsmith sandbox exec my-vm -- cat /etc/os-release
```

`--` 之后的所有内容都会被作为命令发送到沙箱。CLI 将 stdout 打印到 stdout，stderr 打印到 stderr，并以沙箱命令的退出码退出。

## 打开交互式控制台

使用 `sandbox console` 打开一个基于 PTY 的交互式 shell：

```bash
langsmith sandbox console my-vm
langsmith sandbox console my-vm --shell /bin/sh
```

您可以在控制台会话中转发本地 SSH 代理：

```bash
langsmith sandbox console my-vm --forward-ssh-agent
```

`--forward-ssh-agent` 需要在本地设置 `SSH_AUTH_SOCK`。Windows 不支持交互式控制台会话；请改用 SSH 访问。

## 隧道 TCP 端口

当您需要一个本地 TCP 端口来转发到沙箱内运行的服务时，请使用 `sandbox tunnel`。这对于数据库、语言服务器、自定义协议或需要 `localhost` 的本地工具非常有用。

在沙箱中启动服务，然后建立隧道连接：

```bash
langsmith sandbox exec my-vm -- sh -c 'cd /tmp && nohup python -m http.server 8000 > /tmp/http.log 2>&1 &'
langsmith sandbox tunnel my-vm --remote-port 8000 --local-port 18000
```

然后本地连接：

```bash
curl http://127.0.0.1:18000
```

如果省略 `--local-port`，CLI 会使用与 `--remote-port` 相同的值：

```bash
langsmith sandbox tunnel my-vm --remote-port 5432
```

隧道进程在前台运行。使用 `Ctrl+C` 停止它。

你也可以通过沙箱 URL 而非名称来建立隧道：

```bash
langsmith sandbox tunnel \
  --url <SANDBOX_URL> \
  --remote-port 5432
```

<Tip>
对于需要在浏览器中打开或与团队成员共享的 HTTP 应用程序，请使用 [Sandbox 服务 URL](/langsmith/sandbox-service-urls)。对于原始 TCP 协议或本地开发工具，请使用隧道。
</Tip>

## 设置 SSH 访问

使用 `sandbox ssh-setup` 通过沙盒隧道配置标准 SSH 工具（如 `ssh`、`scp`、`rsync` 和 `sftp`）。

```bash
langsmith sandbox ssh-setup my-vm
langsmith sandbox ssh-setup my-vm --identity ~/.ssh/id_ed25519.pub
```

该命令将您的 SSH 公钥上传到沙箱，在沙箱主机密钥可用时获取该密钥，将 `Host sandbox-<name>` 块写入 `~/.ssh/config`，并将沙箱主机密钥写入 `~/.ssh/known_hosts_sandboxes`。

设置完成后，使用以下命令连接：

```bash
ssh sandbox-my-vm
```

沙盒镜像必须在端口 `22` 上运行 `sshd`。如果 `sshd` 未运行，`ssh-setup` 会发出警告，SSH 连接将无法工作，直到您在沙盒内启动它。

<Warning>
`ssh-setup` 会修改本地 SSH 配置，并写入一个调用 `langsmith sandbox tunnel` 的 `ProxyCommand`。根据 CLI 的认证方式，生成的代码块可能包含凭据或对凭据的引用。仅在可信的机器上运行它，不要提交或分享生成的 SSH 配置代码块。
</Warning>

## 命令参考

| 命令 | 描述 |
| --- | --- |
| `langsmith sandbox snapshot list` | 列出快照。 |
| `langsmith sandbox snapshot build <name> --docker-image <image>` | 从 Docker 镜像构建快照。 |
| `langsmith sandbox snapshot capture <name> --box <sandbox>` | 从运行中的沙盒捕获快照。 |
| `langsmith sandbox snapshot get <snapshot-id>` | 查看快照详情。 |
| `langsmith sandbox snapshot wait <snapshot-id>` | 等待快照就绪。 |
| `langsmith sandbox snapshot delete <snapshot-id>` | 删除快照。 |
| `langsmith sandbox create <name>` | 使用默认运行时创建沙盒。 |
| `langsmith sandbox list` | 列出沙盒。 |
| `langsmith sandbox get <name>` | 查看沙盒详情。 |
| `langsmith sandbox update <name>` | 更新沙盒资源或代理配置。 |
| `langsmith sandbox wait <name>` | 等待沙盒就绪。 |
| `langsmith sandbox start <name>` | 启动已停止的沙盒。 |
| `langsmith sandbox stop <name>` | 停止运行中的沙盒，同时保留文件系统状态。 |
| `langsmith sandbox delete <name>` | 删除沙盒。 |
| `langsmith sandbox exec <name> -- <command>` | 在沙盒内运行一次性命令。 |
| `langsmith sandbox console <name>` | 在沙盒内打开交互式 shell。 |
| `langsmith sandbox tunnel <name> --remote-port <port>` | 将本地 TCP 端口转发到沙盒端口。 |
| `langsmith sandbox ssh-setup <name>` | 通过 `sandbox tunnel --stdio` 配置本地 SSH 访问。 |
