title: 版本控制
---

我们的开源软件版本号遵循以下格式：`MAJOR.MINOR.PATCH`，即 [语义化版本控制](https://semver.org/) 所定义的规范。

- **Major**：需要进行代码更改的破坏性 API 更新。
- **Minor**：保持向后兼容性的新功能和改进。
- **Patch**：错误修复和小幅改进。

例如：

- `1.0.0`：首个稳定版本，包含可用于生产环境的 API
- `1.1.0`：以向后兼容方式添加的新功能
- `1.0.1`：向后兼容的错误修复

## API 稳定性

我们通过以下方式说明 API 的稳定性：

### 稳定 API

所有没有特殊前缀的 API 都被视为稳定的，可用于生产环境。我们维护稳定功能的后向兼容性，仅在主要版本中引入破坏性更改。

### Beta API

标记为 `beta` 的 API 功能已完整，但可能会根据用户反馈进行小幅更改。它们可以安全地用于生产环境，但可能在未来版本中需要小幅调整。

### Alpha API

标记为 `alpha` 的 API 是实验性的，可能会有重大更改。在生产环境中使用这些 API 时请谨慎。

### 弃用 API

标记为 `deprecated` 的 API 将在未来的主要版本中移除。我们会尽可能指定计划移除的版本。要处理弃用：

1. 切换到推荐的替代 API
2. 遵循迁移指南（随主要版本一起发布）
3. 尽可能使用自动化迁移工具

### 内部 API

某些 API 明确以几种方式标记为"内部"：

- 部分文档提及内部内容并将其标记为内部。如果文档说明某内容是内部的，则它可能会更改。
- 以单下划线（**`_`**）为前缀的函数、方法和其他对象。这是 Python 中表示某个内容为私有的标准约定；如果任何方法以单个 **`_`** 开头，则它是内部 API。
    - **例外：** 某些方法以 `_` 为前缀，但不包含实现。这些方法*旨在*被子类覆盖以提供实现。这类方法通常是 LangChain **公共 API** 的一部分。

## 发布周期

<AccordionGroup>
    <Accordion title="主要版本">
        主要版本（例如 `1.0.0` → `2.0.0`）可能包括：

        - 破坏性 API 更改
        - 移除已弃用的功能
        - 重大架构改进

        我们提供：

        - 详细的迁移指南
        - 尽可能提供自动化迁移工具
        - 为上一主要版本提供延长支持期
    </Accordion>

    <Accordion title="次要版本">
        次要版本（例如 `1.0.0` → `1.1.0`）包括：

        - 新功能和新能力
        - 性能改进
        - 新的可选参数
        - 向后兼容的增强
    </Accordion>

    <Accordion title="补丁版本">
        补丁版本（例如 `1.0.0` → `1.0.1`）包括：

        - 错误修复
        - 安全更新
        - 文档改进
        - 不涉及 API 更改的性能优化
    </Accordion>
</AccordionGroup>

## 版本支持政策

- **最新主要版本**：完整支持并积极开发（ACTIVE 状态）
- **上一主要版本**：在下一主要版本发布后 12 个月内提供安全更新和关键错误修复（MAINTENANCE 状态）
- **更早版本**：仅社区支持

### 长期支持（LTS）版本

LangChain 和 LangGraph 1.0 均被指定为 LTS 版本：

- 1.0 版本将在 2.0 版本发布前保持 ACTIVE 状态
- 2.0 版本发布后，1.0 版本将进入 MAINTENANCE 模式，至少持续 1 年
- LTS 版本遵循语义化版本控制（semver），允许在次要版本之间安全升级
- 旧版本（LangChain 0.3 和 LangGraph 0.4）将维持 MAINTENANCE 模式至 2026 年 12 月

### 1.0 之前的包

:::python
**Deep Agents**（`deepagents`）是一个正在积极开发的 1.0 之前版本。作为快速迭代的包，API 可能在次要版本之间发生变化，尽管我们会尽可能减少破坏性更改。Deep Agents 在达到 1.0 版本后将采用与 LangChain 和 LangGraph 相同的 LTS 政策。
:::

有关发布状态和支持时间线的详细信息，请参阅 [发布政策](/oss/release-policy)。

## 升级

:::python
<CodeGroup>

```bash LangChain
# Upgrade to the latest version
pip install -U langchain-core langchain

# Upgrade to a specific version
pip install langchain-core==1.0.0
```

```bash LangGraph
# Upgrade to the latest version
pip install -U langgraph

# Upgrade to a specific version
pip install langgraph==1.0.0
```

```bash Deep Agents
# Upgrade to the latest version
pip install -U deepagents

# Upgrade to a specific version
pip install deepagents==0.1.0
```
</CodeGroup>
:::

:::js
<CodeGroup>

</fragment>

```bash LangChain
# Upgrade to the latest version
npm update langchain @langchain/core

# Install a specific version
npm install langchain@1.0.0 @langchain/core@1.0.0
```

```bash LangGraph
# Upgrade to the latest version
npm update @langchain/langgraph

# Install a specific version
npm install @langchain/langgraph@1.0.0
```
## 预发布版本

我们会偶尔发布 alpha 和 beta 版本供早期测试：

- **Alpha**（例如 `1.0.0a1`）：早期预览，预计会有重大变化
- **Beta**（例如 `1.0.0b1`）：功能完整，可能会有小幅更改
- **发布候选**（例如 `1.0.0rc1`）：稳定版本发布前的最终测试
