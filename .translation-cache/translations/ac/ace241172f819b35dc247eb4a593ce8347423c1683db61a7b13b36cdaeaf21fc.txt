title: 版本控制
---

我们的开源软件版本号遵循以下格式： `MAJOR.MINOR.PATCH`，按照以下定义： [语义化版本控制](https://semver.org/).

- **主版本号**：破坏性 API 更新，需要代码更改。
- **次版本号**：新功能和保持向后兼容的改进。
- **修订号**：错误修复和小幅改进。

例如：

- `1.0.0`：首个稳定版本，具有生产级 API
- `1.1.0`：以向后兼容方式添加的新功能
- `1.0.1`：向后兼容的错误修复

## API 稳定性

我们按以下方式说明 API 的稳定性：

### 稳定 API

所有没有特殊前缀的 API 都被视为稳定版本，可用于生产环境。我们为稳定功能保持向后兼容性，仅在主版本中引入破坏性更改。

### Beta API

标记为 `beta` 的 API 功能已完成，但可能会根据用户反馈进行小幅更改。它们可以安全地用于生产环境，但未来版本可能需要进行小幅调整。

### Alpha API

标记为 `alpha` 的 API 是实验性的，可能会有重大更改。在生产环境中使用这些 API 时请谨慎。

### 弃用 API

标记为 `deprecated` 的 API 将在未来的主版本中移除。在可能的情况下，我们会指定预期的移除版本。处理弃用的方法：

1. 切换到推荐的替代 API
2. 遵循迁移指南（与主版本一起发布）
3. 在可用时使用自动化迁移工具

### 内部 API

某些 API 以两种方式明确标记为“内部”：

- 某些文档涉及内部内容并将其标记为内部。如果文档说明某内容是内部的，它可能会更改。
- 函数、方法和其他以英文下划线 (**`_`**) 为前缀的对象。这是 Python 中表示某内容为私有的标准约定；如果任何方法以单个 **`_`**开头，它就是一个内部 API。
    - **Exception:** 某些方法以 `_` 为前缀，但不包含实现。这些方法是 *供* 子类重写以提供实现的。此类方法通常是 LangChain **公共 API** 的一部分。

## 发布周期

<AccordionGroup>
    <Accordion title="Major releases">
        主版本（如 `1.0.0` → `2.0.0`）可能包括：

        - 破坏性 API 更改
        - 移除弃用功能
        - 重大架构改进

        我们提供：

        - 详细的迁移指南
        - 尽可能使用自动化迁移工具
        - 上一主要版本的延长支持期
    </Accordion>

    <Accordion title="Minor releases">
        次要版本（例如， `1.0.0` → `1.1.0`）包括：

        - 新功能和改进
        - 性能改进
        - 新的可选参数
        - 向后兼容的增强
    </Accordion>

    <Accordion title="Patch releases">
        补丁版本（例如， `1.0.0` → `1.0.1`）包括：

        - 错误修复
        - 安全更新
        - 文档改进
        - 不改变API的性能优化
    </Accordion>
</AccordionGroup>

## 版本支持政策

- **最新主要版本**：完全支持，积极开发中（ACTIVE状态）
- **上一主要版本**：在下一个主要版本发布后12个月内提供安全更新和关键错误修复（MAINTENANCE状态）
- **更早版本**：仅社区支持

### 长期支持（LTS）版本

LangChain和LangGraph 1.0都被指定为LTS版本：

- 1.0版本将保持ACTIVE状态，直到2.0版本发布
- 2.0版本发布后，1.0版本将进入MAINTENANCE模式至少1年
- LTS版本遵循语义版本控制（semver），允许在次要版本之间安全升级
- 旧版本（LangChain 0.3和LangGraph 0.4）处于MAINTENANCE模式，直至2026年12月

### 1.0之前的包

:::python
**Deep Agents** (`deepagents`是一个正在积极开发的1.0之前的包。作为快速发展的包，API可能在次要版本之间发生变化，但我们会尽可能减少破坏性更改。Deep Agents在达到1.0后将采用与LangChain和LangGraph相同的LTS政策。
:::

有关发布状态和支持时间线的详细信息，请参阅 [发布政策](/oss/release-policy).

## 升级

:::python
<CodeGroup>

```bash LangChain
# Upgrade to the latest version
pip install -U langchain-core langchain

# Upgrade to a specific version
pip install langchain-core==1.0.0
```

```bash LangGraph
# Upgrade to the latest version
pip install -U langgraph

# Upgrade to a specific version
pip install langgraph==1.0.0
```

```bash Deep Agents
# Upgrade to the latest version
pip install -U deepagents

# Upgrade to a specific version
pip install deepagents==0.1.0
```
</CodeGroup>
:::

:::js
<CodeGroup>

```bash LangChain
# Upgrade to the latest version
npm update langchain @langchain/core

# Install a specific version
npm install langchain@1.0.0 @langchain/core@1.0.0
```

```bash LangGraph
# Upgrade to the latest version
npm update @langchain/langgraph

# Install a specific version
npm install @langchain/langgraph@1.0.0
```
</CodeGroup>
:::

## 预发布版本

我们偶尔会发布alpha和beta版本供早期测试：

- **Alpha** (e.g., `1.0.0a1`）：早期预览，预期会有重大变化
- **Beta** (e.g., `1.0.0b1`）：功能完整，可能有小幅更改
- **候选发布** (e.g., `1.0.0rc1`）：稳定发布前的最终测试
