## 设置

要通过 OpenRouter 访问模型，您需要创建一个 [OpenRouter 账户](https://openrouter.ai/)，获取 API 密钥，并安装 `langchain-openrouter` 集成包。

### 安装

LangChain OpenRouter 集成位于 `langchain-openrouter` 包中：

<CodeGroup>

</fragment>
    ```bash pip
    pip install -U langchain-openrouter
    ```
    ```bash uv
    uv add langchain-openrouter
    ```
</CodeGroup>

### 凭证

前往 [OpenRouter keys page](https://openrouter.ai/settings/keys) 注册并生成 API 密钥。完成后，设置 `OPENROUTER_API_KEY` 环境变量：

```python
import getpass
import os

if not os.getenv("OPENROUTER_API_KEY"):
    os.environ["OPENROUTER_API_KEY"] = getpass.getpass("Enter your OpenRouter API key: ")
```

要启用模型调用的自动跟踪，请设置您的 [LangSmith](/langsmith/observability) API 密钥：

```python
os.environ["LANGSMITH_API_KEY"] = getpass.getpass("Enter your LangSmith API key: ")
os.environ["LANGSMITH_TRACING"] = "true"
```

## 实例化

现在我们可以实例化模型对象并生成聊天补全：

```python
from langchain_openrouter import ChatOpenRouter

model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    temperature=0,
    max_tokens=1024,
    max_retries=2,
    # other params...
)
```

```python
stream = model.stream_events("Write a short poem about the sea.", version="v3")
for token in stream.text:
```


## 工具调用

OpenRouter 使用 OpenAI 兼容的工具调用格式。你可以描述工具及其参数，并让模型返回一个包含要调用的工具及其输入参数的 JSON 对象。

### 绑定工具

通过 `ChatOpenRouter.bind_tools`，你可以将 Pydantic 类、字典模式、LangChain 工具或函数作为工具传递给模型。这些工具在底层会被转换为 OpenAI 工具模式，并在每次模型调用时传递给模型。

```python
from pydantic import BaseModel, Field


class GetWeather(BaseModel):
    """Get the current weather in a given location"""

    location: str = Field(description="The city and state, e.g. San Francisco, CA")


model_with_tools = model.bind_tools([GetWeather])
```

```python
ai_msg = model_with_tools.invoke(
    "what is the weather like in San Francisco",
)
ai_msg
```

```text
AIMessage(content='', response_metadata={'finish_reason': 'tool_calls'}, tool_calls=[{'name': 'GetWeather', 'args': {'location': 'San Francisco, CA'}, 'id': 'call_abc123', 'type': 'tool_call'}], usage_metadata={'input_tokens': 68, 'output_tokens': 17, 'total_tokens': 85})
```

### 工具调用

AIMessage 具有 `tool_calls` 属性，其中包含以标准化格式呈现的工具调用，与模型提供商无关。

```python
ai_msg.tool_calls
```

```text
[{'name': 'GetWeather',
  'args': {'location': 'San Francisco, CA'},
  'id': 'call_abc123',
  'type': 'tool_call'}]
```

### 严格模式

传递 `strict=True` 以确保模型输出与工具定义中提供的 JSON Schema 完全匹配：

```python
model_with_tools = model.bind_tools([GetWeather], strict=True)
```

有关绑定工具和工具调用输出的更多信息，请参阅[工具调用](/oss/langchain/tools)文档。

## 结构化输出

`ChatOpenRouter` 通过 [`with_structured_output`](/oss/langchain/models#structured-output) 方法支持结构化输出。提供两种方法：`function_calling`（默认）和 `json_schema`。

<Accordion title="单个模型调用">

使用 `with_structured_output` 生成结构化模型响应。指定 `method="json_schema"` 使用基于 JSON Schema 的结构化输出；否则该方法默认使用函数调用。

</Accordion>

```python
from langchain_openrouter import ChatOpenRouter
from pydantic import BaseModel, Field

model = ChatOpenRouter(model="openai/gpt-5.5")

class Movie(BaseModel):
    """A movie with details."""
    title: str = Field(description="The title of the movie")
    year: int = Field(description="The year the movie was released")
    director: str = Field(description="The director of the movie")
    rating: float = Field(description="The movie's rating out of 10")

structured_model = model.with_structured_output(Movie, method="json_schema")  # [!code highlight]
response = structured_model.invoke("Provide details about the movie Inception")
response
```

```python
Movie(title='Inception', year=2010, director='Christopher Nolan', rating=8.8)
```

</Accordion>

<Accordion title="Agent 响应格式">

指定 `response_format` 并配合使用 [`ProviderStrategy`](/oss/langchain/structured-output)，以在生成 Agent 最终响应时启用结构化输出。

</Accordion>

```python
from langchain.agents import create_agent
from langchain.agents.structured_output import ProviderStrategy
from pydantic import BaseModel

class Weather(BaseModel):
    temperature: float
    condition: str

def weather_tool(location: str) -> str:
    """Get the weather at a location."""
    return "Sunny and 75 degrees F."

agent = create_agent(
    model="openrouter:openai/gpt-5.5",
    tools=[weather_tool],
    response_format=ProviderStrategy(Weather),  # [!code highlight]
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "What's the weather in SF?"}]
})

result["structured_response"]
```

```python
Weather(temperature=75.0, condition='Sunny')
```

</Accordion>

你可以使用 `strict=True` 与 `function_calling` 和 `json_schema` 方法来强制精确遵循模式。`strict` 参数不支持与 `json_mode`。

```python
structured_model = model.with_structured_output(Movie, method="json_schema", strict=True)
```

## 推理输出

对于支持推理的模型（例如 `anthropic/claude-sonnet-4.5`、`deepseek/deepseek-r1`），你可以通过 `reasoning` 参数启用推理令牌。请参阅 [OpenRouter 推理文档](https://openrouter.ai/docs/guides/best-practices/reasoning-tokens) 了解更多详情：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    max_tokens=16384,
    reasoning={"effort": "high", "summary": "auto"},
)

ai_msg = model.invoke("What is the square root of 529?")

# Access reasoning content via content_blocks
for block in ai_msg.content_blocks:
    if block["type"] == "reasoning":
        print(block["reasoning"])
```

有关内容块的更多信息，请参阅[标准内容块](/oss/langchain/messages#standard-content-blocks)指南。

`reasoning` 字典支持两个键：

- `effort`：控制推理 token 预算。值：`"xhigh"`、`"high"`、`"medium"`、`"low"`、`"minimal"`、`"none"`。
- `summary`：控制响应中返回的推理摘要的详细程度。值：`"auto"`、`"concise"`、`"detailed"`。

推理 token 使用量包含在 `usage_metadata` 中：

```python
print(ai_msg.usage_metadata)
# {'input_tokens': ..., 'output_tokens': ..., 'total_tokens': ...,
#  'output_token_details': {'reasoning': ...}}
```

<Note>
    努力程度到预算的映射取决于模型。例如，Google Gemini 模型将努力程度映射到内部的 `thinkingLevel`，而不是精确的 token 预算。详见 [OpenRouter reasoning docs](https://openrouter.ai/docs/guides/best-practices/reasoning-tokens)。
</Note>

OpenRouter 支持接受多模态输入的模型。请参阅 [OpenRouter models page](https://openrouter.ai/models) 了解可用的模态，具体取决于您选择的模型。

### 支持的输入方式

| 方法 | 图片 | 音频 | 视频 | PDF |
|--------|:-----:|:-----:|:-----:|:---:|
| HTTP/HTTPS URL | ✅ | ❌ | ✅ | ✅ |

## Token 使用量元数据

调用后，token 使用量信息可在响应的 `usage_metadata` 属性中获取：

```python
ai_msg = model.invoke("Tell me a joke.")
ai_msg.usage_metadata
```

```text
{'input_tokens': 12,
 'output_tokens': 25,
 'total_tokens': 37}
```

当底层提供商在其响应中包含详细的 token 分解时，它们会自动显示。当提供商未报告这些信息或其值为零时，这些字段会被省略。

### 推理 tokens

`output_token_details.reasoning` 报告模型用于内部思维链推理的 token 数量。这在使用推理模型（例如 `deepseek/deepseek-r1`、`openai/o3`）或明确启用推理时出现：

```python
from langchain_openrouter import ChatOpenRouter

model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    reasoning={"effort": "high"},
)

ai_msg = model.invoke("What is the square root of 529?")
ai_msg.usage_metadata
```

```text
{'input_tokens': 39,
 'output_tokens': 98,
 'total_tokens': 137,
 'output_token_details': {'reasoning': 62}}
```

### 缓存的输入 token

`input_token_details.cache_read` 报告从提供商的提示缓存中服务的输入 token 数量，而 `input_token_details.cache_creation` 报告首次调用时写入缓存的 token 数量。

提示缓存需要在消息内容块中设置明确的 `cache_control` 断点。在你想要缓存的内容块上传递 `{"cache_control": {"type": "ephemeral"}}`：

```python
from langchain_openrouter import ChatOpenRouter

model = ChatOpenRouter(model="anthropic/claude-sonnet-4.5")

long_system = "You are a helpful assistant. " * 200
messages = [
    ("system", [{"type": "text", "text": long_system, "cache_control": {"type": "ephemeral"}}]),
    ("human", "Say hi."),
]

# First call writes to cache
ai_msg = model.invoke(messages)
ai_msg.usage_metadata
```

```text
{'input_tokens': 1210,
 'output_tokens': 12,
 'total_tokens': 1222,
 'input_token_details': {'cache_creation': 1201}}
```

```python
# Second call reads from cache
ai_msg = model.invoke(messages)
ai_msg.usage_metadata
```

```text
{'input_tokens': 1210,
 'output_tokens': 12,
 'total_tokens': 1222,
 'input_token_details': {'cache_read': 1201}}
```

<Note>
    如果消息内容块上没有 `cache_control`，则提供者不会缓存提示词，这些字段也不会出现。
</Note>

流式传输时，从完整输出中访问令牌使用情况：

```python
stream = model.stream_events("Tell me a joke.", version="v3")
full = stream.output
full.usage_metadata
```

```text
{'input_tokens': 12,
 'output_tokens': 25,
 'total_tokens': 37}
```

## 提供商路由

OpenRouter 上的许多模型由多个提供商提供服务。`openrouter_provider` 参数让您可以控制哪些提供商处理您的请求以及如何选择它们。

### 提供商的排序和过滤

使用 `order` 设置首选提供商序列。OpenRouter 按顺序尝试每个提供商，如果某个提供商不可用，则会回退到下一个：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    openrouter_provider={
        "order": ["Anthropic", "Google"],
        "allow_fallbacks": True,  # default; fall back beyond the order list if needed
    },
)
```

要仅将请求限制在特定提供商，请使用 `only`。要排除某些提供商，请使用 `ignore`：`

```python
# Only use these providers (no fallback to others)
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={"only": ["OpenAI", "Azure"]},
)

# Use any provider except DeepInfra
model = ChatOpenRouter(
    model="meta-llama/llama-4-maverick",
    openrouter_provider={"ignore": ["DeepInfra"]},
)
```

### 按成本、速度或延迟排序

默认情况下，OpenRouter 在提供商之间进行负载均衡，优先选择成本较低的选项。使用 `sort` 更改优先级：

```python
# Prefer the fastest providers (highest tokens/second)
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={"sort": "throughput"},
)

# Prefer the lowest-latency providers
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={"sort": "latency"},
)
```

### 数据收集策略

如果您的使用场景要求提供商不存储或训练您的数据，请将 `data_collection` 设置为 `"deny"`：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    openrouter_provider={"data_collection": "deny"},
)
```

### 按量化筛选

对于开源模型，您可以限制路由到特定的精度级别：

```python
model = ChatOpenRouter(
    model="meta-llama/llama-4-maverick",
    openrouter_provider={"quantizations": ["fp16", "bf16"]},
)
```

### 路由参数

`route` 参数控制高级路由行为：

- `"fallback"`：启用跨提供商的自动故障转移（默认行为）。
- `"sort"`：根据在 `openrouter_provider` 中配置的排序策略进行路由。

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    route="fallback",
)
```

### 组合选项

提供程序选项可以组合在一起：

```python
model = ChatOpenRouter(
    model="openai/gpt-4o",
    openrouter_provider={
        "order": ["OpenAI", "Azure"],
        "allow_fallbacks": False,       # strict—only use providers in order
        "require_parameters": True,     # skip providers that don't support all params
        "data_collection": "deny",
    },
)
```

请参阅 [OpenRouter 提供商路由文档](https://openrouter.ai/docs/guides/routing/provider-selection) 获取完整的选项列表。

## 应用归因

OpenRouter 支持通过 HTTP 标头进行应用归因。您可以通过初始化参数或环境变量设置这些标头：

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    app_url="https://myapp.com",   # or OPENROUTER_APP_URL env var
    app_title="My App",            # or OPENROUTER_APP_TITLE env var
)
```

## 可观测性与追踪

OpenRouter 可以将请求数据广播到配置好的可观测性目标。`ChatOpenRouter` 提供两个相关参数：`session_id` 用于将多个请求分组到同一个逻辑工作流中，以及 `trace` 用于每个请求的追踪元数据。详见 [OpenRouter 广播文档](https://openrouter.ai/docs/guides/features/broadcast/overview)。

### 使用 `session_id` 对请求进行分组

传递 `session_id` 以将多个请求关联到同一个工作流（对话、代理运行、批处理作业、CI 运行等）。最大 256 个字符。

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    session_id="workflow-abc-123",   # or OPENROUTER_SESSION_ID env var
)
```

`OPENROUTER_SESSION_ID` 环境变量在实例化时读取（当没有提供显式 `session_id` 时），这允许进程标记每个请求，而无需通过应用程序代码传递该值。

您也可以在每次调用时覆盖该值：

```python
model.invoke("Summarize the doc", session_id="workflow-abc-123-step-1")
```

### 使用 `trace` 添加追踪元数据

传入 `trace` 以附加每个请求的元数据，OpenRouter 会将这些元数据转发到广播目标。识别的键包括 `trace_id`、`trace_name`、`span_name`、`generation_name` 和 `parent_span_id`；额外的键将作为自定义元数据传递。

```python
model = ChatOpenRouter(
    model="anthropic/claude-sonnet-4.5",
    trace={
        "trace_id": "trace-789",
        "span_name": "summarize",
    },
)
```

`session_id` 和 `trace` 是相互独立的——`session_id` 在 OpenRouter 端将请求分组为逻辑工作流，而 `trace` 则为单个请求添加注解。
