并非每个代理交互都是聊天。有时代理正在执行多步骤计划，而展示进度的最佳方式是**任务列表**，它可以实时更新。深度代理任务列表模式直接从代理的状态中读取 `todos` 数组，在代理执行计划时将每个项目及其当前状态呈现出来。这是一个基于与聊天相同的 `useStream` hook 构建的进度仪表板。它表明代理状态可以为任何 UI 提供支持，而不仅仅是消息气泡。

## 工作原理

深度代理包含一个内置的 **`todos` 状态**，用于跟踪代理执行计划时的任务进度。
当代理执行时，它会更新每个待办事项的状态，从 `"pending"` 到 `"in_progress"`
再到 `"completed"`。@[`useStream`] 钩子通过 `stream.values.todos` 暴露这个
状态，你的 UI 会响应式地渲染它。

流程如下：

1. 用户提交请求
2. 代理创建计划并在其状态中填充 `todos`
3. 代理开始执行每个待办事项，通过 `pending` → `in_progress` → `completed`
   进行状态转换
4. `stream.values.todos` 在代理执行过程中实时更新
5. 你的 UI 会使用当前状态重新渲染待办事项列表

## 设置 `useStream`

无需特殊配置。只需将 @[`useStream`] 指向你的代理，然后从 `stream.values`
读取 `todos`。

<CodeGroup>

</CodeGroup>
```tsx React
import { useStream } from "@langchain/react";

const AGENT_URL = "http://localhost:2024";

export function TodoAgent() {
  const stream = useStream<typeof myAgent>({
    apiUrl: AGENT_URL,
    assistantId: "deep_agent_todo_list",
  });

  const todos = stream.values?.todos ?? [];

  return (
    <div>
      <TodoList todos={todos} />
      {stream.messages.map((msg) => (
        <Message key={msg.id} message={msg} />
      ))}
    </div>
  );
}
```

## 构建 TodoList 组件

待办列表使用状态图标、颜色编码和视觉样式来渲染每个项目，以反映当前状态：

```tsx
function TodoList({ todos }: { todos: Todo[] }) {
  const completed = todos.filter((t) => t.status === "completed").length;
  const percentage = todos.length
    ? Math.round((completed / todos.length) * 100)
    : 0;

  return (
    <div className="rounded-lg border bg-white p-4 shadow-sm">
      <div className="mb-4 flex items-center justify-between">
        <h2 className="text-lg font-semibold">Agent Progress</h2>
        <span className="text-sm text-gray-500">
          {completed}/{todos.length} tasks
        </span>
      </div>

      <ProgressBar percentage={percentage} />

      <ul className="mt-4 space-y-2">
        {todos.map((todo, i) => (
          <TodoItem key={i} todo={todo} />
        ))}
      </ul>
    </div>
  );
}
```

## 进度条

视觉进度条让用户一目了然地了解整体完成情况：

```tsx
function ProgressBar({ percentage }: { percentage: number }) {
  return (
    <div className="space-y-1">
      <div className="flex items-center justify-between text-xs text-gray-500">
        <span>Progress</span>
        <span>{percentage}%</span>
      </div>
      <div className="h-2 overflow-hidden rounded-full bg-gray-200">
        <div
          className="h-full rounded-full bg-green-500 transition-all duration-500"
          style={{ width: `${percentage}%` }}
        />
      </div>
    </div>
  );
}
```

## 单个待办事项

每个项目都会获得一个状态图标、颜色编码的文本，以及针对已完成任务的删除线样式：

```tsx
function TodoItem({ todo }: { todo: Todo }) {
  const config = {
    pending: {
      icon: "○",
      textClass: "text-gray-600",
      bgClass: "bg-gray-50",
      iconClass: "text-gray-400",
    },
    in_progress: {
      icon: "◉",
      textClass: "text-amber-800",
      bgClass: "bg-amber-50 border-amber-200",
      iconClass: "text-amber-500 animate-pulse",
    },
    completed: {
      icon: "✓",
      textClass: "text-green-800 line-through",
      bgClass: "bg-green-50 border-green-200",
      iconClass: "text-green-500",
    },
  };

  const style = config[todo.status];

  return (
    <li
      className={`flex items-start gap-3 rounded-md border px-3 py-2 ${style.bgClass}`}
    >
      <span className={`mt-0.5 text-lg leading-none ${style.iconClass}`}>
        {style.icon}
      </span>
      <span className={`text-sm ${style.textClass}`}>{todo.content}</span>
    </li>
  );
}
```

`in_progress` 图标使用 `animate-pulse` 来吸引用户注意当前正在进行的任务。

## 计算进度

直接从 todos 数组中获取进度指标：

```ts
const todos = stream.values?.todos ?? [];

const completed = todos.filter((t) => t.status === "completed").length;
const inProgress = todos.filter((t) => t.status === "in_progress").length;
const pending = todos.filter((t) => t.status === "pending").length;
const percentage = todos.length
  ? Math.round((completed / todos.length) * 100)
  : 0;
```

这些值会随着代理修改其状态而响应式更新，使进度条和计数器保持同步。

## 与聊天消息结合使用

待办事项列表与常规聊天界面配合使用。一种实用的布局是将待办事项列表作为持久化侧边栏或顶部面板显示，聊天消息显示在下方：

```tsx
function TodoAgentLayout() {
  const stream = useStream<typeof myAgent>({
    apiUrl: AGENT_URL,
    assistantId: "deep_agent_todo_list",
  });

  const todos = stream.values?.todos ?? [];

  return (
    <div className="flex h-screen flex-col">
      {todos.length > 0 && (
        <div className="border-b bg-gray-50 p-4">
          <TodoList todos={todos} />
        </div>
      )}

      <main className="flex-1 overflow-y-auto p-6">
        <div className="mx-auto max-w-2xl space-y-4">
          {stream.messages.map((msg) => (
            <Message key={msg.id} message={msg} />
          ))}
        </div>
      </main>

      <ChatInput
        onSubmit={(text) =>
          stream.submit({ messages: [{ type: "human", content: text }] })
        }
        isLoading={stream.isLoading}
      />
    </div>
  );
}
```

<Tip>
仅当 `todos.length > 0` 时显示待办列表。代理在创建计划之前，没有什么可显示的。显示空组件会浪费空间。
</Tip>

## 使用场景

待办列表模式适用于代理执行结构化计划的任何场景：

- **项目规划**：代理将项目分解为任务，然后按顺序完成
- **研究工作流**：每个研究问题都会成为一个待办项，代理会进行调查并完成
- **数据处理**：摄取、验证、转换等步骤，以及
  export each get their own todo
- **入职流程**：代理逐步完成设置步骤，在配置服务时逐一勾选完成
- **报告生成**：报告的各个部分变成待办事项：收集数据、分析趋势、撰写摘要、格式化输出

## 处理空状态和加载状态

在代理创建计划之前处理初始状态：

```tsx
function TodoList({ todos, isLoading }: { todos: Todo[]; isLoading: boolean }) {
  if (todos.length === 0 && !isLoading) {
    return null;
  }

  if (todos.length === 0 && isLoading) {
    return (
      <div className="rounded-lg border bg-white p-4 shadow-sm">
        <div className="flex items-center gap-2 text-sm text-gray-500">
          <span className="animate-spin">⟳</span>
          Agent is creating a plan...
        </div>
      </div>
    );
  }

  return (
    <div className="rounded-lg border bg-white p-4 shadow-sm">
      {/* ... full todo list rendering */}
    </div>
  );
}
```

## 最佳实践

- **突出显示待办列表**。这是基于计划型代理的主要进度指示器。不要将其埋没在折叠区域以下。
- **为状态转换添加动画效果**。平滑的过渡效果让代理感觉更灵敏。对背景色、文本装饰和透明度使用 CSS 过渡效果。
- **只高亮显示一个 `in_progress` 项目**。代理通常一次只处理一个任务。如果多个项目显示为 `in_progress`，界面会变得嘈杂。考虑只让第一个项目产生脉冲效果。
- **折叠或淡化已完成的项目**。随着列表的增长，已完成的项目变得不那么重要。降低其视觉权重，以便用户专注于仍在进行的内容。
- **显示进度百分比**。像"67% 已完成"这样的单一数字一目了然，即使在房间的另一端也能理解。
- **保持待办列表同步**。由于 `stream.values` 会响应式更新，待办列表会自动保持最新。不要添加手动轮询或刷新逻辑。
