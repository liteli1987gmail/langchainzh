解释器为代理提供了一个可编程的工作空间，让他们可以探索数据、协调工具调用，并将中间工作保持在模型上下文之外。代理编写代码来表达其意图，然后 **in-memory** 运行时执行该代码并返回相关结果。

其中 [沙箱](/oss/deepagents/sandboxes) 是一种以代码优先的方式对环境进行操作（如运行命令、安装依赖和编辑文件），而解释器是一种以代码优先的方式在代理循环内进行操作：组合工具、保存状态，以及决定哪些信息应该返回给模型。

<Warning>
    解释器目前处于 [**beta**](/oss/versioning)状态。API 和生命周期行为可能在版本之间发生变化。
</Warning>

:::python
<Note>
    解释器需要 `langchain-quickjs>=0.1.0` 和 Python `>=3.11`.
</Note>
:::

## 为什么使用解释器？

大多数代理工作交替进行模型推理和工具调用。模型可以在一次响应中触发多个工具调用，但该批次在发出时就固定了。没有另一次模型响应，就无法循环、基于结果分支、重试失败或将一个调用的输出传递到下一个调用，每个结果都返回到模型的上下文中。模型还决定发出多少个调用，因此要求它在数百个项目上分配工作是不可靠的，而且它倾向于覆盖一个样本而不是每一个。

解释器为代理提供了执行该工作的运行时。循环在每次迭代中运行，工具从代码中调用，中间值保留在变量中，只有紧凑的结果返回给模型。

  <Card title="Programmatic subagents" icon="arrows-split" href="/oss/deepagents/programmatic-subagents">

    在代码中对结构化数据进行排序、分组、解析、验证、打分、聚合和探索。

## 选择一种模式

在代理循环内使用解释器来处理代码：组合工具、保存状态和控制返回给模型的内容。使用 [沙箱](/oss/deepagents/sandboxes) 对环境执行代码：shell 命令、包安装、测试、文件系统编辑和操作系统级执行。

| 需求 | 使用 |
| --- | --- |
| 一两个简单的外部调用 | 常规工具调用 |
| 循环、分支、重试或聚合结果的小程序 | 解释器 |
| 应从代码运行的大量选定工具调用 | 带 [程序化工具调用 (PTC) 的解释器](#programmatic-tool-calling-ptc) |
| 大型输入的多个独立工作单元、多个视角或递归分析 | 带 [程序化子代理](/oss/deepagents/programmatic-subagents) |
| Shell 命令、包安装、测试或完整操作系统文件系统访问 | [沙箱](/oss/deepagents/sandboxes) |

## 解释器如何工作

中间件向代理添加了一个 `eval` 工具。当有用时，代理编写 JavaScript 并调用 `eval`; 您不直接调用解释器。该工具在持久化上下文中运行代码，捕获 `console.log`，并返回最后一个表达式的结果。

代理可以像这样编写代码：

```javascript
const rows = [
  { team: "alpha", score: 8 },
  { team: "beta", score: 13 },
  { team: "alpha", score: 21 },
];

const totals = rows.reduce((acc, row) => {
  acc[row.team] = (acc[row.team] ?? 0) + row.score;
  console.log(`${row.team} score: ${acc[row.team]}`)
  return acc;
}, {});

totals;
```

:::python
默认情况下，解释器状态也会通过在每次代理运行后对工作状态进行快照，并在下次运行前恢复，从而在同一线程的多次对话中持久化。
:::

代码在 [**QuickJS**](https://github.com/quickjs-ng/quickjs)上运行，这是一个轻量级的 JavaScript 运行时。默认情况下，解释器代码无法访问宿主文件系统、网络、shell、包管理器或时钟。它可以计算、保存状态并写入 `console.log`，仅此而已。

两个明确的桥接机制扩展了其能力范围：

- **工具**，通过 [程序化工具调用（PTC）](#programmatic-tool-calling-ptc)。将工具白名单作为异步函数公开在 `tools` 命名空间下。这些可以是代理自己的工具，也可以是您定义并传入的独立工具。
- **子代理**，通过 [程序化子代理](/oss/deepagents/programmatic-subagents)。从代码中调度已配置的子代理并用纯 JavaScript 进行编排。

程序化工具调用默认关闭，需要您 [启用它](#enable-ptc)。只要代理拥有子代理，子代理调度默认开启，您也可以将其关闭。除非您主动暴露，否则不会有任何东西越过 QuickJS 边界。

## 程序化工具调用（PTC）

程序化工具调用（PTC）在解释器内的全局 `tools` 命名空间下公开选定的代理工具。与请求模型发出一次工具调用、等待结果、然后决定下次调用不同，代理可以编写在循环、分支、重试或并行批次中调用工具的代码。

这在中间结果只是下一步输入时很有帮助：解释器在返回任何内容给模型之前对其进行过滤或聚合，保持多步骤工作流的 token 效率。它与模型无关，通过中间件而非提供商特定的工具调用 API 实现。

中间件在 `tools`下将每个白名单工具公开为异步函数。代理使用 `await`调用它，在代码中处理结果，模型只看到最终的解释器输出，而非每个中间值。工具名称转换为驼峰命名，而输入对象仍遵循工具的 schema，因此名为 `web_search` 的工具变为 `tools.webSearch(...)`:

```typescript
const result: string = await tools.webSearch({
  query: "deepagents interpreters",
});
```

### 启用 PTC

使用显式白名单启用 PTC：

:::python
```python
from deepagents import create_deep_agent
from langchain_quickjs import CodeInterpreterMiddleware

agent = create_deep_agent(
    model="openai:gpt-5.5",
    middleware=[CodeInterpreterMiddleware(ptc=["web_search"])],
)
```
:::

:::js
```typescript
import { createDeepAgent } from "deepagents";
import { createCodeInterpreterMiddleware } from "@langchain/quickjs";

const agent = createDeepAgent({
  model: "openai:gpt-5.5",
  middleware: [createCodeInterpreterMiddleware({ ptc: ["web_search"] })],
});
```
:::

启用 PTC 后，代理可以从解释器代码中调用白名单工具。此示例并行搜索多个主题并在返回模型之前合并结果：

```javascript
const topics = ["retrieval", "memory", "evaluation"];

const results = await Promise.all(
  topics.map((topic) =>
    tools.webSearch({ query: `${topic} best practices 2025` }),
  ),
);

results.join("\n\n");
```

:::python
<Warning>
    PTC 调用当前通过解释器桥接执行，不会经过正常的工具调用路径。因此， `interrupt_on` 不会对每个 PTC 调用的工具强制执行审批工作流。
</Warning>
:::

## 程序化子代理

程序化子代理允许解释器调度已配置的 [子代理](/oss/deepagents/subagents) 使用内置函数从代码中 `task()` global。跨越多个独立单元的任务（如审查目录中的每个文件或分类一批工单）变成一个将工作分发出去并综合结果的循环。

使用编程式子代理的场景：

- **扇出与综合**：并行对多个项目运行相同类型的工作，然后合并结果。
- **验证**：将发现发送给独立的验证器子代理，只保留已确认的结果。
- **递归工作流**：在工作集中维护

有关配置、示例、编排模式和安全说明，请参阅 [编程式子代理](/oss/deepagents/programmatic-subagents).

## 持久化

`CodeInterpreterMiddleware` 默认情况下在每次代理运行后对解释器状态进行快照，并在下一次运行前恢复它。快照是解释器内存中 JavaScript 状态的序列化副本，包括在代理完成代码运行时的全局变量、变量、函数和导入模块。

在对话轮次之间，生命周期如下：

1. 一轮对话开始，然后 `CodeInterpreterMiddleware` 恢复该线程的最新解释器快照。
2. 代理调用 `eval`，代码可以读取或修改解释器变量。
3. 代理运行完成，中间件将更新后的解释器状态快照保存到图状态中。
4. 下一轮对话从恢复后的解释器状态开始，而不是从空的运行时开始。

在单个代理运行中，重复的 `eval` 调用使用活动的解释器上下文对象。中间件不会在这些调用之间进行快照和恢复；它在运行完成时对上下文进行快照，以便在后续轮次或检查点重放时恢复。

<Note>
    在对话轮次之间，快照仅保留可以合理序列化的值。将它们用于数据，而不是用于运行时对象。函数、类和其他不可序列化的值被恢复为不可访问的产物。如果解释器代码在恢复后访问其中任何一个，eval 工具将抛出类似 `Value for 'fn' was not restored because it is not serializable (type: function).`
</Note>

快照保留解释器内存，而非外部世界的影响。如果解释器代码通过 PTC 调用工具，恢复之前的解释器快照不会撤销该工具调用的副作用。它仅恢复记录或处理结果的解释器变量。

当图使用检查点器时，这与 [LangGraph 时间旅行](/oss/langgraph/use-time-travel)配对使用。恢复图检查点可以恢复存储在图状态中的解释器快照，因此您可以在调试或重放时返回到更早的代理上下文和解释器状态。

```python
from deepagents import create_deep_agent
from langchain_quickjs import CodeInterpreterMiddleware
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()

agent = create_deep_agent(
    model="openai:gpt-5.5",
    checkpointer=checkpointer,
    middleware=[
        CodeInterpreterMiddleware(
            snapshot_between_turns=True,  # Default
        )
    ],
)
```

您可以使用 `snapshot_between_turns=False`.
:::

## 安全

解释器使用 QuickJS 运行不受信任的 JavaScript，具有严格的默认隔离。将此视为一个作用域受限的解释器运行时，而非完整的生产沙箱后端。

您通过 PTC 暴露的每个工具都是解释器代码可以使用的外部能力。将 PTC 允许列表视为权限边界：仅暴露代理所需的工具，避免桥接可以访问敏感系统、花费资金、修改数据或调用不受限制网络的各种工具，除非该行为是有意的。

:::python
| 能力 | 默认可用 | 如何暴露 |
| --- | --- | --- |
| JavaScript 执行 | 是 | 添加解释器中间件 |
| 顶层 `await` | 是 | 在解释器代码中使用 promises |
| `console.log` 捕获 | 是 | 使用以下方式禁用 `capture_console=False` |
| 智能体工具 | 否 | 添加 PTC 白名单 |
| 文件系统访问 | 否 | 添加 [内置文件系统工具](/oss/deepagents/overview#virtual-filesystem-access) 通过 PTC 白名单 |
| 网络访问 | 否 | 通过 PTC 暴露特定网络工具 |
| 挂钟时间或日期时间访问 | 否 | 需要时暴露显式时间工具 |
| Shell 命令、包安装、测试、操作系统级执行 | 否 | 使用 [沙箱后端](/oss/deepagents/sandboxes) |
:::
:::js
| 能力 | 默认可用 | 如何暴露 |
| --- | --- | --- |
| JavaScript 执行 | 是 | 添加解释器中间件 |
| 顶级 `await` | 是 | 在解释器代码中使用 promises |
| `console.log` 捕获 | 是 | 使用以下方式禁用 `captureConsole: false` |
| 智能体工具 | 否 | 添加 PTC 白名单 |
| 文件系统访问 | 否 | 添加 [内置文件系统工具](/oss/deepagents/overview#virtual-filesystem-access) 通过 PTC 白名单 |
| 网络访问 | 否 | 通过 PTC 暴露特定网络工具 |
| 挂钟时间或日期时间访问 | 否 | 需要时暴露显式时间工具 |
| Shell 命令、包安装、测试、操作系统级执行 | 否 | 使用 [沙箱后端](/oss/deepagents/sandboxes) |
:::

:::python
<Note>
    **代码执行工作原理**

    解释器代码在嵌入式 QuickJS 上下文中运行，而不是单独的 VM 或进程中。在 Python 中，此运行时由 [`quickjs-rs`](https://github.com/langchain-ai/quickjs-rs)提供，其在 [安全指南](https://github.com/langchain-ai/quickjs-rs#security).

    将解释器视为能力作用域的执行层，而非主机内存隔离边界。对于不受信任或半信任的代码，请在隔离的工作进程或容器中运行智能体，并保持 PTC 白名单的狭窄范围。
</Note>
:::

## 配置

:::python
`CodeInterpreterMiddleware` 接受以下选项：

| 参数 | 默认值 | 用途 |
| --- | --- | --- |
| `memory_limit` | `64 * 1024 * 1024` <br/>(64 MB) | QuickJS 堆内存限制（字节）。|
| `timeout` | `5.0` | 每次评估超时时间（秒）。|
| `max_ptc_calls` | `256` | 最大 `tools.*` 每次评估的调用次数。仅在受信任环境中使用 `None` 。|
| `tool_name` | `"eval"` | 暴露给模型的解释器工具名称。|
| `max_result_chars` | `4000` | 从结果和 stdout 块返回的最大字符数。|
| `capture_console` | `True` | 是否 `console.log`, `console.warn`和 `console.error` 输出被捕获。|
| `subagents` | `True` | 暴露内置 `task()` 全局变量用于 [编程式子智能体](/oss/deepagents/programmatic-subagents)。设置为 `False` 以要求子智能体通过常规 `task` 工具路径进行调度。|
| `ptc` | `None` | PTC 白名单：工具名称或 `BaseTool` 实例列表。|
| `snapshot_between_turns` | `True` | 解释器状态快照是否在智能体轮次之间保持。|
| `max_snapshot_bytes` | `None` | 最大序列化快照大小。默认为 `memory_limit`. |
:::

:::js
`createCodeInterpreterMiddleware` 接受以下选项：

| 选项 | 默认值 | 用途 |
| --- | --- | --- |
| `ptc` | omitted | PTC 白名单：工具名称或 `StructuredToolInterface` 实例数组。 |
| `memoryLimitBytes` | `64 * 1024 * 1024` <br/>(64 MB) | QuickJS 内存限制（字节）。 |
| `maxStackSizeBytes` | `320 * 1024` | QuickJS 堆栈大小限制（字节）。 |
| `executionTimeoutMs` | `5000` | 每次求值的超时时间（毫秒）。负值表示禁用超时。 |
| `systemPrompt` | `null` | 覆盖内置的解释器系统提示。 |
| `maxPtcCalls` | `256` | 最大 `tools.*` 每次求值的调用次数。仅在 `null` 可信环境中使用。 |
| `maxResultChars` | `4000` | 控制台输出、结果和错误字符串保留的最大字符数。 |
| `toolName` | `"eval"` | 暴露给模型的解释器工具名称。 |
| `captureConsole` | `true` | 是否 `console.log`, `console.warn`和 `console.error` 输出被捕获。 |
| `subagents` | `true` | 暴露内置的 `task()` 全局对象，用于 [编程式子代理](/oss/deepagents/programmatic-subagents)。设为 `false` 以要求子代理通过正常的 `task` 工具路径进行调度。 |
:::
