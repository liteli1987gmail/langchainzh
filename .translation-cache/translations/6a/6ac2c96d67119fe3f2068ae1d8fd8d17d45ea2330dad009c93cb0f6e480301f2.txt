- [数据集版本控制](#version-a-dataset)，用于跟踪随时间的变化。
- 用于评估的[筛选](#evaluate-on-a-filtered-view-of-a-dataset)和[拆分](#evaluate-on-a-dataset-split)数据集。
- 公开[共享数据集](#share-a-dataset)。
- 以各种格式[导出数据集](#export-a-dataset)。

您还将学习如何将[实验](/langsmith/evaluation-concepts#experiment)中的[筛选后的跟踪](#export-filtered-traces-from-experiment-to-dataset)导出回数据集，以便进一步分析和迭代。

<Tip>
  [LangSmith Engine](/langsmith/engine) 可以根据您的生产跟踪自动生成真实数据集示例。

## 数据集版本控制

在 LangSmith 中，数据集是版本化的。这意味着每次您添加、更新或删除数据集中的示例时，都会创建一个新的数据集版本。

### 创建数据集的新版本

每次您在数据集中添加、更新或删除示例时，都会创建数据集的一个新[版本](/langsmith/evaluation-concepts#dataset-organization)。这允许您跟踪数据集随时间的变更，并了解数据集的演变过程。

默认情况下，版本由变更的时间戳定义。当您在**示例**选项卡中点击数据集的特定版本（按时间戳）时，您将找到该时间点的数据集状态。

![Version Datasets](/langsmith/images/version-dataset.png)

请注意，查看数据集的过去版本时，示例是只读的。您还将看到此版本的数据集与最新版本之间的操作。

<Note>
默认情况下，**示例**选项卡中显示数据集的最新版本，**测试**选项卡中显示所有版本的数据集对应的实验结果。
</Note>

在**测试**选项卡中，您将找到在不同版本的数据集上运行的测试结果。

![Version Datasets](/langsmith/images/version-dataset-tests.png)

### 为版本添加标签

您也可以为数据集的版本添加标签，使其具有更易读的名称，这对于标记数据集历史中的重要里程碑很有帮助。

例如，您可以将数据集的某个版本标记为"prod"，并使用它来针对您的 LLM 管道运行测试。

您可以通过在**示例**选项卡中点击 **+ Tag this version** 在 UI 中为数据集的版本添加标签。

![Tagging Datasets](/langsmith/images/tag-this-version.png)

您也可以使用 SDK 为数据集的版本添加标签。以下是使用 [Python SDK](https://docs.smith.langchain.com/reference/python/reference) 为数据集版本添加标签的示例：

```python
from langsmith import Client
from datetime import datetime

client = Client()
initial_time = datetime(2024, 1, 1, 0, 0, 0) # The timestamp of the version you want to tag

# You can tag a specific dataset version with a semantic name, like "prod"
client.update_dataset_tag(
    dataset_name=toxic_dataset_name, as_of=initial_time, tag="prod"
)
```

若要在数据集的特定标记版本上运行评估，请参阅[在特定数据集版本上评估部分](#evaluate-on-a-specific-dataset-version)。

### 使用 `list_examples`

您可以使用 `evaluate` / `aevaluate` 传入可迭代的示例，在数据集的特定版本上运行评估。使用 `list_examples` / `listExamples` 通过 `as_of` / `asOf` 从特定版本标签获取示例，并将其传入 `data` 参数。

<CodeGroup>

</CodeGroup>

```python Python
from langsmith import Client

ls_client = Client()

# Assumes actual outputs have a 'class' key.
# Assumes example outputs have a 'label' key.
def correct(outputs: dict, reference_outputs: dict) -> bool:
  return outputs["class"] == reference_outputs["label"]

results = ls_client.evaluate(
    lambda inputs: {"class": "Not toxic"},
    # Pass in filtered data here:
    data=ls_client.list_examples(
      dataset_name="Toxic Queries",
      as_of="latest",  # specify version here
    ),
    evaluators=[correct],
)
```

```typescript TypeScript
import { evaluate } from "langsmith/evaluation";

await evaluate((inputs) => labelText(inputs["input"]), {
  data: langsmith.listExamples({
    datasetName: datasetName,
    asOf: "latest",
  }),
  evaluators: [correctLabel],
});
```

```java Java
import com.langchain.smith.models.examples.ExampleListParams;

ExampleListParams listParams = ExampleListParams.builder()
    .datasetId(datasetId)
    .asOf("latest")
var examples = client.examples().list(listParams);
```

</CodeGroup>

详细了解如何在 [以编程方式创建和管理数据集](/langsmith/manage-datasets-programmatically#fetch-datasets) 页面上获取数据集的视图。

### 在数据集的过滤视图中进行评估

您可以使用 `list_examples` / `listExamples` 方法从数据集中[获取](/langsmith/manage-datasets-programmatically#fetch-examples)要评估的示例子集。

一种常见的工作流是获取具有特定元数据键值对的示例。

<CodeGroup>

```python Python
from langsmith import evaluate

results = evaluate(
    lambda inputs: label_text(inputs["text"]),
    data=client.list_examples(dataset_name=dataset_name, metadata={"desired_key": "desired_value"}),
    evaluators=[correct_label],
    experiment_prefix="Toxic Queries",
)
```

```typescript TypeScript
import { evaluate } from "langsmith/evaluation";

await evaluate((inputs) => labelText(inputs["input"]), {
  data: langsmith.listExamples({
    datasetName: datasetName,
    metadata: {"desired_key": "desired_value"},
  }),
  evaluators: [correctLabel],
  experimentPrefix: "Toxic Queries",
});
```

```java Java
import com.langchain.smith.models.examples.ExampleListParams;

ExampleListParams listParams = ExampleListParams.builder()
    .datasetId(datasetId)
    .metadata("{\"desired_key\":\"desired_value\"}")
    .build();
var examples = client.examples().list(listParams);
```

</CodeGroup>

如需更多过滤功能，请参阅此[操作指南](/langsmith/manage-datasets-programmatically#list-examples-by-structured-filter)。

### 对数据集分割进行评估

您可以使用 `list_examples` / `listExamples` 方法对数据集的一个或多个[分割](/langsmith/evaluation-concepts#dataset-organization)进行评估。`splits` 参数接受您希望评估的分割列表。

<CodeGroup>

```python Python
from langsmith import evaluate

results = evaluate(
    lambda inputs: label_text(inputs["text"]),
    data=client.list_examples(dataset_name=dataset_name, splits=["test", "training"]),
    evaluators=[correct_label],
    experiment_prefix="Toxic Queries",
)
```

```typescript TypeScript
import { evaluate } from "langsmith/evaluation";

await evaluate((inputs) => labelText(inputs["input"]), {
  data: langsmith.listExamples({
    datasetName: datasetName,
    splits: ["test", "training"],
  }),
  evaluators: [correctLabel],
  experimentPrefix: "Toxic Queries",
});
```

```java Java
import com.langchain.smith.models.examples.ExampleListParams;
import java.util.Arrays;
import java.util.List;

List<String> splits = Arrays.asList("test", "training");

ExampleListParams listParams = ExampleListParams.builder()
    .datasetId(datasetId)
    .splits(splits)
    .build();
var examples = client.examples().list(listParams);
```

</CodeGroup>

有关获取数据集视图的更多详细信息，请参阅[获取数据集](/langsmith/manage-datasets-programmatically#fetch-datasets)指南。

## 分享数据集

### 公开分享数据集

<Warning>
公开分享数据集将使**数据集示例、实验及其关联的运行记录，以及关于此数据集的反馈可供拥有链接的任何人访问**，即使他们没有 LangSmith 账户。请确保您没有分享敏感信息。

此功能仅在 LangSmith 云托管版本中可用。
</Warning>

从 **数据集与实验** 选项卡中，选择一个数据集，点击 **⋮**（页面右上角），点击 **分享数据集**。这将打开一个对话框，您可以在其中复制数据集的链接。

![分享数据集](/langsmith/images/share-dataset.gif)

### 取消分享数据集

1. 在任何公开分享的数据集右上角点击 **公开**，然后在对话框中点击 **取消分享**。![取消分享数据集](/langsmith/images/unshare-dataset.png)

2. 通过点击 **设置** -> **分享链接** 或 [此链接](https://smith.langchain.com/settings/shared) 进入组织的公开分享数据集列表，然后点击要取消分享的数据集旁边的 **取消分享**。

![取消分享轨迹列表](/langsmith/images/unshare-trace-list.png)

## 导出数据集

您可以通过 LangSmith UI 将 LangSmith 数据集导出为 CSV、JSONL 或 [OpenAI 微调格式](https://platform.openai.com/docs/guides/fine-tuning#example-format)。

从 **数据集与实验** 选项卡中，选择一个数据集，点击 **⋮**（页面右上角），点击 **下载数据集**。

![导出数据集按钮](/langsmith/images/export-dataset-button.gif)

## 从实验导出筛选后的轨迹到数据集

在 LangSmith 中运行[离线评估](/langsmith/evaluation-concepts#offline-evaluations)后，您可能希望将满足某些评估标准的[轨迹](/langsmith/observability-concepts#traces)导出到数据集中。

### 查看实验轨迹

![导出筛选后的轨迹](/langsmith/images/export-filtered-trace-to-dataset.png)

首先，点击实验名称旁边的箭头。这将带您进入包含实验生成的轨迹的项目。

![导出筛选后的轨迹](/langsmith/images/experiment-tracing-project.png)

然后，您可以根据评估标准筛选轨迹。在这个示例中，我们筛选所有准确率分数大于 0.5 的轨迹。

![导出筛选后的轨迹](/langsmith/images/filtered-traces-from-experiment.png)

在项目上应用筛选条件后，我们可以多选运行记录添加到数据集，然后点击 **添加到数据集**。

![导出筛选后的轨迹](/langsmith/images/add-filtered-traces-to-dataset.png)
