持久化使 LangGraph 应用能够在单次图运行之外保留有用的信息。当智能体需要继续对话、从中断处恢复、从故障中恢复，或在不同交互中记住信息时，持久化就显得尤为重要。

LangGraph 提供两种互补的持久化系统：

- **[检查点器](/oss/langgraph/checkpointers)** 将线程的图状态持久化为检查点。用于短期、线程作用域的记忆，包括对话连续性、人在环工作流、时间旅行和容错。
- **[存储](/oss/langgraph/stores)** 将应用定义的数据持久化到图状态之外。用于长期、跨线程的记忆，包括用户偏好、事实和共享知识。

大多数应用可以同时使用两者：[检查点器](/oss/langgraph/checkpointers) 跟踪当前线程，[存储](/oss/langgraph/stores) 跟踪跨线程的持久化信息。

## 快速入门

使用检查点器、存储或两者同时编译您的图：

:::python
```python
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.store.memory import InMemoryStore

checkpointer = InMemorySaver()
store = InMemoryStore()

graph = builder.compile(checkpointer=checkpointer, store=store)

result = graph.invoke(
    {"messages": [{"role": "user", "content": "Hi, my name is Bob."}]},
    {"configurable": {"thread_id": "thread-1"}},
)
```
:::

:::js
```typescript
import { MemorySaver, MemoryStore } from "@langchain/langgraph";

const checkpointer = new MemorySaver();
const store = new MemoryStore();

const graph = builder.compile({ checkpointer, store });

const result = await graph.invoke(
  { messages: [{ role: "user", content: "Hi, my name is Bob." }] },
  { configurable: { thread_id: "thread-1" } }
);
```
:::

<Info>
**Agent Server 自动处理持久化**
使用 [Agent Server](/langsmith/agent-server) 时，无需手动实现或配置检查点器或存储。服务器在后台处理持久化基础设施。
</Info>

## 检查点器与存储

| | 检查点器 | 存储 |
| ---- | ---- | ---- |
| 持久化内容 | 图状态快照 | 应用定义的结构化数据 |
| 作用域 | 单个线程 | 跨线程 |
| 内存类型 | 短期、线程作用域内存 | 长期、跨线程内存 |
| 用途 | 对话连续性、人机交互、时间旅行和容错 | 用户偏好、事实和共享知识 |
| 访问模式 | 在图配置中传入 `thread_id` | 从节点或应用代码中读取和写入数据 |
| 完整指南 | [检查点器](/oss/langgraph/checkpointers) | [存储](/oss/langgraph/stores) |
