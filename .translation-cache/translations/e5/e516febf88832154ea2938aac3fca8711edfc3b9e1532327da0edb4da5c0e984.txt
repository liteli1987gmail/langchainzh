Agent 是一个在循环中调用工具的模型，直到给定任务完成。

<Note>
**Agent = Model + Harness**

Harness 的职责：在给定任务中，在正确的时机为模型提供正确的上下文。
</Note>

Harness 是循环周围的所有内容：模型、提示词、工具以及任何塑造其行为的中间件。

## 配置 Harness

`create_agent` 具有高度可扩展性。中间件是自定义的基元：每个部分处理一个关注点，在正确的时机钩入 Agent 循环，并与任何其他部分自由组合。只取你用例需要的内容，跳过其余部分。

常见模式作为一等中间件预先构建。你可以将任何其他内容构建为[自定义中间件](/oss/langchain/middleware/custom)。

### 上下文管理

### 规划和委托

复杂的任务通常超出单个上下文窗口的处理能力。委托让主 Agent 将工作分解成若干部分，交给子 Agent（每个在独立的上下文中运行），主 Agent 专注于协调而非执行。工作可以并行运行；主 Agent 的上下文保持简洁。
