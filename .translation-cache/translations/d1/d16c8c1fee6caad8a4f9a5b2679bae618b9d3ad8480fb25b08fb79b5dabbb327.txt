定义自定义同步 [子代理](/oss/deepagents/subagents) 为 markdown 文件，以便 Deep Agents Code 可以将专业任务委托给它们。

<Note>
    Deep Agents Code 目前不支持异步子代理。
</Note>

每个子代理位于其自己的文件夹中，带有 `AGENTS.md` file:

```text
.deepagents/agents/{subagent-name}/AGENTS.md   # Project-level
~/.deepagents/{agent}/agents/{subagent-name}/AGENTS.md  # User-level
```

项目子代理会覆盖同名用户子代理（参见 [优先级规则](/oss/deepagents/code/data-locations#subagents)).

头部需要 `name` 和 `description` （与 [`SubAgent` 字典规范](/oss/deepagents/subagents#subagent-dictionary-based)相同）。markdown 正文成为子代理的 `system_prompt`。除了基础规范外， `AGENTS.md` 文件支持一个可选 `model` 头部字段，用于覆盖主代理对此子代理的模型。使用 `provider:model-name` 格式（例如， `anthropic:claude-opus-4-8`, `openai:gpt-5.5`）。省略则继承主代理的模型。

<Note>
    其他 `SubAgent` 字段（`tools`, `middleware`, `interrupt_on`, `skills`）目前无法通过 `AGENTS.md` 头部配置——以这种方式定义的自定义子代理会继承主代理的工具。如需完全控制，请直接使用 SDK。
</Note>

## 文件格式

子代理 `AGENTS.md` 文件使用 YAML 头部，后跟 markdown 正文：

```markdown
---
name: researcher
description: Research topics on the web before writing content
model: anthropic:claude-haiku-4-5-20251001
---

You are a research assistant with access to web search.

## Your Process
1. Search for relevant information
2. Summarize findings clearly
```

## 示例：经济高效的子代理

使用更便宜、更快的模型处理简单的委托任务，同时让主代理使用功能更强大的模型：

```markdown
---
name: general-purpose
description: General-purpose agent for research and multi-step tasks
model: anthropic:claude-haiku-4-5-20251001
---

You are a general-purpose assistant. Complete the task efficiently and return a concise summary.
```

这会覆盖内置的通用子代理，将所有委托任务路由到更便宜的模型。参见 [覆盖通用子代理](/oss/deepagents/subagents#override-the-general-purpose-subagent) 了解更多。
