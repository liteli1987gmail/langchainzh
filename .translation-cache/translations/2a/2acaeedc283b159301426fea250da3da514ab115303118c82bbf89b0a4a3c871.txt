---
title: 在服务器日志中包含 HTTP 标头
sidebarTitle: 记录标头
---

默认情况下，[Agent Server](/langsmith/agent-server) 为了保护隐私，会从服务器日志中省略 HTTP 标头。但是，记录请求和关联 ID 可以帮助您调试问题并跨分布式系统跟踪请求。您可以通过修改 [`langgraph.json`](/langsmith/application-structure#configuration-file) 文件中的 `logging_headers` 部分来选择加入所有 API 调用的标头日志记录。

```json
{
  "$schema": "https://langgra.ph/schema.json",
  "http": {
    "logging_headers": {
      "includes": ["request-id", "x-purchase-id", "*-trace-*"],
      "excludes": ["authorization", "x-api-key", "x-organization-id", "x-user-id"]
    }
  }
}
```

`includes` 和 `excludes` 列表接受精确的头部名称或使用 `*` 作为通配符的 glob 模式来匹配任意数量的字符（不区分大小写）。为了保障安全，不支持其他类型的模式。

请注意，排除优先于包含。例如，如果您包含了 `*-id` 但排除了 `x-user-id`，则 `x-user-id` 头部不会被记录。
