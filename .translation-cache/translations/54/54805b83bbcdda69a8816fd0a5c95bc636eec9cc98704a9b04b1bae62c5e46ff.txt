>[Parallel](https://platform.parallel.ai/) 是一个专为 LLM 和 AI 应用构建的实时网络搜索和内容提取平台。

[Task API](https://docs.parallel.ai/task-api/task-quickstart) 在分层处理器菜单（`lite` → `ultra`，以及匹配的 `-fast` 变体）上运行研究级任务。`langchain-parallel` 将其作为四个 LangChain 接口公开，全部列在页面上，方便您为工作负载选择合适的形态。

## 我应该使用哪个接口？

- **一个临时问题，可被 agent 调用**: `ParallelTaskRunTool`（`BaseTool`）。
- **一份长时间运行、多来源的报告**: `ParallelDeepResearch`（`Runnable`）。
- **针对列表的批量富化，带类型化输入和输出**: `ParallelEnrichment`（`Runnable`）。
- **需要完全控制运行信封的低级批处理**: `ParallelTaskGroup`（普通类）。

所有四个接口默认使用 `-fast` 处理器变体（与同类非快速层相比，2-5 倍速度提升，精度相近）。当延迟不如最高质量重要时，可删除 `-fast` 后缀。参见[选择处理器](https://docs.parallel.ai/task-api/guides/choose-a-processor)获取完整菜单。

## 概览

### 集成详情

| 类 | 形态 | 默认处理器 | 包 |
| :--- | :--- | :--- | :--- |
| @[`ParallelTaskRunTool`] | `BaseTool` | `lite-fast` | @[`langchain-parallel`] |
| @[`ParallelDeepResearch`] | `Runnable` | `pro-fast` | @[`langchain-parallel`] |
| @[`ParallelTaskGroup`] | 普通类 | `lite-fast` | @[`langchain-parallel`] |
| @[`ParallelEnrichment`] | `Runnable` | `core-fast` | @[`langchain-parallel`] |

### 凭证

前往 [Parallel](https://platform.parallel.ai) 注册并生成 API 密钥。在您的环境变量中设置 `PARALLEL_API_KEY`：

```python
import getpass
import os

if not os.environ.get("PARALLEL_API_KEY"):
    os.environ["PARALLEL_API_KEY"] = getpass.getpass("Parallel API key:\n")
```

## ParallelTaskRunTool

`ParallelTaskRunTool` 是一个可被 agent 调用的 `BaseTool`。它同步运行一个任务，并返回结构化的 `output`、每个字段的 `basis` 引用以及 `run_id`。

```python
from langchain_parallel import ParallelTaskRunTool

tool = ParallelTaskRunTool()
result = tool.invoke({"input": "Who founded SpaceX, in one sentence?"})

print(result["output"]["content"])
print("run_id:", result["run"]["run_id"])
```

`result["output"]` 始终是一个字典；答案文本位于 `result["output"]["content"]`，每个字段的引用位于 `result["output"]["basis"]`。传入一个 `task_output_schema` 可以使 `content` 以解析后的 pydantic 形状字典而非自由文本字符串的形式返回：

```python
from pydantic import BaseModel, Field

class FounderFact(BaseModel):
    founder: str = Field(description="Full name of the founder")
    year: int = Field(description="Year the company was founded")

structured = ParallelTaskRunTool(task_output_schema=FounderFact)
res = structured.invoke({"input": "Who founded SpaceX and in what year?"})

print(res["output"]["content"])
print(res["output"]["basis"])  # citations + reasoning + confidence per field
```

```text
{'founder': 'Elon Musk', 'year': 2002}
[{'field': 'founder', 'citations': [...], 'reasoning': '...', 'confidence': 'high'}, ...]
```

### parse_basis: citations + low-confidence fields

每个关注置信度的消费者最终都会编写相同的样板代码来遍历结果中的引用、低置信度字段以及 `interaction_id`。`parse_basis()` 为你完成了这一切：

```python
from langchain_parallel import parse_basis

parsed = parse_basis(res)
print("citations per field:", {f: len(c) for f, c in parsed["citations_by_field"].items()})
print("low-confidence fields:", parsed["low_confidence_fields"])
print("interaction_id:", parsed["interaction_id"])
```

### 多轮对话链式调用

结果字典在顶层暴露 `interaction_id`。在下一轮调用时将其作为 `previous_interaction_id` 传递，以跨轮次链式传递上下文：

```python
first = tool.invoke({"input": "Who founded SpaceX?"})
followup = tool.invoke({
    "input": "And what year was it founded?",
    "previous_interaction_id": first["interaction_id"],
})
```

## ParallelDeepResearch

`ParallelDeepResearch` 是一个 `Runnable`。默认使用 `pro-fast`（即"探索性网络研究"的 `-fast` 变体）。为了获得最彻底的多源报告，请传入 `processor="ultra"`。

```python
from langchain_parallel import ParallelDeepResearch

research = ParallelDeepResearch()
result = research.invoke("Summarize the state of net-energy-gain fusion research as of 2026.")
print(result["output"]["content"])
```

<Note>
    深度研究运行并非即时完成。`pro-fast` 通常需要几分钟；`pro` 和 `ultra` 可能需要更长时间。在生产环境中，建议配置 webhook（请参阅 [Webhook signature verification](#webhook-signature-verification)）而非阻塞等待 `invoke` 结果。
</Note>

要使用类型化的深度研究，请传入 `output_schema`：

```python
class CityFact(BaseModel):
    capital: str
    population_millions: float

typed = ParallelDeepResearch(output_schema=CityFact)
out = typed.invoke("Capital and population (in millions) of France?")
print(out["output"]["content"])
```

## ParallelTaskGroup

`ParallelTaskGroup` 创建一个任务组，分发运行，并收集结果。当你需要对批处理信封进行细粒度控制时，直接使用它；否则优先使用 `ParallelEnrichment` 进行类型化的批量运行。

```python
from langchain_parallel import ParallelTaskGroup

group = ParallelTaskGroup()
results = group.run([
    "Founder of Anthropic? One sentence.",
    "Founder of OpenAI? One sentence.",
])

for r in results:
    print(r["output"]["content"])
```

`ParallelTaskGroup` 暴露了 `run`（同步）和 `arun`（异步）方法。延迟由批处理中最慢的运行和所选处理器决定——`lite-fast` 通常在秒级完成，更高级别则需要几分钟。

## ParallelEnrichment

`ParallelEnrichment` 使用您的输入/输出 pydantic schema 构建的 `default_task_spec` 包装了 `ParallelTaskGroup`。它将 pydantic 实例强制转换为字典，分发批处理任务，并按输入顺序返回结果。

```python
from langchain_parallel import ParallelEnrichment

class CompanyInput(BaseModel):
    company: str = Field(description="Company name")

class CompanyOutput(BaseModel):
    headquarters: str
    founding_year: int

enricher = ParallelEnrichment(
    input_schema=CompanyInput,
    output_schema=CompanyOutput,
)
results = enricher.invoke([
    CompanyInput(company="Anthropic"),
    {"company": "OpenAI"},
])

for r in results:
    print(r["output"]["content"])
```

`ParallelEnrichment` 块会等待所有输入解析完成。使用默认的 `core-fast` 处理器，非平凡批次预计需要几分钟；对于短字段请传入更快的处理器，或在后台 worker 中运行大型批次。

## build_task_spec

`build_task_spec` 接受 pydantic 类、原始 JSON-schema 字典或文本描述，并返回可用于 `client.task_run.create` 或 `add_runs(default_task_spec=...)` 的 `TaskSpec` 字典。当你想在 `ParallelTaskRunTool` 或 `ParallelTaskGroup` 上完全控制运行信封时使用它。

```python
from langchain_parallel import build_task_spec

spec = build_task_spec(input_schema=CompanyInput, output_schema=CompanyOutput)
print(list(spec.keys()))  # ['output_schema', 'input_schema']
```

## BYOMCP: 自带 MCP 服务器

`ParallelTaskRunTool` 和 `ParallelDeepResearch` 接受 `mcp_servers=[McpServer(...)]` 参数，将 Streamable-HTTP MCP 端点暴露到运行环境中。

```python
from langchain_parallel import McpServer

mcp = McpServer(
    name="my_internal_mcp",
    url="https://example.com/mcp",
    headers={"Authorization": "Bearer ..."},
)

tool_with_mcp = ParallelTaskRunTool(mcp_servers=[mcp])
result = tool_with_mcp.invoke({"input": "What's our latest internal release?"})
```

## Webhook 签名验证

长时间运行的任务可以通过 webhook 交付结果。使用 `verify_webhook` 验证签名（Standard Webhooks 方案：HMAC-SHA256 对 `<webhook-id>.<webhook-timestamp>.<body>` 进行计算，base64 编码，`v1,<sig>` 并附带重放保护）。有关交付契约，请参阅 [webhook setup](https://docs.parallel.ai/resources/webhook-setup)。

```python
from langchain_parallel import verify_webhook

ok = verify_webhook(
    body,
    webhook_id=request.headers["webhook-id"],
    webhook_timestamp=request.headers["webhook-timestamp"],
    webhook_signature=request.headers["webhook-signature"],
    secret=os.environ["PARALLEL_WEBHOOK_SECRET"],
)
if not ok:
    raise PermissionError("invalid webhook signature")
```

## 链式调用

将 `ParallelTaskRunTool` 绑定到任何支持工具调用的聊天模型，并通过 [`create_agent`](/oss/langchain/agents) 驱动智能体：

```python
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model

llm = init_chat_model(model="claude-opus-4-8")
agent = create_agent(model=llm, tools=[tool])

agent.invoke({"messages": [("human", "Look up Anthropic's founding year.")]})
```
