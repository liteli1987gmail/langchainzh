在本地使用 LangChain 构建代理时，可视化代理内部发生的情况、实时与之交互以及在问题出现时进行调试会很有帮助。 **LangSmith Studio** 是一个免费的可视化界面，用于从本地机器开发和测试您的 LangChain 代理。

Studio 连接到您本地运行的代理，向您展示代理执行的每个步骤：发送给模型的提示词、工具调用及其结果，以及最终输出。您可以测试不同的输入、检查中间状态，并在无需额外代码或部署的情况下迭代优化代理的行为。

本页介绍如何设置 Studio 与您的本地 LangChain 代理。

## 前提条件

开始之前，请确保您具备以下条件：

- **LangSmith 账户**：在 [smith.langchain.com](https://smith.langchain.com).
- **注册（免费）或登录**：请按照 [创建 API 密钥](/langsmith/create-account-api-key) guide.
- 如果您不希望数据被 [追踪](/langsmith/observability-concepts#traces) 到 LangSmith，请在您的应用程序中设置 `LANGSMITH_TRACING=false` 。在追踪禁用的状态下，没有任何数据会离开您的本地服务器。 `.env` 文件。

## 设置本地代理服务器

### 1. 安装 LangGraph CLI

 [LangGraph CLI](/langsmith/cli) 提供了一个本地开发服务器（也称为 [代理服务器](/langsmith/agent-server)），用于将您的代理连接到 Studio。

```shell
npx @langchain/langgraph-cli
```

### 2. 准备您的代理

如果您已有 LangChain 代理，可以直接使用。此示例使用一个简单的邮件代理：

```typescript title="agent.ts"
import { createAgent } from "@langchain/agents";

function sendEmail(to: string, subject: string, body: string): string {
  const email = {
    to,
    subject,
    body,
  };
  // ... email sending logic

  return `Email sent to ${to}`;
}

const agent = createAgent({
  model: "gpt-5.5",
  tools: [sendEmail],
  systemPrompt: "You are an email assistant. Always use the send_email tool.",
});
```

### 3. 环境变量

Studio 需要 LangSmith API 密钥来连接您的本地代理。创建一个 `.env` 文件在项目根目录，并添加您的 API 密钥（来自 [LangSmith](https://smith.langchain.com/settings).

<Warning>
    确保您的 `.env` 文件未被提交到版本控制（如 Git）中。
</Warning>

```bash .env
LANGSMITH_API_KEY=lsv2...
```

### 4. 创建 LangGraph 配置文件

LangGraph CLI 使用配置文件来定位您的代理并管理依赖项。在您的应用目录中创建一个 `langgraph.json` 文件：

```json title="langgraph.json"
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent.ts:agent"
  },
  "env": ".env"
}
```

@[`createAgent`] 函数自动返回一个编译后的 LangGraph 图，这就是配置文件中的 `graphs` 键所期望的值。

<Info>
有关配置文件 JSON 对象中每个键的详细说明，请参阅 [LangGraph 配置文件参考](/langsmith/cli#configuration-file).
</Info>

此时，项目结构应如下所示：

```bash
my-app/
├── src
│   └── agent.ts
├── .env
└── langgraph.json
```

### 5. 安装依赖项

```shell
yarn install
```

### 6. 在 Studio 中查看您的代理

启动开发服务器以将您的代理连接到 Studio：

```shell
npx @langchain/langgraph-cli dev
```

<Warning>
Safari 会阻止 `localhost` 到 Studio 的连接。作为解决方案，请使用 `--tunnel` 通过安全隧道访问 Studio。
</Warning>

服务器运行后，您的代理可以通过 API 访问，地址为 `http://127.0.0.1:2024` 也可以通过 Studio UI 访问，地址为 `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`:

<Frame>
![Studio UI 中的代理视图](/oss/images/studio_create-agent.png)
</Frame>

With Studio connected to your local agent, you can iterate quickly on your agent's behavior. Run a test input, inspect the full execution trace including prompts, tool arguments, return values, and token/latency metrics. When something goes wrong, Studio captures exceptions with the surrounding state to help you understand what happened.

开发服务器支持热重载——对代码中的提示词或工具签名进行更改后，Studio 会立即反映这些更改。从任何步骤重新运行对话线程来测试更改，无需从头开始。此工作流可扩展，从简单的单工具代理到复杂的多节点图。
