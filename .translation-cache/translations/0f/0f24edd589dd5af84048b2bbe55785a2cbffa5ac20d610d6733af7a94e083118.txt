本 notebook 介绍了如何使用 Unstructured [文档加载器](/oss/integrations/document_loaders) 加载多种类型的文件。 `Unstructured` 目前支持加载文本文件、PowerPoint、HTML、PDF、图片等。

请参阅 [Unstructured](/oss/integrations/providers/unstructured) 获取有关本地设置 Unstructured 的更多说明，包括设置所需系统依赖项。

## 概述

## 设置

### 凭据

默认情况下， `langchain-unstructured` 会安装一个占用空间较小的版本，需要将分区逻辑卸载到 Unstructured API，这需要 API 密钥。如果使用本地安装，则不需要 API 密钥。要获取 API 密钥，请前往 [此网站](https://unstructured.io) 获取 API 密钥，然后在下方单元格中设置：

```python
import getpass
import os

if "UNSTRUCTURED_API_KEY" not in os.environ:
    os.environ["UNSTRUCTURED_API_KEY"] = getpass.getpass(
        "Enter your Unstructured API key: "
    )
```

### 安装

#### 常规安装

运行本 notebook 的其余部分需要以下包。

```python
# Install package, compatible with API partitioning
pip install -qU langchain-unstructured unstructured-client unstructured "unstructured[pdf]" python-magic
```

#### 本地安装

如果想在本地运行分区逻辑，需要安装系统依赖项的组合，详见 [此处 Unstructured 文档](https://docs.unstructured.io/open-source/installation/full-installation).

例如，在 Mac 上可以通过以下方式安装所需依赖：

```bash
# base dependencies
brew install libmagic poppler tesseract

# If parsing xml / html documents:
brew install libxml2 libxslt
```

可以通过以下方式安装所需的 `pip` 本地依赖：

```bash
pip install "langchain-unstructured[local]"
```

## 初始化

该 `UnstructuredLoader` 支持从多种不同的文件类型加载数据。要了解关于 `unstructured` 包的更多信息，请参阅其 [文档](https://docs.unstructured.io/open-source/introduction/overview)/. In this example, we show loading from both a text file and a PDF file.

```python
from langchain_unstructured import UnstructuredLoader

file_paths = [
    "./example_data/layout-parser-paper.pdf",
    "./example_data/state_of_the_union.txt",
]


loader = UnstructuredLoader(file_paths)
```

## 后处理

如果需要在提取后对 `unstructured` 元素进行后处理，可以传递一个
`str` -> `str` 函数列表到 `post_processors` 实例化时的 `UnstructuredLoader`kwarg 参数中。这同样适用于其他 Unstructured 加载器。下面是一个示例。

```python
from langchain_unstructured import UnstructuredLoader
from unstructured.cleaners.core import clean_extra_whitespace

loader = UnstructuredLoader(
    "./example_data/layout-parser-paper.pdf",
    post_processors=[clean_extra_whitespace],
)

docs = loader.load()
```

## Unstructured API

如果想使用更小的包快速开始并获取最新的分区功能，可以运行 `pip install
unstructured-client` and `pip install langchain-unstructured`。有关
的更多信息，请参阅 `UnstructuredLoader`页面
[Unstructured 提供商页面](https://python.langchain.com/v0.1/docs/integrations/document_loaders/unstructured_file/).

当您传入时，加载器将使用托管的 Unstructured 无服务器 API 处理您的文档
你的 `api_key` 并设置 `partition_via_api=True`。你可以 [生成一个免费的 Unstructured API 密钥](https://unstructured.io/api-key/).

查看 [Docker 镜像说明以自托管 Unstructured API](https://github.com/Unstructured-IO/unstructured-api#dizzy-instructions-for-using-the-docker-image)
如果你想自托管 Unstructured API 或在本地运行它。

```python
from langchain_unstructured import UnstructuredLoader

loader = UnstructuredLoader(
    file_path="example_data/fake.docx",
    api_key=os.getenv("UNSTRUCTURED_API_KEY"),
    partition_via_api=True,
)

docs = loader.load()
docs[0]
```

### Unstructured SDK 客户端

使用 Unstructured API 进行分区依赖于 [Unstructured SDK
Client](https://docs.unstructured.io/api-reference/api-services/accessing-unstructured-api).

如果你想自定义客户端，你必须传递一个 `UnstructuredClient` 实例给 `UnstructuredLoader`。以下是展示如何自定义客户端功能的示例，例如使用你自己的 `requests.Session()`、传递一个替代的 `server_url`，以及自定义 `RetryConfig` 对象。有关自定义客户端或 SDK 客户端接受的其他参数的更多信息，请参阅 [Unstructured Python SDK](https://docs.unstructured.io/api-reference/api-services/sdk-python) 文档和 [API 参数](https://docs.unstructured.io/api-reference/api-services/api-parameters) 文档。请注意，所有 API 参数都应传递给 `UnstructuredLoader`.

<div class="alert alert-block alert-warning"><b>Warning:</b> 以下示例可能未使用最新版本的 UnstructuredClient，未来版本可能会有重大变更。有关最新示例，请参阅 <a href="https://docs.unstructured.io/api-reference/api-services/sdk-python">Unstructured Python SDK</a> docs.</div>

```python
import requests
from langchain_unstructured import UnstructuredLoader
from unstructured_client import UnstructuredClient
from unstructured_client.utils import BackoffStrategy, RetryConfig

client = UnstructuredClient(
        api_key_auth=os.getenv(
        "UNSTRUCTURED_API_KEY"
    ),  # Note: the client API param is "api_key_auth" instead of "api_key"
        client=requests.Session(),  # Define your own requests session
        server_url="https://api.unstructuredapp.io/general/v0/general",  # Define your own api url
        retry_config=RetryConfig(
                strategy="backoff",
                retry_connection_errors=True,
                backoff=BackoffStrategy(
                        initial_interval=500,
                        max_interval=60000,
                        exponent=1.5,
                        max_elapsed_time=900000,
        ),
    ),  # Define your own retry config
)

loader = UnstructuredLoader(
    "./example_data/layout-parser-paper.pdf",
        partition_via_api=True,
        client=client,
        split_pdf_page=True,
        split_pdf_page_range=[1, 10],
)

docs = loader.load()

print(docs[0].metadata["filename"], ": ", docs[0].page_content[:100])
```

## 分块

该 `UnstructuredLoader` 不支持 `mode` 作为参数来像旧版那样对文本进行分组
加载器 `UnstructuredFileLoader` 等其他工具那样。它转而支持"分块"。在
中，分块机制不同于你可能熟悉的其他分块机制，那些机制基于
on plain-text features--character sequences like "\n\n" or "\n" that might indicate a paragraph
边界或列表项边界来形成块。相反，所有文档都使用关于每种
文档格式的特定知识来将文档划分为语义单元（文档元素），我们仅需要在
单个元素超过所需的最大块大小时才诉诸文本分割。一般来说，
分块会合并连续元素以形成尽可能大的块，同时不超过
最大块大小。分块会产生一个 CompositeElement、Table 或 TableChunk 元素序列。
每个"块"都是这三种类型之一的实例。

参见此 [页面](https://docs.unstructured.io/open-source/core-functionality/chunking) 了解更多
关于分块选项的详细信息，但要重现与 `mode="single"`相同的行为，您可以设置
`chunking_strategy="basic"`, `max_characters=<some-really-big-number>`和 `include_orig_elements=False`.

```python
from langchain_unstructured import UnstructuredLoader

loader = UnstructuredLoader(
    "./example_data/layout-parser-paper.pdf",
    chunking_strategy="basic",
    max_characters=1000000,
    include_orig_elements=False,
)

docs = loader.load()

print("Number of LangChain documents:", len(docs))
print("Length of text in the document:", len(docs[0].page_content))
```

```text
Number of LangChain documents: 1
Length of text in the document: 42772
```

## 加载网页

`UnstructuredLoader` 接受一个 `web_url` kwarg，在本地运行时该参数会填充底层 Unstructured `url` 的 [partition](https://docs.unstructured.io/open-source/core-functionality/partitioning)参数。这允许解析远程托管的文档，例如 HTML 网页。

使用示例：

```python
from langchain_unstructured import UnstructuredLoader

loader = UnstructuredLoader(web_url="https://www.example.com")
docs = loader.load()

for doc in docs:
    print(f"{doc}\n")
```
