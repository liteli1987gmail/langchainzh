Insights 会自动分析您的跟踪记录，检测使用模式、常见智能体行为和失败模式，因此您无需手动审查数千条跟踪记录。

Insights 使用分层分类来理解您的数据并突出显示可操作的趋势。

<Note>
Insights 适用于 LangSmith Plus 和 Enterprise [计划](/langsmith/pricing-plans)。
</Note>

## 前置条件

- 在您的工作区中设置用于 Insights 的[模型配置](/langsmith/model-configurations)。
- 在 LangSmith 中创建规则的[权限](/langsmith/organization-workspace-operations#projects)（生成新的 Insights 报告所需）。
- 在 LangSmith 中查看跟踪项目的[权限](/langsmith/organization-workspace-operations#projects)（查看现有 Insights 报告所需）。

## 生成您的第一份 Insights 报告

<Tabs>
  <Tab title="UI" icon="layout-dashboard">

1. 在左侧菜单中导航至 **跟踪项目** 并选择一个跟踪项目。
1. 点击右上角的 **+新建**，然后选择 **新建 Insights 报告**，为该项目生成新的洞察。
1. 输入您的任务名称。
1. 如果您还没有配置，请在工作区设置中为 Insights[配置模型](/langsmith/model-configurations)。
1. 回答引导性问题，以使您的 Insights 报告专注于您想要了解的有关智能体的内容，然后点击 **运行任务**。

<Tip>切换到手动模式以[手动配置任务](#configure-a-job)。</Tip>

这将启动后台 Insights 报告。报告可能需要长达 30 分钟才能完成。

  </Tab>
  <Tab title="SDK" icon="code">

您可以使用 [Python SDK](/langsmith/smith-python-sdk) 对存储在 LangSmith 外部的数据生成 Insights 报告。这样您就可以分析来自生产系统、日志或其他来源的聊天历史记录。

当您调用 `generate_insights()` 时，SDK 将：

1. 将您的聊天历史记录作为跟踪记录上传到新的 LangSmith 项目。
1. 对上传的跟踪记录生成 Insights 报告。
1. 返回 [LangSmith UI](https://smith.langchain.com) 中结果的链接。

<CodeGroup>

```python Python
import os
from langsmith import Client

client = Client()

chat_histories = [
    [
        {"role": "user", "content": "how are you"},
        {"role": "assistant", "content": "good!"},
    ],
    [
        {"role": "user", "content": "do you like art"},
        {"role": "assistant", "content": "only Tarkovsky"},
    ],
]

report = client.generate_insights(
    chat_histories=chat_histories,
    name="Customer Support Topics - March 2024",
    instructions="What are the main topics and questions users are asking about?",
    openai_api_key=os.environ["OPENAI_API_KEY"],  # optional if already set as workspace secret
)

# client.poll_insights(report=report)
```
</CodeGroup>

  </Tab>
</Tabs>

<Note>
    在 1,000 条线程上生成洞察通常需要花费 $1.00-$2.00（使用 OpenAI 模型）和 $3.00-$4.00（使用当前 Anthropic 模型）。成本随采样线程数和每条线程的大小而增加。
</Note>

## 了解结果

任务完成后，您可以导航到 **Insights** 选项卡，在那里您将看到一个 Insights Report 表格。每个 Report 包含从追踪项目中特定样本生成的洞察。

<Frame caption="单个追踪项目的 Insights Reports"><img src="/langsmith/images/insights-job-results.png" /> </Frame>

点击进入您的任务，查看按自动生成的分类组织的追踪。

您可以通过分类和子分类深入了解，查看底层追踪、反馈和运行统计。

<Frame caption="https://chat.langchain.com 聊天机器人的常见对话主题"><img src="/langsmith/images/insights-nav.gif" /> </Frame>

### 执行摘要

在每个报告的顶部，您会找到执行摘要，其中显示了追踪中发现的最重要模式。这包括：

- 关键发现及显示每种模式出现频率的百分比。
- 可点击的引用（例如 #1、#2、#3），指向代理认为与您的问题高度相关的追踪。

### 顶级分类

您的追踪会自动分组到顶级分类中，这些分类代表了数据中最广泛的模式。

分布条形图显示每种模式出现的频率，让您轻松发现超出或低于预期的行为。

每个分类都有简要描述，并显示其中包含的追踪的聚合指标，包括：

- 典型追踪统计（如错误率、延迟、成本）
- 来自评估器的反馈分数
- 作为任务的一部分提取的[属性](#attributes)

### 子分类

点击任何分类会显示其细分为子分类，使您能够更细致地了解该分类追踪中的交互模式。

在 [Chat Langchain](https://chat.langchain.com) 示例中，在 **Data & Retrieval** 下有 **Vector Stores** 和 **Data Ingestion** 等子分类。

### 单独追踪

您可以通过点击查看分配给每个分类或子分类的追踪，查看追踪表格。从那里，您可以点击进入任何追踪以查看完整的对话详情。

## 配置任务

您可以使用自动生成流程创建 Insights Report，也可以手动配置。

### 自动生成配置

1. 打开 **New Insights** 并确保 **Auto** 开关处于活动状态。
2. 用自然语言回答关于您的代理目的、您想了解什么以及追踪如何结构化的问题。Insights 会将您的回答转换为草稿配置（任务名称、摘要提示、属性和采样默认值）。
3. 选择一个提供商，然后点击 **Generate config** 进行预览，或点击 **Run job** 立即启动。

**提供有用的上下文**

为获得最佳结果，为每个提示写一两句话，给 Insights 提供所需的上下文——您想了解什么、哪些信号或字段最重要，以及您已知哪些内容没有用处。您越清楚您的代理做什么以及其追踪如何结构化，Insights 就越能将示例分组为具体的、可操作的、与您理解数据方式一致的方式。

**描述您的追踪**

解释您的数据是如何组织的：这些是单次运行还是多轮对话？哪些输入和输出包含关键信息？这有助于 Insights 生成关注重要内容的摘要提示和属性。如果需要，您也可以直接从[摘要提示](#summary-prompt)部分直接指定变量。

### 选择模型

Insights 使用两个模型：

- **Thinking 模型**：执行聚类步骤（能力更强，成本更高）。
- **Summarization 模型**：生成每个追踪的摘要（更快，成本更低）。

这两个模型都从您在工作区中配置的提供商中选择。当在您的[模型配置](/langsmith/model-configurations)中为 Insights 启用了特定模型时，您可以单独选择它们。如果没有配置单独的模型，您选择一个提供商（OpenAI 或 Anthropic），Insights 会使用该提供商的默认模型。

为获得最佳结果，请使用来自同一提供商的模型来担任两个角色。

### 手动配置

手动配置让您拥有更多控制——例如，预定义您希望数据分组到的分类，或针对匹配特定反馈分数和过滤器的追踪。

#### 选择追踪

- **样本大小**：要分析的最大追踪数量（1,000 限制）。
- **时间范围**：从该时间范围中采样追踪。
- **过滤器**：其他追踪过滤器。当您调整过滤器时，您会看到有多少追踪匹配您的条件。

#### 分类

默认情况下，顶级分类是从底层追踪自下而上自动生成的。

在某些情况下，您预先知道感兴趣的特定分类，并希望任务将追踪分到那些预定义的分类中。

配置的 **Categories** 部分允许您通过枚举要使用的顶级分类的名称和描述来实现这一点。

子分类仍然在预定义的顶级分类内由算法自动生成。

当任务完成时，发现的顶级分类会自动保存回配置——但仅当配置之前没有定义分类时。这意味着后续的定时运行将重用这些分类以保持一致性。

#### 摘要提示

任务的第一步是为每条追踪创建一个简要摘要。然后对这些摘要进行分类。

在摘要中提取正确的信息对于获得有用的分类至关重要。

您可以编辑用于生成这些摘要的提示。编辑提示时需要考虑两件事：

- 摘要说明：任何不在追踪摘要中的信息都不会影响生成的分类，因此请确保提供清晰的说明，说明从每条追踪中提取哪些重要信息。
- 追踪内容：使用大括号语法指定追踪的哪些部分被传递给摘要生成器。包含大量输入和输出的大型追踪可能成本高昂且嘈杂。将提示缩减为仅包含追踪中最相关的部分可以改善您的结果。

您必须使用以下至少一个模板变量指定要发送给摘要生成器的每条追踪的哪些部分：

| 变量 | 描述 | 示例 |
| --- | --- | --- |
| `run.inputs` | 最近根运行的输入 | `{{run.inputs}}` |
| `run.outputs` | 最近根运行的输出 | `{{run.outputs}}` |
| `run.error` | 错误字符串（如果运行失败） | `{{run.error}}` |
| `run.feedback` | 所有反馈分数作为 JSON 对象 | `{{run.feedback}}` |
| `run.feedback.<key>` | 按键指定的特定反馈分数 | `{{run.feedback.correctness}}` |
| `all_thread_messages` | 线程的完整消息历史（仅适用于带有[线程](/langsmith/threads)的项目） | `{{all_thread_messages}}` |

您可以使用点表示法访问嵌套字段。例如，`{{run.inputs.foo.bar}}` 仅包含最后运行输入中 `foo` 内的 `bar` 字段。

<Note>
对于带有[线程](/langsmith/threads)的项目，Insights 会分析完整对话。只有每个线程中最新的根运行用于 `run.*` 变量。使用 `all_thread_messages` 访问完整对话历史。
</Note>

#### 属性

除了摘要之外，您还可以定义要从每条追踪中提取的其他字符串、数字和布尔属性。
这些属性将影响分类步骤——具有相似属性值的追踪往往会被归为一类。
您还可以查看每个分类的这些属性的聚合。

例如，您可能希望从每条追踪中提取属性 `user_satisfied: boolean`，以引导算法将正面和负面用户体验分开分类，并查看每个分类的平均用户满意度。

#### 过滤属性

您可以在布尔属性上使用 `filter_by` 参数，在生成洞察之前预过滤追踪。启用后，只有属性评估为 `true` 的追踪才会包含在分析中。

当您想将 Insights Report 集中在特定子集的追踪上时，这很有用。例如，仅分析错误、仅检查英语对话，或仅包含符合某些质量标准的追踪。

**工作原理：**
- 创建 Insights 配置时，为任何布尔属性添加 `"filter_by": true`。
- LLM 在摘要生成期间根据属性描述评估每条追踪。
- 在生成洞察之前，属性为 `false` 或缺失的追踪会被排除。

## 计划 Insights Reports

计划 Insights reports 以自动按重复周期运行。创建或编辑配置时，使用 **Schedule** 部分选择：

- **Daily**：每天 UTC 时间 8:00 运行。
- **Weekly on Monday**：每周一 UTC 时间 8:00 运行。
- **Custom**：输入您自己的 cron 表达式（UTC 时间）。

每个计划的运行都会使用您保存的配置生成一个新报告。时间范围是动态计算的。例如，"last 24 hours" 总是在执行时分析最近的 24 小时窗口。

## 保存您的配置

您可以选择使用 **Save as** 按钮保存配置以供将来重用。
如果您想随时间比较 Insights Reports 以识别用户和代理行为的变化，这尤其有用。

在创建新的 Insights Report 时，在窗格左上角的下拉菜单中选择以前保存的配置。
