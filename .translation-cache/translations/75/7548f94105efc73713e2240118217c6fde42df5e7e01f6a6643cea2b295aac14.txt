---
title: 追踪 OpenAI 兼容的 providers
sidebarTitle: OpenAI 兼容 API
description: 将任何 OpenAI 兼容 provider 的 LLM 调用追踪到 LangSmith。
---

许多 LLM provider 以与 OpenAI API 相同的格式接收请求。要将这些 provider 的调用追踪到 LangSmith，请构建一个指向 provider 基础 URL 的 OpenAI 客户端，然后用 @[`wrap_openai`] / [`wrapOpenAI`](https://reference.langchain.com/javascript/modules/langsmith.html) 将其包装。

对于直接 API 调用使用 `wrap_openai` / `wrapOpenAI`。当您需要追踪调用周围的应用程序逻辑或为每个调用设置元数据时，使用 @[`@traceable`]。

| | `wrap_openai` / `wrapOpenAI` | `@traceable` / `traceable` |
|---|---|---|
| Token 追踪 | 自动 | 需要 `run_type="llm"` |
| Run 类型 | LLM（自动设置） | 默认为 Chain |
| 追踪内容 | API 调用 | 包装它的函数 |
| 元数据 | 仅客户端级别（Python）；客户端级别或按调用（TypeScript） | 通过 @[`langsmith_extra`] 按调用 |

<Note>要直接追踪 OpenAI，请参阅[追踪 OpenAI 应用程序](/langsmith/trace-openai)。</Note>

## 追踪 API 调用

<CodeGroup>

```python Python
import os

import openai
from langsmith import wrappers

client = wrappers.wrap_openai(
    openai.OpenAI(
        base_url="https://<provider-base-url>/v1",
        api_key=os.environ["PROVIDER_API_KEY"],
    )
)

completion = client.chat.completions.create(
    model="<provider-model-name>",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(completion.choices[0].message.content)
```

```typescript TypeScript
import OpenAI from "openai";
import { wrapOpenAI } from "langsmith/wrappers/openai";

const client = wrapOpenAI(
  new OpenAI({
    baseURL: "https://<provider-base-url>/v1",
    apiKey: process.env.PROVIDER_API_KEY!,
  })
);

const completion = await client.chat.completions.create({
  model: "<provider-model-name>",
  messages: [{ role: "user", content: "Hello!" }],
});
console.log(completion.choices[0].message.content);
```

</CodeGroup>

## 添加元数据

<Tabs>
<Tab title="Python">

在包装客户端时传递 `tracing_extra`。元数据将应用于使用该客户端的所有调用。

```python
import os

import openai

from langsmith import wrappers

client = wrappers.wrap_openai(
    openai.OpenAI(
        base_url="https://<provider-base-url>/v1",
        api_key=os.environ["PROVIDER_API_KEY"],
    ),
    tracing_extra={"metadata": {"environment": "production"}},
)
```

<Tab title="TypeScript">

将选项作为第二个参数传入 `wrapOpenAI` 用于客户端级别的元数据，或每次调用时传入 [`langsmithExtra`](https://reference.langchain.com/javascript/modules/langsmith.html)。

</Tab>

```typescript
import OpenAI from "openai";
import { wrapOpenAI } from "langsmith/wrappers/openai";

const client = wrapOpenAI(
  new OpenAI({
    baseURL: "https://<provider-base-url>/v1",
    apiKey: process.env.PROVIDER_API_KEY!,
  }),
  { metadata: { environment: "production" } }
);

// Per-call metadata
const completion = await client.chat.completions.create(
  {
    model: "<provider-model-name>",
    messages: [{ role: "user", content: "Hello!" }],
  },
  { langsmithExtra: { metadata: { request_id: "abc123" } } }
);
```

</Tab>
</Tabs>

## 相关指南

某些提供商有专门的设置指南，使用 `@traceable` 或原生回调。这些方法在函数级别进行追踪，而不是直接包装客户端，或与提供商自己的 SDK 和路由层集成。

- [DeepSeek](/langsmith/trace-deepseek)：与 OpenAI 兼容的 API；指南使用带有自定义提供商元数据的 `@traceable`
- [LiteLLM](/langsmith/trace-litellm)：公开 OpenAI 兼容端点的代理；指南涵盖 `@traceable` 和 LiteLLM 内置的 LangSmith 回调
