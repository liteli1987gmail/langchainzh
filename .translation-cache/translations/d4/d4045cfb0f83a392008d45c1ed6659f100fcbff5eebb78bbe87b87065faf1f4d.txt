这是将应用程序部署到 LangSmith Cloud 的完整设置和管理指南。LangSmith Cloud 在 AWS 和 GCP 上运行（请参阅[云概览页面](/langsmith/cloud)了解区域详情）。

本指南涵盖两种部署方法：[LangSmith UI](https://smith.langchain.com)，可从已连接的 GitHub 仓库部署，以及 [`langgraph deploy` CLI 命令](/langsmith/cli#deploy)，可直接从本地机器构建和推送。

<Callout icon="bolt" color="#4F46E5" iconType="regular">
**如果您需要快速设置**，请先尝试[快速入门指南](/langsmith/deployment-quickstart)。
</Callout>

在设置之前，请查看[云概览页面](/langsmith/cloud)以了解 Cloud 托管模式。

## 前置条件

- 一个 [Plus 计划或更高级别](https://www.langchain.com/pricing) 的 LangSmith 账户。
- [验证 LangGraph API 是否在本地运行](/langsmith/local-dev-testing)。如果 API 无法成功运行（即 `langgraph dev`），则部署到 LangSmith 也会失败。

## 创建新部署

选择适合您工作流程的部署方法——LangSmith UI 连接到 GitHub 仓库并支持推送时自动部署，而 `langgraph deploy` CLI 命令可直接从本地项目目录构建和部署。

<Tabs>
  <Tab title="LangSmith UI">
    <Note>
    **需要一次性设置**：GitHub 组织所有者或管理员必须在 LangSmith UI 中完成 OAuth 流程，以授权 `hosted-langserve` GitHub 应用。这只需要每个工作区完成一次。完成初始 OAuth 授权后，所有具有部署权限的开发人员都可以创建和管理部署，无需 GitHub 管理员访问权限。
    </Note>

    从 [LangSmith UI](https://smith.langchain.com) 开始，在左侧导航面板中选择 **Deployments**。在右上角，选择 **+ New Deployment** 创建一个新部署：

    1. 在 **Create New Deployment** 面板中，填写必填字段。对于 **Deployment details**：
        1. 选择 **Import from GitHub** 并按照 GitHub OAuth 工作流程进行操作，安装并授权 LangChain 的 `hosted-langserve` GitHub 应用访问所选仓库。安装完成后，返回 **Create New Deployment** 面板，从下拉菜单中选择要部署的 GitHub 仓库。
            <Note> 安装 LangChain 的 `hosted-langserve` GitHub 应用的 GitHub 用户必须是组织的[所有者](https://docs.github.com/en/organizations/managing-peoples-access-to-your-organization-with-roles/roles-in-an-organization#organization-owners)或账户所有者。此授权只需要每个 LangSmith 工作区完成一次——后续部署可由任何具有部署权限的用户创建。</Note>
        1. 为部署指定一个名称。
        1. 指定所需的 **Git Branch**。部署与分支相关联。创建新修订版本时，将部署关联分支的代码。稍后可在[部署设置](#deployment-settings)中更新分支。
        1. 指定 [LangGraph API 配置文件](/langsmith/cli#configuration-file)的完整路径，包括文件名。例如，如果文件 `langgraph.json` 在仓库的根目录中，请指定 `langgraph.json`。
        1. 使用复选框 **Automatically update deployment on push to branch**。如果选中，当更改推送到指定的 **Git Branch** 时，部署将自动更新。您可以在 [LangSmith UI](https://smith.langchain.com) 的[部署设置](#deployment-settings)中启用或禁用此设置。
        对于 **Deployment Type**：
            - 开发部署适用于非生产用例，配置了最小资源。
            - 生产部署可处理高达 500 请求/秒，配置了高可用存储和自动备份。
        1. 确定部署是否应 **Shareable through Studio**。
            1. 如果未选中，部署只能使用[工作区](/langsmith/administration-overview#workspaces)的有效 LangSmith API 密钥访问。
            1. 如果选中，部署可通过 [Studio](/langsmith/studio) 供任何 LangSmith 用户访问。将提供一个直接指向 Studio 中该部署的 URL，以便与其他 LangSmith 用户共享。
        1. 指定 **Environment Variables** 和密钥。要配置部署的其他变量，请参阅[环境变量参考](/langsmith/env-var-cloud)。
            1. 敏感值（如 API 密钥，例如 `OPENAI_API_KEY`）应指定为密钥。
            1. 也可以指定其他非敏感环境变量。
        1. 系统会自动创建一个与部署同名的 LangSmith [跟踪项目](/langsmith/observability)。
    1. 在右上角，选择 **Submit**。几秒钟后，会出现 **Deployment** 视图，新部署将排队等待配置。
  </Tab>
  <Tab title="LangGraph CLI">
    <Note>
    `langgraph deploy` 命令处于**[测试阶段](/langsmith/release-stages)**。它需要安装并运行 [Docker](https://docs.docker.com/get-docker/)。在 Apple Silicon（M1/M2/M3）上，还需要 [Docker Buildx](https://docs.docker.com/build/install-buildx/) 来进行交叉编译到 `linux/amd64`。
    </Note>

    1. 安装 [LangGraph CLI](/langsmith/cli)：
        ```shell
        uv tool install langgraph-cli
        ```
1. 将您的 LangSmith API 密钥添加到项目根目录的 `.env` 文件中：
        ```shell
        LANGSMITH_API_KEY=lsv2_...
        ```
1. 从项目目录中运行部署命令：
        ```shell
        langgraph deploy
        ```
这会创建一个以项目目录命名的 `dev` 部署。使用 `--name` 来指定不同的名称，或使用 `--deployment-type prod` 创建生产部署：
        ```shell
        langgraph deploy --name my-agent --deployment-type prod
        ```
命令完成后，部署将排队等待配置。部署创建后，可以通过 [LangSmith UI](https://smith.langchain.com) 管理环境变量，也可以在 `langgraph.json` 的 [`env` 字段](/langsmith/cli#configuration-file) 中进行配置。
</Tab>
</Tabs>

## 创建新版本

在[创建新部署](#create-new-deployment)时，默认会创建一个新版本。您可以创建后续版本以部署新的代码更改。

<Tabs>
  <Tab title="LangSmith UI">
    从 [LangSmith UI](https://smith.langchain.com) 开始，在左侧导航面板中选择 **Deployments**。选择一个现有部署来创建新版本。

    1. 在 **Deployment** 视图中，点击右上角的 **+ New Revision**。
    1. 在 **New Revision** 模态框中填写必填字段。
        1. 指定 [API 配置文件](/langsmith/cli#configuration-file) 的完整路径，包括文件名。例如，如果文件 `langgraph.json` 在仓库的根目录下，请指定 `langgraph.json`。
        1. 确定部署是否应该 **Shareable through Studio**。
            - 如果未勾选，部署只能通过 [workspace](/langsmith/administration-overview#workspaces) 的有效 LangSmith API 密钥访问。
            - 如果勾选，部署将通过 [Studio](/langsmith/studio) 对任何 LangSmith 用户开放。将提供一个指向 Studio 的直接 URL，以便与其他 LangSmith 用户共享。
        1. 指定 **Environment Variables** 和 secrets。现有 secrets 和环境变量已预填充。如需为版本配置其他变量，请参阅[环境变量参考](/langsmith/env-var-cloud)。
            1. 添加新的 secrets 或环境变量。
            1. 删除现有的 secrets 或环境变量。
            1. 更新现有 secrets 或环境变量的值。
    1. 选择 **Submit**。几秒钟后，**New Revision** 模态框将关闭，新版本将排队等待部署。
  </Tab>
  <Tab title="LangGraph CLI">
    从项目目录重新运行 `langgraph deploy`。该命令通过名称查找现有部署，并使用您的最新代码更改创建新版本：

    ```shell
    langgraph deploy
    ```

要使用 ID 而非名称来指定特定部署，请使用 `--deployment-id`：

    ```shell
    langgraph deploy --deployment-id <DEPLOYMENT_ID>
    ```

使用 `langgraph deploy list` 查看所有部署并查找其 ID：

    ```shell
    langgraph deploy list
    ```

<Note>
`langgraph deploy` 只能更新最初通过 `langgraph deploy` 创建的部署。通过 LangSmith UI 或 GitHub 集成创建的部署无法使用此命令进行更新。
</Note>

## 查看构建和服务器日志

每个版本都可以查看构建和服务器日志。

<Tabs>
  <Tab title="LangSmith UI">
    从 **部署** 视图开始：

    1. 从 **版本** 表中选择所需的版本。右侧会滑出一个面板，默认选中 **构建** 选项卡，显示该版本的构建日志。
    2. 在面板中，选择 **服务器** 选项卡查看该版本的服务器日志。服务器日志仅在版本部署后才可用。
    3. 在 **服务器** 选项卡中，根据需要调整日期/时间范围选择器。默认情况下，日期/时间范围选择器设置为 **最近 7 天**。
  </Tab>
  <Tab title="LangGraph CLI">
    使用 `langgraph deploy logs` 获取部署的日志。

    如需查看服务器（运行时）日志：

    ```shell
    langgraph deploy logs
    ```

查看构建日志：

    ```shell
    langgraph deploy logs --type build
    ```

实时查看到达的日志：

    ```shell
    langgraph deploy logs --follow
    ```

按时间范围、日志级别或搜索字符串筛选日志：

    ```shell
    langgraph deploy logs --start-time 2026-03-01T00:00:00Z --level ERROR
    ```

如果您有多个部署，请按名称或 ID 指定目标：

    ```shell
    langgraph deploy logs --name my-agent
    langgraph deploy logs --deployment-id <DEPLOYMENT_ID>
    ```

## 查看部署指标

部署上线后，您可以从 [LangSmith UI](https://smith.langchain.com) 监控其性能。

从 LangSmith UI 开始：

1. 在左侧导航面板中，选择 **Deployments**。
1. 选择要监控的现有部署。
1. 选择 **Monitoring** 选项卡以查看部署指标。请参阅[所有可用指标](/langsmith/control-plane#monitoring)列表。
1. 在 **Monitoring** 选项卡中，根据需要使用日期/时间范围选择器。默认情况下，日期/时间范围选择器设置为 **Last 15 minutes**。

## 中断修订版本

中断修订版本将停止该修订版本的部署。

<Warning>
**未定义行为**
已中断的修订版本具有未定义的行为。此操作仅在您需要部署新修订版本且已有修订版本"卡住"时有用。未来此功能可能会被移除。
</Warning>

从 **Deployments** 视图开始：

1. 在 **Revisions** 表中，选择所需修订版本行右侧的菜单图标（三个点）。
1. 从菜单中选择 **Interrupt**。
1. 将出现一个模态框。查看确认消息。选择 **Interrupt revision**。

## 删除部署

<Tabs>
  <Tab title="LangSmith UI">
    从 [LangSmith UI](https://smith.langchain.com) 开始：

    1. 在左侧导航面板中，选择 **Deployments**，其中包含现有部署列表。
    1. 选择所需部署行右侧的菜单图标（三个点），然后选择 **Delete**。
    1. 将出现 **Confirmation** 模态框。选择 **Delete**。
  </Tab>
  <Tab title="LangGraph CLI">
    使用 `langgraph deploy list` 查找要删除的部署 ID：

    ```shell
    langgraph deploy list
    ```

然后按 ID 删除它：

    ```shell
    langgraph deploy delete <DEPLOYMENT_ID>
    ```

要跳过确认提示，请使用 `--force`：

    ```shell
    langgraph deploy delete --force <DEPLOYMENT_ID>
    ```
</Tab>
</Tabs>

## 部署设置

从 **部署** 视图开始：

1. 在右上角，选择齿轮图标（**部署设置**）。
1. 将 `Git Branch` 更新为所需的分支。
1. 勾选/取消勾选复选框以**在推送到分支时自动更新部署**。
    1. 分支创建/删除和标签创建/删除事件不会触发更新。只有推送到现有分支才会触发更新。
    1. 快速连续推送到分支将队列后续更新。一旦构建完成，最近的提交将开始构建，其他排队的构建将被跳过。

## 添加或移除 GitHub 仓库

安装并授权 LangChain 的 `hosted-langserve` GitHub 应用后，可以修改该应用的仓库访问权限以添加新仓库或移除现有仓库。如果创建了新仓库，可能需要显式添加。

1. 从 GitHub 个人资料中，导航到 **Settings** > **Applications** > `hosted-langserve` > 点击 **Configure**。
1. 在 **Repository access** 下，选择 **All repositories** 或 **Only select repositories**。如果选择了 **Only select repositories**，则必须显式添加新仓库。
1. 点击 **Save**。
1. 创建新部署时，下拉菜单中的 GitHub 仓库列表将更新以反映仓库访问权限的变更。
