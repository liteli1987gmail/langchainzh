浏览不同提供商的可用中间件，或为生态系统贡献您自己的中间件。在 [中间件概述](/oss/langchain/middleware/overview) 中了解中间件的工作原理，以及如何在 [深度代理文档](/oss/deepagents/customization#middleware).

## 分享您的中间件

中间件支持上下文工程、工具定制和运行时安全控制。它是 LangChain 中的一个有用的扩展点，我们乐于展示社区用它构建的内容：

<CardGroup cols={2}>
    <Card title="Add an official integration" icon="package" href="/oss/contributing/implement-langchain#middleware">
        按照贡献指南来构建和发布中间件包。
    </Card>
    <Card title="Share a community middleware" icon="users" href="https://github.com/langchain-ai/docs">
        向文档仓库提交 PR，将您的中间件添加到下表中。
    </Card>
</CardGroup>

## 官方集成

| 提供商 | 可用中间件 | 来源 |
|------------|-------------|--------|
| [Anthropic](/oss/integrations/middleware/anthropic) | 提示缓存、bash 工具、文本编辑器、记忆和文件搜索 | [`langchain-ai/langchain`](https://github.com/langchain-ai/langchain/tree/master/libs/partners/anthropic) |
| [AWS](/oss/integrations/middleware/aws) | 提示缓存 | [`langchain-ai/langchain-aws`](https://github.com/langchain-ai/langchain-aws/tree/main/libs/aws) |
| [Microsoft Foundry](/oss/integrations/middleware/azure_ai) | 文本审核、图像审核、提示防护、受保护材料和基础性 | [`langchain-ai/langchain-azure`](https://github.com/langchain-ai/langchain-azure/tree/main/libs/azure-ai) |
| [OpenAI](/oss/integrations/middleware/openai) | 内容审核 | [`langchain-ai/langchain`](https://github.com/langchain-ai/langchain/tree/master/libs/partners/openai) |

## 社区集成

<Note>
社区维护这些中间件集成。它们是基于开源方式贡献的，由社区而非 LangChain 管理或维护。
</Note>

| 中间件 | 描述 | 来源 |
|-----------|-------------|--------|
| [Cisco AI Defense](https://github.com/cisco-ai-defense/ai-defense-langchain-middleware) | 运行时安全检查 | [`cisco-ai-defense/ai-defense-langchain-middleware`](https://github.com/cisco-ai-defense/ai-defense-langchain-middleware) |
| [compact-middleware](https://github.com/emanueleielo/compact-middleware) | Claude Code 的压缩引擎作为 LangChain 中间件。用于长时间运行的代理的多级上下文压缩。 | [`emanueleielo/compact-middleware`](https://github.com/emanueleielo/compact-middleware) |
| [langchain-collapse](https://github.com/johanity/langchain-collapse) | 预防性上下文管理。在连续工具调用组填满上下文窗口之前将其折叠。 | [`johanity/langchain-collapse`](https://github.com/johanity/langchain-collapse) |
| [langchain-task-steering](https://github.com/edvinhallvaxhiu/langchain-task-steering) | 用于有序任务管道的隐式状态机中间件，具有每任务工具作用域、动态提示注入和可组合的完成验证。 | [`edvinhallvaxhiu/langchain-task-steering`](https://github.com/edvinhallvaxhiu/langchain-task-steering) |
| [advisor-middleware](https://github.com/emanueleielo/advisor-middleware) | Claude Code 的顾问模式作为 LangChain 中间件。将快速执行器模型与强大的顾问模型配对，仅在关键决策时进行干预。 | [`emanueleielo/advisor-middleware`](https://github.com/emanueleielo/advisor-middleware) |
| [langchain-router](https://github.com/johanity/langchain-router) | 基于阶段的模型路由。将执行轮次路由到快速模型，保留主模型用于规划和恢复。 | [`johanity/langchain-router`](https://github.com/johanity/langchain-router) |
| [CopilotKit](/oss/langchain/frontend/integrations/copilotkit) | CopilotKit `CopilotKitMiddleware` 和 FastAPI 桥接用于 [深度代理](/oss/deepagents/overview), `create_agent` 图、AG-UI 以及 React 和运行时客户端。 | [`CopilotKit/CopilotKit`](https://github.com/CopilotKit/CopilotKit) |
| [eager-tools](https://github.com/cloudthinker-ai/eager-tools) | 通过在流式块关闭的瞬间分派每个工具调用来减少代理的墙钟时间，使工具执行与 LLM 生成重叠。 | [`cloudthinker-ai/eager-tools`](https://github.com/cloudthinker-ai/eager-tools) |
| [NoPII](https://github.com/Enigma-Vault/NoPII/tree/main/integrations/langchain-nopii-middleware) | 运行时 PII 标记化。检测出站提示中的个人数据，在请求到达 LLM 之前将其替换为确定性保管库标记，并在响应中恢复原始值。 | [`Enigma-Vault/NoPII`](https://github.com/Enigma-Vault/NoPII/tree/main/integrations/langchain-nopii-middleware) |
| [langgraph-state-machine](https://github.com/mahmoud661/langgraph-state-machine) | 用于 LangGraph React 代理的基于节的流程控制。将对话划分为具有作用域工具、提示、自动转换、分支和可选的每节 LLM 覆盖的离散阶段。 | [`mahmoud661/langgraph-state-machine`](https://github.com/mahmoud661/langgraph-state-machine) |
| [text2sql-framework](https://github.com/Text2SqlAgent/text2sql-framework) | 用递归工具使用取代 RAG — 代理使用一个 `execute_sql` 工具探索、编写、测试和自我修正。 | [`Text2SqlAgent/text2sql-framework`](https://github.com/Text2SqlAgent/text2sql-framework) |
