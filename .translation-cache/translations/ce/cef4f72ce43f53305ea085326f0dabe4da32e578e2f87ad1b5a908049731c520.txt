The **Context Hub** 为您的团队提供版本控制和环境感知的管理功能，用于管理您的代理在生产环境中使用的指令和工具。A _context_ 是一个版本化的代理指令和工具包，可以是 skill 或完整的代理，您可以在 LangSmith 中进行管理。

使用本指南创建您的第一个 context，查看其文件和历史记录，并将其推广到环境中，以便您的代理可以拉取它。

## 1. 打开 Context Hub

<Note>
如果您没有看到 **Context** 在左侧导航栏中，请验证 Context Hub 已为您的 workspace 启用，并且您具有所需的权限。
</Note>

在 [LangSmith UI](https://smith.langchain.com)中，选择 **Context** ，位于
左侧导航栏。Context Hub 列出了您 workspace 中的所有代理和 skill。

![Context Hub 主视图，列出现有的 Agent 和 Skill context 及其提交元数据。](/langsmith/images/context-hub-home.png)

## 2. 创建 context

1. 点击 **+ 创建** ，位于 Context Hub 左上角。
1. 选择 context 类型：
    - **Agent:** 一个包含 `AGENTS.md` 文件和工具的完整代理包。
    - **Skill:** 代理可使用的可重用能力，包括 `SKILL.md` file.

    <img src="/langsmith/images/context-hub-create-context.png" alt="The Create dropdown showing the Agent and Skill context type options." width="360" />

1. 填写名称和描述。对于 skill，描述是必填的。您也可以现在输入初始文件内容（`SKILL.md` 对于 skill， `AGENTS.md` 对于代理），或在创建后添加。点击 **创建代理** or **创建 Skill**。LangSmith 创建仓库并打开编辑。

## 3. 查看 context

点击 Context Hub 中的代理或 skill 以查看它。

![一个 Agent context，打开了 AGENTS.md 文件，显示了环境面板、提交历史和文件树。](/langsmith/images/context-hub-agent-view.png)

中间面板显示当前提交的 文件树，右侧面板预览选中的文件。点击中间面板中的文件以打开它，然后在右侧面板中编辑并保存更改以创建新提交。

每次保存的更改都会在左侧的 **提交历史** 面板中创建一个新的 **commit** ，这样您可以浏览、比较和回滚以前的版本而不会丢失
工作。
work.

## 4. 标记并推广提交

一旦提交准备好发布，将其推广到环境，以便下游
代理可以拉取它。

<Note>
Context Hub 目前支持两个用于升级的环境标签： `staging` 和 `production`.
</Note>

1. 选择目标提交后，点击 **升级** 位于右上角。
1. 选择目标环境：
    - **升级到生产环境：** 生产环境代理拉取的提交。
    - **升级到预发布环境：** 用于验证的预发布环境。

    <img src="/langsmith/images/context-hub-promote.png" alt="The Promote dropdown with options to promote a commit to Production or Staging." width="300" />

1. 环境标签（例如， `Production 7ca95573`）会移动到
   升级的提交上。请使用 **标签** 按钮在 **升级** 旁边，可附加
   人类可读的标签到任意提交，以便于参考。

通过环境标签解析上下文的代理运行时（例如， `:production`）现在拉取此升级的提交。
