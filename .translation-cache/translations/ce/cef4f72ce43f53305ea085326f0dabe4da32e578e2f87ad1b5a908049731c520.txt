**Context Hub** 为您的团队提供版本控制、环境感知的方式来管理代理在生产环境中使用的指令和工具。Context 是在 LangSmith 中管理的代理指令和工具的版本化包，可以是 skill 或完整的 agent。

使用本指南创建您的第一个 context，查看其文件和历史记录，并将其推广到环境以便您的代理可以拉取。

## 1. 打开 Context Hub

<Note>
如果您在左侧导航栏中看不到 **Context**，请确认 Context Hub 已为您的 workspace 启用，并且您拥有所需权限。
</Note>

在 [LangSmith UI](https://smith.langchain.com) 中，选择左侧导航栏中的 **Context**。Context Hub 列出 workspace 中的每个 agent 和 skill。

![Context Hub 首页，列出现有的 Agent 和 Skill context 及其提交元数据。](/langsmith/images/context-hub-home.png)

## 2. 创建 context

1. 点击 Context Hub 左上角的 **+ Create**。
1. 选择 context 类型：
    - **Agent：** 包含 `AGENTS.md` 文件和工具的完整 agent 包。
    - **Skill：** agent 可以使用的可重用能力，包括 `SKILL.md` 文件。

    <img src="/langsmith/images/context-hub-create-context.png" alt="Create 下拉菜单显示 Agent 和 Skill context 类型选项。" width="360" />

1. 填写名称和描述。对于 skill，必须填写描述。您现在也可以输入初始文件内容（skill 的 `SKILL.md`、agent 的 `AGENTS.md`），或者在创建后添加。点击 **Create Agent** 或 **Create Skill**。LangSmith 创建 repo 并打开编辑。

## 3. 查看 context

点击 Context Hub 中的 agent 或 skill 进行查看。

![Agent context，展示了打开的 AGENTS.md 文件、环境面板、提交历史和文件树。](/langsmith/images/context-hub-agent-view.png)

中间面板显示当前提交的 文件树，右侧面板预览所选文件。点击中间面板中的文件将其打开，然后在右侧面板中编辑并保存以创建新提交。

每次保存的更改都会在左侧的 **Commit History** 面板中创建一个新的 **commit**，因此您可以浏览、比较和回退之前的版本，而不会丢失任何工作。

## 4. 标记并推广提交

一旦提交准备好发布，请将其推广到环境，以便下游代理可以拉取。

<Note>
Context Hub 目前支持两个用于推广的环境标签：`staging` 和 `production`。
</Note>

1. 选择目标提交后，点击右上角的 **Promote**。
1. 选择目标环境：
    - **Promote to Production：** 生产代理拉取的提交。
    - **Promote to Staging：** 用于验证的预生产环境。

    <img src="/langsmith/images/context-hub-promote.png" alt="Promote 下拉菜单，包含将提交推广到 Production 或 Staging 的选项。" width="300" />

1. 环境标签（例如 `Production 7ca95573`）会移动到被推广的提交。使用 **Promote** 旁边的 **Tag** 按钮可以为任何提交附加易读的标签，方便参考。

通过环境标签解析 context 的代理运行时（例如 `:production`）现在会拉取此推广的提交。
