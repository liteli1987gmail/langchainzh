消息队列允许用户快速连续发送多条消息，而无需等待代理完成当前消息的处理。每条消息立即被接受，排队到活动线程中，并按顺序处理，让你可以完全了解和控制待处理的工作。

## 为什么使用消息队列？

在典型的聊天界面中，用户必须等待代理完成回复后才能发送另一条消息。这在以下几种场景中会造成障碍：

- **批量问题**：用户希望一次提出五个相关问题，而不是等待每个答案
- **后续追问链**：在代理仍在工作时提交澄清或补充信息
- **自动化测试序列**：通过编程方式发送一系列提示来验证代理行为
- **数据录入工作流**：逐个输入结构化数据进行提交处理

消息队列通过立即接受所有提交并按顺序处理来解决这个问题。

这是一个代理用户体验原语，而不是装饰性的聊天功能。SDK 将
队列作为流控制器的一部分进行跟踪，因此你的界面可以显示待处理的
工作、取消过时的请求，并在当前运行
continues.

## 工作原理

传入 `multitaskStrategy: "enqueue"` 当你希望提交内容在
当前正在运行的请求后面等待时。代理处理期间，排队的提交内容
会被添加到活动线程的队列中。当前运行完成后，
下一条排队的消息会自动发送。

使用适用于你框架的配套队列助手读取队列状态：

| 属性 | 类型 | 描述 |
|---|---|---|
| `queue.entries` | `SubmissionQueueEntry[]` | 所有待处理队列条目的数组 |
| `queue.size` | `number` | 当前队列中的条目数量 |
| `queue.cancel(id)` | `(id: string) => Promise<void>` | 按 ID 取消特定的排队条目 |
| `queue.clear()` | `() => Promise<void>` | 取消所有排队的条目 |

每个 @[SubmissionQueueEntry] 对象包含：

| 字段 | 类型 | 描述 |
|---|---|---|
| `id` | `string` | 此队列条目的唯一标识符 |
| `values` | `object` | 提交的输入值（包括消息）|
| `options` | `object` | 随提交传递的任何其他选项 |
| `createdAt` | `string` | 条目创建时间的 ISO 时间戳 |

## 设置 `useStream`

将 @[`useStream`] 连接到你的代理，然后将其与提交队列
助手配对以适用于你的框架。调用 `stream.submit()` 在运行
进行中时发送消息；在应 `multitaskStrategy: "enqueue"` 在活动请求后面等待的提交上传递
。读取 `queue.entries` 和 `queue.size` 以渲染
待处理的工作，并使用 `queue.cancel()` or `queue.clear()` 移除处理开始前的项目
它们开始处理。

<CodeGroup>

```tsx React
import { useStream, useSubmissionQueue } from "@langchain/react";

function Chat() {
  const stream = useStream<typeof myAgent>({
    apiUrl: "http://localhost:2024",
    assistantId: "simple_agent",
  });
  const queue = useSubmissionQueue(stream);

  const handleSubmit = (text: string) => {
    stream.submit({
      messages: [{ type: "human", content: text }],
    });
  };

  const pendingCount = queue.size;
  const entries = queue.entries;

  return (
    <div>
      <MessageList messages={stream.messages} />
      {pendingCount > 0 && <QueueList entries={entries} queue={queue} />}
      <ChatInput onSubmit={handleSubmit} />
    </div>
  );
}
```

## 显示队列

构建一个 `QueueList` 组件来显示每条待处理消息及取消按钮。这让用户可以查看等待中的内容，并能够移除不再需要的项目。

```tsx
function QueueList({ entries, queue }) {
  return (
    <div className="queue-panel">
      <div className="queue-header">
        <span>Queued messages ({entries.length})</span>
        <button onClick={() => queue.clear()}>Clear all</button>
      </div>
      <ul className="queue-entries">
        {entries.map((entry) => {
          const text = entry.values?.messages?.at(-1)?.content ?? "Pending...";
          return (
            <li key={entry.id} className="queue-entry">
              <span className="queue-text">{text}</span>
              <span className="queue-time">
                {new Date(entry.createdAt).toLocaleTimeString()}
              </span>
              <button
                className="queue-cancel"
                onClick={() => queue.cancel(entry.id)}
              >
                Cancel
              </button>
            </li>
          );
        })}
      </ul>
    </div>
  );
}
```

<Tip>
显示每个队列消息的前几个字符作为预览，让用户能够快速识别要取消的项目，而无需阅读完整消息。
</Tip>

## 取消队列中的消息

你有两个级别的取消操作：

### 取消单个条目

通过ID从队列中移除特定消息。代理将跳过它并移动到下一个条目。

```ts
await queue.cancel(entryId);
```

### 清空整个队列

一次移除所有待处理消息。当用户改变上下文或想要重新开始时很有用。

```ts
await queue.clear();
```

<Note>
取消队列条目仅影响 **尚未开始
处理的消息。**如果代理已经在处理某条消息，从队列中取消它
不会有任何效果。使用 `stream.stop()` 来中断当前运行。
</Note>

## 使用 `onCreated`

链接后续提交 `onCreated` 回调在新运行创建时触发，让你能够以编程方式提交后续消息的钩子。这对于构建多步骤工作流很有用，其中下一步问题依赖于前一个提交被接受。

```ts
stream.submit(
  { messages: [{ type: "human", content: "What is quantum computing?" }] },
  {
    onCreated(run) {
      console.log("Run created:", run.runId);
      // Chain a follow-up
      stream.submit({
        messages: [{ type: "human", content: "Give me a simple analogy." }],
      });
    },
  }
);
```

这种模式自然会填满队列。第一条消息
立即开始处理，后续消息则排在后面。

## 开始新线程

当用户想要开始新的对话时，更新响应式 `threadId`
你传递给流的内容。传递 `null` 会清除当前的线程绑定；
下一次提交将创建新线程。

<CodeGroup>

```tsx React
function NewThreadButton() {
  const [threadId, setThreadId] = useState<string | null>(null);
  const stream = useStream<typeof myAgent>({ threadId, onThreadId: setThreadId });

  return (
    <button onClick={() => setThreadId(null)}>
      New conversation
    </button>
  );
}
```

```vue Vue
<script setup lang="ts">
const threadId = ref<string | null>(null);
const stream = useStream<typeof myAgent>({
  threadId,
  onThreadId: (id) => (threadId.value = id),
});
</script>

<template>
  <button @click="threadId = null">New conversation</button>
</template>
```

```svelte Svelte
<script lang="ts">
  let threadId = $state<string | null>(null);
  const stream = useStream<typeof myAgent>({
    threadId: () => threadId,
    onThreadId: (id) => (threadId = id),
  });
</script>

<button onclick={() => (threadId = null)}>New conversation</button>
```

## 最佳实践

- **限制队列大小**：虽然客户端对队列大小没有硬性限制，
但请注意非常大的队列可能会降低用户体验。考虑在队列超过合理阈值时
显示警告（例如，超过10
个项目）。
- **显示队列位置**：为每个队列项目编号，让用户知道处理顺序。
- **保持输入焦点**：提交后保持输入字段焦点，让用户能够立即输入下一条消息。
- **添加过渡动画**：当项目开始处理时，流畅地将它们从队列面板移动到消息列表中。
- **优雅处理错误**：如果队列消息失败，在不阻塞后续队列条目的情况下显示错误。
- **对快速提交进行防抖**：对于自动化或编程式提交，在消息之间添加小延迟，以避免服务器过载。
