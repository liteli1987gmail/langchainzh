消息队列允许用户快速连续发送多条消息，而无需等待代理完成当前消息的处理。每条消息都会被立即接收，加入活跃线程的队列中，并按顺序处理，让你可以完全了解和掌控待处理的工作。

## 为什么使用消息队列？

在典型的聊天界面中，用户必须等待智能体完成回复后才能发送下一条消息。这种模式在以下场景中会造成不便：

- **批量问题**：用户希望一次性提出五个相关问题，而不是逐个等待答案
- **跟进链**：在智能体仍在工作时提交澄清或补充信息
- **自动化测试序列**：以编程方式发送一系列提示来验证智能体行为
- **数据录入工作流**：逐个输入结构化数据进行处理

消息队列通过立即接受所有提交并按顺序处理它们来解决这个问题。

这是智能体用户体验的基础功能，而非表面上的聊天特性。SDK 将队列作为流控制器的一部分进行跟踪，因此你的 UI 可以显示待处理的工作、取消过时的请求，并在当前运行继续进行时保持输入框处于活动状态。

## 工作原理

当你想让某个提交等待当前正在运行的请求时，传递 `multitaskStrategy: "enqueue"`。在智能体处理过程中，排队的提交会被添加到活跃线程的队列中。当前运行完成后，下一条排队的消息会自动发送。

使用框架对应的队列辅助工具读取队列状态：

| 属性 | 类型 | 描述 |
|---|---|---|
| `queue.entries` | `SubmissionQueueEntry[]` | 所有待处理队列条目的数组 |
| `queue.size` | `number` | 当前队列中的条目数量 |
| `queue.cancel(id)` | `(id: string) => Promise<void>` | 通过 ID 取消特定的队列条目 |
| `queue.clear()` | `() => Promise<void>` | 取消所有队列条目 |

每个 @[SubmissionQueueEntry] 对象包含：

| 字段 | 类型 | 描述 |
|---|---|---|
| `id` | `string` | 该队列条目的唯一标识符 |
| `values` | `object` | 提交的输入值（包括消息） |
| `options` | `object` | 随提交传递的任何额外选项 |
| `createdAt` | `string` | 条目创建时间的 ISO 时间戳 |

## 设置 `useStream`

将 @[`useStream`] 连接到你的智能体，然后将其与框架对应的提交队列辅助工具配对。在运行进行时调用 `stream.submit()` 发送消息；对于需要等待当前请求的提交，传递 `multitaskStrategy: "enqueue"`。读取 `queue.entries` 和 `queue.size` 来渲染待处理的工作，并使用 `queue.cancel()` 或 `queue.clear()` 在处理开始前移除条目。

```tsx React
import { useStream, useSubmissionQueue } from "@langchain/react";

function Chat() {
  const stream = useStream<typeof myAgent>({
    apiUrl: "http://localhost:2024",
    assistantId: "simple_agent",
  });
  const queue = useSubmissionQueue(stream);

  const handleSubmit = (text: string) => {
    stream.submit({
      messages: [{ type: "human", content: text }],
    });
  };

  const pendingCount = queue.size;
  const entries = queue.entries;

  return (
    <div>
      <MessageList messages={stream.messages} />
      {pendingCount > 0 && <QueueList entries={entries} queue={queue} />}
      <ChatInput onSubmit={handleSubmit} />
    </div>
  );
}
```

## 显示队列

构建一个 `QueueList` 组件，用于显示每条待处理消息并提供取消按钮。这让用户能够看到等待中的内容，并可以移除不再需要的项目。

```tsx
function QueueList({ entries, queue }) {
  return (
    <div className="queue-panel">
      <div className="queue-header">
        <span>Queued messages ({entries.length})</span>
        <button onClick={() => queue.clear()}>Clear all</button>
      </div>
      <ul className="queue-entries">
        {entries.map((entry) => {
          const text = entry.values?.messages?.at(-1)?.content ?? "Pending...";
          return (
            <li key={entry.id} className="queue-entry">
              <span className="queue-text">{text}</span>
              <span className="queue-time">
                {new Date(entry.createdAt).toLocaleTimeString()}
              </span>
              <button
                className="queue-cancel"
                onClick={() => queue.cancel(entry.id)}
              >
                Cancel
              </button>
            </li>
          );
        })}
      </ul>
    </div>
  );
}
```

<Tip>
显示每个排队消息的前几个字符作为预览，以便用户能够快速识别要取消的项目，而无需阅读完整消息。
</Tip>

## 取消排队的消息

您有两个级别的取消操作：

### 取消单个条目

通过 ID 从队列中移除特定消息。代理将跳过该消息并继续处理下一条记录。

```ts
await queue.cancel(entryId);
```

### 清除整个队列

一次移除所有待处理的消息。当用户切换上下文或想要重新开始时很有用。

```ts
await queue.clear();
```

<Note>
取消队列条目仅影响**尚未开始处理**的消息。如果代理正在处理某条消息，从队列中取消它将不会产生任何效果。请使用 `stream.stop()` 来中断当前运行。
</Note>

## 使用 `onCreated` 链式提交后续消息

`onCreated` 回调在新运行时触发，让您可以以编程方式提交后续消息。这对于构建多步骤工作流非常有用，在这些工作流中，下一个问题取决于先前的提交是否被接受。

```ts
stream.submit(
  { messages: [{ type: "human", content: "What is quantum computing?" }] },
  {
    onCreated(run) {
      console.log("Run created:", run.runId);
      // Chain a follow-up
      stream.submit({
        messages: [{ type: "human", content: "Give me a simple analogy." }],
      });
    },
  }
);
```

这种模式自然地填满队列。第一条消息立即开始处理，后续消息排在其后。

## 开始一个新线程

当用户想要开始新的对话时，更新传入流中的响应式 `threadId`。传入 `null` 会清除当前线程绑定；下一次提交将创建一个新线程。

</CodeGroup>

```tsx React
function NewThreadButton() {
  const [threadId, setThreadId] = useState<string | null>(null);
  const stream = useStream<typeof myAgent>({ threadId, onThreadId: setThreadId });

  return (
    <button onClick={() => setThreadId(null)}>
      New conversation
    </button>
  );
}
```

```vue Vue
<script setup lang="ts">
const threadId = ref<string | null>(null);
const stream = useStream<typeof myAgent>({
  threadId,
  onThreadId: (id) => (threadId.value = id),
});
</script>

<template>
  <button @click="threadId = null">New conversation</button>
</template>
```

## 最佳实践

- **限制队列大小**：虽然客户端对队列大小没有硬性限制，但请注意，过大的队列可能会降低用户体验。建议当队列超过合理阈值时显示警告（例如，超过 10 项）。
- **显示队列位置**：为每个排队的项目编号，以便用户了解处理顺序。
- **保持输入焦点**：提交后保持输入框聚焦，以便用户可以立即输入下一条消息。
- **添加过渡动画**：当项目开始处理时，平滑地将项目从队列面板移动到消息列表。
- **优雅地处理错误**：如果排队的消息失败，显示错误但不要阻塞后续的队列条目。
- **对快速提交进行防抖处理**：对于自动化或编程方式的提交，请在消息之间添加小延迟，以避免给服务器带来过大压力。
