围绕您的目标构建基础架构。 `create_deep_agent` 为您提供生产就绪的基础架构：连接到您的数据、塑造其行为，并添加您的用例所需的功能。

:::python

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    system_prompt="You are a helpful assistant.",
    tools=[search, fetch_url],
    memory=["./AGENTS.md"],
    skills=["./skills/"],
)
```

```python
import asyncio
from langchain_mcp_adapters.client import MultiServerMCPClient
from deepagents import create_deep_agent

async def main():
    async with MultiServerMCPClient(
        {
            "my_server": {
                "transport": "http",
                "url": "http://localhost:8000/mcp",
            }
        }
    ) as client:
        tools = await client.get_tools()

        agent = create_deep_agent(
            model="openai:gpt-5.5",
            tools=tools,
        )

        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": "Use the MCP server to help me."}]},
            config={"configurable": {"thread_id": "1"}},
        )

asyncio.run(main())
```

## 系统提示词

Deep Agents 附带内置系统提示词。深度代理的价值来自于 SDK 在模型之上提供的编排层——规划、虚拟文件系统工具和子代理——模型需要知道这些工具的存在以及何时使用它们。内置提示词教授代理如何使用这些脚手架，这样您就不必为每个项目重新推导它；通过 [配置文件](/oss/deepagents/profiles#harness-profiles) 或您自己的 `system_prompt=` 来调整它，而不是逐字复制。

当中间件添加特殊工具（如文件系统工具）时，它会将这些工具追加到系统提示词中。

每个深度代理还应包含一个针对其特定用例的自定义系统提示词：

### 提示词组装

Deep Agents 从最多四个命名部分构建系统提示词，以便调用者提供的指令、SDK 的内置代理指导以及任何特定模型的 [配置文件](/oss/deepagents/profiles) 覆盖可以与可预测的优先级共存。如果没有这种分层，为 Claude 调优的配置文件后缀（例如）可能会覆盖或被您的 `system_prompt=` 根据调用顺序的参数；命名槽使排序变得明确和稳定。

实际上，大多数调用者只会遇到两个槽： `USER` （你的 `system_prompt=`）和 `BASE` （SDK 默认值）。使用内置配置文件选择模型（目前是 Anthropic 或 OpenAI）会添加一个 `SUFFIX`。完整的四部分组装主要与你创作自定义 `HarnessProfile` 或调试为什么配置文件的文本出现在特定位置相关。

四个命名部分（每个都可能是空的）：

| Name     | Source                                            | Notes                                             |
| -------- | ------------------------------------------------- | ------------------------------------------------- |
| `USER`   | `system_prompt=` 参数到 `create_deep_agent`  | `str` or `SystemMessage`；未设置时省略。     |
| `BASE`   | SDK 默认值（`BASE_AGENT_PROMPT`）             | 除非被配置文件的 `CUSTOM`. |
| `CUSTOM` | [`HarnessProfile.base_system_prompt`](/oss/deepagents/profiles#harness-profiles) | 替换 `BASE` 当匹配的配置文件中设置了它时直接替换。 |
| `SUFFIX` | [`HarnessProfile.system_prompt_suffix`](/oss/deepagents/profiles#harness-profiles) | 当匹配的配置文件中设置了它时追加到末尾。   |

顺序始终是 **`USER` -> (`BASE` or `CUSTOM`) -> `SUFFIX`**，用空行连接（`\n\n`）。两个不变式如下：

1. **`USER` 始终位于前面。** The caller's text precedes any SDK or profile content, so persona/instructions take precedence regardless of which model is selected.
2. **`SUFFIX` 始终位于末尾。** 配置文件后缀最接近对话历史，这是模型调整指导最可靠落地的位置。

组装形状（✓ = 字段已设置，- = 字段未设置）：

| `system_prompt=` | 配置文件 `base_system_prompt` (`CUSTOM`）| 配置文件 `system_prompt_suffix` (`SUFFIX`）| 最终组装的系统提示词 |
| ---------------- | :-------------------------------------: | :---------------------------------------: | ----------------------------- |
| `None`           |                    -                    |                     -                     | `BASE`                        |
| `None`           |                    -                    |                     ✓                     | `BASE` + `SUFFIX`             |
| `None`           |                    ✓                    |                     -                     | `CUSTOM`                      |
| `None`           |                    ✓                    |                     ✓                     | `CUSTOM` + `SUFFIX`           |
| `str`            |                    -                    |                     -                     | `USER` + `BASE`               |
| `str`            |                    -                    |                     ✓                     | `USER` + `BASE` + `SUFFIX`    |
| `str`            |                    ✓                    |                     -                     | `USER` + `CUSTOM`             |
| `str`            |                    ✓                    |                     ✓                     | `USER` + `CUSTOM` + `SUFFIX`  |

工作示例——内置配置文件（Anthropic、OpenAI）仅提供 `system_prompt_suffix`，因此典型调用位于 `str` + `-` + `✓` row:

```python
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    system_prompt="You are a customer-support agent for ACME Corp.",
)
# Final = USER + BASE + SUFFIX
#       = "You are a customer-support agent for ACME Corp."
#         + "\n\n"
#         + BASE_AGENT_PROMPT
#         + "\n\n"
#         + <Claude-specific guidance>
```

<Note>
    传递 `SystemMessage` （而不是字符串）会触发不同的连接路径：右侧组装（`BASE`-or-`CUSTOM` 加上任何 `SUFFIX`）作为额外的文本内容块追加到消息现有的 `content_blocks`。相同的逻辑排序适用（调用方块优先），并且任何 `cache_control` 调用方块上的标记会被保留——这对于放置明确的 Anthropic prompt-cache 断点很有用。
</Note>

<AccordionGroup>
    <Accordion title="Subagent prompts">
         [提示词组装](#prompt-assembly) 覆盖规则也适用于声明式 [子代理](/oss/deepagents/subagents)：每个子代理根据 **自己的模型重新运行配置文件解析**，然后应用已解析配置文件的 `base_system_prompt` / `system_prompt_suffix` 到其创作的 `system_prompt`。子代理的 `system_prompt` 扮演 `BASE` 角色； `CUSTOM` 和 `SUFFIX` 来自与子代理模型匹配的配置文件（可能与主代理的配置文件不同）。

        | `spec["system_prompt"]` | 配置文件 `base_system_prompt` (`CUSTOM`）| 配置文件 `system_prompt_suffix` (`SUFFIX`）| 最终子代理系统提示词 |
        | ----------------------- | :-------------------------------------: | :---------------------------------------: | ---------------------------- |
        | authored                |                    -                    |                     -                     | authored                     |
        | authored                |                    -                    |                     ✓                     | authored + `SUFFIX`          |
        | authored                |                    ✓                    |                     -                     | `CUSTOM`                     |
        | authored                |                    ✓                    |                     ✓                     | `CUSTOM` + `SUFFIX`          |

        没有 `USER` 子代理的片段。规范中的 authored `system_prompt` 是最接近的类比，并保留在 `BASE` 槽位中。仅提供 `system_prompt_suffix` (the common case for built-in Anthropic / OpenAI profiles) just appends to whatever the subagent author wrote. A profile that sets `base_system_prompt` 的 profile 将 *直接替换* authored prompt。
    </Accordion>
    <Accordion title="General-purpose subagent prompt">
        自动添加的 [通用子代理](/oss/deepagents/subagents#the-general-purpose-subagent) 遵循 [prompt assembly](#prompt-assembly) 覆盖规则，并额外增加一层：GP base prompt 按以下顺序解析 **`general_purpose_subagent.system_prompt` （如果已设置）-> `HarnessProfile.base_system_prompt` （如果已设置）-> SDK 通用默认值**。无论如何，profile 后缀都会叠加在上层。

        两个覆盖字段都可以携带 base-prompt 替换，但它们不可互换。 `general_purpose_subagent.system_prompt` 是通用场景的专用配置； `base_system_prompt` 是一个全局覆盖，主要针对主代理。当两者都设置时， **通用场景的专用意图在通用子代理中优先** 这样用户在同时调整两个字段时，永远不会看到 GP 覆盖被静默丢弃：

        ```python
        register_harness_profile(
            "anthropic",
            HarnessProfile(
                base_system_prompt="You are ACME's support orchestrator.",  # main agent
                general_purpose_subagent=GeneralPurposeSubagentProfile(
                    system_prompt="You are a research subagent. Cite sources.",  # GP subagent
                ),
                system_prompt_suffix="Always think step by step.",
            ),
        )
        ```

        | 栈 | 最终系统 prompt |
        | ----- | ------------------- |
        | 主代理 | `"You are ACME's support orchestrator." + SUFFIX` |
        | GP 子代理 | `"You are a research subagent. Cite sources." + SUFFIX` |

        If `general_purpose_subagent.system_prompt` 未设置时，GP 子代理回退到 `base_system_prompt` （如果已设置），最后回退到 SDK 通用默认值。
    </Accordion>
</AccordionGroup>

## 中间件

Deep Agents 支持任何 [中间件](/oss/langchain/middleware/overview)，包括下面列出的内置中间件、来自 LangChain 的预构建中间件、提供商特定中间件，以及您自己编写的自定义中间件。

:::python
将中间件传递给 `middleware` 的参数 `create_deep_agent`。自定义中间件追加在`PatchToolCallsMiddleware`之后的 [默认栈中](#default-stack-main-agent).
:::

:::js
将中间件传递给 `middleware` 的参数 `createDeepAgent`。自定义中间件追加在`PatchToolCallsMiddleware`之后的 [默认栈中](#default-stack-main-agent).
:::

默认情况下，Deep Agents 可访问以下中间件：

### 默认栈（主代理）

从第一个到最后一个：

:::python

1. @[`TodoListMiddleware`]：跟踪和管理待办列表，用于组织代理任务和工作。
2. @[`SkillsMiddleware`]：仅在您传递 `skills`时注入 **紧接着** 待办事项中间件之后，并且 **在...之前** 文件系统中间件，以便技能元数据在文件工具运行之前可用。
3. @[`FilesystemMiddleware`]: 处理文件系统操作，如读取、写入和导航目录。当您传递 `permissions`，文件系统权限强制执行包含在此处，以便它可以评估代理可能调用的每个工具。
4. @[`SubAgentMiddleware`]: 生成并协调子代理，用于将任务委托给专业代理。
5. @[`SummarizationMiddleware`]: 压缩消息历史以保持在上下文限制范围内，当对话变长时（通过 @[create_摘要_中间件]）。
6. @[`PatchToolCallsMiddleware`]: 修复消息历史中悬空的工具调用，当运行在中断后恢复或收到格式错误的工具调用参数时。运行 **之前** 下面的 Anthropic 提示缓存和尾部堆栈。
7. @[`AsyncSubAgentMiddleware`]: 仅在您配置异步子代理时。
8. **您的中间件参数**: 您作为...传递的可选中间件 `middleware` 参数在此处追加（在 Patch 之后，尾部堆栈之前）。
9. **测试工具配置文件补充**: 来自已解析模型配置文件的提供商特定中间件（如果有）。
10. **排除工具过滤**: 当测试工具配置文件列出排除的工具时，中间件会从代理中移除这些工具。
11. @[`AnthropicPromptCachingMiddleware`]: 始终注册；对于非 Anthropic 模型，它不执行任何操作（`unsupported_model_behavior="ignore"`）。运行 **之后** Patch 和您的中间件之后，以便缓存的前缀与实际发送到模型的内容相匹配。
12. @[`MemoryMiddleware`]: 仅在您传递 `memory`.

    <Note>

## 配置文件

A [代理配置文件](/oss/deepagents/profiles#harness-profiles) 是一组可重用的按模型配置，在匹配模型被选中时自动应用。配置文件是实现以下行为的正确工具： `create_deep_agent` 行为跟随模型而非调用位置——例如，针对 Claude 指令风格调优的系统提示后缀、为 GPT 重写的工具描述，或仅在特定提供商有意义的额外中间件。
