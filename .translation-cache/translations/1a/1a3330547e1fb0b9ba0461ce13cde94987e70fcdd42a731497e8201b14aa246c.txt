托管深度代理 API 用于创建、更新、连接和调用[托管深度代理](/langsmith/managed-deep-agents-overview)。请使用适用于 Python、TypeScript 和 React 应用的[托管深度代理 SDK](/langsmith/managed-deep-agents-sdk)。当您需要直接控制请求负载时，请使用 REST API。如需了解推荐的端到端工作流程，请参阅[快速入门](/langsmith/managed-deep-agents-quickstart)。

<Note>
托管深度代理处于**私人测试版**，仅在美国区域的 [LangSmith Cloud](/langsmith/cloud) 上可用。[加入候补名单](https://www.langchain.com/langsmith-managed-deep-agents-waitlist)以请求访问权限。
</Note>

## 设置请求默认值

API 使用 `/v1/deepagents`：

```bash
export LANGSMITH_API_KEY="<LANGSMITH_API_KEY>"
export LANGSMITH_API_URL="https://api.smith.langchain.com"
export DEEPAGENTS_BASE_URL="$LANGSMITH_API_URL/v1/deepagents"
```

请求需要 `X-Api-Key` 请求头：

```txt
X-Api-Key: <LANGSMITH_API_KEY>
```

例如，使用上面设置的 base URL 和 header 列出代理：

```bash
curl "$DEEPAGENTS_BASE_URL/agents" \
  -H "X-Api-Key: $LANGSMITH_API_KEY"
```

缺少 `X-Api-Key` 头返回 `401` 和 `{"error": "Unauthorized"}`。无效或缺少工作区访问权限的密钥返回 `403` 和 `{"error": "Forbidden"}`。这些认证响应使用扁平的 `{"error": "..."}` 响应体，与其他 `4xx` 响应返回的结构化错误体不同。具有有效密钥但角色缺少所需权限也会返回 `403`，但响应体为纯文本，注明缺失的权限（例如 `missing permission mcp-servers:create`），SDK 会在 `error.body` 中暴露该信息。

## 了解资源组

| 资源组 | 用途 |
| --- | --- |
| Agents | 创建和管理托管深度代理资源，包括运行时和后端配置。 |
| Threads | 为托管深度代理创建和管理持久化线程状态。 |
| Runs | 启动和流式传输托管深度代理在线程上的运行。 |
| MCP servers | 注册 MCP 服务器并存储代理工具引用的凭据。 |
| MCP tools | 列出已注册 MCP 服务器公开的工具，以便客户端构建 `tools.json` 条目。 |
| Auth sessions | 为 OAuth MCP 服务器启动和轮询用户 OAuth 会话。 |

托管深度代理不是 LangSmith Deployments。创建托管深度代理会创建托管深度代理资源、单独的 LangSmith 跟踪项目和 Context Hub 代理仓库用于托管文件树。

## 配置沙箱

Create-agent 和 update-agent 负载可以包含一个 `backend` 对象。当代理不需要特定于沙箱的后端行为时，请使用 `state`：

```json
{
  "backend": {
    "type": "state"
  }
}
```

当代理需要 [LangSmith sandbox](/langsmith/sandboxes) 用于代码执行、文件系统操作或长时间运行的任务时，请使用 `sandbox`。Sandbox 后端设置位于 `backend.sandbox_config`，仅在 `backend.type` 设置为 `sandbox` 时有效：

```json
{
  "backend": {
    "type": "sandbox",
    "sandbox_config": {
      "scope": "thread",
      "policy_ids": ["policy-id"],
      "idle_ttl_seconds": 900,
      "delete_after_stop_seconds": 300
    }
  }
}
```

`sandbox` 对象接受以下字段：

| 字段 | 说明 |
| --- | --- |
| `scope` | 沙箱作用域。使用 `thread` 为每个线程分配一个沙箱，或使用 `agent` 让一个沙箱由代理共享。 |
| `policy_ids` | 要应用的沙箱策略 ID。 |
| `idle_ttl_seconds` | 沙箱停止前的空闲超时时间，以秒为单位。 |
| `delete_after_stop_seconds` | 沙箱停止后删除前的延迟时间，以秒为单位。 |

有关后端指导，请参阅[部署代理](/langsmith/managed-deep-agents-deploy#choose-a-backend)。有关独立沙箱概念，请参阅 [LangSmith 沙箱概述](/langsmith/sandboxes)。

## 对代理列表进行分页

[`GET /v1/deepagents/agents`](/langsmith/managed-deep-agents-api/agents/list-agents) 使用游标分页。传递 `page_size`（默认为 `20`，最大值为 `100`）以及前一个请求返回的不透明 `cursor`。响应将结果包装在 `items` 数组中，并附带 `next_cursor` 字段，在最后一页时该字段为 `null`：

```json
{
  "items": [],
  "next_cursor": null
}
```

该端点还接受 `name` 以按名称子字符串进行筛选，`sort_by`（`created_at`、`updated_at` 或 `name`，默认为 `updated_at`），以及 `sort_order`（`asc` 或 `desc`，默认为 `desc`）。

## 了解 API 稳定性

路由在 `/v1/` 版本化，但接口处于私人测试阶段，在正式发布前可能会以不兼容的方式进行更改。详见 [API 稳定性](/langsmith/managed-deep-agents-overview#api-stability) 了解 breaking-change 通知。

该 API 并非每个 LangSmith Deployment 端点的镜像。集成、触发器、技能、沙箱、身份验证提供者和身份验证令牌等端点组不被镜像。
