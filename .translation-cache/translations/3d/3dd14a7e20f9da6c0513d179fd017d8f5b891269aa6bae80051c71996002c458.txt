[AgentQL](https://www.agentql.com/) 的文档加载器使用 [AgentQL 查询](https://docs.agentql.com/agentql-query) 从任何网页中提取结构化数据。AgentQL 可跨多种语言和网页使用，不会随着时间和变化而失效。

## 概述

`AgentQLLoader` 需要以下两个参数：

- `url`：要从中提取数据的网页 URL。
- `query`：要执行的 AgentQL 查询。了解有关[如何在文档中编写 AgentQL 查询的更多信息](https://docs.agentql.com/agentql-query)，或在 [AgentQL Playground](https://dev.agentql.com/playground) 中进行测试。

以下参数的设置是可选的：

- `api_key`：您的 AgentQL API 密钥，来自 [dev.agentql.com](https://dev.agentql.com)。**`可选`。**
- `timeout`：请求超时前的等待秒数。**默认为 `900`。**
- `is_stealth_mode_enabled`：是否启用实验性的反机器人规避策略。此功能可能不会在所有网站上始终有效。启用此模式后，数据提取可能需要更长时间才能完成。**默认为 `False`。**
- `wait_for`：提取数据前等待页面加载的秒数。**默认为 `0`。**
- `is_scroll_to_bottom_enabled`：是否在提取数据前滚动到页面底部。**默认为 `False`。**
- `mode`：`"standard"` 使用深度数据分析，而 `"fast"` 以牺牲一定的分析深度来换取速度，适用于大多数用例。[在此指南中了解更多关于模式的信息。](https://docs.agentql.com/accuracy/standard-mode) **默认为 `"fast"`。**
- `is_screenshot_enabled`：是否在提取数据前截取屏幕截图。作为 Base64 字符串返回到 'metadata' 中。**默认为 `False`。**

AgentQLLoader 是使用 AgentQL 的 [REST API](https://docs.agentql.com/rest-api/api-reference) 实现的。

## 设置

若要使用 AgentQL 文档加载器，您需要配置 `AGENTQL_API_KEY` 环境变量，或使用 `api_key` 参数。您可以从我们的 [开发者门户](https://dev.agentql.com) 获取 API 密钥。

### 安装

安装 **langchain-agentql**。

```python
pip install -qU langchain-agentql
```

### 设置凭据

```python
import os

os.environ["AGENTQL_API_KEY"] = "YOUR_AGENTQL_API_KEY"
```

## 初始化

接下来实例化您的模型对象：

```python
from langchain_agentql.document_loaders import AgentQLLoader

loader = AgentQLLoader(
    url="https://www.agentql.com/blog",
    query="""
    {
        posts[] {
            title
            url
            date
            author
        }
    }
    """,
    is_scroll_to_bottom_enabled=True,
)
```

## 加载

docs = loader.load()

## 延迟加载

`AgentQLLoader` 当前仅加载一个 @[`Document`]。因此，`load()` 和 `lazy_load()` 的行为相同：

```python
pages = [doc for doc in loader.lazy_load()]
pages
```
