本指南介绍如何将 [Google Agent Development Kit (ADK)](https://github.com/google/adk-python) 代理部署到 [LangSmith Agent Server](/langsmith/agent-server) 上，使用 [`deployments-wrap-sdk`](https://pypi.org/project/deployments-wrap-sdk/) 包。

`deployments-wrap-sdk` 提供了一个轻量级包装器，可将配置好的 ADK `Runner` 转换为与 LangGraph 兼容的图，从而无需编写 [Functional API](/oss/langgraph/functional-api) 胶水代码即可部署 ADK 代理。该包装器：

- 将 ADK 会话桥接到 Agent Server 的[检查点持久化](/langsmith/agent-server#persistence)，使会话状态在重启后保留并可在不同运行间恢复。
- 通过 LangGraph 的流式管道转发 ADK token 事件，使部分 token 能在 [`stream_mode="messages"`](/langsmith/streaming) 和 [LangSmith Studio](/langsmith/studio) 中显示。
- 当设置 `LANGSMITH_TRACING` 时，自动为 ADK 启用 [LangSmith 追踪](/langsmith/trace-with-google-adk)。

## 前置条件

- Python 3.11+
- 用于本地开发和部署的 [LangGraph CLI](/langsmith/cli)
- LangSmith API 密钥，请参阅[创建账户和 API 密钥](https://docs.langchain.com/langsmith/create-account-api-key)
- 如果使用 Gemini 模型，需要 Google AI API 密钥，请参阅 [Google AI Studio](https://aistudio.google.com/api-keys)

## 安装

使用 `google-adk` 额外依赖安装该包。额外依赖会引入 `google-adk` 和包装器所需的其他依赖：

```bash
pip install "deployments-wrap-sdk[google-adk]"
```

<Note>
PyPI 分发名称为 `deployments-wrap-sdk`，但 Python 导入路径为 `saf_sdk`。两者指代同一个包。
</Note>

## 快速开始

此最小示例构建一个返回输入内容作为响应的 agent，且无需模型 API 密钥。该 agent 跳过 LLM 调用，以便您在连接真实模型之前验证部署是否正常工作。

创建 `agent.py`：

```python agent.py
from google.adk.agents import Agent
from google.adk.models.llm_response import LlmResponse
from google.adk.runners import Runner
from google.genai.types import Content, Part
from saf_sdk.adk import LangsmithSessionService, wrap


def echo_callback(callback_context, llm_request):
    """Return the user's message instead of calling a real model."""
    user_text = ""
    if callback_context.user_content and callback_context.user_content.parts:
        for part in callback_context.user_content.parts:
            if part.text:
                user_text += part.text
    return LlmResponse(
        content=Content(role="model", parts=[Part(text=f"echo: {user_text}")])
    )


agent = wrap(
    Runner(
        agent=Agent(
            name="echo_agent",
            model="gemini-2.5-flash",
            instruction="Echo the user message.",
            before_model_callback=echo_callback,
        ),
        app_name="adk_echo",
        session_service=LangsmithSessionService(),
    )
)
```

有两点至关重要：

1. **将 `LangsmithSessionService()` 作为运行器的 `session_service` 传递**。如果忘记，`wrap()` 会抛出 `TypeError`。Agent Server 需要这个钩子来通过其检查点加载和保存 ADK 会话状态。
2. **将包装后的 `agent` 导出为模块级变量**。Agent Server 在提供图服务时会导入此符号。

对于实际 agent，请删除 `before_model_callback` 并直接配置模型。例如，通过设置 `model="gemini-2.5-flash"` 并配置 `GOOGLE_API_KEY` 来使用 Gemini，或通过 ADK 的 LiteLLM 适配器（`google.adk.models.lite_llm.LiteLlm`，可通过 `google-adk[extensions]` 获取）使用 Claude/OpenAI。

## 功能和限制

`wrap()` 将 ADK 运行时的一个明确定义的子集桥接到 Agent Server。在移植现有 ADK agent 之前请查看以下边界，因为某些 ADK 功能会原样传递，而其他功能则有意不支持。

### 支持的功能

- **Agent 原语**：`Agent`、`SequentialAgent` 和 `ParallelAgent`，包括通过 `sub_agents` 参数进行的嵌套子代理委托。
- **工具**：Python 函数工具和 `LongRunningFunctionTool`。
- **模型**：直接支持 Gemini 模型，以及 ADK 的 LiteLLM 适配器（`google.adk.models.lite_llm.LiteLlm`，可通过 `google-adk[extensions]` 获取）支持的任何模型。在部署时设置提供商的 API 密钥。
- **Token 流式传输**：ADK 部分事件通过 LangGraph 的异步回调管理器转发，因此 token 块会到达使用 `stream_mode="messages"` 和 Studio 聊天视图的客户端。
- **结构化输出**：使用 `output_schema` 和 `output_key` 配置的 agent 会在图的响应中的 `messages` 之外暴露类型化值。
- **会话持久化**：`LangsmithSessionService` 在部署的检查点存储中保存 ADK 会话状态。状态在重启后保留，并在同一线程的每个后续轮次中加载。
- **追踪**：当 `LANGSMITH_TRACING=true` 时，包装器会自动调用 `configure_google_adk()`（请参阅[启用追踪](#enable-tracing)）。
- **认证**：如果启用了 Agent Server [认证](/langsmith/auth)，经过认证的用户 ID 将成为 ADK 的 `user_id`。否则用户 ID 为 `"anonymous"`。

### 不支持的功能

- **多模态输入**：包装器仅将 `messages[-1].content` 作为单个文本部分转发。入站图像、文件、音频或内联二进制块不会传递给 ADK 运行器。
- **每轮多个新消息**：只有 `messages` 中的最后一项被视为新的用户消息。对话历史记录从 ADK 会话状态重建，而非从 LangGraph 消息列表重建。
- **双向/实时流式传输**：包装器硬编码 `RunConfig(streaming_mode=StreamingMode.SSE)`。不会调用 ADK 的 `Runner.run_live()` 和用于音频或语音代理的双向流式传输模式，因此无法通过 `wrap()` 部署实时音频和语音代理。
- **非文本输出部分**：仅从 ADK 事件中收集 `part.text` 值。代理生成的内联图像、音频或文件不会显示在图的 `messages` 输出中。
- **中间事件作为消息**：响应作为一条包含连接文本的 `AIMessage` 发出。工具调用、工具结果和中间子代理轮次不会作为图的 `messages` 字段中的单独项暴露。请在 [LangSmith traces](/langsmith/observability) 中检查它们。
- **替代 ADK 会话服务**：`runner.session_service` 必须是 `LangsmithSessionService`。ADK 的 `InMemorySessionService`、`DatabaseSessionService` 和 `VertexAiSessionService` 会因 `TypeError` 被拒绝，因为会话状态保存在 LangGraph 检查点中。
- **原生 LangGraph 中断**：包装器不暴露 LangGraph 的 `interrupt` 或 `Command(resume=...)` 机制。基于 `LongRunningFunctionTool` 构建的人机交互流程遵循 ADK 自身的模式：工具返回诸如 `pending_approval` 的状态，代理回复，后续轮次解析待处理的调用。

## 项目布局

一个可部署的项目需要三个文件：

```
my-adk-agent/
├── agent.py              # exports the wrapped agent
├── langgraph.json        # Agent Server config
└── pyproject.toml        # Python dependencies
```

[`langgraph.json`](/langsmith/application-structure#configuration-file-concepts) 指向 Agent Server 的导出符号：

```json langgraph.json
{
  "$schema": "https://langgra.ph/schema.json",
  "dependencies": ["."],
  "graphs": {
    "adk_echo": "./agent.py:agent"
  },
  "env": ".env"
}
```

`pyproject.toml` 声明依赖项：

```toml pyproject.toml
[project]
name = "my-adk-agent"
version = "0.0.1"
requires-python = ">=3.11"
dependencies = [
    "deployments-wrap-sdk[google-adk]>=0.0.1",
]
```

## 安装依赖项

```bash
pip install -e .
```

## 本地运行

使用 [LangGraph CLI](/langsmith/cli) 启动本地 Agent Server：

```bash
langgraph dev
```

这会在 `http://127.0.0.1:2024` 上启动智能体服务，并打开 [LangSmith Studio](/langsmith/studio)，以便您可以与智能体对话。使用 `curl` 直接发送请求：

```bash
# Create a thread
THREAD=$(curl -s -X POST http://127.0.0.1:2024/threads \
  -H "Content-Type: application/json" -d '{}' | python -c "import sys, json; print(json.load(sys.stdin)['thread_id'])")

# Run the agent and wait for the final response
curl -s -X POST "http://127.0.0.1:2024/threads/$THREAD/runs/wait" \
  -H "Content-Type: application/json" \
  -d '{
    "assistant_id": "adk_echo",
    "input": {"messages": [{"type": "human", "content": "Hello"}]}
  }'
```

## 部署到 LangSmith

一旦代理在本地运行，使用 `langgraph deploy` 将其部署到 LangSmith：

```bash
langgraph deploy --name my-adk-agent
```

有关环境配置、部署类型和版本管理，请参阅[部署到云](/langsmith/deploy-to-cloud)。有关自托管设置，请参阅[自托管部署](/langsmith/self-hosted)。

## 启用追踪

`wrap()` 会在启用 LangSmith 追踪时自动调用 `langsmith.integrations.google_adk.configure_google_adk()`，因此您只需在部署上设置环境变量：

```bash .env
LANGSMITH_API_KEY=your-langsmith-api-key
LANGSMITH_TRACING=true
LANGSMITH_PROJECT=my-adk-agent     # optional
GOOGLE_API_KEY=your-google-api-key
```

[追踪](/langsmith/observability) 在 [LangSmith UI](https://smith.langchain.com) 中显示智能体调用、工具调用和 LLM 交互。有关底层追踪集成的更多信息，请参阅[追踪 Google ADK 应用](/langsmith/trace-with-google-adk)。

## API 参考

### `wrap(runner)`

包装一个已配置的 `google.adk.runners.Runner` 并返回一个 LangGraph `Pregel` 图，可以从模块导出并由 Agent Server 提供服务。

| 参数 | 类型 | 描述 |
|---|---|---|
| `runner` | `google.adk.runners.Runner` | 一个已配置的 ADK Runner。其 `session_service` **必须**是 `LangsmithSessionService`。 |

**返回：** 一个名为 `runner.app_name` 的 `Pregel` 图。

**抛出：** 如果 `runner.session_service` 不是 `LangsmithSessionService`，则抛出 `TypeError`。

如果 `runner.agent` 定义了 `output_key`，则该键的值也会在图的输出中暴露，以及 `messages`。这就是使 ADK 结构化输出智能体（`output_schema=...`、`output_key=...`）与 Studio 和 `/runs/wait` 响应配合工作的原因。

### `LangsmithSessionService`

一个由 Agent Server 的检查点存储支持的 `google.adk.sessions.BaseSessionService` 实现。包装器自动管理会话生命周期。它在线程第一轮时创建会话，在后续轮次从检查点加载，并在运行完成时将更新的会话写回。

每个 `Runner` 使用一个新鲜实例：

```python
session_service = LangsmithSessionService()
```

你不需要直接调用它的方法；`wrap()` 会通过 ADK 的标准会话生命周期来驱动这些方法。

### `ADKInput`

包装后代理的默认输入模式。

| 字段 | 类型 | 描述 |
|---|---|---|
| `messages` | `list[AnyMessage]` | （必填）对话消息；包装器将 `messages[-1].content` 作为新的用户消息发送给 ADK 运行器。 |
| `state_delta` | `dict[str, Any] \| None` | （可选）透传给 `runner.run_async(state_delta=...)` 以便在本次轮次中修改 ADK 会话状态。 |

### `ADKOutput`

包装后代理的默认输出模式。

| 字段 | 类型 | 描述 |
|---|---|---|
| `messages` | `list[AnyMessage]` | 代理的响应消息，通过 LangGraph 的 `add_messages` reducer 添加到对话中。 |

将 `messages` 作为类型化字段暴露（而不是普通 `dict`），这样 Studio 就能将图识别为聊天兼容并启用聊天模式开关。

## 工作原理

当运行到达时：

1. 包装后的图从运行配置中读取 `thread_id` 并将其作为 ADK 的 `session_id`。如果启用了[身份验证](/langsmith/auth)，已认证用户的 id 将成为 ADK 的 `user_id`；否则用户 id 为 `"anonymous"`。
2. 包装器从 LangGraph 检查点加载之前的会话（如果有）到 `LangsmithSessionService`，然后让运行器处理最新消息。
3. 运行器发出 ADK 事件。包装器通过 LangGraph 的异步回调管理器转发部分 token 事件，使其通过 `stream_mode="messages"` 进行流式输出，并收集最终文本以生成响应消息。
4. 当运行完成时，包装器将 ADK 会话序列化并通过 `entrypoint.final(save=...)` 保存到检查点。同一个线程上的下一次运行将从该状态恢复。

这意味着 ADK 自己的会话/状态语义被端到端地保留，而部署可以获得标准的 Agent Server 功能：持久化运行、流式输出、多线程持久化和追踪。
