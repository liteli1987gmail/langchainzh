## 概述

向量存储[嵌入](/oss/integrations/embeddings)数据并执行相似性搜索。

```mermaid
flowchart LR

    subgraph "📥 Indexing phase (store)"
        A[📄 Documents] --> B[🔢 Embedding model]
        B --> C[🔘 Embedding vectors]
        C --> D[(Vector store)]
    end

    subgraph "📤 Query phase (retrieval)"
        E[❓ Query text] --> F[🔢 Embedding model]
        F --> G[🔘 Query vector]
        G --> H[🔍 Similarity search]
        H --> D
        D --> I[📄 Top-k results]
    end

    classDef process fill:#E5F4FF,stroke:#006DDD,stroke-width:2px,color:#030710
    class A,B,C,D,E,F,G,H,I process
```

### 接口

LangChain 为向量存储提供了统一的接口，允许你：

- `add_documents` - 向存储中添加文档。
- `delete` - 根据 ID 删除存储的文档。
- `similarity_search` - 查询语义相似的文档。

这种抽象让你能够在不改变应用逻辑的情况下切换不同的实现。

### 初始化

要初始化向量存储，需要为其提供一个嵌入模型：

```python
from langchain_core.vectorstores import InMemoryVectorStore
vector_store = InMemoryVectorStore(embedding=SomeEmbeddingModel())
```

### 添加文档

添加 @[`Document`] 对象（包含 `page_content` 和可选的元数据），如下所示：

```python
vector_store.add_documents(documents=[doc1, doc2], ids=["id1", "id2"])
```

### 删除文档

通过指定 ID 删除：

```python
vector_store.delete(ids=["id1"])
```

### 相似性搜索

使用 `similarity_search` 发起语义查询，该方法返回最相近的嵌入文档：

```python
similar_docs = vector_store.similarity_search("your query here")
```

许多向量存储支持以下参数：

* `k` — 返回结果的数量
* `filter` — 基于元数据进行条件过滤

### 相似度度量与索引

嵌入相似度可以使用以下方式计算：

* **余弦相似度**
* **欧氏距离**
* **点积**

高效搜索通常采用 HNSW（Hierarchical Navigable Small World，分层可导航小世界）等索引方法，但具体细节取决于向量存储。

### 元数据过滤

通过元数据进行过滤（例如，来源、日期）可以优化搜索结果：

```python
vector_store.similarity_search(
  "query",
  k=3,
  filter={"source": "tweets"}
)
```

<important>
对基于元数据过滤的支持因实现而异。
请查阅所选向量存储的文档了解详情。
</important>

    index_name="langchain-demo",
    embedding=embeddings,
    es_url="http://localhost:9200",
)
