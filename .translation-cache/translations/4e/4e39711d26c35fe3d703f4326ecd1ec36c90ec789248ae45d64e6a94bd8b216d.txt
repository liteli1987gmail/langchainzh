在本教程中，我们将构建一个自定义代理，该代理可以使用 LangGraph 回答有关 SQL 数据库的问题。

LangChain 提供内置的[代理](/oss/langchain/agents)实现，使用 [LangGraph](/oss/langgraph/overview) 原语实现。如果需要更深入的定制，可以直接在 LangGraph 中实现代理。本指南演示了 SQL 代理的示例实现。如需实用介绍，请参阅[使用更高级别的 LangChain 抽象构建 SQL 代理](/oss/langchain/sql-agent)。

<Warning>
构建 SQL 数据库的问答系统需要执行模型生成的 SQL 查询。这样做存在固有风险。请确保您的数据库连接权限始终尽可能严格地限制在代理的需求范围内。这将减轻（但不能完全消除）构建由模型驱动的系统的风险。
</Warning>

这个[预构建代理](/oss/langchain/sql-agent)帮助我们快速启动，但我们依赖系统提示来约束其行为——例如，我们指示代理始终以"list tables"工具开始，并在执行查询之前始终运行 query-checker 工具。

我们可以通过在 LangGraph 中自定义代理来实施更高程度的控制。在这里，我们实现了一个简单的 ReAct 代理设置，具有专门用于特定工具调用的节点。我们将使用与预构建代理相同的[状态]。

### 概念

我们将涵盖以下概念：

- 用于从 SQL 数据库读取的[工具](/oss/langchain/tools)
- LangGraph [图形 API](/oss/langgraph/graph-api)，包括状态、节点、边和条件边。
- [人机交互](/oss/langgraph/interrupts)流程

    Be sure that the tables actually exist by calling sql_db_list_tables first!
    Example Input: table1, table2, table3

sql_db_query: Input to this tool is a detailed and correct SQL query, output is a result from the database.
    If the query is not correct, an error message will be returned.
    If an error is returned, rewrite the query, check the query, and try again.
    If you encounter an issue with Unknown column 'xxxx' in 'field list', use sql_db_schema to query the correct table fields.

## 6. 实现人在环中审查

在执行 SQL 查询之前，检查代理的 SQL 查询以防止意外操作或效率低下，这是明智的做法。

在这里，我们利用 LangGraph 的[人在环中](/oss/langgraph/interrupts)功能，在执行 SQL 查询之前暂停运行并等待人工审查。使用 LangGraph 的[持久层](/oss/langgraph/persistence)，我们可以无限期地暂停运行（或者至少在持久层存活期间）。
