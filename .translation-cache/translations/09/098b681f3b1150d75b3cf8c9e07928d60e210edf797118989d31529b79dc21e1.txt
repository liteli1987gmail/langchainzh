Deep Agents Code 在两个目录层次结构中存储数据：

- **`~/.deepagents/`** — Deep Agents 专用数据（代理记忆、技能、会话）
- **`~/.agents/`** — 工具无关数据（跨 AI CLI 工具共享的技能）

## 目录结构

```text
~/.deepagents/
├── .state/                  # Per-machine Deep Agents Code state (managed automatically)
│   ├── sessions.db          #   SQLite database for conversation checkpoints
│   ├── history.jsonl        #   Command input history
│   ├── chatgpt-auth.json    #   ChatGPT OAuth token for the openai_codex provider
│   ├── ...                  #   Other markers & credentials
└── {agent}/                 # Per-agent directory (default: "agent")
    ├── AGENTS.md            # User customizations to agent instructions
    ├── skills/              # User-level skills
    │   └── {skill-name}/
    │       └── SKILL.md
    └── agents/              # Custom subagent definitions
        └── {subagent-name}/
            └── AGENTS.md

~/.agents/                   # Tool-agnostic alias (shared across AI CLIs)
└── skills/                  # Skills available to any compatible tool
    └── {skill-name}/
        └── SKILL.md

{project}/                   # Project-level (in git repo root)
├── AGENTS.md                # Project instructions (root-level)
└── .deepagents/
│   ├── AGENTS.md            # Project instructions (preferred location)
│   ├── skills/              # Project-specific skills
│   │   └── {skill-name}/
│   │       └── SKILL.md
│   └── agents/              # Project-specific subagents
│       └── {subagent-name}/
│           └── AGENTS.md
└── .agents/                 # Tool-agnostic project skills
    └── skills/
        └── {skill-name}/
            └── SKILL.md
```

## 内容存放位置

| Data | Location | Read/Write | Notes |
|------|----------|------------|-------|
| **会话** | `~/.deepagents/.state/sessions.db` | R/W | SQLite checkpoint database |
| **输入历史** | `~/.deepagents/.state/history.jsonl` | R/W | JSON-lines, up/down arrow recall |
| **ChatGPT OAuth 令牌** | `~/.deepagents/.state/chatgpt-auth.json` | R/W | Backs the [`openai_codex`](/oss/deepagents/code/providers) 提供程序；在使用 ChatGPT 登录时创建，并自动刷新。仅可由您的用户账户读取。|
| **基础指令** | 包 `default_agent_prompt.md` | R | 不可变，随 Deep Agents Code 升级更新 |
| **用户自定义** | `~/.deepagents/{agent}/AGENTS.md` | R/W | Appended to base instructions |
| **项目指令** | `.deepagents/AGENTS.md` or `AGENTS.md` | R | 如果存在则两者都加载 |
| **用户技能** | `~/.deepagents/{agent}/skills/` | R/W | Agent-specific skills |
| **共享技能** | `~/.agents/skills/` | R | 工具无关，跨 CLI |
| **项目技能** | `.deepagents/skills/` or `.agents/skills/` | R | 项目范围 |
| **自定义子代理** | `~/.deepagents/{agent}/agents/` | R/W | User-defined subagents |
| **项目子代理** | `.deepagents/agents/` | R | 项目定义的子代理 |

## 优先级规则

当同一项目存在于多个位置时， **更高优先级完全胜出** （不合并）。

### 技能

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/skills/` — 用户 Deep Agents Code
2. `~/.agents/skills/` — 用户工具无关
3. `.deepagents/skills/` — 项目 Deep Agents Code
4. `.agents/skills/` — 项目工具无关 *（最高）*

加载技能时，Deep Agents Code 会验证解析的文件路径是否保持在这些目录之一内。解析到所有技能根目录之外的符号链接会被拒绝。要允许符号链接目标位于其他目录，请参见 [`[skills].extra_allowed_dirs`](/oss/deepagents/code/configuration#skill-directory-allowlist).

### 子代理

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/agents/` — 用户级别
2. `.deepagents/agents/` — 项目级别 *（最高）*

每个子代理都是一个 `AGENTS.md` 文件，包含 YAML frontmatter（`name`, `description`，可选 `model`）和用于系统提示的 markdown 正文。请参阅 [在 Deep Agents Code 中使用子代理](/oss/deepagents/code/subagents) 获取完整的格式参考。

### 指令

所有指令源都会 **合并** （而非覆盖）：

1. 包基础提示词 *（始终加载）*
2. `~/.deepagents/{agent}/AGENTS.md` *（追加）*
3. `.deepagents/AGENTS.md` *（追加）*
4. `AGENTS.md` 在项目根目录 *（追加）*

## `.deepagents` vs `.agents`

| 目录 | 用途 | 使用场景 |
|-----------|---------|-------------|
| `.deepagents/` | Deep Agents Code 专用 | 使用 Deep Agents Code 专用功能的技能和配置 |
| `.agents/` | 工具无关 | 您希望在不同 AI CLI 工具之间共享的技能 |

<Tip>
使用 `.agents/skills/` 来处理适用于任何 AI 编程助手的技能。
使用 `.deepagents/skills/` 来处理依赖于 Deep Agents 专用工具或约定的技能。
</Tip>

## 清理

| 需求 | 操作 |
|------|--------|
| 重置所有数据 | `rm -rf ~/.deepagents` |
| 仅清除会话 | `rm ~/.deepagents/.state/sessions.db*` |
| 清除输入历史 | `rm ~/.deepagents/.state/history.jsonl` |
| 清除存储的 API 密钥 | `rm ~/.deepagents/.state/auth.json` |
| 清除 MCP OAuth 令牌 | `rm -rf ~/.deepagents/.state/mcp-tokens` |
| 清除 MCP 项目信任 | `rm ~/.deepagents/.state/mcp_trust.json` |
| 重新运行首次运行引导 | `rm ~/.deepagents/.state/onboarding_complete` |
| 重置代理指令 | `dcode agents reset --agent {name}` |
| 删除技能 | `rm -rf ~/.deepagents/{agent}/skills/{skill-name}` |

<Warning>
    删除 `~/.deepagents/.state/sessions.db` 将删除所有对话历史和检查点。

    除非您有备份，否则此操作无法撤消 `sessions.db` file.
</Warning>
