[聊天模型](/oss/langchain/models) 是使用一系列 [消息](/oss/langchain/messages) 作为输入并返回消息作为输出的语言模型 <Tooltip tip="Older models that do not follow the chat model interface and instead use an interface that takes a string as input and returns a string as output. These models typically do not include the prefix 'Chat' in their name or include 'LLM' as a suffix.">（与传统纯文本 LLM 相反）</Tooltip>.

## 路由器和代理

路由器和代理让您通过单一 API 和凭证访问来自多个提供商的模型。它们可以简化计费，让您无需更改集成即可在模型之间切换，并提供自动回退等功能。

| 提供商 | 集成 | 描述 |
|-|-|-|
| [OpenRouter](https://openrouter.ai/) | [`ChatOpenRouter`](/oss/integrations/chat/openrouter) | 统一访问 OpenAI、Anthropic、Google、Meta 等模型 |
| [LiteLLM](https://www.litellm.ai/) | [`ChatLiteLLMRouter`](/oss/integrations/chat/litellm) | 统一接口，支持 OpenAI、Anthropic、Azure、Hugging Face 等，具有路由和回退功能 |

## 聊天补全 API

某些模型提供商提供与 OpenAI 兼容的端点 [聊天补全 API](https://platform.openai.com/docs/api-reference/chat)。在这种情况下，您可以使用 [`ChatOpenAI`](/oss/integrations/chat/openai) 与自定义 `base_url` 连接这些端点以获得基本聊天功能。

<Warning>
    `ChatOpenAI` 目标 [官方 OpenAI API 规范](https://github.com/openai/openai-openapi) 仅支持官方功能。第三方提供商的非标准响应字段（例如 `reasoning_content`, `reasoning`, `reasoning_details`) **不会被提取或保留**。当您需要访问非标准功能时，请使用特定提供商的包。

    例如，OpenRouter 有专门的 LangChain 集成。请参阅 [`ChatOpenRouter` 指南](/oss/integrations/chat/openrouter) 了解设置和使用方法。
</Warning>
