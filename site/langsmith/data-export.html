**适用计划限制**

请注意，数据导出功能仅支持 LangSmith Plus 或 Enterprise 计划。

LangSmith 的批量数据导出功能允许您将特定项目和日期范围内的追踪数据导出为 Parquet 格式到 S3 兼容存储桶，字段与 Run 数据格式 一致。这对于在 BigQuery、Snowflake、Redshift 或 Jupyter Notebooks 等工具中进行离线分析非常有用。

此页面涵盖以下内容：

  • - 创建导出目标位置
  • - 创建和配置导出任务，包括计划导出和字段过滤
  • - 监控导出进度

**准备工作：** 导出时间取决于数据量，LangSmith 限制并发导出任务数量。批量导出有 72 小时运行超时限制——请参阅自动重试行为了解更多详情。启动后，LangSmith 将自动处理导出流程的编排和弹性。

1. 创建目标位置

目标位置指示 LangSmith 将导出的数据写入何处。在发出此请求之前，您需要准备：

  • - 您的 LangSmith API 密钥和工作区 ID。
  • - 授予 LangSmith **写入权限**的 S3 或 S3 兼容存储桶（请参阅所需权限）。
  • - 存储桶名称、前缀，以及 AWS 区域（用于 AWS S3）或端点 URL（用于 GCS、MinIO 或其他 S3 兼容提供商）。
  • - 存储桶的访问密钥和密钥。
curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports/destinations' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "destination_type": "s3",
    "display_name": "My S3 Destination",
    "config": {
      "bucket_name": "your-s3-bucket-name",
      "prefix": "root_folder_prefix",
      "region": "your aws s3 region",
      "endpoint_url": "your endpoint url for s3 compatible buckets"
    },
    "credentials": {
      "access_key_id": "YOUR_S3_ACCESS_KEY_ID",
      "secret_access_key": "YOUR_S3_SECRET_ACCESS_KEY"
    }
  }'

凭据以加密形式安全存储。API 将在保存前验证目标和凭据是否有效。如果请求失败，请参阅调试目标错误。

保存响应中的 id；创建导出任务时需要用到它。

有关权限设置、提供商特定配置（AWS S3、GCS、MinIO）和凭据选项，请参阅管理批量导出目标。

2. 创建导出任务

导出任务针对一个项目（或工作区中的所有实验）和一个日期范围。您需要：

  • - 来自上一步的目标 id。
  • - 项目 ID（session_id）或 "all_experiments": true——从**追踪项目**列表中的单个项目视图复制项目 ID。
  • - 采用 UTC ISO 8601 格式的 start_time 和 end_time。
curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "bulk_export_destination_id": "your_destination_id",
    "session_id": "project_uuid",
    "start_time": "2024-01-01T00:00:00Z",
    "end_time": "2024-01-03T00:00:00Z",
    "format_version": "v2_beta"
  }'

start_time 是包含性的，end_time 是排他性的。导出将包含所有满足 run.start_time >= start_time 且 run.start_time < end_time 的运行。

保存响应中的 id，用于监控导出进度。

您可以选择添加 filter 表达式来缩小导出的运行集合。请参阅我们的过滤查询语言和示例了解语法。如果不设置 filter 字段，将导出所有运行。

导出所有实验

提示

自托管：目前仅在 v0.16.1rc1 预览版本上可用。在生产环境中运行之前，请等待 v0.16.1 稳定版本发布。

要导出工作区中的每个实验，而不是使用 session_id 针对单个项目，请在请求中设置 all_experiments: true。LangSmith 会在您针对数据集运行评估时创建一个实验，任何设置了 reference_dataset_id 的追踪项目也都符合条件。

all_experiments 和 session_id 是互斥的——只能设置其中一个。

curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "bulk_export_destination_id": "your_destination_id",
    "all_experiments": true,
    "start_time": "2024-01-01T00:00:00Z",
    "end_time": "2024-02-01T00:00:00Z",
    "format_version": "v2_beta"
  }'

LangSmith 在运行时解析实验会话集合，因此导出会包含您在提交任务后、编排器开始处理前创建的任何实验。

相同的 all_experiments 标志也适用于计划导出——包含 interval_hours 并省略 end_time（而不是提供 end_time）。

提示

**云端限制：每次导出最多 250 个实验**

在 LangSmith 云端 上，每个 all_experiments 导出最多包含 250 个实验。要导出更多实验：

  • - 查询已完成的 all_experiments 导出以查看包含的追踪项目，然后使用 session_id 为剩余实验创建标准批量导出。
  • - 或者，通过 support.langchain.com 联系支持团队，为您的工作区申请更高的限制。

自托管 LangSmith 没有每次导出的限制。

计划定期导出

提示

需要 LangSmith Helm 版本 >= 0.10.42（应用版本 >= 0.10.109）

计划导出定期收集运行数据并导出到配置的目标位置。要创建计划导出，请包含 interval_hours 并省略 end_time：

curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "bulk_export_destination_id": "your_destination_id",
    "session_id": "project_uuid",
    "start_time": "2024-01-01T00:00:00Z",
    "interval_hours": 1,
    "format_version": "v2_beta"
  }'
  • - interval_hours 必须在 1 到 168（一周）之间（含）。
  • - 计划导出必须省略 end_time；一次性导出仍需要 end_time。
  • - 每个生成的导出覆盖 start_time 到 start_time + interval_hours，然后每次后续运行向前推进 interval_hours。由于 end_time 是排他的，连续的导出不会重叠。
  • - 生成的导出在 end_time + 10 minutes 运行，以考虑最近提交的以 end_time 为结束时间的运行。
  • - 生成的导出已填充 source_bulk_export_id 属性。如有需要，必须单独取消——取消源导出**不会**取消已生成的导出。
  • - 要停止计划导出，请取消它。

**示例**

如果创建计划批量导出时设置 start_time=2025-07-16T00:00:00Z 和 interval_hours=6：

导出开始时间结束时间运行时间
12025-07-16T00:00:00Z2025-07-16T06:00:00Z2025-07-16T06:10:00Z
22025-07-16T06:00:00Z2025-07-16T12:00:00Z2025-07-16T12:10:00Z
32025-07-16T12:00:00Z2025-07-16T18:00:00Z2025-07-16T18:10:00Z

限制导出字段

提示

需要 LangSmith Helm 版本 >= 0.12.11（应用版本 >= 0.12.42）。一次性导出和计划导出均支持。

您可以使用 export_fields 参数限制包含的字段来提高导出速度并减小文件大小。省略此参数时，会包含所有字段。

curl --request POST \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID' \
  --data '{
    "bulk_export_destination_id": "your_destination_id",
    "session_id": "project_uuid",
    "start_time": "2024-01-01T00:00:00Z",
    "end_time": "2024-01-03T00:00:00Z",
    "export_fields": ["id", "name", "run_type", "start_time", "end_time", "status", "total_tokens", "total_cost"],
    "format_version": "v2_beta"
  }'
建议

排除 inputs 和 outputs 可以显著提升导出性能并减小文件体积，尤其对于大型运行场景。仅在需要进行分析时才包含这些字段。

数据将按以下 Hive 分区结构导出到您的存储桶中：

3. 监控您的导出任务

使用上一步中的 id 轮询导出状态：

curl --request GET \
  --url 'https://api.smith.langchain.com/api/v1/bulk-exports/{export_id}' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: YOUR_API_KEY' \
  --header 'X-Tenant-Id: YOUR_WORKSPACE_ID'

响应中的 status 字段将为 CREATED、RUNNING、COMPLETED、FAILED、CANCELLED 或 TIMEDOUT 之一。根据数据量的大小，导出可能需要一些时间。状态变为 COMPLETED 后，Parquet 文件即可在您的存储桶中使用。

请参阅监控和排查批量导出，了解如何列出运行记录、停止导出和诊断故障。
