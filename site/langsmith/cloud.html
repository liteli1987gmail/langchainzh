### GCP 服务

以下适用于 GCP 上的**美国、欧盟和亚太** SaaS 区域。

LangSmith 由以下服务组成，全部部署在 Google Kubernetes Engine (GKE) 上：

* LangSmith Frontend：提供 LangSmith UI 服务。
* LangSmith Backend：提供 LangSmith API 服务。
* LangSmith Platform Backend：处理身份验证和其他大容量任务。（内部服务）
* LangSmith Playground：处理将请求转发到各种 LLM 提供商以支持 Playground 功能。
* LangSmith Queue：处理异步任务的处理。（内部服务）

LangSmith 使用以下 GCP 存储服务：

* Google Cloud Storage (GCS) 用于存储运行输入和输出。
* Google Cloud SQL PostgreSQL 用于事务性工作负载。
* Google Cloud Memorystore for Redis 用于队列和缓存。
* GCP 上的 ClickHouse Cloud 用于跟踪数据摄取和分析。我们的服务通过私有端点连接到托管在同一 GCP 区域的 ClickHouse Cloud。

我们使用的一些其他 GCP 服务包括：

* Google Cloud Load Balancer 用于将流量路由到 LangSmith 服务。
* Google Cloud CDN 用于缓存静态资产。
* Google Cloud Armor 用于安全和速率限制。有关我们执行的速率限制的更多信息，请参阅[速率限制](/langsmith/usage-and-billing.html#rate-limits)。

* Amazon ElastiCache for Redis 用于队列和缓存。
* 通过 AWS PrivateLink 在 `us-east-2` 上的 ClickHouse Cloud 用于跟踪数据摄取和分析，与上方的[区域存储](#regional-storage)表一致。

我们使用的一些其他 AWS 服务包括：

* Elastic Load Balancing（网络负载均衡器）和 Istio ingress 用于将流量路由到 LangSmith 服务。文档化的 API 速率限制在 Istio ingress 网关处执行。详情请参阅[速率限制](/langsmith/usage-and-billing.html#rate-limits)。
* Amazon CloudFront 用于缓存静态资产（包括 Web UI 主机名 `aws.smith.langchain.com`）。
* CloudFront 上的 AWS WAF 用于边缘托管规则组（例如，AWS 托管规则通用保护和 Bot Control）。

私有连接（企业版）



**仅限企业版。** 私有连接仅向企业客户提供。联系您的客户代表或 [sales@langchain.dev](mailto:sales@langchain.dev) 以启用此功能。


企业客户可以使用 **AWS PrivateLink** 或 **GCP Private Service Connect (PSC)** 连接到 LangSmith，而无需将流量暴露在公共互联网上。

### AWS PrivateLink

**AWS** 上的客户可以通过 [AWS PrivateLink](https://docs.aws.amazon.com/vpc/latest/privatelink/) 连接到 LangSmith，提供从任何 VPC 的私有连接。本地区域连接支持跨区域连接。

#### 端点服务名称

| 区域 | 服务名称 |
|--------|-------------|
| 美国（`us-east-2`） | `com.amazonaws.vpce.us-east-2.vpce-svc-054f37092752bff6b` |

#### 设置

**1. 请求访问：** 联系您的客户代表或 [sales@langchain.dev](mailto:sales@langchain.dev)，并提供您的 AWS 账户 ID。LangChain 会将您的账户添加到端点服务的允许主体列表中。

**2. 在您的 AWS 账户中创建接口 VPC 端点**。附加一个安全组，允许来自您 VPC CIDR（或需要访问 LangSmith 的实例）的 **TCP 443 入站**：


```bash AWS CLI
aws ec2 create-vpc-endpoint \
  --vpc-id  \
  --service-name  \
  --vpc-endpoint-type Interface \
  --subnet-ids  \
  --security-group-ids  \
  --region
```

```hcl Terraform
resource "aws_vpc_endpoint" "langsmith" {
  vpc_id              = ""
  service_name        = ""
  vpc_endpoint_type   = "Interface"
  subnet_ids          = [""]
  security_group_ids  = [""]
}
```
**3. 等待接受。** LangChain 将接受该连接。端点状态将从 `pendingAcceptance` 变为 `available`。在接受后请等待几分钟，让更改完全传播，然后再测试连接。

#### 配置 DNS

配置 DNS，使 `aws.api.smith.langchain.com` 在您的 VPC 内解析到 VPC 端点的私有 DNS 名称。您可以使用任何私有 DNS 解决方案 — Route 53 私有托管区域、企业 DNS 解析器，或任何可从 VPC 访问的 DNS 服务器。

首先，获取您的端点的 DNS 名称：

```bash
aws ec2 describe-vpc-endpoints \
  --vpc-endpoint-ids  \
  --query 'VpcEndpoints[0].DnsEntries[0].DnsName' \
  --output text --region
```

然后为 `aws.api.smith.langchain.com` 创建一个 CNAME 记录，指向该 DNS 名称。以下是使用 Route 53 的示例：


```bash AWS CLI
aws route53 create-hosted-zone \
  --name aws.api.smith.langchain.com \
  --vpc VPCRegion=,VPCId= \
  --caller-reference langsmith-privatelink-$(date +%s) \
  --hosted-zone-config PrivateZone=true

aws route53 change-resource-record-sets \
  --hosted-zone-id  \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "aws.api.smith.langchain.com",
        "Type": "CNAME",
        "TTL": 300,
        "ResourceRecords": [{"Value": ""}]
      }
    }]
  }'
```

```hcl Terraform
resource "aws_route53_zone" "langsmith_privatelink" {
  name = "aws.api.smith.langchain.com"

  vpc {
    vpc_id = ""
  }
}

resource "aws_route53_record" "langsmith_privatelink" {
  zone_id = aws_route53_zone.langsmith_privatelink.zone_id
  name    = "aws.api.smith.langchain.com"
  type    = "CNAME"
  ttl     = 300
  records = [aws_vpc_endpoint.langsmith.dns_entry[0]["dns_name"]]
}
```


#### 验证连接

从 VPC 中的 EC2 实例或容器中：

```bash
curl https://aws.api.smith.langchain.com/ok
```

### GCP 专用服务连接

**GCP** 上的企业客户可以通过[专用服务连接 (PSC)](https://cloud.google.com/vpc/docs/private-service-connect) 连接到 LangSmith，实现私有连接而不向公网暴露流量。

#### 服务附件 URI

| 区域 | 服务附件 URI |
|--------|----------------------|
| 美国 (`us-central1`) | `projects/langchain-prod/regions/us-central1/serviceAttachments/gateway-psc-publish` |
| 欧盟 (`europe-west4`) | `projects/langchain-prod/regions/europe-west4/serviceAttachments/gateway-psc-publish` |
| 亚太 (`australia-southeast1`) | `projects/langchain-apac-prod/regions/australia-southeast1/serviceAttachments/gateway-psc-publish` |

#### PSC 域名

设置完成后，使用以下域名通过 PSC 连接访问 LangSmith：

| 区域 | 域名 |
|--------|--------|
| 美国 | `us-central1.p.api.smith.langchain.com` |
| 欧盟 | `europe-west4.p.api.smith.langchain.com` |
| 亚太 | `australia-southeast1.p.api.smith.langchain.com` |

#### 设置

**申请访问权限：** 联系您的客户经理或发送邮件至 [sales@langchain.dev](mailto:sales@langchain.dev)，并提供您的 GCP 项目 ID。LangChain 会将您的项目添加到服务附件的允许消费者列表中。

获得访问权限后，使用 gcloud CLI 或 Terraform 创建 PSC 端点并配置 DNS。

#### 创建 PSC 端点

在您的 VPC 中创建一条指向服务附件的转发规则：


```bash gcloud CLI

Create the PSC endpoint

gcloud compute forwarding-rules create langsmith-psc-endpoint \
  --region= \
  --network= \
  --subnet= \
  --target-service-attachment=projects/langchain-prod/regions//serviceAttachments/gateway-psc-publish \
  --load-balancing-scheme=""


Get the assigned IP address

gcloud compute forwarding-rules describe langsmith-psc-endpoint \
  --region= \
  --format="value(IPAddress)"
```

```hcl Terraform
resource "google_compute_forwarding_rule" "langsmith_psc" {
  name                  = "langsmith-psc-endpoint"
  project               = ""
  region                = ""
  network               = ""
  subnetwork            = ""
  target                = "projects/langchain-prod/regions//serviceAttachments/gateway-psc-publish"
  load_balancing_scheme = ""
}
```


#### 配置 DNS

在 VPC 中创建私有 DNS 区域，并添加一条 A 记录指向 PSC 端点 IP：


```bash gcloud CLI

Create a private DNS zone

gcloud dns managed-zones create langsmith-psc \
  --dns-name=".p.api.smith.langchain.com." \
  --visibility=private \
  --networks=


Add an A record pointing to the PSC endpoint IP

gcloud dns record-sets create ".p.api.smith.langchain.com." \
  --zone=langsmith-psc \
  --type=A \
  --rrdatas=
```

```hcl Terraform
resource "google_dns_managed_zone" "langsmith_psc" {
  name        = "langsmith-psc"
  project     = ""
  dns_name    = ".p.api.smith.langchain.com."
  visibility  = "private"

  private_visibility_config {
    networks {
      network_url = ""
    }
  }
}

resource "google_dns_record_set" "langsmith_psc" {
  name         = ".p.api.smith.langchain.com."
  project      = ""
  managed_zone = google_dns_managed_zone.langsmith_psc.name
  type         = "A"
  ttl          = 300
  rrdatas      = [google_compute_forwarding_rule.langsmith_psc.ip_address]
}
```


#### 验证连接

从 VPC 中的虚拟机：

```bash
curl https://.p.api.smith.langchain.com/ok
```

- 速率限制表示为 `count / interval`，其中 count 是间隔（秒）内允许的请求数。例如，`2000 / 10` 表示每 10 秒 2000 个请求。

### 速率限制类别

- **高吞吐量**：用于身份验证、元数据和反馈等核心操作的一般高容量端点。
- **仓库**：仓库和提示管理操作。
- **运行数据摄取**：用于可观测性的单个追踪/运行摄取端点。
- **图表**：图表生成和可视化端点。
- **多部分摄取**：通过多部分上传进行批量运行摄取，用于高容量追踪。
- **运行查询 (API)**：基于 API 密钥的运行查询操作，对复杂查询有更严格的限制。
- **运行查询 (用户)**：基于用户的运行查询操作，为交互式使用提供更高的限制。
- **生成**：AI 驱动的代码和内容生成端点（受限以防止滥用）。
- **提交**：提示版本控制和提交操作。
- **删除**：会话删除和工作流触发操作。
- **运行查找**：通过 UUID 检索特定运行。
- **示例**：获取用于少样本提示的数据集示例。
- **默认（API 密钥）**：针对不匹配特定模式的身份验证 API 请求的兜底速率限制。
- **默认（用户）**：针对不匹配特定模式的身份验证用户请求的兜底速率限制。
- **公共下载**：用于共享资源的高容量公共下载端点。
- **统计**：运行统计和分析端点（适用区域特定的限制）。
- **公共（兜底）**：针对未认证公共访问的默认速率限制。

有关速率限制和其他服务限制的更多信息，请参阅[管理概述](/langsmith/usage-and-billing.html#rate-limits)。
