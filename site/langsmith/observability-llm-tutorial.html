在本教程中，您将构建一个使用检索增强生成（RAG）的客户支持聊天机器人，并在开发的每个阶段添加 LangSmith 可观测性，从早期原型设计到生产环境。

学完本教程后，您将了解如何：

  • - 跟踪单个 LLM 调用和完整的应用程序管道。
  • - 收集和查询用户反馈。
  • - 记录元数据并将其用于过滤和 A/B 测试。
  • - 使用监控仪表板跟踪生产环境性能。

该应用程序将检索相关的文档片段，并用它们来回答用户问题。本教程中的检索器是模拟的；在实际应用中，您可以用向量搜索或类似技术替换它。

前提条件

在开始之前，确保您已具备：

  • - **LangSmith 账户**：在 smith.langchain.com 注册或登录。
  • - **LangSmith API 密钥**：按照创建 API 密钥指南操作。
  • - **OpenAI API 密钥**：从 OpenAI 仪表板生成。
  • - **LangSmith CLI**（可选）：安装后可在终端检查追踪记录。具体说明请参考 LangSmith CLI。

安装所需的软件包：

pip install langsmith openai
npm install langsmith openai
npm install -D typescript tsx

追踪 LLM 调用

首先追踪你的 OpenAI 调用，也就是模型实际被调用的地方。这样可以立即了解你的应用发送的提示词和模型返回的响应。

使用 wrap_openai (Python) 或 wrapOpenAI (TypeScript) 包装 OpenAI 客户端。创建一个名为 app.py（或 app.ts）的文件，包含以下代码：

from openai import OpenAI
from langsmith.wrappers import wrap_openai

client = wrap_openai(OpenAI())

docs = [
    "Acme Cloud supports unlimited users on Enterprise plans. Starter plans are limited to 5 users.",
    "To reset your password, click 'Forgot password' on the login page and follow the instructions sent to your email.",
    "API rate limits are 1,000 requests per hour on the Starter plan and 10,000 requests per hour on Enterprise.",
]

def retriever(query: str) -> list[str]:
    return docs

def support_bot(question: str) -> str:
    context = retriever(question)
    system_message = (
        "You are a helpful customer support agent. "
        "Answer using only the information provided below:\n\n"
        + "\n".join(context)
    )
    response = client.chat.completions.create(
        model="gpt-5.4-mini",
        messages=[
            {"role": "system", "content": system_message},
            {"role": "user", "content": question},
        ],
    )
    return response.choices[0].message.content

if __name__ == "__main__":
    print(support_bot("How many users can I have on the Starter plan?"))

Beta 测试

一旦您的应用在原型设计阶段运行良好，您就可以将其发布给一小部分真实用户。在此阶段，您通常无法确切知道用户将如何与应用交互，因此您需要更丰富的可观测性。您不仅想了解应用做了什么，还想了解用户的反应。

记录元数据

元数据 允许您使用有助于筛选和比较的属性标记运行。例如，使用的模型版本或发出请求的用户。

以下示例追踪检索器（使用 run_type="retriever"）和主函数（带有模型名称的 metadata 属性）：

from langsmith import traceable
from langsmith.wrappers import wrap_openai

client = wrap_openai(OpenAI())

docs = [

def retriever(query: str) -> list[str]:
    return docs

        "Answer using only the information provided below:\n\n"
        + "\n".join(context)
    )
    response = client.chat.completions.create(
        model="gpt-5.4-mini",
        messages=[
            {"role": "system", "content": system_message},
            {"role": "user", "content": question},
        ],
    )
    return response.choices[0].message.content

if __name__ == "__main__":
    support_bot("How many users can I have on the Starter plan?")
const client = wrapOpenAI(new OpenAI());

const docs = [
    "Acme Cloud supports unlimited users on Enterprise plans. Starter plans are limited to 5 users.",
    "To reset your password, click 'Forgot password' on the login page and follow the instructions sent to your email.",
    "API rate limits are 1,000 requests per hour on the Starter plan and 10,000 requests per hour on Enterprise.",
];

const retriever = traceable(  // [!code highlight]
    function retriever(query: string): string[] {  // [!code highlight]
        return docs;  // [!code highlight]
    },  // [!code highlight]
    { run_type: "retriever" }  // [!code highlight]
);  // [!code highlight]

const supportBot = traceable(

        const context = await retriever(question);
        const systemMessage =
            "You are a helpful customer support agent. " +
            "Answer using only the information provided below:\n\n" +

生产环境

有了强大的可观测性保障，您可以自信地将应用发布到生产环境。在生产环境中，流量会大幅增加，无法逐一检查每条跟踪。LangSmith 提供监控工具，帮助您了解整体行为，并在出现异常时深入分析。

监控

在 UI 侧边栏中选择 **Monitoring**，然后从左上角的下拉菜单中选择一个跟踪项目。图表显示项目随时间变化的关键指标，包括跟踪数量、延迟、错误率、反馈分数和成本。有关可用指标和图表配置的更多信息，请参阅 Dashboards。

A/B 测试

提示

分组功能需要在给定的元数据键下至少有两个不同的值。 </aside>

由于您已经记录了 llm 元数据属性，因此可以按该属性对监控图表进行分组，以比较模型随时间的性能表现。从 UI 侧边栏的 **Monitoring** 中，点击左上角的 **Group by**，从下拉菜单中选择 **Metadata**，然后选择 llm。图表将更新为按该属性分组显示结果。有关分组和自定义图表的更多信息，请参阅 Dashboards。

深入分析

当监控图表显示异常情况时，点击数据点可以冻结工具提示，然后点击指标名称（例如 **Input**）可跳转到该时间窗口的筛选后运行表。有关搜索和筛选运行的更多信息，请参阅 Filter traces。

总结

在本教程中，您在整个开发生命周期中为应用添加了 LangSmith 可观测性。同样的跟踪设置不仅能帮助您在原型设计阶段快速迭代，在生产环境中也将继续发挥价值。您将能够洞察单条跟踪和整体性能趋势。
