信息

自托管 LangSmith 是企业计划的附加组件，专为我们最大型、最注重安全的客户设计。请参阅我们的 定价页面 了解更多详情，以及 联系我们的销售团队 如果您想获取许可证密钥以在您的环境中试用 LangSmith。

本页面介绍如何在 **LangSmith** （可观测性、追踪和评估）中设置。您将使用 Helm 来安装 LangSmith 及其依赖项。

完成本页面后，您将拥有：

  • - **LangSmith UI 和 API**：用于 可观测性、追踪和 评估.
  • - **后端服务**：（队列、playground、ACE）。
  • - **数据存储**：（PostgreSQL、Redis、ClickHouse、可选的 blob 存储）。

对于 代理部署：要添加部署功能，请先完成本指南，然后按照 启用 LangSmith 部署、Fleet、Insights 和 Chat.

LangChain 已成功在以下 Kubernetes 发行版上测试了 LangSmith：

  • - Google Kubernetes Engine (GKE)
  • - Amazon Elastic Kubernetes Service (EKS)：如需架构模式和最佳实践，请参阅 在 AWS 上自托管.
  • - Azure Kubernetes Service (AKS)：如需架构模式和最佳实践，请参阅 在 AKS 上自托管.
  • - OpenShift (4.14+)
  • - Minikube 和 Kind（用于开发目的）

可用的模块： - AWS Terraform 模块 - Azure Terraform 模块

前提条件

Ensure you have the following tools/items ready. Some items are marked optional:

  1. 1. LangSmith 许可证密钥
  1. 1. 您可以从您的 LangChain 代表处获取此密钥。 联系我们的销售团队 了解更多详情。
  1. 2. Api Key Salt
  1. 1. 这是您可以生成的密钥。它应该是一个随机的字符字符串。
  2. 2. 您可以使用以下命令生成它：
   openssl rand -base64 32
  1. 3. JWT Secret（可选，但用于基本身份验证）
  1. 1. 这是您可以生成的密钥。它应该是一个随机的字符字符串。
  2. 2. 您可以使用以下命令生成此内容：
   openssl rand -base64 32

数据库

LangSmith 使用 PostgreSQL 数据库、Redis 缓存和 ClickHouse 数据库来存储跟踪记录。默认情况下，这些服务安装在您的 Kubernetes 集群内部。但是，我们强烈建议使用外部数据库。对于 PostgreSQL 和 Redis，最佳选择是您云提供商的托管服务。

更多信息，请参阅以下外部服务设置指南：

  • - PostgreSQL
  • - Redis
  • - ClickHouse

有关每个数据存储的最低支持版本，请参阅 自托管依赖项的最低版本.

Kubernetes 集群要求

  1. 1. 您需要一个可访问的工作 Kubernetes 集群 kubectl您的集群应满足以下最低要求：
  1. 1. 推荐：至少 16 个 vCPU、64GB 可用内存
  • * You may need to tune resource requests/limits for all of our different services based off of organization size/usage. You can find our recommendations in the 自托管扩缩容指南.
  • * We recommend using a cluster autoscaler to handle scaling up/down of nodes based on resource usage.
  • * 我们建议设置 metrics 服务器，以便开启自动扩缩容功能。
  • * 如果您在集群内运行 ClickHouse，则必须有一个至少具有 4 个 vCPU 和 16GB 内存的节点 **可分配** 因为 ClickHouse 默认会请求此数量的资源。
  1. 2. 集群上有有效的动态 PV provisioner 或可用的 PV（仅在集群内运行数据库时需要）
  • * 为了启用持久化，我们将尝试为集群内运行的任何数据库配置卷。
  • * 如果在集群中使用 PV，我们强烈建议在生产环境中设置备份。
  • * **We strongly encourage using a storage class backed by SSDs for better performance. We recommend 7000 IOPS and 1000 MiB/s throughput.**
  • * 在 EKS 上，您可能需要确保已安装 ebs-csi-driver 并为动态配置进行了配置。请参阅 EBS CSI Driver 文档 获取更多信息。

您可以通过运行以下命令进行验证：

      kubectl get storageclass

输出应至少显示一个具有支持动态配置的 provisioner 的存储类。例如：

      NAME            PROVISIONER                 RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE
      gp2 (default)   ebs.csi.eks.amazonaws.com   Delete          WaitForFirstConsumer   true                   161d
提示

我们强烈建议使用支持卷扩展的存储类。这是因为跟踪记录可能需要大量磁盘空间，您的卷可能需要随时间调整大小。 </aside>

请参阅 Kubernetes 文档 了解更多关于存储类的信息。

  1. 1. 要安装 helm 请参阅 Helm 文档
  1. 3. 到 https://beacon.langchain.com 的出站流量（如果不是离线模式运行）
  1. 1. LangSmith 需要访问 https://beacon.langchain.com 进行许可证验证和使用情况报告。这是 LangSmith 正常运行所必需的。您可以在 出站流量 section.

配置您的 Helm 图表：

  1. 1. 创建一个名为 langsmith_config.yaml 的新文件，其中包含上一步中的配置选项。
  2. 1. 您可以在 langsmith_config.yaml 文件。您可以在以下位置找到有关特定配置选项的更多信息 配置 section.
  3. 2. 如果您是 Kubernetes 或 Helm 的新手，我们建议从 Helm Chart 仓库中 examples 目录下的示例配置开始： LangSmith helm chart 示例.
  4. 3. 您可以在以下位置的 Helm Chart 仓库中查看完整的配置选项列表 values.yaml 文件 LangSmith Helm Chart
警告

仅覆盖您需要在 langsmith_config.yaml中需要的设置；不要复制整个 values.yaml. 保持配置最小化可确保您继续从 Helm chart 继承新的默认值和升级。

建议

如果您的集群强制执行非 root 或只读容器策略，请从 只读 Helm 配置示例开始。LangSmith 容器不需要 root 权限。该示例展示了如何设置 runAsNonRoot、服务 UID 和 GID、 fsGroup, RuntimeDefault seccomp 配置、丢弃的能力、禁用权限提升和可写 emptyDir 挂载点以满足需要临时存储的服务。

  1. 2. 至少，您需要设置以下配置选项（使用基本身份验证）：
警告

设置 apiKeySalt 一次，不要更改它。此值用于对所有静态 API 密钥进行哈希。更改它将永久使您组织中的所有现有 API 密钥失效，要求所有用户重新生成其密钥。 </aside>

   config:
     langsmithLicenseKey: "<your license key>"
     apiKeySalt: "<your api key salt>"
     authType: mixed
     initialOrgAdminEmail: "admin@langchain.dev" # Change this to your admin email address
     basicAuth:
       enabled: true
       initialOrgAdminPassword: "secure-password" # Must be at least 12 characters long and have at least one lowercase, uppercase, and symbol
       jwtSecret: <your jwt salt> # A random string of characters used to sign JWT tokens for basic auth.

您还需要指定您正在使用的任何外部数据库的连接详细信息。

部署到 Kubernetes：

  1. 1. 验证您可以连接到 Kubernetes 集群（注意：我们强烈建议安装到空命名空间中）
  1. 1. 运行 kubectl get pods

输出应该类似于：

        langsmith-eks-2vauP7wf 21:07:46 No resources found in default namespace.
提示

如果使用的是非默认命名空间，需要在 helm 和 kubectl 命令中指定命名空间，使用 -n <namespace> flag. </aside>

  1. 2. 确保您已添加 LangChain Helm 仓库。（如果您使用本地 charts 则跳过此步骤）
   helm repo add langchain https://langchain-ai.github.io/helm
  1. 3. 查找 chart 的最新版本。您可以在以下位置找到可用版本 Helm Chart 仓库.
  • * 我们通常建议使用最新版本。
  • * 您也可以运行 helm search repo langchain/langsmith --versions 来查看可用版本。输出将类似于：
    langchain/langsmith              	0.13.0      	0.13.1    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.34      	0.12.73    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.33      	0.12.72    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.32      	0.12.70    	Helm chart to deploy the langsmith application ...
    langchain/langsmith              	0.12.31      	0.12.69    	Helm chart to deploy the langsmith application ...
  1. 4. 运行 helm upgrade -i langsmith langchain/langsmith --values langsmith_config.yaml --version <version> -n <namespace> --wait --debug
  • * 将 <namespace> 替换为您要部署 LangSmith 的命名空间。
  • * 将 <version> 替换为您要安装的 LangSmith 版本（从前一步骤获取）。大多数用户应安装可用的最新版本。

一旦 helm install 命令运行并成功完成，您应该会看到类似以下的输出：

使用 LangSmith

现在 LangSmith 正在运行，您可以开始使用它来追踪您的代码。您可以在 自托管使用指南.

您的 LangSmith 实例现在正在运行，但可能尚未完全设置。

如果您使用了基本配置之一，将为您创建一个默认的管理员用户账户。您可以使用在 langsmith_config.yaml file.

下一步，强烈建议您与基础设施管理员合作：

  • * 为您的 LangSmith 实例设置 DNS 以便于访问
  • * 配置 SSL 以确保传输到 LangSmith 的追踪数据的加密
  • * 使用 单点登录 配置 LangSmith 来保护您的实例安全
  • * 将 LangSmith 连接到外部 Postgres 和 Redis 实例
  • * 设置 Blob 存储 来存储大文件

查看我们的 配置部分 获取有关如何配置这些选项的更多信息。

启用 LangSmith 部署、Fleet、洞察和聊天

要超越可观测性、追踪和评估，您可以在自托管实例上启用以下功能：

  • - **LangSmith 部署**：通过 LangSmith UI 部署、扩展和管理代理。
  • - **Fleet**：无需编写代码即可创建和管理 AI 代理。
  • - **洞察**：获取对追踪和应用程序数据的 AI 支持分析。
  • - **聊天**：跨 LangSmith 的工作区内聊天体验，帮助您分析追踪、线程、提示和实验结果。

按照 启用 LangSmith 部署、Fleet、洞察和聊天 指南来设置这些组件。
