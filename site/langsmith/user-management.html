此页面涵盖 LangSmith 中的用户管理功能，包括访问控制、身份验证和自动用户配置：

  • 设置访问控制: 配置基于角色的访问控制 (RBAC) 以管理工作区内的用户权限，包括创建自定义角色并将其分配给用户。
  • SAML SSO（企业计划）: 为使用 SAML 2.0 的企业客户设置单点登录身份验证，包括热门身份提供商的配置。
  • SCIM 用户配置（企业计划）: 使用 SCIM 在您的身份提供商和 LangSmith 之间自动配置和取消配置用户。

设置访问控制

提示

RBAC（基于角色的访问控制）仅面向企业客户提供。如果您对此功能感兴趣，请联系我们的销售团队。其他计划默认为所有用户使用 Admin 角色。

检查

在设置访问控制之前，您可能会发现阅读管理概述页面很有帮助。

LangSmith 依靠 RBAC 来管理工作区内的用户权限。这允许您控制谁可以访问您的 LangSmith 工作区以及他们可以在其中做什么。具有 workspaces:manage 权限的用户可以管理工作区设置，具有 workspaces:manage-members 权限的用户可以添加、删除和更新工作区成员。内置的 Workspace Admin 角色包含这两个权限。

有关工作区角色及其权限的完整参考，请参阅基于角色的访问控制指南。有关每个角色可以执行的特定操作，请参阅组织和操作参考。

创建角色

默认情况下，LangSmith 附带一组系统角色：

  • Admin：对工作区内的所有资源拥有完全访问权限。
  • Viewer：对工作区内的所有资源拥有只读访问权限。
  • Editor：拥有除工作区管理（添加/删除用户、更改角色、配置服务密钥）之外的所有权限。

如果这些不符合您的访问模型，Organization Admins 可以创建自定义角色来满足您的需求。

要创建角色，请导航到组织设置页面中 Members and roles 部分的 Roles 选项卡。请注意，您创建的新角色可以在组织内的所有工作区中使用。

单击 Create Role 按钮创建新角色。将打开 Create role 表单。

为要控制访问权限的不同 LangSmith 资源分配权限。

为用户分配角色

设置好角色后，您可以将其分配给用户。要为用户分配角色，请导航到组织设置页面中 Workspaces 部分的 Workspace members 选项卡。

每个用户都有一个 Role 下拉菜单，您可以使用它来分配角色。

您还可以使用给定角色邀请新用户。

为您的组织设置 SAML SSO

单点登录 (SSO) 功能可供企业云客户使用，通过单一身份验证源访问 LangSmith。这允许管理员集中管理团队访问，并使信息更加安全。

LangSmith 的 SSO 配置基于 SAML（安全断言标记语言）2.0 标准构建。SAML 2.0 支持将身份提供商 (IdP) 连接到您的组织，以获得更简单、更安全的登录体验。

SSO 服务允许用户使用一组凭据（例如，姓名或电子邮件地址和密码）访问多个应用程序。该服务仅为用户已获得权限的所有应用程序对最终用户进行一次身份验证，并在用户在同一次会话中切换应用程序时消除进一步提示。SSO 的好处包括：

  • 为组织所有者简化跨系统的用户管理。
  • 使组织能够执行自己的安全策略（例如，MFA）。
  • 消除最终用户记住和管理多个密码的需要。通过允许多个应用程序在单一访问点登录来简化最终用户体验。

即时 (JIT) 配置

LangSmith 在使用 SAML SSO 时支持即时配置。这允许通过 SAML SSO 登录的人员自动作为成员加入组织和所选工作区。有关管理 JIT 配置和用户邀请的详细信息，请参阅在 SSO 组织中管理用户访问。

提示

JIT 配置仅对新用户运行，即通过不同登录方式尚无相同电子邮件地址访问该组织的用户。

登录方式和访问

完成组织的 SAML SSO 配置后，用户将能够通过 SAML SSO 以及其他登录方式（如用户名/密码或 Google 身份验证）登录：

  • 通过 SAML SSO 登录时，用户只能访问配置了 SAML SSO 的相应组织。
  • SAML SSO 作为唯一登录方式的用户没有个人组织。
  • 通过任何其他方式登录时，用户可以访问配置了 SAML SSO 的组织以及他们所属的任何其他组织。

仅强制使用 SAML SSO

提示

不支持在强制仅使用 SAML SSO 的组织中进行用户邀请。初始工作区成员资格和角色由 JIT 配置 决定，之后的更改可以在 UI 中管理。 为了在自动化用户管理方面获得更大的灵活性，LangSmith 支持 SCIM。

要确保用户只能使用 SAML SSO 登录访问组织，请勾选 Login via SSO only 复选框并点击 Save。完成后，通过非 SSO 登录方式访问组织的用户需要使用 SAML SSO 重新登录。可以通过取消选中该复选框并点击 Save 来将此设置切换回允许所有登录方式。

提示

您必须通过 SAML SSO 登录才能将此设置更新为 Only SAML SSO。这是为了确保 SAML 设置有效，并避免将用户锁定在组织之外。

有关故障排除，请参阅 SAML SSO 常见问题。如果在设置 SAML SSO 时遇到问题，请通过 support.langchain.com 联系 LangChain 支持团队。

Supabase 属性映射

提示

Supabase 属性映射是一个仅限云的功能。自托管部署直接在 IdP 中配置 SAML/OIDC 属性——请参阅使用 OAuth2.0 和 OIDC 设置 SSO。

LangSmith 云使用 Supabase 作为 SAML SSO 后端。Supabase 自动将一小组标准 SAML 属性（如 email 和 sub）传递到用户的 JWT。您 IdP 发出的任何其他非标准 SAML 属性（例如，SSO 群组同步的 groups）必须通过 Supabase 显式转发，然后 LangSmith 才能读取。

属性流程 (1:1)：

  1. IdP：发出带有配置名称的 SAML 属性（例如，groups）。
  2. Supabase：仅当属性名称出现在 SSO 提供商的 Supabase Attribute Mapping 表中时，才将属性转发到用户的 JWT。标准属性自动转发；非标准属性除非明确列出，否则会被丢弃。
  3. LangSmith：按名称读取 JWT 声明（例如，SSO 群组同步的 Groups claim field 的值）。

属性名称端到端保留：IdP 属性名称、Supabase Attribute Mapping 条目和下游 LangSmith 设置都使用相同的字符串。

配置

在 Settings → Members and roles → SSO Configuration 中，向下滚动到 Supabase Attribute Mapping 部分，为每个要转发的非标准属性添加一行：

列描述
Attribute nameIdP 发出的 SAML 属性名称。必须与 LangSmith 下游期望的 JWT 声明名称匹配（对于 SSO 群组同步，这匹配 Groups claim field 值）。
Array如果属性是多值的（字符串列表），请勾选此项。对于标量（单值）属性，请保持未选中状态。示例：对于 groups 勾选此项；对于 full_name 保持未选中。

单击 Add row 添加每个其他属性，然后 Save。空的映射表意味着没有非标准属性流向 JWT。

为您的组织设置 SCIM

提示

在寻找比 SCIM 更轻量级的替代方案，不需要 IdP 管理员参与推送群组？请参阅下面的SSO 群组同步——它在登录时直接从 SSO 令牌读取群组成员资格，并重用相同的命名约定。

跨域身份管理系统 (SCIM) 是一个开放标准，允许自动化用户配置。使用 SCIM，您可以自动在 LangSmith 组织和工作区中配置和取消配置用户，使用户访问与您组织的身份提供商保持同步。

提示

SCIM 适用于企业计划上的组织。联系销售了解更多。

SCIM 适用于 Helm chart 版本 0.10.41（应用程序版本 0.10.108）及更高版本。

SCIM 支持仅限 API（请参阅下面的说明）。

SCIM 消除了手动用户管理的需要，并确保用户访问始终与您组织的身份系统保持同步。这允许：

  • 自动化用户管理：根据用户在 IdP 中的状态，自动在 LangSmith 中添加、更新和删除用户。
  • 减少管理开销：无需在多个系统上手动管理用户访问。
  • 提高安全性：离开组织的用户会自动从 LangSmith 中取消配置。
  • 一致的访问控制：用户属性和群组成员资格在系统之间同步。
  • 扩展团队访问控制：高效管理具有多个工作区和自定义角色的大型团队。

NameID 格式

LangSmith 使用 SAML NameID 来识别用户。NameID 是 SAML 响应中的必需字段，且不区分大小写。

NameID 必须：

  1. 每个用户唯一。
  2. 是一个永不更改的持久值，例如随机生成的唯一用户 ID。
  3. 每次登录尝试时完全匹配。它不应依赖用户输入。

NameID 不应该是邮箱地址或用户名，因为邮箱地址和用户名更可能随时间更改，而且可能区分大小写。

NameID 格式必须是 Persistent，除非你使用的字段（如邮箱）需要不同的格式。

提示

SSO 组同步是 SCIM 的更简单替代方案，适用于无法或不想配置 SCIM 组推送的组织。它不是按单独的同步间隔将组从您的 IdP 推送到 LangSmith，而是在登录时直接从 SSO 令牌中的可配置声明读取组成员资格，并使用与 SCIM 相同的命名约定应用组织级别和工作区级别的角色分配。

Production 中的工作区管理员LS:Organization User:Production:Admin
Engineering 中的工作区编辑者LS:Organization User:Engineering:Editor
Marketing 中的工作区查看者LS:Organization User:Marketing:Viewer
Production 中的自定义角色 AnnotatorsLS:Organization User:Production:Annotators

行为

  • 命名约定：组名称遵循与 SCIM 相同的格式（例如，LS:Organization Admins 用于组织管理员，LS:Organization User:Production:Editor 用于工作区范围）。有关完整格式，请参阅组命名约定。分隔符按组织通过 scim_group_name_separator 配置，与 SCIM 共享。
  • 格式错误的组名称：不符合约定的组名称会被静默跳过（记录）并且不会阻止有效组的登录。
  • 登录门控：当启用Require matching group to sign in且 SSO 令牌包含零个匹配组时，登录被阻止。
  • 优先级：SSO 组同步不会修改 SCIM 来源、手动分配或 JIT 配置的成员资格。它完全掌控自己的分配，并根据令牌中的组成员资格在每次登录时替换它们。
  • 组织管理员传播：如果用户从其组获得组织管理员角色，他们将在所有工作区被授予工作区管理员（与 SCIM 行为相同）。

注意事项

  • 取消配置延迟：与 SCIM（主动推送）不同，SSO 组同步仅在登录时更新。从 IdP 中移除的组中的用户在下次 LangSmith 登录前会保留其现有工作区访问权限。Require matching group to sign in 门控通过在下一次登录时完全阻止用户来缓解此问题。
  • 无回溯同步：更改角色映射或启用该功能不会更新现有用户，直至他们再次登录。
  • 需要命名约定：客户必须按照 SCIM 约定命名其 IdP 组。如果您的 IdP 组遵循不同的命名策略，基于 description 映射的 SCIM（请参阅组属性）可能是更好的选择。
