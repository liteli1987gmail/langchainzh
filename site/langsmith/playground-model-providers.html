Amazon Bedrock

在使用此模型之前，请确保您拥有 AWS 凭证或 IAM 角色。

身份验证

Amazon Bedrock 支持两种身份验证方法。**推荐使用 IAM 信任实体**，因为这样可以避免与 LangSmith 共享长期 AWS 访问密钥。

IAM 信任实体（推荐）

提示

**不适用于自托管 LangSmith。**请改用访问密钥（或 Bedrock API 密钥）。

使用 IAM 信任实体身份验证时，您在 AWS 账户中创建一个 IAM 角色，并允许 LangSmith 担任该角色。LangSmith 中不存储任何访问密钥。相反，LangSmith 使用 AWS STS 在每次请求时担任该角色。

设置步骤：

  1. 1. 在您的 AWS 账户中创建一个 IAM 角色，并授予调用 Bedrock 模型的权限（例如 bedrock:InvokeModel）。
  2. 2. 添加信任策略，允许 LangSmith 的 AWS 账户（808407022534）担任该角色，并使用您的 LangSmith 工作区 ID 作为外部 ID：
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::808407022534:root"
      },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "sts:ExternalId": "<your-langsmith-workspace-id>"
        }
      }
    }
  ]
}
建议

你可以在 LangSmith 工作区设置 中找到你的工作区 ID。

  1. 3. 在 LangSmith Playground 中，点击 **Key** 图标打开 Bedrock provider 的密钥配置（IAM Trusted Entity 选项在模型配置下拉菜单中不可用）。然后展开 **IAM Trusted Entity** 部分，输入你创建的角色的 ARN。

!Bedrock 密钥和 API 密钥配置，IAM Trusted Entity 部分已展开

有关信任策略的更多详细信息，请参阅 AWS 文档。

访问密钥

或者，你可以使用 AWS 访问密钥（AWS_ACCESS_KEY_ID 和 AWS_SECRET_ACCESS_KEY）进行身份验证。在 Playground 的 Bedrock provider 配置中输入这些密钥。此方法更简单，但安全性较低，因为它需要存储长期凭证。

可用模型

AWS Bedrock 提供对多个 provider 基础模型的访问：

  • - **Anthropic：** Claude 模型。
  • - **Amazon：** Titan 模型。
  • - **Cohere：** Command 模型。
  • - **Meta：** Llama 模型。
  • - **其他：** 根据地区可用的其他 provider。

有关当前可用模型的列表，请参阅 AWS Bedrock 文档。

配置参数

参数取决于底层模型 provider：

#### Anthropic 模型 使用 Anthropic 配置（请参阅下面的 Anthropic 部分）。

#### Amazon Titan | 参数 | 范围 | 描述 | |-----------|-------|-------------| | **Temperature** | 0.0 - 1.0 | 响应随机性 | | **Max Tokens** | 1+ | 最大响应长度 | | **Top P** | 0.0 - 1.0 | 核采样 |

AWS 特定设置

  • - **Region：** 用于模型部署的 AWS 区域。

工具调用

取决于底层模型： - **Anthropic 模型：** auto, any。 - **Cohere 模型：** auto。

Anthropic

在使用此模型之前，请确保你拥有 Anthropic API 密钥。

可用模型

Anthropic 在其 Claude 系列中提供三个级别的模型：

  • - **Opus：** 最高智能和能力。
  • - **Sonnet：** 平衡的性能和成本。
  • - **Haiku：** 快速且经济实惠。

最新的 Claude 模型支持扩展思考能力，用于展示推理过程。

有关当前可用模型的列表，请参阅 Anthropic 文档。

配置参数

参数范围默认值描述
**Temperature**0.0 - 1.0可选随机性控制（取消勾选以使用模型默认值）
**Max Output Tokens**1+1024最大响应长度
**Top P**0.0 - 1.0可选核采样（取消勾选以使用模型默认值）
**Top K**1+可选限制为 top K 个 token（取消勾选以使用模型默认值）
提示

Temperature、Top P 和 Top K 是可选的。取消勾选后，Claude 使用其内部默认值。

扩展思考

适用于支持的 Claude 模型。启用后，模型可以在响应前展示推理过程，类似于 OpenAI 的 o 系列。

参数范围描述
**Enable Extended Thinking**开关显示/隐藏思考过程
**Budget Tokens**1+用于思考的最大 token 数（默认值：1024）

启用后，响应包括：

  1. 1. 包含模型推理的"思考"部分。
  2. 1. 最终响应。

高级选项

  • - **Base URL：** 覆盖自定义部署的 API 端点。

工具调用

  • - **支持的工具选择：** auto, any（需要至少一个工具）。
  • - **并行执行：** 否（仅顺序）。

Azure OpenAI

在使用此模型之前，请确保你拥有 Azure OpenAI 凭据（端点 + API 密钥）。

可用模型

Azure OpenAI 提供与 OpenAI 相同的模型系列：

  • - **GPT 系列：** 通用聊天模型。
  • - **o 系列：** 专注推理的模型。
  • - **传统模型：** GPT-3.5 和 GPT-4 变体。

模型可用性因 Azure 区域而异，使用前需要部署。

有关当前可用模型的列表，请参阅 Azure OpenAI 文档。

配置参数

Azure OpenAI 支持与 OpenAI 相同的参数：

标准参数

参数范围描述
**Temperature**0.0 - 2.0控制随机性。越低 = 越专注，越高 = 越有创意。
**Max Output Tokens**1+响应的最大长度
**Top P**0.0 - 1.0核采样阈值。温度的替代方案。
**Presence Penalty**-2.0 - 2.0惩罚新主题（正值）或鼓励它们（负值）
**Frequency Penalty**-2.0 - 2.0惩罚重复（正值）或允许重复（负值）
**Seed**整数用于可重复的输出

高级参数

**Reasoning Effort：** 适用于推理优化的模型（o 系列和更新的 GPT 模型）。

**Service Tier：** 适用于更新的模型。

**其他参数：** - **JSON Mode：** 强制有效 JSON 响应。 - **Parallel Tool Calls：** 并发执行多个工具。

Azure 特定功能

  • - **部署管理：** 模型使用前必须先部署。
  • - **区域可用性：** 选择 Azure 区域以满足数据驻留要求。
  • - **内容过滤：** 内置内容审核和安全功能。
  • - **托管标识：** 支持 Azure AD 身份验证。
  • - **私有端点：** VNet 集成以实现安全访问。

工具调用

  • - **支持的工具选择：** auto, required, none 或特定工具名称。
  • - **并行执行：** 是。

OpenAI

在使用此模型之前，请确保你拥有 OpenAI API 密钥 或 Azure OpenAI 凭据。

有关当前可用模型的列表，请参阅 OpenAI 文档。

配置参数

标准参数：

参数范围描述
**Temperature**0.0 - 2.0控制随机性。越低 = 越专注，越高 = 越有创意。
**Max Output Tokens**1+响应的最大长度
**Top P**0.0 - 1.0核采样阈值。温度的替代方案。
**Presence Penalty**-2.0 - 2.0惩罚新主题（正值）或鼓励它们（负值）
**Frequency Penalty**-2.0 - 2.0惩罚重复（正值）或允许重复（负值）
**Seed**整数用于可重复的输出

高级参数：

**Reasoning Effort**：适用于推理优化的模型（o 系列和更新的 GPT 模型）。

控制响应前的推理深度。努力程度越高 = 复杂任务的质量越好，延迟越长。

值描述
none禁用推理（标准聊天行为）
minimal最小推理
low轻度推理
medium中度推理（默认）
high深度推理

当 reasoning_effort 处于活动状态（非 none）时，temperature、top_p 和惩罚参数会自动禁用。

**Service Tier**：适用于更新的模型。

控制请求优先级和处理分配。

值描述
auto系统根据负载决定（默认）
default标准处理队列
flex更低成本，可变延迟（如果模型支持）
priority高优先级队列，更低延迟，更高成本

**其他参数：** - **JSON Mode：** 强制有效 JSON 响应。 - **Responses API：** 改进的流式传输（默认：已启用）。 - **Parallel Tool Calls：** 并发执行多个工具。

OpenAI 兼容端点

身份验证因端点而异。常见选项：

  • - **API 密钥**：存储为工作区密钥，并作为 Authorization: Bearer <key> 转发。
  • - **无**：适用于无身份验证的本地端点（例如 localhost 上的 Ollama）。
  • - **OAuth2 client_credentials**：存储在模型配置中。LangSmith 在请求时生成短期 bearer 令牌，并在到期前刷新。请参阅 OAuth 客户端凭据。

配置

**必需：** - **Base URL：** 你的端点 URL（例如 https://your-endpoint.com/v1）。 - **Model Name：** 你的模型标识符。

适用于任何实现 OpenAI 兼容 API 格式的框架或服务，包括：

  • - 自托管开源推理服务器
  • - 模型路由代理
  • - 自定义模型端点

配置参数

所有 OpenAI 兼容参数：

参数范围描述
**Temperature**0.0 - 2.0响应随机性
**Max Tokens**1+最大响应长度
**Top P**0.0 - 1.0核采样
**Frequency Penalty**-2.0 - 2.0减少重复
**Presence Penalty**-2.0 - 2.0鼓励新主题

**高级：** - **JSON Mode：** 如果端点支持。 - **Streaming：** 如果端点支持。 - **Function Calling：** 如果端点实现 OpenAI 格式。

工具调用

  • - **支持的工具选择：** auto, required, none（如果端点支持）。
  • - **并行执行：** 是（如果端点支持）。

所有提供商的通用配置

额外参数

所有提供商都支持**用于额外参数的 JSON 编辑器**，这些参数未在 UI 中公开：

{
  "logprobs": true,
  "top_logprobs": 5,
  "custom_parameter": "value"
}

**使用场景：** - 特定提供商的 Beta 功能 - UI 中尚未提供的参数 - 用于追踪的自定义元数据

**限制：** 无法覆盖 UI 中已设置的参数（例如，如果温度已在上方设置，则无法在此处设置）

速率限制

**每秒请求数 (RPS)** - 对所有提供商开放，适用于在数据集上运行：

  • - **范围：** 0 - 500 RPS
  • - **目的：** 遵守 API 速率限制，控制成本
  • - **默认值：** 因提供商而异

运行实验或评估时设置此项可避免触及速率限制。
