from langchain_mcp_adapters.client import MultiServerMCPClient
from deepagents import create_deep_agent

async def main():
    async with MultiServerMCPClient(
        {
            "my_server": {
                "transport": "http",
                "url": "http://localhost:8000/mcp",
            }

        agent = create_deep_agent(
            model="openai:gpt-5.5",
            tools=tools,
        )

        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": "Use the MCP server to help me."}]},
            config={"configurable": {"thread_id": "1"}},
        )

asyncio.run(main())

系统提示词


Deep Agents 附带内置系统提示词。Deep agent 的价值来自于 SDK 在模型之上提供的编排层——规划、虚拟文件系统工具和子 agent——模型需要知道这些功能的存在以及何时使用它们。内置提示词会教 agent 如何使用这些脚手架，这样你就不必为每个项目重新推导它；通过 [profile](/oss/python/deepagents/profiles.html#harness-profiles) 或你自己的 `system_prompt=` 来调整它，而不是逐字复制。

当中间件添加特殊工具（如文件系统工具）时，它会将这些工具追加到系统提示词中。

每个 deep agent 还应包含一个针对其特定用例的自定义系统提示词：






### 提示词组装

Deep Agents 从最多四个命名部分构建系统提示词，以便调用者提供的指令、SDK 内置的 agent 指导以及任何特定模型的 [profile](/oss/python/deepagents/profiles.html) 覆盖可以与可预测的优先级共存。如果没有这种分层，针对 Claude 调优的 profile 后缀（例如）可能会被你的 `system_prompt=` 参数覆盖，或者反过来覆盖它，具体取决于调用顺序；命名槽位使排序变得明确且稳定。

在实践中，大多数调用者只会遇到两个槽位：`USER`（你的 `system_prompt=`）和 `BASE`（SDK 默认值）。选择具有内置 profile 的模型——目前是 Anthropic 或 OpenAI——会添加一个 `SUFFIX`。完整的四部分组装主要与你创作自定义 `HarnessProfile` 或调试为何 profile 的文本出现在特定位置相关。

四个命名部分（每个都可能不存在）：

| 名称       | 来源                                                      | 备注                                                               |
| ---------- | --------------------------------------------------------- | ------------------------------------------------------------------ |
| `USER`     | `create_deep_agent` 的 `system_prompt=` 参数              | `str` 或 `SystemMessage`；未设置时省略。                           |
| `BASE`     | SDK 默认值（`BASE_AGENT_PROMPT`）                         | 除非被 profile 的 `CUSTOM` 替换，否则始终存在。                    |
| `CUSTOM`   | [`HarnessProfile.base_system_prompt`](/oss/python/deepagents/profiles.html#harness-profiles) | 当匹配的 profile 设置此项时，完全替换 `BASE`。                    |
| `SUFFIX`   | [`HarnessProfile.system_prompt_suffix`](/oss/python/deepagents/profiles.html#harness-profiles) | 当匹配的 profile 设置此项时，追加到最后。                         |

顺序始终为 **`USER` -> (`BASE` 或 `CUSTOM`) -> `SUFFIX`**，用空行连接（`\n\n`）。由此得出两个不变式：

1. **`USER` 始终在前面。** 调用者的文本优先于任何 SDK 或 profile 内容，因此角色/指令的优先级不受所选模型的影响。
2. **`SUFFIX` 始终在最后。** Profile 后缀最靠近对话历史，模型调优指导在这里最可靠地生效。

组装形状（✓ = 字段已设置，- = 字段未设置）：

| `system_prompt=` | profile `base_system_prompt`（`CUSTOM`） | profile `system_prompt_suffix`（`SUFFIX`） | 最终组装系统提示词 |
| ---------------- | :---------------------------------------: | :------------------------------------------: | ------------------ |
| `None`           |                    -                       |                      -                        | `BASE`             |
| `None`           |                    -                       |                      ✓                        | `BASE` + `SUFFIX`  |
| `None`           |                    ✓                       |                      -                        | `CUSTOM`           |
| `None`           |                    ✓                       |                      ✓                        | `CUSTOM` + `SUFFIX`|
| `str`            |                    -                       |                      -                        | `USER` + `BASE`    |
| `str`            |                    -                       |                      ✓                        | `USER` + `BASE` + `SUFFIX`|
| `str`            |                    ✓                       |                      -                        | `USER` + `CUSTOM`  |
| `str`            |                    ✓                       |                      ✓                        | `USER` + `CUSTOM` + `SUFFIX`|

工作示例——内置 profile（Anthropic、OpenAI）仅提供 `system_prompt_suffix`，因此典型调用位于 `str` + `-` + `✓` 行：




```python
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    system_prompt="You are a customer-support agent for ACME Corp.",
)

Final = USER + BASE + SUFFIX


      = "You are a customer-support agent for ACME Corp."


        + "\n\n"


        + BASE_AGENT_PROMPT


        + "\n\n"


        + 

```


<div class="callout"><strong>提示</strong>


    传递 `SystemMessage`（而非字符串）会触发不同的拼接路径：右侧组装结果（`BASE` 或 `CUSTOM` 加上任意 `SUFFIX`）作为额外的文本内容块追加到消息现有的 `content_blocks` 上。相同的逻辑顺序适用（调用方块优先），调用方块的任何 `cache_control` 标记都会保留——这对于放置明确的 Anthropic prompt-cache 断点很有用。

</div>



    
### 子代理提示词


        [提示词组装](#prompt-assembly) 覆盖规则同样适用于声明式 [subagents](/oss/python/deepagents/subagents.html)：每个子代理根据**其自己的模型**重新运行 profile 解析，然后将已解析 profile 的 `base_system_prompt` / `system_prompt_suffix` 应用到其编写的 `system_prompt`。子代理的 `system_prompt` 扮演 `BASE` 角色；`CUSTOM` 和 `SUFFIX` 来自匹配子代理模型的 profile（可能与主代理的 profile 不同）。

        | `spec["system_prompt"]` | profile `base_system_prompt`（`CUSTOM`） | profile `system_prompt_suffix`（`SUFFIX`） | 最终子代理系统提示词 |
        | ----------------------- | :-------------------------------------: | :---------------------------------------: | ---------------------------- |
        | authored                |                    -                    |                     -                     | authored                     |
        | authored                |                    -                    |                     ✓                     | authored + `SUFFIX`          |
        | authored                |                    ✓                    |                     -                     | `CUSTOM`                     |
        | authored                |                    ✓                    |                     ✓                     | `CUSTOM` + `SUFFIX`          |

        子代理没有 `USER` 段。spec 中编写的 `system_prompt` 是最接近的类比，保留在 `BASE` 槽位。仅提供 `system_prompt_suffix` 的 profile（内置 Anthropic / OpenAI profile 的常见情况）只会追加到子代理作者编写的提示词上。设置 `base_system_prompt` 的 profile 将直接*替换*编写的提示词。
    

    
### 通用子代理提示词


        自动添加的[通用子代理](/oss/python/deepagents/subagents.html#the-general-purpose-subagent)遵循[提示词组装](#prompt-assembly)覆盖规则，并额外增加一层：GP 基础提示词按以下优先级解析：**`general_purpose_subagent.system_prompt`（若已设置）-> `HarnessProfile.base_system_prompt`（若已设置）-> SDK 通用默认值**。无论哪种情况，profile suffix 都会叠加在上面。

        这两个覆盖字段都可以携带基础提示词替换，但它们不可互换。`general_purpose_subagent.system_prompt` 是通用子代理专用配置；`base_system_prompt` 是全局覆盖，主要针对主代理。当两者都设置时，**通用子代理专用配置对通用子代理生效**，因此用户同时调整这两个字段时永远不会看到 GP 覆盖被静默丢弃：




        ```python
        register_harness_profile(
            "anthropic",
            HarnessProfile(
                base_system_prompt="You are ACME's support orchestrator.",  # main agent
                general_purpose_subagent=GeneralPurposeSubagentProfile(
                    system_prompt="You are a research subagent. Cite sources.",  # GP subagent
                ),
                system_prompt_suffix="Always think step by step.",
            ),
        )
        ```

| Stack | Final system prompt |
| ----- | ------------------- |
| Main agent | `"You are ACME's support orchestrator." + SUFFIX` |
| GP subagent | `"You are a research subagent. Cite sources." + SUFFIX` |

如果 `general_purpose_subagent.system_prompt` 未设置，GP 子代理将回退到 `base_system_prompt`（如果已设置），最后回退到 SDK 通用默认值。





中间件


Deep Agents 支持任何[中间件](/oss/python/langchain/middleware/overview.html)，包括下面列出的内置中间件、LangChain 的预构建中间件、提供商特定中间件以及您自己编写的自定义中间件。

将中间件传递给 `create_deep_agent` 的 `middleware` 参数。自定义中间件在[默认堆栈](#default-stack-main-agent)中的 [`PatchToolCallsMiddleware`](https://reference.langchain.com/python/deepagents/middleware/patch_tool_calls/PatchToolCallsMiddleware) 之后追加。




默认情况下，Deep Agents 可以访问以下中间件：

### 默认堆栈（主代理）

从第一个到最后一个：

1. [`TodoListMiddleware`](https://reference.langchain.com/python/langchain/agents/middleware/todo/TodoListMiddleware)：跟踪和管理待办事项列表，用于组织代理任务和工作。
2. [`SkillsMiddleware`](https://reference.langchain.com/python/deepagents/middleware/skills/SkillsMiddleware)：仅在传递 `skills` 时使用。在待办事项中间件**之后**立即注入，并在文件系统中间件**之前**，以便文件工具运行前技能元数据可用。
3. [`FilesystemMiddleware`](https://reference.langchain.com/python/deepagents/middleware/filesystem/FilesystemMiddleware)：处理文件系统操作，如读取、写入和导航目录。当您传递 `permissions` 时，文件系统权限 enforcement 包含在此处，以便它可以评估代理可能调用的每个工具。
4. [`SubAgentMiddleware`](https://reference.langchain.com/python/deepagents/middleware/subagents/SubAgentMiddleware)：生成并协调子代理，用于将任务委托给专业代理。
5. [`SummarizationMiddleware`](https://reference.langchain.com/python/langchain/agents/middleware/summarization/SummarizationMiddleware)：压缩消息历史以保持上下文限制，当对话变长时（通过 [create_summarization_middleware](https://reference.langchain.com/python/deepagents/middleware/summarization/create_summarization_middleware)）。
6. [`PatchToolCallsMiddleware`](https://reference.langchain.com/python/deepagents/middleware/patch_tool_calls/PatchToolCallsMiddleware)：修复消息历史中悬空的工具调用，当运行在中断后恢复或收到格式错误的工具调用参数时。在 Anthropic 提示缓存和下面的尾部堆栈**之前**运行。
7. [`AsyncSubAgentMiddleware`](https://reference.langchain.com/python/deepagents/middleware/async_subagents/AsyncSubAgentMiddleware)：仅在配置异步子代理时使用。
8. **您的中间件参数**：作为 `middleware` 参数传递的可选中间件在此处追加（Patch 之后，尾部堆栈之前）。
9. **Harness profile extras**：来自已解析模型配置文件的提供商特定中间件（如果有）。
10. **排除工具过滤**：当 harness 配置文件列出排除的工具时，中间件从代理中移除这些工具。
11. **提示缓存**（[`AnthropicPromptCachingMiddleware`](https://reference.langchain.com/python/langchain-anthropic/middleware/prompt_caching/AnthropicPromptCachingMiddleware) 和 [`BedrockPromptCachingMiddleware`](https://reference.langchain.com/python/langchain-aws/middleware/prompt_caching/BedrockPromptCachingMiddleware)）：两者始终注册并在 Patch 和您的中间件**之后**运行，以便缓存前缀与实际发送给模型的内容匹配。每个在不支持的模型上无操作（`unsupported_model_behavior="ignore"`），因此 Anthropic 中间件适用于 Anthropic 模型，Bedrock 中间件适用于支持缓存的 AWS Bedrock 模型。
12. [`MemoryMiddleware`](https://reference.langchain.com/python/deepagents/middleware/memory/MemoryMiddleware)：仅在传递 `memory` 时使用。

    
<div class="callout"><strong>提示</strong>


    `MemoryMiddleware` 放置在 profile extras 和提示缓存中间件**之后**，这样对注入内存的更新不太可能使缓存前缀失效。`create_deep_agent` 实现注释中也提到了相同的排序问题。
    
</div>


13. `HumanInTheLoopMiddleware`：仅在传递 `interrupt_on` 时使用。在配置的 tool calls 处暂停以等待人工批准或输入。
