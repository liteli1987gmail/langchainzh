快速开始


Deep Agents Code 会自动集成[以下模型提供商](#provider-reference)：除了安装相关提供商包外，无需额外配置。

1. **安装提供商包**

    每个模型提供商都需要其对应的 LangChain 集成包。这些包作为可选依赖项发布，以保持应用程序轻量化。OpenAI、Anthropic 和 Gemini 默认包含。使用 `/install` 在会话中安装任何其他依赖项，或使用 `dcode --install` 在 shell 中安装：

    
        ```txt In session
        /install groq
        ```

        ```bash Shell
        dcode --install groq
        ```


运行 `/install` 不带参数运行以列出有效的额外组件。若要在初始 CLI 安装期间预装额外组件，请设置 `DEEPAGENTS_CODE_EXTRAS`：

    ```bash
    DEEPAGENTS_CODE_EXTRAS="baseten,groq" curl -LsSf https://langch.in/dcode | bash
    ```

2. **设置凭据**

    使用 [`/auth`](/oss/python/deepagents/code/configuration.html#use-%2Fauth-recommended) 凭据管理器为您的提供商添加 API 密钥：

    ```txt
    /auth
    ```

对于非交互式运行、CI/CD 或没有 TUI 的环境，请使用 [`dcode auth set`](/oss/python/deepagents/code/configuration.html#manage-credentials-from-the-shell-dcode-auth) 从 shell 中存储相同的密钥，或改用提供商的 environment variable 变量。详见 [Provider credentials](/oss/python/deepagents/code/configuration.html#provider-credentials) 获取完整的密钥解析顺序、[`DEEPAGENTS_CODE_` 前缀](/oss/python/deepagents/code/configuration.html#deepagents_code_-prefix) 用于将密钥限定在 Deep Agents Code 范围内，以及 [Provider reference](#provider-reference) 中各提供商的 environment variable 变量。

要配置模型参数，请参阅 [Model parameters](#model-parameters)。

### Sign in with ChatGPT

`openai_codex` 提供商允许您使用付费 **ChatGPT** 订阅而非 `OPENAI_API_KEY` 来使用 OpenAI 的 Codex 模型。您使用 ChatGPT 账户登录后，它会在 `/auth` 和 `/model` 切换器中显示为独立的提供商，与基于 API 密钥的 `openai` 提供商分开。


    
### Start the sign-in


        在任何会话中运行 `/auth` 并选择 **`openai_codex`**。由于 ChatGPT 通过浏览器让您登录，这会启动浏览器登录流程，而非要求输入 API 密钥。
    


    
### Authorize in your browser


        Deep Agents Code 会打开您的浏览器进入 ChatGPT 登录页面。如果无法打开浏览器（例如通过 SSH），它也会在屏幕上显示登录 URL，以便您可以复制到另一台设备的浏览器中。
    


    
### Select a Codex model


        登录后，Codex 模型会出现在 `/model` 切换器的 `openai_codex` 提供商下。您可以通过其 spec 直接切换到其中一个：



        ```txt
        /model openai_codex:gpt-5.5
        ```




您的登录状态会在会话之间保持。要查看状态或退出登录，请运行 `/auth`，选择 `openai_codex`，然后选择重新认证或退出登录。


<div class="callout"><strong>提示</strong>


    `openai_codex` 与 `openai` 是独立的。如果要使用标准 API 密钥调用 OpenAI 模型，请使用常规的 `openai` 提供商（例如 `/model openai:gpt-5.5`）。

</div>

切换模型


要在 Deep Agents Code 中切换模型，请执行以下操作之一：

1. **使用交互式模型切换器** - 使用 `/model` 命令。

    
<div class="callout"><strong>提示</strong>


        并非所有模型都会在此处显示。如果您的模型不在列表中，请直接传递模型名称（例如 `/model gpt-5.5`），或将其添加到 `config.toml` 中。
    
</div>

2. **直接指定模型名称** 作为参数，例如 `/model gpt-5.5`。您可以使用所选提供商支持的任何模型，无论其是否出现在选项 1 的列表中。模型名称将传递给 API 请求。
3. **在启动时指定模型** 通过 `--model`，例如

    ```txt
    dcode --model openai:gpt-5.5
    ```


### 模型解析顺序


    Deep Agents Code 启动时，会按以下顺序确定要使用的模型：

    1. **`--model` 标志**——如果提供，始终优先。
    2. **`[models].default`** 在 `~/.deepagents/config.toml` 中——用户的长期偏好。
    3. **`[models].recent`** 在 `~/.deepagents/config.toml` 中——最近通过 `/model` 切换的模型。自动写入；不会覆盖 `[models].default`。
    4. **环境自动检测**：回退到第一个可用的启动凭证，按顺序检查：`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GOOGLE_API_KEY`、`GOOGLE_CLOUD_PROJECT`（Vertex AI）。

    此启动回退仅检查这四个凭证。其他支持的提供商（例如 Groq）仍可通过 `--model`、`/model` 和保存的默认值（`[models].default` / `[models].recent`）使用。



### 切换器中显示哪些模型

`/model` 选择器根据已安装的提供商包动态构建其列表。展开下方查看完整标准和故障排除方法。


### 切换器如何构建其模型列表


    交互式 `/model` 选择器根据已安装的提供商包和 `config.toml` 中配置的模型构建其列表。

    模型在以下情况下会显示：

    1. 提供商包已安装。
    2. 模型可从提供商包、本地提供商或您的 `config.toml` 获取。
    3. 模型配置未将文本输入或输出标记为不支持。

    如果某个模型缺失，可直接使用 `/model <provider>:<model>` 或将其添加到 [`[models.providers.<name>].models`](/oss/python/deepagents/code/configuration#adding-models-to-the-interactive-switcher)。

    
<div class="callout"><strong>建议</strong>


        凭证状态**不会**影响模型是否被列出。您仍然可以选择凭证缺失的模型。提供商会在请求时报告身份验证错误。
    
</div>

### 设置默认模型

你可以设置一个持久化的默认模型，它将应用于所有未来的 CLI 启动：

* **通过模型选择器：** 打开 `/model`，导航到所需模型，然后按 `Ctrl+S` 将其固定为默认。再次在当前默认模型上按 `Ctrl+S` 可清除它。
* **通过命令：** `/model --default provider:model`（例如 `/model --default anthropic:claude-opus-4-8`）
* **通过配置文件：** 在 `~/.deepagents/config.toml` 中设置 `[models].default`（参见[配置](/oss/python/deepagents/code/configuration.html)）。
* **从 shell：**

    ```bash
    dcode --default-model anthropic:claude-opus-4-8
    ```

查看当前默认值：

```bash
dcode --default-model
```

要清除默认值：

* **从 Shell：**

    ```bash
    dcode --clear-default-model
    ```

* **通过命令：** `/model --default --clear`
* **通过模型选择器：** 在当前固定的默认模型上按 `Ctrl+S`。

如果没有设置默认模型，Deep Agents Code 会使用最近使用的模型。

### 模型参数

向模型传递额外的构造函数 kwargs——采样控制、推理/思考预算、上下文窗口大小、请求超时，以及底层聊天模型类接受的任何其他参数。设置它们的三个位置，按优先级排序（最高优先）：

1. **启动时一次性设置 `--model-params`。** JSON 字符串，仅限当前会话：

    ```bash
    # OpenAI reasoning effort
    dcode --model openai:gpt-5.5 --model-params '{"reasoning": {"effort": "high"}}'

    # Anthropic extended thinking
    dcode --model anthropic:claude-opus-4-8 --model-params '{"thinking": {"type": "enabled", "budget_tokens": 10000}, "max_tokens": 16000}'
    ```

2. **通过 `/model --model-params` 进行会话中切换。** 相同的 JSON 语法——在不需要重启的情况下切换参数（以及可选的模型）：

    ```txt
    /model --model-params '{"temperature": 0.7}' anthropic:claude-opus-4-8
    /model --model-params '{"num_ctx": 16384}'           # opens selector, applies params to choice
    ```

3. **持久化在 `config.toml` 中。** 在每次启动时应用的 Provider 级别默认值（包含可选的 per-model 子表）：

    ```toml
    [models.providers.anthropic.params]
    thinking = { type = "enabled", budget_tokens = 10000 }
    max_tokens = 16000

    [models.providers.openai.params]
    reasoning = { effort = "high", summary = "auto" }
    output_version = "responses/v1"

    [models.providers.ollama.params]
    num_ctx = 16384
    temperature = 0

    # Per-model override—wins over provider-level keys
    [models.providers.ollama.params."qwen3:4b"]
    temperature = 0.5
    ```

CLI 标志会覆盖配置文件中的 `params`，且仅在当前会话有效（会话中途的更改不会持久化）。`config.toml` 中每个模型对应的子表会覆盖 provider 级别的键（浅层合并——详见[模型构造函数参数](/oss/python/deepagents/code/configuration.html#model-constructor-params)了解完整语义）。`--model-params` 不能与 `--default` 组合使用。

对于重试次数，请优先使用 `--max-retries` 或顶层 `[retries]` 配置。


<div class="callout"><strong>建议</strong>


    底层聊天模型构造函数接受的任何 kwarg 都有效。具体支持的参数请参考各 provider 的参考文档——例如 [`ChatAnthropic`](https://reference.langchain.com/python/langchain-anthropic/langchain_anthropic/chat_models/ChatAnthropic)、[`ChatOpenAI`](https://reference.langchain.com/python/langchain-openai/langchain_openai/chat_models/base/ChatOpenAI)、[`ChatOllama`](https://reference.langchain.com/python/langchain-ollama/langchain_ollama/chat_models/ChatOllama)。未识别的 kwargs 会被直接转发到上游 API 请求，因此新发布的参数无需更新 CLI 即可使用。

</div>



<div class="callout"><strong>提示</strong>


    请勿将凭证（`api_key`）放在 `params` 中——请使用 [`api_key_env`](/oss/python/deepagents/code/configuration.html#provider-configuration) 指向环境变量。

</div>


要在模型的运行时 *profile*（`max_input_tokens`、`tool_calling`、功能标志）上覆盖字段（这与构造函数参数不同），请参阅[Profile 覆盖](/oss/python/deepagents/code/configuration.html#profile-overrides-advanced)。
