`PGVectorStore` 是一个使用 postgres 作为后端的 LangChain 向量存储实现。

本笔记本介绍如何使用 `PGVectorStore` API。

代码位于一个名为的集成包中：[langchain-postgres](https://github.com/langchain-ai/langchain-postgres/)。


初始化


### PGEngine 连接池

建立 PostgreSQL 作为向量存储的要求和参数之一是 `PGEngine` 对象。`PGEngine` 配置了一个共享连接池以连接到你的 Postgres 数据库。这是行业最佳实践，用于管理连接数量并通过缓存数据库连接来降低延迟。

`PGVectorStore` 可以与 `asyncpg` 和 `psycopg3` 驱动程序配合使用。

使用 `PGEngine.from_connection_string()` 创建 `PGEngine` 时，你需要提供：

1. `url`：使用 `postgresql+asyncpg` 驱动程序的连接字符串。

**注意：** 本教程演示的是异步接口。所有异步方法都有对应的同步方法。

```python

See docker command above to launch a Postgres instance with pgvector enabled.

CONNECTION_STRING = (
    f"postgresql+asyncpg://{POSTGRES_USER}:{POSTGRES_PASSWORD}@{POSTGRES_HOST}"
    f":{POSTGRES_PORT}/{POSTGRES_DB}"
)

To use psycopg3 driver, set your connection string to `postgresql+psycopg://`

```

```python
from langchain_postgres import PGEngine

pg_engine = PGEngine.from_connection_string(url=CONNECTION_STRING)
```

要使用 `PGEngine.from_engine()` 创建 `PGEngine`，需要提供：

1. `engine`：`AsyncEngine` 对象

```python
from sqlalchemy.ext.asyncio import create_async_engine


Create an SQLAlchemy Async Engine

engine = create_async_engine(
    CONNECTION_STRING,
)

pg_engine = PGEngine.from_engine(engine=engine)
```

### Initialize a table

`PGVectorStore` 类需要一个数据库表。`PGEngine` 引擎提供了一个辅助方法 `ainit_vectorstore_table()`，可用于为您创建具有正确 schema 的表。

有关自定义 schema 的信息，请参阅[创建自定义向量存储](#create-a-custom-vector-store)或[使用现有表创建向量存储](#create-a-vector-store-using-existing-table)。

```python
await pg_engine.ainit_vectorstore_table(
    table_name=TABLE_NAME,
    vector_size=VECTOR_SIZE,
)
```

#### 可选提示：💡

你也可以通过传递 `schema_name` 来指定模式名称，就像传递 `table_name` 一样。例如：

```python
SCHEMA_NAME="my_schema"

await pg_engine.ainit_vectorstore_table(
    table_name=TABLE_NAME,
    vector_size=768,
    schema_name=SCHEMA_NAME,    # Default: "public"
)
```

管理向量存储


### 添加文档

向向量存储添加文档。元数据存储在 JSON 列中，请参阅"创建自定义向量存储"以了解如何存储用于过滤器的元数据。

```python


from langchain_core.documents import Document

docs = [
    Document(
        id=str(uuid.uuid4()),
        page_content="Red Apple",
        metadata={"description": "red", "content": "1", "category": "fruit"},
    ),
    Document(
        id=str(uuid.uuid4()),
        page_content="Banana Cavendish",
        metadata={"description": "yellow", "content": "2", "category": "fruit"},
    ),
    Document(
        id=str(uuid.uuid4()),
        page_content="Orange Navel",
        metadata={"description": "orange", "content": "3", "category": "fruit"},
    ),
]

await store.aadd_documents(docs)
```

### 添加文本

如果文本未格式化为 Document，可直接添加到向量存储中。

```python


all_texts = ["Apples and oranges", "Cars and airplanes", "Pineapple", "Train", "Banana"]
metadatas = [{"len": len(t)} for t in all_texts]
ids = [str(uuid.uuid4()) for _ in all_texts]

await store.aadd_texts(all_texts, metadatas=metadatas, ids=ids)
```

### 删除文档

可以使用 id 删除文档。

```python
await store.adelete([ids[1]])
```

```python
query = "I'd like a fruit."
```

添加索引


通过应用向量索引加速向量搜索查询。了解更多关于[向量索引](https://cloud.google.com/blog/products/databases/faster-similarity-search-performance-with-pgvector-indexes)的信息。

如果未提供名称，索引将使用默认索引名称。要添加多个索引，需要使用不同的索引名称。

```python
from langchain_postgres.v2.indexes import HNSWIndex, IVFFlatIndex

index = IVFFlatIndex()
await store.aapply_vector_index(index)

index = HNSWIndex(name="my-hnsw-index")
await store.aapply_vector_index(index)
```

设置索引参数以调整索引，在召回率和 QPS 之间获得最佳平衡。

```python
index = IVFFlatIndex(name="my-ivfflat", lists=120)
await store.aapply_vector_index(index)
```

### 重建索引

使用存储在索引表中的数据重建索引，替换索引的旧副本。某些索引类型在添加大量新数据后可能需要重新索引。

```python
await store.areindex()  # Re-index using the default index name
await store.areindex("my-hnsw-index")  # Re-index using the index name
```

### 删除索引

删除向量索引。

```python
await store.adrop_vector_index()  # Delete index using the default name
await store.adrop_vector_index("my-hnsw-index")  # Delete index using the index name
```

### 创建自定义向量存储

使用特殊列名或自定义元数据列自定义向量存储。

`ainit_vectorstore_table`

* 使用字段 `content_column`、`embedding_column`、`metadata_columns`、`metadata_json_column`、`id_column` 来重命名列。
* 使用 `Column` 类创建自定义 id 或元数据列。Column 由名称和数据类型定义。可以使用任何 Postgres [数据类型](https://www.postgresql.org/docs/current/datatype.html)。
* 使用 `store_metadata` 创建一个 JSON 列来存储额外的元数据。

#### 可选提示：💡

要使用非 uuid id，必须自定义 id 列：

以下是此表如何映射到 `PGVectorStore`：

* **`id_column="product_id"`**：ID 列唯一标识 products 表中的每一行。

* **`content_column="description"`**：`description` 列包含每个产品的文本描述。这些文本由 `embedding_service` 用来创建进入 embedding_column 的向量，这些向量表示每个描述的语义含义。

* **`embedding_column="embed"`**：`embed` 列存储从产品描述创建的向量。这些向量用于查找具有相似描述的产品。

* **`metadata_columns=["name", "category", "price_usd", "quantity", "sku", "image_url"]`**：这些列被视为每个产品的元数据。元数据提供有关产品的附加信息，例如其名称、类别、价格、可用量、SKU（库存单位）和图片 URL。此信息对于在搜索结果中显示产品详情或用于过滤和分类很有用。

* **`metadata_json_column="metadata"`**：`metadata` 列可以灵活的 JSON 格式存储有关产品的任何附加信息。这允许存储不符合标准列的多样化和复杂的数据。

```python
TABLE_NAME = "products"

custom_store = await PGVectorStore.create(
    engine=pg_engine,
    table_name=TABLE_NAME,
    # schema_name=SCHEMA_NAME,
    embedding_service=embedding,
    # Connect to existing VectorStore by customizing below column names
    id_column="product_id",

```

注意：

1. 可选：如果 `embed` 列是新创建的，或者其维度与嵌入模型支持的维度不同，则需要一次性为旧记录添加嵌入，如下所示：

    `ALTER TABLE products ADD COLUMN embed vector(768) DEFAULT NULL`

1. 对于新记录，通过 [`VectorStore`](https://reference.langchain.com/python/langchain-core/vectorstores/base/VectorStore) 添加的嵌入会自动生成。
