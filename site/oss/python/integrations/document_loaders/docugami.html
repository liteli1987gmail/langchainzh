  2. 2. 获取您工作区的访问令牌，并确保将其设置为 DOCUGAMI_API_KEY 环境变量。
  3. 3. 获取已处理文档的 docset 和文档 ID，如下所述： help.docugami.com/home/docugami-api

快速开始

  1. 1. 创建一个 Docugami 工作区 （提供免费试用）
  2. 2. 添加您的文档（PDF、DOCX 或 DOC），并允许 Docugami 提取并将其聚类为相似文档集，例如 NDA、租赁协议和服务协议。系统支持的文档类型没有固定限制，创建的聚类取决于您的特定文档，您可以 更改 docset 分配 later.
  3. 3. 通过 Developer Playground 为您的工作区创建访问令牌。 详细说明
  4. 4. 探索 Docugami API 以获取已处理的 docset ID 列表，或仅获取特定 docset 的文档 ID。
  5. 6. 如下所述使用 DocugamiLoader，以获取文档的丰富语义块。
  6. 7. 可选地，构建并发布一个或多个 报告或摘要。这有助于 Docugami 根据您的偏好使用更好的标签改进语义 XML，然后将这些标签作为元数据添加到 DocugamiLoader 输出中。使用自我查询检索器等技术进行高精度文档问答。

与其他分块技术的优势

适当的文档分块对文档检索至关重要。存在多种分块技术，包括依赖空白和基于字符长度的递归分块分割的简单方法。Docugami 提供了不同的方法：

  1. 1. **智能分块：** Docugami 将每个文档分解为一个分层语义 XML 树，其中包含不同大小的块，从单个单词或数值到整个部分。这些块遵循文档的语义轮廓，提供比任意长度或简单的基于空白分块更有意义的表示。
  2. 2. **语义注释：** 块使用在整个文档集中一致的语义标签进行注释，便于跨多个文档进行一致的层次查询，即使它们的编写和格式不同。例如，在租赁协议集中，您可以轻松识别关键条款，如房东、租户或续订日期，以及更复杂的信息，如任何转租条款的措辞或特定管辖区是否在终止条款中包含例外部分。
  3. 3. **结构化表示：** In addition, the XML tree indicates the structural contours of every document, using attributes denoting headings, paragraphs, lists, tables, and other common elements, and does that consistently across all supported document formats, such as scanned PDFs or DOCX files. It appropriately handles long-form document characteristics like page headers/footers or multi-column flows for clean text extraction.
  4. 4. **附加元数据：** 如果用户使用过 Docugami，块还会使用附加元数据进行注释。此附加元数据可用于高精度文档问答，不受上下文窗口限制。参见下面的详细代码演练。

该 metadata 每个 Document（实际上，一个真实的 PDF、DOC 或 DOCX 片段）包含一些有用的附加信息：

  1. 1. **ID 和来源：** 片段来源的 Docugami 内的文件（PDF、DOC 或 DOCX）的 ID 和名称。
  2. 2. **xpath:** 文档 XML 表示中该片段的 XPath。可用于直接引用文档 XML 内实际片段的来源引用。
  3. 3. **structure:** 片段的结构属性，例如 h1、h2、div、table、td 等。如果调用方需要，可用于过滤掉某些类型的片段。
  4. 4. **tag:** 使用各种生成和提取技术为片段标注的语义标签。更多详情： github.com/docugami/DFM-benchmarks

您可以通过在 DocugamiLoader instance:

  1. 1. 您可以设置最小和最大片段大小，系统会尽量遵守这些限制，同时最大程度减少截断。您可以设置 loader.min_text_length 和 loader.max_text_length 来控制这些。
  2. 2. 默认情况下，仅返回片段的文本。不过，Docugami 的 XML 知识图谱包含其他丰富信息，包括片段内实体的语义标签。设置 loader.include_xml_tags = True 如果您希望返回的片段包含附加的 xml 元数据。

    stripped_metadata = chunk.metadata.copy()
    for key in chunk.metadata:

            # remove semantic metadata
            del stripped_metadata[key]
    chunk.metadata = stripped_metadata
