将 HTML 文档分割成易于管理的块对于各种文本处理任务（如自然语言处理、搜索索引等）至关重要。在本指南中，我们将探索 LangChain 提供的三种不同的文本分割器，您可以使用它们来有效地分割 HTML 内容：

- [**HTMLHeaderTextSplitter**](#using-htmlheadertextsplitter)
- [**HTMLSectionSplitter**](#using-htmlsectionsplitter)
- [**HTMLSemanticPreservingSplitter**](#using-htmlsemanticpreservingsplitter)

这些分割器各自具有独特的功能和使用场景。本指南将帮助您了解它们之间的差异、为什么您可能选择其中一个而不是另一个，以及如何有效地使用它们。

分割器概述


### [HTMLHeaderTextSplitter](#using-htmlheadertextsplitter)


<div class="callout"><strong>信息</strong>


**当您想根据标题保留文档的层级结构时，此功能非常有用。**

</div>


**描述**：基于标题标签（如 `<h1>`、`<h2>`、`<h3>` 等）对 HTML 文本进行分割，并为每个与给定块相关的标题添加元数据。

**功能**：
- 在 HTML 元素级别分割文本。
- 保留文档结构中编码的上下文丰富信息。
- 可以按元素返回块，也可以合并具有相同元数据的元素。

___

### [HTMLSectionSplitter](#using-htmlsectionsplitter)


<div class="callout"><strong>信息</strong>


当您想将 HTML 文档分割成更大的区块（如 `<section>`、`<div>` 或自定义区块）时，此功能非常有用。

</div>


**描述**：类似于 HTMLHeaderTextSplitter，但专注于基于指定标签将 HTML 分割成区块。

**功能**：
- 使用 XSLT 转换来检测和分割区块。
- 内部使用 `RecursiveCharacterTextSplitter` 处理大区块。
- 考虑字体大小来确定区块。
___

### [HTMLSemanticPreservingSplitter](#using-htmlsemanticpreservingsplitter)


<div class="callout"><strong>信息</strong>


**当您需要确保结构化元素不被分割到不同的块中、保留上下文相关性时，此功能是理想选择。**

</div>


**描述**：将 HTML 内容分割成可管理的块，同时保留表格、列表和其他 HTML 组件等重要元素的语义结构。

**功能**：
- 保留表格、列表和其他指定的 HTML 元素。
- 允许为特定 HTML 标签添加自定义处理器。
- 确保文档的语义含义得以保持。
- 内置标准化和停用词移除功能

___

### 选择合适的分割器

- **当出现以下情况时使用 `HTMLHeaderTextSplitter`**：您需要基于标题层级分割 HTML 文档并维护有关标题的元数据。
- **当出现以下情况时使用 `HTMLSectionSplitter`**：您需要将文档分割成更大、更通用的区块，可能基于自定义标签或字体大小。
- **当出现以下情况时使用 `HTMLSemanticPreservingSplitter`**：您需要在分割文档时保留表格和列表等语义元素，确保它们不被分割且上下文得以保持。

| 功能 | `HTMLHeaderTextSplitter` | `HTMLSectionSplitter` | `HTMLSemanticPreservingSplitter` |
|--------------------------------------------|------------------------|---------------------|-------------------------------|
| 基于标题分割 | 是 | 是 | 是 |
| 保留语义元素（表格、列表） | 否 | 否 | 是 |
| 为标题添加元数据 | 是 | 是 | 是 |
| HTML 标签自定义处理器 | 否 | 否 | 是 |
| 保留媒体（图片、视频） | 否 | 否 | 是 |
| 考虑字体大小 | 否 | 是 | 否 |
| 使用 XSLT 转换 | 否 | 是 | 否 |

使用 HTMLHeaderTextSplitter


[HTMLHeaderTextSplitter](https://reference.langchain.com/python/langchain-text-splitters/html/HTMLHeaderTextSplitter) 是一种"结构感知"型[文本分割器](/oss/python/integrations/splitters/)，它在 HTML 元素级别分割文本，并为每个与给定块"相关"的标题添加元数据。它可以逐元素返回块，也可以将具有相同元数据的元素组合在一起，目标是 (a) 将相关文本（或多或少地）按语义分组，以及 (b) 保留文档结构中编码的上下文丰富信息。它可以用作分块管道的一部分，与其他文本分割器配合使用。

它类似于用于 Markdown 文件的 [MarkdownHeaderTextSplitter](/oss/python/integrations/splitters/markdown_header_metadata_splitter.html)。

要指定要分割的标题，请在实例化 `HTMLHeaderTextSplitter` 时指定 `headers_to_split_on`，如下所示。


```python
from langchain_text_splitters import HTMLHeaderTextSplitter

headers_to_split_on = [
    ("h1", "Header 1"),
    ("h2", "Header 2"),
    ("h3", "Header 3"),
]

html_splitter = HTMLHeaderTextSplitter(headers_to_split_on)
html_header_splits = html_splitter.split_text(html_string)
html_header_splits
```

使用 HTMLSemanticPreservingSplitter


`HTMLSemanticPreservingSplitter` 旨在将 HTML 内容分割成易于管理的块，同时保留表格、列表和其他 HTML 组件等重要元素的语义结构。这确保了这些元素不会被拆分到不同的块中，从而避免表头、列表标题等上下文相关内容丢失。

这个分割器的核心设计理念是创建上下文相关的块。使用 `HTMLHeaderTextSplitter` 进行常规递归分割可能导致表格、列表和其他结构化元素被从中截断，造成重要上下文丢失并产生质量不佳的块。

`HTMLSemanticPreservingSplitter` 对于分割包含表格和列表等结构化元素的 HTML 内容至关重要，尤其是在需要完整保留这些元素的情况下。此外，它能够为特定的 HTML 标签定义自定义处理器，这使其成为处理复杂 HTML 文档的多功能工具。

**重要提示**：`max_chunk_size` 不是块大小的绝对最大值，最大尺寸的计算是在保留的内容不属于该块时进行的，以确保内容不会被拆分。当我们将保留的数据重新添加到块中时，块大小有可能超过 `max_chunk_size`。这对于保持原始文档的结构至关重要。

3. 我们有意设置了一个较小的块大小，以演示元素的不拆分行为
