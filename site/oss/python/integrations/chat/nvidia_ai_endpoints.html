概述


`langchain-nvidia-ai-endpoints` 包包含 LangChain 与 [NVIDIA AI 基础模型](https://www.nvidia.com/en-us/ai-data-science/foundation-models/) 驱动的聊天模型和嵌入模型的集成，并托管在 [NVIDIA API Catalog](https://build.nvidia.com/) 上。

一个很好的起点是 [Nemotron](https://www.nvidia.com/en-us/ai-data-science/foundation-models/nemotron/)，这是 NVIDIA 为 AI 智能体构建的开源模型系列。Nemotron 模型采用混合 Mamba-Transformer 专家混合架构，在保持领先准确性的同时，吞吐量比同类模型高出 3 倍，上下文窗口最高可达 100 万 token。模型权重、训练数据和实现配方均以 NVIDIA 开放模型许可证公开发布。

NVIDIA AI 基础模型运行在 NIM 微服务上：这些容器镜像通过 [NVIDIA NGC Catalog](https://catalog.ngc.nvidia.com/) 分发，暴露标准 OpenAI 兼容 API，并使用 TensorRT-LLM 优化以实现最大吞吐量。可以通过托管的 [NVIDIA API Catalog](https://build.nvidia.com/) 访问，也可以使用 NVIDIA AI Enterprise 许可证在本地部署。

本页介绍如何通过 `ChatNVIDIA` 使用 LangChain 与 NVIDIA 模型交互，包括 Nemotron 和 API Catalog 中的其他模型。

有关通过此 API 访问嵌入模型的更多信息，请参阅 [`NVIDIAEmbeddings`](/oss/python/integrations/embeddings/nvidia_ai_endpoints.html) 文档。

5. 复制并保存密钥为 `NVIDIA_API_KEY`。
6. 要验证您的密钥，请使用以下代码。

```python



if os.environ.get("NVIDIA_API_KEY", "").startswith("nvapi-"):

else:
    nvapi_key = getpass.getpass("NVAPI Key (starts with nvapi-): ")
    assert nvapi_key.startswith(
        "nvapi-"
    ), f"{nvapi_key[:5]}... is not a valid key"
    os.environ["NVIDIA_API_KEY"] = nvapi_key
```

启用思考模式


一些 NVIDIA 推理模型支持可配置的思考模式。通过向 `invoke` 传递 `thinking_mode=True` 来为单个请求启用思考：

```python
from langchain_nvidia_ai_endpoints import ChatNVIDIA

model = ChatNVIDIA(model="nvidia/nemotron-3-nano-30b-a3b")

response = model.invoke("Solve this step by step: 17 * 23", thinking_mode=True)
print(response.content_blocks)
```

使用 `with_thinking_mode` 当您想要一个启用了思维功能且可重用的可运行对象时：

```python
thinking_model = model.with_thinking_mode(enabled=True)
response = thinking_model.invoke("Solve this step by step: 17 * 23")
```

要列出已知支持思考模式的模型，请调用 `get_available_models` 并检查 `supports_thinking`。这会查询 NVIDIA 模型列表 API，然后过滤返回的模型元数据：

```python
thinking_models = [
    model for model in ChatNVIDIA.get_available_models() if model.supports_thinking
]
```

`ChatNVIDIA` 将 `thinking_mode=True` 转换为您所选模型所需的控制机制。有些模型使用请求参数，例如 `chat_template_kwargs={"enable_thinking": True}`。其他模型则使用基于提示的控制方式，LangChain 会向系统消息追加模型特定的思考前缀，或者在不存在系统消息时创建一个。如果某个模型同时配置了两种机制，则请求参数优先。

在构造 `ChatNVIDIA` 时不要传递 `thinking_mode=True`。未知的构造函数参数会成为原始的 `model_kwargs`，这些参数会被直接合并到 API 请求负载中。思考模式转换仅在调用 kwargs 或绑定 kwargs 时运行，因此请使用以下形式之一：

```python
response = model.invoke("Solve this step by step: 17 * 23", thinking_mode=True)
```

```python
thinking_model = model.with_thinking_mode(enabled=True)
response = thinking_model.invoke("Solve this step by step: 17 * 23")
```

### Nemotron 模型用于智能体 AI

[Nemotron](https://www.nvidia.com/en-us/ai-data-science/foundation-models/nemotron/) 是 NVIDIA 开源模型系列，专为智能体工作流打造。关键特性：

- **效率**：混合 Mamba-Transformer MoE 架构提供高达 3 倍的吞吐量提升，相比同类稠密模型
- **长上下文**：原生支持高达 1M token 的上下文窗口
- **智能体推理**：专门针对多步骤规划、工具使用和自主软件工程任务进行训练
- **开源**：权重、训练配方和精选数据集以 NVIDIA Open Model License 发布

与 NVIDIA Dynamo 配合使用


[NVIDIA Dynamo](https://developer.nvidia.com/dynamo) 是一个分布式推理服务框架，专为在数据中心规模的多节点环境中部署模型而构建。它通过将推理的各个阶段分解到不同的 GPU 上，智能地将请求路由到适当的 GPU 以避免冗余计算，并通过将数据缓存到经济高效的存储层来扩展 GPU 显存，从而简化并自动化了分布式服务的复杂性。

`ChatNVIDIADynamo` 是 `ChatNVIDIA` 的直接替代品，会自动在每个请求中注入 `nvext.agent_hints`。这些提示会告诉 Dynamo 部署：

- **`osl`**（输出序列长度）——预期生成的 token 数量，以便调度器规划显存分配
- **`iat`**（到达间隔时间）——请求到达的速度，以便路由器预测负载
- **`latency_sensitivity`**——请求的延迟敏感程度，以便交互式调用获得优先路由
- **`priority`**——请求优先级，以便后台工作可以为关键路径请求让路

每个请求都会自动生成唯一的 `prefix_id`，使路由器能够跟踪 KV 缓存亲和性。


<div class="callout"><strong>提示</strong>


本节假设您已有一个运行中的 [NVIDIA Dynamo 部署](https://docs.nvidia.com/dynamo/latest/getting-started/quickstart)。

</div>


### 基本用法

将 `ChatNVIDIA` 替换为 `ChatNVIDIADynamo`，每个请求都会自动包含路由提示。所有标准的 `ChatNVIDIA` 参数均受支持。

```python
from langchain_nvidia_ai_endpoints import ChatNVIDIA, ChatNVIDIADynamo

BASE_URL = "http://localhost:8099/v1"
MODEL = "your-model-name"


Standard ChatNVIDIA — no Dynamo hints

llm_standard = ChatNVIDIA(base_url=BASE_URL, model=MODEL)


ChatNVIDIADynamo — identical interface, automatically injects agent_hints

llm = ChatNVIDIADynamo(base_url=BASE_URL, model=MODEL)

result = llm.invoke("What is KV cache optimization?")
print(result.content)
```

`ChatNVIDIADynamo` 在 `ChatNVIDIA` 支持的参数基础上，额外接受四个参数：

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `osl` | `int` | `512` | 预期输出序列长度（令牌数） |
| `iat` | `int` | `250` | 预期到达间隔时间（毫秒） |
| `latency_sensitivity` | `float` | `1.0` | 更高的延迟敏感度会获得优先路由 |
| `priority` | `int` | `1` | 更低的优先级设置会获得更多调度优先级 |

### 在构造时设置默认值

在创建模型实例时配置 Dynamo 提示。这在模型实例始终承担特定角色时很有用，例如高优先级的交互式助手与低优先级的后台摘要器。

```python

High-priority: short responses, latency-critical

llm_critical = ChatNVIDIADynamo(
    base_url=BASE_URL,
    model=MODEL,
    osl=20,
    priority=0,
    latency_sensitivity=10.0,
)


Low-priority: long responses, latency-tolerant

llm_background = ChatNVIDIADynamo(
    base_url=BASE_URL,
    model=MODEL,
    osl=512,
    priority=10,
    latency_sensitivity=0,
)
```

### 覆盖每次调用

Dynamo 参数也可以在每次调用时被覆盖。这在同一个模型实例处理具有不同特征的请求时非常有用。

```python
result = llm.invoke(
    "Classify this as positive or negative: 'I love this product!'",
    osl=10,
    iat=100,
    latency_sensitivity=1.0,
    priority=10,
)
print(result.content)
```

### 使用 Dynamo hints 进行流式传输

Dynamo hints 包含在初始流式请求中。Dynamo 使用它们在 token 开始流动之前选择最优的工作节点。

```python
stream = llm_critical.stream_events("Give a one-sentence summary of GPU computing.", version="v3")
for token in stream.text:
    print(token, end="", flush=True)
```

### 检查负载内容

要调试，请使用内部 `_get_payload` 方法检查 `ChatNVIDIADynamo` 发送到 NIM 端点的确切负载内容。

```python


payload = llm_critical._get_payload(
    inputs=[{"role": "user", "content": "Hello!"}],
    stop=None,
)
print(json.dumps(payload["nvext"], indent=2))
```

```json

    "prefix_id": "langchain-dynamo-a1b2c3d4e5f6",
    "osl": 20,
    "iat": 250,
    "latency_sensitivity": 1.0,
    "priority": 10
  }
}
```
