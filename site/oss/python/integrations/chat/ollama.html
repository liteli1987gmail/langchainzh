Ollama 允许您在本地运行开源大语言模型 (LLM)，例如 gpt-oss。

Ollama 将模型权重、配置和数据打包成单个包，由 Modelfile 定义。它优化了设置和配置细节，包括 GPU 使用。

有关支持模型和模型变体的完整列表，请参阅 Ollama 模型库。

设置

首先，按照这些说明设置并运行本地 Ollama 实例：

  • * 下载并将 Ollama 安装到可用的支持平台（包括 Windows Subsystem for Linux 即 WSL、macOS 和 Linux）
  • * macOS 用户可以通过 Homebrew 安装 brew install ollama，并通过 brew services start ollama 启动
  • * 通过 ollama pull <model-name> 获取可用的 LLM 模型
  • * 可通过模型库查看可用模型列表
  • * 例如，ollama pull gpt-oss:20b
  • * 这将下载模型的默认标记版本。通常，默认指向最新的最小参数量模型。

> 在 Mac 上，模型将下载到 ~/.ollama/models > > 在 Linux（或 WSL）上，模型将存储在 /usr/share/ollama/.ollama/models

  • * 按如下方式指定感兴趣模型的确切版本 ollama pull gpt-oss:20b（在此示例中查看 Vicuna 模型的各种标签）
  • * 要查看所有已拉取的模型，请使用 ollama list
  • * 要直接从命令行与模型聊天，请使用 ollama run <model-name>
  • * 查看 Ollama 文档获取更多命令。您可以在终端中运行 ollama help 查看可用命令。

要启用模型调用的自动追踪，请设置您的 LangSmith API 密钥：

工具调用

Ollama 工具调用 使用 OpenAI 兼容的 Web 服务器规范，您可以使用默认的 BaseChatModel.bind_tools() 方法，如 LangChain 工具文档 中所述。

请确保选择一个支持工具调用的 ollama 模型。

我们可以将工具调用与针对工具使用进行微调的 LLM（如 gpt-oss）结合使用：

ollama pull gpt-oss:20b

有关创建自定义工具的详细信息，请参阅自定义工具属性。下面，我们演示如何使用 @tool 装饰器在普通 Python 函数上创建工具。

from typing import List

from langchain.messages import AIMessage
from langchain.tools import tool
from langchain_ollama import ChatOllama


@tool
def validate_user(user_id: int, addresses: List[str]) -> bool:
    """Validate user using historical addresses.

    Args:
        user_id (int): the user ID.
        addresses (List[str]): Previous addresses as a list of strings.
    """
    return True


llm = ChatOllama(
    model="gpt-oss:20b",
    validate_model_on_init=True,
    temperature=0,
).bind_tools([validate_user])

result = llm.invoke(
    "Could you validate user 123? They previously lived at "
    "123 Fake St in Boston MA and 234 Pretend Boulevard in "
    "Houston TX."
)

if isinstance(result, AIMessage) and result.tool_calls:

对数概率

ChatOllama 支持通过 logprobs 和 top_logprobs 参数实现令牌级别的对数概率。对数概率表示每个令牌在每个生成步骤中的可能程度。

基本用法

from langchain_ollama import ChatOllama

llm = ChatOllama(model="llama3.1", logprobs=True)

response = llm.invoke("What color is the sky?")
logprobs = response.response_metadata["logprobs"]
for entry in logprobs[:5]:
    print(f"Token: {entry['token']!r:>15}  logprob: {entry['logprob']:.4f}")

每个 token 的 Top-K 替代选项

使用 top_logprobs 返回每个位置上最可能的替代 token：

llm = ChatOllama(model="llama3.1", logprobs=True, top_logprobs=3)

response = llm.invoke("The capital of France is")
logprobs = response.response_metadata["logprobs"]
for entry in logprobs[:3]:
    print(f"Chosen: {entry['token']!r}")
    if entry.get("top_logprobs"):
        for alt in entry["top_logprobs"]:
            print(f"    {alt['token']!r:>12}  logprob: {alt['logprob']:.4f}")

推理模型和自定义消息角色

某些模型（例如 IBM 的 Granite 3.2）支持自定义消息角色，以启用思考过程。

要访问 Granite 3.2 的思考功能，请传递一条角色为 "control"、内容设置为 "thinking" 的消息。由于 "control" 是一个非标准消息角色，我们可以使用 ChatMessage 对象来实现它：

from langchain.messages import HumanMessage
from langchain_core.messages import ChatMessage
from langchain_ollama import ChatOllama

llm = ChatOllama(model="granite3.2:8b")

messages = [
    ChatMessage(role="control", content="thinking"),
    HumanMessage("What is 3^3?"),
]

response = llm.invoke(messages)
print(response.content)
Here is my thought process:
The user is asking for the value of 3 raised to the power of 3, which is a basic exponentiation operation.

Here is my response:

3^3 (read as "3 to the power of 3") equals 27.

This calculation is performed by multiplying 3 by itself three times: 3*3*3 = 27.

请注意，该模型除了输出最终回复外，还会展示其思考过程。
