摘要

当接近令牌限制时自动摘要对话历史，保留最近消息同时压缩较旧上下文。摘要适用于以下场景： - 超出上下文窗口的长对话。 - 具有大量历史记录的多轮对话。 - 需要保留完整对话上下文的应用程序。

提示

摘要是面向文本的上下文压缩。它不会调整大小、下采样或以其他方式压缩图像/音频/视频内容。通过 keep 保留的最近消息仍包含其原始多模态块，而被摘要的较早的多模态消息仅由生成的文本摘要表示。对于图像密集型应用程序，请将媒体存储在文件系统或对象存储中，并通过消息历史传递 URL 或文件引用。

const agent = createAgent({
  model: "gpt-5.5",
  tools: [weatherTool, calculatorTool],
  middleware: [
    summarizationMiddleware({
      model: "gpt-5.4-mini",
      trigger: { tokens: 4000 },
      keep: { messages: 20 },
    }),
  ],
});

建议

用于生成摘要的模型。可以是模型标识符字符串（例如 'openai:gpt-5.4-mini'）或 BaseChatModel 实例。

触发摘要生成的条件。可以是：

  • - 单个条件对象（所有属性必须满足 - AND 逻辑）
  • - 条件对象数组（任意条件满足即可 - OR 逻辑）

每个条件可以包含： - fraction（number）：模型上下文大小的比例（0-1） - tokens（number）：绝对 Token 数量 - messages（number）：消息数量

每个条件必须至少指定一个属性。如果未提供，摘要生成不会自动触发。

摘要生成后保留多少上下文。必须指定以下之一：

  • - fraction（number）：保留的模型上下文大小比例（0-1）
  • - tokens（number）：保留的绝对 Token 数量
  • - messages（number）：保留的最近消息数量

自定义 Token 计数函数。默认为基于字符的计数方式。

摘要生成的自定义提示词模板。如果未指定，将使用内置模板。模板应包含 {messages} 占位符，对话历史将被插入到该位置。

生成摘要时包含的最大 Token 数量。消息将在摘要生成前被修剪至符合此限制。

添加到摘要消息的前缀。如果未提供，将使用默认前缀。

**已弃用：** 请使用 trigger: { tokens: value } 代替。触发摘要生成的 Token 阈值。

**已弃用：** 请使用 keep: { messages: value } 代替。需要保留的最近消息数量。

完整示例

摘要中间件会监控消息的 Token 数量，并在达到阈值时自动对旧消息进行摘要生成。

**触发条件** 控制摘要生成的执行时机： - 单一阈值在该阈值达到时触发 - 包含多个阈值的触发子句仅在所有阈值都达到时触发（AND 逻辑） - 触发条件列表在任意一项满足时触发（OR 逻辑） - 每个阈值可以使用 fraction（模型上下文大小的比例）、tokens（绝对数量）或 messages（消息数量）

**保留条件** 控制保留多少上下文（必须指定其中之一）： - fraction - 保留的模型上下文大小比例 - tokens - 保留的绝对 Token 数量 - messages - 保留的最近消息数量

人在环中

在工具调用执行前暂停代理执行，以等待人工批准、编辑或拒绝。人在环中 适用于以下场景：

  • - 需要人工批准的高风险操作（例如数据库写入、金融交易）。
  • - 必须进行人工监督的合规工作流。

模型调用限制

限制模型调用次数以防止无限循环或过度成本。模型调用限制适用于以下场景：

  • - 防止失控的代理发出过多 API 调用。
  • - 在生产部署中强制执行成本控制。
  • - 在特定调用预算内测试代理行为。
const agent = createAgent({
  model: "gpt-5.5",
  checkpointer: new MemorySaver(), // Required for thread limiting
  tools: [],
  middleware: [
    modelCallLimitMiddleware({
      threadLimit: 10,
      runLimit: 5,
      exitBehavior: "end",
    }),
  ],
});

达到限制时的行为：

  • - 'continue'（默认）- 用错误消息阻止超限的工具调用，允许其他工具和模型继续运行。模型根据错误消息决定何时结束。

  • - 'end' - 立即停止执行，并为超限的工具调用返回 ToolMessage 和 AI 消息。仅在限制单个工具时有效；如果其他工具有待处理的调用则抛出错误。

自定义 PII 类型

您可以通过提供 detector 参数来创建自定义 PII 类型。这允许您检测超出内置类型的特定用例模式。

**创建自定义检测器的三种方式：**

1. **正则表达式模式字符串** - 简单模式匹配 1. **RegExp 对象** - 更灵活地控制正则表达式标志

1. **自定义函数** - 带有验证功能的复杂检测逻辑

// Method 1: Regex pattern string
const agent1 = createAgent({
  model: "gpt-5.5",
  tools: [],
  middleware: [
    piiMiddleware("api_key", {
      detector: "sk-[a-zA-Z0-9]{32}",
      strategy: "block",
    }),
  ],
});

// Method 2: RegExp object
const agent2 = createAgent({
  model: "gpt-5.5",
  tools: [],
  middleware: [
    piiMiddleware("phone_number", {
      detector: /\+?\d{1,3}[\s.-]?\d{3,4}[\s.-]?\d{4}/,
      strategy: "mask",
    }),
  ],
});

// Method 3: Custom detector function
function detectSSN(content: string): PIIMatch[] {
  const matches: PIIMatch[] = [];
  const pattern = /\d{3}-\d{2}-\d{4}/g;
  let match: RegExpExecArray | null;

  while ((match = pattern.exec(content)) !== null) {
    const ssn = match[0];
    // Validate: first 3 digits shouldn't be 000, 666, or 900-999
    const firstThree = parseInt(ssn.substring(0, 3), 10);
    if (firstThree !== 0 && firstThree !== 666 && !(firstThree >= 900 && firstThree <= 999)) {
      matches.push({
        text: ssn,
        start: match.index ?? 0,
        end: (match.index ?? 0) + ssn.length,
      });
    }
  }
  return matches;
}

const agent3 = createAgent({
  model: "gpt-5.5",
  tools: [],
  middleware: [
    piiMiddleware("ssn", {
      detector: detectSSN,
      strategy: "hash",
    }),
  ],
});

建议

对于自定义检测器：

  • - 使用正则字符串处理简单模式
  • - 需要标志时使用 RegExp 对象（例如，不区分大小写的匹配）
  • - 需要超出模式匹配范围的验证逻辑时使用自定义函数
  • - 自定义函数使您可以完全控制检测逻辑，并可实现复杂的验证规则

配置选项

要检测的 PII 类型。可以是内置类型（email、credit_card、ip、mac_address、url）或自定义类型名称。

如何处理检测到的 PII。选项：

  • - 'block' - 检测到时抛出错误
  • - 'redact' - 替换为 [REDACTED_TYPE]
  • - 'mask' - 部分掩码（例如 ****-****-****-1234）
  • - 'hash' - 替换为确定性哈希值（例如 <email_hash:a1b2c3d4>）

自定义检测器。可以是：

  • - RegExp - 用于匹配的正则表达式模式
  • - string - 正则表达式模式字符串（例如 "sk-[a-zA-Z0-9]{32}"）
  • - function - 自定义检测器函数 (content: string) => PIIMatch[]

如果未提供，则使用 PII 类型的内置检测器。

使用 LLM 在调用主模型之前智能选择相关工具。LLM 工具选择器适用于以下场景：

  • - 工具数量众多（10+）且大多数每次查询都不相关的智能体。
